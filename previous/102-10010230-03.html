Cancellation, Suspension, Rejection Reason Codes (CSR)

This table describes the coding required on the Benefit Action (BA) screen when a decision is made to cancel, suspend, or reject a PES payment.

Note: Service Officers must document the relevant section of the legislation that applies to the Suspension/Cancellation/Rejection code. For further information, see Legislation.

Legend

  • Reason Code = 3 letter code used on the ISS system to reject, suspend or cancel
  • Description = expanded meaning of the code
  • Action
  • REJ = rejection
  • SUS = suspension
  • CAN = cancellation
  • Initiation
  • Man = reasons that can be manually input on the Benefit Action (BA) screen
  • Auto = automatically occurring CSRs (cannot be keyed manually)

Note:

  • on the BA screen use the '?' in the cancellation reason field for an online list of codes
  • PES payments can only be restored by the PES Smart Centre Processing Service or by staff with the appropriate PES skill tag in Service Delivery. A PES Restoration required Fast Note should be sent to the PES Smart Centre Processing team. Select Auto Text > Students, ABSTUDY & Apprentices > Update > PES Restoration required > Confirm
  • AIC PES payments can only be restored by AIC Smart Centre Processing teams. An AIC PES Restoration required Fast Note should be sent to the AIC Smart Centre Processing team. Select Auto Text > Students, ABSTUDY & Apprentices > Update > AIC PES Restoration required > Confirm
