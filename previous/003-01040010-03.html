All online claims for AGDRP/NZ DRP and DRA/NZ DRA meet the minimum identity requirements. No further identity documents need to be provided or coded in the Document Verification Service (DVS).

For phone claims and claim forms, where identity document/s do not verify in the DVS, staff must not follow the process to record alternative identity in Process Direct.

If a customer does not meet eligibility criteria for payment, identity documents do not need to be checked or requested.

For how to confirm customers meet identity requirements, select the relevant tab.

Cancel DRA or NZ DRA as customer did not complete the minimum identity requirements + Read more ...

In the customer’s record in Customer First, cancel the DRA or NZ DRA:

  • Change the System drop down menu to ISS
  • In the Next field enter 'SVAAA'
  • Go to the Benefit Action (BA) screen:
  • For Service Reason select either:
    • DRA for DRA, or
    • EXG for NZ DRA

  • Select cancellation code OTH
  • Note: when selecting ‘OTH’, notes are needed on the BA screen within the cancellation activity, explain the reason - ‘minimum identity requirements not met’. For DRA only - a debt activity may be created if the cancellation date is in the past. Send the debt shell activity to region code XWA with keyword DRADET
  • Finalise the manual review
  • Go to Step 7

  • Create a minimum standards ACTDOC that includes debt information using the Callback/ACTDOC guided procedure

  • Select the ACTDOC activity

Cancel DRA or NZ DRA as customer did not complete the minimum identity requirements + Read more ...

In the customer’s record in Customer First, cancel the DRA or NZ DRA:

  • In the Next field enter 'SVAAA'

  • For Service Reason select either:

  • Cancel the DRA or NZ DRA from date of grant +1

For DRA only - a debt activity may be created if the cancellation date is in the past. Send the debt shell activity to region code XWA with keyword DRADET

10

Create minimum standards ACTDOC + Read more ...

  • Create a minimum standards ACTDOC that includes debt information using the Callback/ACTDOC guided procedure
    • To find the Callback/ACTDOC guided procedure, key Callback/ACTDOC in the free text field under Search Workspace
  • Select the Service Reason
    • AUS Gov Disaster Recovery for AGDRP
    • Disaster Recovery Allowance for DRA
    • Emergency Exgratia for NZ DRP and NZ DRA
  • Select Type ACTDOC
  • Under Category select Debt Raising
  • Select Generate Document Content
  • In Document Content, add:
    • Name of the Disaster Event that has been cancelled Raise a debt for: $(enter amount)
    • Customer was overpaid as they did not complete the minimum identity requirements as discussed on (enter date)
  • Select Finish at the top left corner of the screen
  • Key AL in the Next field to go to the Activity List (AL) screen
  • Select the ACTDOC activity
  • Select ‘O’ Transfer the activity to another site or region
  • Select Continue
  • In Regional Office key: XWA
  • In New Keywords
    • AGDRP and NZ DRP add keyword: DRPDET
    • DRA and NZ DRA add keyword: DRADET
  • Select Continue
  • Select OK to Are you sure you want to update the activity?
  • Return to the AL screen to make sure the correct code shows under Region

Staff do not need to take any further action to have the ACTDOC actioned.

Procedure ends here
