Create a Customer Reference Number (CRN) for an Organisation in Customer First

Table 1

Step

Action

1

Receipt of Registration and change of details form (TAS020) + Read more ...

A completed TAS020 is needed to create a new claimant.

To generate a new claimant record in myHSI, create an organisation CRN in Customer First.

Upon receipt of the TAS020, the Service Officer checks the TAS020 to ensure it is authorised by an acceptable signatory.

Has an acceptable signatory signed the TAS020?

  • Yes, go to Step 2
  • No, ask the business for a new TAS020 signed by an acceptable signatory

2

Search for an organisation + Read more ...

To avoid creating a duplicate record, check the organisation is not a claimant on myHSI.

To search for an organisation:

  • go to myHSI>Tasmanian Freight Equalisation Scheme (TFES) General
  • select Search for Claimants
  • select either Name or Australian Business Number (ABN), from the dropdown menu
  • key in name or ABN, select Search
  • check results against the claimant details

Note: if search results in a record with 'Migrated' status, see Step 5 in Table 2.

3

Pre-existing CRN? + Read more ...

Does the organisation have a pre-existing CRN?

  • Yes, enter the CRN into section 5 of the TAS020. The process in Customer First ends here, see Table 2
  • No, go to Step 4

4

Set up a CRN + Read more ...

Open Customer First:

  • key 'PED' into Next field, change Env to 'B', press [Enter]
  • key 'ON' into Next field, press [Enter]
  • key the Organisation name using the information on the TAS020
  • at the bottom of the page key the Activity Details:
    • Activity: key 'Add'
    • Source: key 'LET'
    • DOR: key 'T'
    • Service Reason: key 'PRY'
    • select Continue
    • select Add Organisation

On the next page:

  • key the Postal Address section under non-Australian address heading (physical address is optional)
  • select Continue until CRN appears in CRN/BP box at the top of page under the Organisational name and address
  • copy CRN onto TAS020
  • continue to select Continue, until the screen states Completed
  • select X to close Customer First

Create a new claimant record

Table 2

Step

Action

1

Create a new claimant in myHSI + Read more ...

Once the Organisation CRN has been created in Customer First:

  • go to myHSI TFES Claimant
  • select Add new claimant from the left hand menu
  • key the CRN and select Next
  • key details from TAS020 using upper case
  • if claimant is an individual rather than a business:
    • key the individual's surname first for the purpose of the Claimant Code
    • key the surname in section 6 of the TAS020. John Citizen would be keyed as CITIZEN so that the system generates the claimant code CIT001
  • key 'New Claimant Added' in Claimant Comments

Once the Claimant Code is created, action the update of the name, change CITIZEN to JOHN CITIZEN:

  • In View Claimant Details screen, select Update
  • In Update Claimant Details screen, update the claimant's name to the correct format
  • Change Special Circumstances to Pending Eligibility Checks using the dropdown menu
  • Record the updated action in Claimant Comments, for example, Claimant name and Special Circumstances updated
  • Upload the TAS020 to Claimant Documents Tab

2

Add the registration activities to the TFES tool + Read more ...

Create new activity

Select Create New in TFES Tool:

  • key the:
    • Date received
    • Date uploaded, and
    • Claimant Code
  • in Claim ID field key Register Claimant

Allocate new activity

To allocate an activity in the TFES Tool:

  • selecting Allocate
  • select the activity from the unassigned records
  • select service officer from dropdown menu, select Save Allocations
  • open work allocation, update Work Type to Register Claimant with the applicable claimant type as indicated in section 13 of TAS020
  • update the comments section, for example, Register new Claimant include the name of the Service Officer the eligibility will be allocated to and if there are any claims lodged with the TAS020

Change of status

If the claimant record goes to 'Pending Approval' in myHSI, change the status in the TFES tool to Awaiting QA officer.

If the claimant record goes to 'eligible' in myHSI, change the status in the TFES tool to Completed.

3

Add the eligibility activity to the TFES tool + Read more ...

Create an activity for the eligibility check in TFES tool:

  • Select Create New
  • Key the:
    • Date received
    • Date uploaded, and
    • Claimant Code
  • in Claim ID field key Eligibility Check
  • allocate to the next available Service Officer using the Eligibility Allocations document located G:\Transport_Resources\REGISTERS\ELIGIBILITY ALLOCATIONS

4

Add the online registration to the TFES tool + Read more ...

Create an activity for online registration:

  • Select Create New
  • key the:
    • Date received
    • Date uploaded, and
    • Claimant Code
  • in Claim ID field key Online Registration

Allocate new activity

To allocate an activity in TFES Tool:

  • select Allocate
  • select the activity from the unassigned records
  • select service officer from dropdown menu, select Save Allocations
  • open work allocation, update Work Type to Online Registration and Add Users
  • allocate activity and change status to Awaiting internal advice

After eligibility is finalised, the QA officer changes the status of the online registration activity in the TFES tool to QA back to Assessor. This indicates the online registration can be processed.

5

Troubleshooting + Read more ...

Claimant codes created in an unacceptable format

For example, B& 011 cannot be changed once submitted. A new claim code needs to be created:

  • create a new Organisation CRN
  • create a new myHSI record in the correct format
  • the unacceptable version in myHSI to remain as 'pending approval' or change it to 'ineligible'
  • add a note in the name field of the record in myHSI 'do not use'

Claimant search returns a status of 'Migrated'

This means the record has been inactive since 2009 and needs:

  • an Organisation CRN, See Table 2
  • updating with the information in the TAS020, see Updating claimant information for TFES

Northbound domestic claimants + Read more ...

Contact claimant to confirm eligibility, check:

  • Are the goods shipped eligible and listed on Schedule 1 of the Ministerial Directions?
  • Is the claimant incurring and paying the cost of shipping the goods, and the claimant is not rules out under clause 2.2 in the Ministerial Directions?
  • Are the goods manufactured or produced in Tasmania and for permanent use or sale on mainland Australia?
  • Is the claimant the Tasmanian producer or the mainland buyer of the shipment?
  • Clause 2.2 in the Ministerial Directions to determine if the claimant is eligible or not eligible

Make 2 genuine attempts to contact the claimant, by phone, email or both.

  • If contact is unsuccessful, make a second attempt within 48 hours
  • If possible, leave a message on a message bank, or in an email with your name, a return phone number and positional email address

Allow 5 business days.

If contact with the claimant is unsuccessful, send an email advising if the information is not provided in 21 days, the application will be rejected. Send the email from the TFES positional mailbox to the:

  • contact person listed in section 9 of the TAS020
  • email address provided in section 12 of the TAS020

Has the claimant answered all questions in 21 days?

  • Yes, go to Step 4
  • No, go to Step 6

Eligible for assistance recommendation + Read more ...

Southbound Manufacturers and Miners eligibility (M Scheme)

Table 4

Note: for Manufacturers and Miners to be eligible to claim southbound, they need to be in Division B or C under the Australian and New Zealand Standard Industry Classification (ANZSIC).

Step

Action

1

Review and research the claimant + Read more ...

The Service Officer:

  • Checks the claimant record in myHSI against the TAS020 to ensure the claimant details have been entered correctly
  • Researches and gathers information about the claimant to determine if the business can be classified under Division B or C of Australian and New Zealand Standard Industry Classification (ANZSIC). Conduct an internet search on the company, including:
  • their internet site
  • their public image (how they market themselves to the public)
  • business location
  • business activities
  • ABN search

Australian Business Register (ABR) Check

Check to confirm the:

  • business activity entered by the claimant on the TAS020, and
  • claimant's ANZSIC class code, which is then entered in the Principal Business Activities on the Claimant Details screen in myHSI

Is the ABN on the TAS020 correct as confirmed on ABN lookup?

  • Yes, go to Step 3
  • No, contact the claimant to get the correct details, go to Step 2

2

Contact the claimant + Read more ...

Ask the claimant for the correct ABN.

Use the ABN to conduct searches in ABR Explorer to determine the industry, Division and ANZSIC Class Code.

If the industry classification needs clarifying, check the Australian Bureau of Statistics (ABS) ANZSIC website.

Look under Statistics>Classifications find ANZSIC – Industry Classifications. The website provides the Division and Class Code of the industry classification found in ABR Explorer.

3

Assess business eligibility + Read more ...

If more information is needed to help make a decision on eligibility send the claimant the Manufacturers Miners questionnaire.

Use the following scenarios to help make a recommendation whether a claimant is eligible or ineligible.

Scenario 1

ABR Explorer shows claimant is registered with the Australian Tax Office (ATO) in Division B or C of ANZSIC.

Assessment determines business activities also fall within Division B or C of ANZSIC.

  • Recommendation = Eligible
  • Prepare Manufacturers and Miners north and southbound eligible letter

Scenario 2

ABR Explorer shows claimant is not registered with the ATO in Division B or C of ANZSIC.

Assessment determines business activities fall within Division B or C of ANZSIC.

  • Recommendation = Eligible
  • Prepare Z2232 Southbound manufacturers and miners conditional approval letter
    • The letter advises the claimant to update their ABR within 3 months. If the claimant does not change their ABR within this timeframe they will be deemed ineligible

Scenario 3

ABR Explorer shows claimant is registered with ATO in Division B or C of ANZSIC.

Assessment determines business activities do not fall within Division B or C of ANZSIC.

  • Recommendation = Ineligible
  • Prepare Z2230 Southbound manufacturers and miners ineligible letter

Scenario 4

ABR Explorer shows claimant is not registered with ATO in Division B or C of ANZSIC.

Assessment determines business activities do not fall within Division B or C of ANZSIC.

  • Recommendation = Ineligible
  • Prepare Z2230 Southbound manufacturers and miners ineligible letter

4

Administration + Read more ...

Complete the Registration –Rejection of claimant eligibility template:

  • Include a recommendation to approve or reject the application
  • Upload the template and all research information to the Claimant Document tab
  • Change the status of the activity to awaiting QA Officer in the TFES Assessor tool

Go to myHSI in TFES General and search for the claimant record:

  • scroll to the bottom of the View Claimant Details screen and select update
  • check the Claimant Name field matches the Entity Name on ABR Explorer lookup. If the details do not match:
    • key the Entity Name from the ABR lookup in the Claimant Name field
  • if eligible select Manufacturer or Mining in the Claimant Category dropdown menu
  • select No in Ineligible Southbound field
  • update the Principal Business Activity with ANZSIC classification from ABR Explorer include:
    • the ANZSIC code and name of the classification
    • select update
    • key comments regarding claimants eligibility
  • select Notes tab:
    • select Add Note
    • under Note details
    • Category dropdown menu Claimant Notes
    • in the Subject field Key General Information
    • Note Text key information about the business found during the assessment of eligibility in Step 2
    • select Add

If the recommendation is Eligible Southbound Manufacturer or Miner for Notes tab:

  • Select Add Note
  • In Notes Details:
    • Category dropdown menu, select M Scheme
    • Subject field, key A – Z
  • In Note Text key:
    • Claimant: *name
    • Date of Approval: *date eligibility is approved
    • Description: *ANZSIC division, business activity and ANZSIC code

If Scenario 2 applies – Assessed as eligible, but ANZSIC classification is to be updated within 3 months.

  • add a note in Claimant Comments stating 'update the ABR include the 3 month end date
  • create a Review Activity in the claimant's record:
    • select Create New Activity on the Activity List
    • select Claimant Review against Activity Type
    • select Audit (Internal) Review against Claimant Review Type
    • key claimant's CRN
    • key the date as the end date of the 3 month period
    • Prepare letter Z2232 Southbound manufacturers and miners conditional approval letter

If Scenario 3 or 4 applies – Assessed as ineligible

  • leave Claimant Category as Unspecified
  • select Yes in Ineligible Southbound field and key date of decision
  • leave Business Activities as 0000
  • add a note in the:
    • Claimant Comments stating the 'claimant is not eligible', include the reasons
    • Claimant Notes titled 'Ineligible claimant' key details of claimant's business activities and results of assessment

5

Finalise + Read more ...

The QA Officer locates the eligibility activity in the TFES tool and performs the quality checks. They email the outcome to the Service Officer stating the registration/rejection is approved or needs follow up.

When finalised, the Service Officer sends the letter prepared in Step 2 to claimant to tell them the outcome.
