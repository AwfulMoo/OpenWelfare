1

Sort postal correspondence + Read more ...

When postal correspondence is received in a Digitisation Solutions site with scanning functionality:

  • on the front of the document, clearly date stamp the correspondence with the date received
  • prepare for scanning by sorting correspondence in document sizes:
    • standard A4 size documents, and
    • non A4 sizes (including mixed sizes)
  • where possible sort into:
    • non-customer/administrative correspondence, for example, stationery invoices, letters to H. If clearly identified as non-customer mail do not open the envelope. Forward the correspondence to the intended recipient's site
    • international referral questionnaires
    • court related documents - substituted service, court orders and
    • cheques - domestic and internal
      Note: do not scan cheques
  • remove any staples and binding so documents can go through the multi-function device (MFD) feeder, and
  • add a batch cover sheet to the top of the batch of customer records

For standard A4 size documents, arrange correspondence into batches of 10 customer records. Insert a barcode separator page between each customer record. The separator page assists in keeping each customer record independent of each other. Very thick documents, non-standard and mixed size documents must have their own batch cover sheet and be scanned individually.

If a customer has sent in non-standard size receipts or documentation:

  • make a copy using the photo copier; and substitute the copy in batch, and
  • attach the original small receipts/document to the batch for filing

Note: batches should not exceed 300 pages. For very large documents, scan as a separate batch or add to a batch with fewer pages.

Other correspondence + Read more ...

Multiple customer documents + Read more ...

Employer withholding (EW)

Payment reconciliation documents received in a single envelope from an employer for a single organisation record must be filed together and separated from other employer’s correspondence by barcode separator pages.

Where several employers' documents are contained in the same envelope, these can be split by separator pages.

These documents will be attached to the employer record in Cuba.

Third party responses

Even if a third party has sent in several documents in a single envelope, all third party responses (including EF1s and information from banks) must be individually filed and each separated by a barcode separator page. Each document will be attached to the relevant customer’s record.

Multiple customer names on 1 document

Correspondence with multiple customer names and/or information must be separated.

The National Digitisation Solutions Support Team will:

  • photocopy the original
  • batch file the unedited copy
  • manually black out the other customers' information on each copy to produce a separate copy for individual customers, and
  • prepare in batches with separator and cover sheet as described above

1

Image correspondence (postal or hand delivered) + Read more ...

Paper documents must be scanned within 48 hours of arrival in an office. Image documents using the Fuji Xerox AP6000 multi-function device (MFD).

Clean MFD

Clean the MFD scanner areas daily:

  • follow the instructions on the cover of the MFD
  • dust off any paper 'crumbs' from guillotined, spliced and perforated documents that may build up in the lid of the MFD. These may impact the image quality. For example, the inside upper left hand strip of the MFD
  • clear away any documents left behind from other people photocopying etc, to ensure they do not accidentally get scanned
  • check that there are no error messages on either machine

2

Check documents before scanning + Read more ...

Before scanning commences:

  • sort and match like size documents into the following batches:
    • standard A4 paged documents
    • any other mixed correspondence/forms
  • remove staples/splice forms

If the documents are being scanned on the date of receipt:

  • ensure the front page is face up, and
  • place a barcode separator sheet between each document

If the documents are being scanned on a day after the date of receipt:

  • collate documents by the date they were received
  • place a barcode separator sheet between each document
  • place an 'older date' barcode sheet at the beginning of the batch. See folder within DST team or seek TL support
  • scan the cover sheet as the first page because it informs the software what date to apply to the documents in that batch

3

Prepare to commence scanning documents + Read more ...

Before scanning commences:

  • complete a batch file cover page, which is stored in the tray near the MFD, with the following noted on it:
    • date and time
    • machine number (either 10.112.182.36 or 10.112.182.37)
  • activate MFD - if the Power Save Mode button on the MFD is green, press to turn the MFD on

Note: if scanning a relatively small document, be ready to record the document number on the batch file cover sheet. This number will be displayed on the top left hand side of the screen when scanning commences.

4

To image standard size bulk documents: + Read more ...

  • place bulk documents on the MFD feeder for scanning, ensuring they are:
    • face up
    • lined up
    • in batches of 10 documents or single documents
    • long side against MFD feeder
  • the MFD screen should light up with All Services menu
  • press the silver All Services button (on MFD) if a different screen is showing
  • select Stored Programming (touch screen button)
    • select the relevant program (touch screen button)
    • use Program 1 Today's Date for documents being scanned on the same day they were received, or
    • use Program 11 Older Date for documents being scanned after the date they were received. It takes a few seconds to populate the pre-programmed settings
  • press Start button (green button on MFD), and
  • note the batch filing details on the batch cover sheet:
    • date
    • time scanning started (hour and minute)
    • machine number
    • Service Officer name
    • document number (batch number) displayed on the top of the screen in the blue area whilst the documents are scanning

If unable to record the document number, go to Step 7.

5

To scan mixed standard size documents: + Read more ...

  • place documents on MFD feeder for scanning, ensuring they are:
    • face up
    • lined up
    • long side against MFD feeder with the exception of A3 documents which are lined up against the top edge
  • the MFD screen should light up with All Services menu
  • press the silver All Services button if a different screen is showing
  • select Scan to Mailbox
  • press General settings
  • press Black - scanning in black minimises the image size
  • two sided head to head
  • press layout adjustments
  • select mixed sized originals (on)
  • Original Orientation should show upright image
  • press Save
  • check the Resolution is set to 300DP
  • press the Start button (green button on MFD) to begin scanning. The pages that display during scanning should match the pages of the document
  • note the batch filing details on the batch cover sheet:
    • the time started (hour and minute) [time from the MFD], and
    • the last 4 digits of the document name (displayed on screen in top left corner whilst the documents are scanning)

If unable to record the document number, go to Step 7.

7

Record document number + Read more ...

If unable to record the document number before all the documents were scanned:

  • select the Job Status button (on MFD)
  • select Completed jobs (touch screen button) to display a list of completed jobs
  • record the document number of the last job completed on the batch cover sheet
  • on the right of the display screen, select Document List (touch screen button)
  • leave the separator sheets in between each document
  • batch file the scanned documents together with the batch file cover sheet on top

8

Troubleshooting + Read more ...

Job in Memory light is on

The Job in Memory light should turn off 10 minutes after scanning is finished and switch to a green Online light to indicate the scanned images are being saved to the mailbox via the network.

Documents are torn

If a document is torn:

  • repair the tear using clear sticky tape
  • scan the document face-down on the MFD glass screen, closing the lid before doing so
  • scan the document 1 page at a time to ensure they do not jam the MFD
  • manually increase the resolution to 400DPI. When the screen displays all the Stored Programming settings, press the 300DPI button (touch screen button) and change the setting to 400DPI

Document jams

The MFD screen generally provides step by step instructions to remove jammed paper. As documents are being scanned electronically, paper jams will only occur when a document is fed though the document feeder on the lid of the MFD.

If a document jam occurs:

  • open the top cover, which will also open the side cover on the top left-hand side
  • check for jammed paper
  • lift the lid and check there also for jammed paper
  • remove jammed paper gently to prevent the original document/s from tearing

What must be batch filed and stored + Read more ...

All original hard copy correspondence received must be appropriately stored for future retrieval and use. For further information see Records Management.

With correspondence received via digital fax, the 'original' is the electronic image stored on Cuba. There is no need to print a hard copy of the original to batch file.

2

Batch filing process + Read more ...

Batch filing for scanning sites (Collins Street, Melbourne)

The original document must be batch filed on the same day it was scanned and by batch number. Ensure that the date is correct.

Batch filing for non-scanning sites

The original document must be batch filed at the site in which it was received, on the same day it was received. For further information on batch filing, see Records Management.

3

Store documents + Read more ...

All hard copy originals received must be batch filed on a daily basis in the received site.

Hard copy originals received in scanning sites (Melbourne Central)

Each batch (individually scanned documents or a group of 10 documents scanned together) must have a batch cover sheet.

The batch box labelled with the same date as the batch header folder and stored in a locked facility.

Hard copy originals received in non-scanning sites

Each faxed document must be filed with:

  • the fax cover sheet (as the batch cover sheet)
  • the fax transmission confirmation sheet, and
  • the original correspondence

Batched documents must be stored in chronological order (a manila folder or batch box can be used to separate daily, weekly and monthly batches depending on volumes received). The originals must be stored in a locked facility (room, compactus, or cabinet - depending on the volume of hard copy originals received).

Customer not able to be identified + Read more ...

RACS1

  • Correspondence is for a RACS1 customer

RACS2

  • Correspondence is for a RACS2 customer

RACS3

  • Correspondence is for a RACS3 customer

Multi Customer

  • Correspondence contains information for more than 1 customer

Non-Customer

  • Correspondence relating to child support but not specific to a particular Child Support customer
  • Correspondence that requires a response but there is insufficient information to identify the customer

Pending Delete

  • Image is not suitable to be stored on Cuba (for example, was not intended for Child Support) or:
    • is in relation to a domestic and/or family violence concern received from a non-customer or third party
    • is unsolicited third party correspondence
    • is a FOI request

Unidentified

  • Information contained in the correspondence is not sufficient to identify the customer

Unident Emp (unidentified employer)

  • Information contained in the correspondence is not sufficient to identify the employer

  • Poor quality or incomplete postal correspondence
  • Application for Child Support (101) where no customer record exists and TFN has not been provided by the applicant.

2

Action correspondence category 'Multi Customer' + Read more ...

View the correspondence and associated notepad in the List Correspondence window. If the image contains information about more than on customer and:

  • it was received via post and there is a source document containing multiple customer information
    • retrieve the original document
    • edit a copy of the document (the original must be retained unedited)
    • split or manually black out the other customer's information so that only the information relevant to 1 customer is visible on separate copies of the correspondence
    • stamp the document with the 'Rescanned' stamp, enter the date it was rescanned and initial it
    • scan the document to Rescan
    • index the document
    • change the correspondence category of the original image to Pending Delete or delete the image (appropriate access required)
    • complete Action Reason, and
    • batch file the unedited copy

or

  • the multiple customer image resulted from more than 1 document being scanned as 1 image:
    • retrieve the original documents from batch storage
    • rescan as separate documents to the Rescanned Image mailbox, stamped with 'rescanned' and the date rescanned
    • change the correspondence category of the original image to Pending Delete or delete the image (appropriate access required)
    • complete the Action Reason, and
    • batch file the original document under the batch number in which it was rescanned and place in batch file storage

or

  • the correspondence was electronically received directly into Cuba (fax, email, webmaster forms):
    • open the image in Adobe acrobat pro
    • electronically extract the relevant pages for each customer
    • electronically black out (redact) any details of other parties from each customer's copy
    • attach a copy of the redacted images for each customer to an email with the subject heading 'Image Number': (original file name)'
    • save the email as a draft in Outlook
    • move the email to the Digitisation Solutions mailbox folder DocSplit$
    • forward the email to the Digital Solutions Team
    • confirm the correspondence category of the original image as Multi customer
    • complete the Action Reason, and
    • check the item is not to be reviewed box to allocate the default review date 31/12/9999

3

Action correspondence category 'Non-Customer' + Read more ...

View the correspondence and associated notepad in the List Correspondence window and index the correspondence to a customer record.

If the customer cannot be identified and a response is required:

  • send an email with the subject line 'Correspondence Requiring a Response' and list the GUID numbers to either:
    • New Customers and Mainstream Services,or
    • International Child Support
  • complete the Action Reason, and
  • check the item is not to be reviewed box to allocate the default review date 31/12/9999

If the correspondence is not intended for Child Support or it is not customer related:

  • refer or return the correspondence to the addressee as required
  • change the correspondence category of the original image to Pending Delete or delete the image (appropriate access required)
  • complete the Action Reason
  • if the correspondence is intended for Centrelink, copy the contents to a new email and email to Correspondence intended for Centrelink with the subject line CLK Mail

4

Action correspondence category 'Pending Delete' + Read more ...

View the correspondence and associated notepad in the List Correspondence window and delete the correspondence if appropriate.

If the correspondence relates to a domestic and family violence concern, send the correspondence to the appropriate agency. See Resources for a link to the Australian Government Institute of Family Studies for State/Territory child protection contacts detail

If the correspondence relates to fraud, email it to a Fraud Investigation Team. From the Sections list, select Investigation - state, then the email address is at the end of the list of contacts.

It is possible that the correspondence may be referred to both fraud and a child protection agency.

If the correspondence is a FOI request email a copy to Freedom of Information.

Index the document to the customer and delete the intray.

If the image should not be deleted, index the correspondence to the customer record. If the customer cannot be identified:

  • change the correspondence category to the most appropriate of either Unidentified, Unidentified Employer or Non-Customer
  • document the steps taken to locate the customer in the comments field and
  • if future checks are unlikely to locate the customer, check the item is not to be reviewed box to allocate the default review date 31/12/9999, or
  • if future checks may identify the customer, set an appropriate review date based on the future checks required

5

Action correspondence category 'Unidentified Emp' + Read more ...

View the correspondence and associated notepad in the List Correspondence window and index the correspondence to a customer record.

If unable to index the correspondence to a customer record:

  • email the GUID to National Employer Services Team
  • change the correspondence category to Action Reason Other and update the image notepad with details of the referral
  • check the item is not to be reviewed box to allocate the default review date 31/12/9999

6

Action correspondence category 'Unidentified' + Read more ...

View the correspondence and associated notepad in the List Correspondence window and index the correspondence to a customer record.

If insufficient information is provided to identify the customer and the correspondence includes contact details, change the correspondence category to Non-Customer.

If the customer cannot be identified:

  • document the steps taken to locate the customer and
  • if it is possible the customer can be identified at a future date, set a review date appropriate to the document type, or
  • if the customer is not likely to be identified at a future date, check the item is not to be reviewed box to allocate the default review date 31/12/9999

7

Action correspondence category 'Re-scanned' + Read more ...

View the correspondence and associated notepad in the List Correspondence window then:

  • index the correspondence if the image quality is sufficient to identify the customer, or
  • change the correspondence category to Unidentified if the image quality is suitable but the customer cannot be identified

If required, retrieve the document from batch storage and compare the hardcopy to the scanned image. If the quality of the hardcopy is better than the scanned document:

  • image the document again
  • index the correspondence to a customer record
  • change the correspondence category of the original image to Pending Delete or delete the image (appropriate access required)

If both the image and the hardcopy are of poor quality:

  • change the correspondence category to Unidentified
  • document that a better quality image cannot be obtained

8

Action correspondence category 'Specified Image' + Read more ...

Digitisation Solutions Support Service Officers are advised by their Team Leader when items are to be scanned as 'specified image'.

View the document and associated notepad. If the customer can be identified but the image quality is poor/not suitable:

  • Digitisation Solutions Support Service Officers located in the Melbourne office:
    • retrieve the document from batch storage and compare the original to the image
    • image the document again if the original document is of better quality
    • change the category of the original image to Pending Delete
    • if the Digitisation Solutions Service Officer has delete access, delete the original image
  • Digitisation Solutions Support Service Officers not located in the Melbourne office:
    • change the category of the image to Review Image
    • select the Action Reason
    • select a review date for the following day

If the image quality is suitable and unable to be attached to the document, change the category according to the reason the image cannot be attached.

9

Action correspondence category ‘Detached’ + Read more ...

Correspondence is detached from a customer record when:

  • it had been indexed (attached) to the wrong customer record
  • it is a multi-customer document
  • it is a multi-document from the same customer that needs to split
  • evidence that cannot be kept has been provided by a child (under 18)

View the document and associated notepad to determine the reason the image was detached, then:

  • index the correspondence to the appropriate customer record
  • if unable to index the correspondence, change the correspondence category to the most appropriate category based on the reason the image was detached, or
  • edit the document as required and rescan the document and index it to the customer

10

Action correspondence category for RACS Customers + Read more ...

The Digitisation Solutions Support team will ensure that these images are actioned by a RACS officer.

Forwarding originals + Read more ...

Forwarding originals is managed by National Digitisation Solutions Support Team. Original documents should only be sent if there is a significant business requirement.

If a physical document needs to be sent to another site, use approved mail or courier. For more information, see Resources for a link to Information security.

Record the name of the courier company and the consignment note number or barcode in a Cuba notepad. This will assist investigations if any problems occur.

If a file must be transferred to another site, a courier should be used and the file sent to a specific staff member.

To send the file:

Use double envelopes, marked with the name of the receiving officer. Double enveloping is used for information up to the level of 'Highly Protected'. For more information, see Resources for a link to Control and handling of security classified documents and material.

For a large number of files, a T1 box or similar can be used. Using security packaging requirements, clearly label the box with the name of the receiving officer:

  • Use a Services Australia approved courier
  • Record the consignment note details in the File Location notepad on Cuba for customer files
  • Use a File Transfer Register maintained centrally by the team. Ensure all information on the register is recorded in case tracking is required

The receiving staff member should complete the following when the file is received:

  • A return email confirming possession of the file
  • If files are registered on TRIM, the receiving officer must update the file location
  • Update file location notepad on Cuba

Retrieval of scanned documents + Read more ...

Documents are stored at Child Support sites for a prescribed period, usually 3 months, and then sent to the Centrelink Records Management Unit (RMU).
