Accessing the Aged Care Provider Portal (ACPP)

Table 1

Step

Action

1

PRODA access + Read more ...

Does the ‘A user’ have a PRODA account?

  • Yes, go to Step 2
  • No, the ‘A user’ must have a PRODA account to access ACPP. See Helping users create a Provider Digital Access (PRODA) account for assistance. Procedure ends here

2

Accessing and logging out of the ACPP + Read more ...

To access the ACPP:

  • for the first time, go to Step 3
  • as a returning user, go to Step 6

To log out of:

  • the ACPP, go to Step 7
  • PRODA, go to Step 8

3

Accessing ACPP for the first time + Read more ...

‘A users’ need to link their PRODA account and this only needs to be done once.

Note: ‘A users’ can access PRODA by:

  • going directly to the PRODA webpage, or
  • selecting the myGov Sign in on the Services Australia website. Select Aged Care Provider Portal under Health professionals and select Login PRODA

To link a PRODA account:

  • Go to the Services Australia website
  • Select the myGov Sign in on the right hand side
  • Select PRODA under Health professionals
  • Select Log on
    • The PRODA Login screen will display
  • Key in the username in the Username field
  • Key in the password in the Password field
  • Select Login

4

PRODA 2-step verification and landing screens + Read more ...

When the ‘A user’ has entered the username and password details:

  • The PRODA 2-step verification screen will display, and
  • PRODA sends an automated email to the ‘A user’s registered email address containing a one-time passcode

On the PRODA 2 step verification screen:

  • Key the passcode into the Enter Code field
  • Select Next. The PRODA landing screen will display
  • Select the Aged Care Provider Portal tile under Available services. The Linking screen will display

5

Linking an Aged Care User ID to PRODA + Read more ...

‘A users’ are prompted to link their Aged Care User ID to PRODA to obtain access to the ACPP. The Aged Care User ID is also referred to as the A number.

In the Linking screen:

  • Key in the Aged Care User ID or A number into the Aged Care User ID field
  • Press [Enter]

Is linking successful?

  • Yes, the Aged Care Provider Portal tile displays under My Linked services
    • ‘A users’ must select the Aged Care Provider Portal tile for the Aged Care Provider Portal landing screen to be displayed. See the ACPP landing page screen key features table for the process of going to and using the ACPP
    • Procedure ends here
  • No
    • The ‘Unauthorised error’ message displays to ‘A users’
    • ‘A users’ must resolve the linking issue prior to linking their Aged Care User ID to PRODA
    • ‘A users’ must contact the Aged Care Payments Team to help resolve the linking issue
    • Procedure ends here

6

Accessing the ACPP as a returning user + Read more ...

Note: ‘A users’ can access PRODA by going directly to the PRODA webpage.

To access the ACPP as a returning user:

  • Go to the Services Australia website
  • Select the myGov Sign in on the right hand side
  • Select PRODA under Health professionals
  • Select Log on
    • The PRODA Account Login screen will display
  • Key in the username in the Username field
  • Key in the password in the Password field
  • Select Login
    • The PRODA 2-step verification screen will display

PRODA sends an automated email to the ‘A user’s’ registered email address containing a one-time passcode.

On the PRODA 2-step verification screen:

  • Key the passcode into the Enter Code field
  • Select Next
    • The PRODA landing screen will display
  • Select the Aged Care Provider Portal tile under My Linked services
    • The Aged Care Provider Portal landing screen will display

See the ACPP landing page screen key features table for steps in going to and using the ACPP.

Procedure ends here.

7

Logging out of the ACPP + Read more ...

To log out of the ACPP:

  • Select the Return to PRODA button at the top far right corner of the screen (next to the ‘A user’s’ name)
    • The Return to PRODA pop up notification will display
  • In the Return to PRODA pop up notification, the ‘A user’ can either select:
    • Yes, the ‘A user’ will continue to log out of the ACPP and the PRODA landing screen will display. Go to Step 8
    • No, the ‘A user’ has not logged out of the ACPP and will return to the ACPP screen they were last on. The ‘A user’ can continue to use the ACPP. Procedure ends here

Note: the Return to PRODA button is available on every ACPP screen.

8

Logging out of PRODA + Read more ...

To log out of PRODA, in the PRODA landing screen:

  • select Logout at the top far right corner of the screen
  • the You are about to log out of the system pop up notification will display

In the You are about to log out of the system pop up notification, the ‘A user’ can either select:

  • Yes - the ‘A user’ will continue to log out of PRODA and the Logged out successful message will display
  • No - the ‘A user’ has not logged out of PRODA and will return to the PRODA landing screen

3

Going to and using the Current claim screen + Read more ...

On the ACPP landing page, going to the Current claim screen can be done via 4 methods:

  • the Claim summaries panel
    • select the current claim month
    • the Current claim screen will display
  • the Main menu (Hamburger button)
    • select the Main menu icon
    • select Current claim under the Claims section
    • the Current claim screen will display
  • the Quick link navigation bar
    • select the Current claim hyperlink in the Quick links navigation bar
    • the Current claim screen will display
  • the Quick find navigation bar
    • key Current claim in the Quick find search bar
    • select Go
    • the Current claim screen will display

Using the Current claim screen

The column headings in the Current claim screen are frozen. These are visible by scrolling the page down to view details.

The Current claim screen details the aged care service’s name, ABN details, the claim status and the claim month. For residential and flexible care services, approved places are displayed.

On the Current claim screen, the ‘A user’ can:

  • search the current claim
    • type in a keyword in the Search bar to search for all records that contain the keyword
      Note: the search results are displayed where the keyword is used anywhere on the record, even if it appears in the middle of another word
  • select the Legend button to display the explanation for the various codes. The Legend button will display for residential and flexible care services
  • view the Functional capacity score (MBI) and Care setting days for flexible care services
  • view the events by Status type
    • the Status type can either be All, Accepted, Being Processed, Deleted, Held or Rejected
  • select the number of records to be displayed from the Show records drop down list (bottom left hand corner of the screen)
  • go to a different page by selecting either the
    • First, Previous, Next or Last options, or
    • the relevant page number
    • finalise the current claim

4

Finalise the current claim + Read more ...

On the Current claim screen, ‘A users’ can finalise the current claim by selecting the Finalise claim button.

A warning message will display to ‘A users’ about possible outstanding events that are still being processed and they will be asked to confirm if they want to proceed.

‘A users’ can select:

  • Yes, go to Step 5
  • No, to cancel the claim finalisation process
    • The current claim is not finalised and the Current claim screen will display. Procedure ends here

5

Using the Certification and Authorisation screen + Read more ...

Note:

  • If the claim has been submitted successfully, a successful receipt message will display
  • An error message will display if there is an issue with the claim finalisation process
  • For residential and flexible care services:
    • the over occupancy report will display if their service is over occupied when finalising the claim
    • the claim cannot be finalised unless they resolve the over occupancy issue through their current process
  • For home care services:
    • an information message will display if invoice amounts have not been entered for all care recipients
    • the claim cannot be finalised until these amounts have been entered

The Certification and Authorisation screen will ask the ‘A users’ if they have read, understood and accept the terms of conditions prior to finalising the claim.

On the Certification and Authorisation screen:

  • check the box labelled ‘I accept the terms and conditions’
  • select:
    • Submit to finalise the claim
    • Back if ‘A users’ do not want to accept the terms and conditions or finalise the claim and want to return to the Current claim screen

Procedure ends here.

6

Going to and using the All service claims screen + Read more ...

The All service claims screen can be used to view and/or check the status of all claims (current and historical claims).

On the ACPP landing page, going to the All service claims screen can be done via 2 methods:

  • the Main menu (Hamburger button)
    • select the Main menu
    • select All service claims under the Claims section
    • the All service claims screen will display
  • the Quick find navigation bar
    • key ‘All service claims’ in the Quick find search bar
    • select Go
    • the All service claims screen will display

Using the All service claims screen

Note: for residential and home care purposes, the All service claims screen displays a maximum of 24 months of claim history. This includes the current claim month.

In the All service claims screen, the claim history will display in table format:

  • Claim month - select the up or down arrows to display the months in ascending or descending order
  • Date finalised - select the up or down arrows to display the dates in ascending or descending order
  • Authorised person - select the up or down arrows to display the Authorised person’s name in alphabetical order
  • Channel - can either be Services Australia or the Portal. Select the up or down arrows to display the Channel form in alphabetical order
  • Date approved - select the up or down arrows to display the dates in ascending or descending order
  • Status - can either be Approved, Open or Processing and select the up or down arrows to display the Status type in alphabetical order

In the All service claims screen, ‘A users’ can also:

  • search for a specific current claim
    • type in a keyword in the Search bar to search for all records that contain the keyword
      Note: the search results are displayed where the keyword is used anywhere on the record, even if it appears in the middle of another word
  • change the claim year
    • select the year from the Claims for year drop down list. Note: the Claims for year will default to the current year
  • select the number of records to be displayed from the Show records drop down list (bottom left hand corner of the screen)
  • go to a different page by selecting either the
    • First, Previous, Next or Last options, or
    • the relevant page number

Procedure ends here.

  • Select the Circumstances tab
  • Select Leave
    • The Leave summary screen will display

Using the Leave summary screen

Note:

  • The Leave summary screen’s layout may be different depending on the service in context’s care type
    • The Leave balances table displays for residential care services only
  • To view the leave balance for
    • home care recipients, see Using the Care recipients’ entitlements screen of the Aged Care Provider Portal
    • flexible care recipients, see Using the Care recipients’ details screen of the Aged Care Provider Portal

The Leave summary screen displays the following:

  • The ‘Leave balances are displayed for claimable leave types as at [XX/XX/XX] [XX:XX AM/PM]. Used days consist of leave registered including leave without an end date’ information message
  • Care Recipient banner - displays the care recipient’s name, IDs, Date of Birth, Record type and Type of care status
  • Leave balances table
    • Leave type
    • Period
    • Approved allocated days
    • Used days
    • Remaining days
  • Leave registered table
    • Start date
    • End date
    • Leave duration (days)
    • Leave type
    • Created date
    • Last updated
    • Status - select the relevant hyperlinked Status to display details for an individual leave event

In the Leave summary screen, ‘A users’ can:

  • select Register leave to register (create) a leave event in an individual care recipient profile. Go to Registering a leave event of Table 8 for the process
  • type in a keyword in the Search bar to search for all records that contain the keyword
  • select the number of records to display from the Show records drop down list (bottom left corner of the screen)
  • select Back to profile to return to the care recipient’s profile
