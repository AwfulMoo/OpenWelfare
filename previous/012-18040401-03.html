This page contains how to complete the monthly MI pay run for MI applications and refunds.

Process monthly MI pay run

Complete monthly MI pay run

This table explains how to action the monthly MI pay run.

Step

Action

1

Create Autofill spreadsheet + Read more ...

Create the following folders in the shared drive within the relevant month folder:

  • Finals to Delegate
  • Finals to Shared Services
  • Remittance Advice [Month] [Year]

Open the ‘Autofill spreadsheet template’ and save the file as ‘[month year] Autofill spreadsheet’ into the relevant month folder.

On the claim paper file, check that the:

  • file has been signed off by the delegate as a payment
  • payment figures on the registration cover sheet, assessment worksheet, file note, file cover slip and the Work on Hand (WOH) spreadsheet match

Note: if there is a discrepancy between the registration cover sheet and the figure that is to be paid, complete a new registration cover sheet with the updated financial information. This scenario sometimes occurs if the AO overlooks updating the registration sheet when assessing a section 38 claim and the financial information has changed.

Key each of the applications requiring payment for the month into the ‘Autofill spreadsheet template’. A single application may have multiple entries depending on the number of general ledger (G/L) account codes for the application.

Key the following into the ‘Autofill spreadsheet template’:

  • TRIM identifier: relates to the Insurer, and is the letter after the ‘ARN’ sequence. For example, for ARN1234-3U-H, the Insurer is Avant Mutual Group and the TRIM identifier is ‘U’. See the Resources page for Vendor codes and general ledger account codes
  • Vendor: code will automatically populate based on the TRIM identifier. Each Insurer has a unique vendor number. For example, the vendor number for Avant Mutual Group (‘U’) is ‘MDO1000002’. See the Resources page for Vendor codes and general ledger account codes
  • Invoice date: is the date the payment application was received. See the first page of the payment application form and the registration worksheet. Use the following format ‘DD.MM.YYYY’. If assessing a section 38 response, key the date the original payment application which prompted the section 38 was received
  • Scheme: refers to the scheme under which the payments are being made. The scheme is identified by the last letter of the Application Reference Number (ARN) number (for example, High Cost Claim Indemnity Scheme (HCCS), Run-Off Cover Indemnity Scheme (ROCS), Incurred But Not Reported (IBNR) Indemnity Scheme, ROCS/HCCS and IBNR/HCCS). For example, for ARN1234-3U-H the scheme is ‘HCCS’
  • ARN: is the unique identifier for each payment application. Key the full ARN related to the payment application, for example ‘ARN1234-3U-H’
  • G/L account code: G/L account codes are used to track administered funds across Services Australia. There are 11 different G/L codes for the MI section, 1 for each of the payments schemes for settlement, plaintiff and defence costs and 2 separate ones for claims handling fee (for ROCS and IBNR). A single claim may have multiple G/L account codes. For example a HCCS claim with settlement and defence costs will have 2 G/L codes, 51012 and 51014. The application will have a separate row in the spreadsheet for each G/L code. The G/L codes for each application are included on the first page of the Registration Worksheet. See the Resources page for Vendor codes and general ledger account codes
  • Cost centre code: a different cost centre code exists for each of the payment schemes. The cost centre code will automatically populate based on the G/L account code. See the Resources page for Vendor codes and general ledger account codes
  • Claim Reference #: is the identifier allocated to the claim by the insurer. See the payment application form and the first page of the Registration Worksheet. Note: do not use commas and make sure leading ‘0’ is included
  • Dr initial and last name: the column includes the title, first initial and surname of the Medical Practitioner relating to the payment application. For example, Dr J Citizen
  • AMT: this column includes the amount to be paid for the relevant G/L account code. It is important:
    • that no formatting is applied, such as dollar signs or commas as this will create errors once uploaded into SAP by Accounting Operations
    • amounts are to have 2 decimal points and key refunds as a negative amount, for example a $100.00 refund is '-100.00' and make text amount red
    • to check the refund amount and ensure the insurer has sufficient money for the refund to be deducted. If there is insufficient money for the refund to be processed, the refund will need to be carried over to the next month. If in the next month the insurer still has insufficient money a debt recovery process may be undertaken
  • Payrun date: this column relates to the date the payment is to be made. Use the format ‘DD.MM.YYYY’ This field will automatically populate for each new line once the first line is keyed
  • Line item text: this column includes a summary of payment information and auto populates based on information previously keyed

Key the above information for each payment application that has been approved for payment.

Save the ‘Autofill spreadsheet’.

2

Create Delegate and Shared Services spreadsheets + Read more ...

Spreadsheet for Delegate 1

Open the ‘Master spreadsheet’ template, save as ‘[month year] Final Master Spreadsheet’ in the ‘Finals to Delegate’ folder.

Copy and paste the following fields from the ‘Autofill spreadsheet’ into the ‘[month year] Final Master spreadsheet’. Copy and paste as special values:

  • Autofill spreadsheet column B (vendor) into Master spreadsheet column A (vendor)
  • Autofill spreadsheet column C (invoice date) into Master spreadsheet column B (inv date)
  • Autofill spreadsheet column D (Scheme) into Master spreadsheet header text column C (header text)
  • Autofill spreadsheet column E (ARN) into Master spreadsheet column D (reference)
  • Autofill spreadsheet column F (GL account code) into Master spreadsheet column E (GL account code)
  • Autofill spreadsheet column G (cost centre code) into Master spreadsheet column F (Cost Ctr)
  • Final Master Spreadsheet, Blank column with no heading – do not delete this column
  • Final Master Spreadsheet, Tax Code: leave blank. This section would only be populated if tax is to be paid
  • Final Master Spreadsheet, Order: leave blank
  • Final Master Spreadsheet, Qty: leave blank
  • Final Master Spreadsheet, Qty Unit: leave blank
  • Autofill spreadsheet column N (AMT) into Master spreadsheet column L (AMT). Remove any commas or dollar signs by formatting the cells to ‘general’. Use 2 decimal places
  • Autofill spreadsheet column O (payrun date) into Master spreadsheet column M (BLINE date)
  • Autofill spreadsheet column P (line item text) into Master spreadsheet column N (line item text)

Remove the formatting from all cells, including the background fill.

Note: ensure any refund amounts are in red.

Select all the data (not including the headers):

  • select ‘Custom Sort’ from ‘Sort & Filter’ in the Home tab
  • sort according to the following parameters:
  • Column – sort by Reference (column D)
  • Sort on – Values
  • Order – A to Z

Select OK.

Add a heading to the document labelled ‘[Month year] Final Master Spreadsheet’. Save and print the spreadsheet.

Note: print all spreadsheets in ‘landscape’ page layout,‘A3’ size and in colour.

Spreadsheet for Delegate 2 – Sorted by ARN

This spreadsheet contains all the payment data sorted according to the Application Reference Number (ARN).

Open the ‘Final Master Spreadsheet’ and save as filename ‘[month year] Final Master Spreadsheet by ARN’ in the ‘Finals to delegate’ folder:

  • select all the data (including the headers) and from the Data tab, select Subtotal. A dialogue box will display the message ‘Microsoft Excel cannot determine which row in your list or selection contains column labels, which are required for this command
  • select OK
  • in the ‘Subtotal’ dialogue box, key the following parameters:
    • at each change in: Reference
    • use function: sum
    • add subtotal to: AMT
  • select OK

Ensure the following boxes are checked ‘Replace current subtotals’ and ‘Summary below data’ and select OK.

Bold the subtotals and grand totals. Add the heading ‘[Month Year] Final Master spreadsheet by ARN’.

Save and print the spreadsheet.

Spreadsheet for Delegate 3 – Sorted by scheme

Open the ‘Final Master Spreadsheet’ and file save as ‘[month year] Final Master Spreadsheet by Scheme in the ‘Finals to delegate’ folder.

Select all the data (not including the headers). In the Home tab, select ‘Custom Sort’ from ‘Sort & Filter’.

Sort according to the following parameters:

  • Column – sort by Cost Ctr (column F)
  • Sort on – values
  • Order – A to Z

Note: sort by Cost Ctr (column F) and not by Header Text (column C). If sorted by column C any split claims will be incorrectly sorted.

Separate the different schemes by inserting a new line, add a heading (for example, 'HCCS total') and using the AutoSum formula total the amounts to be paid for all applications related for each scheme.

Include a grand total at the bottom of the spreadsheet by summing the sub totals for each of the schemes. Select the AutoSum formula to automatically pick up the subtotals.

Bold the subtotals and grand totals. Add the heading ‘Final Master spreadsheet by Scheme’. Save and print the spreadsheet.

Spreadsheet for Delegate 4 – Sorted by insurer and cost centre

Open the ‘Final Master Spreadsheet’ and file save as ‘[month year] Final Master Spreadsheet by Insurer and Cost Centre’ in the ‘Finals to delegate’ folder.

Select all the data (not including the headers). Select ‘Custom Sort’ from ‘Sort & Filter’ in the Home tab

Sort according to the following parameters.

  • Column – sort by Vendor (column A)
  • Sort on – values
  • Order – A to Z

Add another level by selecting the option in the dialogue box, and key the following parameters:

  • Column – sort by Cost Ctr (column F)
  • Sort on – values
  • Order – A to Z

Select OK.

The data will now be sorted according to the vendor first (the insurer) and then sorted by the Cost Centre (IBNR, HCCS, ROCS)

Separate the different insurers by inserting a new line, add a heading (for instance, Avant Insurance Limited total) and total all payment applications related to the insurer.

Create a grand total at the bottom of the page by adding together the subtotals for all of the insurers. Ensure the grand total is bold.

Add the heading ‘[Month Year] Final Master spreadsheet by Insurer and Cost Centre’. Save and print the spreadsheet.

Spreadsheets for Shared Services

For the pay run, 3 separate spreadsheets are provided to Shared Services, 1 for each of the payment scheme (HCCS, IBNR and ROCS).

To create the spreadsheets open the ‘Final Master spreadsheet’. Select all the data (not including the headers).

Select ‘Custom Sort’ from ‘Sort & Filter’ in the Home tab

Sort according to the following parameters:

  • Column – sort by Cost Ctr (column F)
  • Sort on – values
  • Order – A to Z

Select OK.

Copy all of the ‘C1600’ cost ctr codes into a new Excel spreadsheet

Note: do not add any headings or totals.

Save as ‘MI Payrun – KF – ROC Scheme’ into the ‘Finals to Shared Services’ folder.

Repeat instructions from the heading 'Spreadsheets for Shared Services' for the:

  • ‘C1597’ cost centre codes saving the document as MI ‘Payrun – KH – HCC Scheme’
  • ‘C1596’ cost centre codes saving the document as ‘MI Payrun – KI – IBNR Scheme’

Ensure the figures for the spreadsheets match the grand total amount on the ‘Autofill spreadsheet’. Save and print spreadsheet(s).

3

Prepare delegate minute + Read more ...

The delegate is required to sign a minute requesting authorisation to release the payments for all approved MI claims for the month. The minute contains details of the total amount payable under each MI scheme and the total amount payable to each MII and MDO. An electronic and hardcopy of the signed delegate memo is maintained for record keeping purposes.

Open the template 'Delegate memo for payrun template' And Save As <month year> Delegate memo

Note: if the total amount payable to the insurer is a negative figure. The payment needs to be carried over to the next month payment, where there is sufficient funds. In this case, use the template ‘Delegate Minute for Payrun Recoveries Removed Template’ and save as ‘[month year] Delegate minute’.

Update the relevant details in the minute (highlighted in red).

Complete the tables in the minute using the information provided in the spreadsheets created above.

Note: only include insurers and schemes relevant to the pay run.

Proof read the minute and format all font colour to black in the document and remove all comments by selecting the review tab, delete drop down, delete all comments.

Print the minute one-sided.

Present the MI Pay run corporate file with the following documents to the Assistant Director to review and provide clearance:

  • Delegate Minute
  • Final Master Spreadsheet Final
  • Final Master Spreadsheet Sorted by ARN
  • Final Master Spreadsheet Sorted by Scheme
  • Final Master Spreadsheet Sorted by Insurer and Cost Centre
  • MI Payrun – KH – HCCS Scheme (raw data for uploading into SAP)
  • MI Payrun – KI – IBNR Scheme (raw data for uploading into SAP)
  • MI Payrun – KF– ROCS Scheme (raw data for uploading into SAP)

4

Submit data to Shared Services + Read more ...

Once the delegate has authorised release of payments, an email is sent from the MI mailbox to Services Australia’s Accounting Operations team requesting the release of payments. The Accounting Operations team will advise the MI team via email when the payments have been released. This usually occurs on the same business day.

Once the documents have been approved by the delegate.

Using the stationery email template in the mail box. send an email to the National Accounting Operations Branch and the Accounts Payable Helpdesk mailbox and cc: any relevant additional recipients with the following information:

  • MI Payrun – KH – HCCS Scheme (raw data for uploading into SAP)
  • MI Payrun – KI – IBNR Scheme (raw data for uploading into SAP)
  • MI Payrun – KF– ROCS Scheme (raw data for uploading into SAP)
  • Signed copy of the delegate minute

Note: see the Accounts Payable Helpdesk team for contacts and information.

Accounting Operations will forward the email to Services Australia’s Finance section who will provide the data to the Reserve Bank of Australia (RBA) for payment release.

Note: this step serves as a notification to the MI Team only, no action is required at this point.

Once payments have been released to the RBA, the Finance team will send an email to the MI mailbox advising that payments have been released as requested. The Finance team will also attach a PDF payment summary and PDF payment advices.

Print the attached files:

  • PDF payment summary
  • PDF payment advices

The email will contain SAP Output PDF documents which must be checked by the MI Programme Officer to ensure they reconcile with the payment amount(s) requested.

Note: if there is a discrepancy between the total payment request and the total payment released, escalate the issue to Assistant Director or Director immediately and investigate the cause of the discrepancy.

File all email correspondence sent and received.

Scan and save all the attachments from the Finance team to the relevant pay run folder in the shared drive.

5

Update HP Records Manager payment information + Read more ...

Open the ‘Payment Advices’ received from Services Australia’s Finance area. For each of the MI applications, place the cover sheet on the paper file and update the registration worksheet in the corporate file to include the Document Number.

For each of the MI applications paid, ensure the payment information is updated in HP Records Manager using the registration worksheet, including:

  • Application status (paid)
  • Status updated (date)
  • Invoice Reference (document number)
  • CHF Paid (for IBNR and ROCS only)
  • Plaintiff Costs Paid
  • Defence Costs Paid
  • Judgment Costs Paid
  • Amount Paid
  • Date Paid (date payment made as per payment advice)
  • if applicable, ensure the Financial recoveries tab is updated

Handover corporate files and payment checklist to Executive Level 1 for Quality Check once all payment data has been entered into HP Records Manager.

Once the corporate files have been updated, they need to be archived. See Monthly Medical Indemnity (MI) administrative process.

6

Prepare and send payment advice to each insurers + Read more ...

Save a PDF copy of each insurer’s payment advice as ‘Insurer Payment Advice – [Month Year]’ in the relevant pay run folder in the shared drive.

Open the payment advice stationery email template in the mailbox and populate the details in red for each insurer.

Attach the payment advice from the relevant pay run folder to the relevant email for the Insurer.

Provide the email to another Programme Officer to be checked for accuracy and corrections before sending.

Send the email to the appropriate insurers via secure email.

File the email in the relevant folder in the MI mailbox.

7

Update the Payment Corporate file + Read more ...

A hard copy corporate file is maintained for monthly payments made under the MI schemes. The file includes all information and documentation for the pay run process.

A single corporate file exists for all payment schemes. Include the following paper-based documents in the payment file:

  • PDF payment summary
  • PDF payment advice
  • email from Finance confirming payments were released
  • delegate memo, and
  • all payment summary spreadsheets

Compile the paper file, placing the documents in the order listed above (front to back).

8

Update Claims on Hand spreadsheet + Read more ...

The status of the payment applications for the relevant month are updated after the pay run process has been completed and all relevant letters have been sent to the insurers.

Open the Claims on Hand spreadsheet and update each payment application for the relevant month to ‘paid’. See the Resources page for assistance.

Go to Monthly Medical Indemnity (MI) administrative process.
