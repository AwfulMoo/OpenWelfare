Review of decision by an ARO

Table 1

Step

Action

1

Request allocation of a review + Read more ...

To request allocation of a review, use the 'Request an ARO review' webform. See the Resources page for a link.

Allocation requests are monitored from Monday to Friday, 8:30 to 17:00 (AEDT). Within these times, the Allocation Team aim to action a request within 1 hour. AROs should have 2-3 actionable reviews to avoid down time due to the allocation process.

2

Locate an allocated review + Read more ...

Use the APLSEL screen in APL to monitor reviews on hand. To perform a search of all unfinalised cases for the ARO logon, in the APLSEL screen:

  • in the Lvl: field, key 'ARO', and
  • in the Logon: field, key ARO logon

3

SME quality check + Read more ...

When the SME makes a decision, it will be unfavourable or favourable. An unfavourable decision is an adverse decision. A favourable decision advantages the customer, but may still be adverse. For example, a debt is recalculated and the debt amount is reduced but the remainder must be repaid. A decision that advantages the customer and has no adverse effect is a fully favourable decision.

  • If the SME’s decision was not fully favourable, go to step 4
  • If the SME’s decision was fully favourable and the customer not advised, go to step 4
  • If the SME did not make a decision before the review was allocated to an ARO, go to Step 4
  • If the SME’s decision was fully favourable and the customer advised, or the customer withdrew the review request during the quality check, an ARO review is not required:
    • DOC the ARO review is not required because the SME made a fully favourable decision or the customer withdrew their review request
    • Delete the ARO level review from the APL system record
    • Go to the Case List (ACL) screen
    • Select the relevant APL
    • Enter 'DEL' in the Nxt: field
    • Select ARO
    • Enter 'Y' to confirm deletion
    • If required, input the date finalised as the date the SME quality check was completed, and
    • Overtype the Outcome field with the relevant code
    • Procedure ends here

4

Check that a decision has been made + Read more ...

Identify the decision that has been referred for review. This will be the original decision or, if the SME changed the original decision, the SME’s decision.

A review can only be done for a decision that has been made and that the customer is affected by. If a decision has not been made, or a decision has been made but has no effect on the customer, a review cannot be done.

If there is difficulty confirming if a decision was made, contact the Review of Decision Helpdesk. See the Resources page for a link.

If a decision has been made and the customer is affected, go to Step 5

If a decision has not been made or the customer is not affected by the decision, delete the APL record:

  • Go to the Case List (ACL) screen
  • Select the relevant APL
  • Enter 'DEL' in the Nxt: field
  • Key 's' and select ALL
  • Annotate the ARO progress DOC to say that the APL record has been deleted because a decision has not been made or the customer is not affected by the decision
  • Create an ACTDOC to say that the review number [APL number] has been incorrectly recorded as a decision has not been made or the customer is not affected by the decision
  • Include in the ACTDOC that the customer will need to be contacted to inform them that a review cannot be done and to advise them of any action they need to take to have the decision made, for example, to lodge outstanding documents
  • Use Work Optimiser to re-categorise the ACTDOC work item to the SME team if a quality check was done
  • Use Work Optimiser to re-categorise the ACTDOC to the team where the review was recorded if a quality check was not done
  • Provide feedback via the Staff Feedback Tool to the officer who added the APL record and to the SME if a quality check was done
  • Procedure ends here

5

Check if the review is to be reallocated + Read more ...

A review can be reallocated to another ARO where an ARO cannot do all or some components of the review because:

  • a decision incorporated in the review is not in their skillset, or
  • they will be on leave for more than 5 working days, or
  • the review type is done by specialised AROs

Note: Australian Victim of Terrorism Overseas Payment (AVTOP) reviews, Family Tax Benefit (FTB) and Child Care Subsidy (CCS) immunisation type reviews, and Additional Child Care Subsidy (ACCS) - Child Wellbeing, Grandparent and Temporary Financial Hardship - reviews, are done by specialised AROs.

The ARO will discuss with their Assistant Director whether reallocation is appropriate. If the Assistant Director agrees that reallocation is appropriate they will submit the Reallocate an ARO review webform. See Resources for a link to the webform.

Note: all reallocations must be done using the Reallocate an ARO review webform. Arrangements should not be made to reallocate reviews without using this form.

The ARO will:

  • Check the APL system is correctly recorded. If the reallocation is because the review was incorrectly recorded in the APL system, amend the coding so the review is correctly allocated to another ARO
  • Annotate the ARO Progress DOC to say reallocation has been requested and the reason, include the text 'Do not delete'

6

Check jurisdiction + Read more ...

Contact the Review of Decision Helpdesk if there is difficulty confirming if there is jurisdiction to review the decision. See the Resources page for a link to the helpdesk.

Does an ARO have jurisdiction to review the decision?

  • Yes, go to Step 7
  • No, see Step 9 in the Contacting customers and completing the review table and send a 'no jurisdiction' letter

7

Check if the SME referral meets required ARO standards + Read more ...

If a quality check has not been done as outlined in Quality checks by a Subject Matter Expert (SME), referral to Authorised Review Officer (ARO) and implementation of ARO decisions, request further information or action from the SME as required. Follow the process in Step 7 in the Contacting customers and completing the review table.

Provide feedback via the Staff Feedback Tool to the officer who made the ARO referral.

When the referral meets the required standard, go to Step 8.

8

Review the decision + Read more ...

Investigate the circumstances of the review:

  • Check the APL system record to ensure the review is correctly recorded. Make necessary corrections and updates, including adding a priority indicator, if appropriate
  • Add extra APL system records according to the ARO counting rules, using the Internal Review/Explanation script. Use the Reallocate an ARO review webform to request these system records. See Resources for a link to the webform. Add this text to the Details box of the webform: 'Please allocate this review to (insert LOGON) as APL (insert APL number) has already been allocated to me’
  • Evaluate all available evidence that may have a bearing on the review outcome, regardless of the age of the evidence or when it was provided to the department
  • Identify and add affected third parties. See ‘Multiple parties affected by a decision’ in the Additional considerations for AROs table on the Resources page
  • Research Operational Blueprint, policy and legislation
  • Contact third parties if necessary

Do not complete the review immediately if the SME has requested:

  • the customer provide documents
  • a debt to be recalculated
  • assessments to be completed, etc.

Wait until the actions are complete or the time has expired for the customer to return the documents before finalising the review.

Only review the decisions the customer has asked to be reviewed and that are available for review. See 'Scope of an ARO review' in the Additional considerations for AROs table on the Resources page.

Do not wait for decisions consequential to the ARO decision to be made. See 'Consequential decisions' in the Additional considerations for AROs table on the Resources page.

If further action by the Subject Matter Expert (SME) is required, see Step 7 in the Contacting customers and completing the review table.

At any time in the ARO review process, if the decision is:

  • a Disability Support Pension (DSP) medical based claim rejection, and
  • the customer has provided new evidence relating to the 13 week claim period

The ARO has the option of seeking a new recommendation from the Medical Assessment Team (MAT). The MAT will:

  • consider the new medical evidence in the context of any previous MAT, JCA or DMA recommendations
  • make a new recommendation and scan this to Document Tools

Can a fully favourable (set aside) decision be made?

  • Yes, and
    • the decision is a Disability Support Pension (DSP) medical based claim rejection or Medical Risk Based review cancellation, see Step 8 in the Contacting customers and completing the review table
    • the decision is not a DSP medical based claim rejection or Medical Risk Based Review cancellation, see Step 9 in the Contacting customers and completing the review table
  • No, see Step 1 in the Contacting customers and completing the review table

Contacting customers to complete the ARO review

Table 2: This table describes the steps an Authorised Review Officer (ARO) follows when contacting the customer and completing the review.

Step

Action

1

Contact with customer + Read more ...

If there are phone contact details recorded, make 2 genuine attempts to contact the customer.

Follow the protocols for Calling a customer or returning a customer’s call.

If the customer has no phone details:

  • create and send a QARO letter giving the customer 10 days to respond
  • include advice that if they do not contact in this period their review will be finalised using the available evidence
  • update the ARO Progress DOC to record the QARO has been sent

Note: for the text to include in the QARO letter, see the Resources page.

If the customer:

  • responds by phone, go to Step 2
  • cannot be contacted by phone or does not respond to the QARO letter, go to Step 6
  • does not answer their phone on the first attempt and a message facility exists:
    • leave a message asking them to call back
    • if they do not call back, make a second genuine attempt to contact on a different day
    • if they answer their phone on the second attempt, go to Step 2
    • if they do not answer their phone on the second attempt, go to Step 6

2

Phone conversation with customer + Read more ...

If the customer requests an interpreter, or it is evident an interpreter is needed, see Offering an interpreter and tell them they will be called back with an interpreter. Arrange an on-demand interpreter, see Booking an interpreter and call the customer back using a conference call.

Some calls will be evaluated within the Quality Call Framework against the Quality Call Standards. See the ARO Quality Call Framework Evaluation task card on the Resources page.

If the customer requests a face-to-face interview, the ARO contacts their Assistant Director to make arrangements. The Assistant Director will arrange for an ARO in a service centre close to where the customer lives to do the review. The review will be reallocated to that ARO.

Clarify the reason the customer requested a review, explain why the original decision was made and answer the customer’s questions.

If the customer withdraws their review request:

  • inform them the review request will be taken never to have been made. Explain the implications, for example, if they request a review of the same decision at a later time, and the decision is changed, full arrears may not be paid
  • annotate the ARO Progress DOC with the time of the discussion and that the customer withdrew their review request
  • run the ARO Finalise Review Case script to record the review outcome. The script does not provide the WDS (Withdrawn/Dismissed) option, which is the correct Explanation field code.
    • Select Automatic Decision instead and select Next
    • Confirm the Office Action details and select Next
    • An error box will be presented and Automatic Decision highlighted
    • Correct the field to ‘WDS’ (Withdrawn/Dismissed) and select OK
    • Navigate through the rest of the script to complete the review recording
  • send a letter confirming the review request was withdrawn. See ‘ARO Letter’ in the Requirements for documents used by AROs table on the Resources page

If a customer requests a review of an unrelated decision, the Internal Review/Explanation script must be used to create a new APL record. See 'Scope of an ARO review' in the Additional considerations for AROs table on the Resources page.

The new APL will be referred to a Subject Matter Expert for quality check. If an ARO referral is required, it may not be allocated to the ARO who recorded the review.

Note: If the customer presents as suicidal or in a life-threatening situation during the phone call, refer them to an Integrity and Information Group Social Worker. Current information on how to make a referral is available in the Office Notes section of the BI Social Work Virtual Team (BSW) page in Office Locator. See the Resources page for a link.

For other Social Worker referrals or consultations, for example the customer talks about suicide or self-harm (but is not in a life-threatening situation), or is affected by family and domestic violence, use the Social Worker Referral guided procedure in Customer First.

A Job Capacity Assessment or Disability Medical Assessment may be required if the decision is:

  • a Disability Support Pension (DSP) medical based claim rejection that may be set aside, or
  • a Medical Risk Based review cancellation that may be set aside
  • For information, see Step 8

Is the decision known?

  • Yes, go to Step 3
  • No, and:
    • additional information is required, go to Step 4
    • no request for additional information is required, go to Step 5

3

Decision known: + Read more ...

Advise the customer:

  • Of the decision and reasons for it
  • The consequences of the decision. For example:
    • If recovery of a debt has been paused pending the outcome of the review, recovery will restart
    • If the customer is being Paid Pending Review (PPR), advise them that PPR will now stop
  • They may request a review by the Administrative Appeals Tribunal (AAT)
  • They can claim an alternative payment or lodge another claim for the same payment, if appropriate. For example, the customer did not qualify for the payment when they made their claim, but may qualify now

When the customer conversation has ended, update:

  • the ARO Progress DOC with the time and date of the conversation
  • ARO Notes with full details of the conversation with the customer

Go to Step 9.

4

Decision not known - additional information required: + Read more ...

  • Ask the customer to provide additional information
  • Advise the customer:
    • this information must be supplied within 7 days and, if not, the review will be finalised with the available evidence. The 7-day period can be extended if needed
    • a decision will be made once the information is provided
    • when a decision is made a decision statement will be sent
    • they may request a review of the decision by the AAT

When the conversation ends, update the ARO Progress DOC with the date and time of the conversation and information requested. Monitor the case using APLSEL.

The customer is responsible for providing additional evidence if they want that evidence taken into account for the review.

When the information is supplied or the case falls due, go to Step 6.

5

Decision not known - no request for additional information required: + Read more ...

  • advise the customer:
    • the decision must be investigated further before a decision can be reached
    • once a decision is made a decision statement will be sent
    • they may request a review of the decision by the AAT

When the conversation ends, update the ARO Progress DOC with the date and time of the conversation and a brief outline of the discussion. Go to Step 6.

6

Further investigation + Read more ...

Conduct any required further research on the customer record, Operational Blueprint, policy, legislation and contact third parties when necessary.

If a third party is contacted, update the ARO Progress DOC with:

  • the time and date of the call and name of the third party
  • the information requested from the third party
  • the due date for information
  • update ARO Notes with full details of the conversation, and
  • monitor the review for the return of information

If:

  • further action by the Subject Matter Expert (SME) is required before the review can be finalised, go to Step 7
  • the review is for a DSP medical based claim rejection or Medical Risk Based review cancellation that may be set aside, go to Step8
  • the decision can be made. Go to Step 9

7

Further action required by Subject Matter Expert (SME) + Read more ...

Further action by a SME might be required before the review can be completed. For example, this may be due to:

  • the SME referral being below standard
  • recalculation of a debt, or
  • assessment of qualification or payability for a new claim
  • Action is to be taken by the responsible SME/team in accordance with Timeliness standards for callbacks and ACTDOCs

Create and transfer an ACTDOC to the relevant team + Read more ...

Using the Callback/Actdoc guided procedure in Customer First:

  • Include the text:
    ARO review not complete. Action required by SME. APL <APL number>.
    When complete, use Work Optimiser to reassign to ARO Logon.
    <Add clear and comprehensive instructions describing the action required.>
  • Do not update the customer record
  • Do not close this DOC
  • <ARO NAME> <ARO LOGON> <FULL PHONE NUMBER>.
  • Add the keywords 'URGENT', '!ARO' and 'ACTxxx' (Where ACTxxx is the customer's most relevant payment). See Creating Callback and ACTDOC work items

Note: do not ask SMEs to undertake assessments if the ARO can complete the review without an assessment being done. For more information, see the Additional considerations for AROs table on the Resources page.

For requests to Families and Childcare Branch SME, refer the ACTDOC to Region Office Code FCC.

If recalculation of a debt is needed before a review can be completed:

  • create an ACTDOC including specific information to be used to recalculate the debt. This may include:
    • the start and end dates of the debt period
    • any supplementary add on payments, for example Rent Assistance, including date range
    • specific details of any income/assets/other circumstances to be used for the recalculation
    • the keyword '!ARO'
    • action the debt officer is required to take when they re-allocate the ACTDOC back to the ARO
  • transfer the ACTDOC to the Regional Office Code of the team responsible for the original debt decision. For requests that Debt Management Branch (DMB) are responsible for, transfer the ACTDOC to the following Region Office Codes (ROC):
    • 'XWA' for debt raising decisions. Include the additional keyword ‘DQADC2’
    • 'NS7' for debt recovery decisions
    • 'CTN' for compensation recovery decisions

The DMB are not responsible for all Department of Human Services debts. The majority of DMB debt raising teams have a ROC code beginning with 'X'. HU1 and HU2 are also DMB debt raising teams. AROs should ensure they are referring recalculation requests to the correct team.

Use Work Optimiser to refer the work item representing the ACTDOC to the appropriate team. That is, enter the Customer Reference Number (CRN) and select the corresponding work item. If the work item type is incorrect, re-categorise the work item using the appropriate work item type.

No action taken by the SME after 72 hours of creating the work item + Read more ...

Sometimes ACTDOCs cannot be actioned within 72 hours. Examples include complicated debt recalculations using MultiCal or additional information required from external parties. The SME is to advise the ARO within 72 hours if a delay is expected.

If action has not been taken within 72 hours and the SME has not advised of a delay:

  • complete the ARO to SME Escalation webform requesting follow up. See Resources for a link to the webform
  • hold the review, awaiting SME action

When SME action has been completed + Read more ...

  • Complete the work item. In the Worklist in Work Optimiser, unassign the work item and choose a reason of Work Item Complete
  • Close the DOC
  • If SME action was required because an assessment was needed to complete the review:
    • if the decision is for a DSP medical review or claim rejection to be set aside, go to Step 8
    • for all other decisions, go to Step 9

8

DSP medical based claim rejection or Medical Risk Based review cancellation decision to be set aside (a fully favourable review decision to be made) + Read more ...

This Step does not apply if setting aside the decision is not being considered. For example, where the customer is not qualified for DSP on non-medical grounds, where DSP is not payable under the income or assets test, or if the customer is clearly not medically qualified.

The information in Claim determinations - social security payments in the Additional considerations for AROs table must be considered, before setting aside a rejection decision and substituting a grant decision.

AROs are authorised to assign new impairment ratings and set aside a rejection or cancellation decision.

The ARO’s decision must be made in accordance with good decision making practice and reasons provided in line with the scenarios below.

Claim rejected based on Medical Assessment Team (MAT) recommendation

A Service Officer has rejected the claim based on the MAT consideration of medical evidence. The ARO considers a qualifying impairment rating could be assigned (the ARO may discuss the matter with their Assistant Director):

  • the ARO has the option of seeking advice from the Medical Assessment Team (MAT) by email. MAT will:
    • consider the new medical evidence in the context of any previous MAT, Job Capacity Assessment (JCA) or Disability Medical Assessment (DMA) recommendations
    • make a new recommendation and scan this to Document Tools
  • if the ARO still believes a qualifying impairment rating could be assigned, they will make a referral for a JCA using the relevant JCA referral Fast Note. See Job Capacity Assessment (JCA) Referral
  • the ARO will monitor for the return of the JCA
  • when the JCA is returned the ARO will either:
    • affirm the decision. Go to step 9, or
    • if not a manifest grant, refer for a DMA by completing a Disability Medical Assessment during appeal form. Scan it to the customer record, running the DMA referral guided procedure. See Disability Medical Assessment (DMA) referrals. The Disability Medical Assessment during appeal form is available on Resources
    • when a decision is made, go to Step 9

Note: the MAT process does not apply to DSP Medical Risk Based review cancellation decisions.

Rejection or cancellation based on Job Capacity Assessment (JCA)

A Service Officer has rejected the claim or decided to cancel DSP after a DSP Medical Risk Assessment and JCA. The ARO however, considers setting aside the rejection or cancellation decision. The ARO:

  • may discuss the decision with their Assistant Director where a rejection decision is being reviewed
  • must discuss the decision with their Assistant Director where a Medical Risk Based cancellation decision is being reviewed
  • has the option to seek advice or clarification from the Health Professional Advisory Unit (HPAU)
  • will email the Assessment Services Quality Team using the approved template if the content of the JCA report is incorrect. The JCA email template is available from Resources
  • can liaise with Assessment Services to discuss the intention to set aside the decision
  • refers the case for a DMA (if not a manifest grant) by completing a Disability Medical Assessment during appeal form. Scans it to the customer record and runs the DMA referral guided procedure. See Disability Medical Assessment (DMA) referrals. The Disability Medical Assessment during appeal form is available on Resources
  • monitors for the return of the DMA
  • makes a decision. Go to Step 9

Rejection or cancellation based on Disability Medical Assessment (DMA)

A Service Officer has rejected a claim or decided to cancel DSP (as a result of a DSP Medical Risk Assessment) based on a DMA. The ARO however, considers setting aside the decision. The ARO:

  • must discuss the decision with their Assistant Director
  • must seek Health Professional Advisory Unit (HPAU) opinion/additional evidence if HPAU consultation has not already occurred

Following the HPAU opinion/additional evidence, if the ARO decides to set aside the decision:

  • The ARO emails the Government Contracted Doctor Liaison Officer (GCDLO) advising the decision is to be set aside and includes feedback/reasons. They will use the Government Contract Doctor (GCD) email template available on Resources.

If the GCDLO does not contact the ARO within 2 days, the review will be completed.

When a decision is made, go to Step 9.

9

ARO Letter/Decision Statement + Read more ...

It is a legal requirement that AROs advise the customer of the outcome of a review in writing. This is done using a decision statement. It is also a legal requirement that AROs advise any affected/added parties of the outcome of a review. See 'Multiple parties affected by a decision' in the Additional considerations for AROs table on the Resources page.

Related decisions (that is, where more than 1 decision is incorporated in an APL system record) are covered in 1 decision statement. Unrelated decisions (that is, where there are separate APL system records) are covered in separate decision statements. Use the ARO Letter Template to create the decision statement. See 'ARO Letter' in the Document requirements table in Resources.

Note:

Separate decision statements are required for family assistance care percentage decisions and their associated debt decisions. See, 'Family assistance care decisions' in the Additional considerations for AROs table on the Resources page.

AROs must have their decision statement proof read by a peer (another ARO) before finalising the review. This is to ensure the decision statement meets required standards. For information about peer proof reading of ARO letters, see the Quality page on the Appeals Branch SharePoint. AROs need to record the details of the peer check in the ARO Progress DOC.

After the decision statement has been completed, go to Step 10.

10

ARO to finalise the review + Read more ...

Amalgamate APL system records when appropriate, see Scope of an ARO review in the Additional considerations for AROs table on the Resources page and Appeals (APL) system records.

Update the ARO Progress DOC. See the Requirements for documents used by AROs table in Resources.

Run the ARO Finalise Review Case script to record the outcome of the ARO review in the APL system and create an ARO Decision DOC. See ARO Decision DOC in the Requirements for documents used by AROs table on the Resources page.

Suggested text for ARO Decision DOC:

  • I have reviewed the decision of [dd/mm/yyyy] to [decision]
  • This decision was affirmed/varied/set aside because [reason]
  • Relevant law/policy: [references]
  • ARO decision uploaded to the customer record using form types ARODEC and AROEVD
  • Implementation instructions - [enter free text]
  • This decision has effect from [dd/mm/yyyy]

When updating the ARO Decision DOC for decisions requiring implementation:

  • include implementation instructions (see the Requirements for documents used by AROs table on the Resources page)
  • if a priority review, forward the ARO Decision DOC to the Appeals Support Team
  • if not a priority review, forward the ARO Decision DOC to the relevant site/team that made the original decision

To forward an ARO Decision DOC to the Appeals Support Team:

  • In the first Keywords: field, add the keyword ‘IMPL_PRI’. Overtype any keyword in this field.
  • Find a keyword in the list below that matches the decision. Add this keyword to the second Keywords: field. Overtype any keyword in this field.
    • IMPL_CAR (Carers)
    • IMPL_DBT (Debts except Families)
    • IMPL_DIS (Disability)
    • IMPL_EMP (Employment)
    • IMPL_FAM (Families, including debts)
    • IMPL_SEN (Seniors)
    • IMPL_STU (Student)
    • IMPL_OTH (Other)

Complete the ARO Finalise Review Case Review script.

Close all other DOCs that are no longer required when finalising the review.

If the ARO decision needs implementation and it is not a priority review, go to the Worklist in Work Optimiser, unassign the work item and re-categorise it to the appropriate work item.

Note: if the decision has not been implemented within 7 days and the customer contacts the ARO, complete the ARO to SME Escalation webform. See the Resources page for a link. The Escalations Team will take follow-up action to ensure the decision is implemented.

When the ARO Finalise Case Review script is complete, go to Step 11.

11

Upload and save decision + Read more ...

Upload the decision statement and ARO Notes to the customer record on the same day the review is finalised.

If Document Upload is not available, use a separator page to scan documents under the AROEVD category. See the Resources page for the Scanning separator page for AROEVD documents.

Note: scan any new evidence to the customer's record. The UNS008 separator page and scanning code is used for medical evidence. The AROEVD code is used for all other evidence and the Confidential header document used if necessary. Once scanned, store this evidence for 70 days (either in the office batch storage or with the ARO). The evidence should then be destroyed.

When the decision documentation is uploaded, go to Step 12.

12

Feedback to the Subject Matter Expert (SME), original decision maker or another officer + Read more ...

Use the Staff Feedback Tool to provide feedback to the SME, the original decision maker or to another officer involved in the decision and review where:

  • there is an identified learning need, or
  • the review was recorded incorrectly in the APL System, or
  • positive feedback should be given

13

Administrative Appeals Tribunal (AAT)(first review) request for statement + Read more ...

An AAT first review ARO statement request is added to the customer’s record by the External Review of Decision (ERoD) team. This will create a work item for action by the ARO.

AROs should check their Worklist in Work Optimiser twice daily for any AAT first review statement requests. The work item will be:

  • ERoD Referral (ZIAM_AAT1_DOC) for standard cases
  • ERoD Referral Priority (ZIAM_AAT1_DOC_URG) for priority cases

When a request is received, action the request in accordance with Evidence preparation for Administrative Appeals Tribunal (AAT) review hearing papers.
