Incoming NZ embargo schedules + Read more ...

For Centrelink International Services (CIS) staff

NZ sends the incoming NZ embargo schedule (customer listing) to Services Australia using the Autonomy Process Automation (APA). See Agreement Liaison Details (ALD) for more information on online transmission systems.

Access the embargo schedule and save to the NZ Secure local drive. Name it using the current list number and month. For example, ‘1712 JUN’.

  • select Conditional formatting > Highlight Cells Rules > Duplicate Values

If the duplicate CRNs belong to:

  • the same customer, combine the lines adding the 2 amounts paid
  • different customers, correct the incorrect CRN

If the schedule is not available, contact the NZ authorities to resolve.

For International Finance Team

When funds have been receipted, check that the receipted amounts match the amount on the embargo schedule.

Note: when comparing amounts, disregard the administrative processing fee.

  • No, go to Step 3

Receipts do not match the schedule + Read more ...

For International Finance Team

Contact the NZ pension authorities to resolve.

In the main embargo processing file, for customer records on the schedule where:

  • the embargo has failed, add a Comment against the relevant customers indicating ‘Failed embargo’ and remove the stated amount for that customer from the total amount on the schedule. Create a Fast Note on the relevant customer records:
    • Ext Detail, key ‘NZ Failed Embargo’
    • Text, key ‘NZ embargo has failed. Process arrears debt from NZ Service Decision on customer record as normal.’
  • monies will be received in future, add a Comment against the relevant customers indicating ‘Embargo on Next Schedule’ and remove the stated amount for that customer from the total amount on the schedule. Create a Fast Note on the relevant customer records:
    • Ext Detail, key ‘Embargo on Next Schedule’
    • Text key, ‘NZ embargo not received. Check to ensure embargo is received on next schedule’
    • place on hold for the date the next schedule is expected to be received

Convert the schedule into the main processing file + Read more ...

For International Finance Team

Open the Filemaker template and save as ‘FULL LIST yyxx MMM’ using the current list number.

Update the following list information:

  • period
  • schedule reference
  • date received

  • the spreadsheet will automatically calculate the AUD amounts and create the list totals
  • check the NZD and AUD totals against the amount received to ensure the data is complete and correct

Go to the ‘Review’ tab, select ‘Share workbook’ and ‘Allow changes by more than one user at the same time’ > OK > Save > Close.

5

Upload embargoes to the system and set up schedule + Read more ...

For Centrelink International Services (CIS) staff

After the NZ embargo schedule and receipts are received each month and the main processing file is on the local secure drive, set up a ‘Source’ file to run the ‘International DOC Creation’ script to create embargo Work Items.

The NZ Embargo DOC file contains the details the script uses to create a DOC on each customer record in the Source file. Update 2 files on the local shared drive:

  • G: > Software > Scripts > Local > International > B > BatchJobs > MagicKey > NZ Embargo DOC

Note: delete any previous ‘Results’ files from the last time the script was run.

Open the NZ Embargo DOC script file on the local drive and change the following variables in:

  • Document Header:
    • Receipt Date, key date the NZ embargo schedule was received
    • Ext Detail, change the list number only to YY/XX where YY is the current financial year and XX is the sequential schedule number
  • Variables:
    • Schedule Period From Date, key the start of the previous calendar month. Use the format DD/MM/YYYY
    • Schedule Period To Date, key the end of the previous calendar month. Use the format DD/MM/YYYY
    • Date of Payment, key the date the monies were received from NZ

Close the file and select Yes to save changes.

6

Prepare ‘Source’ file + Read more ...

For Centrelink International Services (CIS) staff

The ‘Source’ file contains the specific NZ arrears details for each customer on the schedule that will be included in the DOC. The data must be copied into an existing script file in a specific format.

Use the ‘NZ Embargo Source Filemaker’ spreadsheet to create the format required when data is copied from the schedule that adds the exchange rate. Go to the secure drive:

  • G: > TAS > HOB-188-Collins > NZ_Secure > Embargo > Incoming

Open the NZ schedule and the ‘NZ Embargo Source Filemaker’:

  • select all the data on the front worksheet in the NZ schedule:
    • select the diagonal arrow between ‘A’ and ‘1’ to select the whole worksheet
    • copy [Ctrl + C] and paste [Ctrl + V] all the data into the ‘RAW’ worksheet in the ‘NZ Embargo Source Filemaker’
    • close the schedule and select No. Do not save changes
    • select Cell C6 (under the date) and key in the exchange rate from the embargo receipt
  • go to the ‘Source’ worksheet, select Cell A1 and scroll down to the last row that contains data:
    • select Shift and the last cell to select all the data in the column
    • copy [Ctrl + C] the data
    • open the existing ‘Source’ file from the local drive in Notepad
    • select all the data in the file [Ctrl +A] and paste [Ctrl + V] the data from the spreadsheet into the file
    • close the ‘NZ Embargo Source Filemaker’ file and select No. Do not save changes

Close the ‘Source’ file and select Yes to save changes.

The files are now ready to run the script.

7

Create Work Items for NZ embargo schedule processing + Read more ...

For Centrelink International Services (CIS) staff

From Customer Record, run the International DOC Creation Script. The script will attempt to create a work item on each customer’s record and will record success or failure in a ‘Results’ file:

  • select NZ Embargo DOC > Batch Job option
  • ensure the script:
    • Start is from record 1, and
    • End is the Final Record
  • Accept the default ‘Results’ file of ‘1000’ records
  • ensure the file path is the correct directory on the local drive
  • select Next to run the script

If a script error occurs, it is most likely the CRN provided by NZ is incorrect:

  • check the ‘Results’ file to find the record number that failed
  • find the correct CRN using the NZ reference number (SWN) in the Record Number Search (RNS) screen
  • open the Source file, correct the CRN in the record number that had the error and Save changes
  • restart the script from the previous record. For example, if the error occurred on record number 152, restart the script from record 151

The script will create a DOC on the CRN with the Embargo list number.

For any other script failures, manually DOC the record with the following information:

  • Service Reason: FPS
  • Enquiry Type: OVS
  • Who: AGP
  • Tfr to: CRM
  • Receipt date: date of receipt of list
  • Keywords: NZEMBA
  • Extra details: NZ LIST (list number) EMB RCVD
  • Text:
    • schedule of New Zealand arrears payment from MSD (NZ) received for the period (Date)
    • include date of payment, customer name, arrears amount in NZD and AUD

Once all records have been uploaded and script is finished, delete the ‘Results’ file in the script secure drive.

Generate the arrears debt + Read more ...

Code the foreign pension arrears debt using the arrears amount and arrears period from the NZ Service Decision and the applicable system exchange rate from the Foreign Pension Details screen. Do not use the exchange rate from the embargoes schedule.

When coding the arrears period to ensure the debt is calculated for the arrears period correctly, on the Residence Savings Screen (RSS) for the:

  • Long Term Country Code, key AU
  • Date of Event, key the actual date the NZ Pension was granted

2

Compare arrears debt to NZ embargo amount + Read more ...

Official exchange rates the agency uses to convert foreign currency recorded in the system are not the same as the exchange rate used to convert an embargo payment we receive from NZ on any particular day. This means that the amount received from NZ will not reflect the amount of the arrears debt the agency calculates. Despite this difference, consider the customer’s debt to be fully repaid.

Under the Agreement with New Zealand, in Australia the rate of NZ pension is ‘capped’ to the Australian Notional Rate and NZ pensions are always treated as a direct deduction, even for autonomous customers. Therefore the amount of the NZ embargoed arrears should be 100% of the Australian debt amount.

There may still be cases where there are legitimate positive or negative adjustments remaining after the embargo. For example:

  • the embargo amount received from NZ is incorrect. For example, NZ recovered their own debt from the arrears, there may still be some negative adjustment remaining after the embargo is coded
  • the customer is a member of a couple and their partner is not in receipt of an income support payment or the customer has not authorised recovery of their partner’s debt. Half (50%) of the amount received will be embargoed and 50% will be issued to the customer after their debt is recovered

A tolerance of 5% difference between the arrears debt and the embargo amount (AUD value) is accepted as being solely due to exchange rates.

The main processing file will automatically calculate if there is a (+ or -) 5% difference. The recovered column will show:

  • Yes, if the difference is within 5%
  • No, if the difference is over 5%

Is the calculated arrears debt within 5% of the NZ embargo amount received?

  • Yes, consider the debt fully recovered, go to Step 4
  • No, go to Step 3

3

Check embargo amount + Read more ...

To ensure the arrears have been calculated correctly, check the arrears amount received from NZ matches the rate information provided on the original NZ Service Decision.

Multiply the NZ rates by the number of days in the arrears periods as provided in the NZ Service Decision and compare to the amount received in NZ dollars.

Check the agency’s arrears debt calculation is correct. See Calculating Centrelink entitlements and gathering information when investigating debts.

If the NZ arrears amount is established as:

  • correct, go to Step 4
  • not correct, contact the NZ authorities to clarify. If the NZ authorities:
    • will send the remaining amounts at a later date, add a comment (such as ‘Remaining arrears next schedule’) to both the Main embargo processing file and the arrears debt activity. Put the activity on hold for the date the next schedule is due. Procedure ends here
    • have requested return of amounts, contact the International Finance Team to arrange return of funds. See the Resources page table
    • confirm some NZ embargo amounts are remaining to recover for their arrears debt, go to Step 4
    • confirm that no NZ amounts remain for the debt, finalise the arrears debt as normal. See Table 1 of Comparable foreign payment lump sum arrears debts
    • will not send any further amounts, go to Step 4

4

Partner debts + Read more ...

If a member of a couple is granted a NZ pension and their partner receives an income support payment, the partner will also have a debt. The Agreement does not give the authority to recover another persons’ debt from an embargoed amount. The customer must provide express authorisation before we can also recover the partner’s debt from their arrears.

An authorisation is sent to the customer in the NZ pension claim pack. If received, the authorisation is scanned to the customer record and also noted in the relevant DOC when the NZ claim is sent overseas (OVS).

Is there a partner debt?

  • Yes, check the Document Tools and the OVS DOC for the authorisation. If the customer:
    • has not authorised the recovery of the partner’s debt from their arrears or not returned the authorisation form, go to Step 5
    • has authorised the recovery of their partner’s debt from their arrears, go to Step 6
  • No, go to Step 6

5

No partner authorisation, partner debt to be raised + Read more ...

Raise the partner’s half of the debt as per normal debt raising procedures. See Table 1 of Comparable foreign payment lump sum arrears debts.

The embargoed arrears can be used to recover the customer’s debt and residual arrears released to the customer, go to Step 6.

6

Raise the arrears debt and add embargo messaging + Read more ...

Before raising the debt:

  • check the ADEX explanation screen for debt information by selecting ‘W’ on the Debt List (OPDL) screen and selecting the http link
  • determine whether the debt is fully recovered according to the policy above
  • find the ‘Amount received in debt period’

Run the Debt Action script. Select:

  • in the ‘Category of income debt’ dialogue box, select ‘Lump Sum Arrears of Foreign Pension’
  • manually add the relevant embargo text in both the ‘DOC text screen’ and ‘Advice of Debt Display’ dialogue boxes.
  • manually add messaging in the Accounts Payable letter to advise debt is recovered via embargoed arrears. See the Resources page for approved text for each CUS/PTR debt scenario

Once finalised cancel the DMN/RCV activity to prevent the debt going to another debt team, as the debt needs to be recovered via embargo.

7

Coding the embargo recovery of the debt + Read more ...

Once the debt has been raised, the embargo recovery is recorded.

If the customer has authorised the recovery of their partner’s debt from their arrears, follow the same process for the partner’s debt.

Staff require security access to the Debt List (OPDL) screen.

On the OPDL screen, 'S'elect the debt to be adjusted to navigate to the Adjust Repayment (OPAR) screen. In the following fields:

  • Source, key 'POG' (Other Agency adjustment)
  • Amount $, key the amount to be recovered by embargo. If:
    • the debt is considered to be fully recovered according to the exchange rate policy above, the amount recorded must be amount of the debt, or
    • the amount received from NZ is not sufficient to cover the debt (for example the NZ authorities incorrectly used the arears to recover a pre-existing NZ debt and did not send enough to Services Australia), the amount recorded must be the amount received from NZ converted to Australian dollars. Advise the customer of the residual amount of debt left owing to be paid by the customer. Note: residual amounts less than $50 cannot be waived as the total debt amount was not less than $50
  • Date Paid, key the date of action
  • Reason, key Debt recovery by NZ embargo' (see the Resources page for approved text)

Finalise the activity.

Go to the Foreign Claim Summary (FGS) screen:

  • select the relevant NZ pension
  • ‘C’orrect the status/reason code from ‘GEP’ to ‘FIN-GRA’, and
  • Finalise the activity

In some cases, a valid excess amount may be remaining and needs to be issued to the customer. For example, they are a member of a couple and their partner does not receive an income support payment, in which case 50% of the arrears amount is sent to the customer.

The main processing file will automatically calculate if any excess is owed to the customer.

Is an excess amount to be paid to the customer?

  • Yes, go to Step 8
  • No, go to Step 9

8

Excess amounts and Vendors + Read more ...

Any valid excess amount after the embargo is coded must be paid to the customer through Essentials.

International Finance Team

At the end of each week:

  • check the main processing file for customers who have a payment owing to them (the ‘pay’ column will display an amount)
  • place all vendor requests into an ‘Action’ spreadsheet, and
  • update once they have created a Vendor in Essentials

CIS

Check the ‘Action’ file weekly for confirmed vendors:

  • access Essentials and issue the payment to the customer via SAP GUI
  • see Vendor codes and invoice instructions on the Resources page
  • update the ‘Action’ file with Vendor Doc number, Date and user ID
  • if the payment fails for any reason, contact the International Finance Team to resolve

9

Complete Embargo lists + Read more ...

Once all action is taken on the embargo, the details must be recorded on the main embargo processing file so the schedule can be reconciled before the next schedule is received:

  • access the Main embargo processing file.
  • record all details of the embargo for each record:
    • User, key staff logon ID
    • Date Assessed, key date the action was completed
    • CUS Type, key customer’s income support payment type (“FPS’ if none).
    • CUS Debt, key customer’s arrears debt amount.
    • PTR Type, key partner’s income support payment (NPS if none).
    • PTR Debt, key partner’s arrears debt amount.
    • Comments, record whether the embargo action is awaiting vendor, completed or if there are any issues encountered

Auto calculated fields:

  • CUS ADJ, key embargo amount coded for customer’s arrears debt. Note: in accordance with the policy on NZ pension, this is usually 100% or 50% of the embargo amount and not related to the arrears debt
  • CUS REC, key Yes if recovered, or No if there is residual debt
  • PTR ADJ, key embargo amount coded for partner’s arrears debt. In accordance with the policy on NZ pension, this is usually 100% or 50% of the embargo amount and not related to the arrears debt
  • PTR REC, key Yes if recovered, or No if there is residual debt
  • Pay, excess amount paid to the customer (blank if none)

Run the ‘International DOC Creation’ script and select ‘WFI – NZ Embargo’ hotkey to record all details of the embargo action. Close the DOC once completed.

10

Finalising the embargo + Read more ...

The International Finance Team will access the main embargo processing file to prepare lists of payments required and journal results. Once all action is complete for a CRN, the International Finance Team will lock the CRN so no further changes can be made.

Outgoing NZ embargo schedules + Read more ...

At the beginning of each month, the Group Deduction Notice (GDN) report of all individual embargoed amounts from new claims of Australian pension for the previous month is sent to CIS’ positional mailbox.

The GDN includes the totals of all bulk amounts deposited into the embargo holding account. As payments are based on the international 4 weekly payment cycle, there is usually only one deposit per month and two for one month each year. However, there may be other small deposits from regular pay strips in local environments.

2

Receipt all deposits + Read more ...

Receipt all amounts that are deposited in the embargo holding account. Check:

  • the total amount on all receipts matches the total amount on the GDN
  • the NZ Embargo CRN in Customer Record matches the information on the GDN

3

Authorise payment to NZ + Read more ...

Complete the Request to Pay (NZ outgoing embargo) form to authorise payment to NZ,

  • reformat the GDN:
    • delete all information above the 'Customer name' row
    • insert 'SCH ID: AMMMYYYYAU' where MMMYYYY is the month and year of the schedule
    • insert title 'CENTRELINK ARREARS SCHEDULE'
    • where necessary, delete all rows between payment strips and amend the overall total to reflect the amount to be paid to New Zealand
    • insert the exchange rate and total amount in NZD, as advised by the International Finance Team
  • save the GDN to the local secure drive as 'AUS ARREARS SCHEDULE YYYYNN' where YYYYNN is the current year (numeric) and month (numeric)
  • access the electronic liaison system with New Zealand
  • attach the schedule to an 'Australian Arrears Schedule' liaison form and send to New Zealand

Send funds to nominated NZ account by the 15th day of each month.
