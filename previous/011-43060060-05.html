This page contains information on accessing the Operational Statistics System (OSS).

On this Page:

OSS screen access

Control lines for accessing OSS

This table lists the control lines to access the OSS.

Control line

Explanation

Transaction identifier

The transaction identifier consists of 4 characters (the same transaction identifiers used for Medicare OSS):

  • character 1
    • = Operational Statistics System
  • character 2
    • N = Medicare
  • character 3, either:
    • O = Operator
    • B = Branch or Source Office Manager
    • R = Regional or Smart Centre Manager
    • S = State Office Manager
    • A = Central Office (Australia totals)
  • character 4
    • I = Initial enquiry

Modifier

Either:

  • SBA
  • PHR

The modifier used to tell the system return billing agent statistics or health fund statistic.

D/M

Either:

  • D = daily
  • M = modifier

Used to determine the period.

Source Office

Source office codes are assigned to claims when they are background assessed. The first character identifies the state followed by the region identifier. For example, AJ = A (for NSW) followed by J (regional identifier).

  • Offline transaction source office codes have been set up specifically for simplified billing claims. See the Background assessed simplified billing claims table.
  • Online transaction source office codes (pend processing) are recalled by using any source office code.

Date

The dates the statistics relate to any valid date in the format DDMMYY or date range with the dates separated by a comma.

Note: 00MMYY is used when months of statistics are required.

Back loaded modifier

Valid value B.

Used to return statistics for claims that originated in another source office but were finalised by the source office in the control line.

OSS processing and work on hand screen field descriptions

Note:

Within the:

  • simplified billing system the originating source office is the transmitting source office for the claim such as AJAP, CDBS, EAAP, IAAP and KAAP
  • health fund system the originating source office is the transmitting source office for states such as AJEF, CDFP, EAEF, GAEF, IAEF and KAEF

Claims statistics by source office code

Recalling claims statistics for source office codes

This table lists information displayed in the work on hand processing screen fields when recalling claims statistics by source office code.

Column name

Description

This location

Productivity figures when the originating source office and processing source office are the same as that on the control line.

Other location

Productivity figures when the source office keyed on the control line is not the originating source office but is the processing source office.

Total

The aggregated total of This location and Other location figures.

Work on hand

Pat/Prov Segments

The number of PAT/PROV segments for background assessing and pend that are still on hand.

Lines

The number of lines for background assessing and pend.

Unresolved Lines

The number of lines not yet pend processed. This field is only available in the pended column.
