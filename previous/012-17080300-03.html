This page contains information on how an Assessing Officer (AO) is to complete assessment of a Medical Indemnity Online Claiming (MIOC) High Cost Claim Indemnity Scheme (HCCS) initial or subsequent payment application or a Section 38 response to determine if the HCCS payment application meets the relevant eligibility requirements under the HCCS scheme.

Assessing MIOC HCCS Initial and Subsequent applications

Assessing MIOC HCCS S38 Responses

Assessing MIOC HCCS Initial and Subsequent applications

Table 1: This table explains how an Assessing Officer (AO) completes an assessment of a MIOC HCCS initial or subsequent payment application.

Step

Action

1

Open an application for assessment + Read more ...

Complete actions prior to assessing an application.

Open the Medical Indemnity Online Claiming (MIOC) system.

From the Home screen > select Approvals from the top task bar.

Note: Assessing Officers can also refine the list of applications using the filters (example: Insurer, application type, scheme etc.) above the list of current applications.

  • Select a HCCS application with a recommended status of Quality assurance
  • Select the High Cost Claim Scheme (HCCS) radio button under Medical indemnity scheme
  • Select the Quality assurance radio button under Recommended action
  • Select View for the application required

The screen displays a summary of the application and all information entered by the applicant, including case notes. Any warnings associated with the application will also be displayed here. An internal user can undertake the following actions:

  • Download PDF of the application
  • Withdraw the application on the insurers behalf if needed
  • Recommend action to the delegate once an assessment has been undertaken, that is:
    • Approve for payment
    • Request further info from the insurer, or
    • Deem not payable

2

Make a note that the application is being assessed + Read more ...

MIOC will allow multiple users to work in an application at the same time. A case note needs to be created to let other users know that you are assessing the application.

  • Select Notes from the drop down menu at the top of the screen. This will take the user to the Notes section in the application
  • Select Add new [note]
  • Select Other from the drop down Subject menu
  • Enter “<initials> is assessing this application” in the Note text box
  • Select Save
  • Add the claim ARN and your initials under the ‘Assessed’ column in the ‘MIOC Work on Hand’ document located in the shared drive at
    G:\med_indemn\Claims by ARN\Work on Hand

Note: if another user has already made a note to this effect, select Approvals from the task bar at the top of the screen and select another application to assess. Repeat the process at steps 1 and 2 above.

3

Review any case notes the applicant has included in the application + Read more ...

The applicant may have added some notes to their application to to be aware of.

  • Select Notes from the drop down menu at the top of the screen. This will take the user to the Notes section in the application
  • If notes have been added, select the collapsible section beside the ‘Subject’ of the note to view the note
  • Select Supporting documents from the drop down menu at the top of the screen. This will take the user to the Supporting documents section in the application
  • Select the collapsible section beside the listed Document to open the Note field and check for any notes that may have been added

4

Create a folder in the drive for assessment documents + Read more ...

Open the ‘Claims by ARN’ folder located in the shared drive at G:\med_indemn\Claims by ARN.

When a claim is submitted in the MIOC system, the system will auto-generate an application number which is Services Australia’s unique claim reference number.

For an initial application:

  • Select the folder with the ARN number range that the ARN application number falls within
  • Create a new folder using the format ARN[Application number][Insurer identifier letter]. For example, ARN6000U. The Insurer identifier letter is the letter assigned to the insurer (see Vendor codes on the Resources page)

For a subsequent application:

  • Select the sub-folder for the ARN. For example ARN6000U
  • Create a new sub folder, within the relevant claim/ARN sub folder, using the format ARN[Application number]-[Insurer identifier number and letter]-[Scheme identifier letter]. For example, ARN6000-1U-H. The Insurer identifier number is the number of the payment application relevant to the claim, i.e. 1 for first application, 2 for second application, and so on. The Scheme Identifier letter represents the scheme that the claim is being made under:
    • H for HCCS
    • R for ROCS
    • I for IBNR
    • IH for IBNR/HCCS
    • RH for ROCS/HCC

5

Open and save a HCCS MIOC Assessment worksheet + Read more ...

  • Open the HCCS MIOC Assessment worksheet template located in the shared drive at:
    G:\med_indemn\Claims by ARN\Assessment\MIOC assessment\HCCS

Save the document as ‘HCCS MIOC Assessment Worksheet_ARN[Application number]-[Insurer identifier number and letter]-[Scheme identifier letter]’, for example ‘HCCS MIOC Assessment Worksheet_ARN6000-1U-H’ in the relevant application folder.

6

Complete the HCCS MIOC Assessment worksheet + Read more ...

The Assessment worksheet is broken down into the following Sections to correspond with the sections completed by the applicant in the MIOC application:

  • Claim Details
  • Incident Details
  • Insured Person’s Details
  • Policy Details
  • Settlement and Plaintiff Costs
  • Defence Costs (and MIOC Expense Reconciliation)
  • Application Finalisation
  • Total Application Costs

At the top of the assessment worksheet, complete the sections for ARN, Insured person’s name (taken from the Insured person’s details section in MIOC) and Claim ID (taken from the Claim details section in MIOC).

Note: for a subsequent application lodged against a migrated application, Assessing Officers will need to check the Insured Person’s name in HPE Content Manager (TRIM).

Assessing Officers can copy and paste information from MIOC into the document.

Follow the instructions and prompts noted on the assessment worksheet to complete the assessment process and determine an assessment outcome.

Complete all relevant questions of the assessment worksheet, selecting the Answer column checkbox that corresponds with the information provided in the MIOC application. Check that the evidence provided supports the information stated in the MIOC application.

MIOC determines what supporting evidence or documents are required based on the applicant’s answers to the questions.

All supporting documents are uploaded by the applicant in MIOC.

  • Select Supporting documents from the drop down menu at the top of the screen. This will take you to the Supporting documents section in the application
  • Select Download beside the document you want to view and Open

The applicant may have supplied more than 1 relevant document as evidence of the information provided, or the same document for different categories of evidence.

For each question on the worksheet, locate the supporting document in MIOC, note the pdf/word document details in the Supporting Evidence location column of the worksheet. Note any relevant comments and/or discrepancies in the Comments column of the worksheet, as guided by the prompts and instructions on the worksheet.

Question 1: is this an Initial application?

  • Answer the question by checking the ‘Yes’ checkbox for an initial application and move to question 3, or ‘No’ for a subsequent application and complete Question 2
  • Where the application is a subsequent application with no changes from the initial application (as indicated in MIOC under Claim Details), check the ‘No’ checkbox at question 2. Confirm all details against the previous applications for questions 3 to 13, noting any discrepancies, and go to question 14
  • For an initial application, or a subsequent application where changes to the initial application have occurred, select the ‘Yes’ checkbox at question 2 and complete Questions 3 to 26

Questions 2 to 10: For the Claim Details, Incident Details and Insured Person’s Details sections of the assessment worksheet, note the relevant information provided in the MIOC application on the assessment worksheet.

For example: at Question 6, note the Date of Notification (DoN) in the Comments column of the worksheet, open the relevant supporting document provided in MIOC and locate the specific details relating to notification of the claim to the insurer. Record the document Filename and Page number/s in the Supporting evidence location column on the worksheet. Determine if the information provided in the MIOC application is correct and supported by the evidence provided. Note any relevant findings or discrepancies in the Comments column on the worksheet.

Supporting Evidence:

  • The evidence of the insurer approving indemnity of the insured person can be in the form of a Claims Verification Form (CVF), contract of insurance/policy schedule or a letter or email from the insurer
  • The Date of Incident (DoI) can be identified from court documents, such as the Statement of Claim (SoC), Notice of Claim (NoC), or a letter from the member stating details of the claim
  • The Australian state, territory or external territory in which the incident took place can be identified from court documents or a letter from the member stating details of the claim
  • The DoN can be evidenced by date stamped correspondence from the insured person to the insurer, or by a telephone activity log evidencing verbal notification
  • The evidence that the insured person was a practitioner of an eligible medical practitioner, confirming the member’s specialisation, may be identified from court documents, or other supporting documents, such as the CVF or policy schedule

Questions 11 to 17: Check the appropriate checkboxes in the Answer column of the worksheet and note the relevant Policy Details, taken from the MIOC application, in the Comments column of the worksheet. Open the supporting document provided in MIOC. Check that the details provided in MIOC are supported by the evidence provided. Record the location of the supporting evidence on the worksheet and note any comments or discrepancies.

Check the following:

  • the policy date range covers the DoN
  • the retroactive date of the policy is prior to the DoI
  • the limit of the insurance for the claim
  • there are no special conditions or sublimits applicable to the claim
  • any policy deductibles have been deducted from the claim

Questions 18 to 20: Answer Settlement and Plaintiff Costs questions. Note the total amount, amount claimed and any apportionment details, and record the relevant amounts and details on the worksheet for Settlement and Plaintiff Costs. Check that the details provided in MIOC are supported by the evidence provided. Record the location of the supporting evidence on the worksheet and note any comments and discrepancies.

Questions 21 to 23: If defence costs are being claimed, perform a reconciliation of the costs being claimed.

  • Open the ‘MIOC Expense Reconciliation template’ located in the shared drive at:
  • G:\med_indemn\Claims by ARN\Assessment\MIOC assessment
  • Save the document as ‘MIOC Expense Rec_ARN[Application number]-[Insurer identifier number and letter]-[Scheme identifier letter]’, for example ‘MIOC Expense Rec_ARN6000-1U-H’, in the relevant application folder
  • Copy and paste the invoice information listed under Defence Costs details (under invoice date, invoice number, amount claimed and cost shared) from MIOC into the spreadsheet
  • Check that each defence cost being claimed is supported by the invoice or evidence provided and that the agency’s calculation of the total defence costs reconciles with the total defence costs stated in the MIOC application
  • When all invoices have been checked, copy the expense reconciliation information from the template to the end of the assessment worksheet under MIOC Expense Reconciliation. Select the relevant checkbox to indicate the invoices have not been claimed in a previous application or if it is an initial application and doesn’t apply

  • Where a financial discrepancy is identified, provide relevant comments in ‘Additional Notes’ under Recommendation in the assessment worksheet. For example, if GST has been claimed in an invoice, the discrepancy may result in a letter under section 38 of the Medical Indemnity Act 2002 being sent to the applicant to clarify the discrepancy.

Questions 24 to 25: Answer the Application Finalisation questions. If there is no evidence to the contrary, select the ‘No’ checkbox and note ‘No evidence to contradict answer’ in the Comments column.

Question 26: Check that the amount calculated at ‘Total amount claimed in this application’ in MIOC reconciles with the MIOC Expense Reconciliation spreadsheet. Note the total amount and claimed amount in the Comments column on the assessment worksheet under Total Application Costs.

Note any issues, discrepancies or comments in the Additional Notes section of the worksheet.

Create a case note in MIOC in the Notes section detailing any issues, discrepancies or comments from the assessment, particularly if it could result in the application being deemed not payable or where further information is required.

7

Creating case notes where issues are found during assessment + Read more ...

If there is information about the assessment that the approver (delegate) needs to be aware of, create a case note in MIOC in the Notes section.

This must be done for every application where there is a recommendation of Deemed not payable or Request further info.

The note must be detailed enough to enable the approver (delegate) to understand the reasons for your recommendation.

Assessing Officers can also add a note if the recommendation is Approve but this is not mandatory.

To create a case Note:

  • Select Notes from the drop down menu at the top of the screen. This will take the user to the Notes section in the application
  • Select Add new [note]
  • Select the relevant section from the drop down Subject menu (for example, for notes related to the policy, select Policy Details)
  • Enter the relevant details in the Note text box (noting there are character number and symbol limitations, for example % is not accepted)
  • Select Save

To finalise the assessment and make a recommendation to the delegate, see Medical Indemnity (MI) Schemes finalising assessment

HCCS QA and assessment worksheet - initial application

Table 1: This table explains how AOs are to complete the HCCS QA and assessment worksheet for an initial application.

Step

Action

1

Complete QA check of registration + Read more ...

AOs must conduct a detailed QA of the information the Registering Officer (RO) has keyed in relation to the:

  • HCCS registration form and worksheet
  • payment application
  • calculation spreadsheet
  • HP Records Manager, and
  • system checks

as outlined in Medical Indemnity (MI) Schemes registration and Medical Indemnity (MI) Schemes registration worksheets.

Complete the QA of the registration. If:

  • any errors are identified in the registration process, hand the claim back to the RO for correction. Once the RO has amended any errors, undertake the QA check again before progressing to assessment.
  • no errors are identified for:
    • an initial HCCS payment application, go to Step 2
    • a subsequent HCCS payment application, go to the HCCS assessment worksheet - subsequent HCCS application table

2

Initial HCCS payment application assessment worksheet + Read more ...

Open the HCCS assessment worksheet for an initial HCCS payment application template, located at Medical Indemnity > Claims by ARN > Assessment > HCCS_Assessment Worksheet_Template

Save the template by selecting File then Save As:

  • filename, 'ARNXXXX-XX-X_Assessment worksheet
  • location: Medical Indemnity > Claims by ARN > ARN XXXX-XX-X (find the relevant ARN folder)

Complete the assessment worksheet and where possible, use the information on the payment application:

  • Application Reference Number (ARN), use information from the registration worksheet
  • Claim Reference number, insert the reference number the insurer uses, see page 1 of the payment application
  • Linked Files, use information from the registration worksheet

Go to page 2 of the assessment worksheet and complete the Applicant field. The applicant is the MI insurer who is applying for payment. See information located on the payment application.

Note: for all questions, key the folio number(s) of where the relevant/supporting document(s) are located in the ‘Folio #’ column.

At any time through the assessment if it is deemed the claim is not payable, Service Officers must consult with the Assistant Director (AD). In some circumstances, the AD will consult with the Director and they may need to obtain legal or policy advice to ensure the claim is not payable.

3

Questions 1 – 4 of the assessment worksheet + Read more ...

Question 1, review the declaration to ensure it has been correctly completed by the applicant:

  • tick the appropriate box on the assessment worksheet
  • key in relevant folio number/s

Question 2, check if indemnity has been approved. To substantiate that the member/policy holder has been granted indemnity in relation to this claim:

  • the insurer will need to provide evidence, such as a Claims Verification Form (CVF), policy schedule and/or a letter or email from the insurer confirming indemnity was approved.
  • tick the appropriate box on the assessment worksheet
  • key in relevant folio number(s)

Question 3, identify the member’s/policy holder’s specialty at date of incident, and:

  • key the 6-letter HCCS Specialist Code from the HCCS payment application onto the assessment worksheet, see the Resources page
  • key the code description in the Comments field
  • tick the appropriate box on the assessment worksheet
  • key in relevant folio number(s)

Note: if the member’s specialisation has not been provided on the HCCS payment application, refer to other supporting documents such as the CVF or policy schedule to determine the correct speciality code. If the correct code cannot be accurately determined, highlight the question as further information may be required.

Question 4, identify where the incident occurred:

  • tick the appropriate box on the assessment worksheet
  • key the state in which the incident occurred
  • key in relevant folio number(s)

4

Questions 5 - 6 of the assessment worksheet + Read more ...

Question 5, identify the date the applicant was notified of the incident(s) giving rise to the claim and/or was notified of the claim. Key the:

  • date of notification (DON). For more information on DON see the Resources page
  • tick the appropriate box on the assessment worksheet
  • key in the relevant folio number(s)

Note: check that the date of notification on the application and the supporting evidence. The date of notification can be evidenced by date stamped correspondence from the member to the insurer, or by a telephone activity log evidencing verbal notification. The Resources page contains a link to the HCCS Overview around eligibility for the Date of Notification under the HCCS Scheme.

Key any conflicting evidence regarding the date of notification in the Comments section. This will be needed to be documented in the file note at a later step and possibly result in a letter under section 38 of the Medical Indemnity Act 2002 being sent to the applicant to clarify the discrepancy. The Resources page contains a list of frequently asked questions and answers about section 38.

Question 6, identify the applicable HCCS threshold, this is based on the date the applicant was notified of the claim (see Question 5):

  • tick the appropriate box to represent to the correct HCCS threshold
  • key the date the applicant was notified of the claim as this determines the relevant threshold that applies to this claim
  • key in the relevant folio number(s)

5

Questions 7 - 8 of the assessment worksheet + Read more ...

Question 7, identify if there are any other parties involved in the claim.

If the answer is ‘Yes’:

  • the ARN of any linked claims. Note: if an application has not been submitted for the other parties, leave blank
  • key names of all other parties associated with the claim
  • key how many parties are involved in the apportionment
  • key the member’s percentage of total liability of the settlement costs and plaintiffs costs
  • identify any HCCS/non HCCS apportionment between parties
  • identify whether independent apportionment advice has been provided
  • key in the relevant folio number(s)

If the answer is ‘No’:

  • key in the relevant folio number(s)
  • continue to Question 8

Note: see details in payment application (apportionment and cost sharing arrangements section, defence costs), court documents (statement of claim), notice of claim, judgment or deed of release.

Question 8, check if the HCCS amount sought reconciles with the breakdown provided by the applicant:

  • tick the appropriate box
  • key the amount sought by the applicant on the payment application
  • key in the relevant folio number/s

Where a financial discrepancy is identified for example GST has been claimed in an invoice, this will need to be noted in the file note (later step) and possibly result in a letter under section 38 of the Medical Indemnity Act 2002 being sent to the applicant to clarify the discrepancy. The Resources page contains more information on GST and Medical Indemnity.

See the References page for a link to the Act.

6

Questions 9-12 of the assessment worksheet + Read more ...

Question 9, check the CVF and policy documentation to identify if the member has any special conditions as part of their MI insurance policy, such as a deductible or restriction:

  • tick the appropriate box
  • key the details of any special conditions in the Comments field (if applicable)
  • key in the relevant folio number(s)

See the Resources page for more information about CVF.

Question 10, check if the sum of qualifying payment(s) exceeds the HCCS threshold amount indicated in Question 6 of the assessment worksheet:

  • tick the appropriate box
  • key in the relevant folio number(s)

Question 11, check if the amount of the practitioner’s indemnity is sufficient to cover the total amount of the HCCS:

  • tick the appropriate box
  • key the indemnity limit in the Comments field (for example, $20,000,000)
  • key in the relevant folio number(s)

Note: this information will be located on the CVF, policy schedule and/or a letter or email from the insurer confirming that indemnity was approved.

Question 12, check the date of the incident:

  • tick the appropriate box
  • key the incident date(s) in the relevant field
  • key in the relevant folio number(s)

Key any conflicting evidence regarding the date of incident (DOI) in the Comments section and ensure the retroactive date on the policy is before the DOI. For more information on DOI see the Resources page This will need to be addressed in the file note at a later step and possibly result in a letter under section 38 of the Medical Indemnity Act 2002 being sent to the applicant to clarify the discrepancy.

Note: this information will be located in court documents such as the Statement of Claim, Notice of Claim or in a letter from the member stating details of the claim. The date of incident or date range of a series of related incidents, will also be included on the HCCS payment application.

7

Questions 13 – 15 of the assessment worksheet + Read more ...

Question 13, determine if the claim relates to an incident or a series of related incidents that occurred in the course of, or in connection with the practice of a medical profession by the member:

  • tick the appropriate box
  • key in the relevant folio number(s)

Note: this information will be located in the application, court documents such as Statement of Claim, Notice of Claim, or in a letter from the member stating the details of the claim.

Question 14, determine if the claim against the member has been finalised:

  • tick the appropriate box
  • key in the relevant folio number(s)

Note: this information will be located in court documents such as the Settlement documents, Judgment, Deed of Release or Notice of Discontinuance. Check that this question has also been answered on the HCCS payment application.

Question 15, check if the applicant has been paid and/or is liable to pay an amount in relation to the claim:

  • under a contract of insurance with the policy holder, or
  • under discretionary cover, incident-occurring based cover, retroactive cover, indemnity arrangement between it and the member:
    • tick the appropriate box
    • key in the relevant folio number(s)

See the Resources page for more information.

8

Questions 16 – 17 of the assessment worksheet + Read more ...

Question 16, check if the applicant has paid and/or is liable to pay the amount:

  • in their ordinary course of business, or
  • if an externally administered body corporate has paid an amount or is liable to pay a provable amount in the ordinary course of its business as if it were not under external administration:
    • tick the appropriate box
    • key in the relevant folio number(s)

Note: this information will be located on the HCCS payment application and is part of the statutory declaration.

Question 17, determine if the incident or some of the incidents to which the claim relates, occurred in the course of the provision of treatment to a public patient in a public hospital.

Note: any incidents that are occur in the treatment of a public patient in a public hospital are not payable under the HCCS.

Evidence submitted with the claim, such as the Statement of Claim or Notice of Claim, can be used to determine if the incident occurred in the treatment of a private patient.

  • tick the appropriate box
  • key in the relevant folio number(s)

If the answer to Question 17 is:

  • Yes:
    • key the name of the hospital or the location where the incident occurred
    • tick the box to indicate whether the hospital was public or private. The Resources page contains further information on determining public vs private
    • in the Comments field, key the address where the incident occurred
    • ensure any amounts attributable to the treatment of a public patient in a public hospital have been deducted from the total amounts claimed in the HCCS payment application
  • No, move on to the next part of the form

9

Finalise HCCS assessment worksheet and make recommendation to delegate + Read more ...

Go to page 1 of the assessment worksheet. Based on the responses to the question answered in the assessment worksheet, answer the Yes/No question in the middle of the page. Based on those responses, make a recommendation to the delegate to either:

  • deem the claim payable under the HCCS:
    • in the Assessment/Audit Comments field, key in capital letters, 'ALL CRITERIA MET - OK TO PAY'
  • Recommend a section 38 letter be sent:
    • in the Assessment/Audit Comments field, key in capital letters 'SECTION 38 LETTER REQUIRED'
  • deem the claim not payable under the HCCS as it has not met the HCCS eligibility requirements:
    • in the Assessment/Audit Comments field, key in capital letters 'DEEMED NOT PAYABLE'

In the Provisional recommendation section, complete all the remaining fields including details of the AO:

  • save the worksheet. Do not close the worksheet as it will be required for finalising the HCCS payment application
  • go to Finalising assessment of Medical Indemnity (MI) payment application

HCCS assessment worksheet - subsequent HCCS payment application

Step

Action

Open the HCCS assessment worksheet for subsequent claims + Read more ...

Save the template as File > Save As:

  • filename, 'ARNXXXX-XX-X_Assessment worksheet sub
  • location: Medical Indemnity > Claims by ARN > ARN XXXX-XX-X (find the relevant ARN folder)

2

Check subsequent application Questions 1-4 + Read more ...

Question 1, review the declaration to ensure it has been correctly completed by the applicant:

  • tick the appropriate box on the assessment worksheet
  • key in relevant folio number(s)

Question 2, determine if the claim against the member has been finalised:
