Identifying the problem with the payment summary

Table 1

Step

Description

1

MFU tax activity + Read more ...

Is the MFU related to a previous corrected and cancelled MFU activity?

  • Yes, go to Table 2
  • No. For:
    • Compliance and debt operations (CDO) staff, go to Step 2
    • Payments ICT Service Desk staff, go to Step 3
    • non-CDO staff, go to Step 4

2

Determine what MFU tax activity is required + Read more ...

For debt processing staff, MFU tax activities due to debt processing (including those customer records with the suppression indicator set to yes) are assigned within Customer First Workload Manager (CF WLM). Debt processing staff will be able to locate these work items for action by accessing their worklist in the WorkOptimiser.

P&I Operations will provide instruction on locating MFU tax activities for action. If the keyword is:

  • TXEOFY and TXNEG (for EOFY) or TXAUTO and TXNEG (for recalculation), action is required for a payment summary with a negative amount. Go to Step 5
  • TXEOFY and TXSUPP, go to Step 6

3

Find MFU activity + Read more ...

Only authorised staff will have access to the AMSY/CRFF Search (L3AS) or AMSY/CRFF National Search (L3AN) screen within ISIS to locate any tax MFUs.

For authorised staff, go to the L3AS screen. To locate and action MFU activities for:

  • Payment Summary not issued - invalid payment links (Message ID l018TX), in the following fields, key:
    • Class, PEN, NSS, ABY, PGA etc
    • Kwds, TXEOFY* or TXEOFYnn and/or TXLINK* or TXLINKnn, go to Step 7
  • Payment Summary not issued - Adj period mismatch (Message ID l019TX), in the following fields, key:
    • Class, PEN, NSS, ABY, PGA etc
    • Kwds, TXEOFY* or TXEOFYnn and/or TXADJ* or TXADJnn, go to Step 8

Note: where a keyword ends with 'nn' this represents the financial year. For example TXEOFY18

4

Determine what action needs to be taken for the MFU tax activity + Read more ...

MFU tax activity (message I014TX) is created where the taxable entitlement, non-taxable entitlement, Remote Area Allowance (RAA) or tax deducted amount is less than zero. It alerts debt processing staff that the payment summary cannot be issued due to a negative amount(s) in the tax details.

CDO will allocate MFU tax activities to non CDO debt raising teams where the debt requires corrective action. This is usually as a result of an incorrect debt raised for the current financial year.

CDO will reclassify MFU tax activities within Customer First Workload Manager (CF WLM) when allocating to non-CDO debt processing teams. Non CDO debt processing teams will be able to locate their work items on their worklist in the Work Optimiser:

  • follow the instructions in the TAXMFU Referral Fast Note attached to the MFU tax activity
  • if review of the MFU tax activity identifies specialist access on the Tax Payment Summary (TXGS) screen is required to correct the debt, update TAXMFU Referral Fast Note with investigation notes and return to CDO for action

To identify reclassified MFU tax activities in Work Optimiser, key the relevant keywords in the Attributes field:

  • I014TX (for negative amount MFU tax activity)
  • TXEOFYnn and TXNEGnn (for End Of Financial Year) or
  • TXAUTOnn and TXNEGnn (for recalculation)

Note: where a keyword ends with 'nn' this represents the financial year that is, TXEOFY18

Procedure ends here.

5

Payment Summary not issued - invalid amount (Message ID l014TX) + Read more ...

This MFU activity is created where the taxable entitlement, non-taxable entitlement, Remote Area Allowance (RAA) or tax deducted amount is less than zero. It alerts debt processing staff that the payment summary cannot be issued due to a negative amount(s) in the tax details. This is usually as a result of an incorrect debt raised for the current financial year, or necessary tax adjustments not being made via the Tax Adjustment (TXAJ) screen for compensation debts to correct both customer and partner records.

If the:

  • MFU is unable to be completed refer the case to Technical Peer Support (TPS) / operational leader
  • issue cannot be resolved, report it to the ICT Level 2 Service Desk by completing the Tax MFU form on mySupport

Note: the payment summary cannot be issued until the negative amounts are corrected.

For instructions on how to identify the problems in the payment summary calculation, go to Step 9.

6

Payment Summary not issued - auto issue suppressed (Message ID l015TX) + Read more ...

This activity will be created at end of financial year processing where the Suppress Auto Issue Y/N: indicator has been set to Y on the TXGS screen for a customer. The activity is created under the logon ID of the person who originally set the indicator. The MFU is usually created to remind staff that they may need to amend tax details and reissue the payment summary to the customer.

Note: this MFU is only a reminder to staff and will not prevent the payment summary from being issued.

  • Select the activity, the TXGS screen will show why the automatic issue was suppressed and what action is required
  • Make changes to the tax data if required
    • Key ‘C’ against the summary to be changed
    • Select ‘Continue’
    • Make required changes on the Tax Payment Summary Details (TXGD) screen
    • Select ‘Continue’
  • Re-issue the payment summary on the TXGS screen:
    • Key ‘I’ against the summary to be issued
    • Select ‘Continue’
    • Key ‘Y’ in the Issue Group Cert/Statement field
    • Select ‘Continue’
  • Once actioned complete the MFU activity
  • Record details on a DOC

Procedure ends here.

7

Payment Summary not issued - invalid payment links (Message ID l018TX) + Read more ...

This MFU is created where the calculation routine was unable to link regular, irregular and one-off payments to a correct component and is created to alert Payments ICT Service Desk to a problem with the group's' in the pay file not linked correctly, affecting the calculation of the customer's payment summary.

Note: this MFU tax activity will not stop the issue of the payment summary but will need to be followed up by the ICT Service Desk.

For instructions on how to identify the problems in the payment summary calculation, go to Step 9.

8

Payment Summary not issued - Adj period mismatch (Message ID l019TX) + Read more ...

This MFU is created where an adjustment (either manual via TXAJ screen, automatic as a result of BTR (Benefit Transfer) processing, or advance adjustments) is recorded for a period where the customer has not received payments. It will alert Payments ICT Service Desk staff to a problem with negative amount(s) in the calculation of the customer's payment summary.

Note: the payment summary cannot be issued until the adjustment error is corrected.

For instructions on how to identify the problems in the payment summary calculation, go to Step 9.

9

Identify the problem + Read more ...

Tax system processing has been designed to set exceptions in the tax calculation in a priority order and create the appropriate MFU tax activity where necessary. Only 1 MFU activity will be created per record. Staff should check for the possibility of more than one exception in the tax calculation when a MFU activity is generated.

To determine the underlying issue:

  • go to the TXGS screen and check the specified line for the financial year indicated in the keywords
  • select the Payment Summary line of the financial year required and press the Select (S) button (key 'S' if in the Legacy system) and the Tax Payment Summary Details (TXGD) screen will be displayed. The TXGD screen displays the details of the calculated payment summary for the specific financial year
  • note the period and total of the negative amount
  • go to the Tax Year To Date Totals (TXYTD) screen
  • key the period of the financial year and 'ALL' in the Ben Type field. The TXYTD screen will launch the Tax Detailed Explanation web page automatically for periods after 01 July 2003. This screen shows the tax calculation for the period entered
  • on the web page, select the Period in the Payment Summary Information block. This will display the Calculation Summary block

ICT Service Desk staff, go to Step 11

CDO staff, go to Step 10

10

CDO investigate negative amount + Read more ...

For the period in question, if the total amount calculated is less than the total of the Debt Adjustments and the Other Adjustments, then the following is required:

If the negative amount on the Tax Detailed Explanation web page is:

  • from Other Adjustments, select this heading. If the negative amount relates to:
    • an 'RCV' (Advance Recoveries) entry, see Processing Manual Follow-up (MFU) tax activities for Advance recoveries
    • a 'BTR' (Benefit Transfer) entry, see Processing Manual Follow-up (MFU) tax activities for benefit transfers
  • due to:
    • a single debt, see Processing Manual Follow-up (MFU) tax activities for debt processing
    • multiple debts listed for the same period, see Processing Manual Follow-up (MFU) tax activities for multiple debts raised for the same period

Note: standard debt adjustments to the payment summaries occur once a repayment plan is in place and the first repayment is received. This is an automatic process and does not require an amendment to the TXGS/TXGD screens. See Taxability and tax deductions from payments for further information.

Procedure ends here.

11

ICT Service Desk + Read more ...

Where the problem is identified as:

  • invalid payment links, Payments ICT Service Desk are to confirm payment summary details and correct where appropriate. Procedure ends here
  • Adjustment period mismatch, Payments ICT Service Desk are to confirm payment summary details and manually amend as appropriate

Are corrections to the payment summary required?

  • Yes, go to Step 12
  • No, procedure ends here

12

Manually correct the payment summary + Read more ...

Go to the TXGS screen and at the financial year line to be amended:

  • key 'C' and press [Enter]
  • the TXGD screen for the financial year is displayed. Correct the line for the period containing the error using reason code 'ADM'
  • press [Enter]
  • finalise the activity on the TXGS screen
  • go to the Assessment Results (AR) screen and complete the activity

Note: any manual corrections are overridden when a tax calculation occurs for the financial year corrected

13

Issuing the updated Payment Summary + Read more ...

See Safeguarding customer privacy when raising and recovering debts, before issuing the Payment summary. Take the appropriate actions to protect the customer’s privacy.

To issue the payment summary:

  • go to the TXGS screen, type 'I' at the new line and press [Enter]
  • key 'Y' at Issue Group Cert/Statement and press [Enter]
  • finalise the activity
  • complete the MFU tax activity
  • record details of the changes on a DOC

Workaround – Tax MFU activity regenerated

Table 2

Step

Action

1

Regenerated MFU tax activity created + Read more ...

When a manual change is made, the process may cause the record to revert to how it was before the manual change. This may regenerate a TAXMFU. If this occurs:

  • assess if MFU tax activity is regenerated
  • identify the TAXMFU activity, check if the newly created MFU is related to previous corrected and cancelled MFU activity
  • go to the Tax Payment Summary (TXGS) screen
  • identify the financial year the MFU is related to
  • determine if the MFU been generated following a manual update
  • for the determined financial year: Compare the details on the newly created entry on the TXGD screen with the previous TXGD entry details when the previous MFU was created

2

Validate the MFU activity + Read more ...

Check the Tax Year To Date (TXYTD) and Debt List (OPDL) screens for the relevant financial year identified within the MFU for any newly created debts or Benefit Transfers, which would generate the MFU.

Are there new debts or any changes which generated the MFU?

  • Yes, the MFU has been correctly generated. Go to Step 2 in Table 1 to action
  • No, go to Step 3

3

Amended Payment Summary + Read more ...

Check the TXGS screen for a previously issued amended payment summary for the relevant financial year related to the TAXMFU.

Has the amended payment summary been issued?

  • Yes, go to Step 4
  • No:
    • manually amend the details on the TXGD screen. Go to Step 9 in Table 1 to action
    • issue the Amended Payment Summary. See Step 13 in Table 1
    • cancel the Tax MFU. Procedure ends here

4

Confirm if updates are required + Read more ...

Consider if amendments were to be made to the TXGD screen and whether the updated changes would be different to what was previously issued to the customer.

Would changes applied to the TXGD screen match the previous amendment?

  • Yes, go to Step 5
  • No:
    • the MFU has been correctly generated. Go to Step 9 in Table 1 to action
    • issue the Amended Payment Summary. See Step 13 in Table 1
    • cancel the Tax MFU. Procedure ends here

5

Finalise process by cancelling the MFU and document outcome + Read more ...

Cancel the Tax MFU and do not change any details on the TXGS/TXGD screens.

Record the following information in a DOC:

  • identified financial year for the MFU
  • previously amended summary details, and
  • reason for cancellation

Note: no further MFUs should generate unless a new debt is raised, a debt is waived or the waiver is removed, or a BTR process occurs.

If this issue still occurs, lodge a request to mySupport.
