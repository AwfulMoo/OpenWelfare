Updates and Change of Circumstances (CoC)

Table 1

Item

Action

1

Families and Child Care (FCC) Non Claim Update + Read more ...

A number of different activity types may be included in this work type.

Investigate the record to determine the reason for the activity:

  • 'V'iew the activity notes via the History Activity Detail (HAY) screen
  • Select 'I' via the Activity List (AL) screen to view 'I'nformation about the activity (help text)
  • Check the 'K'eywords
  • Check the Document List (DL) screen for information which may be relevant
  • For child updates, use the Activity Package Viewer (PKGV) screen to determine an associated parent Activity ID (AMR)
  • Check the User ID source via the Event Summary (ES) or History Activity Details (HAY) to determine the origin of the activity

Ensure the appropriate Operational Blueprint process is followed depending on the activity allocated.

2

Change of Circumstances Families and Child Care + Read more ...

This work item is created when a customer has notified a change of circumstances online that requires manual action.

  • Check the Interaction History and Notes tab for information about the update and the reason it did not finalise
  • Follow the appropriate Operational Blueprint process to action the update
  • Change the work item status to Completed when all required action is finalised

3

Change of Circumstances - Paid Parental Leave + Read more ...

This work item is created when a customer has notified a change of circumstances online that requires manual action.

See Customer notifies a change of circumstances for Parental Leave Pay (PPL).

4

Change of Circumstances - Non Lodgement Advice + Read more ...

The customer has provided their non-lodgement advice via online or the Express Plus Centrelink mobile app but the update has not automatically completed.

There are two different work item types that could be allocated for manual completion:

  • ZIAM_NONLDG_GEN (work item opens in Customer First), and
  • ZCOC_FAO_NONLDG (work item opens in Process Direct)

See Recording not required to lodge advice and previous year income.

5

FCC Income Estimate + Read more ...

Income Estimate Update work items are created when the customer has advised their income estimate via online or the Express Plus Centrelink mobile app, but the update has not automatically completed.

There are two different work item types that could be allocated for manual completion:

  • ZIAM_EST_GEN, and
  • ZCOC_FAM_INC

The work items may be duplicated, therefore it is important to check for and complete both the:

  • Integrated System (ISIS) activity (viewable via AL screen), and
  • any associated Customer First (CF) SAP Interaction Record/s or Process Direct = Workload Management System for CCS customers

If the update has not completed a note on the record will provide more information about why the estimate update did not automatically complete.

To view the Note in Customer First:

  • Select the CRN/BP hyperlink to go to Customer Details
  • Select the Interaction History tab
  • Select the relevant ID hyperlink
  • Select the Note tab
  • Select the Text hyperlink to view the full activity notes

For FTB or FTB customers who also receive CCS, the system will generate a work item in Customer First.

For CCS only customers, the system will generate a work item in Process Direct. Staff may be allocated a work item via their inbox or they can access it manually via Customer Summary - Transaction tile.

See Updating income estimates for the current financial year.

6

Change of Circumstances - Payment Choice + Read more ...

Staff will need to:

  • Check the DL screen for a DOC
  • Check the Notes tab of the Interaction Record to view activity notes and the reason the update has not finalised automatically
  • Make updates to the customer's payment choice as required, and mark the interaction as Complete

See:

  • Helping families provide a reasonable annual income estimate for family assistance payments
  • Family Tax Benefit (FTB) claim options and payment choices

Scans

Table 4

Item

Action

1

FCC Request for Information + Read more ...

This work item is created when a Request Documents Q777 request for information was issued and a scanned response has been received.

  • Check the Interaction History tab for the Interaction ID this will have a status of document attached
  • Check the Document List (DL) screen for information which would outline why the information was requested and what action is required
  • Update the customer's details as per the information provided by the customer
  • Ensure the document status is changed to Complete

See Information requests for FTB, PPL, DAP, and CCB lump sum claims.

2

Unstructured Scans + Read more ...

This work item is created when an unstructured scan is received.

  • Check the Document List (DL) screen to identify if there was a triggering event that created this activity
  • Review the scan and action as appropriate
  • When actioned and completed:
    • Select the Transaction ID from the Activity List (AL) screen
    • Select Edit
    • Change Status from Document Added to Completed
