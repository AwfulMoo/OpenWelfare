Helping users create a PRODA account

Table 1

Step

Action

1

Access PRODA + Read more ...

Provide the URL address: servicesaustralia.gov.au/proda

The PRODA homepage will display.

Select New PRODA account to take them through a workflow.

2

Create a PRODA account + Read more ...

Note: to ensure their PRODA account is secure and to maintain security of their personal information, PRODA accounts should not be created for someone else and should not be shared with another person.

The user needs to read the Terms and Conditions.

By beginning the process, the user confirms that they acknowledge the terms and conditions of use.

Did the user accept the terms and conditions?

  • Yes, select Register now. Go to Step 3
  • No, they will not be able to create a PRODA account. Procedure ends here

3

Your details page + Read more ...

We recommend that the user enters their current legal name, as shown on their identity documents, to create an account. The names will be used to verify the identity documents at a later step.

If the current legal name differs across the identity documentation then a name change will need to be verified with a change of name document.

If there are any additional given names, enter them in the Additional names field, multiple names can be separated by a space.

If the user only has a single name, enter it in the Surname field and leave the First name field blank.

Ask the user to check the details entered are correct and select Next.

Note: if the user creates the account without an additional name the system will display a pop-up message Update your additional name(s). If the user does not have an additional name select No.

4

Create your login details page + Read more ...

Provide username

The user will need to create a username. The username will need to be unique within the system and is not case-sensitive, will need to be 5 or more characters and cannot contain spaces or special characters.

Provide password

The user will need to create and confirm their password. The password must:

  • have at least 10 characters
  • have a mix of upper and lower case letters, and
  • include at least 1 number or special character ( ! @ $ % ^ & * + = - ` \ [ ]{ } | ; ‘ : . / < > ? ) with no spaces

Note: the on screen password guide will contain all green ticks when the password meets all the password criteria.

Select Next.

5

Security questions and answers + Read more ...

The user must select 3 pre-determined questions from the dropdown boxes and then type their answer to each question. Note: the answers are not case sensitive.

Select Next.

Security questions and answers help

Your answers should be:

  • Memorable - something you will remember
  • Constant - not change over time
  • Safe - not be something that is easily guessed, or researched online (for instance via social media)
  • Different - not the same answer as other questions

Your answer must be at least 3 characters long and cannot be consecutive letters or numbers (e.g. 'abc' or '123').

6

Your email address page + Read more ...

The user is to provide and confirm their email address for the following reasons:

  • so we can send a secure code each time they login to the system
  • to assist in account recovery should they forget their username and/or password; and
  • to contact them regarding their account

Advise: 'Your email address needs to be unique within the system, and should not be a group email account (used by multiple people). If you, or someone else has created an account with the email address you are trying to use, the system will not allow you to continue. If this occurs, select "Contact us" at the bottom of the screen for further support.'

Select Next.

7

Verify your email address page + Read more ...

Note: the activation code is used to verify the email address entered by the user. This is different to the verification code sent to the user each time they access their PRODA account.

PRODA Activation code sent

  • If the email address is unique (not currently in use on another PRODA account), PRODA will send the user an email to the email address entered
  • The email will contain a confirmation code

Note: if the user did not receive the email in their inbox, they may need to check their junk mail.

Has the confirmation code been received?

  • Yes, ask the user to input the code and select Next. Go to Step 9
  • No, go to Step 8

8

Confirmation code not received + Read more ...

The user will need to:

  • select the Didn't receive your code? link, and check the contact details they have entered for errors. Note: occasionally, the code can take some time to come through
    • Check the junk mail folder
    • Close this window and check the email address within the sentence 'We sent a code …' is correct
    • If a mistake is identified, select 'Back' to correct it on the previous screen
    • If the email address is correct, wait until the Resend code button is enabled and select it
  • wait 5 minutes until the Resend code button is re-enabled and then select it again

Has the confirmation code been received?

  • Yes, ask the user to input the code/s and select Next. Go to Step 9
  • No, the user will need to use an alternative email address that they have access to and select Back to input an alternative email address on the previous screen.

9

Confirmation email + Read more ...

An email will be sent to their unique email address (PRODA Account Created) confirming their account creation.

This email will also contain their Registration Authority (RA) number They will need the RA number for some services in Health Professional Online Services (Delegations etc.). They can also find their RA number after the account has been successfully created on the Profile page.

Helping users add identity to a PRODA account

Table 2

Step

Action

1

Verifying your identity page + Read more ...

Advise: 'The next step in this process is to verify your identity. If you have changed your name and any of the 3 documents you select does not have your current name (for example, your passport has your previous name, you will need to provide an additional document that confirms your name change, such as a change of name or marriage certificate.'

2

Can the user provide the following identity documents now? + Read more ...

Online identity documents

  • One of the following documents:
    • Australian passport
    • ImmiCard
    • Citizenship certificate
    • Australian birth certificate
    • Australian Visa (supported by a Foreign passport, which is needed for verification)
    • Certificate of registration by descent
  • Two of the following documents:
    • Medicare card
    • Australian driver's licence
    • ImmiCard (can only be used once)
    • Australian passport (can only be used once)

Note: if a current name is different to the name on any of the identity documents, the user will need to provide a change of name document that can link the names:

  • Marriage certificate
  • Change of name certificate

Note: if the user advises they don't have enough of these documents to finalise the process online. Advise: 'There is an extended document list, documents from the extended document list can be submitted by selecting 'I don't have any of these documents. You will need to submit your identity documents via a paper form that can be downloaded at the end of the process.'

Can the user provide the following identity documents now?

  • Yes, go to Step 3
  • No, ask them to log off. Once they have all relevant documents available, they can log on again with their identity documents handy to complete their registration. Procedure ends here

3

Verify your first document + Read more ...

Recommended: Passport, Australian Drivers Licence and Medicare card.

Ask the user to:

  • select 1 document from the list
  • provide the document information for the selected document

Note: information for only 1 document can be entered at a time.

Select Next.

The information must match the documentation.

A confirmation message will display once the information has been entered and verified.

The user will need to repeat the above steps for all required documents.

The Resources page contains a list of the documents and what information the user inputs.

4

Have all documents 'Passed' successfully? + Read more ...

  • Yes, select Next. To continue process, see Step 1 in the Helping users set up their verification code table
  • No, if:
    • they need to verify the change of name, go to Step 5
    • identity cannot be verified online, go to Step 7

5

Confirm your name - pop-up message + Read more ...

If the user created their account in a name that did not match any of their identity documents, the user will be asked if they want to use the name on their identity documents as their account name.

Does the user want to use the name on their identity documents as their account name?

  • Yes, select Yes. To continue process, see Step 1 in the Helping users set up their verification code table
  • No, the user will need to complete the change of name process. Go to Step 6

6

Change of name + Read more ...

The fourth document selection is only required if an additional name(s) has been entered when verifying the other documents entered. The user must establish a link between the additional name(s) entered and the name used to create the account using a change of name document.

The documents required to verify a change of name are:

  • Marriage certificate (Bride only)
  • Change of Name Certificate

Ask the user to:

  • select 1 document from the list that will verify their name change
  • provide the document information for the selected document

Note:

  • Only 1 documents information can be entered at a time
  • If the user selects Marriage certificate, they will need to enter the related information into the required fields.
  • The required fields will change depending on the year and state.
  • The Resources page contains information on what the user enters for these documents.

Select Next.

Have all documents 'Passed' successfully?

  • Yes, select Next. To continue process, see Step 1 in the Helping users set up their verification code table
  • No, if identity cannot be verified online, go to Step 7

7

If document verification fails + Read more ...

Service Officers can log on to the PRODA Staff Portal to check the document information entered by the user.

If unable to identify any errors in the PRODA Staff Portal:

  • Ask the user to select I don't have any of these documents, and Yes to download form
  • Advise the user their account creation has been pended. The user will need to complete the form and provide copies of the documents that have failed. Ensure to advise the user to provide a contact number on the form. If any further information is required, a Service Officer will call/email them once the form has been processed

Procedure ends here.

Helping users set up their verification code

Table 3

Step

Action

1

Verification code preference page + Read more ...

On this page, the user selects their Preferred code delivery option.

If they select:

  • email, a code will be sent to them via email each time they log on
  • mobile app, a message will be displayed to the user asking if they have already downloaded the PRODA app and linked their PRODA account
    • See the Set up help mobile app table, in Helping users self-manage their Provider Digital Access (PRODA) account
  • SMS, the user will be prompted to add their mobile number and confirm it. The user will need to enter the verification code sent to their mobile phone number. A code will be sent to them via a text message each time they log on

The verification code will need to enter as part of the log on process.

Select an option and select Next.

To continue process, see Step 1 in the Helping users link existing record identifiers to their PRODA account table.
