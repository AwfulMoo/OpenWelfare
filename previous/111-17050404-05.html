HPOS Reference Number fields

Table 1: the expected result for each Health Professional Online Services (HPOS) Number field.

HPOS Reference Number field

Expected Result

Blank

An address screen in HPOS Support with no address will display.

Invalid

For example: HPOS Reference Number does not exist in HPOS Support)

A HPOS Support Error page will display.

Valid

For example: HPOS Reference Number exists in HPOS Support but is incorrect for the PaNDA work item

Incorrect HPOS Form Upload file in HPOS Support. The documents in HPOS Support will not match the documents in PaNDA for the associated provider.

Valid

For example: HPOS Reference Number exists in HPOS Support and is correct for the PaNDA work item)

Correct HPOS Form Upload file in HPOS Support. The documents in HPOS Support will match the documents in PaNDA for the associated provider.

Terms used in PaNDA

Table 2

Term

Description

Age in Days

Numerical value indicating the age of the work item

Assign to Me

Will change the allocation of the Work Item to the user that selected the button

Associate Document

Standard button used to associate Work Items

Cancelled

Status applied when a work item is associated (orphaned)

Capture date

Date the file is processed through the system

Complete

Status used when the work item has been finalised

Document

Terminology used to refer to images/files within a work item

Document ID

The unique ID of the document in context

Due Date

The due date of the work item

Employee Responsible

User ID of the employee allocated to the work item

History Tab

A log of work item activity

Information Tab

Work item details (Program, Work Type, Primary Reference)

Landing Page

Display of Work Optimiser - Medicare functions represented as Tiles

Notes Tab

Icon where a text field is displayed for users to enter processing notes between 1-255 characters

PaNDA Document Upload

Work Optimiser - Medicare function that allows users to upload a Medicare document to either create a new work item or associate with an existing work item

PaNDA Search

Primary method for accessing documents by program, Work Type, status and date of lodgement

PaNDA Work Type Configuration

PaNDA Work Type information includes Work Type name, program, Work Type description, naming conventions and KPI details

Pend

Status used when work items must be held for a period of time

Primary Reference

Unique identifier (Medicare Number, Provider Number)

Reassign

Standard button providing the user with a list of reasons for reassignment to another user

Recategorised

Enables the user to re-categorise the work item to another work item type

Receipt Date

Date the form was received by the department

Save

Standard button used to save any changes to the Work Item

Secondary Reference

Unique identifier used where a primary reference is unavailable (patient or provider Name)

Send for Delegation

Standard button used to refer a work item to a Delegate for assessment

Service Support Required

Reassign/unassign reason where technical support is required - it is more likely that the proficiency is raised or the work item is reassigned to a specific user in this instance

Source Channel

The channel the document has come through (SCAN, EMAIL, HPOS, Upload)

Source Location

The exact place the document was scanned/uploaded, emailed or faxed from

Status

Represents the progress of a work item and can be displayed as Open, Held (Pended) or Complete

Tile

Terminology used for menu boxes found on the Landing Page

Unassign

Standard button providing the user with a list of reasons for unassignment to the allocation pool

User Unavailable

Reassign/unassign reason where work items can be returned to the pool or assigned to another user where the assigned user is unavailable

Work Item

Terminology used in PaNDA to refer to what a user receives when they indicate they are ready for work

Work Item ID

The unique PaNDA ID of the work item

Work Type

Terminology used in PaNDA to identify a work item usually based on the programme it belongs to

Worklist

Workload management inbox where work items can be allocated
