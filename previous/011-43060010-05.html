Simplified Billing EDI, manual and ECLIPSE claim enquiry control lines

The FCEI transaction is used to check the status of Simplified Billing EDI and ECLIPSE claims. Details of all claims (unpaid, paid including LDA) can be obtained depending on the modifier used in the control line.

Table 4: This table lists the FCEI control line modifiers and the information returned by each.

Control line

Returns

FCEI, Minor ID or Registration number

All claims for the Minor ID or Registration number in ascending claim ID order.

FCEI, Minor ID or Registration number, Claim ID

Claims for the Minor ID or Registration number with the same claim ID prefix.

Note: this is useful to see if a claim has been adjusted.

FCEI, Minor ID or Registration number, Claim ID ,DOL=ddmmyy

Details of any status changes for the Claim ID

FCEI, Minor ID or Registration number, Claim ID ,ST/ALL,FR/ddmmyy,TO/ddmmyy

Claims for the Minor ID or Registration number with a date of lodgement between the from (FR) and to (TO) dates in the control line, regardless of their current status.

FCEI, Minor ID or Registration number ,ST/PAID,FR/ddmmyy,TO/ddmmyy

Claims for the Minor ID or Registration number with a date of lodgement between the from (FR) and to (TO) dates in the control line, with a status of PAID.

Note: LDA claims excluded.

Status values - ECLIPSE claims only

Table 5: This table lists the status values that are specific to Simplified Billing ECLIPSE claims.

FCEI status

Description

In Pend

Claim pended

After initial assessment in background, the claim could not be automatically system paid. The claim requires Service Officer intervention via pend processing (FAPI)

PAID-SD

Claim paid same day

Claim has been completed successfully and reached payment status today (Service Office cannot reopen the batch)

PAID-LD

Claim paid latter day

Claim has been completed successfully and reached payment status more than a day ago (Service officer cannot reopen the claim)

HUB 2GET

ECLIPSE HUB to collect

The ECLIPSE claim has been assessed successfully and is waiting for the ECLIPSE HUB to collect the details and transmit them back to the private health insurer or billing agent.

HUB ASS

ECLIPSE HUB sending assessment

The ECLIPSE claim assessment result details are being sent to the private health insurer via the HUB.

FUND ASS

Private health insurer is assessing ECLIPSE claim

The ECLIPSE claim details have been sent back to the private health insurer via the HUB.

FUND REJ

Private health insurer has rejected ECLIPSE claim

The private health insurer has assessed the ECLIPSE claim and has sent back a rejection of the claim.

BACK REJ

Background rejection of ECLIPSE claim

The ECLIPSE claim is currently being rejected in background

REJECTED

ECLIPSE claim rejected

The ECLIPSE claim has been rejected by the private health insurer and successfully rejected in Simplified Billing in background.
