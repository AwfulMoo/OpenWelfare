Item

Description

Worklist + Read more ...

Staff can use Worklist to view and manage all work assigned to them. Two standard worklist views are available to staff:

  • My Work - displays work assigned to them in Work Optimiser, Customer First, and Process Direct
  • Customer Search - to find work related to a customer or organisation record

From the Worklist, staff can select:

  • Work Item ID hyperlink for each work item to navigate to:
    • Work item Details
    • Customer First or Process Direct (where this option has been enabled)
  • Search to search for specific work that has been assigned to them, using the available search criteria. Staff can choose the information displayed for search results with a customised Saved Search
  • Select Filters to customise the search criteria
  • Select Columns to customise the columns that display in the search results
  • Sort Options to sort, filter, and group search results
  • Ready or Quick Ready to have a work item assigned to them for processing
  • Actions to display more options:
    • Hold to User to keep a piece of work in their Inbox until it is complete
    • Assign to Me to assign a piece of work to their inbox for processing
    • Reassign to redirect an assigned piece of work from their inbox to another staff member for processing
    • Unassign to return a piece of work from their inbox to the National Pool

The Read feature also allows staff to identify when work items were last viewed. Work items can display as:

  • Unread - the work item has not been viewed
  • Just Now - the screen has not been reloaded since the work item was viewed
  • Today - the work item has been viewed today
  • Yesterday - the work item was viewed yesterday
  • [number of days] Days Ago – the number of days since the work item was last viewed

To access the Worklist from the Process Direct inbox:

  • Select Inbox from the Process Direct Launchpad
  • Select WLM Worklist from the Main Category drop down menu

2

Worklist Saved Searches + Read more ...

Staff can save personalised searches to display preferred columns in the Worklist. This feature is useful for staff that set reminders when they use Hold to User.

The search results will only display work assigned to the staff member.

  • Use Select Columns to select the preferred columns to display
  • Select Go
  • When the results display, access the Saved Searches list
  • Select Save As
  • Enter a name for the saved search, and select the Set as default checkbox if needed
  • Select Confirm

The default view is My Work. This opens when launching the Worklist.

To change the default view to a personalised view:

  • Select My Work. A list of Saved Searches displays
  • Select Manage in the Saved Searches window
  • Select the preferred default
  • Select Confirm

3

Staff Profile + Read more ...

This is a snapshot of a staff member’s WLM profile. It can help staff understand why they receive particular types of work.

Staff can view:

  • Capabilities: skill tags assigned in ESSentials, including the proficiency level
  • Temporary Work Groups they have been added to so they can receive priority work items
  • Queues they have been added to so they can receive specific work types
  • Customer Relationships: they have with customers
    • Conflict of Interest
    • Restricted Access
    • Child Support Bundle on Creation

If staff select Enable Email Notifications, they will get an email when a work item is assigned to their inbox. This is useful for staff who:

  • have a case management relationship with a customer, and
  • do not undertake processing work daily

4

My Dashboard + Read more ...

Staff can view a summary of their personal WLM activity for the current and previous day.

To swap between current and previous day information, select the Yesterday/Today icon:

  • Yesterday shows information current at 23:59 that day, based on the user’s time-zone
  • Today shows point-in-time information. To view the most up-to-date details, select Refresh icon

Tiles at the top of My Dashboard display the number of work items in two groups.

Work Item Progress shows information about the status of work items assigned that day:

  • Open: work items assigned to staff
  • On Hold: work assigned that was placed on hold
  • Finalised: work items completed since 00:00 on the current day

Work Item Movement shows information about work coming into, or out of the Worklist that day:

  • Daily Allocations work assigned to staff since 00:00 that day
  • Daily Unassigned work items by staff since 00:00 that day
  • Daily Reassigned work items by staff since 00:00 on the current day

Select one of the tiles to display the information in a graph.

More detailed work item information can be viewed in an itemised list. Select a bar segment from the graph to view the information.

For a list of work item status definitions, see the Resources page.

Receiving new work + Read more ...

WLM allocates work items to staff using business rules. These rules take into account a range of factors including:

  • Priority of Work Items: based on many factors including Key Performance Indicators (KPIs) and other business needs. Priorities may change in line with rules set by business owners
  • Staff capability: a combination of skill tags, proficiency and delegation level
  • Work Item Status: e.g. open, hold, due, complete
  • Bundling: see Item 3

To receive a new work item in Work Optimiser, select Ready or Quick Ready at bottom right of the Worklist screen:

  • The work item will display in the Worklist results. Note: WLM allocates a work item each time Ready or Quick Ready are selected. Having multiple actionable work items in the Worklist may impact customer outcomes. Any staff unsure about when or how often to receive new work should seek technical support
  • If enabled, the Work Item ID hyperlink can be selected to go to the relevant system (Customer First or Process Direct) to complete the work

Staff are taken to the work item and details summary. If enabled, staff can select the OPEN IN button to go to the relevant processing system to action the work.

Work allocated via Quick Ready will not be bundled.

3

Bundled Work + Read more ...

Multiple work items may be allocated together. This is known as a bundle.

Bundles are allocated to staff with the assigned skill tag for the parent work item, e.g. a new claim.

Bundles may include work items on both the customer and partner records.

Program business owners decide how bundling works for each work type. Workload Management (WLM) Business Rules include the bundling rules.

Work allocated via Quick Ready will not be bundled.

When staff are allocated a bundle

An Allocate the following bundle pop up box displays for them. It shows the work items in the bundle before they get assigned to their inbox:

  • Staff are automatically assigned all work in the bundle they are skilled for
  • Staff can accept extra work items they may not have a skill tag for, but have the capability to complete

The pop up box has a table with these details and functions:

  • Work Item count: blue text towards the top of the table confirms the total number of bundled work items on the customer and partner records
  • Work type Description for each bundled work item
  • Allocate: a checkbox indicates items that will be assigned from the bundle to their inbox:
    • Select the checkbox next to a work item to accept it
    • Select the checkbox in the table header to accept all items in the bundle
    • Staff cannot de-select a work item they have the skill tag for
  • Bundle Reason: why each item in the bundle has or has not been assigned to the staff member
  • User Reason: why the staff member has or has not accepted each item in the bundle:
    • Customer Contact is Required
    • Service Officer has Incorrect skills
    • Work Item not Actionable
    • To improve processing efficiency. This auto displays when a staff member accepts a work item not assigned to them
  • Work item ID for each item in the bundle
  • Submit to assign all selected work items to inbox

Staff should complete any outstanding work on a customer's record if they can. This gives a better outcome for customers.

Using Reassign and Unassign to reallocate work + Read more ...

Staff can reallocate work assigned to them.

Select the Actions button to display the Actions menu. Select:

  • Reassign to assign the work item to a specific staff member
  • Unassign to return the work item to the national allocation pool

To reallocate work items from the Worklist:

  • Select the checkbox for the work item/s to be reallocated
  • Select Reassign or Unassign from the bottom right of the screen
  • If reassigning, select User ID to search for and enter the staff member's logon
  • Select the Reallocation reason from the drop down list. For a list explaining these reasons, see Workload Management
  • Select Submit

5

Recording a conflict of interest + Read more ...

If a staff member is allocated work for a customer they know personally, e.g. family member, friend, personal acquaintance or neighbour:

  • They can unassign the work item using the reason Conflict of Interest
  • This creates a Conflict of Interest relationship between the staff member and the customer
  • The work item will return to the allocation pool and no further work for this customer will be allocated to the staff member

Conflict of interest relationships display in Staff Profile under Customer Relationships.

Team Leaders can add or remove a Conflict of Interest relationship from staff they line manage, if appropriate. See Work Optimiser for Team Leaders and Analysts.

Note: recording a conflict of interest relationship for WLM does not record inadvertent access to a customer’s information.

6

Searching for work items by CRN/ CSRN/ Medicare Number + Read more ...

Staff can find work related to a specific customer or organisation by searching for the Customer Reference Number (CRN), Child Support Reference Number (CSRN) or Medicare Number:

  • Select the down arrow icon next to the search view (e.g. My Work) at the top left of the Worklist. A list of saved searches will display
  • Select Customer Search
  • Enter the customer or organisation CRN/ CSRN/ Medicare Number in Customer ID field
  • Select Go

Staff can then allocate work to their inbox:

  • Select the work item/s from the customer search results
  • Select Actions from the bottom right of the screen
  • Select Assign to Me from the menu
  • The work item will be assigned to the staff member and display in the staff member's Worklist

Update work item details for open work items - Medicare only + Read more ...

Make sure that misnamed work items are updated straight away with all relevant fields before completing the item.

If needed, update the work item details by updating these fields (if provided):

Update work item status - Medicare only + Read more ...

Close the document and select the back arrow to go back to the Work Item Details screen.

  • Open

  • Pended

    • Select Save. See Item 8
  • Complete

Pend work item - Medicare only + Read more ...

Select Pended in the Status field.

Select appropriate reason from the Pend Reason drop down menu.

Users can place a work item on hold (pend) for a set time. The time is input by the user for a future date pend reason, or is pre-set based on the pend reason.

See the Resources page for a list of pend reasons and explanations.

11

Complete Work Item - Medicare only + Read more ...

On completion of the work item in the source system (where the work is processed, e.g. Mainframe, CDMS):

  • Select Complete in the Status field
  • A Complete reason is only required when:
    • the work item has been completed in the source system (it appears to have already been completed - a duplicate work item)
    • a delegate is updating the work item with their decision before returning a work item to a user or the allocation pool
  • Select Save

A message will display at the bottom of the screen advising that the work item has been successfully saved.

It may take up to 2 minutes for the work item to disappear from the Worklist.

Processing notes + Read more ...

Staff can view, add and edit processing notes to record relevant information on a work item including:

  • processing status
  • customer contact and
  • categorisation updates

Processing notes must not replace updates to records in processing and/or source systems required as part of standard business procedures.

Staff must add a processing note when:

  • unassigning a work item
  • giving information on action taken and
  • the reason the work item has been returned to the allocation pool

Notes can only be added to a work item assigned to a user in Worklist and a 255 character limit applies. Notes are read only for all other work items. IBM Content Navigator (FileNet) will only capture the last processing note recorded.

When to recategorise + Read more ...

If an allocated work item has been classified incorrectly, it can be recategorised to the correct work type. A work item should only be recategorised when:

  • the initial classification is incorrect, or
  • recategorisation is part of an approved and documented business process

Business owners can restrict the ability to do this for a work type in the WLM business rules.

Staff should action any outstanding work for the customer if they can and are appropriately skilled. Even if it has been classified incorrectly.

Medicare only: recategorising to the correct work type will help with locating work items in PaNDA search.

For more information, see Workload Management.

2

How to recategorise + Read more ...

Some work types:

  • Have limited recategorisation options, or
  • Are restricted from being recategorised by the program owner

Work items can be recategorised directly from:

  • The Worklist search results, or
  • The Work Item Details screen

To recategorise via the Work Item Details screen:

  • Select the work item
  • Staff need to select
    • Unassign to recategorise the work item and remove it from their Worklist or
    • Reassign to include their login details and retain the work item in their Worklist
  • In the Reason pop-up, select Re-categorised from the drop-down list
  • Select the Re-categorise field to display a pop-up
  • In the pop-up select the new Category:
    • Work Type Program and Work Type Process Category filters can be used. These will narrow options and can help select the correct work type
    • If unsure of the correct work type after using these filters, do not try to recategorise. Seek help from a Team Leader or on-site Workload Management SME
  • Select Submit to save changes

A warning displays if the work item has already been recategorised.

Once recategorised, a message at the bottom of the screen says the work item is unassigned.

Relationships + Read more ...

Staff can use relationship profiles to create a WLM relationship between themselves and specific customers or organisations. This is known as an Relationship Management.

Relationships allow staff to tailor services for customers who may benefit from a ‘case management’ approach. They are commonly used by processing staff for a claim to be Held to User.

To create a Relationship, staff need to first create a relationship profile. Once a profile has been created, it can be used for relationships with multiple customers.

Note: including a CRN/CSRN in a relationship profile is not mandatory. This allows staff to pre-save profiles of work types for use with Hold to User.

Staff can view and edit relationship profiles for which they are the Employee Responsible.

Hold to User + Read more ...

If staff cannot complete a piece of work, they may need to place it on hold. For example, if the customer needs to provide more information.

Hold to User is a function in Work Optimiser that holds a work item in a user’s worklist until it can be completed.

When and how staff Hold to User is decided by different business processes across the agency.

If staff are unsure if they should use Hold to User for your work, seek advice through your local technical support process.

Hold to User with Relationship Management

  • Commonly used for claims processing, or to provide a case management servicing approach. For example if a customer is vulnerable, in hardship or has complex circumstances
  • The work item is held to the worklist, and new work for the customer is automatically assigned to the Service Officer. For this to occur, specific work types must be included in a relationship profile
  • The same relationship profile can be used for multiple customers

Hold to User without Relationship Management

  • WLM holds the work item in the worklist
  • Used by some teams to hold work in WLM that cannot be held in a source system

The Reminder feature allows staff to set a reminder to show when a work item is due. Reminders are:

  • Useful when work needs to be held and updated over an extended period. For example, during a review that requires multiple contacts with the customer
  • Displayed in the Worklist in red text when they are due

Note: staff must create a new default Worklist Saved Search to view Reminders in the Worklist. Use Select Columns to add the Reminder Date/Time column.

2

Updating Hold to User + Read more ...

  • Place the work item on HOLD in the Source System if appropriate
    • If the work item is placed on HOLD in the Source System
    • Wait until the work item status changed to On Hold in WLM
    • Locate the work item entering the CRN/CSRN into the Customer Search available in the Worklist
    • Select the work item then select Actions and then select Assign to Me
  • Select the work item in Worklist
  • Select Actions
  • Select Hold to User

The Hold to User pop up box will appear with the following options:

  • Reminder Date/Time: Reminders are optional. They are typically used and recommended for work that is not placed on HOLD in a Source System prior to Hold To User being used in WLM
  • Create Relationship with this Customer and Work Type: Select this option if Relationship Management is required for case management style servicing
    • If a Relationship Profile has not been created, a message appears advising the Service Officer to create a Relationship Profile
    • If a Relationship Profile with matching details and overlapping dates already exists, the profile cannot be created
  • Confirm: updates the work item status to HTU
  • If Create Relationship was selected, the CRN/CSRN will be added to the Relationship Profile

If a CRN/CSRN is added using the above method, once the work item is complete (or is unassigned or reassigned), the CRN/CSRN will be automatically removed from the Relationship Profile. Work will no longer be automatically assigned for this customer.
