CDC participant contacts to request a one-off expense transfer + Read more ...

If there is a DOA DOC on the participant's record identifying them as vulnerable for the CDC, go to Cashless Debit Card Vulnerable Participants

Note: if the participant has switched CDC Issuers, a transfer of funds from the losing CDC account issuer cannot be processed.

Requests for a one off expense, also known as a direct entry transfer, may be considered where the participant is unable to use the other available CDC payment options. A one off payment/transfer is sent via direct entry to the business/seller. Requests must be for a specific purchase and must be supported by documentation.

Examples of suitable documentation include, but are not limited to:

  • Invoice
  • Receipt
  • Bill
  • Statement

Where a participant requests to access additional funds to attend a funeral or other sensitive/traumatic event, Service Officers should encourage the participant to access their Other Expenses Limit. If this is not possible, Service Officers should use their discretion to assess the legitimacy and appropriateness of the request. Where available, supporting documentation should be supplied.

Can the participant make the transfer via another method?

  • Yes, DOC the record with details of the discussion. Procedure ends here
  • No, go to Step 2

2

Confirm Card Issuer + Read more ...

A participant’s card issuer can be confirmed on the Payment Details Lodgement (PDL) screen in Customer First by checking the BSB for the restricted account.

  • Traditional Credit Union participants' BSB will be 704 119
  • Indue participants' BSB will be 704 772

Go to Step 3.

3

Confirm Other Expense Limit balance + Read more ...

Contact the card issuer to confirm the participant’s current Other Expenses Limit balance.

Where there are funds available in the Other Expenses Limit, the participant must use this to make the purchase. If the full amount is not available, a transfer may be considered to cover the outstanding balance.

The default Other Expenses Limit for NT and Cape York participants is $0.

Can the participant use the Other Expenses Limit to make the purchase?

  • Yes, DOC the record with details of the discussion. Process ends here
  • No, there are insufficient funds available to pay the expense. Go to Step 4

4

Obtain transfer details + Read more ...

Discuss and confirm the following:

  • The amount required
  • The reason for payment
  • Whether suitable supporting documentation has been provided (supporting documentation is mandatory for all one-off expenses)
    • If supporting documentation is available, copy the Document ID from Document Tools, or save a copy to upload within the workflow.
  • The payment destination for the transfer (bank details)

5

Action the One-off Expense request + Read more ...

Run the Request Funds Transfer workflow from the Customer First Workspace>Debit Card Trial menu.

  • Select the correct card issuer, noting there may be two Active card issuers
  • Select ‘Add new request’
  • Complete the Authentication details and select ‘Continue’
  • Decision Type: One-off Expense
  • Confirm the participant has available funds via the balance display at the top of the workflow, or by selecting ‘View CDC Transaction History’
  • Select the payment destination. For One-off Expenses, the system will default to ‘Third Party Account’. Complete all Account Details fields with the information obtained from the participant. If the participant is facilitating a one-off expense to a third party themselves, or requires access to the funds as cash, select the ‘Unrestricted Account’ radio button. This will deliver the funds to the participant’s current payment destination as displayed on the Payment Destination Summary (PAS) screen
  • Upload any supporting documents – this is mandatory for One-off Expenses (excluding vulnerable participants' standard $200.00 other expenses limit transfer):
    • Local File - Click the 'Browse' button to find and load the supporting document, then add a name that best describes it clearly, or
    • Document ID - Locate the scanned document via the Document Tools page on the main menu and paste the Document ID (in full, including brackets( into the input field, then add a name that best describes it clearly
  • Add mandatory notes
  • For one-off transfers, the Immediate Transfer radio button should be set as 'No'. Do not update this field. If the request is urgent, go to Table 2

To finalise the request:

  • Non-CDC staff, go to Step 6
  • CDC Processing and Hotline staff, go to Step 7

6

Lodge the request + Read more ...

To lodge the request, ensure all fields in the workflow have been completed.

It is mandatory to add notes to ensure applicable information is supplied to assess the request.

Once complete:

  • Select the ‘Lodged’ radio button.
  • Complete the Processing Declaration
  • Select ‘Finalise’

A Workload Manager activity will be created for assessment. Advise the participant their request has been lodged.

Where a request is pending supporting documentation, and the ‘Lodged’ option has been selected, place the activity on Hold for 7 days.

  • Select the activity from the Activity List (AL)
  • Select the ‘Hold/Resubmit’ radio button and complete the ‘Resubmit Date’ field on the Activity Registration (AY) screen with a date 7 days in the future
  • Select ‘Complete’

Advise the participant to supply supporting documentation within those 7 days. After that time, the lodged request will be rejected.

Procedure ends here

7

Approve or Reject the request + Read more ...

For Cashless Debit Card (CDC) Hotline and Processing Team only

To finalise the One-off Expense request, ensure all fields in the workflow have been completed.

It is mandatory to add notes to ensure applicable information is recorded to support the request.

Does the request meet the requirements for a One-off Expense?

  • Yes, select the ‘Approve’ radio button
  • No, select the ‘Reject’ radio button
  • Complete the Processing Declaration
  • Select ‘Finalise’. This will automatically process the transfer

Advise the participant of the outcome. If the request has been rejected, discuss alternate transfer options.

If a request has been received via Workload Manager, go to Step 8

8

Actioning WLM requests + Read more ...

For Cashless Debit Card (CDC) Hotline and Processing Team only

Lodged One-off Expense requests will be allocated via Workload Manager.

  • Read and assess the request, ensuring it meets the One-off Expense criteria
  • Review any past requests to ensure the participant is not attempting to circumvent the program. Some historic transfers will not display in the workflow. View the Document List (DL) screen for more information. If there are concerns with the number of requests, contact the participant to discuss whether the request is necessary.
  • Where a lodged requests presents after being on hold for 7 days, ensure appropriate supporting documentation has been supplied.

Does the request meet the requirements for a One-off Expense?

  • Yes, select the ‘Approve’ radio button.
    • Complete the Processing Declaration
    • Select ‘Finalise’. This will automatically process the transfer
  • No, select the ‘Reject’ radio button.
    • Complete the Processing Declaration
    • Select ‘Finalise’

Advise the participant of the outcome. If the request has been rejected, discuss alternate transfer options. Procedure ends here.

Temporary Replacement Card + Read more ...

If the participant has attended a Service Centre and no longer has access to their Cashless Debit Card, offer a temporary replacement card. Ensure the participant's physical and postal address details are up to date and correct. See Temporary Cashless Debit Cards (CDC). Procedure ends here.

If the CDC participant has contacted via phone, advise that temporary replacement cards are available at:

  • all service centres throughout Australia
  • participating Agent sites in the Northern Territory and Cape York
  • selected Australia Post Outlets and Local Partners
  • TCU branches

The CDC participant can contact the card issuer directly to order a personalised replacement card.

See Resources for assistance locating these services

Can the participant attend one of these locations?

  • Yes, advise the participant to attend in person to obtain a temporary replacement card. DOC the record with details of the discussion. Procedure ends here
  • No, advise the participant to contact the card issuer to order a personalised replacement card. Go to Step 3

Obtain transfer details + Read more ...

Discuss and obtain the following information:

  • Why the participant requires funds urgently and whether they can wait until the following day
  • Why the participant is unable to obtain a temporary card
  • Check the participant’s previous and next payment dates
  • Check the record for similar requests
  • Request the destination account details for the RTGS transfer (account name, BSB, account number)
  • If the destination account is the account their welfare payment is paid to, confirm the details provided match the Payment Destination Summary (PAS) screen
  • If the participant provides a BSB and Account Number that the Department has not previously paid, ask additional Proof of Record Ownership (PORO) questions to ensure the legitimacy of the request
  • If the participant has provided supporting documentation, copy the Document ID from Document Tools or save a copy to attach to the request

Note: RTGS should only be considered for exceptional and unforeseen circumstances where a participant is unable to wait the usual processing time of one to two days via Direct Entry.

Action the Urgent Transfer request + Read more ...

RTGS transfers may arrive in the participant’s nominated unrestricted account on the same day (dependent on their banking institution and time of transfer request). All RTGS requests must be submitted to the card issuer by 2:30pm AEST. If processing after 2.30pm, consider processing as a one-off expense.

Run the Request Funds Transfer workflow from the Workspace > Debit Card Trial menu.

  • select the card issuer, noting there may be two Active card issuers
  • Select ‘Add new request’
  • Complete the Authentication details and select ‘Continue’
  • Decision Type: Urgent Transfer
  • Urgent Transfer Reason:
    • Emergency - where a participant does not have a CDC and is unable to obtain one
    • Other - where a participant requires access to funds urgently and is unable to use their card or online transfers
  • Confirm the participant has available funds via the balance display at the top of the workflow, or by selecting ‘View CDC Transaction History’
  • Select ‘View Card Details’ to confirm if the participant has ordered a new CDC. If the participant has not ordered a new card, discuss placing an order after the transfer
  • Are funds available and has Other Expenses Limit been exceeded?
    • If the participant has available funds, and contact with the card issuer has confirmed the participant has exceeded their $200 Other Expenses Limit, select the ‘Yes’ radio button
    • If the participant has not exceeded their $200 Other Expenses Limit, select the ‘No’ radio button. Advise the participant they must use their Other Expenses Limit to cover the expense. If there is insufficient funds in the limit to fully cover the expense, consider making up the difference with an Urgent Transfer
  • Select the payment destination. For urgent transfers, the system will default to ‘Unrestricted account’. If the payment is to a third party select the ‘Third Party Account’ radio button. Complete all Account Details fields with the information obtained from the participant.
  • Upload any supporting documents:
    • Local File - Click the 'Browse' button to find and load the supporting document, then add a name that best describes it clearly, or
    • Document ID - Locate the scanned document via the Document Tools page on the main menu and paste the Document ID (in full, including brackets( into the input field, then add a name that best describes it clearly

  • Add mandatory notes.

To finalise the request:

  • Face to Face and non-CDC processing staff, go to Step 8
  • CDC Processing and Hotline staff, go to Step 9

  • Review any past Urgent Transfer requests to ensure the participant is not attempting to circumvent the program. Some historic transfers will not display in the workflow. View the Document List (DL) screen for more information. If there are concerns with the number of requests, contact the participant to discuss whether the request is necessary

Does the request meet the requirements for an Urgent Transfer?

  • Yes, select the ‘Approve’ radio button

    • Complete the Processing Declaration

11

Notification of request completion + Read more ...

The card issuer will respond to the email request upon completion.

Where staff receive an email response from the card issuer, check the following details:

  • date email referral was sent
  • time the email referral was sent
  • date the email response was received
  • time the email response was received

Check the type of request against the agreed timeliness standards. The Resources tab contains a copy of the Service Level Agreement.

Where a request falls outside of the agreed timeframe, register the details in the SLA Tracker.

The tracker must be updated and maintained and is required to be sent to the Deduction and Confirmation Branch – Cashless Debit Card on the 14th of each month.

Annotate the request approval Fast Note to advise request is complete.
