Using the Provider search screen + Read more ...

Note:

  • Service Officers can use the Provider search screen to search for active and inactive providers
  • Service Officers can
    • search by a full name or partial name when searching by Provider details
    • select Clear to clear the search criteria
  • Service Officers may need to refine their search criteria if
    • the Your search matched X providers. Only 200 providers are displayed in alphabetical order below. Please refine your search criteria if needed information message displays and they cannot locate the relevant provider from the search results
    • the No results information message displays
  • All fields are optional unless marked with an asterisk

In the Provider search screen, Service Officers can search for a provider by selecting either:

  • Search by NAPS ID - default option
    • Key in the NAPS ID in the NAPS ID field
    • Select Search
    • The Provider details screen for the relevant provider will display
  • Search by BP ID - select the drop down menu to display this search option
    • Key in the BP ID in the BP ID field
    • Select Search
    • The Provider details screen for the relevant provider will display
  • Search by Provider details - select the drop down menu to display this search option
    • Key in the Provider name in the Provider name field. This is a free text field and all other fields are optional if this field is completed
    • Key in the ABN in the ABN field. This field only allows numerical values and all other fields are optional if this field is completed
    • Key in the Suburb in the Suburb field. All other fields are optional if this field is completed
    • Select the relevant State from the drop down menu in the State field. All other fields are optional if this field is completed
    • Key in the Postcode in the Postcode field. This field only allows numerical values and all other fields are optional if this field is completed
    • Select Search
    • The Provider search results screen will display
    • Select the relevant hyperlinked Providers NAPS ID in the search results table
    • The Provider details screen for the selected provider will display
  • Search by Recent searches in the Recent searches section
    • Select the relevant hyperlinked Providers NAPS ID in the search results table
    • The Provider details screen for the selected provider will display

In the Provider search results screen, Service Officers can also

  • type in a keyword in the Search bar to search for all records that contain the keyword
  • filter the search results by the provider’s operational status to narrow the search results
  • sort the search results in alphabetical order by Provider name
  • select the number of records to display from the Show records drop down list (bottom left corner of the screen)
  • go to a different page by selecting either:
    • First, Previous, Next or Last options, or
    • the relevant page number
  • select Back to search to return to the Provider search screen

Procedure ends here.

Using the User search screen + Read more ...

    • online (Aged Care Provider Portal (ACPP)) and

10

New user requests menu options + Read more ...

There are different processes in place depending on the new user request’s status. New user requests with:

  • either the Pending, Processed or Error status in ACSP means the request is incomplete and requires action (i.e. manual intervention) from Service Officers
  • either the Approved or Rejected status in ACSP means the request is completed and no further action is required from Service Officers
  • with the Cancelled status in ACSP means the request has been cancelled due to technical issues. See Step 21 for further details.

Service Officers can choose from the following menu options:

  • To register or reject the new user request - go to Step 11
  • To resolve a new user request with an Error status - go to Step 20
  • To view a new user request with a Cancelled status - go to Step 21

12

New user request registration process - automated checks menu options + Read more ...

Prior to continuing the new user request registration process, there are automated checks in place to ensure the ‘A user’ is not already registered.

There are 3 outcomes from these automated checks:

  • a 100 percent user match - go to Step 13
  • a partial user match - go to Step 14
  • a No duplicates found match - go to Step 15

13

A 100 percent user match result + Read more ...

Note:

  • If Service Officers selects Back, they return to the New user request summary page screen
  • The new user request’s status in the ACSP and the ACPP will remain as Pending and Being Processed status respectively

If the automated checks have resulted in a 100 percent user match, the New user request screen displays:

  • The user is already registered. The request must be rejected information message
  • New user details - details the Family name, Given name and email address of the new ‘A user’
  • Existing users - the table details the matched results i.e. displays the existing ‘A user’s’ details

Service Officers must select:

  • Reject request
    • The new user request is rejected
    • The new user request’s status in both the ACSP and the ACPP is updated to a Rejected status
    • Procedure ends here

14

A partial user match result + Read more ...

Note:

  • Service Officers can also select Back to return to the New user request summary page screen
  • Selecting the Back option will result in the new user request’s status in both the ACSP and the ACPP remaining as Pending and Being Processed status respectively

If the automated checks have resulted in a partial user match, the New user request screen displays:

  • the Please check the table below for similar users. If the user is already registered, please reject the application information message
  • New user details - details the Family name, Given name and email address of the new ‘A user’
  • Existing users - the table details the partial matched results e.g. displays the existing ‘A users’ details

To determine if the ‘A user’ is an existing user, Service Officers must cross check the new ‘A user’s’ details against all results in the Existing user details table. These checks are required to minimise the chance of creating duplicate ‘A users’ IDs (‘A’ numbers).

Is the new ‘A user’ an existing user?

  • Yes,
    • Select Reject Request
    • The new user request is rejected
    • The new user request’s status in both the ACSP and the ACPP is updated to a Rejected status
    • Procedure ends here
  • No,
    • Select Continue registration
    • The successful New user request receipt screen will display
    • Go to Step 16

15

A ‘No duplicates found’ match result + Read more ...

Note:

  • Service Officers can also select Back to return to the New user request summary page screen
  • Selecting the Back option will result in the new user request’s status in both the ACSP and the ACPP remaining as Pending and Being Processed status respectively

  • the No duplicates found. Please continue with the user registration information message

17

Creation of Tivoli Access Manager (TAM) access + Read more ...

IT Identity Access will need to create the Tivoli Access Manager (TAM) access for the ‘A user’ to log into the ACPP.

Service Officers must email these details to the delegated Service Officer responsible for submitting the TAM access. The delegated Service Officer will email IT Identity Access with the below details:

  • The user’s personal details (Given Name, Family Name and User ID)
  • The required access level as indicated in the ACSP

Note:

  • Details of all new user registrations (per service) should be compiled into 1 spreadsheet and forwarded to IT Identity Access section in one email to decrease turnaround time
  • When requesting TAM access for users, the user ID must be specified as a capital 'A', not a lowercase 'a'. The use of the lowercase causes access issues

    • Family name
    • Given name
    • Primary phone number
    • Secondary phone number
    • Email address
    • Service details
    • Access level
    • Start date
    • End date
    • Status - the status is Error

To resolve these requests, Service Officers must ensure the escalation process is actioned appropriately:

  • Complete the Aged Care Portal issues Incident form via mySupport
  • Once ICT has advised the error has been fixed, go to Step 7
