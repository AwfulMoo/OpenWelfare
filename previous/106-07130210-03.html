Rejecting a claim for JSP or YA (job seeker)

Portions of this process are for suitably skilled Service Officers only.

Step

Action

1

Making a decision to reject a claim for JSP or YA (job seeker) + Read more ...

Claim rejection

Code a streamlined rejection if:

  • the customer is in receipt of precluding payment (CPP), or
  • another payment is more appropriate (COP), or
  • the customer failed to supply documents (FSD)

Go to Step 3.

Note: if the customer has failed to supply their Tax File Number (TFN) or proof of Identity (POI), do not streamline reject FSD. Discretion should be used when an automatic rejection may be the most appropriate rejection option.

Rejecting a JSP claim FRC or FSD

If the Service Officer verbally requested more information and the customer fails to supply the information, reject the claim FRC or FSD.

When making an unfavourable decision, speak to the customer. If the customer has previously been contacted and more information was requested verbally, the customer does not need to be contacted again.

Make 1 genuine attempt to contact the customer. If the customer is subscribed, issue a pre-call notification (SMS only) between 8:00 am to 8:00 pm using the customer's local time. Let them know a Service Officer will call them today from a private number. See Centrelink letters online and Electronic Messaging.

Discuss with the customer:

  • explain the decision
  • give them a chance to provide more information, and
  • advise their review and appeal rights

If the customer wants to withdraw their claim before it is processed (whether or not they have attended an appointment), see Withdrawal of claims.

  • if the new claim is still in process, go to Step 2
  • if the claim has been finalised (record is assessed) but the customer is no longer qualified, for example, they have returned to work before their start day, or the customer wishes to withdraw their claim, go to Step 4

2

New claim in process + Read more ...

Service Officers must make sure the customer does not meet the qualification, payability, customer obligation, or mutual obligation requirements for the income support payment they are applying for and the decision to reject the claim is correct.

Note: if the customer is receiving a Disability payment from the Department of Veterans' Affairs (DVA), do not reject their claim based on income before checking whether they may be entitled to receive Defence Force Income Support Allowance (DFISA). The system will be able to identify these customers if the claim is correctly coded including DVA Disability payment details.

If a customer's claim is to be rejected either Failed to respond to correspondence (FRC) or Failed to supply documents (FSD), check if the customer is not vulnerable, at risk, or in crisis, see Claim submission exceptions in Circumstance Change Monitor (CCM).

If the customer has been advised verbally or in writing to give more information, there is no need to contact the customer. Record the decision on the customer's record.

Can the claim be streamline rejected?

  • Yes, code a streamlined rejection. Go to Step 3
  • No, reject the claim manually. Go to Step 5

3

Streamline rejection + Read more ...

To streamline reject the new claim using Process from the Claim Application screen, from the Transaction Summary (TS) screen:

  • select Process
  • from the Superkey field type (BA) to navigate to the Benefit Action (BA) screen
  • select Add New
  • on the Create Benefit Action screen, Benefit Type Code, Benefit Action and Date of Effect will automatically default
  • select the streamline indicator box. A tick will appear
  • in the Reason field select appropriate reason code from the drop down box
  • select Save
  • select Assess this will navigate to the SWE screen
  • once all errors, if any, have been addressed
  • select Assess, this will navigate to the Entitlement (ELD) screen, make sure the correct rejection reason is displaying
  • select Finalise, the finalisation DOC will appear, if extra information is required add this to the DOC. For reason FSD, the Service Officer must list all outstanding documents
  • select Finish

Note: for streamline rejections, any changes to circumstances that may impact a customer's and/or partner’s current payments or benefits must be updated after the streamline rejection is completed.

Procedure ends here.

4

Customer's claim status is 'Assessed' on the system + Read more ...

If a customer has been granted a benefit or allowance but has not yet received a payment and advises they have returned to work or wish to withdraw their claim, manually reject the claim.

Note: if the customer's payment status is 'current', cancel the payment at the customer's request.

Go to the Benefit Action (BA) screen and complete the following:

  • in the Srv Rsn field code ‘JSP’ or 'YA'
  • in the Action field code 'REJ' or 'CAN’
  • in the Reason field code 'CLR' for a withdrawn claim or 'NUE' for returned to work
  • code the Source and DOR fields
  • press [Enter]. If the Date of Effect (DOE) does not automatically default to the start day, staff must manually enter the DOE. This is usually the date of claim

Record the details of decision on a DOC, including details of any waiting periods applied to the claim.

Go to Step 6.

5

To reject claim manually + Read more ...

Reject a new claim using Process from the Claim Application screen:

If the system does not automatically reject the claim, make sure all coding is completed for the claim assessment.

  • From the Superkey field type (BA) to navigate to the Benefit Action (BA) screen
  • Select Add New
  • On the Create Benefit Action screen, Benefit Type Code, Benefit Action and Date of Effect will automatically default
  • In the Reason field select appropriate reason code from the drop down box
  • Select Save
  • Select Assess this will navigate to the SWE screen
  • Once all errors, if any, have been addressed
  • Select Assess this will navigate to the Entitlement (ELD) screen, make sure the correct rejection reason if displaying
  • Select Finish, the finalisation DOC will appear, if extra information is required add this to the DOC
  • Select Finalise

6

Check if manual letter is required + Read more ...

Check if the rejection reason needs a manual letter. A manual letter will be required if a Manual Follow-up (MFU) activity has been created on the Activity List (AL) screen after the rejection activity is finalised.

  • If there is an MFU requiring a manual letter be sent to customer, go to Step 7
  • If there is no MFU, procedure ends here

7

Send manual letter + Read more ...

Send a manual letter (Q134) to the customer giving the reason(s) for rejecting the claim and advising their review and appeal rights.

Sometimes an automatic advice will reject. This creates an advice activity on the AL screen. Selecting the activity may display the letter. Others must be selected from the Text Find (TF) screen.

8

Follow up actions required + Read more ...

  • If a claim is finalised in a service centre earlier than scheduled, Service Officers are to check appointment profiles in Customer First. Cancel these appointments
  • Record details on a DOC, including details of any waiting periods applied to the claim
  • Make sure the customer is fully aware of the implications of not claiming, for example, entitlements and access to a Health Care Card (HCC)
  • Change of circumstances identified within claim which are related to a current payment, i.e. claiming for JSP, streamline reject but identifies children have left care within claim, create a Fast Note or ACTDOC. See Creating documents (Fast Note, Callbacks and ACTDOCs) and direct to appropriate payment team
