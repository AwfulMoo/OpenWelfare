Assessing the statement by claimant

Table 1: This table describes the steps for assessing if the returned statement by claimant is complete prior to processing in the CCMS, and the actions to take when it is incomplete or missing information. Steps to request missing information or documents should have occurred at mail validation. This table also describes actions to be taken where Section 33D requirements have not been met.

Step

Action

1

Section 33D due date + Read more ...

Has the Section 33D due date passed?

Refer to Notice of past benefits and notice of change for Medicare Compensation Recovery.

  • Yes, go to Step 2
  • No, go to Step 3

2

APO Processing + Read more ...

Has the case been set up as an Advance Payment Option (APO) for processing?

  • Yes, go to Step 1 Table 2
  • No, re-route the case to the Judgment queue to continue processing. Procedure ends here

3

Assess the statement by claimant + Read more ...

If the statement by claimant has not been returned. Go to Step 1 Table 2. Procedure ends here.

If statement by claimant has been returned, check that the document includes:

  • All pages of the Medicare history statement (where issued) and the declaration
  • Name of the injured person or claimant
  • Address of the injured person or claimant
  • A tick against 1 box regarding nursing home residential care or home care
    Note: if both boxes are ticked the declaration is invalid
  • Valid signature by the injured person or claimant
  • Supporting documents on file or attached if required where claimant has signed

Has all required information been provided?

  • Yes,
    • Go to Step 1 Table 2 table
    • Procedure ends here
  • No,
    • If the document has missing pages or the document is illegible, go to Step 4
    • If the document is incomplete/invalid (unclear which services are ticked declaration not signed, no tick against the nursing home, residential or home care question etc), go to Step 6

4

Document missing pages or illegible + Read more ...

Is the document missing pages or illegible? (check if a ‘missing information scanning coversheet’ is enclosed with the document)

  • Yes, check if the document was received by post, email or fax:
    • If received by post, treat the document as incomplete, go to Step 6
    • If received by email or fax, go to Step 5
    • If the missing or illegible information is not found, treat the document as incomplete. Go to Step 6
  • No, go to Step 6

5

Determine if document is to be rescanned + Read more ...

Was the statement by claimant received by email or fax?

  • Yes, check the internal email or fax box for the original document:
    • If the missing information was not received or the document is illegible, treat the document as incomplete. Go to Step 6
    • If the missing information was received or the document is legible, rescan a copy to the case and suspend for 7 days. See Step 5 Table 3 to suspend work item
  • No, the document was submitted by post:
    • Send the ‘ACTION REQUIRED - Scanning request (48 hours)’ email template
    • See Step 5 Table 3 to suspend work item for 7 days

6

Document invalid or missing required information + Read more ...

Has a ‘z2452 - Provide more information to process the Medicare history statement and declaration’ letter already been issued at mail validation?

  • Yes, if the missing information has:
    • Been received the document can now be processed as complete, go to Step 1 Table 2
    • Not been received the document is incomplete, See Step 1 Table 2
  • No, go to Step 7

7

Determine stage of the claim + Read more ...

Has the case reached judgment or settlement?

  • Yes, go to Step 8
  • No,
    • If a reimbursement arrangement has been made issue a ‘z2452 - Provide more information to process the Medicare history statement and declaration’ letter. Select letter options to identify what is missing or incomplete. If both the Medicare history statement and declaration have been returned blank, both need to be reissued
    • The letter does not provide an extension of time, it asks for the information to be provided as soon as possible
    • See Step 1 Table 2
    • For cases not settled and no reimbursement arrangement, go to Step 9

8

Check for advance payment + Read more ...

Has an advance payment been made?

  • Yes, go to Step 9
  • No,
    • Issue a ‘z2452 - More information is needed to process a compensation claim’ letter. Select letter options to identify what is missing or incomplete. If both the Medicare history statement and declaration have been returned blank, both need to be reissued
    • The letter will not provide an extension of time, it asks for the information to be provided as soon as possible
    • Treat the document as incomplete
    • Go to Step 1 Table 2

9

Additional time required to process a compensation claim + Read more ...

If the statement by claimant is missing supporting documents, i.e. Last Will and Testament, do not grant an extension of time.

An extension of time is considered where an advance payment has been made and where there is information missing in the Statement by Claimant, such as:

  • Not clearly completed
  • Missing pages
  • Missing valid signature and date on declaration
  • Not able to be matched to a case
  • Does not have a tick for the nursing home question

If the original due date for the statement by claimant has not yet been reached

Issue ‘z2452 - Provide more information to process the Medicare history statement and declaration’ letter. Select letter options to identify what is missing or incomplete. If both the Medicare history statement and declaration have been returned blank, both need to be reissued.

The options selected in the letter macro will:

  • Grant a formal extension of time for the missing information to be submitted
    • Go to Step 5 Table 3 table for how to apply the extension to the case in CCMS, or
  • Request supporting documents be supplied as soon as possible
    • An extension cannot be granted
    • Treat the document as incomplete
    • See Step 1 Table 2

If the original due date for the statement by claimant has passed

A manual letter is not issued at this stage. Treat the document as incomplete, go to Step 1 Table 2.

Processing statement by claimant

Table 2: This table describes the process for actioning the statement by claimant in CCMS. Complete this process after checking the validity of statement by claimant.

Step

Action

1

Determine if case is to be referred to supervisor + Read more ...

Refer case to supervisor:

  • If the injured person or claimant’s name has changed in the Consumer Directory Maintenance System (CDMS), or
  • 'Other information' needs to be updated, for example:
    • If a minor has turned 14 years of age and the parent is no longer the claimant
    • To add or change a case flag, or
    • If a valid notice of past benefits exists and an amendment is required so the 'dates' can be deleted

Does the case need to be referred to supervisor?

  • Yes,
    • Select 'Yes' in the workflow
    • See Step 6 Table 3 to add a case note before releasing the work form
    • Go to the CCMS supervisor queue
    • Select the case
    • Update the details
    • Return to the 'Statement by claimant' workflow to continue processing until the case is suspended or closed
  • No, go to Step 2

2

Identify if the case is an advance payment + Read more ...

Is this case an advance payment?

  • Select APO or Non APO (if applicable), go to Step 3

Note: this field is only used to identify Advance Payment Option (APO) or Non Advance Payment Option (Non-APO).

3

Claimant's address + Read more ...

Can the claimant’s address be verified?

  • Yes,
    • Only select 'Yes' if Services Australia has been advised of a change of address (over the phone or in writing) or if other agency records have been updated
    • Enter any new address information in the Address field box
    • Go to Step 4
  • No, go to Step 4

4

Process statement by claimant + Read more ...

Has the document been assessed as complete?

  • Yes, (or where the Section 33D date has passed) the screen will be rebuilt depending on the progress or details of the case
    • If the Medicare history statement and declaration was returned or the Section 33D date has passed, go to Step 5
  • If the Declaration only was returned (after a CP006/z2060) where no Medicare services claimed since date of injury, go to Step 7
  • No,
    • If the document has been assessed as incomplete, go to Step 5
    • If the s24(4) date has passed. See Step 6 Table 3, to refer the case to the notice of judgment or settlement queue to be finalised. Then go to Notice of judgment or settlement for Medicare Compensation Recovery, Table 2

Note: where an extension of time has been requested, see Step 5 Table 3

5

Statement by claimant details + Read more ...

If the case has settled or a reimbursement arrangement has been made, only services between the following dates are to be included in the notice of charge:

  • The date of injury or illness to the date the amount of compensation was fixed by judgment or settlement
  • The date of injury or illness to the date the reimbursement arrangement was made, or
  • The specified liability period for medical expenses or determinations in relation to the compensable injury or illness by the Administrative Appeals Tribunal (if applicable)

Enter statement by claimant details

  • Select the black arrow and the Services data entry screen will open

Where the Section 33D date for the case has passed, remove any services previously selected or deemed:

  • Remove any previously ‘deemed’ services by selecting ‘undeem’
  • Remove any previously selected services by double clicking the red ticks against the line numbers
  • Select the ‘Update’ button once complete

If the statement by claimant has been returned

  • Enter verified services into the Services data entry screen by double clicking the 'X' matching the corresponding line numbers (selected ticks will be red against the line numbers once selected)
  • Alternatively, enter a range of line numbers in the 'Items' box at the top of the Services data entry screen and press [Enter] to select the verified services
  • Cross reference if required - do not include any indicated services that appear on a verified notice of charge used to close another case for the claimant
  • Select the ‘Update’ button once complete

On an expired notice, if a clear statement by claimant has been returned with amendments and approved by the delegate:

  • any previously verified services can be removed if clearly marked as not related by:
    • a solid line crossed through the service,
    • some of the previously verified services re-ticked and others left unticked, or
    • have a comment beside the service it does not relate
  • scroll to the top of the list of services and double click the 'X' matching the corresponding line numbers (de-selected ticks will revert box colour from red to grey)
  • enter verified services into the Services data entry screen by double clicking the 'X' matching the corresponding line numbers using the mouse (selected ticks will be red against the line numbers once selected)
  • alternatively, enter a range of line numbers in the 'Items' box at the top of the Services data entry screen and press [Enter] to select the verified services. If other closed cases were found during mandatory checks at assessment stage, perform cross referencing to ensure services are not duplicated by removing any that appear on the verified notice of charge used to close other case(s)
  • Select the 'Update' button once complete

If the statement by claimant has not been returned (or is incomplete):

  • All ticks should be yellow against the corresponding line numbers
  • Click the 'DEEM' button at the top of the Services data entry screen to ensure all line numbers have been deemed in yellow
  • Select the 'Update' button once complete.

Do nursing home benefits apply - Select 'Yes' or 'No' per the response on the declaration. Always select ‘No’ if the Section 33D date has passed.

Update the Line items - Select 'Yes'.

Date document received - Enter the date stamp on the document or today’s date if the document was not returned, go to Step 6.

6

Prepare calculations for notice of past benefit + Read more ...

If the case has settled with a previous voluntary payment for a case that has not been added, then select:

  • No - re-route the case to the ad-hoc recovery queue and add the voluntary payment first and then return to this work step to continue processing the case, see Step 5 Table 3
  • Yes - a system display of the past Medical expenses, any payments previously paid to the agency and the total amount shown on the notice of past benefits is returned

If 'Yes' was selected check nursing home, residential or home care benefits

For urgent care cost checks, for example, where case is approaching s33D or s24(4) date or for other case specific reasons, complete a manual care costs check. See How to identify recoverable care costs.

  • Where a manual care costs check is required, complete the ‘Care costs check request’ email template and suspend the case for 14 days, see Step 5 Table 3. The Resources page contains a link to the email template

For all other care cost checks:

  • Name of the residential care facility - Enter 'Not supplied'
  • State of the residential care facility - Enter the 'State' the injured person resides in
  • Suspend work item - Select 'Yes'
  • A system message will appear advising ‘case will be pended for 14 days awaiting nursing home, residential or home care check from the agency'
  • Do not release the work form yet, see Step 7 Table 3 to add a case note

Procedure ends here.

If 'No' was selected to check nursing home, residential or home care benefits

  • If the case has not settled
    • Issue notice of past benefits - Press [Enter]
    • Suspend work item - Select 'Yes', do not release the work form yet, see Step 7 Table 3 to add a case note
  • If the case is a Reimbursement arrangement
    • Issue notice of charge - Press [Enter]
    • Suspend work item - Select 'No', see Step 5 Table 3 and refer the case to the Process reimbursements queue to be finalised, go to Notice of reimbursement arrangement for Medicare Compensation Recovery'

Procedure ends here.

  • If the case is Non advance payment
    • Issue notice of charge - Press [Enter]
    • Suspend work item - Select 'Yes', do not release the work form yet, see Step 7 Table 3 to add a case note

Procedure ends here.

  • If the case is an advance payment go to Step 8

7

Date document received + Read more ...

Enter the date stamp on the document or today’s date if the document was not returned.

  • Check if Nursing home, residential or home care benefits apply
  • Select 'Yes' or 'No' per the response on declaration
  • Select 'Yes' is selected to check nursing home, residential or home care benefits
  • Name of the residential care facility - Enter 'Not supplied'
  • State of the residential care facility - Enter the 'State' injured person resides in
  • Suspend work item - Select 'Yes'
  • A system message will appear advising ‘case will be pended for 14 days awaiting nursing home, residential or home care check from the agency'
  • see Step 7 Table 3 to add a case note, before releasing the work form

Procedure ends here.

If 'No' is selected to check nursing home, residential or home care benefits

  • If the case has not settled
    • Issue notice of past benefits - Press [Enter]
    • Suspend work item - Select 'Yes', do not release the work form yet, see Step 7 Table 3 to add a case note
  • If the case is a Reimbursement arrangement
    • Issue notice of charge - Press [Enter]
    • Suspend work item - Select 'No'
    • See Step 5 Table 3, refer the case to the Process reimbursements queue to be finalised
    • Go to Notice of reimbursement arrangement for Medicare Compensation Recovery

Procedure ends here.

  • If the case is Non advance payment:
    • Issue notice of charge - Press [Enter]
    • Suspend work item - Select 'Yes'
    • Do not release the work form yet
    • See Step 7 Table 3 to add a case note
  • If the case is voluntary payment where information is missing:
    • Issue notice of charge with voluntary payment included - Press [Enter]
    • Suspend work item - Select 'No'
    • See Step 5 Table 3 and refer the case to the judgment or settlement queue to be finalised
    • Go to Notice of judgment or settlement for Medicare Compensation Recovery

Procedure ends here.

  • If the case is advance payment, go to Step 8

8

Apportionment of compensation between the parties + Read more ...

Did the finalisation fix the amount of compensation on the basis that liability for the injury would be apportioned between the parties?

Note: notice of judgment or settlement can be located in the view folder

  • Yes,
    • Select 'Yes'
    • Enter percentage of apportionment attributed to the injured person indicated on the notice of judgment or settlement form
    • Answer the question is any amount of compensation payable by periodic payments as indicated on the notice of judgment or settlement form
    • Go to Step 9
  • No,
    • Select 'No'
    • Answer the question is any amount of compensation payable by periodic payments as indicated on the notice of judgment or settlement form
    • Go to Step 9

9

Past medical costs and care costs + Read more ...

If the case has been finalised by judgment, only enter amounts if a copy of a court document has been provided to support the amount fixed.

  • Enter the amounts fixed as indicated on the notice of judgment or settlement form
  • Enter 0.00 if the amounts are nil
  • Go to Step 10

10

Future medical costs and care costs + Read more ...

Have future medical costs and nursing home, residential or home care costs been identified?

  • Yes, enter the amounts awarded as indicated on the notice of judgment or settlement form for both questions, go to Step 11
  • No, go to Step 11

11

Amount payable to Services Australia - Centrelink + Read more ...

  • Enter amount payable as shown on the notice of judgment or settlement form
  • Enter 0.00 if the amount is nil
  • See Step 1 Table 3

Finalisation actions for a statement by claimant

Table 3: This table describes the process for finalising a statement by claimant in the CCMS.

Step

Action

1

Case is ready for reconciliation + Read more ...

Note: if a refund is required see Payments, refunds and debts for Medicare Compensation Recovery Process page Tables 1 to 3

  • Select the green arrow and CCMS will attempt to reconcile with SAP FI
    • If the payment is successfully reconciled, a message is returned confirming 'DONE'. Enter the date the advance payment was received, then Press [Enter]. Go to Step 2
    • If the payment fails to reconcile, a message is returned confirming 'FAILED'. The case should be referred to finance to locate the advance payment in SAP Financial system and allocate the payment to the WIN
  • Select 'Refer case to finance' then Press [Enter]. Do not release the work form yet, go to Step 7 to add a case note

2

Enter the amount indicated in point 15 of the Certificate of Details of Settlement/Judgment + Read more ...

  • Enter the amount in point 15 as instructed in the 'calculated amount to be recovered by Services Australia' message which appears on screen, then Press [Enter]
  • CCMS will check the calculation and will display 'correct' or 'incorrect'. If incorrect, re-check your work and amend as required until the check comes back as correct
  • If the amount to be recovered is a:
    • positive amount, this is a debt to the agency
    • negative amount, this is a refund owed by the agency
  • Go to Step 3

3

Issuing notice of past benefits + Read more ...

Issue notice of past benefits:

  • Select the arrow to issue an advance payment notice of past benefits
  • Update CICS with finalisation details - Press [Enter]. Do not release the work form yet
  • Notice of charge (CP055M/z1931) letter issues to the injured person with details of any reconciliation

If within the Section 33D date, go to Step 4

If the Section 33D date has passed:

  • complete the ‘S33K report’ template on the Resources page
  • if the refund is going directly to the injured person or claimant and EFT details have been supplied complete the ‘EFT to finance (valid advance payments)’ form and select submit. See the Resources page for a link to the template

Go to Step 7, to add case note.

4

Assess for section 29(3) + Read more ...

If there is a:

  • refund owed by the agency, go to Step 5
  • debt to the agency, assess for s29(3)

Section 29(3) applies if the notice of judgment/settlement was provided to the agency more than 28 days after the date of judgment or settlement.

Does section 29(3) apply?

Yes, case is a s29(3):

  • Close work item by selecting the arrow to issue an advance payment notice of past benefits
  • Update CICS with finalisation details, then press [Enter]
  • Delete the CP055M/z1931 letter from the print queue
  • Draft:
    • z2464 'Non-compliance with notification of settlement' to the compensation payer for residual debt
    • only for a verified charge z2637 'Compensation related medical benefits and/or care services' letter
    • cover letter z2451’Compensation claim document enclosed’ to claimant or authorised third party with copies of z2464 and z2637, if applicable
  • Access the Referring a case to Compensation finance email template on the Resources page
  • Send the email template to Medicare Compensation Recovery Team Leaders with attached letter/s for quality checking
  • Add a case note, go to Step 7
  • After quality clearance:
    • send the email to finance
    • post letters
    • scan all correspondence to the WIN as backlog

No, go to Step 5

5

Determine if work item is to be suspended + Read more ...

Suspend this work item?

  • Yes, a 'suspend until' date must be entered. If suspending case the default date is 28 days from date of processing
    • Enter date as 28 days from date of processing and Press [Enter]. Do not release the work form yet, go to Step 7 to add a case note
  • No, go to Step 6

6

Go to next transaction + Read more ...

  • Yes, select which work queue the case needs to go to for further action, then Press [Enter]. Do not release the work form yet, go to Step 6 to add a case note
  • No, never select 'No' for both 'suspend' and 'go to next transaction', if this is done the case will not suspend and will simply reactivate at the bottom of the current work queue.

If Initiate extension processing is required after sending case to extension work step

Note: before issuing an extension, assess if the case qualifies for an extension of time outlined above

  • Refer Case to supervisor queue? Select 'No'
  • Is this case a Class Action Suit? Select 'No'
  • Verify claimant’s address?
    • 'Yes' - if an address change is required
    • 'No' - if an address change is not required
  • Process an extension request? Select 'Yes'
  • Select type of extension?
    • Initial request - 1st extension, or
    • Re-application - 2nd extension
  • The extension request is? Select 'granted'
  • Enter new date statement is due to be returned? Enter new agreed return date
  • Suspend this work item? Select 'Yes', do not release the work form yet, go to Step 7 to add a case note

7

Add a case note + Read more ...

If the Case Notes screen has not opened automatically, select the Add notes button from the list on the right hand side of the main CCMS screen.

  • The top box shows all notes previously recorded on this case
  • In the blank bottom box, record the actions taken, information requested and from whom, reasons for a decision. Use the case notes macro or standard case notes where possible
  • Select Add to save notes. A confirmation box will appear, select 'Yes' to add case note
  • Press [F4] to release the work item, and
    • If this is the end of the process, case will suspend or close, or
    • If further actions are required in other CCMS work queues, follow through on case until further actions are complete
