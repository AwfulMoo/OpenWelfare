Preparing for and making the outbound recovery call

Table 1

Step

Action

1

Debt recovery script + Read more ...

Use the Debt Recovery script when actioning a debt.

Thoroughly investigate the record to understand what actions have been completed.

For:

  • dialler outbound attempt, go to Step 2
  • manual outbound attempt, go to Step 3
  • unable to contact due to no contact number on TDS, go to Step 4
  • investigations after notice has been sent to customer, see Table 2

2

Automated dialler attempt + Read more ...

Did the attempt/s result in a successful engagement with the customer?

  • Yes, go to Step 5
  • No:
    • DOC the attempt/s using the Unsuccessful Engagement Fast Note
    • Procedure ends here

3

Manual attempt + Read more ...

Search for outstanding activities using Workload Management in Customer First. Complete all outstanding work.

Make a reasonable effort to contact the customer.

Note: when leaving messages follow Safeguarding customer privacy when raising and recovering debts.

Did the attempt result in a successful engagement with the customer?

  • Yes, go to Step 5
  • No:
    • DOC the attempt using the Unsuccessful Engagement Fast Note
    • Go to Step 4

4

Send a request to contact letter + Read more ...

See Safeguarding customer privacy when raising and recovering debts, before issuing any debt notices. Take the appropriate actions to protect the customer's privacy.

Send a 'Request client to contact office by phone' (Q164) letter. The script creating the Q164 letter may generate a held activity, finalise this activity.

Hold the No Arrangement (NOARR) activity for 23 days to give the customer sufficient time to respond.

DOC the record and include the address to which the letter was sent to.

When the No Arrangement (NOARR) activity becomes due, go to Table 2.

5

Discuss the reason for the call + Read more ...

Safeguard the customer's privacy before discussing the reason for the call.

Tell the customer that the call: 'may be recorded or listened to for quality and coaching purposes'. See Calling a customer or returning a customer's call for more information.

Determine an appropriate arrangement. Consider:

  • Repayment history and any previous missed payments
  • When the last repayment was made
  • Customers willingness to negotiate
  • Customers financial capacity to pay

See Arranging cash repayments to recover debts and Negotiation of debt repayments with customers for more information.

Was an arrangement made?

  • Yes:
    • Finalise the arrangement
    • Procedure ends here
  • No, go to Step 6

6

Customer refuses to enter into a reasonable repayment arrangement + Read more ...

Did the customer refuse to enter into a reasonable repayment arrangement?

  • Yes:
    • DOC the contact using the Successful Engagement refusal to pay Fast Note
    • Include any relevant information about the discussion
    • See Table 2
  • No:
    • DOC the contact using the Successful Engagement Fast Note
    • See Temporary write off of Centrelink debts
    • Procedure ends here

Gathering information for the purposes of debt recovery

Table 2

Step

Action

1

Consider garnishee options + Read more ...

The system will not allow the cancellation of a No Arrangement (NOARR) activity until a payment arrangement is in place.

If customer contact has been unsuccessful, consider if the debt can be recovered by garnishee.

Does the debt meet the criteria for a garnishee?

  • Yes, refer to the garnishee team. Add keyword to the NOARR debt activity:
    • RCVGRNWG for wage garnishees, or
    • RCVGRNBK for bank garnishees
    • Procedure ends here
  • No, go to Step 2

2

Garnishee criteria not met + Read more ...

Review the case to determine if it is suitable for other recovery or non-recovery actions.

Is the case suitable for other recovery or non-recovery?

  • Yes, consider options:
    • Temporary write off of Centrelink debts, or
    • Referring debts to an External Collection Agent (ECA)
    • Procedure ends here
  • No, go to Step 3

3

Recovery options exhausted + Read more ...

If recovery options have been exhausted, run the Fast Note:

  • Select Auto Text. Use Debt Recovery > Unsuccessful Engagement > Recovery Options Exhausted
  • DOC the decision and include the exhausted recovery options
  • Link the NOARR activity to the Fast Note - select 'Link to existing activity' in the Document Completion: field
  • Complete the Fast Note

Retain the work items and exit the customer's record:

  • Retain the work item in the user inbox until the Priority shows as '0' - this generally takes less than 20 minutes once the user has exited the customer record
  • When the Priority resets to '0', unassign the work item. Use the reason 'Service Support Referral'
  • This prevents the activity from re-allocation
