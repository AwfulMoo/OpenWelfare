NCL activity preventing updates + Read more ...

Not all NCL activities need to be cancelled to process other activities.

Sometimes a claim activity must be cancelled to process other updates on the customer's record. This is a result of 'task locking' - provisional updates in the NCL activity conflict with updates being made in the new activity.

If the current update is urgent or affects the customer's payments and cannot be left until the NCL activity is finalised, it may be necessary to cancel the NCL activity.

  • If a claim in Process Direct is preventing updates in Customer First, go to Step 2
  • For other claims:
    • Collect all relevant claim information, see Table 2
    • Cancel the started NCL activity
    • After completing updates, re-index or regenerate the NCL activity, depending on the claim type, go to Table 2

2

Claim in Process Direct preventing update in Customer First/ Customer Record + Read more ...

When a new claim (NCL) transaction is assessed in Process Direct (PD), the data is sent to the legacy system (ISIS). An activity on the Activity List (AL) screen in Customer First/ Customer Record will display.

If the activity is preventing Service Officers from completing other updates on the customer's record and the claim status is:

  • On Hold or In Process, the claim will need to be cancelled from the AL screen in Customer First/ Customer Record.
  • QMA (Quality Management Check), arrange for someone to complete the quality check. See Urgent QMA process for Smart Centres, Labour Hire Service Delivery Partners. Then make the updates

Note: Child Care Subsidy (CCS) claims have a separate process, see Processing Child Care Subsidy (CCS) claims.

For claims On Hold or In Process

In Customer First identify if the claim on the AL is processed in PD:

  • select the CRN/Business Partner link in the customer header
  • select the Cases tab, then the Super Case for business partner link under Description
  • select Transactions tab
  • select the Claim ID
  • If error ‘Please use Process Direct to complete the function’:
    • doesn't display, the claim is not processed in PD. Procedure ends here.
    • displays in the top right corner, the claim will need to be cancelled. Go to Step 3.

3

Cancel NCL + Read more ...

On the AL screen in Customer First:

  • enter 'C’ against the activity and press [Enter]
  • select ‘C’ on the AYC screen to cancel all provisional changes
  • enter reason PPD – Process in PD, this will:
    • Cancel the Customer First/Customer Record claim
    • Return the claim to Process Direct to be processed
    • All provisional data must be re-coded
    • For further information see Process Direct navigation, common screens and functions
  • Select [Enter]. The NCL activity is now cancelled from AL
  • Complete the other update

Procedure ends here.

2

Re-indexing a claim + Read more ...

This process applies to the following claims:

  • Paper claims not processed in Process Direct
  • ABSTUDY claims lodged via the ABSTUDY Claiming Tool not processed in Process Direct
  • DSP claim review of decision - where the decision has been set aside or varied

All activities must be a replica of the original claim activity. This includes the source and date of receipt (DOR).

Go to original claim activity

  • Select STArted NCL activity on the Activity List (AL) screen, or
  • Go to Event Summary (ES) screen

Check and note from original claim activity:

  • keywords, from the Keywords (KWD) screen
  • Region Code (ROC)
  • customer details
  • claim details
  • source
  • DOR

Manually re-index claim

In Customer First or Customer Record

  • Make sure the correct System displays. For claims in ISS System, key SV and the Payment Service Reason in Next. For example, ABSTUDY claims, key SVABY
  • Go to Name Search (IN) screen, select Continue
    • customer surname
    • customer first name or initial
    • date of birth (DOB)
  • In Claims details section:
    • relevant service reason
    • activity 'NCL', exception is for an Aged Care Assessment where activity is ‘HOM’
    • same Source as original claim
    • same DOR as original claim
  • Select Continue
  • Select the customer from the list, then Continue
  • On Person Details (PED) screen key 'Y' to confirm, then Continue
  • Go back to AL screen
  • Select 'STA NCL activity', select (O) 'Transfer activity to another site/region', then Continue
  • Change 'To Regional Office' code to transfer this activity back to the Region Code (ROC) for the original NCL activity
  • Add all keywords together with re-index keyword REI. Up to 3 keywords can be entered. If more than 3 keywords are needed, add additional keywords by selecting the 'STA NCL' activity from the AL screen, then select either (O) 'Transfer activity to another site/region' or (V) 'View activity details on AY screen'
  • In Notes section add reason claim was re-indexed
  • Select Continue

Procedure ends here.

3

Re-index SAP online and paper claims processed in Process Direct + Read more ...

Select the Customer Summary tile:

  • on the Search Customers screen, enter the Customer Reference Number (CRN) in the CRN field and select Go
  • select the CRN link from the Search Results
  • select Transactions icon
  • select All Transactions button
  • select the Transaction ID of the completed claim that needs to be reassessed
  • the Claim Application page displays
  • select > Re-index Claim

In Process Direct CPTv3

  • Complete the following fields:
    • Primary Task: Re-index Claim
    • Secondary Task: leave blank
    • Date of receipt: will default date of claim submission
    • Service Reason: select payment type
    • Appeal Number: if appropriate
    • Task List: Re-index Claim
    • select Create

In Process Direct CPTv5

Re-indexed claims will generate a new CoC Interaction transaction ID and display as Re-index Claim to be allocated for processing.

Did the > Re-index Claim icon display?

  • Yes, see specific claim processing files to process the claim. Procedure ends here
  • No, go to Step 4

4

No Re-index Claim option in Process Direct + Read more ...

The Re-index Claim function will not display if a new CoC Interaction transaction ID has already been created.

If Re-index Claim is not displayed in :

  • Check if the claim has already been re-indexed and a new Change of Circumstance (CoC) Re-index claim transaction ID is available on the customer's record. See Process Direct navigation, common screens and functions
  • Check the Status of the finalised claim being re-indexed. See Process Direct navigation, common screens and functions. Claims with a Not Required status cannot be re-indexed. Manually create a SOA shell to reassess the claim:
    • Go to the Launchpad
    • Select SOA Shell tile to go to the SOA Shell Creation screen
    • Select Benefit Type, e.g. Age Pension
    • Enter Customer CRN
    • Enter the appropriate Date of Receipt based on initial claim
    • Select Create Claim
