Actions required before processing the claim

For staff processing ABSTUDY apprentice or tertiary student claims in Process Direct

Table 1

Step

Action

1

Locate claim + Read more ...

Claims are allocated via the Work Inbox.

See Process Direct navigation, common screens and functions for help with launching the tool and accessing work items.

Select the work item to view Customer and Claim Information.

Online claim or Assisted Customer Claim (ACC) submitted

An ABSTUDY claim work item is generated on the customer's record. This contains all the claim information provided by the customer online or entered into ACC.

2

Claim Application screen + Read more ...

Select the icons to display the customer's information in detail.

These sections all form part of the pre-check procedure along with the other details that must be reviewed before starting to process the claim.

  • Select the Status icon to amend the claim status if necessary. If the status is Submission Failed, change this to In Process:
    • select Edit
    • select In Process
    • select Save
  • Select the Documents icon to open relevant scanned documents
  • Select the Interaction icon to review relevant DOCs
  • Select to review the claim

3

Claiming Incidentals Allowance only? + Read more ...

Is the customer claiming Incidentals Allowance only (as an Australian Apprentice or tertiary student)?

  • Yes, go to Step 5
  • No, go to Step 4

4

Relationships + Read more ...

Member of a Couple assessments (SS284 or SS293 forms) are required for claims where the customer advises they are living with their ex-partner as separated under one roof (SUOR).

Where either a Relationship Details (SS284) or Relationship Details - Separated under one roof (SS293) form is issued to the customer and sharer (where required), a MoC assessment must be undertaken, before the claim is finalised. See Member of a Couple (MoC) and Separated under one roof (SUOR) assessments.

If applicable, partner details must be verified verbally or in paper form prior to finalising the online claim or ACC. For more information, see Change in relationship from single to partnered.

Relationships must be completed and confirmed before starting to process the claim.

If relationships are updated after processing has been started, the claim must be regenerated.

Select the Relationships icon to check/update the customer's links. Linking must be completed before claim processing is started:

  • If an update is required, select the person from the Claim Relationships tab
  • To add a new relationship (if there are no details under the Claim Relationships or Known Relationships tabs), select New

For help, see Process Direct navigation, common screens and functions.

5

Is the claim ready to process? + Read more ...

  • Yes, see Table 2
  • No, go to Step 6

6

Further action required + Read more ...

For a compensation referral, see Coding Compensation and damages (MOD C) to request a clearance.

Additional supporting documents required

When assessing a claim code any evidence the customer has provided with their claim.

To complete the following actions, select and then the relevant option:

  • Request documents
    • Do not send a second request for documents already requested
    • For help, see Table 6 in Circumstance Change Monitor (CCM)
    • If the requested documents have not been provided (within 14 days of the date the request was made), finalise the claim using the streamline rejection. See Step 3 in Table 2
  • Re-index a claim with a status of Completed
  • Regenerate a claim, excluding claims with a status of Approved
  • Referral, for example, to a Complex Assessment Officer (CAO), International Services. The claim will automatically be placed on hold for 14 days

Actions required to process an ABSTUDY claim

For staff processing ABSTUDY apprentice or tertiary student claims in Process Direct

Table 2

Step

Action

1

Start processing/update claim + Read more ...

From the Claim Application screen, select Process.

Is the customer claiming Incidentals Allowance only (tertiary student or apprentice)?

  • Yes, go to Step 2
  • No, and claim relationships are:
    • correct, go to Step 2
    • incorrect, check and update Relationships if necessary. See Step 4 in Table 1

2

Claim Processing screen + Read more ...

Icons across the top of the page are in the same order as the Claim Overview information.

Selecting the icon will display tables that may contain provisional data and/or confirmed data. This information should have been automatically loaded from the claim. If information has not loaded correctly, this will be displayed as an error, warning or message.

For help, see Process Direct navigation, common screens and functions.

Select Simulate at any time to help identify and correct errors or once all relevant updates have been completed.

For assistance to determine eligibility, see:

  • Assessing a claim for ABSTUDY Allowances and Part-time Award
  • Assessing a claim for ABSTUDY Schooling B Award, Tertiary Award and Masters and Doctorate Award

Based on the information immediately available, is the customer not eligible for ABSTUDY and the claim is to be rejected?

  • Yes, go to Step 3
  • No, go to Step 4

3

Streamline rejection codes + Read more ...

The following streamline rejection codes are available:

  • All claims
    • COP - another payment is more appropriate
    • FRC - failed to reply to correspondence
    • FSD - failed to supply documents
    • OGA - other government assistance
  • Apprentice claims only
    • NAT - not an apprentice
  • Tertiary student only claims
    • NAC - enrolled in unapproved course
    • NAI - not an approved institution
    • NAS - not a student

Is a streamline rejection code available?

  • Yes,
    • For FSD or FRC, go to Step 11
    • For another streamline rejection reason, go to Step 10
  • No, go to Step 4

4

Assess and code ABSTUDY claim + Read more ...

Using the claim summary, check:

  • all the information (provisional data) from the online claim or ACC:
    • has loaded correctly into Process Direct
    • is accurate and in line with any evidence that has been provided. Update where necessary
  • for any conflict between the provisional data and confirmed data. Correct where necessary
  • modules supplied by the customer. Code information into the appropriate table/s

See:

  • Process Direct navigation, common screens and functions
  • Using Digital Assistant Roxy in Process Direct

5

Identity Confirmation + Read more ...

On the TS screen, check the customer's Identity Status.

If the Identity Status is:

  • Confirmed, go to Step 6
  • Not Confirmed, code alternative proof of identity:
    • Select the Not Confirmed link to open a Customer Summary window
    • Select Identity Confirmation > Identity Confirmation Dashboard from the Task Selector
    • Code the identity documents or consider applying alternative identity
    • Close the Customer Summary window to return to the claim

6

Accommodation Details + Read more ...

See Determining proof of rent for the Verification field on the Accommodation Details (AC) screen.

Once verification is provided, update the Accommodation Details (AC) screen. See Processing proof of rent for the Verification field on the Accommodation Details (AC) screen.

7

Tax File Number + Read more ...

Before granting ABSTUDY, customers must supply a valid Tax File Number (TFN). A TFN exemption may apply in limited circumstances.

Has the customer provided a valid TFN?

  • Yes, go to Step 8
  • No, see Requesting a Tax File Number (TFN)

8

Education Details + Read more ...

The customer must meet study requirements for ABSTUDY, but consider options, for example:

  • Aggregated study load
  • Concessional study load
  • Temporary incapacity
  • Reasonable excuse or circumstances beyond the control of the student

9

Parental Income + Read more ...

Parental income is requested as part of the claim, for dependent students.

Where a parent advises a substantial decrease in their income for the current tax year, Service Officers are to accept that the decrease could last for 2 years or more. Parents will not be required to provide evidence of their decrease in income.

See Increase or decrease in parental income for Youth Allowance (YA), ABSTUDY and Assistance for Isolated Children (AIC) Scheme.

10

Errors + Read more ...

After selecting Simulate, relevant errors, warnings and messages may display in the data validation box. These messages inform of the items that need to be addressed to prepare to finalise the claim.

  • Error messages display in red. Information must be updated to address these before the work item can be finalised. When fixed, the error message will no longer display
  • Warnings display in yellow do not block processing of the work item. However, all warnings must be checked and necessary coding completed before the work item is finalised. Warnings will continue to display until the claim is finalised
  • Blue messages indicate the error can be resolved by selecting Auto Correct

Select each of these messages to view a brief explanation and to go to the relevant table. The message should also display in the top of the table that will open. Sometimes staff may need to scroll down to view the table.

Select the All icon to access all tables. This may be an efficient way to address multiple errors at once.

Is the customer eligible for ABSTUDY?

  • Yes, go to Step 14
  • No, go to Step 11

11

Not eligible for ABSTUDY + Read more ...

If the claim is not finalised as a streamline rejection, the appropriate rejection reason should automatically occur in most cases after the claim is coded.

There may be some circumstances where a manual rejection is required. See Codes for ABSTUDY cancellation, suspension and rejection for more information.

When making an unfavourable decision, speak to the customer:

  • explain the decision
  • give them a chance to provide more information, and
  • advise their review and appeal rights

If a customer has died before their claim was finalised, end date Electronic Messaging subscription within the new claim. See Initial contact after someone has died.

If the claim is to be rejected for:

  • a streamline rejection reason, go to Step 12
  • another reason, and
    • a manual rejection is required, go to Step 13
    • the auto rejection will occur, go to Step 14

12

Streamline rejection + Read more ...

Full claim coding is not required if the claim is to be rejected using a streamline rejection:

  • Select > Streamline Rejection
  • Select the most appropriate rejection reason from the Rejection Reason Code drop down menu
  • Ensure there is a tick in the Streamline Indicator box
  • Select Finish
  • Once on the Assessments Results screen, ensure the outcome is correct for the customer, and partner if applicable
  • Select Finish

Go to Step 15.

13

Manual rejection + Read more ...

Make all necessary updates required within the claim. Provisional data that has not been verified should be deleted before finalising the claim so the data is not applied to the customer record.

  • Select the Benefit Action icon
  • Update the rejection reason code. Do not select the streamline indicator box (there should not be a tick)
  • Select Assess and Finish to finalise the claim
  • Select OK

Go to Step 15.

14

Finish + Read more ...

After addressing the data validations:

  • Select Simulate to load the data
  • Select Assess
    • Errors, warnings or messages may display and may need to be addressed
    • If there are no errors, the Assessment Results screen will display. Ensure the outcome is correct for both the customer, and partner if applicable
  • Select Finish. A box will confirm that the claim has been successfully completed
  • Select OK

15

Claim outcome + Read more ...

A Notes field presents. Any notes entered here form the contents of the DOC created in the customer's record. It is not mandatory to enter information in this field.

Getting It Right standards for a significant decision must be met in the Claim Finalisation Note. The note automatically generates after the claim is finalised and is viewed via the Notes tab. For more information see:

  • Online Document Recording (ODR) Resources page for Act References
  • Creating documents (Fast Note, Callbacks and ACTDOCs)

Check family assistance

If the customer has a dependent child (aged under 19 years) in their care and they or their partner are not in receipt of Family Tax Benefit (that is, FTB/CUR or FTB/CZR), issue a Q888 letter inviting the customer to check their eligibility for family assistance. See Creating a Q999 or Q888 letter. Refer to the Resources page for standard text to be included in the Q888 letter.
