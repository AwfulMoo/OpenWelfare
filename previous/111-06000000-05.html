To access and use Customer First, staff must have a security role approved in ESSentials as described in this procedure.

Staff also need resources for the required service delivery system:

  • For Medicare mainframe and eBusiness Portal, search mySupport article Medicare Legacy access
  • For Centrelink Income Security Integrated System (ISIS) staff need CDC, OLE and ENQ resources if not included with their ESSentials position. Search mySupport article How to Request CDC, ENQ and OLE Mainframe and Customer First Resources

Security roles

Security roles are requested and approved in ESSentials. For task card support when requesting a security role, go to:

  • ESSentials > Help Home > User Access > Requestors

Note: users need at least 1 role.

Separate procedures cover security roles for access to:

  • Customer First training environment
  • Workload Management and Work Optimiser
  • Process Direct

Customer First security roles to access and UPDATE

Table 1: This table lists the most common security roles used to access and use Customer First.

Security role

Description

ZCCRM_DHS_CSO

For staff requiring access to Customer First.

For access to:

  • Workspace
  • Search
  • My Assignments
  • Reports

ZCCRM_DHS_CSO_ONSITE_MANAGER

For access to:

  • Performance reporting
  • Staff reporting
  • Needs Based Servicing (NBS) reports
  • Add Service Providers for NBS
  • Appointment profile management capability
  • Export to Excel

Customer First security roles to access and VIEW ONLY

Table 2: This table lists the VIEW ONLY security role used to view Customer First.

Security role

Description

ZC.CF.CSO.VIEW.ONLY

For staff requiring access to view agency information in Customer First. Updates to records are not possible with this role.

For view only access to:

  • Workspace
  • Search
  • My Assignments
  • Reports
