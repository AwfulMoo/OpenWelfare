This page contains the escalation process for urgent Quality On Line
(QOL) requests. For Smart Centre and Labour Hire staff only.

On this page:

Service Officers

QOL Checkers

Service Officers

Table 1

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Make sure the request meets the   |
|                                   | urgent QOL criteria + Read more   |
|                                   | ...                               |
|                                   |                                   |
|                                   | Urgent QOL requests are limited   |
|                                   | to these activities:              |
|                                   |                                   |
|                                   | -   Bereavement Payment (LPB)     |
|                                   | -   Urgent payment/Crisis         |
|                                   |     Payment, or an activity       |
|                                   |     preventing an Urgent/Crisis   |
|                                   |     Payment from being assessed   |
|                                   | -   The activity is rate          |
|                                   |     affecting for that dayâ€™s      |
|                                   |     Entitlement Period End Day    |
|                                   |     (EPED)                        |
|                                   | -   The activity needs to be      |
|                                   |     finalised so the customer can |
|                                   |     access updated information    |
|                                   |     related to their              |
|                                   |     service/payment now           |
|                                   | -   The customer needs to access  |
|                                   |     an Income Statement with      |
|                                   |     accurate details before the   |
|                                   |     end of the day. E.g. they     |
|                                   |     need it for a medical         |
|                                   |     appointment or an emergency   |
|                                   |     assistance referral           |
|                                   | -   It is a concession card       |
|                                   |     refresh and the customer      |
|                                   |     needs to use their card on    |
|                                   |     the same day, e.g. doctor or  |
|                                   |     pharmacy                      |
|                                   | -   The activity is stopping      |
|                                   |     another non-urgent activity   |
|                                   |     from being started (STA) on   |
|                                   |     the Activity List (AL)        |
|                                   |     screen. The non-urgent        |
|                                   |     activity will be completed    |
|                                   |     after QOL checking            |
|                                   | -   The activity is preventing a  |
|                                   |     second activity from being    |
|                                   |     completed on the customer's   |
|                                   |     record. The second activity   |
|                                   |     is rate affecting for that    |
|                                   |     day's EPED                    |
|                                   | -   The activity is preventing a  |
|                                   |     new claim being finalised     |
|                                   | -   The activity is a new claim   |
|                                   |     activity and the customer is  |
|                                   |     in severe financial hardship  |
|                                   | -   The activity is preventing    |
|                                   |     the customer from using self  |
|                                   |     service                       |
|                                   |                                   |
|                                   | Complete activities in a way or   |
|                                   | order that reduces the need for   |
|                                   | urgent QOL requests.              |
+-----------------------------------+-----------------------------------+
| 2                                 | Complete the request for an       |
|                                   | urgent QOL + Read more ...        |
|                                   |                                   |
|                                   | -   Open the Urgent QOL           |
|                                   |     Sharepoint                    |
|                                   | -   Select New Item               |
|                                   | -   Fill in all required details  |
|                                   |     on the form and select Submit |
|                                   |     Logon field: the Service      |
|                                   |     Officer who submitted the     |
|                                   |     activity. Note: it defaults   |
|                                   |     to the user's logon. If       |
|                                   |     another person submitted the  |
|                                   |     activity, enter their logon   |
|                                   | -   Refresh [F5] the page to see  |
|                                   |     the status of the request     |
|                                   |                                   |
|                                   | Staff can view their items in the |
|                                   | My Items tab.                     |
|                                   |                                   |
|                                   | The All Items tab shows the       |
|                                   | current workload and wait times.  |

QOL Checkers

Table 2

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | View items submitted for urgent   |
|                                   | QOL request + Read more ...       |
|                                   |                                   |
|                                   | Open the Urgent QOL Sharepoint.   |
|                                   |                                   |
|                                   | Always Refresh [F5] the screen to |
|                                   | view the most up-to-date items.   |
|                                   |                                   |
|                                   | -   Select All Items view         |
|                                   | -   Staff can filter the list     |
|                                   |     view by System to show        |
|                                   |     activities aligned to their   |
|                                   |     skillset                      |
|                                   | -   New items display at the top  |
|                                   |     of the list, then items Under |
|                                   |     review, then items Finalised  |
|                                   |                                   |
|                                   | Note:                             |
|                                   |                                   |
|                                   | -   the Logon list shows the      |
|                                   |     logon of the Service Officer  |
|                                   |     who submitted the activity    |
|                                   | -   the Created by list shows the |
|                                   |     name of the person who        |
|                                   |     created the urgent QOL        |
|                                   |     request                       |
+-----------------------------------+-----------------------------------+
| 2                                 | Select new item + Read more ...   |
|                                   |                                   |
|                                   | -   Select the Edit icon to       |
|                                   |     select a New item to QOL      |
|                                   | -   Select Under Review and then  |
|                                   |     Close. This will change the   |
|                                   |     request status and prevent    |
|                                   |     another checker from          |
|                                   |     selecting the item            |
|                                   | -   Check the activity aligns     |
|                                   |     with the QOL Checker's        |
|                                   |     skillset                      |
+-----------------------------------+-----------------------------------+
| 3                                 | Can the QOL Checker complete the  |
|                                   | QOL check on the item under       |
|                                   | review? + Read more ...           |
|                                   |                                   |
|                                   | Yes, go to Step 4                 |
|                                   |                                   |
|                                   | No,                               |
|                                   |                                   |
|                                   | -   If the QOL checker selects an |
|                                   |     item they do not have the     |
|                                   |     skillset to complete:         |
|                                   |     - Select the Edit icon for    |
|                                   |     the item                      |
|                                   |     - Select Re-allocate to Pool  |
|                                   |     and Close                     |
|                                   |     - This will revert the status |
|                                   |     of the activity back to New   |
|                                   |     - Procedure ends here         |
|                                   | -   Some activities cannot be     |
|                                   |     actioned through this         |
|                                   |     process, go to Step 5         |
+-----------------------------------+-----------------------------------+
| 4                                 | Finalise item after completing    |
|                                   | QOL check + Read more ...         |
|                                   |                                   |
|                                   | -   Select the Edit icon for the  |
|                                   |     item                          |
|                                   | -   Select Finalise then Close    |
|                                   |                                   |
|                                   | TPS/QOL Checkers are not required |
|                                   | to record TSC entries for urgent  |
|                                   | QOL requests completed via the    |
|                                   | Urgent QOL SharePoint             |
|                                   |                                   |
|                                   | Procedure ends here.              |
+-----------------------------------+-----------------------------------+
| 5                                 | Activity excluded from the urgent |
|                                   | QOL process + Read more ...       |
|                                   |                                   |
|                                   | The following activities cannot   |
|                                   | be actioned through this process: |
|                                   |                                   |
|                                   | -   Compensation Activity         |
|                                   | -   Compliance Risk Branch        |
|                                   |     Activity (e.g. EII)           |
|                                   | -   QMA Required                  |
|                                   | -   QOL already completed or not  |
|                                   |     available for QOL checking    |
|                                   | -   Service Delivery Partner      |
|                                   |     Activity                      |
|                                   |                                   |
|                                   | To finalise the QOL check in      |
|                                   | Sharepoint:                       |
|                                   |                                   |
|                                   | -   Select the Edit icon for the  |
|                                   |     item                          |
|                                   | -   Select the relevant reason    |
|                                   |     the activity is excluded      |
|                                   | -   Select Finalise               |
|                                   |                                   |
|                                   | This will generate an email to    |
|                                   | the Service Officer to direct     |
|                                   | them to the correct process.      |
|                                   |                                   |
|                                   | See the Resources page for more   |
|                                   | information.                      |
|                                   |                                   |
|                                   | Procedure ends here.              |
