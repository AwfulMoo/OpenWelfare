Record access warning message

Table 1

+-----------------------------------+-----------------------------------+
| Item                              | Warning message                   |
+-----------------------------------+-----------------------------------+
| 1                                 | 'User: <USERID> is currently in   |
|                                   | this CRN' + Read more ...         |
|                                   |                                   |
|                                   | This warning message will display |
|                                   | if another Service Officer is     |
|                                   | already in the customer’s record. |
|                                   | When this warning appears,        |
|                                   | Service Officers can continue to  |
|                                   | view and update the customer’s    |
|                                   | record with a new transaction.    |
+-----------------------------------+-----------------------------------+

Transaction errors and warning messages

Table 2: This table outlines the different warning and error messages
that may appear when a Service Officer is in a customer’s record or is
processing a transaction. See Resources for scenarios.

+-----------------------------------+-----------------------------------+
| Item                              | Error or warning message          |
+-----------------------------------+-----------------------------------+
| 1                                 | 'User: <USERID> is currently in   |
|                                   | this transaction' + Read more ... |
|                                   |                                   |
|                                   | This error message will display   |
|                                   | when a Service Officer attempts   |
|                                   | to action the same transaction    |
|                                   | another Service Officer is        |
|                                   | already updating.                 |
|                                   |                                   |
|                                   | This transaction stays locked     |
|                                   | until the Service Officer with    |
|                                   | the lock on the transaction has   |
|                                   | exited the transaction or the     |
|                                   | transaction is unlocked via the   |
|                                   | Transaction Unlocking screen.     |
+-----------------------------------+-----------------------------------+
| 2                                 | 'User: <USERID> is currently in   |
|                                   | transaction ID<XXXXXXXXXX>' +     |
|                                   | Read more ...                     |
|                                   |                                   |
|                                   | This warning message displays     |
|                                   | when a Service Officer accesses   |
|                                   | the customer record, and another  |
|                                   | Service Officer is actioning a    |
|                                   | transaction on that record.       |
+-----------------------------------+-----------------------------------+
| 3                                 | 'You no longer have a lock on     |
|                                   | this transaction, lock is secured |
|                                   | by <USERID>' + Read more ...      |
|                                   |                                   |
|                                   | This error message will display   |
|                                   | when a Service Officer loses the  |
|                                   | lock on the transaction and       |
|                                   | another Service Officer attempts  |
|                                   | to use an action such as Assess,  |
|                                   | Add, Edit or Delete to update the |
|                                   | transaction.                      |
|                                   |                                   |
|                                   | The transaction lock has been     |
|                                   | removed by another Service        |
|                                   | Officer and a new lock on the     |
|                                   | transaction has been established. |
+-----------------------------------+-----------------------------------+

Unlocking a transaction

Table 3

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Determine that the transaction    |
|                                   | needs to be actioned urgently +   |
|                                   | Read more ...                     |
|                                   |                                   |
|                                   | Urgent circumstances include:     |
|                                   |                                   |
|                                   | -   The system incorrectly        |
|                                   |     displays that a Service       |
|                                   |     Officer is in the transaction |
|                                   | -   Customers disclosing that     |
|                                   |     they are in imminent risk, or |
|                                   |     critically ill                |
|                                   |                                   |
|                                   | Is urgent action required?        |
|                                   |                                   |
|                                   | -   Yes, go to Step 2             |
|                                   | -   No, transactions can only be  |
|                                   |     unlocked when it is urgent.   |
|                                   |     Procedure ends here           |
+-----------------------------------+-----------------------------------+
| 2                                 | Attempt to contact + Read more    |
|                                   | ...                               |
|                                   |                                   |
|                                   | Reasonable attempts must be made  |
|                                   | to contact the Service Officer or |
|                                   | their Team Leader for the         |
|                                   | transaction to be exited before   |
|                                   | the transaction is unlocked. For  |
|                                   | example, Teams or phone.          |
|                                   |                                   |
|                                   | Has contact been made and the     |
|                                   | Service Officer exited the        |
|                                   | transaction?                      |
|                                   |                                   |
|                                   | -   Yes, Procedure ends here      |
|                                   | -   No, go to Step 3              |
+-----------------------------------+-----------------------------------+
| 3                                 | Contact unsuccessful + Read more  |
|                                   | ...                               |
|                                   |                                   |
|                                   | If contact has been unsuccessful, |
|                                   | remove the lock:                  |
|                                   |                                   |
|                                   | Key 'Unlock' in the Super Key     |
|                                   | field                             |
|                                   |                                   |
|                                   | -   the Transaction Locking       |
|                                   |     Summary screen will display   |
|                                   |                                   |
|                                   | Answer Yes or No to the following |
|                                   | questions:                        |
|                                   |                                   |
|                                   | -   The transaction meets the     |
|                                   |     urgent standards as outlined  |
|                                   |     in OB <111-21062917>          |
|                                   | -   Contact attempts have been    |
|                                   |     made to the Service Officer   |
|                                   |     or their Team leader if       |
|                                   |     unavailable to request the    |
|                                   |     Service Officer to exit the   |
|                                   |     transaction                   |
|                                   | -   A reasonable time has passed  |
|                                   |     for the Service Officer or    |
|                                   |     their Team Leader to respond  |
|                                   |                                   |
|                                   | Delete will only be enabled once  |
|                                   | the Service Officer has confirmed |
|                                   | the transaction meets urgent      |
|                                   | standards and reasonable attempts |
|                                   | to contact the Service Officer    |
|                                   | and Team Leader have been         |
|                                   | unsuccessful.                     |
|                                   |                                   |
|                                   | -   Select Delete                 |
|                                   | -   The Remove Transaction Lock   |
|                                   |     message will display          |
|                                   | -   Select OK                     |
|                                   | -   To establish a lock on the    |
|                                   |     transaction, select the       |
|                                   |     transactions again from the   |
|                                   |     Transaction tab               |
|                                   |                                   |
|                                   | The Service Officer who           |
|                                   | originally had the lock on the    |
|                                   | transaction will be presented     |
|                                   | with the Warning Message 'You no  |
|                                   | longer have a lock on this        |
|                                   | transaction, lock is secured by   |
|                                   | <USERID>'. The Service Officer    |
|                                   | will no longer be able to make    |
|                                   | any further updates to that       |
|                                   | transaction.                      |
