This page contains information on access to the Provider Digital Access
(PRODA) staff portal and the escalation process for enquiries,
complaints, feedback and incidents.

PRODA access levels, enquiries and escalations

Separate processes apply for managing enquiries from:

-   Healthcare professionals and administrators
-   National Disability Insurance Scheme (NDIS) providers

Select the relevant user group from the drop down box below:

Complaints and feedback - Healthcare users

+-----------------------------------+-----------------------------------+
| Item                              | Definition and escalation         |
+-----------------------------------+-----------------------------------+
| 1                                 | Definitions complaints and        |
|                                   | feedback + Read more ...          |
|                                   |                                   |
|                                   | A complaint might be made by a    |
|                                   | user if they are unhappy about    |
|                                   | something, for example, they      |
|                                   | might express dissatisfaction     |
|                                   | about a product or service        |
|                                   | offered by the PRODA system.      |
|                                   |                                   |
|                                   | Feedback is a suggestion a user   |
|                                   | makes to improve the PRODA        |
|                                   | system. This includes the         |
|                                   | addition of features or more      |
|                                   | information and suggestions to    |
|                                   | improve business processes.       |
|                                   |                                   |
|                                   | Complaints and feedback may       |
|                                   | include:                          |
|                                   |                                   |
|                                   | -   an outage of the PRODA        |
|                                   |     system, such as telephony     |
|                                   |     system, fax outage or actual  |
|                                   |     PRODA system not functioning, |
|                                   |     or                            |
|                                   | -   complex user enquiries        |
+-----------------------------------+-----------------------------------+
| 2                                 | Escalations of complaints and     |
|                                   | feedback + Read more ...          |
|                                   |                                   |
|                                   | Must be referred in the first     |
|                                   | instance:                         |
|                                   |                                   |
|                                   | -   MPB T1 SO to the MPB PSO4     |
|                                   | -   MPB T1 PSO4 to T2 eBus        |
|                                   | -   T2 eBus SO to the T2 eBus     |
|                                   |     PSO4                          |
|                                   |                                   |
|                                   | If necessary, the PSO4 will       |
|                                   | escalate the issue to the Team    |
|                                   | Leader 5 (TL5).                   |
|                                   |                                   |
|                                   | Note: if a PSO4, TL5, TL6 or EL1  |
|                                   | is unavailable and the enquiry is |
|                                   | urgent, escalate directly to Tier |
|                                   | 3 PRODA support.                  |

PRODA incidents and escalation - Healthcare users

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Types of PRODA incidents + Read   |
|                                   | more ...                          |
|                                   |                                   |
|                                   | Serious user incidents            |
|                                   |                                   |
|                                   | A user enquiry must be escalated  |
|                                   | directly to Tier 3 PRODA Support  |
|                                   | if they are reporting privacy and |
|                                   | security related incidents. For   |
|                                   | example:                          |
|                                   |                                   |
|                                   | -   a security incident or        |
|                                   |     privacy incident, e.g. the    |
|                                   |     user believes unauthorised    |
|                                   |     access has occurred           |
|                                   | -   an urgent request or issue    |
|                                   | -   complaints                    |
|                                   | -   fraud                         |
|                                   |                                   |
|                                   | System user incidents             |
|                                   |                                   |
|                                   | System user incidents, while      |
|                                   | generally less critical than the  |
|                                   | incidents listed under serious    |
|                                   | incidents, may be issues relating |
|                                   | to the PRODA system reputation or |
|                                   | the Australian Government.        |
|                                   |                                   |
|                                   | If necessary, the Subject Matter  |
|                                   | Expert (SME) will escalate the    |
|                                   | issue to Tier 3 PRODA Support.    |
+-----------------------------------+-----------------------------------+
| 2                                 | Perform security check + Read     |
|                                   | more ...                          |
|                                   |                                   |
|                                   | Perform security check.           |
|                                   |                                   |
|                                   | Access account information for    |
|                                   | escalation referral.              |
+-----------------------------------+-----------------------------------+
| 3                                 | Escalate incident - Tier 1        |
|                                   | Service Officer + Read more ...   |
|                                   |                                   |
|                                   | If the user is reporting a        |
|                                   | privacy and security related      |
|                                   | incident, escalate directly to    |
|                                   | Tier 3, see Help for staff        |
|                                   | Provider Digital Access (PRODA).  |
|                                   | Procedure ends here.              |
|                                   |                                   |
|                                   | For other incidents               |
|                                   |                                   |
|                                   | Escalate the issue to Subject     |
|                                   | Matter Expert (SME). This may be  |
|                                   | a Program Support Officer (PS04)  |
|                                   | or Team Leader 5.                 |
|                                   |                                   |
|                                   | If SME is unavailable and the     |
|                                   | enquiry is urgent, escalate       |
|                                   | directly to Tier 3 PRODA Support. |
|                                   |                                   |
|                                   | If unsure how to manage or triage |
|                                   | an incident,                      |
|                                   |                                   |
|                                   | Communicate immediately with      |
|                                   | PS04. If they are unavailable,    |
|                                   | contact Team Leader 5/6 or Tier 1 |
|                                   | EL1 immediately for advice.       |
|                                   |                                   |
|                                   | The Resources page contains       |
|                                   | examples of escalating privacy,   |
|                                   | security and unauthorised access  |
|                                   | incidents, fraud incidents and    |
|                                   | system incidents                  |
+-----------------------------------+-----------------------------------+
| 4                                 | Escalate incident - PSO4/Team     |
|                                   | Leader 5 + Read more ...          |
|                                   |                                   |
|                                   | Escalate to Tier 3 immediately to |
|                                   | advise of the 'serious' or        |
|                                   | 'system' user incident:           |
|                                   |                                   |
|                                   | Include in email:                 |
|                                   |                                   |
|                                   | -   brief description of issue    |
|                                   |     including contact details for |
|                                   |     the user, dates and times of  |
|                                   |     the incident (and if possible |
|                                   |     provide screen shots)         |
|                                   | -   brief description of all your |
|                                   |     correspondence and response   |
|                                   |     to the user                   |
|                                   |                                   |
|                                   | See Help for staff Provider       |
|                                   | Digital Access (PRODA).           |
