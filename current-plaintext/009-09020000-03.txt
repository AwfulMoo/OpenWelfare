| 1                                 | Fact of Death Data (FoDD) and     |
|                                   | Death Reports + Read more ...     |
|                                   |                                   |
|                                   | Notification of a deceased CAPS   |
|                                   | customer can be received via a    |
|                                   | data match from the Department of |
|                                   | Births, Deaths and Marriages      |
|                                   | (FoDD report) or notification by  |
|                                   | a member of the public (Death     |
|                                   | report).                          |
|                                   |                                   |
|                                   | Programme Management receive the  |
|                                   | following reports every Friday:   |
|                                   |                                   |
|                                   | -   CAPS death report dd month    |
|                                   |     yyyy                          |
|                                   | -   CAPS records with a FoDD      |
|                                   |     update dd month yyyy          |
|                                   | -   CAPS death report dd month    |
|                                   |     yyyy processed last week      |
|                                   |                                   |
|                                   | CAPS entitlements for all         |
|                                   | deceased customers are required   |
|                                   | to be ceased in the CAPS system.  |
|                                   | The entitlements need to be       |
|                                   | ceased by COB the following       |
|                                   | Wednesday so the customers do not |
|                                   | appear on the next week's report. |
|                                   |                                   |
|                                   | The report is saved on the        |
|                                   | CAPS-administration drive by the  |
|                                   | Programme Management team and     |
|                                   | emailed to the CAPS processing    |
|                                   | team for action.                  |
|                                   |                                   |
|                                   | How the data is entered into CDMS |
|                                   | (i.e. manually or automatically)  |
|                                   | determines how the customer       |
|                                   | record is updated:                |
|                                   |                                   |
|                                   | -   an automatically processed    |
|                                   |     date of death will result in  |
|                                   |     a date of death for the       |
|                                   |     customer and an entitlement   |
|                                   |     end date and reason recorded  |
|                                   |     in CDMS                       |
|                                   | -   manually entered data will    |
|                                   |     result in only a date of      |
|                                   |     death being recorded for the  |
|                                   |     customer                      |
|                                   | -   the CAPS Registration screen  |
|                                   |     ‘Deceased date’ field will be |
|                                   |     populated with the date of    |
|                                   |     death recorded in CDMS for    |
|                                   |     both automatically and        |
|                                   |     manually entered data         |
|                                   |                                   |
|                                   | Go to Step 4.                     |

| 2                                 | High Care report + Read more ...  |
|                                   |                                   |
|                                   | A customer is not eligible to     |
|                                   | receive CAPS if they are a care   |
|                                   | recipient and the level of care   |
|                                   | required is:                      |
|                                   |                                   |
|                                   | -   a high ADL domain category,   |
|                                   |     or                            |
|                                   | -   a high CHC domain category,   |
|                                   |     or                            |
|                                   | -   a high behaviour domain       |
|                                   |     category, or                  |
|                                   | -   a medium domain category in   |
|                                   |     at least two domains, or      |
|                                   |                                   |
|                                   | the level of care is assessed     |
|                                   | using the Aged Care Funding       |
|                                   | Instrument (ACFI) and is divided  |
|                                   | into three domains:               |
|                                   |                                   |
|                                   | -   Activities of Daily Living    |
|                                   |     (ADL)                         |
|                                   | -   Behaviour (BEH), or           |
|                                   | -   Complex health care (CHC)     |
|                                   |                                   |
|                                   | Customers listed in the High Care |
|                                   | report need to have their         |
|                                   | entitlement reviewed and ceased.  |
|                                   | The report is saved on the        |
|                                   | CAPS-administration drive by the  |
|                                   | Programme Management team and     |
|                                   | actioned by the CAPS processing   |
|                                   | team. The report is password      |
|                                   | protected and the password is     |
|                                   | forwarded in a separate email.    |
|                                   |                                   |
|                                   | Review the Customer record to     |
|                                   | determine if payments should      |
|                                   | cease and comments to ensure no   |
|                                   | relevant history applies.         |
|                                   |                                   |
|                                   | Go to Step 4.                     |
