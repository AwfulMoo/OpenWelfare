+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Obtain confirmation of identity + |
|                                   | Read more ...                     |
|                                   |                                   |
|                                   | The customer is required to       |
|                                   | provide confirmation of identity. |
|                                   | Where the customer is:            |
|                                   |                                   |
|                                   | -   Present at a Service Centre,  |
|                                   |     sight acceptable              |
|                                   |     identification. See also:     |
|                                   |     Authenticating a Medicare     |
|                                   |     customer                      |
|                                   | -   Not present at a service      |
|                                   |     centre or is unable to        |
|                                   |     produce acceptable            |
|                                   |     identification. See,          |
|                                   |     Authenticating a Medicare     |
|                                   |     customer                      |
+-----------------------------------+-----------------------------------+
| 2                                 | Identify claimant + Read more ... |
|                                   |                                   |
|                                   | To identify if the customer is    |
|                                   | the claimant, the Service Officer |
|                                   | must ask the customer the         |
|                                   | following questions:              |
|                                   |                                   |
|                                   | -   What date was the service?    |
|                                   | -   Approximately what time was   |
|                                   |     the service?                  |
|                                   | -   Who was the health            |
|                                   |     professional?                 |
|                                   | -   How much was paid for the     |
|                                   |     service?                      |
|                                   |                                   |
|                                   | If the customer successfully      |
|                                   | answers these questions the claim |
|                                   | should be adjusted to reflect the |
|                                   | correct claimant.                 |
+-----------------------------------+-----------------------------------+
| 3                                 | Confirm 'HELD' Medicare benefit   |
|                                   | is outstanding for claim          |
|                                   | requiring an adjustment + Read    |
|                                   | more ...                          |
|                                   |                                   |
|                                   | In Mainframe:                     |
|                                   |                                   |
|                                   | -   key, NHSI,CARD NUMBER,        |
|                                   |     patient’s IRN or first        |
|                                   |     initial                       |
|                                   | -   select [Enter]                |
|                                   |                                   |
|                                   | The patient history is returned   |
|                                   | displaying the 'HELD' payment.    |
|                                   | Under the STAT field it will say  |
|                                   | 'HELD'.                           |
+-----------------------------------+-----------------------------------+
| 4                                 | Reject HELD claim + Read more ... |
|                                   |                                   |
|                                   | In Mainframe:                     |
|                                   |                                   |
|                                   | -   key 'BRNA'                    |
|                                   | -   select [Enter]                |
|                                   |                                   |
|                                   | The EFT payment history enquiry   |
|                                   | (BRNA) screen is displayed.       |
|                                   |                                   |
|                                   | On the BRNA screen:               |
|                                   |                                   |
|                                   | -   key claimant’s Medicare       |
|                                   |     number into payee ID field    |
|                                   | -   select [Enter]                |
|                                   | -   key the relevant 'SEL' field  |
|                                   |     number in the SELECTION field |
|                                   | -   select [F9]                   |
|                                   |                                   |
|                                   | The BRNS screen is displayed, the |
|                                   | system will display 'HELD' in the |
|                                   | STATUS field.                     |
|                                   |                                   |
|                                   | On the BRNS screen:               |
|                                   |                                   |
|                                   | -   tab to Status field           |
|                                   |     displaying 'HELD'             |
|                                   | -   over key the status 'HELD'    |

|                                   | -   tab to TEXT field             |
|                                   | -   add text ‘REJECT HELD PAYMENT |
|                                   |     ADJUSTMENT REQUIRED’          |
|                                   | -   select [F5] to validate and   |
|                                   |     update rejection process      |
|                                   |                                   |
|                                   | This action will change the       |
|                                   | 'HELD' status with a new status   |
|                                   | of 'REJECT'.                      |
