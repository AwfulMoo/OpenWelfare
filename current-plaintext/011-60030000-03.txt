|                                   |                                   |

|                                   | -   RESOLVED - the intertwined    |

|                                   | -   MPI 1 flag warning message -  |

|                                   | Shopping System to continue       |
|                                   | processing                        |
|                                   |                                   |
|                                   | If the Consumer Restrictions      |
|                                   | field displays 'CAUTION' or       |
|                                   | 'SUSPEND' and has not been end    |
|                                   | dated refer to the 'Intertwined   |
|                                   | Record script for PSIS operators' |
|                                   | located on the Resources tab      |
|                                   | Note: if you do not see the full  |
|                                   | restriction information including |
|                                   | the start and end date fields you |
|                                   | will need to open the Restriction |
|                                   | History. This will provide        |
|                                   | confirmation if the status has    |
|                                   | been end dated                    |
|                                   |                                   |
|                                   | If the Consumer Restrictions      |
|                                   | field displays an 'Information    |
|                                   | Message' continue processing      |
|                                   |                                   |
|                                   | If the Consumer Restrictions      |
|                                   | field displays an MPI restriction |
|                                   | flag, this means that the         |
|                                   | customer record is affected by    |
|                                   | unauthorised Medicare activity    |
|                                   | and has been compromised. There   |
|                                   | are 2 flags - MPI 1 and MPI 2     |
|                                   |                                   |
|                                   | The MPI 1 flag prevents any       |
|                                   | updates and/or claiming to that   |
|                                   | record and indicate the customer  |
|                                   | has been transferred to a new     |
|                                   | card. This flag is applied after  |
|                                   | the customer has been transferred |
|                                   | onto a new card and has received  |
|                                   | the new card                      |
|                                   |                                   |
|                                   | If the MPI 1 flag appears, advise |
|                                   | the prescriber that the patient   |
|                                   | has a new Medicare number and ask |
|                                   | the prescriber whether they have  |
|                                   | the new Medicare number           |
|                                   | available. If yes, enter this     |
|                                   | number into CDMS and proceed to   |
|                                   | the Prescription Shopping System. |
|                                   | If not, then perform a manual     |
|                                   | name and date of birth search for |
|                                   | the new Medicare number. You can  |
|                                   | then proceed to the Prescription  |
|                                   | Shopping System. Do not advise    |
|                                   | the prescriber of the flag        |
|                                   | message                           |
|                                   |                                   |
|                                   | The MPI 2 flag is a warning       |
|                                   | message to inform that the record |
|                                   | has been compromised              |
|                                   |                                   |
|                                   | If the MPI 2 flag appears, ask    |
|                                   | the prescriber to confirm the     |
|                                   | patient information. Once         |
|                                   | confirmed, the warning message    |
|                                   | can be acknowledged proceed to    |
|                                   | the Prescription Shopping System. |
|                                   | Do not advise the prescriber of   |
|                                   | the flag message                  |
|                                   |                                   |
