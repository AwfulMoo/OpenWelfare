mySupport

To access and use Customer First, staff must have a security role
approved in ESSentials as described in this procedure.

Staff also need resources for the required service delivery system:

-   For Medicare mainframe and eBusiness Portal, search mySupport
    article Medicare Legacy access
-   For Centrelink Income Security Integrated System (ISIS) staff need
    CDC, OLE and ENQ resources if not included with their ESSentials
    position. Search mySupport article How to Request CDC, ENQ and OLE
    Mainframe and Customer First Resources

Search using the key words, error message, article number or application
name and select the Article.

Security roles

Security roles are requested and approved in ESSentials. For task card
support when requesting a security role, go to:

-   ESSentials > Help Home > User Access > Requestors

Note: users need at least 1 role.

Separate procedures cover security roles for access to:

-   Customer First training environment
-   Workload Management and Work Optimiser
-   Process Direct
