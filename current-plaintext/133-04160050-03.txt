How to enrol a customer who presents at a service centre

Table 2

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Confirm customer’s identity +     |
|                                   | Read more ...                     |
|                                   |                                   |
|                                   | Ask the customer for their        |
|                                   | Customer Reference Number (CRN)   |
|                                   | or their Customer Access Number   |
|                                   | (CAN) to confirm the customer’s   |
|                                   | identity using photo ID.          |
|                                   | Crosscheck the following items    |
|                                   | against previously recorded       |
|                                   | information on the (Identity      |
|                                   | Confirmation Summary screen in    |
|                                   | Customer First:                   |
|                                   |                                   |
|                                   | -   the photo ID serial number    |
|                                   | -   address                       |
|                                   | -   date of birth                 |
|                                   |                                   |
|                                   | Service Officers must be          |
|                                   | satisfied they are creating a PIN |
|                                   | with the right person.            |
+-----------------------------------+-----------------------------------+
| 2                                 | Read privacy statement + Read     |
|                                   | more ...                          |
|                                   |                                   |
|                                   | Read out the Privacy Statement    |
|                                   | and Self Service Conditions of    |
|                                   | Use. Ask for the customer's       |
|                                   | consent to continue.              |
|                                   |                                   |
|                                   | The Privacy Statement and the     |
|                                   | Conditions of Use are available   |
|                                   | on the Resources page.            |
+-----------------------------------+-----------------------------------+
| 3                                 | Enter details in Customer First   |
|                                   | to create a PIN + Read more ...   |
|                                   |                                   |
|                                   | -   Go to Workspace>Self          |
|                                   |     Service>Manage Self Service   |
|                                   |     Access (AMSS)                 |
|                                   | -   Select phone self service     |
|                                   | -   Select the FastTrack process  |
|                                   |                                   |
|                                   | Write down the PIN and give it to |
|                                   | the customer.                     |
|                                   |                                   |
|                                   | Ask the customer to read out the  |
|                                   | number.                           |
|                                   |                                   |
|                                   | Confirm this against the number   |
|                                   | on the screen.                    |
|                                   |                                   |
|                                   | Advise the customer all words in  |
|                                   | the common passphrase must be     |
|                                   | repeated as per the IVR prompt.   |
|                                   | Some customers may want to        |
|                                   | practice the passphrase “In       |
|                                   | Australia my voice identifies me" |
|                                   | prior to creating their           |
|                                   | voiceprint.                       |
+-----------------------------------+-----------------------------------+
| 4                                 | Customer calls self service       |
|                                   | line + Read more ...              |
|                                   |                                   |
|                                   | Ask the customer to call the Self |
|                                   | Service line on 136 240 to        |
|                                   | finalise the enrolment using      |
|                                   | either                            |
|                                   |                                   |
|                                   | -   the Self Service phones in    |
|                                   |     the service centre            |
|                                   | -   their own mobile phone        |
|                                   |                                   |
|                                   | The customer should be encouraged |
|                                   | to call while still in the        |
|                                   | service centre so they have staff |
|                                   | available in case of any          |
|                                   | problems.                         |
|                                   |                                   |
|                                   | Note: some customers may have a   |
|                                   | different CAN to their CRN. The   |
|                                   | Service Officer must advise the   |
|                                   | customer to use their CAN when    |
|                                   | calling the self service line and |
|                                   | prompted for their CRN.           |
|                                   |                                   |
|                                   | Procedure ends here.              |

How to enrol a customer who calls a smart centre

Table 3

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Before taking the call + Read     |
|                                   | more ...                          |
|                                   |                                   |
|                                   | Before the Service Officer has    |
|                                   | answered the call, the customer   |
|                                   | has:                              |
|                                   |                                   |
|                                   | -   provided their CAN or CRN     |
|                                   |     (Customers must use their CAN |
|                                   |     if it is different to their   |
|                                   |     CRN)                          |
|                                   | -   heard the Privacy Statement   |
|                                   |     and Conditions of Use through |
|                                   |     the IVR - Customers Consent   |
|                                   | -   recorded their Voiceprint by  |
|                                   |     repeating the Common Pass     |
|                                   |     Phrase (CPP) of ‘In Australia |
|                                   |     my voice identifies me’ 3     |
|                                   |     times                         |
+-----------------------------------+-----------------------------------+
| 2                                 | When the call connects with a     |
|                                   | voiceprint recorded + Read more   |
|                                   | ...                               |
|                                   |                                   |
|                                   | If the Case Information window    |
|                                   | displays:                         |
|                                   |                                   |
|                                   | -   green, the voiceprint has     |
|                                   |     already been recorded and     |
|                                   |     authenticated. Procedure ends |
|                                   |     here                          |
|                                   | -   amber, the customer has       |
|                                   |     recorded a voiceprint but it  |
|                                   |     needs to be authenticated     |
|                                   | -   red, the customer did not     |
|                                   |     record a voiceprint or there  |
|                                   |     is a problem with             |
|                                   |     authentication                |
|                                   |                                   |
|                                   | For example, the following amber  |
|                                   | icon and the green 'Confirm       |
|                                   | Voiceprint' button display        |
|                                   | because the caller has recorded a |
|                                   | provisional voiceprint in the     |
|                                   | Smart IVR. The customer's Proof   |
|                                   | of Record Ownership (PoRO) needs  |
|                                   | to be finalised for the           |
|                                   | customer's voiceprint enrolment   |
|                                   | to be successful.                 |
|                                   |                                   |
|                                   | []                                |
|                                   |                                   |
|                                   | A green 'Confirm Voiceprint'      |
|                                   | button will appear when the       |
|                                   | customer's voiceprint has been    |
|                                   | successful:                       |
|                                   |                                   |
|                                   | See Authentication status for     |
|                                   | more icon examples.               |
|                                   |                                   |
|                                   | Get the customer’s consent to     |
|                                   | finalise the enrolment. Suggested |
|                                   | wording:                          |
|                                   |                                   |
|                                   | 'You have successfully recorded   |
|                                   | your voiceprint with us. I can    |
|                                   | finalise your voiceprint          |
|                                   | registration for you now.'        |
|                                   |                                   |
|                                   | Does the customer want to         |
|                                   | register their voiceprint?        |
|                                   | (Ensure record ownership and that |
|                                   | this is not a partner, parent of  |
|                                   | a child record, nominee or caller |
|                                   | on a deceased record)             |
|                                   |                                   |
|                                   | Yes, go to Step 3                 |
|                                   |                                   |
|                                   | No,                               |
|                                   |                                   |
|                                   | -   Authenticate the customer     |
|                                   | -   Record Fast Note - Voiceprint |
|                                   |     Recorded: Declined Enrol      |
|                                   |     (customer chose not to enrol) |
|                                   | -   The provisional voiceprint    |
|                                   |     will delete from the system   |
|                                   |     within 24 hours. It cannot be |
|                                   |     accessed once the call ends   |
|                                   | -   Continue with the customer    |
|                                   |     enquiry                       |
|                                   |                                   |
|                                   | Note: If the customer says they   |
|                                   | do not want to enrol their        |
|                                   | voiceprint or do not want to be   |
|                                   | offered the voiceprint option in  |
|                                   | future, see Table 5               |
+-----------------------------------+-----------------------------------+
| 3                                 | Confirming the customer’s         |
|                                   | identity + Read more ...          |
|                                   |                                   |
|                                   | Click Confirm Voiceprint button   |
|                                   | to open the Manage Self Service   |
|                                   | Access (AMSS) workflow            |
|                                   |                                   |
|                                   | Before starting the voiceprint    |
|                                   | registration workflow, confirm    |
|                                   | the customer's identity to make   |
|                                   | sure the correct customer record  |
|                                   | is used. Check the customer's:    |
|                                   |                                   |
|                                   | -   name                          |
|                                   | -   address                       |
|                                   | -   date of birth                 |
|                                   | -   CRN                           |
|                                   |                                   |
|                                   | Select the tick box after         |
|                                   | confirming their identity.        |
|                                   |                                   |
|                                   | Go to Step 4                      |
|                                   |                                   |
|                                   | Note: Service Officers no longer  |
|                                   | need to read the Privacy          |
|                                   | Statement or Conditions of Use,   |
|                                   | as the customer has already heard |
|                                   | this in the IVR.                  |
+-----------------------------------+-----------------------------------+
| 4                                 | Proof of Record Ownership         |
|                                   | (PoRO) + Read more ...            |
|                                   |                                   |
|                                   | Additional questions will display |
|                                   | from the customer’s Centrelink    |
|                                   | record.                           |
|                                   |                                   |
|                                   | Select the Yes radio button in    |
|                                   | the Confirmed column for each     |
|                                   | correct answer.                   |
|                                   |                                   |
|                                   | Has customer completed the        |
|                                   | required PoRO questions?          |
|                                   |                                   |
|                                   | Note: do not use receipt number   |
|                                   | from the current call if using    |
|                                   | "Receipt number issued within the |

|                                   |                                   |
|                                   | Yes,                              |
|                                   |                                   |
|                                   | -   Select the Continue button    |

|                                   | -   Go to Step 5                  |
|                                   |                                   |

|                                   |                                   |

|                                   |     confirmation items.           |
|                                   | -   If the customer is unable to  |
|                                   |     answer enough questions to    |
|                                   |     satisfy PoRO, go to Step 6    |
+-----------------------------------+-----------------------------------+
| 5                                 | Enrolment successful + Read more  |
|                                   | ...                               |
|                                   |                                   |
|                                   | Advise the customer their account |

Table 5

+-----------------------------------+-----------------------------------+
| Item                              | Action                            |
+-----------------------------------+-----------------------------------+

|                                   | Read more ...                     |
|                                   |                                   |
|                                   | Customers are encouraged to       |
|                                   | record their voiceprint Common    |
|                                   | Pass Phrase (CPP) in the          |
|                                   | Interactive Voice Response (IVR). |
|                                   | Customers may record their CPP    |
|                                   | but not wish to enrol. The IVR    |
|                                   | tells customers enrolment is      |
|                                   | optional.                         |
|                                   |                                   |
|                                   | Some customers may present to a   |
|                                   | Service Officer and decide not to |
|                                   | enrol. While the agency prefers   |
|                                   | voiceprint as an authentication   |
|                                   | method for phone contact, Service |
|                                   | Officers cannot demand a customer |
|                                   | use it.                           |
|                                   |                                   |
|                                   | Do not finalise the voiceprint    |
|                                   | enrolment in Manage Self Service  |
|                                   | Access (AMSS).                    |
|                                   |                                   |
|                                   | Advise the customer:              |
|                                   |                                   |
|                                   | -   The passphrase audio has not  |
|                                   |     been stored. This is because  |
|                                   |     they did not complete their   |
|                                   |     enrolment                     |
|                                   | -   Enrolling their voiceprint is |
|                                   |     fast and easy, and provides   |
|                                   |     extra security for their      |
|                                   |     record                        |
|                                   | -   Using a voiceprint means they |
|                                   |     can use phone self service    |
|                                   |     applications to save time     |
|                                   | -   They will get the opportunity |
|                                   |     to enrol in the IVR each time |
|                                   |     they call                     |
|                                   |                                   |
|                                   | If a customer does not want to    |
|                                   | enrol for a voiceprint:           |
|                                   |                                   |
|                                   | -   create a Fast Note and record |
|                                   |     why they do not want to       |
|                                   |     enrol, and                    |
|                                   | -   authenticate the customer     |
+-----------------------------------+-----------------------------------+
| 2                                 | Customer does not want to be      |
|                                   | asked to enrol their voiceprint   |
|                                   | each time they call + Read more   |
|                                   | ...                               |
|                                   |                                   |
|                                   | Advise the customer:              |
|                                   |                                   |
|                                   | -   The agency has a duty of care |
|                                   |     and must offer the highest    |
|                                   |     level of security to          |
|                                   |     customers                     |
|                                   | -   Voiceprints are the preferred |

Managing voiceprints - failed voiceprints or suspensions

Table 6

+-----------------------------------+-----------------------------------+
| Item                              | Description                       |
+-----------------------------------+-----------------------------------+
| 1                                 | Problems recording Voiceprint +   |
|                                   | Read more ...                     |
|                                   |                                   |
|                                   | The IVR will transfer the caller  |
|                                   | to a Service Officer if they are  |
|                                   | not able to record a CPP. The IVR |
|                                   | will prompt them to enrol the     |
|                                   | next time they call with only a   |
|                                   | CAN or their CRN (Customers must  |
|                                   | use their CAN if it is different  |
|                                   | to their CRN).                    |
|                                   |                                   |
|                                   | Reasons why this may happen       |
|                                   | include:                          |
|                                   |                                   |
|                                   | -   A customer may have a         |
|                                   |     different Customer Access     |
|                                   |     Number (CAN) to their         |
|                                   |     Customer Reference Number     |
|                                   |     (CRN). The customer must use  |
|                                   |     their CAN when prompted in    |
|                                   |     the Smart IVR to say their    |
|                                   |     CRN                           |
|                                   | -   No input - the customer does  |
|                                   |     not speak as prompted, or     |
|                                   |     they begin to speak too early |
|                                   |     or too late                   |
|                                   | -   Mismatched audio - the system |
|                                   |     does not identify the         |
|                                   |     customer as who they say they |
|                                   |     are, for example the system   |
|                                   |     detects a recording of a      |
|                                   |     customer's voice is being     |
|                                   |     played, multiple speakers are |
|                                   |     speaking (excluding the       |
|                                   |     Service Officer), or the      |
|                                   |     customer is not saying the    |
|                                   |     correct Common Passphrase     |
|                                   | -   Inconclusive audio - the      |
|                                   |     technology cannot check the   |
|                                   |     vocal sample spoken by the    |
|                                   |     customer. This includes when  |
|                                   |     the customers speech is too   |
|                                   |     soft, too loud, too short or  |
|                                   |     too long. Inconclusive audio  |
|                                   |     may also be caused by         |
|                                   |     excessive background noise.   |
|                                   |     It is recommended enrolment   |
|                                   |     be completed in a quiet       |
|                                   |     location                      |
|                                   | -   The previous Service Officer  |
|                                   |     may have provided the         |
|                                   |     customer with an incorrect    |
|                                   |     PIN number (Accidently wrote  |
|                                   |     down a wrong digit). Or the   |
|                                   |     customer may have written     |
|                                   |     down a wrong digit            |
|                                   | -   The customer must repeat all  |
|                                   |     words in the common           |
|                                   |     passphrase as per the IVR     |
|                                   |     prompt                        |
+-----------------------------------+-----------------------------------+
| 2                                 | Caller asks to stop using their   |
|                                   | voiceprint + Read more ...        |
|                                   |                                   |
|                                   | Proceed with caution. Complete    |
|                                   | the proof of record ownership     |
|                                   | (PoRO) to confirm the customer's  |
|                                   | identity.                         |
|                                   |                                   |
|                                   | Advise the customer:              |
|                                   |                                   |
|                                   | A voiceprint is the most secure   |
|                                   | way customers can deal with       |
|                                   | Centrelink over the phone. If     |
|                                   | they do not have a voiceprint     |
|                                   | they will be offered this service |
|                                   | each time they call               |
|                                   |                                   |
|                                   | Deleting will remove all          |
|                                   | voiceprints Telstra has stored    |
|                                   | for the customer (on behalf of    |
|                                   | Centrelink)                       |
|                                   |                                   |
|                                   | If they have a PIN they can       |
|                                   | continue using it                 |
|                                   |                                   |
|                                   | If they want a PIN:               |
|                                   |                                   |
|                                   | -   they must complete the proof  |
|                                   |     of record ownership (PoRO) to |
|                                   |     confirm their identity        |
|                                   | -   when they enter their PIN     |
|                                   |     they will be asked to enrol   |
|                                   |     their voiceprint              |
+-----------------------------------+-----------------------------------+
| 3                                 | Customer frequently fails         |
|                                   | voiceprint + Read more ...        |
|                                   |                                   |
|                                   | Reasons why this may happen       |
|                                   | include:                          |
|                                   |                                   |
|                                   | -   poor sound quality, for       |
|                                   |     example, due to background    |
|                                   |     noise, poor phone line        |
|                                   | -   the customer must repeat all  |
|                                   |     words in the common           |
|                                   |     passphrase as per the IVR     |
|                                   |     prompt                        |
|                                   | -   Also, go to Step 1, Problems  |
|                                   |     recording a Voiceprint        |
