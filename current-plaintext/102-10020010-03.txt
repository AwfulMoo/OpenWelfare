This page contains information about coding a cancellation of a payment
and the reasons for cancellations.

Coding a cancellation

This table describes the steps to take when coding a cancellation.

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | This procedures applies to the    |
|                                   | following payments + Read more    |
|                                   | ...                               |
|                                   |                                   |
|                                   | -   JobSeeker Payment (JSP) and   |
|                                   |     Youth Allowance (YA) job      |
|                                   |     seekers. If the customer is   |
|                                   |     requesting payments to be     |
|                                   |     cancelled, see Cancellation   |
|                                   |     of JobSeeker Payment (JSP) or |
|                                   |     Youth Allowance (YA) (job     |
|                                   |     seeker) at customer's request |
|                                   | -   Special Benefit (SpB)         |
|                                   | -   Partner Allowance (PA)        |
|                                   | -   Austudy                       |
|                                   |                                   |
|                                   | If the customer is receiving any  |
|                                   | other payment, see Cancellation   |
|                                   | of payments for payment specific  |
|                                   | information.                      |
|                                   |                                   |
|                                   | Has the customer advised they     |
|                                   | have started work?                |
|                                   |                                   |
|                                   | -   Yes, go to Step 2             |
|                                   | -   No, go to Step 3              |
+-----------------------------------+-----------------------------------+
| 2                                 | Customer has started work + Read  |
|                                   | more ...                          |
|                                   |                                   |
|                                   | If the customer is a student in   |
|                                   | receipt of Austudy or YA, see     |
|                                   | Commencing or returning to work   |
|                                   | for Austudy and Youth Allowance   |
|                                   | (Students).                       |
|                                   |                                   |
|                                   | Customers eligible for working    |
|                                   | credit should be advised to       |
|                                   | provide their income details so   |
|                                   | that they can use their Working   |
|                                   | Credit. They may remain current   |
|                                   | at a nil rate for up to 6         |
|                                   | fortnights and retain certain     |
|                                   | benefits before their payment is  |
|                                   | cancelled, as long as some of     |
|                                   | that income is employment income. |
|                                   |                                   |
|                                   | If the customer:                  |
|                                   |                                   |
|                                   | Provides income details, code on  |
|                                   | Employment Income Details (EAN)   |
|                                   | screen. This will allow those     |
|                                   | eligible to utilise their Working |
|                                   | Credit and their 6 fortnights of  |
|                                   | nil payment before their payment  |
|                                   | is cancelled                      |
|                                   |                                   |
|                                   | Has outstanding compliance action |
|                                   | preventing cancellation or        |
|                                   | reporting, refer the job seeker   |
|                                   | to the Participation Solutions    |
|                                   | Team (PST) after coding income    |
|                                   | details                           |
|                                   |                                   |
|                                   | Refuses to supply income details, |
|                                   | payment to be cancelled on the    |
|                                   | Benefit Action (BA) screen 'FTE'  |
|                                   | from date last paid plus one      |
|                                   |                                   |
|                                   | -   For JSP and YA job seekers,   |
|                                   |     if the customer refuses to    |
|                                   |     supply income details, coding |
|                                   |     the Customer Employment       |
|                                   |     Status (EAES) screen will     |
|                                   |     automatically cancel the      |
|                                   |     payment from date paid to     |
|                                   |     plus one                      |
|                                   |                                   |
|                                   | For more information, see the     |
|                                   | appropriate procedure within      |
|                                   | Commencing or returning to work   |
|                                   | (CLK).                            |
|                                   |                                   |
|                                   | Procedure ends here.              |
+-----------------------------------+-----------------------------------+
| 3                                 | Automatic or manual               |
|                                   | cancellation + Read more ...      |
|                                   |                                   |
|                                   | Cancellations can either be       |
|                                   | coded:                            |
|                                   |                                   |
|                                   | -   manually, on the Benefit      |
|                                   |     Action (BA) screen, or        |
|                                   | -   automatically, as a result of |
|                                   |     coding the appropriate        |
|                                   |     screens                       |
|                                   |                                   |
|                                   | Determine if the cancellation     |
|                                   | should be coded automatically or  |
|                                   | manually                          |
|                                   |                                   |
|                                   | For a list of codes for both      |
|                                   | automatic and manual              |
|                                   | cancellation, see Cancellation,   |
|                                   | Suspension and Rejection reason   |
|                                   | codes (CLK)                       |
|                                   |                                   |
|                                   | If the customer is commencing or  |
|                                   | returning to work refer to the    |
|                                   | following procedures:             |
|                                   |                                   |
|                                   | -   Commencing or returning to    |
|                                   |     work for JobSeeker Payment    |
|                                   |     (JSP), Youth Allowance (job   |
|                                   |     seeker), Widow Allowance and  |
|                                   |     Partner Allowance             |
|                                   | -   Commencing or returning to    |
|                                   |     work for Austudy and Youth    |
|                                   |     Allowance (Students)          |
|                                   |                                   |
|                                   | For automatic cancellation, go to |
|                                   | Step 4                            |
|                                   |                                   |
|                                   | For manual coding, go to Step 5   |
+-----------------------------------+-----------------------------------+
| 4                                 | Automatic coding + Read more ...  |
|                                   |                                   |
|                                   | An automatic cancellation occurs  |
|                                   | where details are recorded on the |
|                                   | system (assets are updated and    |
|                                   | this precludes the customer from  |
|                                   | payment). See Assessment of       |
|                                   | Assets for Centrelink payments.   |
|                                   |                                   |
|                                   | If a JSP job seeker is commencing |
|                                   | or returning to study their       |
|                                   | payments will be cancelled. See   |
|                                   | Returning to full-time study.     |
|                                   |                                   |
|                                   | For other automatic cancellation  |
|                                   | codes, see Cancellation,          |
|                                   | Suspension, and Rejection reason  |
|                                   | codes (CLK).                      |
|                                   |                                   |
|                                   | Go to Step 6.                     |
+-----------------------------------+-----------------------------------+
| 5                                 | Manual coding + Read more ...     |
|                                   |                                   |
|                                   | While in the customer's record,   |
|                                   | go to the Benefit Action (BA)     |
|                                   | screen and complete the following |
|                                   | fields. Code:                     |
|                                   |                                   |
|                                   | -   The payment type in the Svc   |
|                                   |     Rsn field                     |
|                                   | -   'CAN' in the Action field     |
|                                   | -   The reason in the Reason      |
|                                   |     field. Use field help (?) or  |
|                                   |     see Cancellation, Suspension  |
|                                   |     and Rejection reason codes    |
|                                   |     (CLK)                         |
|                                   | -   Enter the date of effect of   |
|                                   |     cancellation in the Effect    |
|                                   |     date field or press [Enter]   |
|                                   |     to allow the date of effect   |
|                                   |     to default. Where the         |
|                                   |     customer or their partner is  |
|                                   |     in receipt of Family Tax      |
|                                   |     Benefit (FTB), and their      |
|                                   |     payment is to be cancelled    |
|                                   |     for a period in the past, the |
|                                   |     actual cancellation date must |
|                                   |     be recorded for FTB purposes  |
|                                   |     (not date paid to plus 1)     |
|                                   | -   Source: and DOR               |
|                                   |                                   |
|                                   | Go to the Assessment Results (AR) |
|                                   | screen.                           |
|                                   |                                   |
|                                   | Record details on a DOC. Ensure   |
|                                   | the DOC meets Getting it Right    |
|                                   | (GIR) Minimum Standards, and      |
|                                   | covers how the information was    |
|                                   | received, whether manual advice   |
|                                   | has been sent and other relevant  |
|                                   | information. For more information |
|                                   | see:                              |
|                                   |                                   |
|                                   | -   Online Document Recording     |
|                                   |     (ODR) Resources page for Act  |
|                                   |     References                    |
|                                   | -   Creating documents (Fast      |
|                                   |     Note, Callbacks and ACTDOCs)  |
+-----------------------------------+-----------------------------------+
| 6                                 | Finalise activity and action      |
|                                   | overpayments if appropriate +     |
|                                   | Read more ...                     |
|                                   |                                   |
|                                   | Finalise activity on the          |
|                                   | Assessment Results (AR) screen.   |
|                                   | Service Officers will need to     |
|                                   | adjust any overpayments or        |
|                                   | arrears upon assessment           |
|                                   |                                   |
|                                   | The Finished: field will be       |
|                                   | protected if there are any        |
|                                   | negative adjustments. Each        |
|                                   | negative adjustment needs the     |
|                                   | debt action field completed, via  |
|                                   | the Assessment Consequences (ASC) |
|                                   | screen. Appropriate codes are:    |
|                                   |                                   |
|                                   | -   WVD - Waive debt. See Small   |
|                                   |     debt waivers                  |
|                                   | -   CAL - Calculate debt - If     |
|                                   |     debt definitely exists        |
|                                   | -   INV - Investigation needed if |
|                                   |     it is uncertain if a debt     |
|                                   |     exists or not                 |
|                                   |                                   |
|                                   | Select appropriate code and       |
|                                   | update debt action codes          |
|                                   |                                   |
|                                   | -   Press [Enter] to go back to   |
|                                   |     the AR screen                 |
|                                   | -   Make any other necessary      |
|                                   |     adjustments, for example,     |
|                                   |     selecting the BLD line, and   |
|                                   |     finalise the activity         |
|                                   |                                   |
|                                   | Note: any undetermined debt       |
|                                   | activity will be transferred      |
|                                   | through to the office. The        |
|                                   | Notification Handler (NOHL) will  |
|                                   | not automatically waive debts <   |
|                                   | $50 because these debts may have  |
|                                   | to be raised.                     |
+-----------------------------------+-----------------------------------+
| 7                                 | Check if a manual letter is       |
|                                   | required + Read more ...          |
|                                   |                                   |
|                                   | A manual letter will be required  |
|                                   | if a Manual Follow-up (MFU)       |
|                                   | activity has been created on the  |
|                                   | Activity List (AL) screen after   |
|                                   | the cancellation has been         |
|                                   | finalised.                        |
|                                   |                                   |
|                                   | -   If there is an MFU requiring  |
|                                   |     a manual letter to be sent to |
|                                   |     customer go to Step 8         |
|                                   | -   If there is no MFU            |
|                                   |     cancellation, the advice will |
|                                   |     be automatically sent to the  |
|                                   |     customer. Procedure ends here |
+-----------------------------------+-----------------------------------+
| 8                                 | Send a manual Letter + Read more  |
|                                   | ...                               |
|                                   |                                   |
|                                   | -   Select the MFU activity on    |
|                                   |     the AL screen and complete    |
|                                   |     either a Rej, Can, Sus or     |
|                                   |     Defer pension, allowance or   |
|                                   |     benefit (Q134) letter or a    |
|                                   |     free text (Q999) letter       |
|                                   | -   The letter should advise the  |
|                                   |     customer of the reasons for   |
|                                   |     cancellation of payment       |
|                                   |     and/or imposition of a        |
|                                   |     penalty, and their Review and |
|                                   |     appeal rights                 |
|                                   |                                   |
|                                   | Note: if the customer has a       |
|                                   | correspondence nominee, issue the |
|                                   | same letter to the correspondence |
|                                   | nominee.                          |
+-----------------------------------+-----------------------------------+
| 9                                 | Adjust any overpayments or        |
|                                   | arrears upon assessment + Read    |
|                                   | more ...                          |
|                                   |                                   |
|                                   | When making payment updates       |
|                                   | within Customer First:            |
|                                   |                                   |
|                                   | -   exit the activity             |
|                                   | -   click on the activity which   |
|                                   |     will provide a list of        |
|                                   |     possible actions              |
|                                   | -   select the Assessment Results |
|                                   |     (AR) screen to finalise the   |
|                                   |     activity                      |
|                                   |                                   |
|                                   | This will allow adjustments on    |
|                                   | the Daily Rates Summary (RATS)    |
|                                   | screen.                           |
+-----------------------------------+-----------------------------------+
