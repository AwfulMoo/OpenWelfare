Unfavourable decision, change to reporting or cancellation

Table 2: This table describes the process when a customer requires a
change of payment frequency or no longer has an ongoing entitlement and
their payment must be cancelled.

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Customer does not have an ongoing |
|                                   | entitlement or has Working Credit |
|                                   | to deplete + Read more ...        |
|                                   |                                   |
|                                   | For customers with full-time      |
|                                   | ongoing employment, code the      |
|                                   | employment income, via the        |
|                                   | Earnings and Reporting Workflow,  |
|                                   | with the following:               |
|                                   |                                   |
|                                   | Date of Event - the date the      |
|                                   | customer was paid by their        |
|                                   | employer. If the customer has     |
|                                   | worked but not yet been paid, the |
|                                   | Date of Event will be the         |
|                                   | Entitlement Period End Date       |
|                                   | (EPED) to code hours only         |
|                                   |                                   |
|                                   | Amount - the gross amount paid    |
|                                   |                                   |
|                                   | Frequency field - code as the     |
|                                   | employer pay period frequency     |
|                                   |                                   |
|                                   | -   Weekly (1WE) - where the      |
|                                   |     customer is paid the same     |
|                                   |     amount, on the same day every |
|                                   |     week                          |
|                                   | -   Fortnightly (2WE) - where the |
|                                   |     customer is paid the same     |
|                                   |     amount, on the same day every |
|                                   |     fortnight                     |
|                                   | -   Income for one period (IOP) - |
|                                   |     where income can vary or      |
|                                   |     first employment income       |
|                                   |     amount is for less than the   |
|                                   |     full entitlement period. If   |
|                                   |     this occurs, code the first   |
|                                   |     EPED as IOP and the following |
|                                   |     EPED as the relevant ongoing  |
|                                   |     frequency                     |
|                                   | -   Long Period/Lump sum (LOP) -  |
|                                   |     where income is for more than |
|                                   |     the entitlement period. Start |
|                                   |     and End dates will be         |
|                                   |     required.                     |
|                                   | -   If coding hours only, select  |
|                                   |     HOP to code hours worked in   |
|                                   |     the entitlement period        |
|                                   |                                   |
|                                   | Empl Sts (Employment Status)      |
|                                   | field - as 'FTO' (full-time       |
|                                   | ongoing). This can only be coded  |
|                                   | with a 1WE or 2WE frequency.      |
|                                   | Leave blank for IOP or LOP        |
|                                   | frequency                         |
|                                   |                                   |
|                                   | Hrs PF field - number of hours    |
|                                   | worked. This can be coded with    |
|                                   | 1WE or 2WE frequency. If coding   |
|                                   | income using IOP or LOP, leave    |
|                                   | blank and use frequency HOP to    |
|                                   | code hours. Ensure Empl Sts field |
|                                   | is coded as 'FTO' (full-time      |
|                                   | ongoing). See Recording and       |
|                                   | correcting employment income      |
|                                   | details for more information      |
|                                   |                                   |
|                                   | Income support payments will      |
|                                   | cancel automatically when the     |
|                                   | customer's Working Credit has     |
|                                   | depleted                          |
|                                   |                                   |
|                                   | -   If employment income          |
|                                   |     precluded payment and JSP/YAL |
|                                   |     does not cancel               |
|                                   |     automatically, go to Step 3   |
|                                   |                                   |
|                                   | Check if the customer is eligible |
|                                   | to receive a Special Employment   |
|                                   | Advance (SEA).                    |
|                                   |                                   |
|                                   | Advise the customer about         |
|                                   | providing verification of         |
|                                   | employment income.                |
|                                   |                                   |
|                                   | Before a customer's payment       |
|                                   | ceases, Special Employment        |
|                                   | Advance (SEA) applications can    |
|                                   | be:                               |
|                                   |                                   |
|                                   | -   taken at a service centre, or |
|                                   | -   [] uploaded through the       |
|                                   |     Express Plus Centrelink       |
|                                   |     mobile                        |
|                                   |                                   |
|                                   | When making an unfavourable       |
|                                   | decision, the Service Officer     |
|                                   | must:                             |
|                                   |                                   |
|                                   | -   explain the decision to the   |
|                                   |     customer and give them an     |
|                                   |     opportunity to provide        |
|                                   |     further details or evidence   |
|                                   |     relevant to the decision      |
|                                   | -   tell them about their review  |
|                                   |     and appeal rights, and        |
|                                   | -   if needed, record a request   |
|                                   |     for an explanation or review  |
|                                   |     of the decision               |
|                                   |                                   |
|                                   | This includes unfavourable        |
|                                   | decisions made by Service         |
|                                   | Officers within a Smart Centre.   |
|                                   |                                   |
|                                   | If a Service Officer is not       |
|                                   | speaking with a customer at the   |
|                                   | time of making a decision, they   |
|                                   | must make 2 genuine attempts to   |
|                                   | contact a customer before         |
|                                   | finalising an unfavourable        |
|                                   | decision.                         |
