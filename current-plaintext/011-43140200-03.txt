Process claims

Table 1: claims submitted using digital self service channels are pended
in Mainframe CICs.

Note: MOA skilled Service Officers must attempt first contact resolution
on the phone for Medicare online account claim enquiries. Try to process
all claims regardless of date of lodgement.

|                                   | ...                               |
|                                   |                                   |
|                                   | To locate the pended claim:       |
|                                   |                                   |

|                                   | -   enter 'MQRI' in the control   |

|                                   | -   454 (Resubmit claim for       |
|                                   |     service – Some details not    |
|                                   |     shown or unclear on image)    |
|                                   | -   455 (Resubmit claim for this  |
|                                   |     service – Include account and |
|                                   |     receipt)                      |
|                                   | -   621 (Item not claimable       |
|                                   |     electronically)               |
|                                   |                                   |
|                                   | Review and apply any other        |
|                                   | assessing restrictions by         |
|                                   | following the warning messages.   |
|                                   | Warning messages must be actioned |
|                                   | in order from left to right.      |
|                                   | Further explanations can be       |
|                                   | viewed by placing the cursor at   |
|                                   | the beginning of the return       |
|                                   | message and pressing F1.          |
|                                   |                                   |
|                                   | For a full list of reason codes   |
|                                   | see Indicators, codes, modifiers  |
|                                   | and control lines for claims      |
|                                   | processing in Medicare. Ask a     |
|                                   | team leader if help is needed or  |
|                                   | unable to action.                 |
|                                   |                                   |

|                                   | will automatically be sent to the |

Process a claim with a minor discrepancy

Table 2

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Amending claim data + Read more   |
|                                   | ...                               |
|                                   |                                   |
|                                   | Customers submitting claims using |
|                                   | digital self service channels     |
|                                   | occasionally mistype or omit      |
|                                   | information. This results in a    |
|                                   | minor difference between what     |
|                                   | shows in Mainframe and on the     |
|                                   | invoice.                          |
|                                   |                                   |
|                                   | If fraud is not suspected,        |
|                                   | Service Officers should amend the |
|                                   | claim data to correct the error.  |
|                                   | See the Resources page for        |
|                                   | examples.                         |
|                                   |                                   |
|                                   | Amendments are not acceptable if  |
|                                   | the information is substantially  |
|                                   | different. See the Resources page |
|                                   | for examples.                     |
|                                   |                                   |
|                                   | Amendments are also not           |
|                                   | acceptable if it is suspected the |
|                                   | claim may be fraudulent. These    |
|                                   | claims must be referred for       |
|                                   | investigation.                    |
+-----------------------------------+-----------------------------------+
| 2                                 | Verify details + Read more ...    |
|                                   |                                   |
|                                   | -   Verify all details in         |
|                                   |     Mainframe against what is on  |
|                                   |     the receipt – note any minor  |
|                                   |     discrepancy/ies               |
|                                   | -   Verify the items in           |
|                                   |     mainframe - confirm any items |
|                                   |     that have not been submitted  |
|                                   |     using digital self service    |
|                                   |     claiming process, or need     |
|                                   |     reordering                    |
|                                   | -   If the discrepancy is not     |
|                                   |     minor, reject the services as |
|                                   |     per Step 7 in Process claims  |
|                                   |     table                         |
|                                   | -   If fraud is suspected, it     |
|                                   |     must not be paid. For         |
|                                   |     instructions on how to reject |
|                                   |     these claims, see the         |
|                                   |     ‘Processing issues for claims |
|                                   |     submitted using digital       |
|                                   |     service channels’ table on    |
|                                   |     the Resources page            |
+-----------------------------------+-----------------------------------+
| 3                                 | Modifying claim lines + Read more |
|                                   | ...                               |
|                                   |                                   |
|                                   | Additional lines can only be      |
|                                   | keyed where the invoices claimed  |
|                                   | are for the same provider. Where  |
|                                   | the customer has claimed multiple |
|                                   | invoices with different           |
|                                   | providers, Service officers are   |
|                                   | only to pay the service details   |
|                                   | for one provider on the condition |
|                                   | that the details match what the   |
|                                   | customer has keyed.               |
|                                   |                                   |
|                                   | Invoices for secondary providers  |
|                                   | are to be rejected 453 as the     |
|                                   | customer is required to lodge     |
|                                   | separate accounts for each        |
|                                   | provider.                         |
|                                   |                                   |
|                                   | If the invoice has additional     |
|                                   | items not keyed by the customer:  |
|                                   |                                   |
|                                   | -   insert a new line by keying   |
|                                   |     'I' in the A: field of the    |
|                                   |     claim as shown and press F6   |
|                                   | -   a new line will be inserted   |
|                                   |     into the claim                |
|                                   | -   key the correct claim details |
|                                   |     from the receipt into the new |
|                                   |     line                          |
|                                   |                                   |
|                                   | If the discrepancy is minor and   |
|                                   | the claim is not suspected to be  |
|                                   | fraudulent, modify the line. See  |
|                                   | Resources for examples of         |
|                                   | acceptable and unacceptable       |
|                                   | amendments when processing        |
|                                   | claims:                           |
|                                   |                                   |
|                                   | -   key ‘M’ in the A: field of    |
|                                   |     the affected claim line and   |
|                                   |     press F6                      |
|                                   | -   overtype information in       |
|                                   |     editable field/s              |
|                                   | -   repeat if required and press  |
|                                   |     Enter to finalise claim       |
|                                   |                                   |
|                                   | Note: if any inserted lines are   |
|                                   | incorrect, delete the line by     |
|                                   | keying ‘D’ in the A: field. Where |
|                                   | a payee provider is present on    |
|                                   | the invoice, a PayP line may have |
|                                   | to be inserted, by keying ’Y’ in  |
|                                   | the A: field:                     |
|                                   |                                   |
|                                   | -   correct any errors returned   |
|                                   |     by the system                 |
|                                   | -   when the system displays the  |
|                                   |     message 'enter P to pay',     |
|                                   |     enter 'P' to pay and press    |
|                                   |     Enter                         |
|                                   | -   press F2 to select the next   |
|                                   |     claim                         |

Process a claim that requires Safety Net verification (substantiation)

Table 3: this table describes how to action claims that contain a part
payment, or claims that may require Safety Net verification in the NSCI
screen.

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Determine if Medicare Safety Net  |
|                                   | verification is required + Read   |
|                                   | more ...                          |
|                                   |                                   |
|                                   | Review the claim and ensure that  |
|                                   | the service details on Mainframe  |
|                                   | match the invoice image provided  |
|                                   | by the customer. Correct any      |
|                                   | minor discrepancies.              |
|                                   |                                   |
|                                   | If the invoice is not paid in     |
|                                   | full and contains a part payment, |
|                                   | go to Step 2                      |
|                                   |                                   |
|                                   | If the invoice is:                |
|                                   |                                   |
|                                   | -   paid in full                  |
|                                   | -   contains items that were      |
|                                   |     previously processed as       |
|                                   |     unpaid (payee code 2), and    |
|                                   | -   Mainframe displays a 6-SUS    |
|                                   |     DUP return message, go to     |
|                                   |     Step 3                        |
+-----------------------------------+-----------------------------------+
| 2                                 | Key a part payment in the FEE     |
|                                   | field + Read more ...             |
|                                   |                                   |
|                                   | -   In the FEE field, key the     |
|                                   |     part payment with a P prefix, |
|                                   |     overwriting any FEE amount    |
|                                   |     that was displayed            |
|                                   |     previously. For example, key  |
|                                   |     P2000 if there is a part      |
|                                   |     payment of $20.00.            |
|                                   | -   Press Enter. The system will  |
|                                   |     record the part payment       |
|                                   |     amount and the FEE field will |
|                                   |     be reset to display the       |
|                                   |     schedule fee                  |
|                                   |                                   |
|                                   | Note: if an incorrect part        |
|                                   | payment is keyed, repeat this     |
|                                   | step.                             |
|                                   |                                   |
|                                   | -   Check all return messages and |
|                                   |     finish assessing the claim    |
|                                   |                                   |
|                                   | Procedure ends here.              |
+-----------------------------------+-----------------------------------+
| 3                                 | Verify (substantiate) the claim   |
|                                   | in NSCI + Read more ...           |
|                                   |                                   |
|                                   | -   Take note of the claim        |
|                                   |     details (a temporary          |
|                                   |     screenshot may assist) and    |
|                                   |     clear the Mainframe screen by |
|                                   |     pressing [ESC] or [F12]. The  |
|                                   |     digital self service claim    |
|                                   |     will remain assigned and can  |
|                                   |     be finalised after            |
|                                   |     verification is complete.     |
|                                   | -   Check the NSCI verification   |
|                                   |     screen and verify the         |
|                                   |     relevant service(s) if        |
|                                   |     required                      |
|                                   | -   If the service was for a      |
|                                   |     previous year, use the        |
|                                   |     appropriate control line      |
|                                   |     modifier. For example:        |
|                                   |     NSCI,patient Medicare card    |
|                                   |     number,FROM/010119 to view    |
|                                   |     unverified services for 2019  |
|                                   | -   If the relevant service does  |
|                                   |     not appear in NSCI, that      |
|                                   |     service was either already    |
|                                   |     verified or is not eligible   |
|                                   |     for Safety Net verification   |
|                                   |     (for example, it may be an    |
|                                   |     in-patient service or Child   |
|                                   |     Dental Benefits Schedule      |
|                                   |     service)                      |
|                                   | -   Return to the digital self    |
|                                   |     service claim in MQRI         |
+-----------------------------------+-----------------------------------+
| 4                                 | Reject the digital self service   |
|                                   | claim with RSN 151 + Read more    |
|                                   | ...                               |
|                                   |                                   |
|                                   | -   Reject the digital self       |
|                                   |     service claim with RSN 151    |
|                                   |                                   |
|                                   | RSN 151 has the attached message  |
|                                   | 'Associated service already paid  |
|                                   | adjustment - being processed'.    |

Same day reversal of a claim

Table 4: this table describes how to complete a same day reversal and
reassessment where a Service Officer has processed a MOA claim and
identified an error.

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Identify record for reversal +    |
|                                   | Read more ...                     |
|                                   |                                   |
|                                   | In Mainframe, key NQEI,[insert    |
|                                   | claim ID here],card number and    |
|                                   | press Enter.                      |
|                                   |                                   |
|                                   | claim ID is either:               |
|                                   |                                   |
|                                   | -   HC - MOA                      |
|                                   | -   AC - app                      |
|                                   |                                   |
|                                   | If the paid claim can be          |
|                                   | reversed, it will show as         |
|                                   | PAID-SD:                          |
|                                   |                                   |
|                                   | If there are multiple claims      |
|                                   | listed for that card number:      |
|                                   |                                   |
|                                   | -   Key ‘V’ and press Enter to    |
|                                   |     view the selected digital     |
|                                   |     self-service claim            |
|                                   | -   Press [F2] to return to the   |
|                                   |     list of claims                |
|                                   |                                   |
|                                   | When the correct claim is located |
|                                   | key R to select the claim that    |
|                                   | requires reversal and press       |
|                                   | Enter.                            |
+-----------------------------------+-----------------------------------+
| 2                                 | Reverse the claim + Read more ... |
|                                   |                                   |
|                                   | Overtype NERC with NERR and press |
|                                   | Enter.                            |
|                                   |                                   |
|                                   | This will remove the claim from   |
|                                   | the customer’s NHSI history.      |
|                                   | Service Officers can now          |
|                                   | reprocess.                        |
|                                   |                                   |
|                                   | Go to Step 3 if any further       |
|                                   | changes are required to process   |
|                                   | the claim.                        |
+-----------------------------------+-----------------------------------+
| 3                                 | Reassess claim + Read more ...    |
|                                   |                                   |
|                                   | Key NQEI,[insert claim ID         |
|                                   | here],card number and press       |
|                                   | Enter.                            |
|                                   |                                   |
|                                   | claim ID is either:               |
|                                   |                                   |
|                                   | -   HC - MOA                      |
|                                   | -   AC - app                      |
|                                   |                                   |
|                                   | Key S against the claim with      |
|                                   | status PEND-REV and press Enter.  |
+-----------------------------------+-----------------------------------+
| 4                                 | Complete action + Read more ...   |
|                                   |                                   |
|                                   | To reassess and complete the      |
|                                   | claim, overtype NEPC with NEPP    |
|                                   | and press Enter.                  |
