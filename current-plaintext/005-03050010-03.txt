| 1                                 | AoS must be accepted + Read more  |
|                                   | ...                               |
|                                   |                                   |
|                                   | An AoS case can only be activated |
|                                   | to current (CUR), cancelled or    |
|                                   | expired if the case status is     |
|                                   | accepted (ACC). See AoS status    |
|                                   | codes and explanations.           |
|                                   |                                   |
|                                   | To check if an AoS application    |
|                                   | has been accepted (ACC) search    |
|                                   | the AoS application by:           |
|                                   |                                   |
|                                   | -   the assurer's (Main Partner)  |
|                                   |     Customer Reference Number     |
|                                   |     (CRN)                         |
|                                   | -   AoS case ID or                |
|                                   | -   Visa application number       |
|                                   |                                   |
|                                   | Select Inbox from the landing     |
|                                   | page:                             |
|                                   |                                   |
|                                   | In the Work Item Search select    |
|                                   | Assurance of Support in the Main  |
|                                   | Category dropdown box. Key the:   |
|                                   |                                   |
|                                   | -   assurer’s CRN in the CRN box  |
|                                   |     or                            |
|                                   | -   AoS case ID in the Case Id    |
|                                   |     box or                        |
|                                   | -   visa application number in    |
|                                   |     the Visa Reference box        |
|                                   |                                   |
|                                   | Select Go, the AoS application    |
|                                   | will display in the work item     |
|                                   | table                             |
|                                   |                                   |
|                                   | Select the AoS application        |
|                                   |                                   |
|                                   | Select the Status icon. This will |
|                                   | show the status history. The most |
|                                   | recent status should be Accepted  |
|                                   |                                   |
|                                   | To view the assurer(s) and        |
|                                   | assuree(s) details, select the    |
|                                   | Parties icon                      |
|                                   |                                   |
|                                   | If the Main Partner, Case ID, or  |
|                                   | Visa application number are       |
|                                   | unknown, search by second or      |
|                                   | third assurer or assuree CRNs in  |
|                                   | Process Direct.                   |
|                                   |                                   |
|                                   | If the AoS case needs to be:      |
|                                   |                                   |
|                                   | -   current, go to Step 2         |
|                                   | -   cancelled, go to Step 5       |
|                                   | -   expired, go to Step 6         |
|                                   | -   transferred, go to Step 7     |

| 2                                 | Activating an AoS + Read more ... |
|                                   |                                   |
|                                   | Manual activation should only be  |
|                                   | required when the Department of   |
|                                   | Home Affairs data link has not    |
|                                   | automatically populated the       |
|                                   | details. For example, there is no |
|                                   | information from the datalink, or |
|                                   | the visa details from the         |
|                                   | datalink is different to the visa |
|                                   | details coded in the AoS          |
|                                   | application.                      |
|                                   |                                   |
|                                   | Enter a start date to activate an |
|                                   | AoS, the case status will update  |
|                                   | to current (CUR). The start date  |
|                                   | is the later of:                  |
|                                   |                                   |
|                                   | -   the date the assuree arrived  |
|                                   |     in Australia on the relevant  |
|                                   |     visa, or                      |
|                                   | -   the date the visa was granted |
|                                   |     if the assuree is already in  |
|                                   |     Australia                     |
|                                   |                                   |
|                                   | The visa and arrival information  |
|                                   | are available on the RSLEG and    |
|                                   | RSIM screens via Department of    |
|                                   | Home Affairs datalink or          |
|                                   | confirmed with Centrelink         |
|                                   | International Services.           |
|                                   |                                   |
|                                   | AoS duration                      |
|                                   |                                   |
|                                   | The AoS duration is 1, 2, 4 or 10 |
|                                   | years depending on the AoS visa   |
|                                   | subclass and the date the AoS     |
|                                   | application was lodged:           |
|                                   |                                   |
|                                   | -   202 visa (offshore visa) –    |
|                                   |     one year                      |
|                                   | -   115, 117 (offshore visa), 835 |
|                                   |     and 837 (onshore visa) – 2    |
|                                   |     years                         |
|                                   | -   143 (can be granted onshore   |
|                                   |     or offshore) and 864 (onshore |
|                                   |     visa) – 10 years              |
|                                   | -   103 (offshore visa), 804      |
|                                   |     (onshore visa) and other AoS  |
|                                   |     visa subclass – 4 years if    |
|                                   |     the AoS application was       |
|                                   |     lodged after 1 January 2019   |
|                                   |     and 2 years if lodged before  |
|                                   |     1January 2019                 |
|                                   |                                   |
|                                   | Use the AoS checker to find out   |
|                                   | the AoS duration. To manually     |
|                                   | activate the AoS go to Step 3.    |
+-----------------------------------+-----------------------------------+
| 3                                 | Manually enter start date on      |
|                                   | assuree record (AOAE screen in    |
|                                   | Process Direct) + Read more ...   |
|                                   |                                   |
|                                   | -   Select Customer Summary from  |
|                                   |     the Process Direct landing    |
|                                   |     page                          |
|                                   | -   From the Search Customers     |
|                                   |     screen key the assuree’s CRN  |
|                                   |     in the CRN field, select Go   |
|                                   | -   Select the assuree record     |
|                                   |     from the Search Results field |
|                                   |                                   |
|                                   | On the Start screen               |
|                                   |                                   |
|                                   | Key 'AOAE' in the Super Key field |
|                                   |                                   |
|                                   | Select + (Add)                    |
|                                   |                                   |
|                                   | AoS Start date: key the later of: |
|                                   |                                   |
|                                   | -   the date the assuree arrived  |
|                                   |     into Australia or             |
|                                   | -   the date the relevant visa    |
|                                   |     was granted                   |
|                                   |                                   |
|                                   | Expected Expiry date: this date   |
|                                   | is automatically calculated based |
|                                   | on the start date that is entered |
|                                   | and the type of visa subclass. If |
|                                   | the Expiry Date has not been      |
|                                   | automatically updated check:      |
|                                   |                                   |
|                                   | -   the visa subclass if onshore  |
|                                   |     or offshore                   |
|                                   | -   the AoS application lodgement |
|                                   |     date will also determine the  |
|                                   |     applicable AoS period         |
|                                   | -   enter the Expected Expiry     |
|                                   |     date manually. See, Examples  |
|                                   |     of offshore and onshore visas |
|                                   |     or Example of 143 visa (can   |
|                                   |     be granted onshore or         |
|                                   |     offshore)                     |
|                                   |                                   |
|                                   | If there is more than one         |
|                                   | assuree, code the start date for  |
|                                   | each assuree on each individual   |
|                                   | assuree record. By entering this  |
|                                   | information, the assuree record   |
|                                   | automatically updates and the     |
|                                   | case status becomes current       |
|                                   | (CUR).                            |
|                                   |                                   |
|                                   | Note: a start date must be        |
|                                   | entered even if the expected      |
|                                   | expiry date has passed. Once the  |
|                                   | start date is entered, the case   |
|                                   | will expire immediately and the   |
|                                   | relevant expiry letters will also |
|                                   | be issued.                        |
|                                   |                                   |
|                                   | -   Enter DOR and Channel, and    |
|                                   |     select Assess                 |
|                                   | -   On Entitlement (ELD) screen   |
|                                   |     select Finish                 |
|                                   | -   Record details in Note        |
|                                   |     template and select Finish    |
|                                   |                                   |
|                                   | Activation letter to assurer      |
|                                   |                                   |
|                                   | Once the activation to current    |
|                                   | (CUR) has occurred a letter will  |
|                                   | be automatically sent to the      |
|                                   | assurer(s). The letter will       |
|                                   | contain:                          |
|                                   |                                   |
|                                   | -   advice that their             |
|                                   |     responsibilities as an        |
|                                   |     assurer have begun            |
|                                   | -   details of the assuree(s)     |
|                                   |     whose support has become      |
|                                   |     active                        |
|                                   |                                   |
|                                   | It may take a day for the letter  |
|                                   | to be triggered. If the letter is |
|                                   | not available on the History      |
|                                   | Summary (HS) screen in the        |
|                                   | Process Direct or in Search       |
|                                   | Outbound Correspondence in        |
|                                   | Customer First, check the record  |
|                                   | again the next day.               |
|                                   |                                   |
|                                   | Does the AoS start date need to   |
|                                   | be edited?                        |
|                                   |                                   |
|                                   | -   Yes, go to Step 4             |
|                                   | -   No, procedure ends here       |

| 4                                 | How to edit an AoS Start date +   |
|                                   | Read more ...                     |
|                                   |                                   |
|                                   | If the AoS start date entered on  |
|                                   | the AOAE screen is incorrect:     |
|                                   |                                   |
|                                   | -   Select Edit.                  |
|                                   | -   AoS Start Date: enter the     |

|                                   | -   Expected Expiry Date: enter   |
|                                   |     the expiry date               |
|                                   | -   Select Save                   |
|                                   | -   Receipt Date: today’s date    |
|                                   | -   Channel: Internal             |
|                                   | -   Select Save then Assess       |
|                                   | -   Success > Updates applied     |
|                                   |     successfully! will display    |
|                                   |     select OK                     |
|                                   |                                   |
|                                   | Procedure ends here.              |

| 5                                 | Cancelling an AoS + Read more ... |
|                                   |                                   |
|                                   | Select Inbox from the Process     |
|                                   | Direct landing page               |
|                                   |                                   |
|                                   | -   In the Work Item Search field |
|                                   |     select Assurance of Support   |
|                                   |     from the Main Category        |
|                                   |     dropdown box                  |
|                                   | -   To navigate to the AoS case   |
|                                   |     key the:                      |
|                                   | -   Assurer (main partner) CRN,   |
|                                   |     or                            |
|                                   | -   AoS Case Id, or               |
|                                   | -   Visa Reference                |
|                                   |                                   |
|                                   | Confirm the reason if the         |
|                                   | cancellation appears in the       |
|                                   | following non-discretionary list: |
|                                   |                                   |
|                                   | -   the assuree's visa has been   |
|                                   |     cancelled by the Department   |
|                                   |     of Home Affairs               |
|                                   | -   the assuree, other than for a |
|                                   |     CSP entrant, is subsequently  |
|                                   |     granted a refugee visa after  |
|                                   |     arrival in Australia          |
|                                   | -   death of a sole assurer or    |
|                                   |     assuree will auto expire an   |
|                                   |     AoS case                      |
|                                   |                                   |
|                                   | If the cancellation is not on the |
|                                   | non-discretionary list, staff     |
|                                   | must make a discretionary         |
|                                   | determination. See Resources for  |
|                                   | examples.                         |
|                                   |                                   |
|                                   | Decision not to cancel:           |
|                                   |                                   |
|                                   | -   DOC the decision and issue a  |
|                                   |     Q999 letter detailing reasons |
|                                   |     the cancellation is not       |
|                                   |     approved, include the right   |
|                                   |     to review of a decision.      |
|                                   |     Procedure ends here           |
|                                   |                                   |
|                                   | Decision to cancel:               |
|                                   |                                   |
|                                   | In the AoS case, select AoS       |
|                                   | Action (AOBA) screen from the AoS |
|                                   | case level AoS Task Selector      |
|                                   |                                   |
|                                   | -   Select CAN-Cancelled from the |
|                                   |     Action: dropdown menu         |
|                                   | -   Select a Reason from the      |
|                                   |     dropdown menu                 |
|                                   | -   Enter the Effect Date         |
|                                   | -   Select Next                   |
|                                   | -   Select Assess on the SAP      |
|                                   |     Warnings and Errors (SWE)     |
|                                   |     screen                        |
|                                   | -   Select Finalise on            |
|                                   |     Entitlement (ELD) screen      |
|                                   | -   Record details in a Note      |
|                                   |     template, select Finish       |
|                                   |                                   |
|                                   | This will cancel the case, end    |
|                                   | the assurer-assuree links and the |
|                                   | case status will update to        |
|                                   | cancelled (CAN).                  |
|                                   |                                   |
|                                   | Cancellation reasons are:         |
|                                   |                                   |
|                                   | -   OTH - any other reason        |
|                                   | -   NLR - no longer required      |
|                                   | -   AAG - another visa granted    |
|                                   |     before current AoS expired    |
|                                   |     withdrawn by assurer          |
|                                   | -   CLR - claim withdrawn         |
|                                   | -   DTH - death of an assurer or  |
|                                   |     assuree                       |
|                                   | -   VRJ - visa withdrawn,         |
|                                   |     cancelled or rejected by the  |
|                                   |     Department of Home Affairs    |
|                                   | -   XLC - exceptional             |
|                                   |     circumstances                 |
|                                   |                                   |
|                                   | Enter Effect Date, then save      |
|                                   |                                   |
|                                   | Record details in a Note.         |
|                                   |                                   |
|                                   | To end an AoS for one assuree in  |
|                                   | the multiple assurees case        |
|                                   |                                   |
|                                   | -   Key AOAE in the Super Key to  |
|                                   |     navigate to the Assuree       |
|                                   |     Arrival Details (AOAE) screen |
|                                   | -   Select Edit                   |
|                                   | -   Update the Expiry Date and    |
|                                   |     Expiry Reason                 |
|                                   | -   Receipt Date: todays date     |
|                                   | -   Channel: Internal             |
|                                   | -   Select Save, then Assess      |
|                                   | -   Success > Updates applied     |
|                                   |     successfully! will display,   |
|                                   |     select OK                     |
|                                   | -   On Entitlement (ELD) screen   |
|                                   |     select Finish                 |
|                                   | -   Record details in Note        |
|                                   |     template and select Finish    |
|                                   |                                   |
|                                   | This will cancel the AoS for the  |
|                                   | particular assuree, and the other |
|                                   | assurees will remain active in    |
|                                   | the AoS case. If AoS Start Date   |
|                                   | has not been entered, the AoS     |
|                                   | Start Date must first be updated  |
|                                   | then the appropriate Expiry Date  |
|                                   | can be entered.                   |
|                                   |                                   |
|                                   | Expiry Reasons are:               |
|                                   |                                   |
|                                   | -   AAG – Another visa granted    |
|                                   |     before current AoS expired    |
|                                   | -   CNV – Conversion of heritage  |
|                                   |     codes for migration           |
|                                   | -   DTH – Assurer/Assuree         |
|                                   |     Deceased                      |
|                                   | -   EXP – AoS Period Ended –      |
|                                   |     Assuree Expired               |
|                                   | -   FSB – Failed To Supply Bank   |
|                                   |     Guarantee                     |
|                                   | -   GAC – Granted Australian      |
|                                   |     Citizenship                   |
|                                   | -   INV – AoS Invalid             |
|                                   | -   M2E – More than 2 Assurees    |
|                                   | -   NLR – No Longer Required      |
|                                   | -   OTH – Any Other Reason        |
|                                   | -   REF – Refugee Visa Granted    |
|                                   | -   VCA – Assuree Outside         |
|                                   |     Australian Government         |
|                                   | -   VIS – Another AoS Visa        |
|                                   |     Granted                       |
|                                   | -   XLC – Exception Circumstances |

| 6                                 | AoS expiry + Read more ...        |
|                                   |                                   |
|                                   | If an AoS period has been served  |
|                                   | in full, the case will expire     |
|                                   | automatically and the case status |
|                                   | will update to (EXP).             |
|                                   |                                   |
|                                   | Record the data for all assures.  |
|                                   |                                   |
|                                   | When the start date is coded on   |
|                                   | the Assuree Arrival Details       |
|                                   | (AOAE) screen, the Expected       |
|                                   | Expiry Date field will            |
|                                   | automatically have the expiry     |
|                                   | date calculated. Once this date   |
|                                   | is reached the field              |
|                                   | automatically becomes the AoS     |
|                                   | Expiry Date field.                |
|                                   |                                   |
|                                   | A case may only be expired early  |
|                                   | if another visa requiring an AoS  |
|                                   | has been granted. The full length |
|                                   | of both AoS periods will not      |
|                                   | exceed the period for which the   |
|                                   | first AoS was given.              |
|                                   |                                   |
|                                   | Expiration Letters                |
|                                   |                                   |
|                                   | Upon expiry of the AoS and no     |
|                                   | recoverable payments were made to |
|                                   | the assurees, letters             |
|                                   | automatically generate to advise  |
|                                   | that the AoS period has expired.  |
|                                   |                                   |
|                                   | If a bank guarantee supported by  |
|                                   | a term deposit was provided, the  |
|                                   | expiry letter will also advise    |
|                                   | the assurer(s) the funds can be   |
|                                   | released by the Commonwealth Bank |
|                                   | of Australia (CBA).               |
|                                   |                                   |
|                                   | A letter will also be sent to the |
|                                   | CBA advising of the expiry and    |
|                                   | confirming that any bank          |
|                                   | guarantee can be cancelled and    |
|                                   | any remaining term deposit funds  |
|                                   | can be released to the account    |
|                                   | holders.                          |
|                                   |                                   |
|                                   | The assurer(s) must contact the   |
|                                   | CBA within 31 days of the date of |
|                                   | issue. Otherwise, the CBA will    |
|                                   | ask the assurer to get a new      |
|                                   | expiry letter from Services       |
|                                   | Australia. AoS staff will reissue |
|                                   | the manual OLA letter code (Q271) |
|                                   |                                   |
|                                   | To issue a Q271 manual letter to  |
|                                   | CBA:                              |
|                                   |                                   |
|                                   | -   In Customer First>Customer    |
|                                   |     Record> LD>letter code        |
|                                   |     Q271>Service reasons AoS      |
|                                   | -   Type: Dear Sir/Madam = 1,     |
|                                   |     Account Number, Assurer Name, |
|                                   |     Amount, BGRCVY: '3' for full  |
|                                   |     bond release contact          |
|                                   |     Assurance of Support (AoS)    |
|                                   |     Employment Services Telephone |
|                                   |     line                          |
|                                   | -   Command: FIN                  |

| 7                                 | Transfer of an AoS case + Read    |
|                                   | more ...                          |
|                                   |                                   |
|                                   | After making the determination to |
|                                   | transfer an AoS case. The         |
|                                   | existing AoS case must be         |
|                                   | cancelled and coded as if it is a |
|                                   | new claim.                        |
|                                   |                                   |
|                                   | Making the decision               |
|                                   |                                   |
|                                   | Consider cancelling the AoS case, |
|                                   | see Step 5                        |
|                                   |                                   |
|                                   | Which of the following categories |
|                                   | does the transfer request meet?   |
|                                   |                                   |
|                                   | Non-discretionary:                |
|                                   |                                   |
|                                   | -   a court ordered transfer of   |
|                                   |     an Assurance of Support,      |
|                                   |     refer to Step 5 Cancelling an |
|                                   |     AoS                           |
|                                   |                                   |
|                                   | Discretionary:                    |
|                                   |                                   |
|                                   | -   a long term injury or illness |
|                                   |     (individual)                  |
|                                   | -   liquidation of a company      |
|                                   |     (Body)                        |
|                                   | -   if family and domestic        |
|                                   |     violence is substantiated     |
|                                   | -   any other reason provided by  |
|                                   |     the assurer                   |
|                                   |                                   |
|                                   | Collect evidence to make the      |
|                                   | discretionary determination.      |
|                                   | Including verification from the   |
|                                   | potential new assurer(s) to       |
|                                   | ensure they meet the requirements |
|                                   | to be an assurer.                 |
|                                   |                                   |
|                                   | The Level 2 AoS Helpdesk can been |
|                                   | contacted for assistance. The     |
|                                   | assessor will have the final      |
|                                   | decision.                         |
|                                   |                                   |
|                                   | See Resources for possible        |
|                                   | reasons for a discretionary       |
|                                   | transfer.                         |
|                                   |                                   |
|                                   | Transfer not approved.            |
|                                   |                                   |
|                                   | If the potential new assurer(s)   |
|                                   | do not meet the requirements, the |
|                                   | transfer cannot proceed. DOC the  |
|                                   | rejection of the request to       |
|                                   | transfer. Issue a Q999 letter     |
|                                   | with the reasons the request to   |
|                                   | transfer was rejected. Include    |
|                                   | the right to review of a decision |
|                                   | paragraph. Procedure ends here.   |
|                                   |                                   |
|                                   | Transfer approved cancel the AoS  |
|                                   | case.                             |
|                                   |                                   |
|                                   | In the AoS case, select AoS       |
|                                   | action (AOBA) screen from the     |
|                                   | Task Selector.                    |
|                                   |                                   |
|                                   | -   Select Add                    |
|                                   | -   Select CAN-Cancelled from the |
|                                   |     Action: dropdown menu         |
|                                   | -   Select a Reason from the      |
|                                   |     dropdown menu                 |
|                                   | -   Enter the Effect Date         |
|                                   | -   Select Next                   |
|                                   | -   Select Assess on the SAP      |
|                                   |     Warnings and Errors (SWE)     |
|                                   |     screen                        |
|                                   | -   Select Finalise on            |
|                                   |     Entitlement (ELD) screen      |
|                                   | -   Record details in a Note      |
|                                   |     template, select Finish       |
|                                   |                                   |
|                                   | Once the AoS case is cancelled,   |
|                                   | re-index the new AoS case. See    |
|                                   | Processing applications to become |
|                                   | an assurer.                       |
|                                   |                                   |
|                                   | Note: in non-discretionary cases, |
|                                   | other than identity confirmation, |
|                                   | there is no requirement to        |
|                                   | collect additional evidence, and  |
|                                   | the minimum value of the income   |
|                                   | threshold can be coded.           |

| 8                                 | Bank guarantee and term deposit + |
|                                   | Read more ...                     |
|                                   |                                   |
|                                   | In cases involving a bond during  |
|                                   | a transfer, there are 3           |
|                                   | scenarios:                        |
|                                   |                                   |
|                                   | -   the court orders the transfer |
|                                   |     of the bond from the existing |
|                                   |     assurer to the potential new  |
|                                   |     assurer. Send a copy of the   |
|                                   |     court order to the Level 3    |
|                                   |     AoS Helpdesk with the request |
|                                   |     to transfer the name(s) of    |
|                                   |     bond as ordered, or           |
|                                   | -   in discretionary cases, the   |
|                                   |     existing assurer(s) approves  |
|                                   |     the existing bond be          |
|                                   |     transferred in name to the    |
|                                   |     potential new assurer(s).     |
|                                   |     Send evidence (usually a      |
|                                   |     statutory declaration from    |
|                                   |     the existing assurer(s)) to   |
|                                   |     Level 3 AoS Helpdesk with the |
|                                   |     request to transfer the       |
|                                   |     name(s) of bond, or           |
|                                   | -   the existing assurer(s) will  |
|                                   |     request the release of the    |
|                                   |     existing bond. See Assurance  |
|                                   |     of Support (AoS) bank         |
|                                   |     guarantee and term deposit    |
