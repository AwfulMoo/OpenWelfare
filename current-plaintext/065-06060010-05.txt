Legend - Explanation of reason code fields

Table 1

+-----------------------------------+-----------------------------------+
| Item                              | Description                       |
+-----------------------------------+-----------------------------------+
| 1                                 | Reason Code                       |
|                                   |                                   |
|                                   | Gives the three letter            |
|                                   | abbreviation for the cancellation |
|                                   | reason described.                 |
+-----------------------------------+-----------------------------------+
| 2                                 | Initiation                        |
|                                   |                                   |
|                                   | Shows if the cancellation reason  |
|                                   | is automatic, manual or can be    |
|                                   | both.                             |
|                                   |                                   |
|                                   | -   A = automatic                 |
|                                   | -   M = manual                    |
|                                   | -   B = is when the reason can be |
|                                   |     manually used or if the       |
|                                   |     correct CSHC screen is        |
|                                   |     updated the system will       |
|                                   |     automatically cancel the      |
|                                   |     CSHC. Automatic cancellation  |
|                                   |     or rejection is the preferred |
|                                   |     option in these cases         |
|                                   | -   C = Cancellation              |
|                                   | -   R = Rejection                 |
|                                   | -   SLR = Streamline Rejection    |
|                                   |     (Process Direct)              |
+-----------------------------------+-----------------------------------+
| 3                                 | Description                       |
|                                   |                                   |
|                                   | Gives a definition for the three  |
|                                   | letter abbreviation.              |
+-----------------------------------+-----------------------------------+
| 4                                 | Cancellation and Rejection        |
|                                   | Reasons                           |
|                                   |                                   |
|                                   | Cancellation and rejection        |
|                                   | reasons can be recorded manually  |
|                                   | via the Benefit Action (BA)       |
|                                   | screen, or automatically by the   |
|                                   | system. Automatic cancellations   |
|                                   | or rejections occur when          |
|                                   | information provided by the       |
|                                   | customer disqualifies them from   |
|                                   | entitlement, or the customer      |
|                                   | fails to provide information. For |
|                                   | example, if the customer provides |
|                                   | income details which exceed the   |
|                                   | CSHC income limit, then the       |
|                                   | system is likely to cancel/reject |
|                                   | the card with reason code INC     |
|                                   | (excess income) as the income is  |
|                                   | too high.                         |
|                                   |                                   |
|                                   | It is preferable, when possible,  |
|                                   | to allow the system to            |
|                                   | automatically cancel/reject the   |
|                                   | CSHC by updating the customer's   |
|                                   | information on the relevant       |
|                                   | screens. This enables other       |
|                                   | staff, such as Smart Centre       |
|                                   | staff, to determine the reason    |
|                                   | for cancellation/rejection        |
|                                   | without requesting further        |
|                                   | information, should a customer    |
|                                   | contact us or request a review.   |
