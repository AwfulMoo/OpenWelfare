| 3                                 | Customer does not have all        |
|                                   | required documents + Read more    |
|                                   | ...                               |
|                                   |                                   |
|                                   | Alternative Identity assessments  |
|                                   | help the agency understand the    |
|                                   | circumstances of customers having |
|                                   | difficulty proving who they are.  |
|                                   | If a customer attends in person   |
|                                   | or contacts by phone, staff MUST  |
|                                   | review their current identity     |
|                                   | status and discuss with the       |
|                                   | customer:                         |
|                                   |                                   |
|                                   | -   their ability to provide      |
|                                   |     outstanding identity          |
|                                   |     documents during the          |
|                                   |     assessment or at a later      |
|                                   |     time, and                     |
|                                   | -   available review periods, and |
|                                   | -   if the Alternative identity   |
|                                   |     assessment can be completed   |
|                                   |     by phone or in person, by     |
|                                   |     staff considering if the      |
|                                   |     customer has exceptional      |
|                                   |     circumstances, and            |
|                                   | -   if the customer requires a    |
|                                   |     myGov linking code issued by  |
|                                   |     phone or in person, so        |
|                                   |     customers can upload          |
|                                   |     outstanding identity          |
|                                   |     documents                     |
|                                   |                                   |
|                                   | On the Identity Confirmation      |
|                                   | Dashboard:                        |
|                                   |                                   |
|                                   | In each category staff must       |
|                                   | assess if a customer can or       |
|                                   | cannot give an approved identity  |
|                                   | document.                         |
|                                   |                                   |
|                                   | Can provide:                      |
|                                   |                                   |
|                                   | -   code and verify the identity  |
|                                   |     documents the customer has    |
|                                   |     given                         |
|                                   |                                   |
|                                   | Can provide later:                |
|                                   |                                   |
|                                   | -   select YY - will provide      |
|                                   |     later: this will result in a  |
|                                   |     review period. This may be a  |
|                                   |     reasonable option where you   |
|                                   |     have assessed that an         |
|                                   |     Alternative Identity          |
|                                   |     assessment is required        |
|                                   |     because the customer does not |
|                                   |     have a photo identity         |
|                                   |     document, but has provided a  |
|                                   |     verified PUiC and SUiC        |
|                                   |     document and has indicated    |
|                                   |     that they can provide a CoI   |
|                                   |     within a review period.       |
|                                   |                                   |
|                                   | Cannot provide:                   |
|                                   |                                   |
|                                   | select ZZ - Cannot provide: the   |
|                                   | customer can never give a         |
|                                   | verified identity document        |
|                                   | Note: if the customer can give    |
|                                   | documents for each category but   |
|                                   | cannot complete linkage:          |
|                                   |                                   |
|                                   | -   code and verify CoI & PUiC    |
|                                   |     identity documents and        |
|                                   | -   select ZZ - Cannot provide    |
|                                   |     for the SUiC document         |
|                                   | -   complete an Alternative       |
|                                   |     identity assessment           |
|                                   |                                   |
|                                   | select Next                       |
|                                   |                                   |
|                                   | The Alternative Identity          |
|                                   | assessment workflow will display. |
|                                   | Go to Step 4.                     |

| 4                                 | Alternative Identity assessment + |
|                                   | Read more ...                     |
|                                   |                                   |
|                                   | Based on the customer's           |
|                                   | circumstances, the Alternative    |
|                                   | Identity assessment workflow will |
|                                   | determine:                        |
|                                   |                                   |
|                                   | -   the questions to ask, and     |
|                                   | -   the most appropriate          |
|                                   |     Alternative Identity category |
|                                   |     and review period             |
|                                   |                                   |
|                                   | The workflow will prompt Service  |
|                                   | Officers to verify details on the |
|                                   | customer's record. For example,   |
|                                   | previous addresses, other names   |
|                                   | etc.                              |
|                                   |                                   |
|                                   | -   select Next to continue       |
|                                   | -   complete the full question    |
|                                   |     set                           |
|                                   |                                   |
|                                   | Note: Service Officers must ask   |
|                                   | all questions in the workflow and |
|                                   | accurately record the customer's  |
|                                   | answers.                          |
|                                   |                                   |
|                                   | Once the customer has answered    |
|                                   | all questions, an overview screen |
|                                   | displays their responses.         |
|                                   |                                   |
|                                   | -   check all responses are       |
|                                   |     accurate before continuing    |
|                                   | -   select Previous and return to |
|                                   |     the workflow to amend any     |
|                                   |     response, if required         |
|                                   | -   select Next and FINISH to     |
|                                   |     finalise the workflow. A      |
|                                   |     pre-filled Fast Note will     |
|                                   |     generate and the appropriate  |
|                                   |     review period will auto apply |
|                                   |     to the customer's record      |
|                                   |                                   |
|                                   | The Identity Confirmation         |
|                                   | Dashboard will show the           |
|                                   | Alternative Identity category.    |
|                                   |                                   |
|                                   | -   tell the customer the         |
|                                   |     assessment outcome and review |
|                                   |     period                        |
|                                   |                                   |
|                                   | For claims processed outside      |
|                                   | Process Direct:                   |
|                                   |                                   |
|                                   | -   manually code RF on the POI   |
|                                   |     screen in Customer First when |
|                                   |     finalising the claim          |
|                                   | -   go to Step 9                  |
|                                   |                                   |
|                                   | Centrelink International Services |
|                                   |                                   |
|                                   | -   alternative Identity coding   |
|                                   |     for customers granted under   |
|                                   |     International Social Security |
|                                   |     Agreements is completed       |
|                                   |     through the POI screen (by    |
|                                   |     Centrelink International      |
|                                   |     (CIS) staff only)             |
|                                   | -   see Proof of Identity         |
|                                   |     documents for overseas        |
|                                   |     customers. (Centrelink        |
|                                   |     International (CIS) staff     |
|                                   |     only)                         |
|                                   | -   procedure ends here           |

|                                   | at an Agent site + Read more ...  |

|                                   | contact Silver Service to advise  |

|                                   | customer has presented. The       |
|                                   | Silver Service Officer must       |
|                                   | authenticate Specified Personnel  |
|                                   | at Rural and Remote Agent sites   |
|                                   | and complete proof of record      |
|                                   | ownership. Refer to Silver        |
|                                   | Service phone line for Specified  |
|                                   | Personnel at Agent sites          |
|                                   |                                   |
|                                   | the Silver Service Officer can    |
|                                   | complete an Alternative Identity  |
|                                   | assessment with the agent and     |

|                                   |                                   |
|                                   | important: if the customer does   |
|                                   | not have their CoI document,      |

|                                   | or Aboriginal Population record,  |
|                                   | see Commencement of Identity      |
|                                   |                                   |
|                                   | complete the Alternative Identity |
|                                   | workflow                          |
|                                   |                                   |
|                                   | based on the customer's           |
|                                   | circumstances, the workflow will  |
|                                   | determine:                        |
|                                   |                                   |
|                                   | -   the questions to ask, and     |
|                                   | -   the most appropriate          |
