PARS Approval error messages

Resolve PARS Approval error messages

This table describes how to address error messages that relate to Prior
Financial Authorisation within a claim using the Prior Approval &
Referral System (PARS).

+-----------------------------------+-----------------------------------+
| Item                              | Description                       |
+-----------------------------------+-----------------------------------+
| 1                                 | 4-PAR NFND + Read more ...        |
|                                   |                                   |
|                                   | For items that require an         |
|                                   | approval record on the patient’s  |
|                                   | VEPI/PARS screen.                 |
|                                   |                                   |
|                                   | When there is no prior approval   |
|                                   | record present which covers the   |
|                                   | services being processed or the   |
|                                   | prior approval record for the     |
|                                   | services being processed has been |
|                                   | rejected, the system displays the |
|                                   | message 4-PAR NFND and the first  |
|                                   | item number involved is           |
|                                   | highlighted.                      |
|                                   |                                   |
|                                   | When an item that requires prior  |
|                                   | approval is involved, it will be  |
|                                   | necessary to press the PF2-PARS   |
|                                   | key to display the PARS/VEPI      |
|                                   | screen for the patient:           |
|                                   |                                   |
|                                   | -   if a rejected prior approval  |
|                                   |     record exists for the         |
|                                   |     service/s involved, reject    |
|                                   |     the service/s claimed with    |
|                                   |     RSN 289 (prior approval       |
|                                   |     sought but not granted for    |
|                                   |     this item) and press the      |
|                                   |     Enter key. Processing then    |
|                                   |     continues in the normal       |
|                                   |     manner                        |
|                                   | -   if a prior approval record    |
|                                   |     covering the service/s        |
|                                   |     claimed does not exist,       |
|                                   |     reject the service/s involved |
|                                   |     with RSN 279 (prior approval  |
|                                   |     not sought or advice of       |
|                                   |     admission not lodged) and     |
|                                   |     press the Enter key.          |
|                                   |     Processing then continues in  |
|                                   |     the normal manner             |
|                                   |                                   |
|                                   | Note:                             |
|                                   |                                   |
|                                   | -   the message 4-PAR NFND        |
|                                   |     usually occurs when there is  |
|                                   |     no prior approval or referral |
|                                   |     on the patient’s prior        |
|                                   |     approval/referral VEPI record |
|                                   |     for the treatment type        |
|                                   |     involved. When there is a     |
|                                   |     referral or prior approval    |
|                                   |     record present for the        |
|                                   |     treatment type, the message   |
|                                   |     4-PAR NMCH will usually be    |
|                                   |     displayed                     |
|                                   | -   the message 4-PAR NFND can    |
|                                   |     also be displayed when the    |
|                                   |     patient has moved interstate  |
|                                   |     and the prior approval record |
|                                   |     is on the original file       |
|                                   |     number. In this situation,    |
|                                   |     the current file number will  |
|                                   |     be shown on the screen. This  |
|                                   |     file number should be used to |
|                                   |     process the claim.            |
+-----------------------------------+-----------------------------------+
| 2                                 | 4-PAR NMCH + Read more ...        |
|                                   |                                   |
|                                   | For items that require an         |
|                                   | approval record on the patient’s  |
|                                   | VEPI/PARS screen.                 |
|                                   |                                   |
|                                   | When the system cannot uniquely   |
|                                   | match the service/s being         |
|                                   | processed to a prior approval     |
|                                   | record, the message 4-PAR NMCH is |
|                                   | displayed and the first item      |
|                                   | number involved is highlighted.   |
|                                   |                                   |
|                                   | In this situation, the            |
|                                   | information held on the patient's |
|                                   | PARS/VEPI screen will need to be  |
|                                   | checked by pressing PF2-PARS      |
|                                   | before a decision is made to      |
|                                   | either accept or reject the       |
|                                   | service/s involved.               |
|                                   |                                   |
|                                   | When identifying the PARS records |
|                                   | on the patient's prior approval/  |
|                                   | referral record (VEPI) screen,    |
|                                   | look for records that have an     |
|                                   | effective date close to the       |
|                                   | date/s of service involved.       |
|                                   |                                   |
|                                   | Assess why the service is not     |
|                                   | automatically matching to the     |
|                                   | approval record:                  |
|                                   |                                   |
|                                   | -   if the date of service falls  |
|                                   |     outside the ‘Effective Date’  |
|                                   |     and ‘End Date’ of a relevant  |
|                                   |     PARS record, reject the       |
|                                   |     service/s with RSN code S 282 |
|                                   |     [Date of service falls        |
|                                   |     outside of approval/referral  |
|                                   |     period]                       |
|                                   | -   if the date of service falls  |
|                                   |     within the dates of a         |
|                                   |     relevant PARS record, but the |
|                                   |     item number claimed is        |
|                                   |     different to what is stated   |
|                                   |     in the record on PARS, reject |
|                                   |     the service with RSN code S   |
|                                   |     283 [Item/condition claimed   |
|                                   |     not covered by approval]      |
|                                   | -   if the date of service falls  |
|                                   |     within the dates of a         |
|                                   |     relevant PARS record, but the |
|                                   |     Visit Count of the PARS       |
|                                   |     record has already been       |
|                                   |     reached, reject the service   |
|                                   |     with RSN code S 281 [Number   |
|                                   |     of services claimed exceeds   |
|                                   |     approved/allowed number]      |
|                                   | -   if the servicing provider     |
|                                   |     number making the claim is    |
|                                   |     different to the servicing    |
|                                   |     provider number recorded on   |
|                                   |     PARS, check VPQI address. If  |
|                                   |     the provider is at the same   |
|                                   |     address location, force-match |
|                                   |     the service into the PARS     |
|                                   |     record [PI 3X]. If at         |
|                                   |     different locations, reject   |
|                                   |     the service/s with reason     |
|                                   |     code 285 [Prior approval not  |
|                                   |     sought for the                |
|                                   |     provider/practice location].  |
|                                   |                                   |
|                                   | Note: there may be more than one  |
|                                   | record available for the          |
|                                   | service/s involved and, in these  |
|                                   | cases, all likely records should  |
|                                   | be considered.                    |
+-----------------------------------+-----------------------------------+
| 3                                 | 4-PAR DATE + Read more ...        |
|                                   |                                   |
|                                   | The message 4-PAR DATE is         |
|                                   | displayed when the date of        |
|                                   | service for the item/s being      |
|                                   | processed is outside of the       |
|                                   | effective and/or end date of the  |
|                                   | prior approval record. In these   |
|                                   | situations, the following actions |
|                                   | are taken:                        |
|                                   |                                   |
|                                   | -   check the COMMENTS field and  |
|                                   |     business rules for any        |
|                                   |     further information regarding |
|                                   |     the payment of additional     |
|                                   |     services                      |
|                                   | -   if the service is to be paid, |
|                                   |     press the PF5-ACCEPT key to   |
|                                   |     return to the Pend Resolution |
|                                   |     (VAPC) screen, and            |
|                                   | -   key PI 3X (service prior      |
|                                   |     approved - manual match) to   |
|                                   |     pay the service if a prior    |
|                                   |     approval record is involved   |
|                                   | -   if the service/s involved is  |
|                                   |     to be rejected, press the     |
|                                   |     PF2-RETURN key to return to   |
|                                   |     the Pend Resolution (VAPC)    |
|                                   |     screen, and                   |
|                                   | -   if some of the services are   |
|                                   |     outside of the prior          |
|                                   |     approval/referral period,     |
|                                   |     reject each service that      |
|                                   |     falls outside of the prior    |
|                                   |     approval/referral period with |
|                                   |     RSN 282 (date of service      |
|                                   |     falls outside of              |
|                                   |     approval/referral period)     |
|                                   |                                   |
|                                   | Note: before rejecting services   |
|                                   | that fall outside of a prior      |
|                                   | approval period, always check to  |
|                                   | ensure that you are dealing with  |
|                                   | the correct PARS record.          |
+-----------------------------------+-----------------------------------+
| 4                                 | 4-PAR PROV + Read more ...        |
|                                   |                                   |
|                                   | The message 4-PAR PROV is         |
|                                   | displayed when the provider       |
|                                   | claiming the service/s is not the |
|                                   | provider specified in the prior   |
|                                   | approval record. In these         |
|                                   | situations, the following actions |
|                                   | are taken:                        |
|                                   |                                   |
|                                   | check the COMMENTS field and      |
|                                   | business rules for any further    |
|                                   | information that may allow        |
|                                   | payment for a different provider  |
|                                   |                                   |
|                                   | -   if the service is to be paid, |
|                                   |     press the PF5-ACCEPT key to   |
|                                   |     return to the Pend Resolution |
|                                   |     (VAPC) screen and key PI 3X   |
|                                   |     (service prior approved -     |
|                                   |     manual match) to pay the      |
|                                   |     service                       |
|                                   | -   if the service/s is to be     |
|                                   |     rejected, press the           |
|                                   |     PF2-RETURN key to return to   |
|                                   |     the Pend Resolution (VAPC)    |
|                                   |     screen and reject the         |
|                                   |     service/s claimed with RSN S  |
|                                   |     285 (provider has not sought  |
|                                   |     approval to perform this      |
|                                   |     service)                      |
|                                   |                                   |
|                                   | Note: when the provider rendering |
|                                   | the service is not the approved   |
|                                   | provider shown in the prior       |
|                                   | approval record, the service/s    |
|                                   | can be paid providing the         |
|                                   | provider claiming the service/s   |
|                                   | operates from the same practice   |
|                                   | location as the provider who was  |
|                                   | granted prior approval.           |
+-----------------------------------+-----------------------------------+
| 5                                 | 4-PAR FEE + Read more ...         |
|                                   |                                   |
|                                   | The message 4-PAR FEE is          |
|                                   | displayed when the item has a     |
|                                   | provider negotiated benefit type  |
|                                   | and the amount being claimed is   |
|                                   | higher than the agreed amount     |
|                                   | shown in the MAX FEE field on the |
|                                   | prior approval record. In these   |
|                                   | situations, the following actions |
|                                   | are taken:                        |
|                                   |                                   |
|                                   | -   check the COMMENTS field for  |
|                                   |     any additional information    |
|                                   |     regarding the amount that can |
|                                   |     be paid for the service/s     |
|                                   |     claimed                       |
|                                   | -   if an agreed amount is        |
|                                   |     recorded in the MAX FEE field |
|                                   |     on PARS, press the PF5-ACCEPT |
|                                   |     key to return to the Pend     |
|                                   |     Resolution (VAPC) screen and: |
|                                   | -   key PI 3X (service prior      |
|                                   |     approved - manual match) to   |
|                                   |     pay the service               |
|                                   | -   key RSN S 288 (fee paid in    |
|                                   |     accordance with departmental  |
|                                   |     agreed rates) in the RSN      |
|                                   |     field against the item        |
|                                   |     involved                      |
|                                   | -   the system uses the agreed    |
|                                   |     amount recorded in the MAX    |
|                                   |     FEE field as the fee for the  |
|                                   |     item when calculating the     |
|                                   |     amount to be paid. The amount |
|                                   |     to be paid cannot exceed the  |
|                                   |     maximum fee recorded on QITI  |
|                                   | -   if $0.00 is recorded in the   |
|                                   |     MAX FEE field on PARS, press  |
|                                   |     the PF2-RETURN key to return  |
|                                   |     to the Pend Resolution (VAPC) |
|                                   |     screen and:                   |
|                                   | -   key PI 3X (service prior      |
|                                   |     approved - manual match) to   |
|                                   |     pay the service               |
|                                   | -   key RSN 288 (fee paid in      |
|                                   |     accordance with departmental  |
|                                   |     agreed rates) in the RSN      |
|                                   |     field against the item        |
|                                   |     involved                      |
|                                   | -   the system uses the maximum   |
|                                   |     fee recorded on the QITI as   |
|                                   |     the fee for the item when     |
|                                   |     calculating the amount to be  |
|                                   |     paid. Payment will be limited |
|                                   |     to the amount claimed and     |
|                                   |     cannot exceed the maximum fee |
|                                   |     recorded on the QITI          |
|                                   |                                   |
|                                   | Note:                             |
|                                   |                                   |
|                                   | -   if the agreed amount recorded |
|                                   |     in the MAX FEE field on PARS  |
|                                   |     is higher than the maximum    |
|                                   |     fee recorded for the item on  |
|                                   |     the QITI, more than one item  |
|                                   |     line will need to be used to  |
|                                   |     pay the agreed amount         |
|                                   | -   if the amount claimed by the  |
|                                   |     provider for a service is     |
|                                   |     equal to or less than the     |
|                                   |     agreed amount recorded in the |
|                                   |     MAX FEE field on PARS, the    |
|                                   |     service will be automatically |
|                                   |     matched (processing indicator |
|                                   |     3Y displayed in the PI field) |
|                                   |     to the prior approval record  |
|                                   |     involved                      |
+-----------------------------------+-----------------------------------+
| 6                                 | 4-PAR ITEM + Read more ...        |
|                                   |                                   |
|                                   | The message 4-PAR ITEM is         |
|                                   | displayed when the item being     |
|                                   | claimed is not listed as one of   |
|                                   | the items in the prior approval   |
|                                   | record. In these situations, the  |
|                                   | following actions are taken:      |
|                                   |                                   |
|                                   | -   check the COMMENTS field and  |
|                                   |     business rules for any        |
|                                   |     further information regarding |
|                                   |     the payment of the service/s  |
|                                   |     claimed                       |
|                                   | -   if the service is to be paid, |
|                                   |     press the PF5-ACCEPT key to   |
|                                   |     return to the Pend Resolution |
|                                   |     (VAPC) screen and key         |
|                                   |     processing indicator 3X       |
|                                   |     (service prior approved -     |
|                                   |     manual match) to pay the      |
|                                   |     service                       |
|                                   | -   if the service/s is to be     |
|                                   |     rejected, press the           |
|                                   |     PF2-RETURN key to return to   |
|                                   |     the Pend Resolution (VAPC)    |
|                                   |     screen and reject the         |
|                                   |     service/s claimed with RSN S  |
|                                   |     283 (item claimed not covered |
|                                   |     by approval)                  |
+-----------------------------------+-----------------------------------+
| 7                                 | 4-PAR VSTS + Read more ...        |
|                                   |                                   |
|                                   | The message 4-PAR VSTS is         |
|                                   | displayed when the service being  |
|                                   | processed exceeds the approved    |
|                                   | number of visits on the prior     |
|                                   | approval record. In these         |
|                                   | situations, the following actions |
|                                   | are taken:                        |
|                                   |                                   |
|                                   | -   check the COMMENTS field and  |
|                                   |     business rules for any        |
|                                   |     further information regarding |
|                                   |     the payment of additional     |
|                                   |     services                      |
|                                   | -   if the service is to be paid, |
|                                   |     press the PF5-ACCEPT key to   |
|                                   |     return to the Pend Resolution |
|                                   |     (VAPC) screen and key PI 3X   |
|                                   |     (service prior approved -     |
|                                   |     manual match) to pay the      |
|                                   |     service                       |
|                                   | -   if the service/s is to be     |
|                                   |     rejected, press the           |
|                                   |     PF2-RETURN key to return to   |
|                                   |     the Pend Resolution (VAPC)    |
|                                   |     screen and reject the         |
|                                   |     service/s claimed with RSN S  |
|                                   |     281 (number of services       |
|                                   |     claimed exceeds approved      |
|                                   |     number)                       |
|                                   |                                   |
|                                   | Note: payment for additional      |
|                                   | visits should not be made unless  |
|                                   | this is supported by PARS         |
|                                   | records/comments or it has been   |
|                                   | established that the visit count  |
|                                   | shown on the patient's prior      |
|                                   | approval VEPI record is           |
|                                   | incorrect.                        |
