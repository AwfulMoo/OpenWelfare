|                                   | -   Auto or bulk updates to       |
|                                   |     Shares/Managed Investments    |

|                                   |     changes                       |
|                                   | -   Family Tax Benefit (FTB) Part |

|                                   |     one period (IOP/LOP) which    |
|                                   |     impact the customer's current |

| Administrative action suspensions | Administration action suspensions |
| or cancellations                  | or cancellations such as a        |
|                                   | cancellation 'WUK' (Whereabouts   |
|                                   | Unknown) are not attributable to  |
|                                   | a review unless they were         |
|                                   | directly triggered by the review. |
|                                   |                                   |
|                                   | Auto processed 'WUK'              |
|                                   | cancellations are only            |
|                                   | attributable to the review if the |
|                                   | cancellation was triggered by the |
|                                   | automated Return to Sender        |
|                                   | processâ€š and the returned letter  |

|                                   | Compliance Risk Branch letter.    |
|                                   | The Document Tools facility links |
|                                   | to scanned images.                |

| cancellations                     | never be attributed to a review   |
|                                   | if the cancellation is triggered  |
|                                   | by the customer's age and not     |
|                                   | from information identified as    |
|                                   | part of the review.               |
|                                   |                                   |
|                                   | AGD (Doesn't meet age             |
|                                   | requirements - Auto)              |
|                                   | cancellations which occur by the  |
|                                   | changing/adding of a course end   |
|                                   | date are only attributable to the |
|                                   | review if they were directly      |
|                                   | triggered by the review.          |
|                                   |                                   |
|                                   | System generated AGD              |
|                                   | cancellations are not             |
|                                   | attributable as they are          |
|                                   | triggered by the course end date  |
|                                   | being reached which is not        |
|                                   | related to the review.            |
+-----------------------------------+-----------------------------------+
| Attributing actions to a partner  | The same attribution rules apply  |
| review                            | to a partner review.              |
|                                   |                                   |
|                                   | Consequential activities will not |
|                                   | be automatically attributed to    |
|                                   | the partner review. The Service   |
|                                   | Officer must manually attribute   |
|                                   | the activity if they determine    |
|                                   | that the consequential action     |
|                                   | relates to the review.            |
|                                   |                                   |
|                                   | After the partner review has      |
|                                   | started, a contact letter or      |
|                                   | advice can be issued on the       |
|                                   | primary review which also relates |
|                                   | to the partner's circumstances.   |
|                                   |                                   |
|                                   | Any Review Contact Period present |
|                                   | on the primary review can be used |
|                                   | to assess whether an action was   |
|                                   | indirectly triggered on the       |
|                                   | partner's record.                 |
|                                   |                                   |
|                                   | Attributing an indirectly         |
|                                   | triggered action to a partner     |
|                                   | review based on a contact issued  |
|                                   | as part of the primary customer's |
|                                   | review must be recorded on a DOC  |
|                                   | on the partner's record.          |
+-----------------------------------+-----------------------------------+
| Attributing upward variations     | Rate increases should be          |
|                                   | attributed to a review if the     |
|                                   | review directly triggered that    |
|                                   | action.                           |
|                                   |                                   |
|                                   | Compliance/Service Officer should |
|                                   | not avoid attributing an upward   |
|                                   | variation to a customer's rate    |
|                                   | because it will impact any        |
|                                   | downward variation savings that   |
|                                   | the review would realise.         |
|                                   |                                   |
|                                   | The Compliance/Service Officer    |
|                                   | does not need to investigate or   |
|                                   | attribute a rate increase which   |
|                                   | has been indirectly triggered by  |
|                                   | the review unless it is a         |
|                                   | restoration, grant or re-grant    |
|                                   | action. It is unlikely that the   |
|                                   | review triggered the customer to  |
|                                   | obtain a higher rate of payment   |
|                                   | as this action would most likely  |
|                                   | have occurred regardless of the   |
|                                   | review.                           |
+-----------------------------------+-----------------------------------+
| A voluntary compliance            | The cancellation should be        |
| cancellation is attributable to   | attributed to the first review    |
| more than one compliance or fraud | started.                          |
| review                            |                                   |
|                                   | The only exception to this is     |
|                                   | where the cancellation reason is  |
|                                   | not appropriate to attribute to a |

|                                   | the review triggered the action   |
|                                   | still applies.                    |

|                                   |                                   |
|                                   | -   the customer's DTL date is    |

|                                   |     the same date as the start of |
|                                   |     the review and the customer   |

|                                   |     the day the review started,   |

|                                   |     lodgement rules apply, and    |

|                                   |     Contact Period                |
|                                   |                                   |
|                                   | Note: cancellations for DNL (Did  |
|                                   | Not Lodge) will be excluded from  |

|                                   | Customer is not a fortnightly     |
|                                   | lodger                            |

|                                   | -   the customer's DTL date is    |

|                                   |     received the review letter    |

|                                   | -   the customer has not lodged   |
|                                   |     since the review started      |
|                                   |                                   |
|                                   | Note: the cancellation action is  |
|                                   | likely to fall outside the Review |
|                                   | Contact Period however as long as |
|                                   | the above rule is satisfied it is |
|                                   | deemed that it is acceptable to   |
|                                   | claim this cancellation action.   |

|                                   | permanent full time employment,   |

|                                   | fortnights at zero rate. The      |
|                                   | customer must also be advised     |

|                                   | credits which may allow them to   |
|                                   | receive further payments.         |
|                                   |                                   |
|                                   | These conversations must be       |

|                                   |                                   |

|                                   | processed by Business Integrity   |

|                                   | Period                            |
|                                   |                                   |
|                                   | -   A Compensation cancellation   |

|                                   |     triggered by the review, or   |
|                                   |     indirectly triggered within a |

|                                   |     the reason for the review was |

| General rule to determine if an   | The general rule for determining  |
| action is attributable to the     | whether an action can be          |
| review                            | attributed to a review is where   |
|                                   | the review:                       |
|                                   |                                   |
|                                   | -   directly triggered the        |
|                                   |     action, or                    |
|                                   | -   indirectly triggered the      |
|                                   |     action and the trigger for    |
|                                   |     that action occurred within a |
|                                   |     Review Contact Period and is  |
|                                   |     not directly attributable to  |
|                                   |     another started review or     |
|                                   |     review process                |
|                                   |                                   |
|                                   | The action must also meet         |
|                                   | additional rules outlined in the  |
|                                   | Resources page as there are some  |
|                                   | circumstances where an action,    |
|                                   | such as a suspension or           |
|                                   | cancellation should not be        |
|                                   | included as part of a review      |
|                                   | result.                           |
|                                   |                                   |
|                                   | Note: restoration, grant or       |
|                                   | re-grant actions should only be   |
|                                   | attributed to a review if the     |
|                                   | preceding suspension or           |
|                                   | cancellation is attributed to the |
|                                   | review.                           |

| General rule to determine if an   | A compliance or fraud review is   |
| action related to a review        | conducted because there is a      |
|                                   | justified, documented risk of     |
|                                   | incorrect payment which is a      |
|                                   | result of fraud,                  |
|                                   | misrepresentation, error or       |
|                                   | omission on the part of the       |
|                                   | customer, rather than a system,   |
|                                   | coding or assessment error.       |
|                                   |                                   |
|                                   | The savings associated with a     |
|                                   | compliance or fraud review is     |
|                                   | based on the savings the review   |
|                                   | has realised with updates to the  |
|                                   | customer's payment which relate   |
|                                   | solely to this risk.              |
|                                   |                                   |
|                                   | The general rule for determining  |
|                                   | whether an action is attributable |
|                                   | to a review is:                   |
|                                   |                                   |
|                                   | -   the action relates to the     |
|                                   |     reason for the review         |
|                                   | -   the action was directly       |
|                                   |     triggered or indirectly       |
|                                   |     triggered by the review       |
|                                   | -   the information which changed |
|                                   |     the customer's rate was not   |
|                                   |     known prior to the start of   |
|                                   |     the review, and               |
|                                   | -   the action directly relates   |
|                                   |     to fraud, misrepresentation,  |
|                                   |     error or omission on the part |
|                                   |     of the customer               |
