Updating income for CSHC

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Notice of Assessment (NOA) and    |
|                                   | other documents supplied + Read   |
|                                   | more ...                          |
|                                   |                                   |
|                                   | For verification of income,       |
|                                   | copies are acceptable and         |
|                                   | original documents do not need to |
|                                   | be supplied.                      |
|                                   |                                   |
|                                   | When customers provide documents  |
|                                   | in person to support their claim, |
|                                   | accept and scan documents.        |
|                                   |                                   |
|                                   | Note: when a customer (or         |
|                                   | partner) has declared in the      |
|                                   | online claim they have a taxable  |
|                                   | income of nil and cannot supply   |
|                                   | previous income tax returns, use  |
|                                   | the information provided to       |
|                                   | determine the claim outcome, as   |
|                                   | these customers have completed an |
|                                   | online declaration.               |
|                                   |                                   |
|                                   | [] These customers may be         |
|                                   | eligible for Age Pension and      |
|                                   | should be encouraged to test      |
|                                   | eligibility online.               |
|                                   |                                   |
|                                   | All customers are subject to an   |
|                                   | annual reviews and Australian     |
|                                   | Taxation Office (ATO) data        |
|                                   | matches. Where customers supply   |
|                                   | verification documents in person  |
|                                   | or via mail, scan documents to    |
|                                   | appropriate record and mark       |
|                                   | relevant tasks as submitted       |
|                                   | through:                          |
|                                   |                                   |
|                                   | -   the Request and Manage        |
|                                   |     Customer Tasks (RMCT) in      |
|                                   |     Customer First                |
|                                   | -   Tasks in Process Direct       |
|                                   |                                   |
|                                   | For further information, see      |
|                                   | Sighting, recording and returning |
|                                   | original documents.               |
|                                   |                                   |
|                                   | Is this update being performed as |
|                                   | part of a new claim?              |
|                                   |                                   |
|                                   | -   Yes, see Processing           |
|                                   |     Commonwealth Seniors Health   |
|                                   |     Card (CSHC) claims, procedure |
|                                   |     ends here                     |
|                                   | -   No, go to Step 2              |
+-----------------------------------+-----------------------------------+
| 2                                 | Senior Health Card (SHC) system + |
|                                   | Read more ...                     |
|                                   |                                   |
|                                   | In Customer First, prior to       |
|                                   | making an update to a             |
|                                   | Commonwealth Seniors Health Card  |
|                                   | (CSHC) record, select the correct |
|                                   | system:                           |
|                                   |                                   |
|                                   | -   in the Sys: field key 'SHC'   |
|                                   | -   press [Enter]                 |
+-----------------------------------+-----------------------------------+
| 3                                 | Senior Health Income Details      |
|                                   | (SHID) screen + Read more ...     |
|                                   |                                   |
|                                   | While in the customer's record,   |
|                                   | go to the SHID screen, either:    |
|                                   |                                   |
|                                   | via the SHC Task Selection (SHTS) |
|                                   | screen:                           |
|                                   |                                   |
|                                   | -   key 'SHTS' in the NXT: field  |
|                                   | -   press [Enter]                 |
|                                   | -   select the appropriate        |
|                                   |     option/s for SHC Income       |
|                                   |     Details and Assessment        |
|                                   |     Results                       |
|                                   | -   press [Enter], or             |
|                                   |                                   |
|                                   | key 'SHID' in the NXT: field      |
|                                   |                                   |
|                                   | -   press [Enter]                 |
+-----------------------------------+-----------------------------------+
| 4                                 | Updating the SHID screen + Read   |
|                                   | more ...                          |
|                                   |                                   |
|                                   | On the SHID screen update the     |
|                                   | following fields:                 |
|                                   |                                   |
|                                   | Date of Event:                    |
|                                   |                                   |
|                                   | -   use the date the NOA was      |
|                                   |     issued, or                    |
|                                   | -   for customers who are not     |
|                                   |     required to lodge an income   |
|                                   |     tax return, the date they     |
|                                   |     advised the ATO of this, or   |
|                                   | -   for customers who are         |
|                                   |     providing a current year      |
|                                   |     estimate use the date of      |
|                                   |     notification                  |
|                                   |                                   |
|                                   | Reference Tax Year: put in the    |
|                                   | financial year of the NOA or      |
|                                   | relevant tax year for a           |
|                                   | non-lodger. Note: this cannot be  |
|                                   | greater than 2 financial years in |
|                                   | the past from the current         |
|                                   | financial year. The most recent   |
|                                   | NOA should be used. Members of a  |
|                                   | couple must provide income        |
|                                   | details for the same reference    |
|                                   | tax year. Use the year that the   |
|                                   | financial year ended, for         |
|                                   | example, for 2018/19 code 2019    |
|                                   |                                   |
|                                   | Income Type (code 1 of the        |
|                                   | following):                       |
|                                   |                                   |
|                                   | -   TNA for an NOA and for        |
|                                   |     customers who have been       |
|                                   |     granted an ATO lodgement      |
|                                   |     extension                     |
|                                   | -   EST for an estimate, or       |
|                                   | -   NRL where a customer does not |
|                                   |     provide an NOA as their       |
|                                   |     income is below the tax       |
|                                   |     lodgement threshold or        |
|                                   |     because of an ATO tax offset. |
|                                   |     NRL must not be coded if the  |
|                                   |     customer has income over the  |
|                                   |     tax lodgement threshold       |
|                                   |                                   |
|                                   | Taxable Income $ code 1 of the    |
|                                   | following:                        |
|                                   |                                   |
|                                   | -   Enter the amount from the NOA |
|                                   |     or for customers who have     |
|                                   |     been granted an ATO lodgement |
|                                   |     extension, an estimate of     |
|                                   |     their taxable income. Note:   |
|                                   |     if a person has accessed      |
|                                   |     superannuation under the      |
|                                   |     First Home Super Saver (FHSS) |
|                                   |     Scheme these withdrawals      |
|                                   |     require special               |
|                                   |     consideration.                |
|                                   | -   The current year estimate     |
|                                   | -   Customer's income below the   |
|                                   |     tax lodgement threshold.      |
|                                   |     Note: customers who have      |
|                                   |     provided an estimate for the  |
|                                   |     current financial year that   |
|                                   |     has been accepted must also   |
|                                   |     have their verified income    |
|                                   |     that is over the income limit |
|                                   |     from either of the last 2     |
|                                   |     financial years recorded on   |
|                                   |     the SHID screen. After the    |
|                                   |     estimate has been recorded,   |
|                                   |     record the NOA using the date |
|                                   |     the NOA was issued            |
|                                   |                                   |
|                                   | The following fields are          |
|                                   | optional. Only complete if        |
|                                   | applicable to the customer:       |
|                                   |                                   |
|                                   | -   Foreign Income $: Enter the   |
|                                   |     Australian dollar value from  |
|                                   |     section IT4 on tax return or  |
|                                   |     other foreign income          |
|                                   |     documents if not required to  |
|                                   |     lodge                         |
|                                   | -   Net Investment Loss $: Enter  |
|                                   |     details from IT5/6 on income  |
|                                   |     tax return or other financial |
|                                   |     documents if not required to  |
|                                   |     lodge                         |
|                                   | -   Employer Provided Benefits $: |
|                                   |     (also known as fringe         |
|                                   |     benefits), Enter the details  |
|                                   |     from the payment summary or   |
|                                   |     IT1 on tax return, minus the  |
|                                   |     first $1,000                  |
|                                   | -   Reportable Super              |
|                                   |     Contributions $: Enter        |
|                                   |     details from payment summary  |
|                                   |     or IT2 on income tax return   |
|                                   | -   Personal deductable           |
|                                   |     superannuation contribution   |
|                                   |     $: Income tax Return sections |
|                                   |     D12 or relevant documents.    |
|                                   |     For example, statement from   |
|                                   |     superannuation provider       |
|                                   |                                   |
|                                   | When all updates are completed,   |
|                                   | press [Enter].                    |
|                                   |                                   |
|                                   | Income stream details             |
|                                   |                                   |
|                                   | For coding instructions to update |
|                                   | an existing account- based income |
|                                   | stream or add a new account-based |
|                                   | income stream, see Adding or      |
|                                   | updating an account-based income  |
|                                   | stream.                           |
|                                   |                                   |
|                                   | Note: the SHID screen includes a  |
|                                   | Deemed Income Amount field which  |
|                                   | is a protected field that cannot  |
|                                   | be updated manually. If a         |
|                                   | customer or partner has           |
|                                   | account-based income streams      |
|                                   | recorded that are subject to      |
|                                   | deeming the claim activity will   |
|                                   | need to be taken through          |
|                                   | assessment to ensure the deemed   |
|                                   | income figure (if applicable) on  |
|                                   | the SHID screen is correct. To do |
|                                   | this Service Officers can         |
|                                   | navigate to the Assessment        |
|                                   | Results (AR) screen in Customer   |
|                                   | First, and the Entitlements (ELD) |
|                                   | in Process Direct, without        |
|                                   | finalising the activity and then  |
|                                   | return to the SHID screen to view |
|                                   | the updated Deemed Income Amount. |
+-----------------------------------+-----------------------------------+
| 5                                 | Finalise Activity + Read more ... |
|                                   |                                   |
|                                   | []In Customer First:              |
|                                   |                                   |
|                                   | Go to the Assessment Results AR   |
|                                   | screen:                           |
|                                   |                                   |
|                                   | -   press [Enter]                 |
|                                   | -   record details on a DOC       |
|                                   |                                   |
|                                   | In Process Direct:                |
|                                   |                                   |
|                                   | -   Select Assess                 |
|                                   | -   Navigate to ELD               |
|                                   | -   Select Finalise               |
|                                   | -   Claim will auto-DOC, update   |
|                                   |     any information if required   |
