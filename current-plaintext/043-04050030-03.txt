|                                   | disregarded include:              |
|                                   |                                   |
|                                   | -   amounts already been included |
|                                   |     as attribution income         |
|                                   | -   amounts from prior year       |
|                                   |     profits                       |
|                                   |                                   |
|                                   | The gifting provisions in         |
|                                   | relation to distributions paid to |
|                                   | a non-attributable stakeholder    |
|                                   | will be disregarded if:           |
|                                   |                                   |
|                                   | -   the customer can show they    |
|                                   |     were not in receipt of an     |
|                                   |     income support payment and    |
|                                   |     could not have reasonably     |
|                                   |     known they would require      |
|                                   |     income support at the time    |
|                                   |     the distribution or dividend  |
|                                   |     was paid, or                  |
|                                   | -   the non-attributable          |
|                                   |     stakeholder has been assessed |
|                                   |     as a Genuine Investor and the |
|                                   |     distribution paid to the      |
|                                   |     Genuine Investor is           |
|                                   |     commensurate with their       |
|                                   |     investment in the entity, or  |
|                                   | -   the distribution has been     |
|                                   |     paid in lieu of wages         |
|                                   |                                   |
|                                   | For the non-controller the        |
|                                   | distribution will be assessed as  |
|                                   | income for 52 weeks, unless it is |
|                                   | determined the distribution was   |

|                                   | lieu of wages.                    |

|                                   | involvement with the entity, any  |

| 2                                 | Recording Trust/Company           |
|                                   | distribution amounts + Read more  |
|                                   | ...                               |
|                                   |                                   |
|                                   | Go to the Trust/Company           |
|                                   | Distribution (TRD) screen and     |
|                                   | code the following fields:        |
|                                   |                                   |
|                                   | Event Date: record the date of    |
|                                   | the resolution to distribute      |
|                                   |                                   |
|                                   | -   if the resolution to          |
|                                   |     distribute states a           |
|                                   |     percentage and not a specific |
|                                   |     amount, the beneficiary's     |
|                                   |     legal entitlement to a        |
|                                   |     distribution only commences   |
|                                   |     from the date the amount is   |
|                                   |     known. This is usually when   |
|                                   |     the tax return is completed.  |
|                                   |     The date the amount is known  |
|                                   |     is the date of event          |
|                                   | -   if the resolution declares to |
|                                   |     distribute a specific amount  |
|                                   |     for a beneficiary but the     |
|                                   |     trust tax return has not been |
|                                   |     lodged at the time of the     |
|                                   |     resolution, the date of event |
|                                   |     is the date the resolution    |
|                                   |     was made, not the date the    |
|                                   |     tax return was lodged         |
|                                   |                                   |
|                                   | Amount: record the amount of the  |
|                                   | distribution in whole dollars     |
|                                   | Note: if distributions have not   |
|                                   | ceased, but no distribution has   |
|                                   | been paid for a particular        |
|                                   | financial year, $0 should be      |
|                                   | recorded in order to ensure a     |
|                                   | review is triggered correctly for |
|                                   | the following financial year's    |
|                                   | returns and financial statements  |
|                                   |                                   |
|                                   | CRN: Code the Customer Reference  |
|                                   | Number (CRN). The distribution    |
|                                   | amount will pass through to that  |
|                                   | customer's record and display on  |
|                                   | the Trust/Company Income/ Assets  |
|                                   | Details (TRCIAS) screen in the    |
|                                   | customers record. If the customer |
|                                   | is also a controller of the       |
|                                   | trust/company only the            |
|                                   | distribution amount in excess of  |
|                                   | their attributed income will be   |
|                                   | treated as distribution income    |
|                                   | Note: where the distribution is   |
|                                   | made to a child and that child    |
|                                   | has a CRN because their parent    |
|                                   | may be receiving family           |
|                                   | assistance, do not record the     |
|                                   | child's CRN                       |
|                                   |                                   |
|                                   | Gen INV: or Deprivation:          |
|                                   |                                   |
|                                   | -   If the distribution was made  |
|                                   |     to a genuine investor, code   |
|                                   |     'GNI'                         |
|                                   | -   If the distribution was made  |
|                                   |     during the 1999/2000 year,    |
|                                   |     code 'CON'                    |
|                                   | -   If either 'GNI' or 'CON' have |
|                                   |     been recorded in the Gen INV: |
|                                   |     or Deprivation: field this    |
|                                   |     amount will not be treated as |
|                                   |     being deprivation/gift of the |
|                                   |     controllers                   |
|                                   |                                   |
|                                   | Distrb Ceased: If the             |
|                                   | distribution amount is set to     |
|                                   | zero and the customer has         |
|                                   | received the final distribution,  |
|                                   | enter 'Y'. This will help when    |
|                                   | deciding which trust/company      |
|                                   | records can be archived and which |
|                                   | should be selected for a program  |
|                                   | review. It is essential to leave  |
|                                   | this field blank if the customer  |
|                                   | may receive a distribution in the |
|                                   | future                            |
|                                   |                                   |
|                                   | Multiple Distributions: If a      |
|                                   | Trustee resolves to make multiple |
|                                   | distributions to a customer       |
|                                   | within a financial year, each     |
|                                   | distribution will need to be      |
|                                   | assessed as income for a period   |
|                                   | of 52 weeks from the date of the  |
|                                   | resolution to distribute. For     |
|                                   | example, if the Trustee resolves  |
|                                   | to make a distribution of $5,000  |
|                                   | on 1 August 2015, a TRD screen    |
|                                   | will be recorded with a date of   |
|                                   | event of 1 August 2015 and a      |
|                                   | distribution amount of $5,000. If |
|                                   | the Trustee resolves to make a    |
|                                   | further distribution of $2,000 on |
|                                   | 1 March 2016, a new TRD screen    |
|                                   | will need to be recorded with a   |
|                                   | date of event of 1 March 2016 and |
|                                   | a distribution amount of $7,000   |
|                                   | ($5,000 plus $2,000). If no       |
|                                   | further distributions are made to |
|                                   | the customer, the TRD screen      |
|                                   | needs to be recorded with a       |
|                                   | distribution of $2,000 ($7,000 -  |
|                                   | $5,000) from 30 July 2016 (for    |
|                                   | example 52 weeks after the        |
|                                   | resolution to distribute) and a   |
|                                   | TRD screen with a nil             |
|                                   | distribution and a date of event  |
|                                   | of 28 February 2017. A review     |
|                                   | should be set to follow up any    |
|                                   | further distributions and update  |
|                                   | the TRD screen as required, 52    |
|                                   | weeks after each distribution     |
|                                   |                                   |
|                                   | Code a review on the Review       |
|                                   | Registration (RVR) screen in the  |
|                                   | trust record:                     |
|                                   |                                   |
|                                   | -   Serv Rsn: TRF                 |
|                                   | -   Rev Rsn: TDI                  |
|                                   | -   Due Date: this will be 52     |
|                                   |     weeks from the date of each   |
|                                   |     distribution                  |
|                                   | -   Text: include details of      |
|                                   |     updates that need to be made  |
|                                   |     to the trust record, such as  |
|                                   |     follow up any further         |
|                                   |     distributions                 |
|                                   |                                   |
|                                   | This process will normally form   |
|                                   | part of coding of all an entity's |
|                                   | income, assets and attribution    |
|                                   | details. As such, ensure          |
|                                   | distribution details are included |
|                                   | when completing a DOC outlining   |
|                                   | the actions taken.                |
|                                   |                                   |
|                                   | The Trust/Company Distribution    |
|                                   | Summary Screen (TRDS) will        |
|                                   | display:                          |
|                                   |                                   |
|                                   | -   the event date                |
|                                   | -   the total distributions to    |
|                                   |     non-genuine investors or      |
|                                   |     non-concession amounts        |
|                                   | -   the total distributions to    |
|                                   |     genuine investors or          |
|                                   |     concession amounts            |

|                                   |     claims)                       |
|                                   |                                   |
|                                   | No, go to Item 5                  |

| 5                                 | Trust has been vested and a       |
|                                   | beneficiary only received a       |
|                                   | distribution within the last      |
|                                   | twelve months + Read more ...     |
|                                   |                                   |
|                                   | Any income distributions to       |
|                                   | customers, who are only           |
|                                   | beneficiaries of the trust, will  |
|                                   | need to be maintained as income   |
|                                   | for a period of 52 weeks from the |
|                                   | date the trustee resolved to make |
|                                   | the distribution. This may mean   |
|                                   | the trust cannot be coded as      |
|                                   | being vested until the            |
|                                   | distributions cease to be         |

|                                   | The trust income, assets,         |
|                                   | liabilities and attribution       |
|                                   | should all be set to nil but any  |
|                                   | distributions to beneficiaries    |
|                                   | only will be coded/remain on the  |
|                                   | TRD screen until they cease to be |
|                                   | assessable.                       |
|                                   |                                   |
|                                   | Code a review on the RVR in the   |
|                                   | trust record:                     |
|                                   |                                   |
|                                   | -   ServRsn: TRF                  |
|                                   | -   RevRsn: TDI                   |
|                                   | -   Due Date: this will be 52     |
|                                   |     weeks from the date of the    |
|                                   |     resolution to distribute      |
|                                   | -   Text: include details of      |
|                                   |     updates that need to be made  |
|                                   |     to the trust record, such as  |
|                                   |     reducing distributions to nil |
|                                   |     and coding the ‘Wound Up’     |
|                                   |     indicator on TRMD             |

|                                   | There are special provisions for  |
|                                   | customers experiencing Family     |
|                                   | Domestic Violence (FDV) where     |
|                                   | previous trust or company         |
|                                   | distributions affect their rate   |

|                                   | (DSS) may grant an exemption for  |

|                                   | Helpdesk for assessment.          |
|                                   |                                   |

|                                   |     where the customer is not an  |

|                                   |     the entity.                   |
|                                   |     If the CAO has assessed the   |
|                                   |     customer as an attributable   |
|                                   |     stakeholder, procedure ends   |
|                                   |     here                          |
|                                   | -   in cases where the FDV        |
|                                   |     involves a partner, the       |
|                                   |     customer and partner must be  |
|                                   |     separated for the exemption   |
|                                   |     to apply.                     |
|                                   |     If the customer remains       |
|                                   |     partnered, procedure ends     |
|                                   |     here                          |
|                                   |                                   |

|                                   | required:                         |
|                                   |                                   |
|                                   | -   consent from the customer to  |
|                                   |     be referred to a social       |
|                                   |     worker to obtain FDV          |
|                                   |     verification                  |
|                                   | -   explicit authority from the   |
|                                   |     customer for DSS to access    |
|                                   |     their record, go to Step 2    |

|                                   |                                   |
|                                   | When the required information is  |
|                                   | gathered, go to Step 4            |
