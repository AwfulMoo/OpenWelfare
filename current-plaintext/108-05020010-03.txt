| 3                                 | Salary continuance or income      |
|                                   | protection payments from          |
|                                   | superannuation + Read more ...    |
|                                   |                                   |
|                                   | The following benefits paid from  |
|                                   | a superannuation fund are not     |
|                                   | treated as compensation:          |
|                                   |                                   |
|                                   | -   salary continuance benefits   |
|                                   | -   income protection benefits    |
|                                   | -   disability income benefits    |
|                                   |                                   |
|                                   | If unsure, check with the Level 2 |
|                                   | Policy helpdesk.                  |
|                                   |                                   |
|                                   | Record these payments on the OINS |
|                                   | screen. They include:             |
|                                   |                                   |
|                                   | regular ongoing payments          |
|                                   |                                   |
|                                   | lump sum arrears, paid for a      |
|                                   | period before the start of        |
|                                   | regular ongoing payments, and     |
|                                   |                                   |
|                                   | -   assessed from the date of     |
|                                   |     receipt over a period of not  |
|                                   |     more than 52 weeks            |
|                                   | -   the assessment period is      |
|                                   |     determined by the past period |
|                                   |     for which the lump sum was    |
|                                   |     paid. For example, a lump sum |
|                                   |     payment of income protection  |
|                                   |     arrears for a 6 month period  |
|                                   |     is assessed for 26 weeks from |
|                                   |     the date of receipt of the    |
|                                   |     payment                       |
|                                   |                                   |
|                                   | The assessable amount is the      |
|                                   | gross income less any adjustments |
|                                   | for the offsets applied by the    |
|                                   | insurance company.                |
|                                   |                                   |
|                                   | Before starting the update, check |
|                                   | the Pension/ Annuities Summary    |
|                                   | (SUPS) screen. In the past, these |
|                                   | payments were recorded on SUPS    |
|                                   | with product type DIS.            |
|                                   |                                   |
|                                   | Does a SUPS record already exist? |
|                                   |                                   |
|                                   | Yes:                              |
|                                   |                                   |
|                                   | -   Cancel the record on SUPS and |
|                                   |     move it to OIN when the       |
|                                   |     customer provides an update   |
|                                   |     to the payment                |
|                                   | -   Select the product and        |
|                                   |     Continue to open the          |
|                                   |     Pension/Annuities             |
|                                   |     Identification(SUPI) screen   |
|                                   | -   In the Product Cancellation   |
|                                   |     Reason: field, code 'OTH' and |
|                                   |     in Action: field use 'C'      |
|                                   | -   Continue to open              |
|                                   |     Pension/Annuities             |
|                                   |     Identification (SUPI) screen. |
|                                   | -   The Event Date: field on the  |
|                                   |     Pension/Annuities Detail      |
|                                   |     Variable (SUPV) screen is the |
|                                   |     relevant date of the update   |
|                                   | -   The payment is then moved to  |
|                                   |     OIN in the same activity,     |
|                                   |     using the same event date as  |
|                                   |     the SUPS cancellation         |
|                                   | -   go to Step 5                  |
|                                   |                                   |
|                                   | No, go to Step 5                  |

| 4                                 | Royalties, commissions or         |
|                                   | contract payments + Read more ... |
|                                   |                                   |
|                                   | Royalties received by an          |
|                                   | individual from a community, for  |
|                                   | example, royalties paid to an     |
|                                   | indigenous community, are treated |
|                                   | as employment income if they are  |
|                                   | then distributed to individuals   |
|                                   | in that community. The income is  |
|                                   | coded as annual (ANN) and         |
|                                   | assessed for 52 weeks from the    |
|                                   | date received.                    |
|                                   |                                   |
|                                   | For customers who are commission  |
|                                   | or contract workers, or receive   |
|                                   | royalties and are considered to   |
|                                   | be self-employed:                 |
|                                   |                                   |
|                                   | -   The assessed income is the    |
|                                   |     net income after allowable    |
|                                   |     expenses have been deducted   |
|                                   | -   If the customer receives lump |
|                                   |     sums from a number of         |
|                                   |     contracts with the same       |
|                                   |     company, each instance of     |
|                                   |     income will need to be coded  |
|                                   |     with a unique Description.    |
|                                   |                                   |
|                                   | For commissions, contracting and  |
|                                   | royalties income recorded on OINS |
|                                   | (ROC) the income is assessable    |
|                                   | for 52 weeks and coded with a     |
|                                   | Frequency of ANN.                 |
|                                   |                                   |
|                                   | If the income is a one off and    |
|                                   | received as a lump sum it will be |
|                                   | assessed as income for a period   |
|                                   | of 52 weeks from the date it was  |
|                                   | able to be received. The          |
|                                   | frequency will be ANN.            |
|                                   |                                   |
|                                   | If income is not a one off and    |
|                                   | received as a lump sum:           |
|                                   |                                   |
|                                   | -   for pensions strike an annual |
|                                   |     net rate of income, taking    |
|                                   |     into consideration any other  |
|                                   |     income from the same source,  |
|                                   |     and coded with a Frequency of |
|                                   |     ANN                           |
|                                   | -   for allowances, maintain the  |
|                                   |     net income as an annual (ANN) |
|                                   |     amount                        |
|                                   |                                   |
|                                   | If the customer is eligible for   |
|                                   | the Work Bonus the percentage of  |
|                                   | their income that has been earned |
|                                   | from personal exertion will need  |
|                                   | to be coded. For both employment  |
|                                   | income and self-employment, the   |
|                                   | default Work Bonus Income %:      |
|                                   | percentage is to be coded as      |
|                                   | 100%.                             |
|                                   |                                   |
|                                   | Go to Step 5                      |

| 5                                 | Coding advised income + Read more |
|                                   | ...                               |
|                                   |                                   |
|                                   | For information on the treatment  |
|                                   | of lump sum arrears payments from |
|                                   | income protection and sickness    |
|                                   | accident policies (not paid from  |
|                                   | superannuation), see Resources.   |
|                                   |                                   |
|                                   | If the customer is unable to      |
|                                   | provide the information needed to |
|                                   | complete the OIN screen, go to    |
|                                   | Step 7.                           |
|                                   |                                   |
|                                   | Note: a customer could receive    |
|                                   | more than one payment of the same |
|                                   | type from the same provider. An   |
|                                   | example of how this should be     |
|                                   | recorded is:                      |
|                                   |                                   |
|                                   | -   Code the first amount on the  |
|                                   |     OIN screen (for example,      |
|                                   |     Grant or Scholarship - xxx    |
|                                   |     University - $1,000 - ANN -   |
|                                   |     with correct DOV)             |
|                                   | -   Code the second amount on the |
|                                   |     OIN screen with a slightly    |
|                                   |     different description name    |
|                                   |     (for example, Grant or        |
|                                   |     Scholarship - xxx University  |
|                                   |     2 - $1,000 - ANN - with       |
|                                   |     correct DOV)                  |
|                                   |                                   |
|                                   | This will ensure the system       |
|                                   | correctly assesses both payments. |
|                                   |                                   |
|                                   | Go to OIN and code:               |
|                                   |                                   |
|                                   | Type enter the appropriate type   |
|                                   | Code or 'OTH' for example for     |
|                                   | salary continuance or income      |
|                                   | protection payment from super,    |
|                                   | code 'OTH'                        |
|                                   |                                   |
|                                   | Description: free text to give a  |
|                                   | description of the other income   |
|                                   | (30 character limit). For salary  |
|                                   | continuance or income protection  |
|                                   | payment from super enter the type |
|                                   | of payment and source             |
|                                   |                                   |
|                                   | Reference: enter any reference    |
|                                   | number for the income type/       |
|                                   | source (12 character limit)       |
|                                   |                                   |
|                                   | Event Date: date of event for the |
|                                   | change or start of an assessment  |
|                                   | period                            |
|                                   |                                   |
|                                   | Amount $: amount of other income  |
|                                   |                                   |
|                                   | Freq: the frequency the customer  |
|                                   | receives the amount of other      |
|                                   | income:                           |
|                                   |                                   |
|                                   | -   For lump sum payments or lump |
|                                   |     sum arrears of salary         |
|                                   |     continuance or income         |
|                                   |     protection payments from      |
|                                   |     superannuation which are      |
|                                   |     assessed over the period to   |
|                                   |     which the payment relates,    |
|                                   |     use a frequency appropriate   |
|                                   |     to the period of payment (for |
|                                   |     example, 2WE, MTH)            |
|                                   | -   Due to a known system error,  |
|                                   |     avoid using frequency ‘One’.  |
|                                   |     Use other frequencies as long |
|                                   |     as the amount and frequency   |
|                                   |     produce correct rate.         |
|                                   |     For example, the customer     |
|                                   |     receives $200 each week,      |
|                                   |     code, for the period of       |
|                                   |     payment, Amount $: '400' and  |
|                                   |     Freq: '2WE'                   |
|                                   |                                   |
|                                   | Work Bonus income %- If the other |
|                                   | income type is eligible to have   |
|                                   | the Work Bonus applied, the Work  |
|                                   | Bonus income % will display:      |
|                                   |                                   |
|                                   | -   If the customer is eligible   |
|                                   |     for Work Bonus the percentage |
|                                   |     of their employment or        |
|                                   |     self-employment income from   |
|                                   |     royalty, commission or        |
|                                   |     contract income (type ROC)    |
|                                   |     that has been earned from     |
|                                   |     personal exertion is to be    |
|                                   |     recorded                      |
|                                   | -   The default is to code 100%.  |
|                                   |                                   |
|                                   | Source: information source        |
|                                   |                                   |
|                                   | DOR: date of receipt of           |
|                                   | information                       |
|                                   |                                   |
|                                   | Note: if a customer already has   |
|                                   | an assessment for an amount from  |
|                                   | the same source on the OIN        |
|                                   | screen:                           |
|                                   |                                   |
|                                   | -   do not code another on that   |
|                                   |     line, unless                  |
|                                   | -   the assessment of the         |
|                                   |     previous coding has stopped   |
|                                   |                                   |
|                                   | For example:                      |
|                                   |                                   |
|                                   | if there is coding from 1 January |
|                                   | 2020 to be assessed for 1 year,   |
|                                   | and                               |
|                                   |                                   |
|                                   | another payment has been received |
|                                   | on 1 June 2020, then              |
|                                   |                                   |
|                                   | create a new line for the 1 June  |
|                                   | 2020 payment to record cessation  |
|                                   | of income. At Income Ceased (end  |
|                                   | dated), code:                     |
|                                   |                                   |
|                                   | -   Start Date as the date of     |
|                                   |     change                        |
|                                   | -   Amount as zero                |
|                                   |                                   |
|                                   | record all details of other       |
|                                   | income on a DOC                   |
|                                   |                                   |
|                                   | finalise activity via Assessment  |
|                                   | Results (AR) screen               |

| 6                                 | Set up a review + Read more ...   |
|                                   |                                   |
|                                   | Code a review to check the income |
|                                   | or zero the income after the      |
|                                   | period the income is to affect    |
|                                   | the customer.                     |
|                                   |                                   |
|                                   | For pensioners:                   |
|                                   |                                   |
|                                   | -   A manual review for 52 weeks  |
|                                   |     is not needed if income is    |
|                                   |     coded as 'ANN' on OINS using  |
|                                   |     the 'PEN' system, as the      |
|                                   |     system automatically creates  |
|                                   |     the review for 12 months to   |
|                                   |     cease the assessment          |
|                                   | -   If the income is not coded as |
|                                   |     'ANN', or the income should   |
|                                   |     affect for a different        |
|                                   |     period, use the help facility |
|                                   |     ('?') in the RevRsn: field to |
|                                   |     find a valid reason code for  |
|                                   |     the review                    |
|                                   | -   For more information, see How |
|                                   |     to code and action a manual   |
|                                   |     review                        |
|                                   |                                   |
|                                   | For other customers:              |
|                                   |                                   |
|                                   | -   Code a review for the         |
|                                   |     applicable date in the future |
|                                   |     to review or delete the       |
|                                   |     income, using a review reason |
|                                   |     of INC on the Review          |
|                                   |     Registration (RVR) screen     |
|                                   | -   For example, if the income    |
|                                   |     was received on 1 May 2017    |
|                                   |     and to be assessed for 6      |
|                                   |     weeks, the review should be   |
|                                   |     set for 12 June 2017 to zero  |
|                                   |     off the income at the end of  |
|                                   |     the 6 week period             |
|                                   |                                   |
|                                   | Check that any income coded       |
|                                   | previously using an ongoing       |
|                                   | assessment code (for example,     |
|                                   | MTH, ANN, etc.) on OIN has been   |
|                                   | reduced to $0 from the end of its |
|                                   | assessment period.                |
