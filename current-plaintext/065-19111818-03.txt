This page contains the process for manually matching a Customer First
record to an Aged Care Management Payment System (ACMPS) record.

On this page:

Action a manual match for ACMPS record via ECML

Reasons an ECML match cannot be completed

Reasons an ECML match cannot be completed

Table 2: This table describes the process to follow when a manual match
between ACMPS and Customer First cannot be completed.

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Reason for match not able to be   |
|                                   | completed + Read more ...         |
|                                   |                                   |
|                                   | If a match cannot be finalised,   |
|                                   | determine the action required     |
|                                   | including any error that is       |
|                                   | presented:                        |
|                                   |                                   |
|                                   | ‘E038C3 – Customer already has a  |
|                                   | different external reference id’  |
|                                   |                                   |
|                                   | -   This error indicates that the |
|                                   |     CRN is linked to another      |
|                                   |     ACMPS ID                      |
|                                   |                                   |
|                                   | ‘E040C3 – External Reference Id   |
|                                   | already exists for a different    |
|                                   | customer’                         |
|                                   |                                   |
|                                   | -   This error indicates that     |
|                                   |     this ACMPS ID is already      |
|                                   |     linked to a Customer First    |
|                                   |     record                        |
|                                   |                                   |
|                                   | What is the reason that this      |
|                                   | match cannot be completed?        |
|                                   |                                   |
|                                   | -   Care recipient has multiple   |
|                                   |     ACMPS IDs, go to Step 2       |
|                                   | -   CRN is already linked to an   |
|                                   |     ACMPS ID, go to Step 2        |
|                                   | -   Care recipient has multiple   |
|                                   |     CRNs, go to Step 5            |
|                                   | -   Care recipient is deceased,   |
|                                   |     go to Step 7                  |
+-----------------------------------+-----------------------------------+
| 2                                 | Care recipient has multiple ACMPS |
|                                   | IDs + Read more ...               |
|                                   |                                   |
|                                   | A match cannot be confirmed when  |
|                                   | a CRN is already linked to an     |
|                                   | ACMPS ID. If there are multiple   |
|                                   | ACMPS IDs for a care recipient,   |
|                                   | further remediation is required   |
|                                   | prior to ECML matching being      |
|                                   | completed.                        |
|                                   |                                   |
|                                   | See Multiple ACMPS records        |
|                                   | remediation to determine the      |
|                                   | Master ACMPS record. This may     |
|                                   | require TPS/SSO assistance.       |
|                                   |                                   |
|                                   | Is a Master ACMPS record          |
|                                   | identified and remediated?        |
|                                   |                                   |
|                                   | Yes, and the CRN is currently     |
|                                   | linked to:                        |
|                                   |                                   |
|                                   | -   Master ACMPS, go to Step 3    |
|                                   | -   Source ACMPS, go to Step 4    |
|                                   |                                   |
|                                   | No, awaiting outcome of referral  |
|                                   | to determine ACMPS ID status, go  |
|                                   | to Step 6                         |
+-----------------------------------+-----------------------------------+
| 3                                 | Customer First record is linked   |
|                                   | to the Master ACMPS record + Read |
|                                   | more ...                          |
|                                   |                                   |
|                                   | If the CRN is linked to the       |
|                                   | Master ACMPS ID, the match        |
|                                   | request in ECML is a Source ACMPS |
|                                   | ID only.                          |
|                                   |                                   |
|                                   | See Multiple ACMPS records        |
|                                   | remediation for information on    |
|                                   | moving Identification Numbers and |
|                                   | resending data when required.     |
|                                   |                                   |
|                                   | Type X next to the activity on    |
|                                   | ECML to remove from the list.     |
|                                   |                                   |
|                                   | Procedure ends here.              |
+-----------------------------------+-----------------------------------+
| 4                                 | Customer First record is linked   |
|                                   | to a Source ACMPS record + Read   |
|                                   | more ...                          |
|                                   |                                   |
|                                   | If the CRN is linked to a Source  |
|                                   | ACMPS record, unlink it and then  |
|                                   | link it to the Master ACMPS       |
|                                   | record:                           |
|                                   |                                   |
|                                   | Enter CRN to access customer’s    |
|                                   | record and confirm that Source    |
|                                   | ACMPS ID is linked on the NU      |
|                                   | screen                            |
