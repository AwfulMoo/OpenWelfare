Examples of appropriate and inappropriate Callback and ACTDOC work items

Table 1

+-----------------------------------+-----------------------------------+
| Item                              | Description                       |
+-----------------------------------+-----------------------------------+
| 1                                 | Appropriate Callback work items   |
|                                   |                                   |
|                                   | -   A customer has contacted to   |
|                                   |     talk about a rejected claim.  |
|                                   |     After discussing the decision |
|                                   |     using information on the      |
|                                   |     record, the customer wants a  |
|                                   |     further explanation. This is  |
|                                   |     outlined in Request for an    |
|                                   |     explanation or application    |
|                                   |     for a formal review           |
|                                   | -   Appointment profiles for a    |
|                                   |     service centre state 'local   |
|                                   |     booking only' and there is a  |
|                                   |     requirement for the service   |
|                                   |     centre to book an appointment |
|                                   |     and notify customer of the    |
|                                   |     time                          |
|                                   | -   Customer has contacted to     |
|                                   |     speak to a Financial          |
|                                   |     Information Service (FIS)     |
|                                   |     Officer, and no FIS officer   |
|                                   |     is currently available to     |
|                                   |     accept the call immediately   |
+-----------------------------------+-----------------------------------+
| 2                                 | Inappropriate Callback work items |
|                                   |                                   |
|                                   | -   Customer enquires about the   |
|                                   |     progress of a claim that is   |
|                                   |     outside timeliness standards  |
|                                   |     and has supplied all the      |
|                                   |     required information. See     |
|                                   |     Progress of Claim             |
|                                   | -   Customer wants an explanation |
|                                   |     of a decision and the Service |
|                                   |     Officer has not tried to      |
|                                   |     discuss the decision based on |
|                                   |     information on the file       |
|                                   | -   Customer asks to speak to a   |
|                                   |     certain Service Officer       |
|                                   |     because they prefer to deal   |
|                                   |     with that Service Officer     |
|                                   | -   Customer requests to speak to |
|                                   |     a Service Officer whose name  |
|                                   |     is at the bottom of a letter  |
|                                   |     they have received and the    |
|                                   |     reason for the call is not    |
|                                   |     clarified to determine if the |
|                                   |     call can be finalised at      |
|                                   |     first point of contact        |
|                                   | -   Customer has enquired about   |
|                                   |     progress of claim and claim   |
|                                   |     is still within the           |
|                                   |     timeliness standard           |
|                                   | -   Customer has enquired about   |
|                                   |     actioning scanned documents   |
|                                   |     that can be processed at      |
|                                   |     point of contact              |
|                                   | -   The Callback work item        |
|                                   |     contains information only and |
|                                   |     includes no request for a     |
|                                   |     call back                     |
|                                   | -   There is an existing Callback |
|                                   |     work item regarding the same  |
|                                   |     issue already on customer     |
|                                   |     file                          |
+-----------------------------------+-----------------------------------+
| 3                                 | Appropriate ACTDOC work items     |
|                                   |                                   |
|                                   | -   An Operational Blueprint      |
|                                   |     procedure specifically states |
|                                   |     an ACTDOC work item must be   |
|                                   |     sent to another site          |
|                                   | -   A customer is responding to a |
|                                   |     request for further           |
|                                   |     information that needs to be  |
|                                   |     actioned on Customer Record,  |
|                                   |     and that information cannot   |
|                                   |     be coded at point of contact  |
|                                   | -   A rescan request              |
|                                   | -   A Disability Support Pension  |
|                                   |     (DSP) customer advises they   |
|                                   |     want to voluntarily register  |
|                                   |     for employment assistance     |
+-----------------------------------+-----------------------------------+
| 4                                 | Inappropriate ACTDOC work items   |
|                                   |                                   |
|                                   | -   A claim needs to be reindexed |
|                                   | -   Customer requests to have     |
|                                   |     claim withdrawn               |
|                                   | -   Request to action a scanned   |
|                                   |     document that can be          |
|                                   |     processed at point of contact |
|                                   | -   Request to action an          |
|                                   |     outstanding Customer          |
|                                   |     Declaration Form (CDF) when a |
|                                   |     customer has already attended |
|                                   |     a CDF appointment or has a    |
|                                   |     current Job Plan              |
|                                   | -   Request to restore a customer |
|                                   |     with mutual obligation        |
|                                   |     requirements whose payment is |
|                                   |     CAN/DNL (Did not lodge a      |
|                                   |     Reporting Statement) or FRP   |
|                                   |     (Failed to Report)            |
|                                   | -   The ACTDOC work item contains |
|                                   |     information only and includes |
|                                   |     no request for action         |
|                                   | -   There is an existing ACTDOC   |
|                                   |     work item regarding the same  |
|                                   |     issue already on customer     |
|                                   |     file                          |

Keywords

There are limited reasons why a keyword would need to be added manually.
Keywords must only be added manually when the guided procedure or Fast
Note tools are not working or Operational Blueprint says to do so.

The following list of Callback/ACTDOC approved keywords is not a full
list. Service Officers must check the relevant Operational Blueprint
procedure for appropriate keywords.

The prefix of ACT identifies ACTDOC work items and CALL identifies
Callback work items.

Social workers assessing mutual obligation exemptions

A Callback work item must be sent if a Social Worker has assessed a job
seeker as meeting the criteria for an exemption from mutual obligation
requirements due to a â€˜Major Personal Crisisâ€™.

seeker), Parenting Payment (PP) and Special Benefit (SpB) customers.

Using the Callback/ACTDOC workflow, social workers must
