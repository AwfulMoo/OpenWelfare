General system functionality issues

System functionality issues and how to resolve them

Table 8: This table lists the common system functionality issues and
outlines the process to resolve them.

+-----------------------------------+-----------------------------------+
| Issue                             | Resolution                        |
+-----------------------------------+-----------------------------------+
| Veteran History (VHOI)            | A service displayed on VHOI has   |
|                                   | not necessarily been paid. Any    |
|                                   | voucher that is received by the   |
|                                   | mainframe that is not in error is |
|                                   | displayed on veteran history      |
|                                   | (VHOI). Service Officers must     |
|                                   | check that the associated claim   |
|                                   | has been paid to determine if the |
|                                   | service has been paid.            |
+-----------------------------------+-----------------------------------+
| Veteran services rejected with    | VHOI and statements (VPSI) are    |
| reason code 376 not on statements | stored by patient file number.    |
|                                   | When the veteran file number      |
|                                   | cannot be determined, the system  |
|                                   | is unable to write any            |
|                                   | information to history (VHOI) or  |
|                                   | the statement (VPSI). As a        |
|                                   | consequence, when a service is    |
|                                   | rejected with reason code 376     |
|                                   | (veteran cannot be identified     |
|                                   | with information supplied), the   |
|                                   | information related to that       |
|                                   | service will not be displayed.    |
+-----------------------------------+-----------------------------------+
| Duplicate provider statements not | When a File Fix or other          |
| available (NO STMTAVL)            | information culling action has    |
|                                   | removed a patient from a claim,   |
|                                   | the NO STMTAVL (no statement      |
|                                   | available) message will display.  |
|                                   | However, when one of a number of  |
|                                   | services for a patient have been  |
|                                   | culled, this error message is not |
|                                   | displayed.                        |
+-----------------------------------+-----------------------------------+
| 1-PATIENT Default Selecting and   | While changing or selecting a     |
| Delete Service Line               | patient and using [F3] to         |
|                                   | auto-select the patient in the    |
|                                   | rest of the claim, if a service   |
|                                   | line is deleted using D on the    |
|                                   | action field and [F6], the system |
|                                   | may ABEND. Service Officers can   |
|                                   | resolve the ABEND by entering the |
|                                   | full veteran number, including    |
|                                   | leading 0s. To prevent the ABEND, |
|                                   | Service Officers should complete  |
|                                   | the [F3] patient selection        |
|                                   | function in the entire claim      |
|                                   | prior to attempting to delete a   |
|                                   | service line.                     |
+-----------------------------------+-----------------------------------+
| ABEND error message               | When processing a Veterans'       |
|                                   | claim, sometimes the transaction  |
|                                   | will fail and an 'ABEND' error    |
|                                   | message will appear with an error |
|                                   | code, for example,                |
|                                   |                                   |
|                                   | error: 'DFHAC2206 18:00:43        |
|                                   | CICSPT02 Transaction SUSR failed  |
|                                   | with ABEND AZI6'.                 |
|                                   |                                   |
|                                   | Report the error to the Programme |
|                                   | Support Officer (PSO) who will    |
|                                   | email the VAP Helpdesk with a     |
|                                   | screen shot of the ABEND message. |
+-----------------------------------+-----------------------------------+
| PCC Card Type Online Override     | When a Pension concession card    |
|                                   | (PCC) cardholder is system        |
|                                   | rejected with reason code 744 and |
|                                   | an Online paperless claim has the |
|                                   | ReferralOverrideTypeCde set to H, |
|                                   | the system is inserting reason    |
|                                   | code 350 over the top of the 744  |
|                                   | rejection.                        |
+-----------------------------------+-----------------------------------+
