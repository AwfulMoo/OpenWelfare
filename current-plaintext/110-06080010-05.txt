+-----------------------------------+-----------------------------------+
| 1                                 | Accessing the IRS + Read more ... |
|                                   |                                   |
|                                   | In the Income Security System     |
|                                   | (ISS), code a customer payment    |
|                                   | environment (e.g. NSS, PEN, FAO)  |
|                                   | in the Sys: field and:            |
|                                   |                                   |
|                                   | -   go to the Reviews Main Menu   |
|                                   |     (IRMM) screen. The IRMM       |
|                                   |     screen displays screens used  |
|                                   |     in the IRS, or                |
|                                   | -   other IRS screens can be      |
|                                   |     opened directly by entering   |
|                                   |     the appropriate screen name   |
|                                   |                                   |
|                                   | Navigating in the IRS             |
|                                   |                                   |
|                                   | If unfamiliar with the IRS        |
|                                   | screens, access the IRS Main Menu |
|                                   | by coding 'IRMM' in the Nxt:      |
|                                   | field.                            |
|                                   |                                   |
|                                   | A number of screens will be       |
|                                   | presented. To navigate to any     |
|                                   | screen, code the screen literal   |
|                                   | into the Nxt: field and press     |
|                                   | [Enter].                          |
|                                   |                                   |
|                                   | -   For details about any screen  |
|                                   |     listed, navigate to that      |
|                                   |     screen and use F1 (Screen     |
|                                   |     Help)                         |
|                                   | -   Most IRS screen details can   |
|                                   |     only be accessed when a       |
|                                   |     customer's record and/or      |
|                                   |     review is being viewed        |

| 7                                 | Review Status and Review          |
|                                   | Sub-status codes + Read more ...  |
|                                   |                                   |
|                                   | Review Status:                    |
|                                   |                                   |
|                                   | -   REG - Registered              |
|                                   | -   STA - Started                 |
|                                   | -   CAN - Cancelled               |
|                                   | -   FIN - Finalised               |
|                                   | -   COM - Completed               |
|                                   | -   FUT - Future                  |
|                                   |                                   |
|                                   | A search by a Review Status of    |
|                                   | 'STA' (Started) will not return   |
|                                   | any reviews which have a Review   |
|                                   | Status of 'HLD' (Held).           |
|                                   |                                   |
|                                   | A search by a Review Status of    |
|                                   | 'FIN' (Finalised) will also       |
|                                   | return reviews with a Review      |
|                                   | Status of 'ARP' (reviews subject  |
|                                   | to an Applicable Restoration      |
|                                   | Period).                          |
|                                   |                                   |
|                                   | Review Sub-Status:                |
|                                   |                                   |
|                                   | -   DUE - Due                     |
|                                   | -   HLD - Held                    |
|                                   | -   NOT - Not Approved            |
|                                   | -   SUB - Submitted               |
|                                   | -   ARP - Applicable Restoration  |
|                                   |     Period                        |
|                                   | -   AMD - Amend Random Sample     |
|                                   |     Survey                        |
|                                   |                                   |
|                                   | Note: there are on-line search    |
|                                   | criteria for:                     |
|                                   |                                   |
|                                   | -   RST - Registered/Started      |
|                                   | -   FCO - Finalised/Completed     |

| 9                                 | Navigation Aid for IRS + Read     |
|                                   | more ...                          |
|                                   |                                   |
|                                   | Command and Description:          |
|                                   |                                   |
|                                   | -   'NR' in Nxt: field - Move to  |
|                                   |     next review record selected   |
|                                   |     from the Integrated Review    |
|                                   |     Review Enquiry (IRRE) screen  |
|                                   | -   'PR' in Nxt: field - Move to  |
|                                   |     previous review record        |
|                                   |     selected from the IRRE screen |
|                                   |                                   |
|                                   | Screens to assist with            |
|                                   | attribution, result calculation   |
|                                   | and validating review results     |
|                                   |                                   |
|                                   | Command and Description:          |
|                                   |                                   |
|                                   | -   'XDE' in Nxt: field - Move to |
|                                   |     the rate history of a         |
|                                   |     customer which can identify   |
|                                   |     the rate variation caused by  |
|                                   |     an activity being processed   |
|                                   |     on a customer's record.       |
|                                   | -   'ES' in Nxt: field - Move to  |
|                                   |     the Event Summary (ES) screen |
|                                   |     to identify activities which  |
|                                   |     are attributed, or should be  |
|                                   |     attributed to a review. Also  |
|                                   |     useful to identify the        |
|                                   |     activity which finalised the  |
|                                   |     review.                       |
|                                   | -   'IRRC' in Nxt: field - Move   |
|                                   |     to the Result Components      |
|                                   |     (IRRC) screen to identify     |
|                                   |     which activities are          |
|                                   |     attributed to a review and    |
|                                   |     the rate variations           |
|                                   |     calculated by IRS for each    |
|                                   |     payment component.            |
|                                   | -   'RATS' in Nxt: field - from   |
|                                   |     within an activity can        |
|                                   |     identify the rate variation   |
|                                   |     associated with the           |
|                                   |     processing of that activity.  |
|                                   |     ('S'elect the payment line    |
|                                   |     which was current at the time |
|                                   |     the activity was processed).  |
|                                   | -   'RAC' in Nxt: field - from    |
|                                   |     within an activity while on   |
|                                   |     the Period Rates Summary      |
|                                   |     (RATS) screen can identify    |
|                                   |     the rate variation for each   |
|                                   |     payment component.            |
|                                   | -   'HAL' in Nxt: field - Move to |
|                                   |     the Activity History List     |
|                                   |     (HAL) screen to identify an   |
|                                   |     activity processed on a       |
|                                   |     customer's record. This       |
|                                   |     screen can be useful to       |
|                                   |     identify Activity Management  |
|                                   |     Reference (AMR) numbers for   |
|                                   |     consequential debts created   |
|                                   |     from an activity, and also to |
|                                   |     view to events logged on the  |
|                                   |     activity. For example a       |
|                                   |     'CANIRS' event is logged to   |
|                                   |     cancel a Service Update or    |
|                                   |     Host Review in IRS.           |
|                                   | -   'XPGIRSD' in Nxt: field -     |
|                                   |     Move to the IRS 'X' files     |
|                                   |     (only where access to view    |
|                                   |     data in these 'X' files has   |
|                                   |     been granted). This is useful |
|                                   |     to identify where a           |
|                                   |     restoration has been          |
|                                   |     incorrectly manually          |
|                                   |     unattributed from a review.   |
