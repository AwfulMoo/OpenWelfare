This page contains the process for reinstating a Contracted Service
Provider (CSP) organisation record to the status it held before it was
retired, that is, either 'Active' or 'Deactivated', in the Healthcare
Identifiers (HI) Service.

Reinstate a CSP

Reinstating a CSP organisation record

This table describes how to reinstate a Contracted Service Provider
(CSP) organisation record in the Healthcare Identifiers (HI) Service
that has been retired in error. Note: before completing this procedure,
ensure that the policy conditions are met.

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Perform a search for the          |
|                                   | organisation being reinstated +   |
|                                   | Read more ...                     |
|                                   |                                   |
|                                   | From the HI eHealth program       |
|                                   | (EHP), perform a search for the   |
|                                   | organisation being reinstated.    |
|                                   |                                   |
|                                   | See: Searching for records in the |
|                                   | Healthcare Identifiers Service    |
|                                   | using the eHealth Program (EHP).  |
|                                   |                                   |
|                                   | The Search for Organisation       |
|                                   | screen is displayed with one or   |
|                                   | more matching organisations under |
|                                   | search results.                   |
+-----------------------------------+-----------------------------------+
| 2                                 | Select Amend in the Action column |
|                                   | of the relevant record + Read     |
|                                   | more ...                          |
|                                   |                                   |
|                                   | The Amend CSP Organisation screen |
|                                   | is displayed.                     |
+-----------------------------------+-----------------------------------+
| 3                                 | Select the Organisation Details   |
|                                   | tab + Read more ...               |
|                                   |                                   |
|                                   | The Organisation Details tab      |
|                                   | displays the selected record.     |
+-----------------------------------+-----------------------------------+
| 4                                 | Select Reinstate + Read more ...  |
|                                   |                                   |
|                                   | The Organisation Activity Status  |
|                                   | changes to 'Active' or            |
|                                   | 'Deactivated', depending on its   |
|                                   | status before retirement.         |
+-----------------------------------+-----------------------------------+
| 5                                 | Add reason + Read more ...        |
|                                   |                                   |
|                                   | An automatic comment of           |
|                                   | Reinstated appears in the Reason  |
|                                   | for Activity Status change field. |
|                                   |                                   |
|                                   | Add a reason why the record is    |
|                                   | being reinstated.                 |
+-----------------------------------+-----------------------------------+
| 6                                 | Select Validate and Submit + Read |
|                                   | more ...                          |
|                                   |                                   |
|                                   | If there are:                     |
|                                   |                                   |
|                                   | -   validation errors, correct    |
|                                   |     errors as prompted            |
|                                   | -   no validation errors, the     |
|                                   |     Confirm Amend CSP             |
|                                   |     organisation screen is        |
|                                   |     displayed                     |
+-----------------------------------+-----------------------------------+
| 7                                 | Check details + Read more ...     |
|                                   |                                   |
|                                   | If the details are:               |
|                                   |                                   |
|                                   | -   incorrect, select Edit to     |
|                                   |     return to the Amend CSP       |
|                                   |     organisation screen           |
|                                   | -   correct, select Save          |
|                                   |                                   |
|                                   | A confirmation message is         |
|                                   | displayed.                        |
+-----------------------------------+-----------------------------------+
| 8                                 | Contact Tier 3 support + Read     |
|                                   | more ...                          |
|                                   |                                   |
|                                   | Contact Tier 3 support to arrange |
|                                   | for a letter of notification to   |
|                                   | be mailed to the organisation.    |
+-----------------------------------+-----------------------------------+
