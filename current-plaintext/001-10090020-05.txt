Recording re-engagement reasons

This table describes re-engagement appointment reasons, how to manually
record a 'no re-engagement' requirement in the customer's record.

+-----------------------------------+-----------------------------------+
| Item                              | Description                       |
+-----------------------------------+-----------------------------------+
| 1                                 | Re-engagement appointment         |
|                                   | reasons + Read more ...           |
|                                   |                                   |
|                                   | RAPRC - Re-engagement appointment |
|                                   | booked for a connection           |
|                                   |                                   |
|                                   | RNSNP - Re-engagement appointment |
|                                   | booked for a No Show, No Pay      |
|                                   | failure. (Note: this appointment  |
|                                   | type is no longer in use. No Show |
|                                   | No Pay failures with a            |
|                                   | disengagement flag for failure to |
|                                   | attend a compulsory activity are  |
|                                   | no longer generated, as           |
|                                   | activities in CDP are voluntary)  |
|                                   |                                   |
|                                   | Note: 'RACOA' (serious failure    |
|                                   | waiver) appointments have not     |
|                                   | been included in the above list   |
|                                   | as their purpose differs from     |
|                                   | other re-engagement appointments. |
+-----------------------------------+-----------------------------------+
| 2                                 | No re-engagement codes for        |
|                                   | Provider initiated compliance     |
|                                   | events + Read more ...            |
|                                   |                                   |
|                                   | RNH - No re-engagement, current   |
|                                   | exemption                         |
|                                   |                                   |
|                                   | RNE - No re-engagement, no        |
|                                   | phone/diary appt available        |
|                                   |                                   |
|                                   | RNL - No re-engagement, Provider  |
|                                   | requested withdrawal              |
|                                   |                                   |
|                                   | RYS - Self re-engaged after       |
|                                   | failure event date                |
+-----------------------------------+-----------------------------------+
| 3                                 | Manually recording 'no            |
|                                   | re-engagement' requirements +     |
|                                   | Read more ...                     |
|                                   |                                   |
|                                   | The Participation Compliance      |
|                                   | workflow is the tool used to      |
|                                   | record re-engagement. In limited  |
|                                   | situations if the Participation   |
|                                   | Compliance workflow is not        |
|                                   | available, the system can be used |
|                                   | to record a 'no re-engagement     |
|                                   | requirement' using the following  |
|                                   | steps:                            |
|                                   |                                   |
|                                   | -   on the Incident Summary       |
|                                   |     (BIIS) screen, select the     |
|                                   |     failure                       |
|                                   | -   on the BIDE screen place a    |
|                                   |     'Y' in the View Investigation |
|                                   |     Items: field and press        |
|                                   |     [Enter] to go to the          |
|                                   |     Investigation Items Summary   |
|                                   |     (BITS) screen                 |
|                                   | -   on the BITS screen type       |
|                                   |     'BITE' in Nxt: field          |
|                                   | -   on the Investigation Item     |
|                                   |     (BITE) screen, record the     |
|                                   |     appropriate 'no               |
|                                   |     re-engagement' code then      |
|                                   |     enter an 'I' in the Action:   |
|                                   |     field                         |
|                                   | -   proceed through Assessment    |
|                                   |     Results (AR) screen to        |
|                                   |     finalise the update           |
+-----------------------------------+-----------------------------------+
