|                                   |     Processing System (via        |
|                                   |     eBusiness system)             |

|                                   | -   Key the association’s         |
|                                   |     approval ID into the Approved |
|                                   |     Supplier Number (ASN) field   |
|                                   | -   Key the relevant claim period |
|                                   |     into the Claim Period Number  |
|                                   |     (CPN) field                   |

|                                   |     beside the claim search       |
|                                   |     results then select Open      |
|                                   |     Claim Period                  |
|                                   |                                   |
|                                   | The PBS Online Claims Processing  |
|                                   | System automatically flags these  |
|                                   | claims with the Hold Payment      |
|                                   | indicator and sets the Payment    |
|                                   | Due Date to the day following the |
|                                   | claim upload. The PBS Payment     |
|                                   | ID’s will not populate and the    |
|                                   | payment status will be            |
|                                   | ‘Withheld’.                       |
|                                   |                                   |
|                                   | Untick the Hold Payment box on    |
|                                   | the Query Claim Period screen. Do |
|                                   | not tick the Hold Payment box     |
|                                   | again                             |
|                                   |                                   |
|                                   | Once the Hold Payment indicator   |
|                                   | is unticked, the system will      |
|                                   | calculate the actual Payment Due  |
|                                   | Date using the Claim Package      |
|                                   | Received Date (the date the claim |
|                                   | is received via HPOS) and         |
|                                   | allocate PBS Payment ID’s during  |
|                                   | the overnight payment batch job.  |
|                                   |                                   |
|                                   | Check the Claim Package Received  |
|                                   | Date when unticking the Hold      |
|                                   | Payment indicator, and prioritise |
|                                   | processing accordingly.           |
|                                   |                                   |
|                                   | After the overnight payment batch |
|                                   | process, the Claim Period Status  |
|                                   | will be Generate Statement and it |
|                                   | will remain as this until the due |
|                                   | date is reached which is 15 days  |
|                                   | from the date the claim is        |
|                                   | received.                         |

| 2                                 | Quality assurance checking + Read |
|                                   | more ...                          |
|                                   |                                   |
|                                   | During the automated claims       |
|                                   | assessment process, the PBS       |
|                                   | Online Claims Processing System   |
|                                   | automatically allocates reason    |
|                                   | codes for the items that do not   |
|                                   | meet requirements. The reason     |
|                                   | codes are located at the bottom   |
|                                   | window pane of the Ostomy Item    |
|                                   | Details screen.                   |
|                                   |                                   |

|                                   | Stoma & ParaQuad Reason Codes and |

|                                   | quality assurance.                |
|                                   |                                   |
|                                   | Check the first and last items    |
|                                   | and a minimum of 4 or maximum of  |
|                                   | 10 items (randomly selected) in   |
|                                   | between.                          |
|                                   |                                   |
|                                   | View the items in the system:     |
|                                   |                                   |
|                                   | On the Query Claim Period screen, |
|                                   | select View prescriptions         |
|                                   |                                   |
|                                   | Select Open prescriptions to edit |
|                                   | the claim data if required. The   |
|                                   | following fields can be edited:   |
|                                   |                                   |
|                                   | -   Serial number                 |
|                                   | -   Patient identifier            |
|                                   | -   Date of Supply                |
|                                   | -   Item code                     |
|                                   | -   Quantity                      |
|                                   |                                   |
|                                   | Note: quality assurance must be   |
|                                   | completed on one day (not across  |
|                                   | multiple days).                   |
|                                   |                                   |
|                                   | If any items identified in        |
|                                   | quality assurance have reject     |
|                                   | reason codes allocated to them,   |
|                                   | check the supporting paperwork to |
|                                   | confirm the electronic data is    |
|                                   | correct. For example, if          |
|                                   | rejection reason code 298 - 'Date |
|                                   | of supply was invalid' is         |
|                                   | allocated to an item, check the   |
|                                   | paperwork to see if it has been   |
|                                   | included. If it has, amend the    |
|                                   | item in the system and resubmit   |
|                                   | so that it is paid. If it hasn’t  |
|                                   | then leave rejected.              |
|                                   |                                   |
|                                   | If adjustments are made to the    |
|                                   | claim, they are automatically     |
|                                   | picked up during payment          |
|                                   | processing that night and will    |
|                                   | have a new PBS Payment ID         |
|                                   | allocated, which appears on the   |
|                                   | association’s statement. If       |
|                                   | adjustments are done over         |
|                                   | multiple days, a new payment ID   |
|                                   | is allocated for each day. To     |
|                                   | make reading statements as easy   |
|                                   | as possible, it is preferred that |
|                                   | all adjustments are done on the   |
|                                   | same day to ensure minimal        |
|                                   | payment IDs are allocated.        |
