Control line to access QITI

Using control lines to access QITI

Table 1: This table contains a list of control lines to access QITI.

Note: QITI always defaults to the M01 out of hospital benefit rate. If
the in-patient rate is required, amend the table field to M01H and press
[PF5].

+-----------------------------------+-----------------------------------+
| Item                              | Description                       |
+-----------------------------------+-----------------------------------+
| 1                                 | QITI [Enter]                      |
|                                   |                                   |
|                                   | This returns the most basic blank |
|                                   | enquiry screen, then:             |
|                                   |                                   |
|                                   | -   key the item number and the   |
|                                   |     date being queried, and       |
|                                   | -   press either the [Enter] key  |
|                                   |     or one of the function keys   |
+-----------------------------------+-----------------------------------+
| 2                                 | QITI,item,date [Enter]            |
|                                   |                                   |
|                                   | This returns a brief description  |
|                                   | of the item and general assessing |
|                                   | information as at that date of    |
|                                   | service.                          |
|                                   |                                   |
|                                   | To access other screens, use      |
|                                   | [PF8] to or a specific function   |
|                                   | key for the required information. |
+-----------------------------------+-----------------------------------+
| 3                                 | QITI,item,date [appropriate       |
|                                   | function key, such as [PF10] for  |
|                                   | Assessing Rulings]                |
|                                   |                                   |
|                                   | This returns the screen with the  |
|                                   | specific information held in that |
|                                   | function key.                     |

General QITI use

Fields that can be changed in QITI

Table 2: This table contains a list of which fields that can be changed
in QITI.

+-----------------------------------+-----------------------------------+
| Item                              | Description                       |
+-----------------------------------+-----------------------------------+
| 1                                 | To view a different MBS item      |
|                                   |                                   |
|                                   | Over-key the item number          |
|                                   | displayed in the Item field with  |
|                                   | the new item number and [Enter]   |
+-----------------------------------+-----------------------------------+
| 2                                 | To view a different date of       |
|                                   | enquiry                           |
|                                   |                                   |
|                                   | Over-key the date displayed in    |
|                                   | the Inquiry Date field with the   |
|                                   | new date and [Enter]              |
+-----------------------------------+-----------------------------------+
| 3                                 | To view a different table         |
|                                   |                                   |
|                                   | Over-key the entry in the Table   |
|                                   | field with one of the following:  |
|                                   |                                   |
|                                   | -   M01 - Medicare Out of         |
|                                   |     hospital                      |
|                                   | -   M01H - Medicare In hospital   |
|                                   | -   M01V - DVA                    |
+-----------------------------------+-----------------------------------+
| 4                                 | To view restrictions between two  |
|                                   | MBS items (from the F11 screen)   |
|                                   |                                   |
|                                   | Key in the item in the            |
|                                   | Restrictions from item field and  |
|                                   | [Enter]                           |

Assessing information page fields

Assessing information page field explanations

Table 3: This table contains an explanation of the fields on the
assessing information page.

+-----------------------------------+-----------------------------------+
| Field                             | Explanation                       |
+-----------------------------------+-----------------------------------+
| Time Dependency                   | An indication of whether a time   |
|                                   | dependent restriction applies for |
|                                   | the item number keyed in the ITEM |
|                                   | field. If this field is blank, no |
|                                   | Time Dependency restriction       |
|                                   | applies.                          |
|                                   |                                   |
|                                   | If YES appears in this field, a   |
|                                   | Time Dependency restriction       |
|                                   | applies. Press [PF20] to access   |
|                                   | the number of services that can   |
|                                   | be paid in the period specified.  |
|                                   | An indication of whether the      |
|                                   | restriction applies to the same   |
|                                   | or any provider is also           |
|                                   | displayed.                        |
+-----------------------------------+-----------------------------------+
| Renumbered to Item                | This is only displayed when the   |
|                                   | item number being queried has     |
|                                   | been renumbered.                  |
+-----------------------------------+-----------------------------------+
| Renumbered from Item              | This is only displayed when the   |
|                                   | item number being queried has     |
|                                   | been renumbered from an old       |
|                                   | number.                           |
+-----------------------------------+-----------------------------------+
| Minimum Age                       | The patient must be the age       |
|                                   | displayed or older to be able to  |
|                                   | claim the item queried. If NO is  |
|                                   | displayed, the Minimum Age        |
|                                   | restriction does not apply.       |
+-----------------------------------+-----------------------------------+
| Maximum Age                       | The patient must be the age       |
|                                   | displayed or younger to be able   |
|                                   | to claim the item queried. If NO  |
|                                   | is displayed, the Maximum Age     |
|                                   | restriction does not apply.       |
+-----------------------------------+-----------------------------------+
| Gender Restriction                | The patient must be the gender    |
|                                   | displayed to claim the item       |
|                                   | queried. If NO is displayed, the  |
|                                   | Gender restriction does not       |
|                                   | apply.                            |
|                                   |                                   |
|                                   | See Gender Guidelines             |
|                                   | Implementation.                   |
+-----------------------------------+-----------------------------------+
| RHCA Allowed                      | If YES is displayed, a patient    |
|                                   | who is eligible under a           |
|                                   | Reciprocal Health Care Agreement  |
|                                   | (RHCA) is eligible to claim the   |
|                                   | item queried.                     |
|                                   |                                   |
|                                   | If NO is displayed, an RHCA       |
|                                   | patient is not eligible to claim  |
|                                   | the item queried.                 |
+-----------------------------------+-----------------------------------+
| Aftercare Period                  | If this field is blank, no        |
|                                   | aftercare period applies.         |
|                                   | Otherwise, the period quoted in   |
|                                   | days is the average aftercare     |
|                                   | period for the item queried. This |
|                                   | information must not be provided  |
|                                   | to external customers.            |
+-----------------------------------+-----------------------------------+
| Referral Required                 | The term referral on this screen  |
|                                   | also means request (for the       |
|                                   | purpose of diagnostic imaging and |
|                                   | pathology services).              |
|                                   |                                   |
|                                   | Either YES or NO will be          |
|                                   | displayed, indicating whether     |
|                                   | Referral or Request details must  |
|                                   | be keyed to allow payment of the  |
|                                   | item.                             |
+-----------------------------------+-----------------------------------+
| Force Pend                        | If YES is displayed, this item    |
|                                   | requires research before the      |
|                                   | claim can be paid. Generally,     |
|                                   | [PF10] Assessing Rulings will     |
|                                   | provide guidance.                 |
+-----------------------------------+-----------------------------------+
| Assistant Processing              | Displays whether or not a benefit |
|                                   | can be paid under an item in      |
|                                   | Group T9 'Assistance at an        |
|                                   | operation' when performed by      |
|                                   | another practitioner in           |
|                                   | conjunction with the item being   |
|                                   | queried.                          |
+-----------------------------------+-----------------------------------+
| Composite Applicable              | If YES is displayed, the item     |
|                                   | being queried forms part of a     |
|                                   | composite item.                   |
+-----------------------------------+-----------------------------------+
| Drop Down Items                   | Displays the item number that     |
|                                   | should be used to pay benefit     |
|                                   | when the item number being        |
|                                   | queried has not been requested or |
|                                   | referred.                         |
+-----------------------------------+-----------------------------------+
| Bulking Allowed                   | No services can be bulked.        |
|                                   |                                   |
| (MO1V table only)                 |                                   |
+-----------------------------------+-----------------------------------+
| In Hospital Service               | If FREQUENTLY is displayed, the   |
|                                   | item being queried is generally   |
|                                   | performed on a private in-patient |
|                                   | in a hospital.                    |
|                                   |                                   |
|                                   | If NEVER is displayed, the item   |
|                                   | being queried is only ever        |
|                                   | performed on a patient who is not |
|                                   | admitted to hospital (85% benefit |
|                                   | applies).                         |
|                                   |                                   |
|                                   | If POSSIBLY is displayed, the     |
|                                   | item being queried could be       |
|                                   | performed on a private in-patient |
|                                   | in a hospital or on a patient who |
|                                   | is not admitted to hospital.      |
|                                   |                                   |
|                                   | If ALWAYS is displayed, the item  |
|                                   | being queried must be performed   |
|                                   | on a private in-patient in a      |
|                                   | hospital (75% benefit applies).   |
+-----------------------------------+-----------------------------------+
| Diagnostic Self Deemed            | If NO is displayed, the item      |
|                                   | being queried cannot be           |
|                                   | self-determined and Request or    |
|                                   | Referral details must be keyed    |
|                                   | before the claim can be paid.     |
|                                   |                                   |
|                                   | If YES is displayed, it means     |
|                                   | that the service can be           |
|                                   | self-determined. When an account  |
|                                   | indicates the item is             |
|                                   | self-determined, PI11 should be   |
|                                   | keyed, not Request or Referral    |
|                                   | details.                          |
|                                   |                                   |
|                                   | If UNUSED is displayed, it means  |
|                                   | that it is not applicable to the  |
|                                   | item being queried. Where a DI    |
|                                   | item has a requirement for a      |
|                                   | referral and the details aren’t   |
|                                   | entered within the claim, it will |
|                                   | return an error message.          |
+-----------------------------------+-----------------------------------+
| Eligible as Aftercare             | If YES is displayed, it means the |
|                                   | item being queried can be         |
|                                   | aftercare for a procedure.        |
|                                   |                                   |
|                                   | If NO is displayed, it means the  |
|                                   | item being queried is not         |
|                                   | aftercare for a procedure.        |
+-----------------------------------+-----------------------------------+
| Referral Type                     | If this field is blank, no        |
|                                   | referral is required. Otherwise,  |
|                                   | it reflects the type of referral  |
|                                   | required for the item queried.    |
+-----------------------------------+-----------------------------------+
| LSPN Required                     | If YES is displayed, it means the |
|                                   | item being queried requires a     |
|                                   | Location Specific Practice Number |
|                                   | (LSPN).                           |
+-----------------------------------+-----------------------------------+
| DI Accreditation                  | If YES is displayed, it means the |
|                                   | item being queried requires       |
|                                   | Diagnostic Imaging (DI) equipment |
|                                   | used in performing the service to |
|                                   | be accredited.                    |
+-----------------------------------+-----------------------------------+
| Item Association                  | If YES is displayed, it means the |
|                                   | item being queried is associated  |
|                                   | with another item which must      |
|                                   | either be present in the claim or |
|                                   | on the patient history.           |
+-----------------------------------+-----------------------------------+

QITI function keys

Summary of QITI function keys

Table 4: This table contains a list of QITI function keys and their
description. Note: to access function keys [PF13] to [PF24] use the
shift key plus the function key that corresponds to your selection For
example, [PF13] becomes Shift and [F1].

+-----------------------------------+-----------------------------------+
| Function Key                      | Description                       |
+-----------------------------------+-----------------------------------+
| [PF1]-HELP                        | Place cursor on the field you     |
|                                   | wish explained and press F1.      |
+-----------------------------------+-----------------------------------+
| [PF3]-EXIT                        | Exits from QITI to a blank        |
|                                   | screen.                           |
+-----------------------------------+-----------------------------------+
| [PF4]-ASS INFO                    | Lists assessing information       |
|                                   | attached to the item.             |
+-----------------------------------+-----------------------------------+
| [PF5]-FEE/BEN                     | Displays the schedule fee and     |
|                                   | benefit rate and safety net       |
|                                   | threshold amounts. If the item    |
|                                   | has a derived fee, additional     |
|                                   | information will be displayed.    |
+-----------------------------------+-----------------------------------+
| [PF6]-PROV QUAL                   | Lists the specialty codes and     |
|                                   | their descriptions of the         |
|                                   | practitioners able to perform the |
|                                   | item. Press F8 to display         |
|                                   | ineligible provider               |
|                                   | qualifications where applicable.  |
|                                   | Press F8 to display LSPN          |
|                                   | equipment requirements.           |
+-----------------------------------+-----------------------------------+
| [PF7]-BACKWARDS                   | Scrolls back to the previous      |
|                                   | screen.                           |
+-----------------------------------+-----------------------------------+
| [PF8]-FORWARDS                    | Scrolls forward to the next       |
|                                   | screen.                           |
+-----------------------------------+-----------------------------------+
| [PF9]-FULL DESC                   | Displays the full description of  |
|                                   | the item number including         |
|                                   | Category, Group and Subgroup.     |
+-----------------------------------+-----------------------------------+
| [PF10]-ASS RULE                   | Displays the assessing rulings    |
|                                   | and the date they were added to   |
|                                   | the item.                         |
+-----------------------------------+-----------------------------------+
| [PF11]-RESTRICT                   | Lists the items that restrict;    |
|                                   | indicates which should be paid;   |
|                                   | indicates whether the system      |
|                                   | automatically restricts or pends  |
|                                   | it; and the number of days the    |
|                                   | restriction applies.              |
+-----------------------------------+-----------------------------------+
| [PF12]-CANCEL                     | Cancels out of the QITI screen    |
|                                   | and returns a blank screen.       |
+-----------------------------------+-----------------------------------+
| [PF13]-REQ PROV                   | Lists the specialty codes of the  |
|                                   | practitioners that can request    |
|                                   | the diagnostic imaging service or |
|                                   | refer for the allied health       |
|                                   | service. Press F8 and go to the   |
|                                   | end to display LSPN equipment     |
|                                   | requirements.                     |
+-----------------------------------+-----------------------------------+
| [PF14]-PROC INST                  | Provides processing instructions. |
+-----------------------------------+-----------------------------------+
| [PF20]-TIMEDEP                    | Displays the item/s (including    |
|                                   | the same item where applicable)   |
|                                   | that are time dependant with the  |
|                                   | enquiry item; and the period that |
|                                   | the item/s are unable to be       |
|                                   | claimed with the enquiry item.    |
+-----------------------------------+-----------------------------------+
| [PF21]-HCVS                       | Displays the Hospital Claims      |
|                                   | Validation System (HCSV) and/or   |
| (M01V Table only)                 | Pseudo PARS creation related      |
|                                   | information for the enquired      |
|                                   | item.                             |
+-----------------------------------+-----------------------------------+
| [PF22}-CO-CLAIM                   | Displays the items that have      |
|                                   | co-claiming restrictions. For     |
|                                   | example where certain             |
|                                   | consultations are not payable     |
|                                   | when billed with an item from     |
|                                   | Group T8, on the same day, by the |
|                                   | same provider.                    |
+-----------------------------------+-----------------------------------+

Medicare benefit type

Medicare benefit type fields and associated explanations

Table 6: This table lists the Benefit type fields and associated
explanations which appear on the Schedule fee page.

+-----------------------------------+-----------------------------------+
| Field                             | Explanation                       |
+-----------------------------------+-----------------------------------+
| GAP amount                        | This field shows the Greatest     |
|                                   | Permissible Gap (GPG) which is    |
|                                   | indexed annually on 1st November. |
|                                   | The out of hospital Medicare      |
|                                   | benefit is calculated as 85% of   |
|                                   | the scheduled fee OR the          |
|                                   | scheduled fee less the GPG        |
|                                   | amount, whichever is greater.     |
|                                   |                                   |
|                                   | Note: the GAP amount will show as |
|                                   | $0.00 for items queried on the    |
|                                   | MO1H Table.                       |
+-----------------------------------+-----------------------------------+
| Standard Percent                  | Percentage of the schedule fee    |
|                                   | paid as a benefit. This reflects  |
|                                   | the table viewed (eg. 75% will be |
|                                   | displayed in the M01H table).     |
+-----------------------------------+-----------------------------------+
| Standard Benefit                  | The dollar amount of benefit paid |
|                                   | for the item. This is calculated  |
|                                   | from the scheduled fee and the    |
|                                   | standard percent.                 |
+-----------------------------------+-----------------------------------+
| Bulk Bill Percent                 | Percentage of the schedule fee    |
|                                   | paid as a benefit when the item   |
|                                   | is bulk billed.                   |
+-----------------------------------+-----------------------------------+
| Bulk Bill Benefit                 | The dollar amount of benefit paid |
|                                   | to the provider for the bulk      |
|                                   | billed item.                      |
+-----------------------------------+-----------------------------------+
| EMSN Cap Applies                  | This field only displays for item |
|                                   | numbers where the EMSN cap        |
|                                   | applies. It displays the          |
|                                   | percentage of the schedule fee    |
|                                   | that will be paid as the maximum  |
|                                   | EMSN benefit.                     |
+-----------------------------------+-----------------------------------+
| Maximum cap                       | This field only displays for item |
|                                   | numbers where the EMSN cap        |
|                                   | applies. It displays the maximum  |
|                                   | EMSN cap that applies to the item |
|                                   | number.                           |
+-----------------------------------+-----------------------------------+

Item restriction sub-columns

Item restriction page sub-columns and associated explanations

Table 7: This table lists the Item Restriction page sub-columns and
associated explanations.

+-----------------------------------+-----------------------------------+
| Column                            | Explanation                       |
+-----------------------------------+-----------------------------------+
| ITEM                              | Lists the items that restrict     |
|                                   | with the item being queried.      |
+-----------------------------------+-----------------------------------+
| PAY                               | Indicates which item out of the   |
|                                   | two to pay.                       |
|                                   |                                   |
|                                   | -   High - Pay the higher benefit |
|                                   |     of the two items and reject   |
|                                   |     the other                     |
|                                   | -   Low - Pay the lower benefit   |
|                                   |     of the two items and reject   |
|                                   |     the other                     |
+-----------------------------------+-----------------------------------+
| PROV                              | Indicates whether the restriction |
|                                   | applies.                          |
|                                   |                                   |
|                                   | -   Same - The other item is      |
|                                   |     performed by the same         |
|                                   |     practitioner                  |
|                                   | -   Any - The other item is       |
|                                   |     performed by any other        |
|                                   |     practitioner                  |
+-----------------------------------+-----------------------------------+
| SYS                               | Indicates whether the system will |
|                                   | take automatic action, pend the   |
|                                   | claim or reject the claim.        |
|                                   |                                   |
|                                   | -   Blank - The system will not   |
|                                   |     take any automatic action     |
|                                   | -   Pend - The system will        |
|                                   |     automatically pend the claim  |
|                                   | -   Reject - The system will      |
|                                   |     automatically reject the item |
+-----------------------------------+-----------------------------------+
| RST DAYs                          | Indicates the number of days the  |
|                                   | restriction between the two items |
|                                   | applies.                          |
|                                   |                                   |
|                                   | Also check the Assessing Ruling   |
|                                   | (F10) for additional information. |
