For staff conducting Automation of Income Stream (AoIS) Interventions
only

This page contains information on AoIS interventions when the customer
contacts or does not contact.

| 1                                 | Discuss Income Stream data + Read |
|                                   | more ...                          |
|                                   |                                   |
|                                   | A match has occurred because the  |
|                                   | information on the customer's     |
|                                   | Centrelink record differs from    |
|                                   | information received from an      |
|                                   | income stream provider.           |
|                                   |                                   |
|                                   | Prior to discussing the Income    |
|                                   | Stream data with the customer,    |
|                                   | check the customer record to      |
|                                   | ensure the funds/ asset has not   |
|                                   | already been assessed and/or      |
|                                   | debts raised:                     |
|                                   |                                   |
|                                   | -   Managed Investments Summary   |
|                                   |     (MIS) screen                  |
|                                   | -   Superannuation Summary (SUPS) |
|                                   |     screen                        |
|                                   | -   Savings Summary (SVS) screen  |
|                                   | -   Financial Investment Summary  |
|                                   |     (IVIS) screen                 |
|                                   | -   Pensions Income and Asset     |
|                                   |     Summary (PIAS) screen         |
|                                   | -   Pensions Assessable Asset     |
|                                   |     Summary (PAAS) screen         |
|                                   | -   Assets Summary (NASS) screen  |
|                                   |                                   |
|                                   | If identified, consider the use   |
|                                   | of that information and take the  |
|                                   | required actions prior to coding  |
|                                   | on SUPS. The information found    |
|                                   | and actions taken must be clearly |
|                                   | documented in the Notes.          |
|                                   |                                   |
|                                   | Discuss the Income Stream data    |
|                                   | with the customer and give them a |
|                                   | chance to provide a verbal        |
|                                   | update.                           |
|                                   |                                   |
|                                   | The scope start date will either  |
|                                   | be:                               |
|                                   |                                   |
|                                   | -   the start date of the income  |
|                                   |     stream, or                    |
|                                   | -   if the start date is greater  |
|                                   |     than 7 years, the scope start |
|                                   |     date will be 1 July of the    |
|                                   |     financial year, 7 years       |
|                                   |     before the current financial  |
|                                   |     year. (For example, currently |
|                                   |     in 18/19 financial year,      |
|                                   |     scope start date will be 1    |
|                                   |     July 2011)                    |
|                                   | -   if the customer started       |
|                                   |     receiving an income support   |
|                                   |     payment after the income      |
|                                   |     stream start date, the scope  |
|                                   |     start date will be the date   |
|                                   |     in which the customerâ€™s       |
|                                   |     payment started               |
|                                   |                                   |
|                                   | Discuss the amounts listed in any |
|                                   | of the income fields on the       |
|                                   | Income Stream Data.               |
|                                   |                                   |
|                                   | Use questioning skills to get as  |
|                                   | much information as possible from |
|                                   | the customer over the phone.      |
|                                   |                                   |
|                                   | If the customer indicates the     |
|                                   | information is incorrect, review  |
|                                   | the customer record to determine  |
|                                   | an anomaly or mismatch.           |
|                                   |                                   |
|                                   | Inform the customer:              |
|                                   |                                   |
|                                   | 'Would you mind holding the line  |
|                                   | while I check your record.'       |
|                                   |                                   |
|                                   | Preliminary assessment            |
|                                   |                                   |
|                                   | If there are at least 2           |
|                                   | commonalities between the         |
|                                   | customer and the Income Stream    |
|                                   | data (e.g. name and date of       |
|                                   | birth, or name and address), it   |
|                                   | can be decided that the Income    |
|                                   | Stream data is not a mismatch and |
|                                   | should be continued.              |
|                                   |                                   |
|                                   | Alternative spelling or           |
|                                   | misspelling of a customer's name  |
|                                   | does not mean a mismatch.         |
|                                   |                                   |
|                                   | Can an anomaly or mismatch be     |
|                                   | identified from the customer      |
|                                   | record?                           |
|                                   |                                   |
|                                   | -   Yes, go to Step 2             |
|                                   | -   No, go to Step 3              |
