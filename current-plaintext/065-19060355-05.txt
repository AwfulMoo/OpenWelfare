Task Selector screens

Table 1: an overview of the screens that display on the Process Direct
Task Selector in the Home Equity Access Scheme applications. Screens not
listed in the Task Selector can be accessed via the Super Key field.

+-----------------------------------+-----------------------------------+
| Item                              | Description                       |
+-----------------------------------+-----------------------------------+
| 1                                 | Pensions Assessment Date (PNA)    |
|                                   |                                   |
|                                   | In Process Direct the Pensions    |
|                                   | Assessment (PNA) screen displays  |
|                                   | the customer’s Age Pension age    |
|                                   | date and the Pensions Assessment  |
|                                   | Date.                             |
|                                   |                                   |
|                                   | Age Pension age Details           |
|                                   |                                   |
|                                   | If the customer has reached Age   |
|                                   | Pension age, the Age Pension      |
|                                   | Eligibility indicator will        |
|                                   | display 'Yes'. To be eligible for |
|                                   | the Scheme the customer must be   |
|                                   | Age Pension age or partnered to a |
|                                   | person who is Age Pension age.    |
|                                   |                                   |
|                                   | To view the partner’s Age Pension |
|                                   | age details, select               |
|                                   | [relationships_menu_icon.png] to  |
|                                   | view the Relations menu and       |
|                                   | select the partner to access      |
|                                   | their record.                     |
|                                   |                                   |
|                                   | Pensions Assessment (PNA) Date    |
|                                   |                                   |
|                                   | The PNA date is automatically     |
|                                   | determined based on the           |
|                                   | customer’s circumstances, and can |
|                                   | be changed if required.           |
|                                   |                                   |
|                                   | If the customer is current on a   |
|                                   | pension, no changes need to be    |
|                                   | made to the PNA date for the      |
|                                   | purposes of the Scheme            |
|                                   | application.                      |
|                                   |                                   |
|                                   | If the customer has applied for   |
|                                   | loan-only loan, the PNA date will |
|                                   | default to the date the Scheme    |
|                                   | application was lodged. For       |
|                                   | loan-only applications that       |
|                                   | result in a grant outcome, the    |
|                                   | PNA date must match the Scheme    |
|                                   | grant date to ensure pension      |
|                                   | arrears are not paid. The Scheme  |
|                                   | grant date is the Effect Date     |
|                                   | recorded on the HEAS Details      |
|                                   | (PLS) screen.                     |
+-----------------------------------+-----------------------------------+
| 2                                 | HEAS Summary (PLSS) screen        |
|                                   |                                   |
|                                   | The HEAS Summary (PLSS) screen    |
|                                   | provides a list of current and    |
|                                   | past loans paid to the customer   |
|                                   | through the Scheme. The details   |
|                                   | for each loan can be viewed by    |
|                                   | selecting the relevant line from  |
|                                   | the summary to open the HEAS      |
|                                   | Details (PLS) screen.             |
|                                   |                                   |
|                                   | Review the information on the     |
|                                   | PLSS screen before proceeding to  |
|                                   | add a new loan, noting:           |
|                                   |                                   |
|                                   | -   if a loan already exists with |
|                                   |     an outstanding loan balance   |
|                                   |     (for example, a total balance |
|                                   |     above $0), a new Scheme loan  |
|                                   |     cannot be granted until the   |
|                                   |     existing loan has been        |
|                                   |     settled. See Assessing Home   |
|                                   |     Equity Access Scheme          |
|                                   |     applications for information  |
|                                   |     about how to assess existing  |
|                                   |     loans                         |
|                                   | -   if an outstanding loan does   |
|                                   |     not exist and the claim was   |
|                                   |     lodged online or via Assisted |
|                                   |     Customer claim (ACC), a new   |
|                                   |     row for a provisional loan is |
|                                   |     automatically created. To     |
|                                   |     view and update the details,  |
|                                   |     select Edit                   |
|                                   | -   if the claim was lodged via   |
|                                   |     the SA310 paper form, a row   |
|                                   |     for a new loan needs to be    |
|                                   |     manually created. Select      |
|                                   |     [Adding a new line] to add a  |
|                                   |     new row                       |
+-----------------------------------+-----------------------------------+
| 3                                 | HEAS Details (PLS) screen         |
|                                   |                                   |
|                                   | The HEAS Details (PLS) screen is  |
|                                   | mandatory for all Scheme claims.  |
|                                   | The information recorded on the   |
|                                   | PLS screen determines the loan    |
|                                   | assessment outcome.               |
|                                   |                                   |
|                                   | If the application was submitted  |
|                                   | online or via ACC, the            |
|                                   | information the customer entered  |
|                                   | to the application will be        |
|                                   | pre-populated. Otherwise, a new   |
|                                   | loan needs to be added and the    |
|                                   | data needs to be manually         |
|                                   | entered. To add a new loan select |
|                                   | [Adding a new line] on the PLSS   |
|                                   | screen.                           |
|                                   |                                   |
|                                   | Review the details on the PLS     |
|                                   | screen and make any required      |
|                                   | updates. For information on the   |
|                                   | individual fields and how to code |
|                                   | the PLS screen see Table 2.       |
+-----------------------------------+-----------------------------------+
| 4                                 | Real estate details (REBS)        |
|                                   |                                   |
|                                   | The Real Estate / Business        |
|                                   | Details (REBS) screen displays    |
|                                   | details of the real estate and    |
|                                   | business assets owned by the      |
|                                   | customer and (if applicable)      |
|                                   | their partner. The REBS screen in |
|                                   | Process Direct provides view and  |
|                                   | update access to the following    |
|                                   | screens:                          |
|                                   |                                   |
|                                   | -   Real Estate/Business          |
|                                   |     Identifying (REBI) screen     |
|                                   | -   Real Estate Detail (RE)       |
|                                   |     screen                        |
|                                   | -   Real Estate Valuation         |
|                                   |     Information (REVI) screen     |
|                                   |                                   |
|                                   | To view all information for a     |
|                                   | particular property select        |
|                                   | Expand.                           |
|                                   |                                   |
|                                   | The information on REBS is        |
|                                   | pre-populated where the customer  |
|                                   | has provided details in the       |
|                                   | online application. The           |
|                                   | information will have already     |
|                                   | been reviewed and updated via     |
|                                   | Customer First as part of the     |
|                                   | Assessing Home Equity Access      |
|                                   | Scheme applications process.      |
|                                   | Review the information entered to |
|                                   | the application and ensure that   |
|                                   | the real estate asset(s) being    |
|                                   | used as security for the loan are |
|                                   | coded correctly. If further       |
|                                   | updates are required, update the  |
|                                   | information by adding a new row   |
|                                   | or amending existing details.     |
|                                   |                                   |
|                                   | The Centrelink Property ID on the |
|                                   | REBI screen for each real estate  |
|                                   | property offered as security is   |
|                                   | to be coded in the CLK Loan       |
|                                   | Property Ids field on the HEAS    |
|                                   | Details (PLS) screen. If multiple |
|                                   | properties have been offered,     |
|                                   | separate the ID's with a comma.   |
|                                   |                                   |
|                                   | Note: requests for real estate    |
|                                   | valuations are coded via the REVI |
|                                   | screen, however the request       |
|                                   | functionality does not yet exist  |
|                                   | in Process Direct. See Assessing  |
|                                   | Home Equity Access Scheme         |
|                                   | applications for information on   |
|                                   | how to request a valuation for    |
|                                   | the Scheme applications.          |
+-----------------------------------+-----------------------------------+
| 5                                 | Customer Person Detail Summary    |
|                                   | (CPDS)                            |
|                                   |                                   |
|                                   | The Customer Person Detail        |
|                                   | Summary (CPDS) screen displays    |
|                                   | the current and historical person |
|                                   | details for the customer,         |
|                                   | including name, sex and date of   |
|                                   | birth. Selecting a line on the    |
|                                   | CPDS screen opens the Customer    |
|                                   | Person Details (CPD) screen,      |
|                                   | which displays the full details   |
|                                   | corresponding to the selected     |
|                                   | line.                             |
|                                   |                                   |
|                                   | The information on this screen is |
|                                   | generally updated when the        |
|                                   | customer provides their proof of  |
|                                   | identity documents. If there are  |
|                                   | any changes required to the       |
|                                   | customer’s personal details,      |
|                                   | update the information by adding  |
|                                   | a new row or amending existing    |
|                                   | details.                          |
|                                   |                                   |
|                                   | For more information about        |
|                                   | customer personal details see     |
|                                   | Aliases and other names (legal,   |
|                                   | preferred, sex and gender diverse |
|                                   | customers, cultural               |
|                                   | considerations etc.).             |
+-----------------------------------+-----------------------------------+
| 6                                 | Marital Status (MS)               |
|                                   |                                   |
|                                   | The Marital Status (MS) screen is |
|                                   | used to record the customer’s     |
|                                   | relationship status, and details  |
|                                   | pertaining to the relationship    |
|                                   | status such as Partner Permission |
|                                   | to Enquire (PPE) and qualifiers   |
|                                   | of the relationship status.       |
|                                   |                                   |
|                                   | The information on this screen is |
|                                   | pre-populated where the customer  |
|                                   | has provided details in the       |
|                                   | online application. If any        |
|                                   | changes are required to the       |
|                                   | customer’s relationship status,   |
|                                   | update the information by adding  |
|                                   | a new row or amending existing    |
|                                   | details.                          |
|                                   |                                   |
|                                   | For more information about        |
|                                   | relationship status see           |
|                                   | Assessment of relationship status |
|                                   | – processes, roles and            |
|                                   | responsibilities.                 |
+-----------------------------------+-----------------------------------+
| 7                                 | Address details (ADH)             |
|                                   |                                   |
|                                   | The Address (ADH) screen displays |
|                                   | the customer’s current and past   |
|                                   | addresses and can be used to add, |
|                                   | view or change the customer’s     |
|                                   | address details.                  |
|                                   |                                   |
|                                   | The information on the ADH screen |
|                                   | is pre-populated where the        |
|                                   | customer has provided details in  |
|                                   | their application. If there are   |
|                                   | any changes required, update the  |
|                                   | information by adding a new row   |
|                                   | or amending existing details.     |
|                                   |                                   |
|                                   | If the customer has notified via  |
|                                   | their Scheme application that     |
|                                   | they have changed address, check  |
|                                   | if they still live in the same    |
|                                   | geographically-based environment. |
|                                   | If an environment transfer is     |
|                                   | required, conduct the transfer    |
|                                   | before processing the Scheme      |
|                                   | application.                      |
|                                   |                                   |
|                                   | For more information see:         |
|                                   |                                   |
|                                   | -   Updating address details      |
|                                   | -   Environment transfers and     |
|                                   |     relationships                 |
+-----------------------------------+-----------------------------------+
| 8                                 | Accommodation (ACS)               |
|                                   |                                   |
|                                   | The Accommodation (ACS) screen is |
|                                   | used to record the accommodation  |
|                                   | details for the customer, such as |
|                                   | home ownership details and rent   |
|                                   | information. The information on   |
|                                   | the ACS screen is used for        |
|                                   | determining the customer’s        |
|                                   | eligibility for Rent Assistance   |
|                                   | (RA) with their Home Equity       |
|                                   | Access Scheme qualifying payment. |
|                                   |                                   |
|                                   | The information on the ACS screen |
|                                   | is pre-populated where the        |
|                                   | customer has provided details in  |
|                                   | their application. If changes are |
|                                   | required, update the information  |
|                                   | by adding a new row or amending   |
|                                   | existing details.                 |
|                                   |                                   |
|                                   | Note: the Scheme application does |
|                                   | not gather accommodation          |
|                                   | information from applicants who   |
|                                   | advise they are applying for a    |
|                                   | Scheme-only loan, as Scheme-only  |
|                                   | customers are not eligible to     |
|                                   | receive RA. As such,              |
|                                   | accommodation details are not     |
|                                   | required to be recorded on the    |
|                                   | ACS screen for Scheme-only        |
|                                   | claims.                           |
|                                   |                                   |
|                                   | For more information about the    |
|                                   | ACS screen see Recording          |
|                                   | accommodation details.            |
+-----------------------------------+-----------------------------------+
| 9                                 | Telephone Details (TDS)           |
|                                   |                                   |
|                                   | The Telephone Details (TDS)       |
|                                   | screen displays the customer’s    |
|                                   | current and past telephone        |
|                                   | details and can be used to add,   |
|                                   | view or change the customer’s     |
|                                   | telephony details.                |
|                                   |                                   |
|                                   | The information on the TDS screen |
|                                   | is pre-populated where the        |
|                                   | customer has provided details in  |
|                                   | their application. If there are   |
|                                   | any changes required, update the  |
|                                   | information by adding a new row   |
|                                   | or amending existing details.     |
|                                   |                                   |
|                                   | For more information see Updating |
|                                   | telephone details and/or paying   |
|                                   | Telephony Allowance (TAL).        |
+-----------------------------------+-----------------------------------+
| 10                                | Sharer Details Summary (SDS)      |
|                                   |                                   |
|                                   | The Sharer Details Summary (SDS)  |
|                                   | screen provides access to the     |
|                                   | following screens:                |
|                                   |                                   |
|                                   | -   Sharing Details (SDQ)         |
|                                   | -   Details of a Sharing          |
|                                   |     Relationship (SHR)            |
|                                   | -   Relationship Status Among the |
|                                   |     Customer, Residence and       |
|                                   |     Sharer (SHD)                  |
|                                   | -   Living Arrangement (LAO/LAQ)  |
|                                   |                                   |
|                                   | These screens are used to record  |
|                                   | whether the customer is sharing   |
|                                   | accommodation with a person aged  |
|                                   | 16 years or over who is not an    |
|                                   | immediate family member. Sharer   |
|                                   | details are used to determine     |
|                                   | correct entitlement to Rent       |
|                                   | Assistance (RA).                  |
|                                   |                                   |
|                                   | The information on the sharer     |
|                                   | details screens is pre-populated  |
|                                   | where the customer has provided   |
|                                   | details in their application. If  |
|                                   | changes are required, update the  |
|                                   | information by adding a new row   |
|                                   | or amending existing details.     |
|                                   |                                   |
|                                   | Note: accommodation details do    |
|                                   | not need to be recorded for       |
|                                   | loan-only claims, as loan-only    |
|                                   | customers are not eligible to     |
|                                   | receive RA. Accordingly, the      |
|                                   | Scheme application does not       |
|                                   | gather accommodation information  |
|                                   | from applicants who advise they   |
|                                   | are applying for a loan-only      |
|                                   | loan.                             |
|                                   |                                   |
|                                   | For more information about sharer |
|                                   | details see The Living            |
|                                   | Arrangements question set.        |
+-----------------------------------+-----------------------------------+
| 11                                | Regular Period Frequency (RPF)    |
|                                   |                                   |
|                                   | The Regular Period Frequency      |
|                                   | (RPF) screen displays the payment |
|                                   | frequency for supplements the     |
|                                   | customer may receive with their   |
|                                   | Home Equity Access Scheme         |
|                                   | qualifying payment, including:    |
|                                   |                                   |
|                                   | -   Energy Supplement (CES)       |
|                                   | -   Pension Supplement Minimum    |
|                                   |     (PSMIN)                       |
|                                   |                                   |
|                                   | The screen can be used to change  |
|                                   | the period frequency for these    |
|                                   | supplements from a fortnightly    |
|                                   | payment frequency to a quarterly  |
|                                   | payment frequency, and vice       |
|                                   | versa.                            |
|                                   |                                   |
|                                   | Updates to the RPF screen are not |
|                                   | required for the Scheme           |
|                                   | applications, however the         |
|                                   | functionality is available if the |
|                                   | customer requests a change to the |
|                                   | frequency of their supplements.   |
|                                   | Note: loan-only customers are not |
|                                   | eligible to receive the Energy    |
|                                   | Supplement or Pension Supplement  |
|                                   | Minimum (PSMIN), therefore any    |
|                                   | update will only be applicable    |
|                                   | where the customer receives the   |
|                                   | supplements with their qualifying |
|                                   | payment.                          |
|                                   |                                   |
|                                   | For more information about the    |
|                                   | supplements see:                  |
|                                   |                                   |
|                                   | -   Pension Supplement            |
|                                   | -   Energy Supplement for         |
|                                   |     Commonwealth Seniors Health   |
|                                   |     Card holders                  |
+-----------------------------------+-----------------------------------+
| 12                                | Residence                         |
|                                   |                                   |
|                                   | To be eligible for the Scheme,    |
|                                   | applicants must meet the          |
|                                   | qualifying residence rules        |
|                                   | applicable to their Home Equity   |
|                                   | Access Scheme qualifying payment. |
|                                   | The customer's qualification is   |
|                                   | automatically assessed based on   |
|                                   | the residence information held on |
|                                   | the customer’s record and any new |
|                                   | information entered to the        |
|                                   | application.                      |
|                                   |                                   |
|                                   | Check the residence information   |
|                                   | provided by the customer in their |
|                                   | Scheme application and make any   |
|                                   | required updates. Residence       |
|                                   | updates must be coded even if the |
|                                   | application is to be rejected, in |
|                                   | order to correctly assess the     |
|                                   | customer’s eligibility for the    |
|                                   | Scheme qualifying payment.        |
|                                   |                                   |
|                                   | Common residence screens to       |
|                                   | review and update include:        |
|                                   |                                   |
|                                   | -   RSIMME - Immigration Enquiry  |
|                                   | -   RSLEG - Residence Details     |
|                                   | -   CRES - Country of Residence   |
|                                   | -   RSIM - Immigration Movements  |
|                                   |                                   |
|                                   | If the RSIM screen shows customer |
|                                   | was outside Australia on the date |
|                                   | they lodged the Scheme            |
|                                   | application, the customer is not  |
|                                   | eligible for the Scheme as they   |
|                                   | need to be physically present in  |
|                                   | Australia when the claim is made. |
|                                   |                                   |
|                                   | For information on residence      |
|                                   | coding, see Residence and         |
|                                   | Portability screens.              |
+-----------------------------------+-----------------------------------+
| 13                                | Payment Destination Details (PAS) |
|                                   |                                   |
|                                   | The Payment Destination Details   |
|                                   | (PAS) screen is used to record    |
|                                   | the bank account where the Scheme |
|                                   | loan payments will be sent.       |
|                                   | Payment destination details for   |
|                                   | the Scheme are recorded against   |
|                                   | Benefit System PEN – Pensions.    |
|                                   |                                   |
|                                   | Check the Scheme application to   |
|                                   | identify which bank account the   |
|                                   | customer has nominated to receive |
|                                   | their loan payments into. Ensure  |
|                                   | the correct information is        |
|                                   | recorded on the PAS screen for    |
|                                   | Benefit System PEN – Pensions.    |
|                                   |                                   |
|                                   | Note: if the customer receives a  |
|                                   | pension, the pension and Scheme   |
|                                   | loan must be paid to the same     |
|                                   | bank account. Any changes to the  |
|                                   | payment destination details for   |
|                                   | Benefit System: PEN – Pensions    |
|                                   | will be applied to the pension.   |
|                                   | If a pension recipient has        |
|                                   | nominated a different account for |
|                                   | their Scheme payments, contact    |
|                                   | the customer before applying the  |
|                                   | change to advise that the same    |
|                                   | account must be used for both     |
|                                   | payments, and confirm which       |
|                                   | account they wish to proceed      |
|                                   | with.                             |
|                                   |                                   |
|                                   | For more information see Payment  |
|                                   | destination.                      |
+-----------------------------------+-----------------------------------+
| 14                                | Tax File Number (TFN)             |
|                                   |                                   |
|                                   | The Tax File Number (TFN) screen  |
|                                   | is used for recording the         |
|                                   | customer’s TFN details, TFN       |
|                                   | authorisation details and TFN     |
|                                   | exemption information. The Scheme |
|                                   | cannot be paid unless the         |
|                                   | customer has provided a valid TFN |
|                                   | and authorisation, or has a valid |
|                                   | exemption.                        |
|                                   |                                   |
|                                   | The information on the TFN screen |
|                                   | will automatically populate with  |
|                                   | the customers TFN details if      |
|                                   | already provided. TFN details for |
|                                   | the Scheme are recorded against   |
|                                   | the Service Reason for the Scheme |
|                                   | qualifying payment, for example   |
|                                   | Age Pension (AGE).                |
|                                   |                                   |
|                                   | Check the information provided in |
|                                   | the Scheme application to ensure  |
|                                   | the populated details are         |
|                                   | correct, including that the       |
|                                   | customer has provided             |
|                                   | authorisation to use the TFN for  |
|                                   | the Scheme purposes. If a new     |
|                                   | provisional entry has been        |
|                                   | automatically created, check the  |
|                                   | details for correctness and make  |
|                                   | edits if required.                |
|                                   |                                   |
|                                   | If the customer is partnered,     |
|                                   | switch to the partner’s record    |
|                                   | and ensure their TFN details have |
|                                   | also been updated correctly.      |
|                                   | Partners of the Scheme applicants |
|                                   | are requested to provide a TFN,   |
|                                   | however it is not mandatory       |
|                                   | unless they are applying for      |
|                                   | their own Scheme loan.            |
|                                   |                                   |
|                                   | If the applicant is unable to     |
|                                   | provide their partner’s TFN,      |
|                                   | exemption reason SEC (Secretaries |
|                                   | Exemption) may be applied to the  |
|                                   | partner’s record. Recording of    |
|                                   | the TFN exemption must be applied |
|                                   | by the Customer Details Helpdesk. |
|                                   | To request an exemption for the   |
|                                   | partner, submit a request to the  |
|                                   | Helpdesk.                         |
|                                   |                                   |
|                                   | For more information about TFN    |
|                                   | requirements and exemptions see   |
|                                   | Requesting a Tax File Number      |
|                                   | (TFN).                            |
+-----------------------------------+-----------------------------------+
| 15                                | Office Code (OC)                  |
|                                   |                                   |
|                                   | The Office Code (OC) screen       |
|                                   | displays the office code(s) for   |
|                                   | all of a customer’s service       |
|                                   | reasons applicable as at a        |
|                                   | particular date. The system       |
|                                   | automatically assigns an office   |
|                                   | code based on the customer’s      |
|                                   | address details, but can be       |
|                                   | changed if the customer requests  |
|                                   | to use another office.            |
|                                   |                                   |
|                                   | For Scheme applicants, the office |
|                                   | code is recorded against the      |
|                                   | qualifying payment type, for      |
|                                   | example Age Pension. The same     |
|                                   | office code must be used for the  |
|                                   | qualifying payment and the        |
|                                   | Scheme.                           |
|                                   |                                   |
|                                   | Check the OC screen to see if any |
|                                   | updates are required. If a new    |
|                                   | provisional entry has been        |
|                                   | automatically created, check the  |
|                                   | details for correctness and make  |
|                                   | edits required. If the            |
|                                   | provisional entry is the same as  |
|                                   | the confirmed data, the           |
|                                   | provisional data can be deleted.  |
|                                   |                                   |
|                                   | For more information about office |
|                                   | codes, including a link to Office |
|                                   | Locator, see Updating address     |
|                                   | details.                          |
+-----------------------------------+-----------------------------------+
| 16                                | Other Contact Details (OCD)       |
|                                   |                                   |
|                                   | The Other Contact Details (OCD)   |
|                                   | screen is used to display and     |
|                                   | update details of a customer’s    |
|                                   | other contact features. These     |
|                                   | consist of:                       |
|                                   |                                   |
|                                   | -   Interpreter Language          |

|                                   | loan information was last         |
|                                   | updated. Details can only be      |
|                                   | updated on or after the most      |
|                                   | recent Effect Date. The details   |
|                                   | for historical Effect Dates are   |
|                                   | display-only.                     |

Pre grant interview

Table 3: information about the Scheme pre-grant interview. The Scheme
loans must not be granted until the interview has been conducted and the
customer accepts the loan offer.

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Purpose                           |
|                                   |                                   |
|                                   | Prior to granting the Scheme      |
|                                   | loan, a verbal offer of loan must |
|                                   | be made and an interview          |
|                                   | conducted with the applicant and  |
|                                   | partner, if any. The intent of    |
|                                   | the interview is to discuss the   |
|                                   | loan available to the applicant   |
|                                   | once the actual Maximum Loan      |
|                                   | Amount (MLA) and rate has been    |
|                                   | determined. The applicant must    |
|                                   | verbally agree to proceed with    |
|                                   | the loan. The interview also      |
|                                   | serves as a control to address    |
|                                   | the risk of vulnerability or      |
|                                   | elder abuse, fraud and capacity   |
|                                   | issues in a way that incorporates |
|                                   | and is aligned with best practice |
|                                   | for lenders of loans.             |
|                                   |                                   |
|                                   | The interview provides an         |
|                                   | opportunity for the Decision      |
|                                   | Maker (DM) to complete all of the |
|                                   | following:                        |
|                                   |                                   |
|                                   | -   inform the applicant of the   |
|                                   |     decision and provide the      |
|                                   |     details of the loan offer     |
|                                   | -   ensure the applicant has an   |
|                                   |     understanding of the Home     |
|                                   |     Equity Access Scheme terms    |
|                                   |     and conditions, including the |
|                                   |     requirements to maintain      |
|                                   |     eligibility criteria and      |
|                                   |     comply with ongoing           |
|                                   |     notification obligations      |
|                                   | -   provide information about the |
|                                   |     customers withdrawal rights   |
|                                   |     and review/appeal rights      |
|                                   | -   as far as possible, identify  |
|                                   |     if the customer and/or        |
|                                   |     partner does not understand,  |
|                                   |     or have capacity to           |
|                                   |     understand, the loan          |
|                                   | -   identify if there are signs   |
|                                   |     the customer will not be able |
|                                   |     to comply with the            |
|                                   |     terms/conditions              |
|                                   | -   as far as possible, identify  |
|                                   |     indicators of financial elder |
|                                   |     abuse or fraud                |
|                                   | -   reiterate the benefits of     |
|                                   |     seeking independent legal or  |
|                                   |     financial advice before       |
|                                   |     entering into the Scheme      |
|                                   | -   answer any questions the      |
|                                   |     applicant may have about the  |
|                                   |     Scheme                        |
|                                   | -   seek agreement from the       |
|                                   |     applicant and partner (if     |
|                                   |     any) to proceed on the loan   |
|                                   |     offer                         |
|                                   |                                   |
|                                   | The interview should be conducted |
|                                   | as close to the date of grant as  |
|                                   | possible. Typically the           |
|                                   | application is finalised on the   |
|                                   | same day the interview is         |
|                                   | conducted, providing the          |
|                                   | applicant has agreed to proceed   |
|                                   | with the loan.                    |
|                                   |                                   |
|                                   | For details of the information to |
|                                   | be covered during the Scheme      |
|                                   | interview, go to Step 2.          |
+-----------------------------------+-----------------------------------+
| 2                                 | Interview details                 |
|                                   |                                   |
|                                   | The Scheme pre-grant interview    |
|                                   | must include discussion of the    |
|                                   | following information.            |
|                                   |                                   |
|                                   | Details of the loan offer,        |
|                                   | including:                        |
|                                   |                                   |
|                                   | -   the value of the loan         |
|                                   |     security used in the          |
|                                   |     assessment                    |
|                                   | -   nominated amount (if          |
|                                   |     applicable)                   |
|                                   | -   Maximum Loan Amount (MLA)     |
|                                   |     available                     |
|                                   | -   fortnightly loan payment rate |
|                                   |     and/or advance payment amount |
|                                   | -   fortnightly pension rate      |
|                                   |     (including if zero)           |
|                                   | -   start date for the loan as of |
|                                   |     the next fortnight            |
|                                   |     Note: an offer to backdate    |
|                                   |     the start date should not be  |
|                                   |     made unless the customer      |
|                                   |     specifically requests it      |
|                                   | -   projected loan balance        |
|                                   |     information                   |
|                                   |                                   |
|                                   | The applicable interest rate and  |
|                                   | method of compounding interest,   |
|                                   | noting interest continues to      |
|                                   | accrue (compounding fortnightly)  |
|                                   | until the Scheme debt is repaid   |
|                                   | in full by the customer or their  |
|                                   | estate. Note: the Scheme interest |
|                                   | rate is set by the Minister for   |
|                                   | the Department of Social Services |
|                                   | and is published on the Federal   |
|                                   | Register of Legislation           |
|                                   |                                   |
|                                   | Taxation status of the loan       |
|                                   | component (i.e. non-taxable)      |
|                                   |                                   |
|                                   | Costs involved in registering,    |
|                                   | changing or withdrawing a charge  |
|                                   | or caveat on the titles of        |
|                                   | secured real estate. Withdrawal   |
|                                   | of the caveat/charge occurs at    |
|                                   | the point of settlement of the    |
|                                   | loan                              |
|                                   |                                   |
|                                   | Liability of the customer, or     |
|                                   | their estate, for all components  |
|                                   | of the Scheme loan including      |
|                                   | principal loan payments, costs    |
|                                   | and interest                      |
|                                   |                                   |

|                                   | (NNEG) may apply at settle time.  |

|                                   | debt be greater than the value of |
|                                   | the property (market value) at    |

|                                   | customer, partner or estate will  |

|                                   | the property’s current market     |

|                                   | Access Scheme loans               |

|                                   | Scheme loan amounts for           |

|                                   | the estate before entering into   |
|                                   | the Scheme                        |
|                                   |                                   |
|                                   | Once granted:                     |
