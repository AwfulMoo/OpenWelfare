|                                   |     Access Manager (TAM) resource |

|                                   |     tile. The Access Request      |

|                                   | -   No, close the Existing        |
|                                   |     Assignments screen. Go to     |
|                                   |     Step 3                        |
+-----------------------------------+-----------------------------------+
| 2                                 | How to request the business role  |
|                                   | in ESSentials + Read more ...     |
|                                   |                                   |
|                                   | Note: all fields are optional     |
|                                   | unless marked with an asterisk.   |
|                                   |                                   |
|                                   | To request the business role:     |
|                                   |                                   |
|                                   | -   Select the Add Role button    |
|                                   |     underneath the User Access    |
|                                   |     tab. The Select Roles pop up  |
|                                   |     box will display              |
|                                   | -   In the Functional Area field, |
|                                   |     select Aged Care from the     |
|                                   |     drop down menu                |
|                                   | -   Select the Search button. The |
|                                   |     Search results will display   |
|                                   |     in the Available assignment   |
|                                   |     block                         |
|                                   | -   Search for the required       |
|                                   |     business role and select the  |
|                                   |     row                           |
|                                   | -   Select the down arrow to move |
|                                   |     the selected role to the      |
|                                   |     Selected assignment block     |
|                                   | -   Select OK. The Access Request |
|                                   |     screen will display           |
|                                   | -   In the Description* field,    |
|                                   |     key in the business reason    |
|                                   |     for the role request          |
|                                   | -   In the Valid From* and Valid  |
|                                   |     To* dates field, adjust the   |
|                                   |     dates if required             |
|                                   | -   Select Submit to finalise the |
|                                   |     role request                  |
|                                   |                                   |
|                                   | The access request requires line  |
|                                   | manager approval, then role owner |
|                                   | approval.                         |
|                                   |                                   |
|                                   | The Service Officer will receive  |
|                                   | an email once access has been     |
|                                   | approved, go to Step 3            |
+-----------------------------------+-----------------------------------+
| 3                                 | How to check if Service Officers  |
|                                   | have a P number + Read more ...   |
|                                   |                                   |
|                                   | Service Officers must have a      |
|                                   | Tivoli Access Manager (TAM)       |
|                                   | account to access the ACSP. The   |
|                                   | Service Officer’s P number is     |
|                                   | confirmation that they have an    |
|                                   | existing TAM account.             |
|                                   |                                   |
|                                   | To check the P number via the ICT |
|                                   | Security Portal (ISP):            |
|                                   |                                   |
|                                   | Open ICT Security Portal (ISP)    |
|                                   |                                   |
|                                   | Select Copy or View Access        |
|                                   | underneath the Request Management |
|                                   | menu option                       |
|                                   |                                   |
|                                   | -   The Copy or View Access       |
|                                   |     screen will display           |
|                                   |                                   |
|                                   | Select the magnifying icon next   |
|                                   | to the User to View field         |
|                                   |                                   |
|                                   | -   The Search object list screen |
|                                   |     will display                  |
|                                   |                                   |
|                                   | Key the relevant details in the   |
|                                   | search field. Note: the Search    |
|                                   | field defaults to First Name      |
|                                   |                                   |
|                                   | Select Search                     |
|                                   |                                   |
|                                   | -   The search results will       |
|                                   |     display                       |
|                                   | -   The Agency Login ID field     |
|                                   |     displays the Service          |
|                                   |     Officer’s P number            |
|                                   |                                   |
|                                   | Does the Service Officer have a P |
|                                   | number?                           |
|                                   |                                   |
|                                   | -   Yes, go to Step 4             |
|                                   | -   No, the Service Officer needs |
|                                   |     to complete the Medicare      |
|                                   |     Legacy Access form in         |
|                                   |     mySupport for the creation of |
|                                   |     a Tivoli Access Manager (TAM) |
|                                   |     account. Once ICT has         |
|                                   |     confirmed completion of the   |
|                                   |     TAM account, go to Step 4     |
+-----------------------------------+-----------------------------------+
| 4                                 | How to request the Tivoli Access  |
|                                   | Manager (TAM) resource + Read     |
|                                   | more ...                          |
|                                   |                                   |
|                                   | Note: all fields are optional     |
|                                   | unless marked with an asterisk.   |
|                                   |                                   |
|                                   | To request the TAM resource:      |
|                                   |                                   |
|                                   | Open mySupport                    |
|                                   |                                   |
|                                   | In the Search field, key in       |
|                                   | ‘Medicare Access Change’. The     |
|                                   | search results will display       |
|                                   |                                   |
|                                   | Select the Medicare Access Change |
|                                   | mySupport form. The Medicare      |
|                                   | Access Change mySupport form will |
|                                   | display                           |
|                                   |                                   |
|                                   | In the Requested For* field,      |
|                                   | ensure the details are correct    |
|                                   | (while this defaults to the       |
|                                   | requester’s log on ID, this can   |
|                                   | change to another log on ID by    |
|                                   | selecting the drop down menu)     |
|                                   |                                   |
|                                   | In the What TAM (CDMS/PDS etc.)   |
|                                   | group(s) are to be added?*        |
|                                   | section, key in the required      |
|                                   | Tivoli Access Manager (TAM)       |
|                                   | resources (ac_staff_portal).      |
|                                   | Note: valid TAM groups must be    |
|                                   | specified. Requests with invalid  |
|                                   | group(s) will not be processed    |
|                                   |                                   |
|                                   | In the What additional Mainframe  |
|                                   | access is required? section, key  |
|                                   | in ‘Not applicable’               |
|                                   |                                   |
|                                   | In the What TAM group(s) are to   |
|                                   | be removed? section, key in ‘Not  |
|                                   | applicable’                       |
|                                   |                                   |
|                                   | In the What Mainframe access is   |
|                                   | to be removed? section, key in    |
|                                   | ‘Not applicable’                  |
|                                   |                                   |
|                                   | In the What is the business case  |
|                                   | for this access change?* section, |
|                                   | key in the business reason:       |
|                                   |                                   |
|                                   | -   Suggested text:               |
|                                   |     “Please add TAM account for   |
|                                   |     my [log on ID] in PRODUCTION  |
|                                   |     Environment and assign TAM    |
|                                   |     group (ac_staff_portal). This |
|                                   |     role allows me to access the  |
|                                   |     production version of the     |
|                                   |     Aged Care Staff Portal. I     |
|                                   |     require this role as part of  |
|                                   |     my job for aged care          |
|                                   |     purposes. Please do not       |
|                                   |     remove my other existing TAM  |
|                                   |     groups.”                      |
|                                   |                                   |
|                                   | Select the Submit button. The     |
|                                   | Request Details screen will       |
|                                   | display                           |
|                                   |                                   |
|                                   | In the User Define Request Title* |
|                                   | section, key in the title ‘Aged   |
|                                   | Care Staff Portal’                |
|                                   |                                   |
|                                   | In the User Define Request        |
|                                   | Description section, key in       |
|                                   | description                       |
|                                   |                                   |
|                                   | Select Submit                     |
|                                   |                                   |
|                                   | The access request requires line  |
|                                   | manager approval, then role owner |
|                                   | approval. The Service Officer     |
|                                   | will receive an email once access |
|                                   | has been approved.                |
