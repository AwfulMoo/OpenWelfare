This is a Family and Domestic Violence Interaction Point. If the
customer is with another person, on speaker phone, or already identified
with family and domestic violence concerns, continue with the current
business conversation. Otherwise, go to the Family and domestic violence
procedure to conduct the risk identification and referral process.

This page contains information on creating or updating a customer or
nominee password on a Display on Access (DOA) DOC in Customer First.

Updating or creating a password for a customer or nominee

|                                   | person, on speaker phone, or      |
|                                   | already identified with family    |
|                                   | and domestic violence concerns,   |

|                                   | business conversation. Otherwise, |

| 2                                 | Customer has a password on        |
|                                   | record + Read more ...            |
|                                   |                                   |
|                                   | When a customer contacts the      |
|                                   | password should appear on a       |
|                                   | Display on Access (DOA) DOC. If   |
|                                   | there is only one DOA DOC         |
|                                   | present, it will auto display     |
|                                   | when the Service Officer enters   |
|                                   | the record.                       |
|                                   |                                   |
|                                   | If there are multiple DOA DOCs    |
|                                   | present, the Document List (DL)   |
|                                   | screen will display a list under  |
|                                   | the 'Display on access documents' |
|                                   | heading.                          |
|                                   |                                   |
|                                   | Ask the customer to provide their |
|                                   | password. This is in addition to  |
|                                   | authenticating the customer and   |
|                                   | confirming Proof of Record        |
|                                   | Ownership (PoRO).                 |
|                                   |                                   |
|                                   | If a customer has been            |
|                                   | authenticated via Voice           |
|                                   | Biometrics technology or the      |
|                                   | Interactive Voice Response (IVR)  |
|                                   | process, it is necessary to ask   |
|                                   | the customer to provide their     |
|                                   | password.                         |
|                                   |                                   |
|                                   | Can the customer provide their    |
|                                   | password?                         |
|                                   |                                   |
|                                   | -   Yes, procedure ends here      |
|                                   | -   No, go to Step 3              |
+-----------------------------------+-----------------------------------+
| 3                                 | Customer cannot quote password +  |
|                                   | Read more ...                     |
|                                   |                                   |
|                                   | Call the customer back on a phone |
|                                   | number listed on their customer   |
|                                   | record and follow the regular     |
|                                   | authentication process (without   |
|                                   | the password requirement).        |
|                                   |                                   |
|                                   | If there is no phone number       |
|                                   | recorded, or if the caller states |
|                                   | they do not have access to a      |
|                                   | recorded phone number:            |
|                                   |                                   |
|                                   | The Service Officer should ask    |
|                                   | the caller additional Proof of    |
|                                   | Record Ownership (PoRO) questions |
|                                   | until they are satisfied the      |
|                                   | caller is the record holder. More |
|                                   | PoRO questions should be asked    |
|                                   | than would be done for a regular  |
|                                   | call, due to the increased fraud  |
|                                   | risk.                             |
|                                   |                                   |
|                                   | Where the Service Officer doubts  |
|                                   | they are speaking to the          |
|                                   | customer, information must not be |
|                                   | released or updates made to the   |
|                                   | record. If the Service officer    |
|                                   | suspects the person of            |
|                                   | fraudulently attempting to access |
|                                   | another person's record, the      |
|                                   | Service Officer must report the   |
|                                   | suspected fraud.                  |
|                                   |                                   |
|                                   | Once PoRO is established the      |
|                                   | existing password can be removed, |
|                                   | and a new password added          |
|                                   |                                   |
|                                   | Procedure ends here.              |
+-----------------------------------+-----------------------------------+
| 4                                 | Nominee has a password on         |
|                                   | record + Read more ...            |
|                                   |                                   |
|                                   | When a nominee contacts the       |
|                                   | password will appear on the       |
|                                   | Nominee Relationship Summary      |
|                                   | (NORS) screen.                    |
|                                   |                                   |
|                                   | To access the NORS screen:        |
|                                   |                                   |
|                                   | -   in the customer record go to  |
|                                   |     the Nominee Link Summary      |
|                                   |     (NOLS) screen                 |
|                                   | -   'S'elect the nominee and      |
|                                   |     press [Enter]                 |
|                                   |                                   |
|                                   | From the Nominee Task Selector    |
|                                   | (NOTS) screen select the NORS     |
|                                   | screen and press Enter.           |
|                                   |                                   |
|                                   | The nominee password is           |
|                                   | displayed. Can the nominee quote  |
|                                   | their password?                   |
|                                   |                                   |
|                                   | -   Yes, procedure ends here      |
|                                   | -   No, go to Step 5              |

| 6                                 | Customer requests password on     |
|                                   | their record + Read more ...      |
|                                   |                                   |
|                                   | If a customer requests to have a  |
|                                   | password on their record, ensure  |
|                                   | the customer has been             |
|                                   | authenticated and Proof of Record |
|                                   | Ownership (PoRO) is confirmed.    |
|                                   | This can be done over the         |
|                                   | telephone.                        |
|                                   |                                   |
|                                   | Tell the customer that the        |
|                                   | password should be something that |
|                                   | would not be easy for someone to  |
|                                   | guess, but easy for them to       |
|                                   | remember. They should avoid       |
|                                   | information which could be common |
|                                   | knowledge such as family names    |
|                                   | and birth dates.                  |
|                                   |                                   |
|                                   | If a customer has a PIN and a     |
|                                   | password on their record it is    |
|                                   | necessary to still ask for the    |
|                                   | password.                         |
+-----------------------------------+-----------------------------------+
| 7                                 | Coding password on a Display on   |
|                                   | Access (DOA) DOC + Read more ...  |
|                                   |                                   |
|                                   | Access the Fast Note workflow to  |
|                                   | create a DOA DOC . Complete the   |
|                                   | following fields:                 |
|                                   |                                   |
|                                   | -   Who Contacted                 |
|                                   | -   Service Reason                |
|                                   | -   Date of Receipt               |
|                                   | -   Channel                       |
|                                   | -   Source                        |
|                                   |                                   |
|                                   | Select Auto Text, use Data        |
|                                   | Quality > Updates > Password for  |
|                                   | enquires                          |
|                                   |                                   |
|                                   | Enter the customer's password     |
|                                   | into the Extra Details and fields |
|                                   |                                   |
|                                   | Select Display on access          |
|                                   |                                   |
|                                   | Select the appropriate DOA reason |
|                                   | from the Reason: drop-down list   |
|                                   | and enter the Expiry Date if      |
|                                   | required (delete the Expiry Date  |
|                                   | if not required)                  |
|                                   |                                   |
|                                   | Select Confirm                    |
|                                   |                                   |
|                                   | Select Continue once the template |
|                                   | content is complete               |
|                                   |                                   |
|                                   | If the Password for Enquires Fast |
|                                   | Note has been selected, the       |
|                                   | correct status will be applied on |
|                                   | the DOC.                          |
|                                   |                                   |
|                                   | See Creating, reviewing and       |
|                                   | deleting documents (Fast Note,    |
|                                   | Callbacks, ACTDOCs and DOA DOCs)  |
|                                   | if Fast Note workflow is          |
|                                   | unavailable.                      |
|                                   |                                   |
|                                   | Note: the password does not       |
|                                   | replace or form part of an        |
|                                   | authentication/Proof of Record    |
|                                   | Ownership (PoRO) check, which is  |
|                                   | required at every customer        |
|                                   | contact. It acts only as an extra |
|                                   | identifier. Explain this to       |
|                                   | customers, otherwise they may     |
|                                   | think the password is sufficient  |
|                                   | to be authenticated.              |

| 8                                 | Changing a customer's password +  |
|                                   | Read more ...                     |
|                                   |                                   |
|                                   | -   Code 'DL' in Nxt: this will   |
|                                   |     display the Document List     |
|                                   |     (DL) screen                   |
|                                   | -   Click on the Annotate field   |
|                                   |     of the password DOC           |
|                                   | -   Type the new password as      |
|                                   |     requested by the customer     |
|                                   | -   Press Continue to finalise    |
|                                   |     the annotation                |
+-----------------------------------+-----------------------------------+
| 9                                 | Deleting a password from          |
|                                   | customer's record + Read more ... |
|                                   |                                   |
|                                   | This is a Family and Domestic     |
|                                   | Violence Interaction Point. If    |
|                                   | the customer is with another      |
|                                   | person, on speaker phone, or      |
|                                   | already identified with family    |
|                                   | and domestic violence concerns,   |
|                                   | continue with the current         |
|                                   | business conversation. Otherwise, |
|                                   | go to the Family and domestic     |
|                                   | violence procedure to conduct the |
|                                   | risk identification and referral  |
|                                   | process.                          |
|                                   |                                   |
|                                   | -   Code 'DL' in Nxt: this will   |
|                                   |     display the Document List     |
|                                   |     (DL) screen                   |
|                                   | -   Highlight the 'Password for   |
|                                   |     enquires' DOC and click on    |
|                                   |     the 'More' drop down box and  |
|                                   |     select Expire or Extend       |
|                                   | -   Code a date in the Expiry:    |
|                                   |     field (this can be today's    |
|                                   |     date to take effect           |
|                                   |     immediately)                  |
|                                   | -   Press Continue                |
|                                   | -   DOC now appears on the DL     |
|                                   |     screen as a normal DOC        |
