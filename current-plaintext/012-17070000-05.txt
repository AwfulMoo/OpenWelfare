Key Performance Indicators

Note: claims and/or services referred to DVA prior to the completion of
processing or pending DVA approval will be rejected once the processing
targets are reached.

Maximum number of days for processing claims

Table 1: This table indicates the maximum number of business days
Service Officers should take to process each claim type. Note: this
table reflects current business and is based on the existing Memorandum
of Understanding. It is used as the basis of performance reporting to
DVA.

+-----------------------+-----------------------+-----------------------+
| Type of claim         | Online (paperless)    | Manual                |
+-----------------------+-----------------------+-----------------------+
| Medical               | 2                     | 20                    |
+-----------------------+-----------------------+-----------------------+
| Hospital              | 5                     | 20                    |
+-----------------------+-----------------------+-----------------------+
| Allied                | 2                     | 20                    |
+-----------------------+-----------------------+-----------------------+
| Community Nursing     | 2                     | 20                    |

Keying conventions

Manual data entry keying conventions

Table 2: This table describes keying conventions for manual data entry
fields. Use Control F to locate the relevant field.

Note: not all fields are mandatory. If mandatory information is
unavailable, keying ? in the required field will cause the claim to PEND
for further assessing.

+-----------------------+-----------------------+-----------------------+
| Field                 | Claim type            | Keying convention     |
+-----------------------+-----------------------+-----------------------+
| File #                | All                   | The DVA file number   |
+-----------------------+-----------------------+-----------------------+
| Name                  | All                   | The first 3           |
|                       |                       | characters of the     |
|                       |                       | first given name      |
+-----------------------+-----------------------+-----------------------+
| DOS                   | All                   | The date the service  |
|                       |                       | was rendered - DDMMYY |
|                       |                       | format                |
+-----------------------+-----------------------+-----------------------+
| Item                  | All                   | The item number which |
|                       |                       | identifies the        |
|                       |                       | service rendered.     |
|                       |                       |                       |
|                       |                       | If the service is a   |
|                       |                       | consultation in a     |
|                       |                       | hospital, nursing     |
|                       |                       | home or institution,  |
|                       |                       | documentation should  |
|                       |                       | show the total number |
|                       |                       | of patients seen by   |
|                       |                       | the doctor during     |
|                       |                       | that visit. The       |
|                       |                       | number of patients    |
|                       |                       | seen is keyed as part |
|                       |                       | of the item number,   |
|                       |                       | e.g. '33/2'. This     |
|                       |                       | information is        |
|                       |                       | obtained from the     |
|                       |                       | ‘Description of       |
|                       |                       | Service’ or ‘Item     |
|                       |                       | No.’ section of the   |
|                       |                       | voucher.              |
|                       |                       |                       |
|                       |                       | If more than 1        |
|                       |                       | service is shown on   |
|                       |                       | the voucher, press    |
|                       |                       | [PF2- FULL /BASIC] to |
|                       |                       | allow keying in the   |
|                       |                       | additional ITEM       |
|                       |                       | fields.               |
+-----------------------+-----------------------+-----------------------+
| Amount                | G, P, R, S, A, H      | The amount claimed    |
|                       |                       | for the service       |
|                       |                       | performed. Key as     |
|                       |                       | cents, e.g. $33.35 as |
|                       |                       | '3335'.               |
|                       |                       |                       |
|                       |                       | Do not key for Local  |
|                       |                       | Medical Officer (LMO) |
|                       |                       | attendance items (1   |
|                       |                       | to 98), as the system |
|                       |                       | will automatically    |
|                       |                       | insert the payable    |
|                       |                       | fee.                  |
+-----------------------+-----------------------+-----------------------+
| L                     | G, R, S, A            | The treatment         |
|                       |                       | location.             |
|                       |                       |                       |
|                       |                       | Do not key for LMO    |
|                       |                       | attendance items      |
|                       |                       | (items 1 to 98), as   |
|                       |                       | the system will       |
|                       |                       | automatically insert  |
|                       |                       | the appropriate       |
|                       |                       | location value. For   |
|                       |                       | other items, the      |
|                       |                       | system will           |
|                       |                       | automatically insert  |
|                       |                       | the R (rooms) in this |
|                       |                       | field unless another  |
|                       |                       | value is keyed.       |
|                       |                       |                       |
|                       |                       | If the Visit box is   |
|                       |                       | ticked, key 'V'.      |
|                       |                       |                       |
|                       |                       | If the Hospital box   |
|                       |                       | is ticked, key 'H'.   |
+-----------------------+-----------------------+-----------------------+
| KM                    | G, R, S               | The number of         |
|                       |                       | kilometres travelled  |
|                       |                       | by the provider to    |
|                       |                       | treat the patient.    |
+-----------------------+-----------------------+-----------------------+
| R                     | G, R, S, H            | The disposal          |
|                       |                       | (separation) code     |
|                       |                       | identifying the       |
|                       |                       | patient status at the |
|                       |                       | time of discharge:    |
|                       |                       |                       |
|                       |                       | -   A - discharged by |
|                       |                       |     hospital          |
|                       |                       | -   B - discharged    |
|                       |                       |     own risk          |
|                       |                       | -   C - transferred   |
|                       |                       |     to nursing home   |
|                       |                       | -   D - transfer to   |
|                       |                       |     psychiatric       |
|                       |                       |     hospital          |
|                       |                       | -   E - transfer to   |
|                       |                       |     other hospital    |
|                       |                       | -   F - death with    |
|                       |                       |     autopsy           |
|                       |                       | -   G - death without |
|                       |                       |     autopsy           |
|                       |                       | -   H - transfer to   |
|                       |                       |     other             |
|                       |                       |     accommodation     |
|                       |                       | -   I - type change   |
|                       |                       |     separation        |
|                       |                       | -   R - deceased      |
|                       |                       | -   S - still an      |
|                       |                       |     inpatient         |
|                       |                       | -   W - nursing       |
|                       |                       | -   X - other         |
|                       |                       |     hospital          |
|                       |                       | -   Z - home          |
+-----------------------+-----------------------+-----------------------+
| SD                    | G, P, R, S            | Key 'SD'              |
|                       |                       | (self-determined) if  |
|                       |                       | the notation SD is    |
|                       |                       | present on the        |
|                       |                       | documentation. This   |
|                       |                       | indicates the         |
|                       |                       | provider has          |
|                       |                       | self-determined the   |
|                       |                       | service involved.     |
+-----------------------+-----------------------+-----------------------+
| RSN                   | G, P, A, N            | Key 'P' (pend) if the |
|                       |                       | service line needs to |
|                       |                       | be examined by the    |
|                       |                       | pend Service Officer. |
+-----------------------+-----------------------+-----------------------+
| Ref Pr                | P, R, S, N            | The provider number   |
|                       |                       | of the referring or   |
|                       |                       | requesting provider.  |
|                       |                       |                       |
|                       |                       | Note: if more than 1  |
|                       |                       | service is shown on   |
|                       |                       | the voucher, do not   |
|                       |                       | key this field for    |
|                       |                       | subsequent services.  |
+-----------------------+-----------------------+-----------------------+
| Ref Dt                | P, R, S, N            | The date the provider |
|                       |                       | requested the         |
|                       |                       | service - DDMMYY      |
|                       |                       | format.               |
|                       |                       |                       |
|                       |                       | Note: if more than 1  |
|                       |                       | service is shown on   |
|                       |                       | the voucher, do not   |
|                       |                       | key for subsequent    |
|                       |                       | services.             |
+-----------------------+-----------------------+-----------------------+
| TO                    | R, H                  | The accommodation to  |
|                       |                       | date - DDMMYY format  |
+-----------------------+-----------------------+-----------------------+
| NO                    | R, H                  | The number of days.   |
+-----------------------+-----------------------+-----------------------+
| SCP                   | P                     | The licensed Specimen |
|                       |                       | Collection Point      |
|                       |                       | number. The alpha     |
|                       |                       | prefix and dash are   |

|                       |                       | SPC-021 is keyed as   |
|                       |                       | '021'.                |
+-----------------------+-----------------------+-----------------------+
| Ref To                | S                     | If a referral period  |
|                       |                       | other than 12 months  |
|                       |                       | has been specified,   |
|                       |                       | key the number of     |

|                       |                       |                       |
|                       |                       | Note: if more than 1  |
|                       |                       | service is shown on   |
|                       |                       | the voucher, do not   |
|                       |                       | key for subsequent    |
|                       |                       | services.             |
+-----------------------+-----------------------+-----------------------+
| LSPN                  | S                     | Location Specific     |
|                       |                       | Practice Number       |
|                       |                       | (diagnostic equipment |
|                       |                       | registration number). |
|                       |                       | Alpha prefix needs to |
|                       |                       | be keyed.             |
+-----------------------+-----------------------+-----------------------+
| A/C No                | H                     | Key if an account     |
|                       |                       | reference or invoice  |
|                       |                       | number has been       |
|                       |                       | supplied. Non-numeric |
|                       |                       | characters are not    |

|                       |                       | is keyed as '12345'.  |
+-----------------------+-----------------------+-----------------------+
