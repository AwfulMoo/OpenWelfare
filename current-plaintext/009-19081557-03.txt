| 10                                | Payment level CA will cancel +    |
|                                   | Read more ...                     |
|                                   |                                   |
|                                   | Payment level CA will cancel when |
|                                   | the carer is being assessed for   |
|                                   | CP (Standard) and the carer is    |
|                                   | not:                              |
|                                   |                                   |
|                                   | -   eligible for CP (Standard),   |
|                                   |     and                           |
|                                   | -   getting payment level CA for  |
|                                   |     another carer receiver        |

|                                   | The carer’s claim for CP will be  |
|                                   | rejected and the carer is not     |
|                                   | eligible for CP (XWP) from the    |
|                                   | date from which the claim was     |
|                                   | assessed because their only or    |
|                                   | last CA has been cancelled and    |
|                                   | there is a break in continuous    |
|                                   | payment level CA entitlement.     |
|                                   |                                   |
|                                   | The grandfathering status on the  |
|                                   | GFD screen will automatically     |
|                                   | change to ‘NOT’ and the customer  |
|                                   | will lose their grandfathering    |
|                                   | eligibility for CP (XWP). Using   |
|                                   | Process Direct, the AR result     |
|                                   | will show CA is cancelled and the |
|                                   | change to NOT will be applied     |
|                                   | after the claim is finalised.     |
|                                   |                                   |
|                                   | The carer will automatically be   |
|                                   | advised their CA has been         |
|                                   | cancelled. However, if the CP     |
|                                   | (Standard) claim is finalised on  |
|                                   | the same day the CP (XWP) was     |
|                                   | cancelled, a manual GMANPENMAN    |
|                                   | letter may be generated to advise |
|                                   | the CP rejection. Where CA        |
|                                   | payment is not stripped the same  |
|                                   | day, the SO may delay finalising  |
|                                   | the CP (Standard) claim until the |
|                                   | next day to allow an automatic CP |
|                                   | rejection advice.                 |
|                                   |                                   |
|                                   | Procedure ends here.              |

|                                   | Read more ...                     |
|                                   |                                   |
|                                   | To remain eligible for CP (XWP),  |
|                                   | the carer must have been eligible |
|                                   | for payment level CA continuously |
|                                   | since 19 March 2020. It’s not     |
|                                   | necessary for CA to be paid for   |
|                                   | the same care receiver since 19   |
|                                   | March 2020.                       |

|                                   | CP (XWP) will be cancelled ‘CAC’  |

|                                   | the Grandfather Override (GFD)    |
|                                   | screen changed to ‘NOT’ when the  |
|                                   | carer experiences a break in      |
|                                   | continuous payment level CA       |
|                                   | entitlement due to:               |
|                                   |                                   |
|                                   | -   CA suspension or cancellation |
|                                   | -   a change from payment level   |
|                                   |     CA to HCC only or             |
|                                   | -   a change to zero rate CA due  |
|                                   |     to ETT                        |
|                                   |                                   |
|                                   | Has CP (XWP) cancelled CAC?       |
|                                   |                                   |
|                                   | Yes, go to Step 3                 |
|                                   |                                   |
|                                   | No, and CP (XWP) is suspended or  |
|                                   | cancelled:                        |
|                                   |                                   |
|                                   | -   O6W, go to Step 2             |
|                                   | -   for a reason other than CAC   |
|                                   |     or O6W, consider the reason   |
|                                   |     for suspension or             |
|                                   |     cancellation and alternative  |
|                                   |     income support options if     |
|                                   |     applicable. Procedure ends    |
|                                   |     here                          |

| 2                                 | CP (XWP) is suspended or          |
|                                   | cancelled due to the reason O6W + |
|                                   | Read more ...                     |
|                                   |                                   |
|                                   | CP(XWP) for customers who leave   |
|                                   | Australia temporarily for more    |
|                                   | than 6 weeks will generally       |
|                                   | cancel ‘CAC’ (due to CA being     |
|                                   | suspended O6W).                   |
|                                   |                                   |
|                                   | Manually grandfathered CP (XWP)   |
|                                   | (GRF was entered manually on the  |
|                                   | GFD screen) will be suspended O6W |
|                                   | when CA is suspended with reason  |
|                                   | O6W.                              |
|                                   |                                   |
|                                   | Even if CA is restored on return  |
|                                   | to Australia, there has been a    |
|                                   | break in payment level CA         |
|                                   | entitlement and CP (XWP)          |
|                                   | eligibility is lost.              |
|                                   |                                   |
|                                   | An MFU is produced when CA is     |
|                                   | stopped and CP (XWP) has been     |
|                                   | manually grandfathered. The MFU   |
|                                   | states: Check if customer is      |
|                                   | still eligible for CAR (XWP).     |
|                                   |                                   |
|                                   | In these cases:                   |
|                                   |                                   |
|                                   | -   only staff at the APS5 level  |
|                                   |     or above can make changes to  |
|                                   |     the grandfathering status.    |
|                                   |     For Smart Centre staff, these |
|                                   |     requests meet the criteria    |
|                                   |     for direct referral to a      |
|                                   |     Senior Support Officer (SSO). |
|                                   |     Contact with a Technical Peer |
|                                   |     Support (TPS) is not needed   |
|                                   | -   to make the update, on the    |
|                                   |     Grandfathering Summary (GFS)  |
|                                   |     screen, select the 2020       |
|                                   |     Former WP–CARER line on the   |
|                                   |     GFD screen                    |
|                                   | -   key ‘NOT’ from the date the   |
|                                   |     customer’s only or last       |
|                                   |     payment level CA care         |
|                                   |     receiver was suspended        |
|                                   | -   cancel CP (XWP) from the same |
|                                   |     date                          |
|                                   | -   code the ‘NOT’ status before  |
|                                   |     the customer returns to       |
|                                   |     Australia to prevent CP (XWP) |
|                                   |     from automatically being      |
|                                   |     restored on their return      |
|                                   | -   the customer will need to     |
|                                   |     claim CP (Standard) or an     |
|                                   |     alternative income support    |
|                                   |     payment on return to          |
|                                   |     Australia                     |
|                                   |                                   |
|                                   | Procedure ends here.              |
