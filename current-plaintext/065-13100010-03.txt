| 5                                 | Approve or reject an automatic    |
|                                   | advance + Read more ...           |
|                                   |                                   |
|                                   | Automatic advances calculated as  |
|                                   | either $0 or over $5,000,000 will |
|                                   | be in a Held status for manual    |
|                                   | intervention.                     |
|                                   |                                   |
|                                   | A SME and/or Team Leaders in the  |
|                                   | Aged Care Payments Team (Aged     |
|                                   | Care and Allied Health Branch)    |
|                                   | with the following business roles |
|                                   | have the authority to manually    |
|                                   | approve or reject automatic       |
|                                   | advances:                         |
|                                   |                                   |
|                                   | -   Aged Care Technical Program   |
|                                   |     Advisor                       |
|                                   | -   Business Administrator        |
|                                   | -   IT Administrator              |
|                                   |                                   |
|                                   | For automatic advances calculated |
|                                   | between $0 and $5,000,000:        |
|                                   |                                   |
|                                   | -   these will be in a Calculated |
|                                   |     status on the Advances        |
|                                   |     summary screen for a service  |
|                                   | -   there is one business day to  |
|                                   |     manually reject these         |
|                                   |     automatic advances            |
|                                   | -   the system will automatically |
|                                   |     approve these advances if     |
|                                   |     they are not manually         |
|                                   |     rejected during this period   |
|                                   |                                   |
|                                   | All fields are optional unless    |
|                                   | marked with an asterisk.          |
|                                   |                                   |
|                                   | To approve an automatic advance:  |
|                                   |                                   |
|                                   | -   on the Automatic advances for |
|                                   |     approval details screen,      |
|                                   |     select Approve                |
|                                   | -   when the Approve? alert       |
|                                   |     notification displays,        |
|                                   |     complete the Notes field.     |
|                                   |     Document clear and accurate   |
|                                   |     notes detailing the action    |
|                                   |     taken                         |
|                                   | -   select Yes to display a       |
|                                   |     message stating ‘Automatic    |
|                                   |     advance has been successfully |
|                                   |     approved’                     |
|                                   | -   select Close to return to the |
|                                   |     Automatic advances for        |
|                                   |     approval screen. Procedure    |
|                                   |     ends here                     |
|                                   |                                   |
|                                   | To reject an automatic advance in |
|                                   | Held status:                      |
|                                   |                                   |
|                                   | -   on the Automatic advances for |
|                                   |     approval details screen,      |
|                                   |     select Reject                 |
|                                   | -   when the Reject? alert        |
|                                   |     notification displays,        |
|                                   |     complete the Notes field.     |
|                                   |     Document clear and accurate   |
|                                   |     notes detailing the action    |
|                                   |     taken                         |
|                                   | -   select Yes to display a       |
|                                   |     message stating ‘Automatic    |
|                                   |     advance has been successfully |
|                                   |     rejected. If you want to pay  |
|                                   |     an advance to this service    |
|                                   |     for [claim month], go to the  |
|                                   |     Advances summary page and     |
|                                   |     create a manual advance’      |
|                                   | -   select Close to return to the |
|                                   |     Automatic advances for        |
|                                   |     approval screen. Procedure    |
|                                   |     ends here                     |
|                                   |                                   |
|                                   | To reject an automatic advance    |
|                                   | with a Calculated status:         |
|                                   |                                   |
|                                   | -   select the Main menu icon >   |
|                                   |     Service Search > key the      |
|                                   |     service’s details and         |
|                                   |     Search > from the search      |
|                                   |     results, select the Service   |
|                                   |     NAPS ID hyperlink to go to    |
|                                   |     the ACSP Service in Context   |
|                                   |     home page > in the Quick find |
|                                   |     search bar, key Advances > Go |
|                                   | -   when the Advances summary     |
|                                   |     screen displays, select the   |
|                                   |     relevant advance record’s     |
|                                   |     Status from the Advances      |
|                                   |     summary table                 |
|                                   | -   on the Advance details        |
|                                   |     screen, select Reject         |
|                                   | -   when the Reject? alert        |
|                                   |     notification displays,        |
|                                   |     complete the Notes field.     |
|                                   |     Document clear and accurate   |
|                                   |     notes detailing the action    |
|                                   |     taken                         |
|                                   | -   select Submit to display a    |
|                                   |     message stating ‘Automatic    |
|                                   |     advance has been successfully |
|                                   |     rejected. If you want to pay  |
|                                   |     an advance to this service    |
|                                   |     for [claim month], go to the  |
|                                   |     Advances summary page and     |
|                                   |     create a manual advance’      |
|                                   | -   select Close to return to the |
|                                   |     Advances summary screen.      |
|                                   |     Procedure ends here           |
