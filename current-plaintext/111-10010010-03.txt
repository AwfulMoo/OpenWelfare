Classifying escalated images

This table describes how Classification Support queue staff classify
images escalated to Classification Support in the National Queue.

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Access the National Queue and     |
|                                   | view escalated images + Read more |
|                                   | ...                               |
|                                   |                                   |
|                                   | To access the National Queue,     |
|                                   | open a Process Direct session     |
|                                   | using the desktop icon.           |
|                                   |                                   |
|                                   | Note: to receive a Classification |
|                                   | Support work item in Process      |
|                                   | Direct, Classification Support    |
|                                   | Officers must have the            |
|                                   | appropriate skill tag(s) coded in |
|                                   | ESSentials.                       |
|                                   |                                   |
|                                   | -   Select Ready to allocate the  |
|                                   |     next available work package   |
|                                   |     that the Classification       |
|                                   |     Officer has the capability to |
|                                   |     complete. The Work Package    |
|                                   |     details screen will display   |
|                                   | -   Check the image has been      |
|                                   |     escalated properly. See:      |
|                                   |     Classifying Centrelink        |
|                                   |     digital images via the        |
|                                   |     National Queue                |
|                                   |                                   |
|                                   | If the escalation was:            |
|                                   |                                   |
|                                   | -   suitable, go to Step 2        |
|                                   | -   not suitable, classify the    |
|                                   |     image and provide feedback to |
|                                   |     the Service Officer Procedure |
|                                   |     ends here                     |
+-----------------------------------+-----------------------------------+
| 2                                 | Process the escalated image +     |
|                                   | Read more ...                     |
|                                   |                                   |
|                                   | If the CRN cannot be found, go to |
|                                   | Step 3                            |
|                                   |                                   |
|                                   | If the record belongs to a        |
|                                   | Restricted Access customer see    |
|                                   | the Resources page for more       |
|                                   | information on how to action the  |
|                                   | image                             |
|                                   |                                   |
|                                   | If the image contains             |
|                                   | inappropriate material            |
|                                   |                                   |
|                                   | -   Make sure a Team Leader has   |
|                                   |     been notified and reviewed    |
|                                   |     the image                     |
|                                   | -   Request an image deletion     |
|                                   | -   Pending deletion, classify    |
|                                   |     document as F9. Finalise work |
|                                   |     item using Document Decision  |
|                                   |     option Image requires         |
|                                   |     administrative processing.    |
|                                   |     Click Save. A message will    |
|                                   |     appear to say that the work   |
|                                   |     item has been unassigned      |
|                                   |                                   |
|                                   | Go to Step 5 if:                  |
|                                   |                                   |
|                                   | -   the document does not relate  |
|                                   |     to a customer's record (e.g.  |
|                                   |     office phone bill, customer   |
|                                   |     complaint), or                |
|                                   | -   there is not enough           |
|                                   |     information on the document   |
|                                   |     to identify the CRN, or       |
|                                   | -   a Centrelink document, and a  |
|                                   |     rescan request is not         |
|                                   |     applicable                    |
|                                   |                                   |
|                                   | The Resources page contains       |
|                                   | classification details for form   |
|                                   | codes and unstructured documents. |
+-----------------------------------+-----------------------------------+
| 3                                 | No CRN on the image + Read more   |
|                                   | ...                               |
|                                   |                                   |
|                                   | Does the image contain a phone    |

|                                   |     Adding a customer to the      |
|                                   |     system. Once a customer       |
|                                   |     record has been found or      |

|                                   | -   No, go to Step 5              |

|                                   | more ...                          |
|                                   |                                   |
|                                   | In the Documents section, add the |

|                                   | type: field if applicable.        |
|                                   |                                   |
|                                   | Select the Document Decision      |
|                                   | Image checked and ready to be     |
|                                   | processed. Click Save. A message  |
|                                   | will appear to say that the work  |
|                                   | item has been unassigned.         |
|                                   | Procedure ends here.              |

|                                   | in the document to:               |
|                                   |                                   |
|                                   | -   locate a customer record      |

|                                   | -   create a new record           |
|                                   |                                   |
|                                   | Do not use orphan processing when |
|                                   | the customer or nominee has       |
|                                   | provided name, address and        |
|                                   | contact information. The customer |
|                                   | and/or nominee is to be contacted |
|                                   | to obtain more information.       |
|                                   |                                   |
|                                   | Administrative documents          |
|                                   |                                   |
|                                   | -   do not relate to any customer |
|                                   |     (e.g. invoices)               |
|                                   | -   are part of a process where   |
|                                   |     the documents cannot be       |
|                                   |     attached to a record (e.g.    |
|                                   |     tip offs or inappropriate     |
|                                   |     images)                       |
|                                   | -   there is a business reason    |
|                                   |     why the documents cannot be   |
|                                   |     attached to a record but they |
|                                   |     still need processing (e.g. a |
|                                   |     single page contains details  |
|                                   |     for multiple customers)       |
|                                   |                                   |

|                                   | for Classification Support in the |

|                                   | specific actions for some forms.  |

|                                   | document may also be an Orphan    |

|                                   | cannot be identified.             |
|                                   |                                   |
|                                   | If the document:                  |

|                                   | does not relate to a customer's   |
|                                   | record (e.g. office phone bill,   |

|                                   | to a customer's record            |
|                                   |                                   |
|                                   | -   Classify the document using   |
|                                   |     the form code, or most        |

|                                   | -   If the image is completely    |
|                                   |     blank with no identifying     |
|                                   |     features, classify as F9, and |
|                                   |     then click Save. A message    |
|                                   |     will say the work item has    |
|                                   |     been unassigned               |
|                                   | -   The Resources page has more   |
|                                   |     information about the         |
|                                   |     appropriate team to refer the |
|                                   |     document/s to                 |
|                                   |                                   |
|                                   | relates to a person's record but  |
|                                   | there is not enough information   |
|                                   | on the document to identify the   |
|                                   | customer's CRN                    |
|                                   |                                   |
|                                   | -   Classify the document using   |
|                                   |     the form code, or most        |
|                                   |     appropriate UNS code if no    |
|                                   |     form code is available.       |
|                                   |     Finalise the work item as     |
|                                   |     Image contains insufficient   |
|                                   |     information to classify and   |
|                                   |     click Save                    |
|                                   | -   If the image is completely    |
|                                   |     blank with no identifying     |

|                                   |     will say the work item has    |
|                                   |     been unassigned               |
|                                   | -   Go to Step 6                  |
