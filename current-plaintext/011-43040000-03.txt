Order of entry

Note: if the order of entry is not followed, the system may return the
error message '1-TOO MANY'. In this case, delete the adjustments and
start again in the preferred order.

Table 1: This table describes the order for entry of bulk bill
adjustments to claim lines for each patient.

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Keying + Read more ...            |
|                                   |                                   |
|                                   | Process any adjustment involving  |
|                                   | over keying, including            |
|                                   | transferring, the service line(s) |
|                                   | with adjustment details.          |
+-----------------------------------+-----------------------------------+
| 2                                 | Insertion of service lines + Read |
|                                   | more ...                          |
|                                   |                                   |
|                                   | Process any adjustments involving |
|                                   | the insertion of service lines.   |
+-----------------------------------+-----------------------------------+
| 3                                 | Deletion of service lines + Read  |
|                                   | more ...                          |
|                                   |                                   |
|                                   | Process any deletions of service  |
|                                   | lines.                            |
+-----------------------------------+-----------------------------------+
| 4                                 | Suppress + Read more ...          |
|                                   |                                   |
|                                   | Suppression will usually be done  |
|                                   | in a separate adjustment claim.   |
+-----------------------------------+-----------------------------------+

Recall an original bulk bill LDA claim

Table 9: This table describes how Service Officers recall an original
bulk bill LDA claim.

Note: the claim ID (to be adjusted) and date of lodgement (DOL) are
equal to the original claim.

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Key control line + Read more ...  |
|                                   |                                   |
|                                   | Key control line 'DHSI,PROVIDER   |
|                                   | NO,CLAIM ID,DOL=DDMMYY'.          |
|                                   |                                   |
|                                   | DHSC screen returns displaying    |
|                                   | the message 'CARD' indicating the |
|                                   | vouchers within the claim are     |
|                                   | displayed in card number order.   |
|                                   |                                   |
|                                   | Note:                             |
|                                   |                                   |
|                                   | -   when the DHSI control line is |
|                                   |     entered in the first          |
|                                   |     instance, adjustment          |
|                                   |     processing is initiated and   |
|                                   |     the date of lodgement used on |
|                                   |     the control line is the DOL   |
|                                   |     of the original claim         |
|                                   | -   key A at the end of the       |
|                                   |     control line to display the   |
|                                   |     claim in alphabetical order - |
|                                   |     the return message 'ALPHA' is |
|                                   |     displayed                     |
|                                   | -   after initiation of           |
|                                   |     adjustment processing, the    |
|                                   |     DOL shown on the control line |
|                                   |     of the DHSC screen is the new |
|                                   |     adjustment claim DOL          |

Delete a DHSC line

Table 12: This table describes how Service Officers delete a line on the
DHSC screen for bulk bill LDA claims.

Note: the FEE and BENEFIT fields contain zero amounts and H is displayed
in the PAYEE field, indicating that this line will be recorded for
history purposes only, as a total overpayment.

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Line deletion + Read more ...     |
|                                   |                                   |
|                                   | -   Key T in the A field          |
|                                   | -   Press [Enter]                 |
|                                   |                                   |
|                                   | The system returns the message    |
|                                   | 'LINE DLTD' and displays:         |
|                                   |                                   |
|                                   | -   the original deleted line -   |
|                                   |     all fields are protected,     |
|                                   |     except the A field which      |
|                                   |     contains an O                 |
|                                   | -   a restated line - all fields  |
|                                   |     are protected, except the A   |
|                                   |     field which contains an R     |

Insert a line during adjustment

Table 13: This table describes how Service Officers insert a line during
a bulk bill adjustment.

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Activate insert function + Read   |
|                                   | more ...                          |
|                                   |                                   |
|                                   | Key I over the R in the A field   |
|                                   | of any restated line.             |
|                                   |                                   |
|                                   | Note: I must be keyed over the R  |
|                                   | otherwise the insert function     |
|                                   | will not be activated.            |
+-----------------------------------+-----------------------------------+
| 2                                 | New line insertion + Read more    |
|                                   | ...                               |
|                                   |                                   |
|                                   | Press [Enter].                    |
|                                   |                                   |
|                                   | The system returns the message    |
|                                   | 'INSERT', together with a new     |
|                                   | line that enables new details to  |
|                                   | be keyed.                         |
|                                   |                                   |
|                                   | Note: an N appears in the A field |
|                                   | to indicate a new line.           |
+-----------------------------------+-----------------------------------+
| 3                                 | Enter details + Read more ...     |
|                                   |                                   |
|                                   | Key details in the following      |
|                                   | fields:                           |
|                                   |                                   |
|                                   | -   ITEM                          |
|                                   | -   DOS                           |
|                                   | -   BN ASSG                       |
|                                   |                                   |
|                                   | Note: these fields are            |
|                                   | highlighted and must be keyed.    |
|                                   | All usual keying fields in the    |
|                                   | new line remain unprotected.      |
|                                   |                                   |
|                                   | The system:                       |
|                                   |                                   |
|                                   | -   repeats the card number and   |
|                                   |     patient name from the         |
|                                   |     restated line and these       |
|                                   |     fields are protected          |
|                                   | -   inserts PAYEE Code 6 and      |
|                                   |     zeros in the FEE and BENEFIT  |
|                                   |     fields, these fields are      |
|                                   |     protected                     |
|                                   | -   leaves the PI, RSN and TX     |
|                                   |     fields blank and not required |
|                                   |     when inserting a line         |
+-----------------------------------+-----------------------------------+
| 4                                 | Finalise + Read more ...          |
|                                   |                                   |
|                                   | -   key new line details          |
|                                   | -   press [Enter]                 |
|                                   |                                   |
|                                   | The system returns:               |
|                                   |                                   |
|                                   | -   restated line                 |
|                                   | -   new line                      |
|                                   | -   any messages that relate to   |
|                                   |     the accuracy of the new       |
|                                   |     details                       |
|                                   |                                   |
|                                   | Note: a new line can also be      |
|                                   | inserted over an 'N' once a new   |
|                                   | line has been generated.          |
+-----------------------------------+-----------------------------------+
