Customer First screen information

Table 1

+-----------------------------------+-----------------------------------+
| Item                              | Description                       |
+-----------------------------------+-----------------------------------+
| 1                                 | History Activity Detail (HAY) +   |
|                                   | Read more ...                     |
|                                   |                                   |
|                                   | This screen displays the details  |
|                                   | of a historical, current or       |
|                                   | future activity.                  |
|                                   |                                   |
|                                   | To view activities on this        |
|                                   | screen, select an activity from:  |
|                                   |                                   |
|                                   | -   the HAL screen, or            |
|                                   | -   one of the AL, ES or FAL      |
|                                   |     screens and enter 'HAY' into  |
|                                   |     the next field                |
|                                   |                                   |
|                                   | Within an activity, the HAY       |
|                                   | screen provides the following     |
|                                   | information:                      |
|                                   |                                   |
|                                   | Activity ID - the Activity        |
|                                   | Identification number, or AMR.    |
|                                   | All activities are allocated a    |
|                                   | number sequentially when they are |
|                                   | created                           |
|                                   |                                   |
|                                   | Activity type - displays the      |
|                                   | service reason or system and the  |
|                                   | expanded literal for the activity |
|                                   | type/code. E.g. FTB/Manual        |
|                                   | Followup. This is followed by     |
|                                   | additional service reason codes   |
|                                   | where applicable. A maximum of    |
|                                   | three can be displayed            |
|                                   |                                   |
|                                   | Status - displays the status of   |
|                                   | the activity. It also displays    |
|                                   | 'Mandatory' if the activity must  |
|                                   | be processed and cannot be        |
|                                   | cancelled                         |
|                                   |                                   |
|                                   | Review reason - displays the      |
|                                   | reason for the review if the      |
|                                   | activity is a review              |
|                                   |                                   |
|                                   | Source - displays the code that   |
|                                   | defines the type of source        |
|                                   | document for the activity         |
|                                   |                                   |
|                                   | Resubmit Date - date a held       |
|                                   | activity is due to come off hold  |
|                                   |                                   |
|                                   | Date of Receipt - displays the    |
|                                   | Date of Receipt of the source     |
|                                   | (i.e. Date of phone call, date    |
|                                   | form received, etc.)              |
|                                   |                                   |
|                                   | Next Auto Action - displays the   |
|                                   | next action for a review or       |
|                                   | activity. An example of this is   |
|                                   | RM1 (reminder one) this would     |
|                                   | mean a FMI (Form Issue) has been  |
|                                   | done                              |
|                                   |                                   |
|                                   | Date of Auto Action - the date    |
|                                   | the action is scheduled to occur  |
|                                   |                                   |
|                                   | Keywords - displays all the       |
|                                   | keywords associated with the      |
|                                   | activity. This field will not     |
|                                   | display if there are no keywords  |
|                                   | present                           |
|                                   |                                   |
|                                   | TDIs - To Do Indicators - TDIs    |
|                                   | can require both manual and       |
|                                   | automatic action before they      |
|                                   | allow the activity to be          |
|                                   | completed. This field will not    |
|                                   | display if no TDI is present      |
|                                   |                                   |
|                                   | Notes - displays the comments or  |
|                                   | notes relating to a specific      |
|                                   | review or activity for a          |
|                                   | customer. This field will not     |
|                                   | display if there are no           |
|                                   | comments/notes present            |
|                                   |                                   |
|                                   | Status Changes - this section     |
|                                   | contains the details of the       |
|                                   | status changes the activity has   |
|                                   | undergone, including:             |
|                                   |                                   |
|                                   | -   Date of the last status       |
|                                   |     update                        |
|                                   | -   Status of the activity at     |
|                                   |     every change                  |
|                                   | -   Logon ID associated with the  |
|                                   |     status change of the          |
|                                   |     activity. This Logon ID       |
|                                   |     includes users and Batch2     |
|                                   |     processing logons eg. *B2     |
|                                   | -   Region - the Office Code      |
|                                   |     relevant to the user involved |
|                                   |     with the activity             |
|                                   |                                   |
|                                   | Work Allocation History - this    |
|                                   | section contains the details of   |
|                                   | the work allocation changes the   |
|                                   | activity has undergone,           |
|                                   | including:                        |
|                                   |                                   |
|                                   | -   Date of the last work         |
|                                   |     allocation change. It changes |
|                                   |     with each update made to the  |
|                                   |     allocation of the activity    |
|                                   | -   Status of the activity at     |
|                                   |     every change in its           |
|                                   |     allocation                    |
|                                   | -   Logon ID associated with the  |
|                                   |     work allocation changes of    |
|                                   |     the activity. This Logon ID   |
|                                   |     includes users and Batch2     |
|                                   |     processing logons eg. *B2     |
|                                   | -   Workgroup name relevant for   |
|                                   |     this activity. This is used   |
|                                   |     for Automatic Work Allocation |
|                                   |     purposes                      |
|                                   | -   Position name relevant for    |
|                                   |     this activity. This is used   |
|                                   |     for Automatic Work Allocation |
|                                   |     purposes                      |
|                                   | -   Region - the Office Code      |
|                                   |     relevant to the user involved |
|                                   |     with the activity             |
|                                   |                                   |
|                                   | Tasks - displays all the tasks    |
|                                   | started and completed as part of  |
|                                   | the activity                      |
|                                   |                                   |
|                                   | Events - displays the events and  |
|                                   | consequences of the activity      |
|                                   |                                   |
|                                   | Ripples and Activity Source -     |
|                                   | displays the source or the        |
|                                   | destination of the rippled        |
|                                   | activity, including the CRN and   |
|                                   | environment indicator. This will  |
|                                   | display one of two messages       |
|                                   | depending on if the activity was  |
|                                   | sourced from a ripple or was      |
|                                   | sending a ripple to another       |
|                                   | destination. This field will only |
|                                   | be displayed if a ripple is       |
|                                   | present                           |
|                                   |                                   |
|                                   | Source ID - displays the source   |
|                                   | activity ID from which this       |
|                                   | activity was created. E.g. When a |
|                                   | debt is automatically created     |
|                                   | this field will display, when an  |
|                                   | MFU is created this field will    |
|                                   | display. This is for the ease of  |
|                                   | referencing back to a source      |
|                                   | activity ID. This field will only |
|                                   | be displayed if a source activity |
|                                   | ID is present                     |
|                                   |                                   |
|                                   | Package Participants - displays   |
|                                   | the CRN and Activity ID of the    |
|                                   | participants involved in the      |
|                                   | assessment package. It also       |
|                                   | displays by virtue of an asterisk |
|                                   | the package owner. An assessment  |
|                                   | package consists of all           |
|                                   | activities involved in a single   |
|                                   | assessment. All the participants  |
|                                   | are displayed on the HAY screen   |
|                                   | of all participants (except       |
|                                   | children who do not have a HAY    |
|                                   | screen) including the Package     |
|                                   | owner. This field will only       |
|                                   | display if a package is present   |
|                                   |                                   |
|                                   | Errors and Warnings - displays    |
|                                   | all errors and warnings affecting |
|                                   | the activity. This includes the   |
|                                   | number and literal message. This  |
|                                   | field will only display if an     |
|                                   | error or warning is present       |
|                                   |                                   |
|                                   | Other Users - contains            |
|                                   | information on users who have     |
|                                   | updated the activity, but have    |
|                                   | not caused a status or work       |
|                                   | allocation change, and have not   |
|                                   | started or completed tasks on the |
|                                   | record                            |
+-----------------------------------+-----------------------------------+

|                                   |     information including package |

|                                   | Enter the child CRN and child     |
|                                   | Activity ID (AMR) to view the     |
|                                   | following:                        |
|                                   |                                   |
|                                   | -   Package ID                    |
|                                   | -   CRN                           |
|                                   | -   Link type                     |
|                                   | -   AMR                           |
+-----------------------------------+-----------------------------------+
