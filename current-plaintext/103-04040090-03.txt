Reissuing rejected payments

Table 2: This table describes the procedure to reissue rejected payments
to customers and PPL customers where the PPL delivery is by the
customer's employer. Portions of this process can only be completed by
the Paid Parental Leave Employer Processing Team (PPLEPT).

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Family assistance customer + Read |
|                                   | more ...                          |
|                                   |                                   |
|                                   | Before reissuing the rejected     |
|                                   | family assistance payments, check |
|                                   | the Payment Details (PD) screen   |
|                                   | to ensure both Family Tax Benefit |
|                                   | (FTB) Part A and Part B           |
|                                   | components have been rejected.    |
|                                   |                                   |
|                                   | -   If the entire payment has     |
|                                   |     been rejected 'This payment   |
|                                   |     has been rejected on ...'     |
|                                   |     will display                  |
|                                   | -   If only part of the payment   |
|                                   |     has been rejected the message |
|                                   |     will specify an amount        |
|                                   |     returned. For example,        |
|                                   |     $180.00 has been rejected on  |
|                                   |     28 May 2011                   |
|                                   |                                   |
|                                   | Have both family assistance       |
|                                   | components been rejected?         |
|                                   |                                   |
|                                   | -   Yes, go to Step 3             |
|                                   | -   No, code a manual review for  |
|                                   |     3 working days to check both  |
|                                   |     payments have been rejected.  |
|                                   |     Procedure ends here until     |
|                                   |     manual review elapses in 3    |
|                                   |     days, then go to Step 2       |
+-----------------------------------+-----------------------------------+
| 2                                 | Check record after manual review  |
|                                   | has elapsed + Read more ...       |
|                                   |                                   |
|                                   | Have both payments rejected in    |

+-----------------------------------+-----------------------------------+
| 3                                 | Confirm customer's entitlement to |
|                                   | the payment + Read more ...       |
|                                   |                                   |
|                                   | -   If payment is correct and can |
|                                   |     be reissued, go to Step 4     |
|                                   | -   If the payment looks          |
|                                   |     incorrect or the system will  |
|                                   |     not allow the payment to be   |
|                                   |     reissued, staff must follow   |
|                                   |     normal escalation procedures  |
|                                   |     to have this record escalated |
|                                   |     to ICT for investigation or   |
|                                   |     confirmation that the payment |
|                                   |     can be reissued               |
|                                   | -   If the full amount of the     |
|                                   |     arrears are not to be         |
|                                   |     reissued to the customer, the |
|                                   |     record must be escalated via  |
|                                   |     mySupport to have the payment |
|                                   |     blocked and then the payment  |
|                                   |     the customer is eligible for  |
|                                   |     must be reissued via the      |
|                                   |     Payment Adjustment (PYAJ)     |
|                                   |     screen. See Using the Payment |
|                                   |     Adjustment (PYAJ) screen      |
