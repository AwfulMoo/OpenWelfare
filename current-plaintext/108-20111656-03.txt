This page contains the steps needed to check the status of a customer on
the MCR, as well as their Link authorities, to determine the state of
STP data flow. For further explanation regarding the fields on the MCR
Person Summary screen, please refer to Resources.

Escalating issues

MCR Person summary for a customer

Table 1

+-----------------------------------+-----------------------------------+
| Step                              | Details                           |
+-----------------------------------+-----------------------------------+
| 1                                 | Load Customer Summary screen in   |
|                                   | Process Direct + Read more ...    |
|                                   |                                   |
|                                   | Load the Customer Summary screen  |
|                                   | in the Process Direct workspace.  |
|                                   |                                   |
|                                   | Search for the customer’s CRN and |
|                                   | open the customer’s record into   |
|                                   | context.                          |
+-----------------------------------+-----------------------------------+
| 2                                 | View a customer's MCR Person      |
|                                   | summary + Read more ...           |
|                                   |                                   |
|                                   | To view a customer’s MCR Person   |
|                                   | summary, either:                  |
|                                   |                                   |
|                                   | -   use Task Selector to select:  |
|                                   |     ‘MCR - Mutual Client          |
|                                   |     Register’                     |
|                                   | -   use Super Key code: ‘MCR’     |
|                                   |                                   |
|                                   | The Mutual Client Register (MCR)  |
|                                   | screen will appear, displaying    |
|                                   | two blocks of information. The    |
|                                   | top block is the Person summary   |
|                                   | for the customer.                 |
+-----------------------------------+-----------------------------------+
| 3                                 | Status + Read more ...            |
|                                   |                                   |
|                                   | If an attempt has been made to    |
|                                   | add the customer to the MCR, the  |
|                                   | Person summary block will have    |
|                                   | information populated, and the    |
|                                   | status will be one of the         |
|                                   | following:                        |
|                                   |                                   |
|                                   | -   Active                        |
|                                   | -   Inactive (or Inactive –       |
|                                   |     Deceased)                     |
|                                   | -   Unavailable                   |
|                                   | -   Pending                       |
|                                   |                                   |
|                                   | Is the Person summary block       |
|                                   | blank, but the customer has a     |
|                                   | current benefit or debt?          |
|                                   |                                   |
|                                   | -   Yes, go to Step 8             |
|                                   | -   No, go to Step 4              |
+-----------------------------------+-----------------------------------+
| 4                                 | Active on MCR + Read more ...     |
|                                   |                                   |
|                                   | -   If the Person is Active on    |
|                                   |     the MCR, go to Table 2.       |
|                                   | -   Otherwise, go to Step 5.      |
+-----------------------------------+-----------------------------------+
| 5                                 | Unavailable on MCR + Read more    |
|                                   | ...                               |
|                                   |                                   |
|                                   | If the Person is Unavailable on   |
|                                   | the MCR, it may be a result of    |
|                                   | personal details not matching     |
|                                   | with ATO. Proceed to check with   |
|                                   | customer whether the following    |
|                                   | details on their Centrelink       |
|                                   | record are complete, up to date   |
|                                   | and spelled correctly:            |
|                                   |                                   |
|                                   | -   Name (1 current + 2           |
|                                   |     historical)                   |
|                                   | -   Address (current + historical |
|                                   |     addresses)                    |
|                                   | -   Phone number (current MOB +   |
|                                   |     current HOM + current WRK,    |
|                                   |     then MOB historical)          |
|                                   | -   Email address (1 current + 2  |
|                                   |     historical)                   |
|                                   |                                   |
|                                   | Any updates / change of           |
|                                   | circumstances should trigger the  |
|                                   | MCR process to determine if the   |
|                                   | person needs to be added to the   |
|                                   | MCR with the ATO.                 |
|                                   |                                   |
|                                   | -   If any changes are made to    |
|                                   |     any of the above details, the |
|                                   |     system should automatically   |
|                                   |     retry adding the customer to  |
|                                   |     the MCR, in which case, go to |
|                                   |     Step 6                        |
|                                   | -   If all details are correct    |
|                                   |     and no changes are made to    |
|                                   |     any of the customer’s         |
|                                   |     personal details, go to Step  |
|                                   |     8                             |
+-----------------------------------+-----------------------------------+
| 6                                 | Retry MCR + Read more ...         |
|                                   |                                   |
|                                   | After a few minutes, click the    |
|                                   | ‘Refresh’ button (top-right of    |
|                                   | Person summary table), and check  |
|                                   | if any new rows are added, or if  |
|                                   | any Person status updates.        |
|                                   |                                   |
|                                   | Did anything change in the Person |
|                                   | summary table?                    |
|                                   |                                   |
|                                   | -   Yes, go to Step 7             |
|                                   | -   No, go to Step 8              |
+-----------------------------------+-----------------------------------+
| 7                                 | Retry Pending on MCR + Read more  |
|                                   | ...                               |
|                                   |                                   |
|                                   | If any Person rows have Pending   |
|                                   | status, wait a few minutes and    |
|                                   | try clicking the ‘Refresh’ button |
|                                   | again.                            |
|                                   |                                   |
|                                   | If status updates to Active, go   |
|                                   | to Table 2.                       |
|                                   |                                   |
|                                   | Otherwise if it stays at Pending  |
|                                   | or changes to Unavailable, check  |
|                                   | for any STP/MCR related notices   |
|                                   | in NNU. If there is an active     |
|                                   | issue, relay to the customer that |
|                                   | there may be a delay due to the   |
|                                   | outage, and to try again later.   |
|                                   | Procedure ends here.              |
|                                   |                                   |
|                                   | If there are no active related    |
|                                   | notices in NNU, go to Step 8.     |
+-----------------------------------+-----------------------------------+
| 8                                 | Further investigation + Read more |
|                                   | ...                               |
|                                   |                                   |
|                                   | If the customer is expecting to   |
|                                   | receive ongoing STP data, inform  |
|                                   | them that further investigation   |
|                                   | will be required. Explain that    |
|                                   | not all customers will be able to |
|                                   | be matched (eg a young person who |
|                                   | does not have a Tax File Number), |
|                                   | so while further investigations   |
|                                   | will be conducted, it may still   |
|                                   | result in them not receiving STP  |
|                                   | data.                             |
|                                   |                                   |
|                                   | Go to the mySupport form to       |
|                                   | request support.                  |
|                                   |                                   |
|                                   | In the User Defined Request       |
|                                   | Description, be sure to list as   |
|                                   | much information as possible,     |
|                                   | including:                        |
|                                   |                                   |
|                                   | -   The customer’s MCR Person     |
|                                   |     status – Pending, Inactive,   |
|                                   |     Unavailable, or blank         |
|                                   | -   Reasoning as to why customer  |
|                                   |     would expect to be Active on  |
|                                   |     MCR, e.g., currently          |
|                                   |     receiving benefit and needs   |
|                                   |     to regularly report earnings  |
|                                   | -   If any of the customer’s      |
|                                   |     personal details were updated |
|                                   |     (as part of Step 5)           |
|                                   | -   Date and time of enquiry      |
|                                   | -   Preferred method for          |
|                                   |     contacting the customer       |
+-----------------------------------+-----------------------------------+

MCR Link summary for a customer

Table 2

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Confirm Active status + Read more |
|                                   | ...                               |
|                                   |                                   |
|                                   | After completing the above steps  |
|                                   | and confirming the Person is      |
|                                   | Active on the MCR, go to Step 2.  |
+-----------------------------------+-----------------------------------+
| 2                                 | Link authorities + Read more ...  |
|                                   |                                   |
|                                   | The bottom block of the Mutual    |
|                                   | Client Register (MCR) screen is   |
|                                   | the Link summary for the          |
|                                   | customer. At least one Link       |
|                                   | authority should be listed,       |
|                                   | however it is possible for a      |
|                                   | person to have more than one Link |
|                                   | authority type, listed for        |
|                                   | different periods.                |
|                                   |                                   |
|                                   | Is at least one Link authority    |
|                                   | listed?                           |
|                                   |                                   |
|                                   | -   Yes, go to Step 3             |
|                                   | -   No, go to Step 6              |
+-----------------------------------+-----------------------------------+
| 3                                 | Link authority status + Read more |
|                                   | ...                               |
|                                   |                                   |
|                                   | Check the status of each Link     |
|                                   | authority listed. Status will be  |
|                                   | one of the following              |
|                                   |                                   |
|                                   | -   Active                        |
|                                   | -   Inactive                      |
|                                   | -   Unavailable                   |
|                                   | -   Pending                       |
+-----------------------------------+-----------------------------------+
| 4                                 | No STP data being received + Read |
|                                   | more ...                          |
|                                   |                                   |
|                                   | If at least one Link authority    |
|                                   | listed is Active on the MCR and   |
|                                   | the customer is concerned about   |
|                                   | no STP data coming through,       |
|                                   | explain this may be because:      |
|                                   |                                   |
|                                   | the customer is not currently     |
|                                   | working                           |
|                                   |                                   |
|                                   | the customer’s employer(s) does   |
|                                   | not report income to ATO using    |
|                                   | STP, or                           |
|                                   |                                   |
|                                   | the employer may be late          |
|                                   | submitting their payroll data to  |
|                                   | the ATO                           |
|                                   |                                   |
|                                   | -   Check with person how         |
|                                   |     regularly their employer      |
|                                   |     processes pays to determine   |
|                                   |     if perhaps the employer has   |
|                                   |     not sent data through yet     |
|                                   |                                   |
|                                   | If the customer has verified none |
|                                   | of these scenarios apply to their |
|                                   | circumstances, check for any      |
|                                   | STP/MCR related notices in NNU.   |
|                                   | If there is an active issue,      |
|                                   | relay to the customer that there  |
|                                   | may be a delay due to the outage, |
|                                   | and to try again later.           |
|                                   |                                   |
|                                   | Are there any active related      |
|                                   | notices in NNU?                   |
|                                   |                                   |
|                                   | -   Yes, go to Step 5             |
|                                   | -   No, go to Step 6              |
+-----------------------------------+-----------------------------------+
| 5                                 | Update Link authority + Read more |
|                                   | ...                               |
|                                   |                                   |
|                                   | If any Link authority row has     |
|                                   | Pending status, wait a few        |
|                                   | minutes and try clicking the      |
|                                   | ‘Refresh’ button (top-right of    |
|                                   | Link summary table). The status   |
|                                   | should either update to Active or |
|                                   | Unavailable.                      |
|                                   |                                   |
|                                   | If it updates to Active, go to    |
|                                   | Step 4.                           |
|                                   |                                   |
|                                   | If it stays at Pending or changes |
|                                   | to Unavailable, check for any     |
|                                   | STP/MCR related notices in NNU,   |
|                                   | relay to the customer that there  |
|                                   | may be a delay due to the outage, |
|                                   | and to try again later. Procedure |
|                                   | ends here.                        |
|                                   |                                   |
|                                   | If there are no active related    |
|                                   | notices in NNU, go to Step 6      |
+-----------------------------------+-----------------------------------+
| 6                                 | No active related notices in      |
|                                   | NNU + Read more ...               |
|                                   |                                   |
|                                   | If the customer is expecting to   |
|                                   | receive ongoing STP data, explain |
|                                   | that further investigation will   |
|                                   | be required. Before proceeding to |
|                                   | mySupport form to request         |
|                                   | support, first check with the     |
|                                   | customer whether they have ever   |
|                                   | received STP data previously.     |
|                                   |                                   |
|                                   | In the User Defined Request       |
|                                   | Description, be sure to list as   |
|                                   | much information as possible,     |
|                                   | including:                        |
|                                   |                                   |
|                                   | -   The customer’s MCR Person     |
|                                   |     status is Active              |
|                                   | -   Reasoning as to why customer  |
|                                   |     would expect to be receiving  |
|                                   |     STP data – eg currently       |
|                                   |     receiving benefit and needs   |
|                                   |     to regularly report earnings  |
|                                   | -   The status for each Link      |
|                                   |     authority – Active, Pending,  |
|                                   |     Inactive, Unavailable, or     |
|                                   |     blank                         |
|                                   | -   If the customer has received  |
|                                   |     STP data previously           |
|                                   | -   If at least one Link          |
|                                   |     authority status is Active:   |
|                                   |     confirmation that customer    |
|                                   |     has verified the scenarios in |
|                                   |     Step 4 do not apply to their  |
|                                   |     circumstances                 |
|                                   | -   Date and time of enquiry      |
|                                   | -   Preferred method for          |
|                                   |     contacting the customer       |
+-----------------------------------+-----------------------------------+

Escalating issues

Table 3

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Escalation + Read more ...        |
|                                   |                                   |
|                                   | For issues not covered in the     |
|                                   | above MCR Link summary or MCR     |
|                                   | Person summary sections, first    |
|                                   | check for any STP/MCR related     |
|                                   | notices in NNU.                   |
|                                   |                                   |
|                                   | If there are no active related    |
|                                   | notices in NNU, proceed to        |
|                                   | mySupport form and Request        |
|                                   | Support.                          |
|                                   |                                   |
|                                   | In the User Defined Request       |
|                                   | Description, be sure to list as   |
|                                   | much information as possible.     |
|                                   |                                   |
|                                   | The information should include:   |
|                                   |                                   |
|                                   | -   Reasoning behind the          |
|                                   |     mySupport request - e.g.      |
|                                   |     Person summary table not      |
|                                   |     displaying correctly          |
|                                   | -   Date and time of enquiry      |
|                                   | -   Preferred method for          |
|                                   |     contacting the customer       |
|                                   |                                   |
|                                   | The information may include:      |
|                                   |                                   |
|                                   | -   The customer’s MCR Person     |
|                                   |     status - e.g. what it is vs   |
|                                   |     what is expected              |
|                                   | -   The Link authority status -   |
|                                   |     e.g. what it is vs what is    |
|                                   |     expected                      |
|                                   | -   Any other details relevant to |
|                                   |     the issue                     |
