| Item                              | Description                       |
+-----------------------------------+-----------------------------------+
| 1                                 | Arrears generated                 |
|                                   | incorrectly/stop payment + Read   |
|                                   | more ...                          |
|                                   |                                   |
|                                   | When an activity is finalised by  |
|                                   | a Service Officer and incorrect   |
|                                   | arrears are generated, the        |
|                                   | request to prevent these arrears  |
|                                   | from being issued must be         |
|                                   | referred to the Information       |
|                                   | Communication Technology (ICT)    |
|                                   | Service Desk before close of      |
|                                   | business.                         |
|                                   |                                   |
|                                   | Due to the urgency of this        |
|                                   | request, Service Officers must    |
|                                   | escalate this directly to their   |
|                                   | local Service Support Officer     |
|                                   | (SSO) or Team Leader (TL) for     |
|                                   | immediate action.                 |
|                                   |                                   |
|                                   | The SSO or TL will take the       |
|                                   | necessary action to have the      |
|                                   | arrears stopped by referring the  |
|                                   | matter via mySupport to the ICT   |
|                                   | Service Desk.                     |
|                                   |                                   |
|                                   | The Resources page contains a     |
|                                   | link to the mySupport for         |
|                                   | requesting a stop to an incorrect |
|                                   | payment.                          |
+-----------------------------------+-----------------------------------+
| 2                                 | Notification of death             |
|                                   | escalations + Read more ...       |
|                                   |                                   |
|                                   | When an activity relating to      |
|                                   | coding the death of a customer or |
|                                   | partner is escalated to an SSO,   |
|                                   | the escalation must be actioned   |
|                                   | within 24 hours.                  |
|                                   |                                   |
|                                   | If the escalation cannot be       |
|                                   | finalised at the first contact    |
|                                   | (e.g. has to be referred to ICT), |
|                                   | it must be reviewed within 14     |
|                                   | days.                             |
|                                   |                                   |
|                                   | The action taken must be clearly  |
|                                   | documented on the record using    |
|                                   | the Escalated to SSO Fast Note.   |
|                                   | Select Auto text > Generic        |
|                                   | Escalation.                       |
+-----------------------------------+-----------------------------------+
| 3                                 | First Contact Service Offer       |
|                                   | (FCSO) escalations + Read more    |
|                                   | ...                               |
|                                   |                                   |
|                                   | TPS/FLW contacts escalated to     |
|                                   | SSOs with FCSO/SAC in the Enq 1   |
|                                   | field that have been referred to  |
|                                   | ICT are to be finalised on the    |
|                                   | 15th day from the original        |
|                                   | contact.                          |
+-----------------------------------+-----------------------------------+
| 4                                 | If action is required, who        |
|                                   | completes the work? + Read more   |
|                                   | ...                               |
|                                   |                                   |
|                                   | -   If the escalation is          |
|                                   |     finalised on day one, and the |
|                                   |     required work is within       |
|                                   |     technical scope of the        |
|                                   |     Service Officer, the Service  |
|                                   |     Officer is to complete        |
|                                   | -   If the escalation is not      |
|                                   |     finalised on day one, the SSO |
|                                   |     is to complete all necessary  |
|                                   |     work on the customer's        |
|                                   |     record. The exception to this |
|                                   |     is for New Claim escalations  |
|                                   |     from Processing staff. These  |
|                                   |     should be referred back to    |
|                                   |     the original Service Officer  |
|                                   |     for completion                |
|                                   | -   On the rare occasion the      |
|                                   |     customer needs to be called   |
|                                   |     back, this is the             |
|                                   |     responsibility of the Service |
|                                   |     Officer and must be clearly   |
|                                   |     stated on the escalation      |
|                                   |     notes                         |
|                                   | -   If coding or workaround is    |
|                                   |     not standard procedure, it    |
|                                   |     must be completed by the SSO  |
|                                   |     on all occasions              |
+-----------------------------------+-----------------------------------+
| 5                                 | System issue referrals + Read     |
|                                   | more ...                          |
|                                   |                                   |
|                                   | Prior to referring a system       |
|                                   | problem to ICT, SSOs must check   |
|                                   | the following:                    |
|                                   |                                   |
|                                   | -   Operational Blueprint         |
|                                   | -   Operational Messaging         |
|                                   | -   SharePoint                    |
|                                   | -   Network News Update (NNU)     |
|                                   | -   ICT Service Desk              |
|                                   | -   mySupport                     |
|                                   |                                   |
|                                   | TPS/FLW staff who encounter       |
|                                   | escalations for system issues     |
|                                   | that are supported by instruction |
|                                   | in a Network News Update (NNU)    |
|                                   | can escalate the issue directly   |
|                                   | to ICT. The relevant referral     |
|                                   | point will be monitored within    |
|                                   | the NNU.                          |
|                                   |                                   |
|                                   | TPS/FLW must include the Incident |
|                                   | (SD) number in the escalation so  |
|                                   | the SSO is aware that the         |
|                                   | activity has already been         |
|                                   | forwarded to ICT.                 |
|                                   |                                   |
|                                   | System issues encountered by FCC  |
|                                   | processing staff or Families      |
|                                   | telephony staff undertaking off   |
|                                   | phone claim processing must be    |
|                                   | referred by a SSO to the National |
|                                   | ICT Referral Officer (NIRO), and  |
|                                   | not directly to the ICT Service   |
|                                   | Desk.                             |
|                                   |                                   |
|                                   | The Resources page contains more  |
|                                   | information and a link to         |
|                                   | mySupport.                        |
+-----------------------------------+-----------------------------------+
| 6                                 | Policy escalations and reference  |
|                                   | inconsistencies + Read more ...   |
|                                   |                                   |
|                                   | If SSOs encounter a complex       |
|                                   | policy enquiry, it should be      |
|                                   | referred to the relevant Level 2  |
|                                   | helpdesk.                         |
|                                   |                                   |
|                                   | If an SSO encounters unclear      |
|                                   | reference material, feedback      |
|                                   | should be provided to the         |
|                                   | relevant team.                    |
|                                   |                                   |
|                                   | See the Resources page for links  |
|                                   | to the relevant helpdesks.        |
+-----------------------------------+-----------------------------------+
| 7                                 | Documenting escalations + Read    |
|                                   | more ...                          |
|                                   |                                   |
|                                   | Any escalation that SSOs are      |
|                                   | unable to finalise on day one     |
|                                   | must be updated following the     |
|                                   | comment guidelines below.         |
|                                   |                                   |
|                                   | At a minimum it is expected that  |
|                                   | the following be included in the  |
|                                   | Comments field within the         |
|                                   | escalation and escalation DOC on  |
|                                   | the customer's record:            |
|                                   |                                   |
|                                   | -   Action taken to resolve the   |
|                                   |     issue                         |
|                                   | -   Resources used                |
|                                   | -   Detailed information if there |
|                                   |     is action required by the     |
|                                   |     Service Officer               |
|                                   | -   If the escalation is not      |
|                                   |     finalised on day one, details |
|                                   |     of why                        |
|                                   | -   If referred to ICT, the date  |
|                                   |     referred and the SD number    |
|                                   |                                   |
|                                   | A DOC must be created in the      |
|                                   | customer's record if information  |
|                                   | is received and not processed on  |
|                                   | Day One. SSOs are to check the    |
|                                   | customer's record and action if   |
|                                   | required. For more information    |
|                                   | about DOC standards see:          |
|                                   |                                   |
|                                   | -   Online Document Recording     |
|                                   |     (ODR)                         |
|                                   | -   Creating documents (Fast      |
|                                   |     Note, Callbacks and ACTDOCs)  |
+-----------------------------------+-----------------------------------+
| 8                                 | Escalation finalisation and       |
|                                   | follow up guidelines + Read more  |
|                                   | ...                               |
|                                   |                                   |
|                                   | All escalations are to be         |
|                                   | finalised in cases where 'hand    |
|                                   | off' has occurred and/or no       |
|                                   | further action is required in the |
|                                   | Smart Centre. This includes when: |
|                                   |                                   |
|                                   | -   an ACTDOC has been sent to a  |
|                                   |     Service Centre/decision maker |
|                                   |     (DM), or                      |
|                                   | -   an NNU confirms there is no   |
|                                   |     current work around for that  |
|                                   |     particular issue, and         |
|                                   | -   the issue will be fixed in a  |
|                                   |     future systems release as     |
|                                   |     advised by ICT or an NNU. All |
|                                   |     details must be documented on |
|                                   |     the customer's record         |
|                                   |                                   |
|                                   | System issues escalated to ICT    |
|                                   | that are affecting the customer's |
|                                   | entitlement are to be followed up |
|                                   | by SSOs each week or on the       |
|                                   | customer's Entitlement Period End |
|                                   | Date (EPED) and notes added to    |
|                                   | the escalation in TSC and the DOC |
|                                   | on the customer file              |
+-----------------------------------+-----------------------------------+
| 9                                 | Prioritising escalations + Read   |
|                                   | more ...                          |
|                                   |                                   |
|                                   | Smart Centre telephony SSOs       |
|                                   | should search the escalation      |
|                                   | inbox in the Technical Support    |
|                                   | Centre (TSC) firstly focussing on |
|                                   | escalations generated by          |
|                                   | Technical Peer Support (TPS)      |
|                                   | staff from their own site and     |
|                                   | prioritise by selecting the       |
|                                   | oldest unselected escalations     |
|                                   | first. Urgent/payment affecting   |
|                                   | enquiries should be prioritised.  |
|                                   |                                   |
|                                   | Smart Centre processing SSOs      |
|                                   | should search the escalation      |
|                                   | inbox in TSC focussing on         |
|                                   | escalations generated by TPS/FLW  |
|                                   | staff in sites within their       |
|                                   | footprint and prioritise by       |
|                                   | selecting the oldest unselected   |
|                                   | escalations first. Urgent/payment |
|                                   | affecting enquiries should be     |
|                                   | prioritised.                      |
|                                   |                                   |
|                                   | All SSOs with the relevant skills |
|                                   | are to provide daily assistance   |
|                                   | with handling escalations lodged  |
|                                   | under the Direct referral         |
|                                   | process.                          |
|                                   |                                   |
|                                   | The Resources page contains a     |
|                                   | link to a taskcard to help with   |
|                                   | locating escalations in TSC.      |
+-----------------------------------+-----------------------------------+
