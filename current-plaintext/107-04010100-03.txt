| 3                                 | Requesting the refund + Read more |
|                                   | ...                               |
|                                   |                                   |
|                                   | Use the Request for Refund (RFR)  |
|                                   | process to recall whole payments  |
|                                   | from bank accounts, including     |
|                                   | joint accounts. Part of a payment |
|                                   | cannot be recalled.               |
|                                   |                                   |
|                                   | To recall a payment:              |
|                                   |                                   |
|                                   | Go to the Payment Summary (PS)    |
|                                   | screen                            |
|                                   |                                   |
|                                   | Key 'F' next to the payment line  |
|                                   |                                   |
|                                   | Press [Enter]                     |
|                                   |                                   |
|                                   | The Request for Refund/Recall     |
|                                   | (RFR) screen will display the:    |
|                                   |                                   |
|                                   | -   details of the payment to be  |
|                                   |     refunded                      |
|                                   | -   type and amount of the        |
|                                   |     payment, and                  |
|                                   | -   payment destination           |
|                                   |                                   |
|                                   | Check the details to make sure    |
|                                   | the correct payment is being      |
|                                   | refunded:                         |
|                                   |                                   |
|                                   | -   Check the Reason for Refund   |
|                                   |     field. This should default to |
|                                   |     3RD (third party)             |
|                                   | -   Complete the Source and DOR   |
|                                   |     fields                        |
|                                   | -   In the Finished? (Y/Q/N)      |
|                                   |     field, code 'Y', or code 'Q'  |
|                                   |     if the activity is to be      |
|                                   |     Quality On Line (QOL) checked |
|                                   | -   Press [Enter]                 |
|                                   | -   The system will display the   |
|                                   |     following warning:            |
|                                   |     W030PL - Is this an RFR for a |
|                                   |     deceased customer? If Yes,    |
|                                   |     code DEA screen first         |
|                                   |                                   |
|                                   | Press [Enter]                     |
|                                   |                                   |
|                                   | On the PS screen, the payment     |
|                                   | will have the code RFR            |
|                                   |                                   |
|                                   | The system will issue Request for |
|                                   | Refund letter. It will ask for    |
|                                   | the return of the payment(s)      |
|                                   |                                   |
|                                   | 'S'elect the payment to show the  |
|                                   | Payment Details (PD) screen. The  |
|                                   | payment will be marked 'THIS      |
|                                   | PAYMENT WAS REQUESTED TO BE       |
|                                   | REFUNDED ON DD MM YYYY'           |
|                                   |                                   |
|                                   | This starts the review process.   |
|                                   | The system will:                  |
|                                   |                                   |
|                                   | -   set up a Manual Follow-up     |
|                                   |     (MFU) on the Future Activity  |
|                                   |     List (FAL) screen for 14 days |
|                                   |     in the future with the        |
|                                   |     keyword REFUND                |
|                                   | -   After 14 days a second        |
|                                   |     activity with the keyword     |
|                                   |     REFUND creates with a due     |
|                                   |     date of 14 days. If payment   |
|                                   |     is returned the review is     |
|                                   |     cancelled                     |
|                                   |                                   |
|                                   | To reissue the payment, see       |
|                                   | Replacing direct credit, New      |
|                                   | Payments Platform (NPP) or Real   |
|                                   | Time Gross Settlement (RTGS)      |
|                                   | payments at a customer's request. |

Removing the debt when placed on an incorrect account

Table 2:

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Has the debt been raised on the   |
|                                   | customer's record? + Read more    |
|                                   | ...                               |
|                                   |                                   |
|                                   | -   Yes, go to Step 2             |
|                                   | -   No, go to Step 3              |
+-----------------------------------+-----------------------------------+
| 2                                 | Reducing the determined debt to   |
|                                   | zero + Read more ...              |
|                                   |                                   |
|                                   | If a third party received the     |
|                                   | incorrect payment, raise a debt   |
|                                   | against the third party.          |
|                                   |                                   |
|                                   | Finalise 'no debt' (FND) the      |
|                                   | customer's debt.                  |
|                                   |                                   |
|                                   | 'S'elect the relevant Debt Id     |
|                                   | from the Debt List (OPDL) screen  |
|                                   |                                   |
|                                   | Go to the Debt Amount (OPDA)      |
|                                   | screen and complete the           |
|                                   | following:                        |
|                                   |                                   |
|                                   | -   Reason Changed $ field, key   |
|                                   |     the reason why the debt       |
|                                   |     amount is reducing to zero,   |
|                                   |     (for acceptable values: see   |
|                                   |     the Resources page)           |
|                                   | -   Recalculated amount:$ field   |
|                                   |     key '0' to zero the debt      |
|                                   |     amount                        |
|                                   | -   Press [Enter] twice to        |
|                                   |     confirm the recalculated Nil  |
|                                   |     debt amount                   |
|                                   |                                   |
|                                   | On the Assessment Finalisation    |
|                                   | (OPAF) screen, in the Finished?   |
|                                   | (Y/Q/N) field key:                |
|                                   |                                   |
|                                   | -   'Y' to complete, or           |
|                                   | -   'Q' if the activity is to be  |
|                                   |     Quality On Line (QOL) checked |
|                                   | -   Press [Enter] to finalise     |
|                                   |                                   |
|                                   | Record the details on a DOC       |
|                                   |                                   |
|                                   | Go to Step 4.                     |
+-----------------------------------+-----------------------------------+
| 3                                 | Finalising the undetermined debt  |
|                                   | activity as finalised - 'no debt' |
|                                   | (FND) + Read more ...             |
|                                   |                                   |
|                                   | The undetermined debt activity on |
|                                   | the customer record must be       |
|                                   | finalised - 'no debt' (FND).      |
|                                   | Raise the debt against the third  |
|                                   | party:                            |
|                                   |                                   |
|                                   | 'S'elect the undetermined debt    |
|                                   | activity DMN/NDB on the Activity  |
|                                   | List (AL) screen                  |
|                                   |                                   |
|                                   | Go to the Modify/Assess Debt      |
|                                   | (OPMA) screen and complete the    |
|                                   | following:                        |
|                                   |                                   |
|                                   | -   Calculated Amount $ field,    |
|                                   |     key '0'                       |
|                                   | -   FND Reason field, key the     |
|                                   |     reason why the debt amount is |
|                                   |     zero, (for acceptable values  |
|                                   |     see the Resources page)       |
|                                   | -   Addtnl Reference No field,    |
|                                   |     key the third party CRN       |
|                                   | -   Press [Enter] twice to        |
|                                   |     confirm the recalculated debt |
|                                   |     amount                        |
|                                   |                                   |
|                                   | On the Assessment Finalisation    |
|                                   | (OPAF) screen, in the Finished?   |
|                                   | (Y/Q/N) field key:                |
|                                   |                                   |
|                                   | -   'Y' or                        |
|                                   | -   'Q' if the activity is to be  |
|                                   |     Quality On Line (QOL) checked |
|                                   | -   Press [Enter]                 |
|                                   |                                   |
|                                   | Record the details on a DOC.      |
