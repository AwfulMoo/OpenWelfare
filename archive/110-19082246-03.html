6

Cancel product with reason TRN + Read more ...

Before proceeding, consider the source of the funds used to purchase the new income stream.

For example, the customer may have commuted their old income stream and purchased a new product using the commuted funds as well as funds from another source. An update to other assets may be needed. If funds cannot be accounted for, contact the customer to discuss.

When making updates to other assets, apply the DOV as the date the changes occurred.

Go to SUPS:

  • make a screenshot of the product information within the Income Stream table
    • note the product name from SUPI and each line from the purchase date onwards from SUPV
  • return to SUPI and change the Product Cancellation Reason to TRN
  • on the SUPV screen:
    • change the date of event (DOV) to the purchase date of the new product
    • if the line cannot be changed, add a new line with the required DOV
  • delete any lines with a DOV after the purchase date

Once the original product is cancelled, recode the new product, including the deleted lines. Note: if an AISR update for the old product has been applied post-purchase because of a delay in processing, do not re-code lines from this update to the new product.

For help with coding a new product, see:

  • Adding or updating an account-based income stream
  • Adding or updating a market-linked income stream

Apply category 2 income stream provider data + Read more ...

  • Event date: code today’s date
  • Gross income amount:
    • the Gross Annual Payment field in the provider data (code as dollars and cents)
    • if there is a more recent update than the current account balance date in the provider data, retain the gross income amount on the record
  • Income Frequency: 'ANN-Annually'
  • Original Purchase Price: code exactly as stated in the Provider Data (including cents if stated)
  • 1 July account balance for August reviews only. Code as whole dollars only
    • If completing a February review post 1 July, use the previous financial year 1 July account balance reduced by $1
      Note: increase by $1 if the error ‘E853CQ - Gross Income not equal to or greater than’ is triggered
    • If 1 July balance has already been coded for the current Financial year, e.g. for a February review coded post 1 July, retain the existing coded balance unless incurring a E853CQ - Gross Income not equal to or greater than error
      Note: when incurring E853CQ, delete the existing coding and recode that DOV with the new 1 July balance, before applying the provider data from today
  • Current account balance: code as whole dollars only. Note: if there is a more recent update than the current account balance date in the provider data, retain the balance from the more recent date of event
  • Latest commutation: code as needed to make sure the total commutations in the provider data and the total commutations on the customer's record match
  • Commutation date:
    • 31 January for February reviews
    • 31 July for August reviews, or
    • the date the product was cancelled on the customer's record (for example, today’s date) if the product has been cancelled

Go to Step 17.

Was the income streams purchased prior to 1 January 2015? + Read more ...

Products purchased prior to 1 January 2015 may be grandfathered when the customer has had continuous entitlement to:

  • An Income Support Payment, or
  • Commonwealth Seniors Health Card (CSHC) from 31 December 2014

For a pre-2015 product, check if the product remains as Category 2 or if a grandfathered override reason needs to be coded. See Adding or updating an account-based income stream.

18

Is the update cancelling the benefit? + Read more ...

An assessment of the current ongoing entitlement is required for customers only in receipt of a CSHC and/or CA. A cancellation may be caused by the addition of deemed income to out-of-date Adjusted Taxable Income (ATI) information.

Is this a CSHC or CA only customer and the benefit is cancelling upon finalisation?

  • Yes, go to Step 19
  • No, go to Step 22

19

Check the ATI reference/financial tax year and income type is up-to-date + Read more ...

Where the reference/financial tax year is not within the two years prior to the current tax year, an update of the ATI is needed before cancellation:

  • For CSHC, the reference tax year is found on the SHC Income Details (SHID) screen. See Cancellation of Commonwealth Seniors Health Card (CSHC)
  • For CA only customers check financial tax year used on the Carer Allowance Adjusted Taxable Income (CAATI) screen. This is only available in Process Direct. See Carer Allowance (CA) income test – determining reference tax year and assessable income components

The reference/financial tax year is usually the previous tax year. This is the tax year immediately before the current tax year. If a customer is unable to provide the previous tax year, then it is the year immediately prior.

An estimate of the current year’s income is only required when the reference year exceeds the income threshold, and the income has changed.

Where the reference tax year is before the previous year, then a check with the customer is needed to determine if:

  • their circumstances have changed, and
  • an update of the reference year is needed

Is the reference/financial tax year a current year estimate?

  • Yes, go to Step 22
  • No, go to Step 20

20

Request an update of the ATI + Read more ...

Make 2 genuine attempts to contact the customer via phone to request a Notice of Assessment (NOA) for either of the two previous tax years. Determine if they are a non-lodger, and/or provide the opportunity to provide a current year estimate.

Was contact successful?

  • Yes, go to Step 20
  • No, request the Notice of Assessment or current income via a Request for Information (RFI). Make sure the correct benefit type is selected to allow for notice of assessment (NOA) to display as an option. See Requesting Information (CLK)

Put the activity on hold:

  • Select the Status icon
  • Select Edit icon
  • Change the On Hold Reason to Customer to Provide Information
  • Change the date to when information is expected to be returned, see Reasonable timeframe
  • Select the Notes icon and annotate the notes for the activity to state that documents have been requested. See Requesting information (CLK)
  • Exit Process Direct

Has the NOA or response been provided by the end of the reasonable time?

  • Yes, go to Step 21
  • No, go to Step 22

See Lodgement of the Notice of Assessment (NOA) and other evidence of income for Commonwealth Seniors Health Card (CSHC) and Carer Allowance (CA) income test - acceptable evidence of income

Note: the advice can be verbal.

Partial commutation + Read more ...

Market-linked income streams can only have partial commutations in very limited circumstances:

  • within the first 6 months of the purchase date if the product’s purchase date is before 20 September 2007 and it is a non-commutation funded income stream
  • to pay a superannuation contribution surcharge
  • to pay an excess contribution tax
  • income stream split following a divorce property settlement

For more information, see Commutation of asset-test exemption (ATE) income streams.

To determine if the partial commutation is allowed, escalate to the FINS Level 2 Policy Help Desk.

      System issue: the sum of the Previous Commutations and Latest Commutation cannot be more than the purchase price. Code the Latest Commutation field as the original purchase price less the total of previous commutations. If this was coded in a prior review, leave the Current commutation amount and date fields blank.
  • Commutation date only if there is a related latest commutation:
    • 31 January for February reviews
    • 31 July for August reviews, or
    • the cancellation date on the customer’s record if the previous product entry has been cancelled

Note: to maintain an accurate asset assessment for the customer, DOV coding for extra assets purchased as the result of a commutation must align with the DOV of the relevant balance update.

Go to Step 6.

Apply market-linked income stream provider data + Read more ...

Check the provider name, product reference number and ABN from the provider data in the Income Stream (SUPS, SUPI, SUPV) table.

If the information on the customer's record is different, select the Edit icon and update as needed.

Action the handoff by adding a new entry to SUPV by selecting the Add new line icon to the right of the Income Stream Details table.

Refer to the provider data and update:

  • Event date: code today’s date
  • Market linked gross amount: the Gross Annual Payment field in the provider data, code as dollars and cents. Note: if there is a more recent update than the current account balance date in the provider data, retain the gross income amount on the record
  • 1 July account balance for August reviews only. Code as whole dollars only
    • If completing a February review post 1 July, use the previous financial year 1 July account balance reduced by $1
      Note: increase by $1 if the error ‘E853CQ - Gross Income not equal to or greater than’ is triggered
    • If 1 July balance has already been coded for the current Financial year, e.g. for a February review coded post 1 July, retain the existing coded balance unless incurring a E853CQ - Gross Income not equal to or greater than error
      Note: when incurring E853CQ, delete the existing coding and recode that DOV with the new 1 July balance, before applying the provider data from today
  • Current account balance: code as whole dollars only. Note: if there is a more recent update than the current account balance date in the provider data, retain the balance from the more recent date of event
  • Latest commutation: code as needed, as dollars and cents. Leave blank if no commutation coding is needed
    • Commutation/s greater than the purchase price:
      The partial commutation amount can be more than the original purchase price when the underlying investments perform very well.
      System issue: the sum of the Previous Commutations and Latest Commutation cannot be more than the purchase price. Code the Latest Commutation field as the original purchase price less the total of previous commutations. If this was coded in a prior review, leave the Current commutation amount and date fields blank.
  • Commutation date only if there is a related latest commutation:
    • 31 January for February reviews
    • 31 July for August reviews, or
    • the cancellation date on the customer’s record if the previous product entry has been cancelled

If the market-linked income stream is in its final year with a payment factor of 1, go to Step 4.

For all others, go to Step 6.

4

Market-linked income stream (MLI) in the final year + Read more ...

Due to a known system issue, the minimum and maximum calculation during the final year for MLI is not correct (see NNU 12279).

If the error “E022SN - The Gross Annual Income Amount must be between minimum ####.## and maximum ####.##” is displayed, it requires the following workaround:

  • Open the existing SUPI entry by selecting the Edit icon next to the product listed under the Income Streams (SUPS, SUPI, SUPV) table
  • Note the product reference from the SUPI screen
  • Cancel the current product by selecting OTH ‘Other’ in the Product Cancellation Reason field
  • Save
  • Check the data shows nil income and asset value in the Income Stream Details (SUPV) table
  • Make sure the latest line on SUPV is today’s DOV

Code new entry on Other Assets (OAS) screen:

  • Type: OTH
  • Description: SUPS MLI PRN (insert product reference number)
  • DOV: today
  • Value: If the product is not Section 9A/B/BA compliant, code the current account balance as the value. If the product is Section 9A/B/BA compliant code half the current account balance as the value.
  • Ownership: 100% CUS

Code new entry on Other Income (OINS) screen

  • Type: OTH
  • Description: SUPS MLI PRN (insert product reference number)
  • Reference: leave blank
  • DOV: today
  • Value: code assessable income: In the final period, this is calculated by the Gross Income less the deductible which is ((purchase price – commutations) / relevant number)
  • Frequency: ANN

Once finalised, DOC the record (see Resources for a DOC template) and code a manual review for the expiry date. A manual review must be set to expire all market linked products in the final year.

Go to Step 5

Finalise the activity + Read more ...

The activity in Process Direct needs completing so the related Integrity Review System (IRS) review is finalised.

To finalise the activity:

  • Go to the Sap Warnings and Errors (SWE) screen via the Super Key
  • Select Assess
  • Review the entitlements (ELD) screen for any unexpected debts or arrears. For example, no debts or arrears would be expected for AIS/MLI products if the only DOV actioned was for today’s date

Has an unexpected debt or arrears been raised for the activity, inconsistent with any historical changes and date of events (DOV)?

  • Yes, raise an interaction via Roxy DA. Procedure ends here
  • No,
    • Select Finish when satisfied with outcome
    • Add Notes about the activity in the display box
    • Finalise the activity

If a debt shell is raised because of the activity, and the review is completed in Customer First, add keyword AISRDEBT to the debt shell.

Add a Display on Access DOC using Fast Note to show that commutations to date (with breakdown) have been investigated and assessed as allowable or not allowable.
