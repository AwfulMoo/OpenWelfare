This page contains details of the Service Strategy (PQSS) screen, which displays the Service Components that make up the Service Strategy for the customer. A Service Component can be expanded to view the associated Service Actions, which in turn can be expanded to view the Service Details. The hierarchy for these is indicated by the indentation. IDs are displayed in brackets on the Screen.

The Service Strategy (PQSS) screen will display + Read more ...

  • Date Last Profiled: is a display only field, and shows the date profiles were last refreshed.

  • DOE column. This is display only, and shows Date of Effect for the Service Component status.

Updating Service Action Status

Table 2: This table describes the process for updating Service Action Status.

Step

Action

1

To update Service Action status + Read more ...

Service Action status can be updated, corrected or deleted via the Service Action STS: field. Only one status change is allowed at one time.

2

Multiple Service Actions + Read more ...

The status of multiple Service Actions can be updated within the same Service Component in a single activity. Do not update the status of multiple Service Actions in more than one Service Component within one activity. This prevents the system attempting to stamp a single activity with more than one review ID.

There are three options to change a Service Action status:

  • I - Insert/Update: - To be used for entering the new status, the activity will be stamped with the Integrated Review System (IRS) review identification (ID). The results from this activity will be attributed to the applicable Service Update in IRS.
  • C - Correct: - To be used for correcting an existing status when the existing data was entered incorrectly. The correct action should not be used if the intention is to update existing information. If there is a change in a status, a new occurrence of the status should be updated.
  • D - Delete: - The delete action is only to be used when the data should never have been recorded.

3

Coding + Read more ...

  • Code 'I', 'C' or 'D' in the action field (to the left of the Service Action)
  • Change the Service Action: field Status column appropriately
  • Complete the Source: and DOR: fields
  • Press [Enter]

Note: if a Service Action status is being deleted, a 'confirm deletion' field will display. Code 'Y' (yes) and 'N' (no).

4

Changing Service Action status + Read more ...

Changing a customer's Service Action status will create a XXX/SVC activity (XXX=system) on the Activity List (AL) screen.

  • To add an additional activity to the Service Update, go to Step 5
  • If no other activities are to be attributed, procedure ends here

5

Create an additional activity for a Service Update + Read more ...

If there is another activity that is associated with the Service Component it needs to be added to the Service Update so that the results correctly reflect the outcomes of the Service Update.

Has the activity been started?

Note: an activity should only be created if there are no activities on the record that can be selected.

  • Yes, go to Step 6
  • No, go to Step 7

6

Where the activity has been started + Read more ...

Navigate to the PQSS screen.

  • Code an 'A' in the Action: field against the relevant Service Action
  • The started activity will be stamped with the review ID and all updates made will be attributed to the selected Service Update
  • Procedure ends here

7

Where no activity has been started + Read more ...

Navigate to the PQSS screen.

  • Code an 'A' in the Action: field against the relevant Service Action
  • An XXX/SVC activity will be created (XXX=system)

If the results of the activity attributed to the selected Service Update, are not to be attributed go to the Activity List (AL) screen and 'C'ancel the started activity.
