This page contains details on the action required when a person has been incorrectly enrolled in the Medicare program.

On this page:

Incorrectly enrolled

Acceptable documents for incorrectly recorded date of birth or gender

Amend incorrectly recorded date of birth or gender

Incorrectly enrolled

Table 1

Step

Action

1

Person is present in a Medicare Service Centre + Read more ...

Where an ineligible person has been enrolled in Medicare and the person is present in a Medicare Service Centre:

  • explain there has been an error in their enrolment and they are ineligible to remain enrolled in Medicare
  • consumer end date the persons enrolment and include message in Comments field

2

Information provided by another person + Read more ...

Where an ineligible person has been enrolled in Medicare and the information has been provided by another person (in person, over the phone or in correspondence), further investigation is necessary:

  • recall documents from archives (where applicable)
  • write to the person informing them it is necessary to confirm their Medicare eligibility
  • add a comment in the Personal Comments field detailing what information was received and that an incorrect enrolment letter has been sent

Note: it is important to remember that details regarding a suspected incorrect enrolment cannot be disclosed to anyone other than the person concerned. This is especially important when alerted to an incorrect enrolment by a member of the public.

For further assistance see Escalation process for Medicare eligibility.

Acceptable documents for incorrectly recorded date of birth or gender

Table 2

Step

Action

1

Acceptable documents where a customer date of birth or gender is incorrectly recorded + Read more ...

When a customer contacts for a change to date of birth or gender incorrectly recorded, the following documents are acceptable:

  • Australian birth certificate
  • Australian Drivers licence
  • Australian Passport
  • Australian Certificate of name change
  • Australian Citizenship Certificate
  • Immicard
  • Certificate of Identity
  • Foreign passport with current Australian visa

See Table 3

Amend incorrectly recorded date of birth or gender

Table 3

Step

Action

1

Request to update incorrectly recorded date of birth or gender + Read more ...

  • Request received via phone, go to Step 2
  • Request received via PaNDA, go to Step 4
  • Request processed in a Service Centre:
  • Complete a public security check
  • If Service Officer is satisfied with supporting documents DVS is not required, go to Step 6

2

Request via phone + Read more ...

The caller must be the person who is changing their date of birth or gender.

Complete a public security check.

Has the person passed a public security check?

  • Yes, go to Step 3
  • No, request the person emails or mails supporting documents. Procedure ends here

3

Customer consent + Read more ...

Read the following to the customer:

'Do you understand that identification documents you provide to Services Australia will be checked with the issuing authority to confirm validity. The documents are subject to agency compliance and audit processes. In providing these documents, you consent to the Agency validating the documents with the issuing authority.'

Has verbal consent been given?

  • Yes, go to Step 5
  • No, add comments in Personal - verbal consent not obtained, document not verified through DVS. Procedure ends here

4

Get verbal consent + Read more ...

Attempt to contact the customer.

If successful contact:

  • Perform a public security check
  • Read the following to the customer:
    • 'Do you understand that identification documents you provide to Services Australia will be checked with the issuing authority to confirm validity. The documents are subject to agency compliance and audit processes. In providing these documents, you consent to the Agency validating the documents with the issuing authority.'

Has verbal consent been given?

  • Yes, go to Step 5
  • No, add comments in Personal - verbal consent not obtained, document not verified through DVS. Procedure ends here

If unsuccessful contact:

  • issue Request for information to complete or change Medicare enrolment z0137 letter

5

Access DVS tool + Read more ...

Do you have access to the DVS tool:

HSDD:

  • Yes, access DVS tool and attempt to verify documents
  • No, contact Medicare document verification team for document verification

F2F:

  • Yes, access DVS tool and attempt to verify document
  • No, see Escalation process for Medicare Eligibility

Is verification successful?

  • Yes, go to Step 6
  • No, go to Step 9

6

Log in to Consumer Directory Maintenance System (CDMS) + Read more ...

On the CDMS home page:

  • select Amend on consumer details
  • key the Medicare card number in ID Number field
  • select Personal

7

Change customers date of birth or gender + Read more ...

In Consumers, select the radio button against the customer to be changed.

In Personal Details, amend the customer's details, as required:

  • date of birth, or
  • gender
  • Birth Date Accuracy field, select a value to indicate the accuracy of the date of birth

Does the error message 12300 DATE OF BIRTH>MEDICARE ENTITLEMENT START DATE display?

  • Yes, go to Table 4 and follow the process
  • No, make any other amendments in the remaining fields, as needed

Select:

  • Validate
  • Validate
  • Update to confirm all details

A Medicare Payment Integrity (MPI) restriction may be present preventing amendment. If this occurs, please contact a Medicare Eligibility Programme Support Officer (PSO).

8

Record comments + Read more ...

Where DVS has been used, record comments in Personal tab in CDMS and result in DVS database.

Request received via phone:

  • DVS consent acquired (insert date). (Insert document type and document reference text/document number), VRN: (insert VRN number)
  • For example, PaNDA work item ID: 1235467891. DVS consent acquired 10/12/2020. Marriage certificate, reference number 256256/2014, VRN: 12345
  • Scroll down and select Validate
  • Action any warning messages and Validate
  • If the details are correct, select Update

Where DVS has not been used (change made in a service centre), record comments in Personal tab in CDMS:

Record the document reference text/document number. For example, Marriage certificate, reference number 256256/2014.

Procedure ends here.

9

Document verification unsuccessful + Read more ...

  • F2F Service Officers - see Escalation process for Medicare Eligibility
  • HSDD Service Officers - call Medicare Eligibility PSO

Was PSO document verification successful?

  • Yes, complete application. Go to Step 6
  • No, Service Officer attempts to contact customer

Was customer contact successful?

  • Yes, customer provides other identification document. Go to Step 4
  • No, customer contact is unsuccessful and/or second attempt at verification fails: return original document to customer with letter Z0137

Amending an incorrect date of birth when error message 12300 is displayed

Table 4

Step

Action

1

Enter the customer’s Medicare number in Amend Consumer Details on CDMS + Read more ...

Select the Personal tab and select the radio button of the person requiring the date of birth correction (only applicable if more than one person on the Medicare record).

Select Entitlement tab and take note of the Entitlement Type and Date:

  • Select the relevant Eligibility Status radio button and select Amend
  • Select Documents tab to confirm if any identity documents have already been recorded

2

Eligibility documents + Read more ...

Have eligibility documents been previously recorded?

  • Yes, select Return. Go to Step 4
  • No, go to step 3

3

No document recorded + Read more ...

  • select Document of Identity from the Eligibility Document dropdown box
  • in the Reference Text field, enter in ‘DOB OB Work Around’ and any other reference related to the document provided
  • in the Reference Date field, enter the customer’s correct date of birth
  • Select Validate
  • Select OK when the message 00634 appear
  • Select Return
  • Select Validate
  • Select Update on the Confirm Amended Medicare Entitlement Details
  • Select OK if the message 00144 appears
  • Select OK when the message 00454 appear

4

Deleting current Entitlement Type + Read more ...

Make sure you have taken note of the Entitlement Type before deleting.

  • Under the Eligibility Status field select the radio button with the Entitlement Start Date to be deleted
  • Select Delete
  • Select No when the message 00633 appears
  • Select OK when the message 00646 appears
  • Select Validate and Update
  • Select OK when the message 00144 appears
  • Select OK when the message 00454 appears

5

Amending the date of birth + Read more ...

  • Select the Personal tab on the bottom of the screen
  • amend the customer’s date of birth to their correct date of birth
  • Select the Date is valid (not estimated) from the Birth Date Accuracy dropdown box select Validate
  • If the error message 00395 appears because the indigenous status has not previously been recorded:
  • Select OK
  • Select the correct status from the Aboriginal/Torres Strait Islander/South Sea Islander dropdown box
  • if status is not known, select Not Stated from the dropdown box
  • select Validate
  • Select OK when the message 00645 appears
    • Select Validate and Update
  • Select OK when the message 00359 appears
  • If error message 12302 appears
    • Select OK
    • Click on the Documents button
    • A warning Message 00661 will appear
    • Select OK
    • Select Validate

Note: if My Health Registration message is returned – select NO (unless request for a My Health Registration has been provided).

6

Re-entering Entitlement type details + Read more ...

Select the Entitlement tab.

Re-enter the Entitlement Type, Country of Relevance and Entitlement Start Date.

The Entitlement Start Date is the customer’s date of birth or relevant start date (if customer eligible after their date of birth):

  • Select relevant entitlement from the Eligibility Type dropdown box
  • Select country from the Country of Relevance dropdown box
  • key the Entitlement Start Date, is the customer’s date of birth or relevant start date (if customer eligible after their date of birth)
  • Select Validate
  • Select OK when the message 00645 appears
  • Select Validate and Update
  • Select OK when the message 00144 appears
  • Select OK when the message 00454 appears

Note: if no identity document was provided, and DOB OB workaround from the Document of Identity dropdown box as part of this work around, it must be deleted.

Were DOB OB workaround keyed as identity documents?

  • Yes, go to Step 7
  • No, go to Step 8

7

Deleting DOB OB workaround documents + Read more ...

  • Select the Documents tab
  • Select the documents to be deleted (Ref text will read - DOB OB WORK AROUND) and select Delete
  • Select NO when warning message 00183 appears
  • Select OK when message 00644 appears
  • Select Return, Validate and Update
  • Select OK when message 00454 appears

8

Only 1 person on the Medicare record + Read more ...

Is there only 1 person on the Medicare record?

  • Yes, go to Step 9
  • No, go to Step 10

Procedure ends here.

9

Confirm Group Eligibility Start Date + Read more ...

Where the customer is the only person on the Medicare record, amend the Group Eligibility Start Date to the customer’s date of birth (or other relevant start date if eligible after their date of birth).

Note: this date should only be amended if the customer is the only person on the Medicare record.

  • Select the Group tab
  • Select Card Request
  • Amend the Group Eligibility Start Date to either the customer’s date of birth (or other relevant start date if eligible after their date of birth)
  • Select Validate
  • Select OK when the message 00645 appears
  • Select Validate and Update
  • Select OK when the message 00105 appears

10

If assistance is required to finalise the transaction + Read more ...

See, Escalation process for Medicare eligibility.
