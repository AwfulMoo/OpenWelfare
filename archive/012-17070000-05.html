Manual data entry keying conventions

Table 2: This table describes keying conventions for manual data entry fields. Use Control F to locate the relevant field.

Note: not all fields are mandatory. If mandatory information is unavailable, keying ? in the required field will cause the claim to PEND for further assessing.

Field

Claim type

Keying convention

File #

All

The DVA file number

Name

All

The first 3 characters of the first given name

DOS

All

The date the service was rendered - DDMMYY format

Item

All

The item number which identifies the service rendered.

If the service is a consultation in a hospital, nursing home or institution, documentation should show the total number of patients seen by the doctor during that visit. The number of patients seen is keyed as part of the item number, e.g. '33/2'. This information is obtained from the ‘Description of Service’ or ‘Item No.’ section of the voucher.

If more than 1 service is shown on the voucher, press [PF2- FULL /BASIC] to allow keying in the additional ITEM fields.

Amount

G, P, R, S, A, H

The amount claimed for the service performed. Key as cents, e.g. $33.35 as '3335'.

Do not key for Local Medical Officer (LMO) attendance items (1 to 98), as the system will automatically insert the payable fee.

L

G, R, S, A

The treatment location.

Do not key for LMO attendance items (items 1 to 98), as the system will automatically insert the appropriate location value. For other items, the system will automatically insert the R (rooms) in this field unless another value is keyed.

If the Visit box is ticked, key 'V'.

If the Hospital box is ticked, key 'H'.

KM

G, R, S

The number of kilometres travelled by the provider to treat the patient.

R

G, R, S, H

The disposal (separation) code identifying the patient status at the time of discharge:

  • A - discharged by hospital
  • B - discharged own risk
  • C - transferred to nursing home
  • D - transfer to psychiatric hospital
  • E - transfer to other hospital
  • F - death with autopsy
  • G - death without autopsy
  • H - transfer to other accommodation
  • I - type change separation
  • R - deceased
  • S - still an inpatient
  • W - nursing
  • X - other hospital
  • Z - home

SD

G, P, R, S

Key 'SD' (self-determined) if the notation SD is present on the documentation. This indicates the provider has self-determined the service involved.

RSN

G, P, A, N

Key 'P' (pend) if the service line needs to be examined by the pend Service Officer.

Ref Pr

P, R, S, N

The provider number of the referring or requesting provider.

Note: if more than 1 service is shown on the voucher, do not key this field for subsequent services.

Ref Dt

P, R, S, N

Statistics screens

Statistics screens, control lines and functions

Table 3: This table describes the control lines and functions for the individual operator daily statistics screen and general supervisor statistic screens. Note: statistics are not available until the next working day.

Item

Control line and functions

1

General operator enquiry - individual operator daily statistics screen

OVOI,PRO,D,SOURCE OFFICE CODE,DDMMYY

This screen displays the amount of TAS processing completed by the operator who is logged on for the date and source office specified.

Note: personal operator statistics are only available to the individual who is logged on and they are not available until the next working day.

2

General supervisor - statistics screens

These are used by supervisors/managers with the appropriate access authority.

OVSI,PRO,D,SOURCE OFFICE CODE,DDMMYY

Source office daily processed claims statistics screen displays the total amount of TAS processing completed for the source office on the date specified.

OVSI,WOH,D,SOURCE OFFICE CODE,DDMMYY

Source office daily work on hand statistics screen displays the total amount of TAS work on hand for the source office on the date specified.

OVSI,PRO,D,STATE CODE,DDMMYY

State daily processed claims statistics screen displays the total amount of TAS processing completed for the state on the date specified.

OVSI,WOH,D,STATE CODE,DDMMYY

State daily work on hand statistics screen displays the total amount of TAS work on hand for the state on the date specified.
