This page contains processes for validating a registration, registering new applicants and assessing Disaster Health Care Assistance Schemes (DHCAS) claims.

On this page:

Register a new applicant

Validate a registration

Table 1: how processing staff assess whether registration requirements are met for DHCAS. A completed DHCAS registration form will contain the information outlined in this table.

Step

Action

1

Assess document requirements + Read more ...

To receive the next registration for processing, select the Ready button in PaNDA.

2

Documents supplied + Read more ...

Have all the required documents been supplied?

Refer to the DHCAS category and services table on the Resources page

  • Yes, go to Step 3
  • No, registration is rejected and returned
    • prepare the relevant DHCAS Rejection of Registration letter based on the reason and return all documents to the applicant
    • close the claim in PaNDA
    • procedure ends here

3

Form signed and dated + Read more ...

Has the form been signed and dated?

  • Yes, if the claimant:
    • only submitted the registration form continue processing the registration using the Register a new applicant table
    • submitted both a registration form and a claim form (see Resources page for links to registration and claim forms) at the same time, process the registration and the claim
  • No, the claim needs more information, prepare the DHCAS claim needs more information letter citing claim form is incomplete or incorrect paragraph. Return all documents back to the applicant

Register a new applicant

Table 2: how processing staff register new applicants to DHCAS and create a new claimant file. Team Leaders escalate all new application registrations to Programme Management for assessment. Programme Management respond to the Team Leader if the applicant is eligible for registration.

Step

Action

1

Open Disaster Health Care Assistance Scheme + Read more ...

  • Open Intranet
  • From the Apps drop down list, select Disaster Health Care Assistance
  • Enter UserID (P number)
  • Enter Password, generated by RSA SecurID Token

If denied access, confirm current access with a Team Leader.

2

Check for previous registration + Read more ...

  • From the Special Assistance menu on the left hand side of the screen, select the Search button under the Registration heading
  • Select the relevant adverse event from the Assistance Scheme drop down box
  • Enter into the appropriate fields the information based on the registration form:
    • surname
    • first name
    • gender
  • Select the Search button

Is the applicant already registered?

  • Yes,
    • ensure all details match and contact the applicant to advise that they have already registered
    • if details don’t match, for example address is different, see Table 4
  • No, go to Step 3

3

Add a new registration + Read more ...

If any information on the form needs clarifying, contact the claimant

  • From the Special Assistance menu on the left hand side of the screen, select New under the Registration heading. This will display the Special Assistance – New Registration screen
  • Enter into the appropriate fields the information based on the registration form:
  • Start Date - this is the date the registration form was signed
  • Registered Category - this can be selected after the adverse event has been chosen from the Assistance Scheme drop down box

Complete remaining tabs if any of the following information has been supplied:

  • Cover Type - if a claimant’s registration is for Allied or Hospital only. This field is not compulsory and should be left blank if unknown
  • Financial Institution - if a claimant has provided their bank details at the time of registration. Note: if bank details are not provided, contact the claimant to record their bank details verbally. Payments under DHCAS can only be made via EFT. Claims cannot be paid via cash or cheque
  • Fund - if the claimant provides any Private Health Fund details at the time of registration
  • Select Add New to complete claimant registration

4

Add a claimant file + Read more ...

  • From the Special Assistance menu on the left hand side of the screen, select Search under the Registration heading
  • Enter into the appropriate fields the details of the new claimant
  • Select Search which will display the registration number to be used in the file naming convention: REGNUMBER SURNAME FIRSTNAME
  • Open the Claimant Files folder located on Special Assistance drive
  • Create New Folder for the claimant
  • Rename the folder as per the naming convention: REGNUMBER SURNAME FIRSTNAME
  • Complete DHCAS Registration Completed letter:
    • letter needs to be Quality Checked (QC) before posting to applicant
    • post letter to applicant. Note: this is a manual process, as the letter is not system generated
    • move applicant’s registration documents to the new Claimant File located on the Special Assistance drive

Once registered with a DHCAS adverse event, to be paid they require a vendor number.

See Table 3

Request a new vendor number

Table 3: how the Vendor Maintenance team request a new vendor number for either claimants that have not previously registered, or suppliers or providers.

Step

Action

1

Request a new vendor number + Read more ...

A new vendor number request must be actioned through the Vendor Maintenance team:

  • complete a 'Vendor Information Request Form' (under ‘V’ in the Services Australia Financial forms index)
  • email form to the Vendor Maintenance team (see Medicare EFT manual payments)

2

Filing process + Read more ...

If the vendor number is for a supplier or provider, Pend the claim documentation noting a request has been forwarded for a new vendor number.

Once a vendor number has been received from the Vendor Maintenance team:

  • retrieve the document from Pend
  • continue processing the claim
  • update the ‘Errors to be reported’ Vendor list with the new Vendor number

Ask a Team Leader to email Vendor Maintenance (see Medicare EFT manual payments) to request a new list. The lists are in the Special Assistance drive, under References.

Confirm and update claimant registration

Table 4: how processing staff confirm and update a claimant’s records before processing a claim.

Step

Action

1

Open Disaster Health Care Assistance Scheme system + Read more ...

  • Open Intranet
  • From the Apps drop down list, select Disaster Health Care Assistance
  • Enter User ID (P number)
  • Enter Password, generated by RSA SecureID Token

If denied access, confirm current access with Team Leader.

2

Confirm claimant’s registration details + Read more ...

  • From the ‘Special Assistance’ menu on the left hand side of the screen, select Search under the Registration heading
  • Select the relevant adverse event from the Assistance Scheme drop down box
  • Enter into the appropriate fields the claimant’s details:
    • Medicare Card Number
    • surname
    • first name
    • gender
  • Select Search
  • Select Update
  • HIC Special Assistance - Update Registration screen displays

If the Assistance Scheme cannot be identified, search by Surname and First Name.

3

Update claimant’s registration details + Read more ...

Select one of the following tabs to start the update, based on the information supplied:

  • Mailing Details
  • Cover Type – this records the claimant’s private health insurance type of cover (Allied or Hospital). If the claimant has both, then this needs to be noted in the ‘memo’ category
  • Financial Institution
  • Fund – this records the claimant’s private health insurance name
  • Confirm or amend all registration details as per the claimant’s claim form or letter/email notifying of a change
  • Select Update Now

Continue to assess the claim.
