Identity SME escalation process

Role

Action

Timeliness standard

Service Officer

When Identity document returns a result of 'Failed Verification' ('N' result), staff must:

  • check that the details submitted match the document (spelling, dates etc.) and re-submit (Max 5 attempts) if incorrectly coded, and
  • check the personal details have been entered as they appear on the identity document
  • if it still results in 'Failed Verification' ('N' result), or the identity document is locked, escalate to the Identity SME via the ‘DVS Response of N’ Identity Fast note, and scan identity document to the customer’s record
  • Reason for request must be included
  • Do not close the Fast Note DOC
  • Staff working on the Silver Service line must select Silver Service ID Escalation Fast note from the template drop down box

When Identity document returns a result of ‘Data out of Range (‘D’ result)’ staff must:

  • compare identity document to images in Identity documents A to Z.
  • if you still require assistance, escalate to the Identity SME via the ‘DVS Response of N’ Identity Fast note, and
  • scan identity document to the customer’s record
  • reason for request must be included

If an Identity document requires deletion,

  • consider if the documents needs to be moved to another category, see table 2, step 11 on the Process page
  • if the document still requires deletion, escalate to an Identity SME via the ‘DVS Response of N’ Identity Fast note
  • Reason for deletion must be included

If the Photo check box is incorrectly ticked, for customer,

  • without a confirmed identity status
    • escalate to an Identity SME via the ‘DVS Response of N’ Identity Fast note
  • with a confirmed identity status
    • email full details to DVS Operations
    • reason for request must be included

24 hours

Identity SME

The SME must:

  • For failed verification ‘N’ result use DVS Admin UI then attempt to resolve the issue (up to 5 times in the Dashboard)
  • For document deletion request, investigate the record and move the identity documents between categories if possible
  • If the SME does not have delete access:
    • refer the request to a SME with delete access in the same Service Zone
    • if a SME with delete access cannot be identified refer to line manager for assistance, or email DVS Operations
  • If unable to verify the document:
    • Check the document is scanned to the record. Failed DVS verification escalations must not be escalated to DVS Operations until the document is scanned to the record
    • Escalate to the DVS Operations Team via WLM
    • Reallocate the work item in WLM using the reallocation reason of 'Recategorised'
    • In the work item field, enter ZIAM_ACT_ESCALATED and then select 'SUBMIT'. This will flow the work item to the DVS Operations Team for action
  • If there is no identity document scanned to the record:
    • Unlock the locked category
    • Annotate the escalation DOC and close the activity
    • If necessary, provide feedback to the Service Officer who created the escalation

24 hours

DVS Operations Team
  • Investigate case
  • Action customer record
  • Provide response to the Identity SME if appropriate

5 days
