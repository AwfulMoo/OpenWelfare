For processing staff only

This page contains the instructions on how to process the Manual Identity Verification Provider Digital Access (HW080) form for a user.

Locating the correct PRODA account and checking the form and required identity documents

Table 1

Step

Action

1

Identity status + Read more ...

Accounts have 3 different identity statuses:

  • Known accounts have been created and the professional's identity has been verified by passing the Identity Verification Service (IVS) process. The professional may use this account to access services and the PRODA system. If the account is 'Known' and there are no identifiers listed on the form, Service Officers are not required to process a provided form
  • Unknown accounts have been created but the professional's identity has not been verified. The user can either add their identity themselves or submit the HW080 form
  • Pending accounts have been created but the professional cannot pass the PRODA Document Verification Service (DVS) validation using their existing documentation and will be required to submit the HW080 form

2

Has HW080 form been completed and signed? + Read more ...

  • Yes, go to Step 3
  • No, the user will need to provide a signed completed form. Go to Step 10

3

Search for account-holder in PRODA + Read more ...

Access the PRODA system.

The system must be set to No Inquirer.

Enter supplied details in PRODA.

Enter either of the following for the Account details in PRODA:

  • Username
  • Registration Authority (RA) Number

Select Search.

What is the result?

  • Manage account screen displays, go to Step 8
  • no records have been found: Error ID PIA0051displays, go to Step 4

4

No records found + Read more ...

Enter demographic search details:

  • first name (optional)
  • surname (mandatory)
  • date of birth (optional)
  • gender (optional)

What is the result?

  • Manage account screen displays, check the HW080 for required action. Where the selection for the Purpose of this form is:
    • Complete the identity verification part of PRODA account creation, go to Step 8
    • Verify a change of name, go to Step 9
  • multiple records match and a list of search results displays, go to Step 5
  • no records have been found: Error ID PIA0051 displays, go to Step 6

5

Multiple records match + Read more ...

Is the correct user listed in the search results?

  • Yes, go to Step 7
  • No, go to Step 6

6

Still unable to match + Read more ...

Check the PRODA clean-up report.

Is the correct user identified?

  • Yes, send the 'Your PRODA account enquiry - PRODA clean up report' email to the user. Procedure ends here
  • No, call the user and ask them to provide the 'PRODA Account Created' email. This is an auto generated email sent to the user when the account is created. If they cannot locate this email, ask them to check in their junk mailbox
    • If unable to contact the user by phone, send them an email using the unable to access created PRODA account email template (the Resources page contains a link to the template)
    • Staff must add a comment to the user account if contacted by phone (for example, 'Spoke to user, unable to locate the account with information provided on HW080 form')
      Select Comments button under Account details
      Select Add new comment button
      Type comment and select Save button
    • Procedure ends here

7

Correct user identified + Read more ...

Select the radio button for the correct user

Select Manage account

Check HW080 for what action is required.

What has been selected at the Purpose of this form?

  • Complete the identity verification part of PRODA account creation, go to Step 8
  • Verify a change of name, go to Step 9

8

Ready to begin identity verification process? + Read more ...

Processing can start if all the required acceptable documents for PRODA have been provided.

Documents already passed online and documents provided with the form need to meet the PRODA Evidence of Identity criteria:

  • Check the Identity status table to see if the user has already passed some Evidence of Identity documents online
  • Check if the user has provided enough Evidence of Identity with the form to meet the identity verification document/s criteria (this includes the documents already passed online)

Ready to continue?

  • Yes, see Step 1 in the Add Evidence of Identify documentation table
  • No, go to Step 10

9

Begin verifying change of name? + Read more ...

The process can be continued if all the required acceptable documents for PRODA have been provided to verify a change of name and are attached to the HW080 form.

Ready to continue?

  • Yes, see Step 1 in the Add evidence of change of name table
  • No, go to Step 10

10

Contact user + Read more ...

Phone the user to request the missing information. If the form has not been signed, the user will need to provide a new completed signed form.

If the user has not provided a phone number, send them an email using the 'PRODA form missing/incomplete information email' template. See Resources page.

Staff can add a comment to the user account (for example, 'Sent an email to user about missing/incomplete information):

  • Select Comments button under Account details
  • Select Add new comment button
  • Type comment and select Save button

Procedure ends here.

Add Evidence of Identity documentation

Table 2

Step

Action

1

Begin identity verification + Read more ...

On the Manage account screen, select the Add Identity Document(s) button under the Identity Status section.

The Add Identity Document(s) screen will display a list of previously added documents under Added documents.

Note: only documents that have not already been passed online need to be added to their PRODA account.

Note: ensure a duplicate document is not added to the user's account. Documents must not be added to a user’s account more than once.

2

Add additional documents + Read more ...

  • Select Add Document(s) button
  • Select radio button for the category and type of supporting identity document supplied with the form
  • Select Add Document(s) button again

See Evidence of Identity for Provider Digital Access (PRODA).

3

Result of adding document + Read more ...

If a Fail message displays after attempting to add a document, see the Failed identity documents table.

If the Manage account screen displays:

  • The document is added to the Identity status document list
  • Repeat Step 2 for each identity document supplied
  • Once the correct number and type of relevant identity documents have been added successfully, Identity status will display as Known. Go to Step 4
  • If Identity status displays as Unknown or is still Pending after the correct number of documents have been added:
    • If supporting change of name document/s provided. See Step 1 in the Add evidence of change of name table
    • Check Manage names. The user must submit a change of name document for every alternative first name or surname entered as part of the 3 identity verification documents
    • If no change of name is required, escalate to the Programme Support Officer (PSO) for further investigation
    • If more Evidence of Identity documents or change of name documents are required, contact the user to request the missing information. See Step 10 in the Locating the correct PRODA account and checking the form and required identity documents table

4

Has the user provided an identifier at Q2? + Read more ...

  • Yes, see Processing Medicare identifiers for Healthcare providers and administrators
  • No, go to Step 5

5

Inform user + Read more ...

Send an email to the user's PRODA email address notifying them their form has been successfully processed. Use PRODA form processing complete (email template). See Resources page for the email template.

The Service Officer should file the completed form in their daily work to be archived.

Procedure ends here.

Add evidence of change of name

Table 3

Step

Action

1

Manage account screen + Read more ...

Select the Add Identity Document(s) button under the Identity Status section.

The Add Identity Document(s) screen will display a list of previously added documents under Added documents.

The following documents are valid change of name documents:

  • Marriage Certificate issued by Births, Deaths and Marriages
    • When a Marriage Certificate is used to verify a change of name, ensure the account holder is female. Currently the system only caters for a female using this certificate through the PRODA Document Verification Service (DVS)
    • If an account holder is male and provides a Marriage Certificate to verify a change of name, do not enter it into PRODA - escalate to Tier 3 for investigation, see Help for staff Provider Digital Acces (PRODA).
  • Change of Name Certificate
  • Amended Australian Birth Certificate

The above identity document details vary based on the issuing state/territory.

2

Add additional documents + Read more ...

  • Select Add document button
  • Select Change of Name radio button for the type of supporting identity document supplied with form
  • Select the document type:
    • Marriage certificate
    • Change of name certificate
    • Australian birth certificate (if the birth certificate was amended to reflect a change of name)
  • Select Add document button again

See Evidence of Identity for Provider Digital Access (PRODA).

3

Result of adding document + Read more ...

If a Fail message displays after attempting to add a document, see the Failed identity documents table.

If the Manage account screen displays:

  • the document is added to the Identity status document list
  • repeat Step 2 for each identity document supplied
  • once the correct number and type of relevant identity documents have been added successfully, Identity status will display as Known
  • go to Step 4

The user must submit a change of name document for every alternative name entered as part of the 3 identity verification documents.

4

Has the user provided an identifier at Q2 + Read more ...

  • Yes, see Processing Medicare identifiers for Healthcare providers and administrators
  • No, go to Step 5

5

Inform user + Read more ...

Send an email to the user's PRODA email address notifying them their form has been successfully processed. Use PRODA form processing complete (email template). See the Resources page for the email template.

The Service Officer should file the completed form in their daily work to be archived.

Procedure ends here.

Investigating failed identity documents

Table 4

Step

Action

1

Failed identity document + Read more ...

Note: if the error message PIA0113 - 'This document has already been verified with a different account' displays, check the Activity History to see the related account.

If the user has managed to set up 2 separate accounts, ensure both accounts belong to the same person and escalate to Tier 3. Do not duplicate the document.

Has the document been verified for a different account?

  • Yes, escalate to Tier 3 with reason - duplicate account (both related account numbers), see Help for staff Provider Digital Access (PRODA)
  • No, go to Step 2

2

Investigate failed document + Read more ...

The Service Officer is to check that the supporting document is entered correctly and is valid.

Check the Evidence of Identity for Provider Digital Access (PRODA) for what information needs to be keyed into the relevant fields.

The following Commencement of Identity (CoI) documents cannot be manually passed and must be escalated to Tier 3 for investigation, see Help for staff Provider Digital Access (PRODA):

  • Australian passport
  • Australian birth certificate
  • Australian citizenship certificate
  • Foreign passport (with an Australian visa linked)
  • Certificate of registration by descent
  • ImmiCard

Primary and secondary documents that cannot be verified through the DVS can be manually passed, after a peer review by another Service Officer.

Peer reviewer: must check spelling, all dates (ensure the date of birth on the document matches the date of birth on the account), gender and that the information has been entered in the correct fields.

See Step 1 of the Peer review and manually overriding documents table.

Peer review and manually overriding documents

Table 5

Step

Action

1

Commencement of Identity (CoI) documents + Read more ...

Is the document a CoI document?

  • Yes, CoI documents cannot be manually overridden. Escalate to Tier 3 for investigation, see Help for staff Provider Digital Access (PRODA). Procedure ends here
  • No, go to Step 2

2

CoI passed on account + Read more ...

Is a CoI document passed on the account?

  • Yes, go to Step 3
  • No, code the CoI document first. If the document is unable to be coded, follow normal escalation procedures for the Programme Support Officer (PSO) to escalate to Tier 3 for investigation, see Help for staff Provider Digital Access (PRODA). Procedure ends here

3

Manually override the document + Read more ...

Enter the document information.

If the name on the document is the same as the 'Current name' on the account, do not select the <name> is not the name on the document.

Note: if the <name> is not the name on the document has already been selected, and the name is the same <first> and <surname>, unselect the check box and select submit again.

If the name on the document is different to the 'Current name', does the name match a name in Manage names on the PRODA account?

  • Yes, go to Step 4
  • No, if the user has not passed any Evidence of Identity and the name they created the account in is not the name on their documents, code the documents provided in the name on the documents. Once the documents have been added to the account, and if the account remains in Unknown or Pending, ask the user if they can provide a change of name document:
    • If the user is unable to provide one due to creating the account in a name they are known by in the community, escalate to the Programme Support Officer (PSO) for investigation
    • If the user is able to provide a change of name document, ask the user to send it in. See the Resources page in Evidence of Identity for Provider Digital Access (PRODA) for accepted change of name documents

4

Check data on PRODA account + Read more ...

Check all data entered on the PRODA account is entered correctly.

Cross-check information with other documents provided including date of birth and gender.

Note: if the date of birth on this document is different to the CoI document already passed, this document cannot be used. Ask the user to provide another document that has the correct date of birth.

If all above criteria is met, contact eBusiness Service Centre (EBSC) Program Support Officer (PSO) and request a peer review.

All manual overrides require PSO approval.

Service Officers and PSOs are to double check the date of birth and gender matches, and the spelling of names to ensure no incorrect name sets are being introduced to the account.

Can the document be added?

  • Yes, go to Step 5
  • No, if the PSO has identified that the document cannot be manually overridden, the Service Officer is to follow normal escalation procedures for the PSO to escalate to Tier 3 for further investigation, see Help for Staff Provider Digital Access (PRODA). Procedure ends here

5

Add failed document + Read more ...

To manually pass the document:

  • Tick the Add failed identity document to record box
  • Enter findings to support the reason the document can be accepted into the Reason for adding failed document comment box, include the peer reviewer's Logon ID or P number (mandatory)
  • Select Submit
  • Approved:
    • the document is added to the Identity status document list
    • repeat above steps for each identity document
    • go to Step 6

6

Inform user + Read more ...

Send an email to the user's PRODA email address notifying them their form has been successfully processed. Use PRODA form processing complete (email template). See Resources page for a link.

7

Add comments to the account + Read more ...

Staff must add a comment to the user account:

  • select Comments under Account details
  • select Add new comment
  • enter reason the document can be accepted and include the peer reviewer's Logon ID or P number (mandatory)
  • select Save

Service Officers should file the completed form in their daily work to be archived.
