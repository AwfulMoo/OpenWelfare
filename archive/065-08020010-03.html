Assessing the Scheme applications

Table 1

Step

Action

1

Access Process Direct and select Work Item + Read more ...

Upon allocation of a Work Item for a Scheme application, access Process Direct and locate the Work Item using the Search fields. Select the Work Item to view the customer's details and claim information.

Review the details on the customer's record and the information they provided in the application. To view information in Process Direct:

  • For applications lodged online or via Assisted Customer Claim (ACC):
    • Select at top left to view the claim summary provided by the customer. Note: for combined partner online claims, there is 1 claim Work Item to assess for both the customer and partner’s loan applications
  • For paper applications and manually created SOA shells:
    • Select Documents icon and open the SA310 or SA496

If the customer is a Member of a Couple (MoC) check if the partner has also lodged a Scheme application. If both MoC have applied, a separate Work Item is created for each person and bundled together so that both Work Items are automatically assigned. If the Work Items have not automatically bundled, allocate the partner's Work Item via Work Optimiser and assess both applications at the same time.

For information on reassigning Work Items in Work Optimiser see Searching and managing Work Items including hold to user.

Go to Step 2.

2

Progress of claim + Read more ...

When a Scheme online application is submitted, a Progress of Claim Note is automatically created and can be viewed via the Notes icon in Process Direct.

In addition to the Process Direct Progress of Claim Note, the lodgement of both online and paper applications for Home Equity Access Scheme is also automatically recorded on the customer’s record in Customer First.

The Process Direct Progress of Claim Note can be used to document the progress of the application and any decisions made throughout the assessment. This note is not visible in Customer First, use Fast Note, select Auto Text > use Older Australians > Home Equity Access Scheme > HEAS application progress

Close the Fast Note once all relevant details are entered.

Go to Step 3.

3

Death of customer + Read more ...

Start assessing the application to determine whether the customer meets the basic eligibility criteria for the Scheme. The assessment is conducted in Process Direct unless stated otherwise. If the basic eligibility criteria is not met, the application will be progressed for rejection.

For detailed information on the Scheme eligibility criteria refer to Eligibility for the Home Equity Access Scheme.

Check the Death Action (DEA) screen.

Has the customer passed away since lodging an application?

  • Yes, application to be rejected for reason DEA (death of customer). Go to Assessment outcome
  • No, go to Step 4

4

Voluntary withdrawal of application + Read more ...

Check the Notes and Documents on the customer and (if applicable) partner's records to determine whether a request to withdraw the application has been received.

Note: a verbal withdrawal of a Scheme application should only be actioned if the Home Equity Access Scheme CAO is satisfied of the customer's or correspondence nominee's identity. For information on how to establish and confirm the identity of the person verbally requesting the withdrawal, see Authenticating a Centrelink customer.

Has the customer or correspondence nominee requested the Scheme application be withdrawn?

  • Yes, application to be withdrawn by rejecting for reason CLR (customer request). Go to Assessment outcome. Do not withdraw the application by changing the status via the Status icon, as this will not generate a letter to the customer to confirm the withdrawal. Claims withdrawn via the Status icon require a manual letter to be sent
  • No, go to Step 5

5

Age requirements + Read more ...

Go to the Pensions Assessment Date (PNA) screen for the customer and (if applicable) partner to check if they have reached Age Pension age. Using the Determine Home Equity Access Scheme eligibility table for guidance, assess whether the customer meets the Scheme age requirements.

Does the customer meet the Scheme age requirements?

  • Yes, go to Step 6
  • No, application to be rejected for reason AGD (Age requirements not met). Go to Table 5

6

Application lodged in Australia + Read more ...

Check if the customer was physically present in Australia on the date they lodged their Scheme application, by checking their answers in the application. Also check the customer's travel movements (if available) using the Travel Outside Australia Summary (TOAS) and Immigration Movements (RSIM) screens. See Activating the Department of Home Affairs datalink and contingency procedures if datalink is unavailable for more details.

Is there evidence that the customer was not in Australia on the date of lodgement?

  • Yes, customer was outside Australia on the date of lodgement. Application to be rejected for reason DIS (Secretary Discretion). Go to Assessment outcome
  • No, customer was in Australia on the date of lodgement. Go to Step 7.

7

Australian real estate + Read more ...

Check the information the customer entered into the Scheme application and identify the real estate asset(s) offered as security for the Scheme loan. See Security for the Home Equity Access Scheme for details of the types of real estate asset(s) that can be accepted as loan security for the Scheme applications, including what ownership means.

Does the customer own real estate in Australia that they have offered as security for the loan?

  • Yes, go to Step 8
  • No, application to be rejected for reason REA (Real estate requirements not met). Go to Table 5

8

Loan security value + Read more ...

Check the information entered to the application by the customer and assess whether their estimate gives sufficient equity in Australian real estate asset(s) to secure the loan. Refer to the Establishing sufficient security for the Home Equity Access Scheme table for guidance.

Does the customer's estimate provide sufficient equity in the Australian real estate to secure the loan?

  • Yes, go to Step 9
  • No, application to be rejected for reason SEC (Inadequate security). Go to Table 5

9

Adequate and appropriate insurance + Read more ...

Has the customer advised in the Scheme application that they have adequate and appropriate insurance for the real estate they have offered as loan security?

  • Yes, go to Step 10
  • No, application to be rejected for reason INS (Insurance requirements not met). Go to Table 5

10

Bankruptcy and personal insolvency + Read more ...

Has the customer advised in the Scheme application that they are bankrupt or subject to a personal insolvency agreement?

  • Yes, application to be rejected for reason BAN (Bankruptcy or personal insolvency). Go to Table 5
  • No, go to Step 11

11

Pre-existing loans + Read more ...

A customer can only have 1 loan at a time through the Scheme. If the customer has an existing loan, it must be settled in full before a new loan can be granted. If the customer can provide sufficient additional security or has not yet reached the Maximum Loan Amount (MLA) for the existing loan, they may be able to have loan payments made under the existing loan instead of having a new loan created.

The HEAS Summary (PLSS) screen provides an overview list of any current and past loans for the customer. Check the PLSS screen and review whether any previous loans exist with an outstanding balance (i.e. a total current balance greater than zero).

Does the customer have an existing loan with an outstanding balance?

  • Yes, go to Step 12
  • No, go to Step 14

12

Restoration of existing loan + Read more ...

Review whether loan payments can be made under the existing loan. This includes checking that the customer meets all the current eligibility criteria and identifying whether the Maximum Loan Amount (MLA) has been reached. Consider whether changes can be made to the loan security value to increase the MLA.

For more information on how to determine if a loan payment can be started again, see Home Equity Access Scheme reviews.

As part of the review, contact the customer to discuss their loan situation and preferences, including whether they wish to withdraw the Scheme application and restart the existing loan.

Can loan payments be started under the existing loan, and has the customer consented to start the loan?

  • Yes, existing loan can be started and customer consents to withdraw their new application and start the existing loan. The following actions are required:
    • For customers who applied online or via Assisted Customer Claim (ACC), help the customer to withdraw the application via their Centrelink online account. For assistance see Cancel or withdraw an online claim. Update the Progress of Claim Note to document the decision
    • Application to be rejected for reason CLR (customer request). Go to Assessment outcome
    • In Customer First, make any coding updates to the customer's record based on the information in the claim, then manually restore the existing loan. See Home Equity Access Scheme cancellation and rejections for the steps to follow. Note: if a customer has applied to get an advance payment only (without a fortnightly loan payment), the Loan Topup Type must be set to ADV on the HEAS Details (PLS) screen to ensure fortnightly payments do not start again. See Advance payments for Home Equity Access Scheme loans for more details
  • No, the existing loan cannot be started, or the customer does not consent to start the existing loan. Go to Step 13

13

Settlement of existing loan + Read more ...

If the existing loan cannot be restarted, or the customer does not consent to starting the existing loan, the outstanding loan must be settled in full before the new Scheme application can be progressed. If the customer does not intend to settle the loan, the application is to be withdrawn or rejected.

Does the customer intend to repay and settle the existing loan?

  • Yes, place the Scheme application on hold for 4 weeks pending settlement of the loan. Update the Progress of Claim Note to document the assessment progress. Once the loan has been settled, the assessment of the new application can recommence. For information on how the customer can make repayments against their loan see Repayment of Home Equity Access Scheme loans
  • No, application is to be manually rejected. Use rejection reason CLR (customer request) if the customer requests withdrawal of Scheme application, or use reason DIS (Secretary Discretion). Go to Assessment outcome

Note: to request withdrawal of an application a customer or their correspondence nominee must do so either in writing, verbally or online. A verbal withdrawal of an application should only be actioned if the CAO is satisfied of the customer's or their nominee's identity.

For information on how to establish and confirm the identity of a person verbally requesting a withdrawal, see Authenticating a Centrelink customer. The customer must be advised that decisions regarding withdrawn applications cannot be appealed.

Place the Scheme application on hold + Read more ...

Place the Scheme application on hold to await finalisation of the pension claim. If a timeframe is unclear, place the Scheme application on hold for 4 weeks. Set the On Hold reason to CR In Process. Update the Progress of claim Note to document the reason for placing the application on hold.

  • Category: Claim
  • Summary: Pension new claim (AGE/DSP/CAR)
  • Content:
    • Customer has applied for a Home Equity Access Scheme loan and needs pension qualification to be assessed in full. Do not streamline reject the pension claim. The pension claim must be fully coded and assessed (including medical and carer eligibility, if applicable), even if the outcome is a rejection due to income or assets, the Scheme application has been placed on hold pending outcome of pension claim. When the pension claim is finalised, take the Scheme claim off hold. Note: taking the activity off hold will result in it being allocated, based on previously established priorities. The user logon on this activity does not impact work allocation

Qualifying payment REJ/CAN/SUS + Read more ...

Has a Home Equity Access Scheme qualifying payment been rejected (REJ), cancelled (CAN) or suspended (SUS)?

7

REJ/SUS/CAN reason code + Read more ...

Check the Pensions Status History (PNSH) screen to determine the reason why the qualifying payment was rejected, cancelled or suspended.

Was the reason a payability based reason?

  • Yes, go to Step 8
  • No, the qualifying payment is not payable for another reason
    • Rejections and cancellations - application to be rejected for reason STS (no qualifying payment). Go to Assessment outcome
    • Suspensions - suspensions are a temporary situation and may lead to a cancellation, or the qualifying payment may become current again
    • Place the application on hold for 4 weeks for reason CR in Process pending further review of the suspension status
    • Update the Progress of claim Note to document the reason for placing the application on hold
    • Process ends here pending confirmation of pension qualification

8

REJ/CAN/SUS effect date + Read more ...

Check the Pensions Status History (PNSH) screen to determine the date the qualifying payment was rejected, cancelled or suspended.

Is the Effect Date within 13 weeks of the Scheme application lodgement date?

  • Yes, the customer is accepted to be qualified (but not payable) for the qualifying payment, providing they have not since lost qualification. To determine this:
    • Check the information on the customer's record and in the Scheme application to see if a change of circumstance may mean the customer has since lost qualification for the qualifying payment
    • If there is evidence that determines they have lost qualification, the pension determination cannot be used to assess eligibility for the purposes of the Scheme application go to Step 9, or go to Step 12
  • No, the pension determination cannot be used to assess eligibility for the purposes of the Scheme application. Go to Step 9

9

Determine age + Read more ...

Has the customer reached Age Pension age?

  • Yes, application to be assessed as a loan-only application. Go to Step 10
  • No, application to be rejected for reason STS (no qualifying payment). Go to Assessment outcome

Australian Residence + Read more ...

See Residence assessment for customers claiming Age Pension.

Does the customer meet the qualifying residence requirements for Age Pension?

  • Yes, go to Step 12

11

Assets hardship + Read more ...

Check the Manual Assessment (MAS) screen to see if the customer is being paid under the assets hardship provisions (Manual Rate Reason code HAR). For information about the provisions see Assets hardship provisions - Pensions.

Is the customer being paid under the assets hardship provisions?

  • Yes, contact customer to advise that the Scheme cannot be paid while income support is being paid under the assets hardship provisions. If the customer prefers to retain payment under the hardship provisions, seek their consent to withdraw the Scheme application, ensuring the customer is aware that decisions regarding withdrawn applications cannot be appealed against. Once consent has been obtained:
    • For customers who applied online or via Assisted Customer Claim (ACC), help the customer to withdraw the application via their Centrelink online account. For assistance see Cancel or withdraw an online claim. Update the Progress of Claim Note to document the decision
    • Go to Assessment outcome. Application to be rejected for reason CLR (customer request)
  • No, go to Step 12

Completed and signed application + Read more ...

Review the information provided by the customer in their Scheme application and check that all mandatory questions have been answered, including that:

  • Members of a Couple (MoC) have made a joint decision for the customer to receive loan payments, and

2

Signature requirements + Read more ...

It is a legislative requirement that Scheme applications are signed by the applicant and (if any) their partner. Extreme caution must be applied where a Scheme application has been signed and/or lodged by another person, including if one member of a couple (MoC) has signed for both members. An application signed or lodged by another person is an indicator that the applicant or their partner may not be completely aware of the application or have a full understanding of the loan.

Is there a concern the applicant or their partner (where applicable) may not have a full awareness or understanding of the application?

  • Yes, consider appropriate action to take, which may include:
    • Contacting the applicant and/or their partner to gain an understanding of their circumstances and their awareness of the application
    • Requesting signed consent to the application via the SA310 or SA496 Part A
    • Assessing relevant documentation that may support an inability by the applicant or partner to sign, for example on medical or legal grounds. Such cases are to be escalated to the Level 2 Policy Helpdesk for consideration
    • Offering referrals to applicants who may be affected by elder abuse - see Abuse of older Australians for guidance on how to recognise, support and appropriately refer customers
    • Making a decision that the application is not valid (Not Effective) or rejecting the application for reason DIS (Secretary Discretion)
    • Once appropriate action has been determined, for rejection outcome go to Assessment outcome, or go to Step 3 to proceed with the assessment
  • No, go to Step 3

3

Review documents + Read more ...

In Process Direct, go to Documents and open relevant scanned documents provided by the customer.

Refer to the list of documents required for Home Equity Access Scheme applications and verify whether the customer has provided all the documents applicable to their circumstances.

Has the customer provided all the documents required for assessment of their application?

  • Yes, go to Step 9
  • No, go to Step 4

4

Land title search + Read more ...

Customers must provide a recent title search with their Scheme application. The search must have been done within 13 weeks of the date the application was lodged. Services Australia may conduct a title search on behalf of the customer through Equifax, however the searches incur a cost to the agency as they are only done when certain circumstances apply.

When instructed to conduct a title search, upload a copy of the search result(s) to the customer's record via the Document Tools guided procedure in Customer First.

Has the customer provided a title search?

  • Yes, check that the search was conducted within 13 weeks of the application lodgement date. If the search was done:
    • within 13 weeks of lodgement, accept the search as is. Go to Step 4 to request the other documentation
    • outside of 13 weeks, conduct a new title search via Equifax and upload it to the customer's record
    • Go to Step 5 to proceed with a request for information. The title search does not need to be included in the request
  • No, if the search is the only document not provided, conduct a title search via Equifax then go to Step 10,or include the title search in the request for information (go to Step 5)

Request information + Read more ...

Issue a request for information and allow time to respond. The method for generating the letter will depend whether the Scheme application is subject to the Circumstance Change Monitor (CCM) process.

How was the Scheme application lodged?

  • Paper application or manually created Social Application (SOA) shell (e.g. reindexed claims), go to Step 7

6

Request information for CCM claim + Read more ...

In Process Direct select at the bottom right of the customer's Scheme application, then select Request Documents. Enter the following information:

  • Select PLS - Home Equity Access as the Benefit type
  • Select New Claim as the Benefit Status
  • Search Available Documents and use the double right facing arrow to add documents to the request
    • To find the SA310/SA496 Part A - Part F of the SA310/SA496 search for 'Home Equity Access Scheme application'. Multiple parts can be selected and added to the request
    • Search 'PLS' for other Scheme-specific documents
    • If necessary document(s) are not available for selection, select 'Other' and use the free text fields to describe the required documents (up to five free text documents can be added)
  • Review the default settings for all other fields and update if appropriate
  • If free text is required set the 'Free Text required' radio button to Yes and enter the relevant text. Any free text entered will display at the top of the letter (prior to the dot point list of documents) so check spelling and grammar before proceeding
  • Select Finish to complete and issue the request. Check that the application has been placed on hold for a time which includes as allowance for the delivery of the notice. Manually update the timeframe if appropriate

For help see Process Direct - request information.

Once the letter has been successfully issued, update the Progress of claim Note to document a request has been issued. Include the date the letter was sent and details of which documents have been requested.

Process ends here pending due date for return of information.

7

Request information for non-CCM claim + Read more ...

In Customer First open the Request for Information (RFI) Guided Procedure (GP). Enter the following information:

  • Select PLS- Home Equity Access Scheme as the Benefit type (this will appear as ‘Home Equity Access Scheme’ in the letter)
  • Select New Claim as the Benefit Status
  • Select Social Security (Admin) Act 1999 - Section 63 as the Authority to Request
  • Review the default settings for all other fields and update if appropriate
  • Search Available Documents and use the double right facing arrow to add documents to the request. Note: the guided procedure does not contain the SA310/SA496 or Scheme-specific documents
    • If necessary document(s) are not available for selection, exit the GP and to go Step 8
  • Enter any free text information to include in the DOC created by the RFI GP. Note: the free text is not included in the letter
  • Select Finish to complete and issue the request
  • In Process Direct manually place the application on hold for a time which includes as allowance for the delivery of the notice via the Status page. Use Hold Reason Customer to Provide Information

For help see Request additional information for non CCM claims.

Once the letter has been successfully issued, update the Progress of claim Note to document a request has been issued. Include the date the letter was sent and details of which documents have been requested.

Process ends here pending due date for return of information.

8

Manual letters + Read more ...

If the letter generated via Process Direct or the Request for Information (RFI) Guided Procedure (GP) is not suitable for the customer's circumstances, a manual letter may need to be issued. A manual letter must only be issued only as a last resort so before creating one, consider whether the documents are genuinely required or whether a verbal request for information may be appropriate.

See Requesting Information (CLK) for information about verbal requests. Where there is any doubt that the customer may not have the capacity to fully understand a verbal request, a written notice must be issued.

To create a manual letter for the Scheme application:

  • In Customer First, create a manual letter by following the instructions in Creating a Q999 or Q888 letter. A Q888 should be used unless a document or form needs to be enclosed with the letter, in which case a Q999 should be used
  • Draft the letter content using the approved text template in the Resources page
  • In Process Direct:
    • Manually place the application on hold for a time which includes as allowance for the delivery of the notice via the Status page. Use Hold Reason Customer to Provide Information
    • Update the Progress of claim Note to document that a request has been issued. Include the date the letter was sent and details of which documents have been requested

Process ends here pending due date for return of information.

9

Failure to respond to a request for information + Read more ...

Has the customer failed to respond to a request for information and a time which includes as allowance for the delivery of the notice has lapsed?

  • Yes, application to be rejected for reason FRC (Failure to return correspondence). Go to Assessment outcome
  • No, customer has responded to the request and all required documents have been provided. Go to Step 10

Note: if the request for information included a request to provide a recent land title search, check that the search was conducted within 13 weeks of the application lodgement date or the date the request for information was issued. If not, conduct a new search via Equifax and upload a copy of the result(s) to the customer's record via the Document Tools guided procedure in Customer First.

10

Bankruptcy Register Search + Read more ...

Verify that the customer is not bankrupt or subject to a personal insolvency agreement by conducting a Bankruptcy Register Search (BRS) through Equifax.

  • If Equifax returns a Negative result, annotate the DOC to indicate the search returned negative, with no requirement to upload a copy of the search result to the customer’s record
  • If Equifax returns a Positive result, annotate the DOC to indicate the search returned positive, and upload a copy of the Equifax PDF file to the customer’s record via the Document Tools guided procedure in Customer First

Note: BRS searches incurs a cost so must only be conducted once the Scheme eligibility assessment is complete and all required information has been provided.

Is the customer bankrupt or subject to a personal insolvency agreement?

  • Yes, application to be rejected for reason BAN (Bankruptcy or personal insolvency). Go to Assessment outcome
  • No, go to Step 11

Insurance documents + Read more ...

  • If the renewal contains a Certificate of Currency that has a period of currency that provides coverage for the date of eligibility decision, then that is sufficient for the eligibility decision on that day

Valuation required + Read more ...

If the real estate asset(s) has not been valued in the last 12 months, or has not previously been recorded on the real estate screens, a valuation is required before the Scheme application can be processed to enable accurate assessment of the customer's equity in the asset and calculation of the Maximum Loan Amount (MLA).

Authority to inspect is mandatory for principal home properties greater than two hectares where there is an assessable component for an income support payment.

Has the customer and (if applicable) their partner provided authority for a valuer to inspect the real estate asset(s) being offered as security for the loan?

  • Yes, go to Step 4
  • No:

    • Otherwise, go to Step 4

4

Code real estate details on REBS + Read more ...

Go to the Real Estate/Business Summary (REBS) screen in Customer First and check if the real estate asset offered as loan security has previously been coded. The real estate must be updated with any new information provided in the customer's application, including the customer's estimate of the current market value.

  • If the asset is not already listed on REBS screen, add the property in a new activity via the Real Estate/Business Identifying (REBI) screen
  • If the asset does already exist on REBS, select the property and add any relevant updates

Do not finalise the activity. Use the Date of Lodgement of the application as the Date of Event for the updates.

Note: updates to new or existing real estate require coding of the following screens:

  • Real Estate/Business Identifying (REBI) screen. Note: free text entered to the Land Titles Reference field on the REBI screen will appear in correspondence to the customer. Ensure text is correct and appropriate
  • Real Estate (RE) screen
  • Real Estate Valuation Information (REVI) screen

For assistance coding these screens see Recording real estate.

Continue to Step 5 to determine any additional coding requirements. Also see the Resources page for a matrix of coding requirements for the various scenarios.

5

Determine if the real estate is an assessable asset + Read more ...

Review the real estate information to determine if the real estate asset(s) offered as loan security is assessable under the assets test. The asset is not assessable if the customer:

  • Does not receive a pension and has applied for the Scheme only, or
  • Receives a pension and the real estate asset is their principal home:
    • assessed under the Private Land Use test with less than 2 hectares of land, or
    • grandfathered from the Private Land Use test with less than 2 hectares of land, or
    • assessed under the Extended Land Use test

Do any of these circumstances apply?

  • Yes, the property is not assessable under the assets test. Go to Step 6
  • No, part or all of the property is assessable under the assets test. Go to Step 8

6

Principal home used as loan security + Read more ...

Is the customer using their principal home as loan security?

  • Yes, add the following coding to the real estate activity in Customer First, but do not finalise the activity
    • On the REBI screen ensure the Principal Home indicator is Y
    • On the RE screen code the Current House and Curtilage field as $1 less than the Current Market Value supplied by the customer. Note: this is a workaround required due to a system limitation
    • On the REVI screen, record 'High priority- Pensions Loans Scheme' in the Other Circumstances field
    • Go to Step 9
  • No, go to Step 7

7

Loan only customers + Read more ...

Is the customer applying for loan only payments (that is, without a payable rate of a qualifying payment)?

  • Yes, add the following coding to the real estate activity in Customer First, but do not finalise the activity
    • On the REVI screen, record 'High priority- Home Equity Access Scheme' in the Other Circumstances field
    • Go to Step 9
  • No, the property is assessable go to Step 8

8

Loan security is an assessable asset + Read more ...

The coding required for a customer receiving a pension is dependent on whether the Scheme security is partly or wholly assessable.

In Customer First update the real estate activity to include the following details in the Other Circumstances field on the REVI screen:

  • If the Scheme security is the principal home and is partly assessable under the assets test, record 'High priority- Home Equity Access Scheme'
  • If the Scheme security is wholly assessable under the assets test, record 'High priority- Home Equity Access Scheme'
  • If the assessable portion of the asset includes commercial components (i.e. generates income), also include notes in the Other Circumstances field to describe the commercial aspect e.g. B&B.

Go to Step 9

9

Request valuation on REVI screen + Read more ...

Finalise the provisional activity for the real estate update via the Assessment Results (AR) screen, before proceeding to request a valuation.

Note: only APS 5 and above can submit a valuation request.

To submit a request, return to the REVI screen to request the valuation by entering 'VAL' on the Request Valn/Profile field. Press Enter to submit the valuation request.

This will automatically send a request for a valuation to the valuation services provider. A successful request for a valuation will create a PEN/REA activity on the Future Activity List (FAL) screen. The activity will display the RE ID that appears on the REBI screen for the applicable property.

Note: if the valuation is unsuccessful, or a valuation cannot be requested via REVI, contact the Valuation Helpdesk for advice.

Go to Step 10.

10

Preventing assessment under the assets test + Read more ...

If the property is a non-assessable asset for the purposes of the assets test, additional coding must be applied immediately after the valuation request has been sent to prevent the asset from being assessed.

Code the following details via Customer First, as applicable based on the customer's circumstance and type of loan security:

Customer receives pension and loan security is principal home which is not assessable under the assets test

  • On the REBI screen, update the Sold/Gifted/Ceased field to 'NAA'
  • On the RE screen, code:
    • % Asset Owned field as '0'
    • Current House and Curtilage field as the same value as the Current Market Value field

Customer applied for loan-only and loan security is principal home

  • On the RE screen, code the Current House and Curtilage field as the same value as the Current Market Value field

Update the Source and Date of Receipt (DOR) fields as appropriate and finalise the activity.

Go to Step 11.

11

Place the Scheme application on hold + Read more ...

In Process Direct, place the Scheme application on hold by selecting the Status icon and updating the following information:

  • Set the status of the application to On Hold
  • Set the On Hold reason to Referral to valuation
  • Set the On Hold Expiry Date to 14 days in the future

For information about how to view and update the claim status in Process Direct CPTv5, see the Review Work item table

Update the Progress of Claim Note to document the reason the application has been placed on hold.

Are there any other properties offered as security for the loan?

  • Yes, go to Step 1 for the next property
  • No, procedure ends here pending return of the valuation. Once the valuation(s) are returned, go to Step 12

12

Valuation has been returned + Read more ...

When the real estate valuation has been returned, in Customer First update the real estate details on the REVI screen as follows:

  • Delete the words 'HIGH PRIORITY' from the Other Circumstances field. The rest of the notes including 'HOME EQUITY ACCESS SCHEME' must remain in the field
  • Enter today's date as the Event Date and Date of Receipt (DOR)
  • INT on Source field
  • I on Action field
  • Finalise the coding

The updated valuation details should automatically display in the application in Process Direct. When processing the application, ensure details have refreshed correctly.

Update the Progress of Claim Note to document that the valuation process has been completed.
