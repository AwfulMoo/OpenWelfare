Debt Management and Information System (DMIS)

Step

Action

1

Overview of DMIS + Read more ...

DMIS includes the following functions:

  • online addition, amendment and deletion of debt records
  • access to DMIS screens in all payment systems
  • facility to move between systems by changing the system code in the header
  • online browse access
  • facility to adjust remittances recorded via ESSentials
  • full repayment history
  • screen and field help

2

How to access DMIS + Read more ...

Access DMIS via any of the host systems by entering a screen name in the Nxt: field. The system codes in the Sys: field can be:

  • PEN
  • FAO
  • NSS
  • PGA
  • ISS

3

DMIS menu screens + Read more ...

There are 4 menu screens in DMIS:

  • DMIS Main Menu (OPMM)
    • This gives a reference point to the main screens and secondary menus within DMIS
  • DMIS Debt Update Menu (OPDM)
    • This lists the screens for updating certain debt details on DMIS
  • DMIS Customer/Recovery Menu (OPCM)
    • This lists the screens for updating recovery arrangements and other details relating to customers
  • Report Lists (OPRL)
    • A screen to get historical reports from DMIS

4

Navigating within DMIS + Read more ...

To access most DMIS screens:

  • enter the screen name in the Nxt: field. For example 'OPDL'
  • press [Enter]

DMIS will go to an appropriate screen when selecting an activity. Which screen it goes to will depend on the type of activity.

Some screens are update screens, and staff will need to enter data.

To exit an update screen without making any changes, press [PF16] (Shift + PF4).

It is possible to enter a new screen code in the Nxt: field at the same time.

5

Moving between screen fields, DMIS navigation and help + Read more ...

Navigation, help and movement through the DMIS screen fields is the same as other systems.

  • Field Help ('?'). Keying '?' in to a field will explain the:
    • functions of the field
    • the acceptable values for it, and
    • any guidelines for entering those values
  • [PF1] is Screen Help. This explains the functions of the displayed screen
  • [PF2] is Message Help. This gives details about:
    • displayed messages. Such as errors, warnings and information
    • how to respond to a displayed message
  • [PF4] gives information about the valid function keys for the current screen and system
  • [PF24] (Shift PF12) lets staff move forward between different debts on most screens. This happens when staff first select the debt from the Debt List (OPDL) screen
  • [PF23] (Shift PF11) lets staff move backwards between different debts on most screens. This happens when staff first select the debt from the Debt List OPDL

To navigate on any screen:

  • press [Tab] to move through the fields from left to right, top to bottom
  • press [Shift]+[Tab] to move in reverse order through the fields, from right to left, bottom to top
  • press [Home] to go to the Nxt: field
  • use the arrow keys to move the cursor to a new position
  • use the mouse to select on the required field

6

Changing system within DMIS + Read more ...

DMIS integrates with the income security environment.

Staff can go to other systems, including Customer Archive and Retrieval (CAR), and keep the DMIS record in the screen header.

To switch systems:

  • key the name of the system in the Sys: field
  • press [Enter]

7

Scrolling screens + Read more ...

Many of the DMIS screens are scrolling screens where previous occurrences of the screen are held. The most recent information will show first.

The header will show the number of pages if there is more than 1 page for a screen.

To scroll thorough the screens:

  • press [PF8] to scroll forward
  • press [PF7] to scroll backward
  • over key the page number in the header with the wanted page number and press [Enter]

8

Creating a record in DMIS + Read more ...

A record is created in DMIS via:

  • interface with the:
    • host systems. See Explaining and actioning negative adjustments on the Assessment Results (AR) screen
    • Integrated Review System (IRS)
    • Compensation Management System (CMS)
    • Assurance of Support (AOS) system
    • Department of Veterans' Affairs (DVA) enabling clearances to be automatically actioned
  • a manual registration of a debt record within DMIS. See Create a new debt record manually on the Debt Management and Information System (DMIS)

Each debt has only 1 record. Customers who are jointly and severally liable for a debt, are added to the debt on the Multiple Liability (OPML) screen. See Debts where more than one person is liable for recovery.

9

Transaction History and DMIS + Read more ...

DIMS keeps a full history of all debt activities.

This includes the:

  • activity type
  • date
  • last officer to access the file
  • region
    • in most cases this is the office on the actioning officer's user profile
  • status
  • Activity Id

No DMN/RCV activity should map to the service centre. Debt Management Teams action all DMN/RCV activities.

Integrated Activity Management (IAM) keeps a transaction history:

  • Access this via the Activity History List (HAL) or Event Summary (ES) screens
  • Select an activity from the ES screen
    • The Activity History Details (HAY) screen will show

The Debt Action Summary (OPAS) screen shows a history of the actions taken on a debt. It lists:

  • actions that result in a change to the debt amount and/or outstanding balance
  • waivers, debt adjustments and repayments etc.

The Key Dates (OPKD) screen shows key dates and will show:

  • automatic notices sent
  • the debt raised date
  • the due date
  • earliest referral date to an External Collection Agent (ECA)
