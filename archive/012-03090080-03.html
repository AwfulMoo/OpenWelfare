This page contains the procedures on how Service Officers identify and resolve duplicate eHealth program (EHP) individual records.

On this page:

Identify and flag possible duplicate EHP records

A duplicate record is where two or more identifiers have been created for the same individual.

Only Tier 3 Service Officers can complete this procedure.

Identify and flag possible duplicate EHP records in the eHealth program

Table 1: This table describes how Tier 3 Service Officers identify and resolve records which have been identified as possible duplicates.

Step

Action

1

Search for the primary record + Read more ...

From the Healthcare Identifier EHP, search for the primary record being retained.

See Searching for records in the Healthcare Identifiers Service using the eHealth Program (EHP).

2

Select Amend + Read more ...

Select Amend in the Action column of the primary record.

  • If the primary record is a Healthcare Provider Identifier - Individual (HPI-I) record, the Amend HPI-I Details screen is displayed
  • If the primary record is a Public Officer/OMR record, the Amend Administrator details screen is displayed

3

Select Record Potential Duplicate + Read more ...

Select:

  • the Linked Organisation Details tab
  • Record Potential Duplicate

4

Insert details of the secondary record + Read more ...

Key the duplicate ID number of the secondary record in the Potential Duplicate ID Number field.

Add appropriate details in the Comments field. Example: Record identified as a potential duplicate.

Select Finished.

Details just entered display in the Recorded Potential Duplicates area of the screen.

5

Warning message + Read more ...

Does the Potential Duplicate Individual Number is already flagged as a potential duplicate warning message display?

  • Yes, this means that the secondary record is already recorded as a potential duplicate against the primary record and no further action is required. Procedure ends here
  • No, go to Step 6

6

Submit the details + Read more ...

Select Submit.

7

Review the details + Read more ...

Review the details on the confirmation screen. If the details are:

  • Correct, select Save. The confirmation screen displays
  • Incorrect, select Edit. The Linked Organisation Details tab is displayed. Repeat Steps 3 to 6 to correct the details

Resolve possible duplicate EHP records

Where at least one of the records identified as a possible duplicate is registered by a Data Source, Tier 3 staff complete the resolution process. Tier 3 staff have the relevant access to edit these records.

Only Tier 3 Service Officers can complete this procedure.

Resolve possible duplicate EHP records in the eHealth program

Table 2: This table describes how Tier 3 Service Officers resolve records which have been identified as duplicates.

Note: any Public Key Infrastructure (PKI) certificate linked to the secondary record must be removed prior to the resolution.

Step

Action

1

Search for the secondary record + Read more ...

From the Healthcare Identifier EHP, search for the secondary record.

See Searching for records in the Healthcare Identifiers Service using the eHealth Program (EHP).

2

Select Amend + Read more ...

Select Amend in the Action column of the primary record.

  • If the primary record is a Healthcare Provider Identifier - Individual (HPI-I) record, the Amend HPI-I Details screen is displayed
  • If the primary record is a Public Officer/OMR record, the Amend Administrator details screen is displayed

3

Remove any linked PKI certificates + Read more ...

Go to the Certificates Tab.

Remove any linked PKI certificates by selecting delete in the action column of the PKI certificate.

Note: if the PKI is left attached to the secondary record, the system does not allow the same Registration Authority (RA) number to be linked to the primary record after the merge.

4

Enter 2 at the end of all the secondary record details + Read more ...

While still in the secondary record, go to the:

  • Name Details tab
  • Address Details tab
  • Electronic details tab

Key in 2 at the end of the surname, end of the street name and the end of each of the electronic details listed.

Note: if a change to these details is not made, the system does not allow the merge as there is an existing exact match to other details.

Select:

  • Submit
  • Save

5

Search for the primary record + Read more ...

From the Healthcare Identifier EHP, search for the primary record being retained.

See Searching for records in the Healthcare Identifiers Service using the eHealth Program (EHP).

6

Select Amend + Read more ...

Select Amend in the Action column of the primary record.

  • If the primary record is a Healthcare Provider Identifier - Individual (HPI-I) record, the Amend HPI-I Details screen is displayed
  • If the primary record is a Public Officer/OMR record, the Amend Administrator details screen is displayed

7

Select the Resolve Duplicate link + Read more ...

Select the:

  • Linked Organisation Details tab
  • View Duplicate Status link
  • Resolve Duplicate link in the Action column of the appropriate record

8

Action the duplicate record + Read more ...

Is the record identified on the Duplicate Resolution screen a duplicate?

  • Yes:
    • Select Resolve as Duplicate from the Duplicate Resolution drop down menu
    • Add relevant details in the Comments field. Example: Record resolved as duplicate
    • Select Add
  • No:
    • Select Remove Duplicate Flag from the Duplicate Resolution list
    • Add relevant details in the Comments field. Example: Record not a duplicate

Select Add.

The following warning message displays at the top of the screen: You are about to resolve the EHP Individual as a duplicate record of <ID Number>. This action will close the Secondary EHP Individual and transfer all individual details to the Primary EHP Individual record. All HPI-Os linked to this record will have their link end dated.

9

Submit details + Read more ...

Select Submit.

Either the Confirm Amend HPI-I Details or Confirm Amend Administrator screen is displayed.

10

Review details + Read more ...

Review the details on the Confirmation screen.

If the details are:

  • Correct, select Save. A confirmation message displays
  • Incorrect, select Edit. The Linked Organisation Details tab is displayed. Repeat Steps 3 to 9 to correct details

11

Delete duplicate information and relink the PKI certificate + Read more ...

Access the primary record and check each tab for any duplicate information.

Delete any duplicated information.

While still in the primary record, go to the Certificates tab and relink the PKI certificate that was un-linked from the secondary record.

Remove duplicate flag from EHP records (Tier 3)

eHealth Program (EHP) records have been previously flagged as potential duplicates by a data source (for example Australian Health Practitioner Regulation Agency (Ahpra)), automatic matching report or other Tier 3 staff. Only Tier 3 staff have the relevant access to edit these records.

Remove duplicate flag from EHP records (Tier 3)

Table 3: This table describes how Tier 3 staff remove the duplicate flag from EHP records which have been identified as duplicates.

Step

Action

1

Search for one of the duplicate records + Read more ...

Search for either of the records that have been flagged as potential duplicates of each other from the EHP.

See Searching for records in the Healthcare Identifiers Service using the eHealth Program (EHP).

2

Select Amend + Read more ...

Select Amend in the Action column of the primary record.

  • If the primary record is a Healthcare Provider Identifier - Individual (HPI-I) record, the Amend HPI-I Details screen is displayed
  • If the primary record is a Public Officer/OMR record, the Amend Administrator details screen is displayed

3

Generate the Duplicate Resolution screen + Read more ...

Select the:

  • Linked Organisation Details tab
  • View Duplicate Status link
  • Resolve Duplicate link in the Action column of the appropriate record

The Duplicate Resolution screen is displayed.

4

Remove the duplicate flag and enter details + Read more ...

Select Remove Duplicate Flag from the Duplicate Resolution list.

Add relevant details in the Comments field. Example: record not a duplicate.

Select Add.

The You are about to remove the duplicate flag warning message displays at the top of the screen.

5

Submit details + Read more ...

Select Submit.

The Confirm Amend HPI-I Details or Confirm Amend Administrator screen displays.

6

Review details + Read more ...

Review the details on the confirmation screen. If the details are:

  • Correct, select Save. A confirmation message is displayed
  • Incorrect, select Edit. The Linked Organisation Details tab is displayed. Repeat Steps 3 to 6 to correct details

7

Identify and resolve duplicate EHP individual records if necessary + Read more ...

If required, complete:

  • Identify and flag possible duplicate EHP records (Tier 3)
  • Resolve possible duplicate EHP records (Tier 3)
