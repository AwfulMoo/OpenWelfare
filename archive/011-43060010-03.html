This page contains information on how to process Electronic Claim Lodgement and Information Processing Service Environment (ECLIPSE) claims, identify rejected and partially rejected ECLIPSE claims and find ECLIPSE claims in Search Analyse Investigate Network Tool (SAINT).

On this page:

Process Simplified Billing ECLIPSE claims

Identify rejected, partially rejected or paid ECLIPSE claims

Find ECLIPSE claim details in SAINT

Process Simplified Billing ECLIPSE claims

Table 1: This table describes how to process and finalise payments for Simplified Billing ECLIPSE claims once picked up for the Work Distribution, Allocation and Management System (WDAM).

Step

Action

1

Process ECLIPSE claim + Read more ...

Log on to Mainframe, key:

  • MQRI, press [Enter]
  • enter SB in the GIVE ME NEXT CLAIM field, press [Enter]

The system will return the next most urgent Simplified Billing electronic claim in FAPI (FAPC) screen. This may be either an ECLIPSE or EDI claim.

For access to WDAM, complete a WDAM Access request to Simplified Billing Specialist queue on the Resources page.

2

Assess ECLIPSE Claim + Read more ...

The pend processing screen will be displayed.

Review and apply assessing restrictions by following warning messages:

  • warning message must be actioned in order from left to right and further explanations can be viewed by placing the cursor under the return message and pressing [F1]
  • check patient history to determine if services should be paid or rejected
  • all assessing rules apply and must be followed, for example:
    • multiple operation rule
    • diagnostic service rules
    • restrictions
    • time dependencies
    • amputations
    • anaesthetics formulas/RVG
    • text field messages must be read - place the cursor under the item number and press [F4]

Note:

  • Assistant items must be processed last in a batch. Key [F8] to scroll through services
  • Ensure consistency between surgical item and assistant item text fields. If no surgical items have been claimed, manually calculate the assistant item using [F4] to calculate the items the assistant provider has endorsed in the text field
  • Where applicable, key the appropriate processing indicator in the PI field as per the return warning messages
  • When rejecting a service, the payee code (P) field must be changed from 8 to F, and the applicable RSN code keyed in the RSN field

1-PREV CON return messages. The Resources page contains the 1-PREV CON Matrix

3

Finalise ECLIPSE Claim + Read more ...

Once the claim has been assessed, select [Enter].

Note: a claim cannot be completed until all errors have been resolved. Once all errors have been resolved, the system will return the message 'NO ERRORS OK TO PAY'.

4

Pay ECLIPSE Claim + Read more ...

  • Over key FAPC to FPPI and select [Enter] to update the claim status
  • Over key FPPI to FPPP and select [Enter] to pay the claim. The claim will return ‘PAID OK’
  • Select [Enter] to return to the Online Claims Management screen to select the next claim and repeat above

Identify rejected, partially rejected or paid ECLIPSE claims

Table 2: This table describes how to identify ECLIPSE claims rejected, partially rejected or paid claims by Medicare in mainframe.

Step

Action

1

Confirm how claim was transmitted + Read more ...

Was the claim transmitted via the ECLIPSE channel and initiated by the health professional or practice staff?

  • Yes, go to Step 2
  • No, go to Step 10

2

Identify the health professional + Read more ...

Identify the health professional or practice staff performing a provider security check. See Perform telephone security check for Medicare health professionals.

3

Identify the claim + Read more ...

Get the following details from the health professional or practice staff to identify the claim:

  • Minor ID
  • Claim ID (ECLIPSE claim IDs have a % suffix, for example 12345%)
  • Date of lodgement

Can the health professional or practice staff provide the above details?

  • Yes, go to Step 8
  • No, if the site cannot locate the claim ID, go to Step 4 and request the additional details

See the Resources page for a list of billing agents and private health insurers.

4

Requesting additional details to identify a claim + Read more ...

Get the following details from the health professional or practice staff to identify the patient:

  • Medicare card number and Individual Reference Number (IRN)
  • patient's first name
  • patient's surname
  • patient's date of birth (if required)

Can the health professional or practice staff provide the above details?

  • Yes, go to Step 5
  • No, ask them to phone back when they have the required details. Procedure ends here

5

Identify ECLIPSE claims - NHSI screen + Read more ...

To identify ECLIPSE claims on the Medicare claims history (NHSI) screen:

  • key control line NHSI, cardnumber, patient name or IRN
  • select [Enter]

Patient history displays, including rejected services for ECLIPSE claims only if Medicare has not rejected all the services in the claim.

6

Confirm details + Read more ...

Confirm the details of service:

  • date of service
  • provider details
  • service items

To confirm the claim ID and date of lodgement press:

  • PF2-CLAIM ID to identify claim ID. ECLIPSE claim IDs have a % suffix for example 12345%
  • PF4-FEE/SRC to check the date of lodgement

Use patient history modifiers to refine the search on the history screen, it can display as either:

  • NHSI,cardnumber,patient name, or
  • NHSI,cardnumber,IRN, or
  • NHSI,cardnumber,DOS/DDMMYY. This control line returns all services on the patient history with the specified date of service

Does the claim ID have a suffix % ?

  • Yes,
    • if the claim is paid, payee code 8 displays. To confirm the claim status go to Step 9
    • if the claim has been rejected, payee code F displays. To search the status of claim go to Step 7
  • No, confirm the claim type with the health professional:
    • private health insurers or billing agents submit Manual or EDI claims. Go to Step 10
    • for ECLIPSE patient claims, tell the health professional to contact the eBusiness Service Centre. Procedure ends here

See also: Indicators, codes, modifiers and control lines for claims processing in Medicare.

7

Check claim details + Read more ...

Are the claim details available on the patient history?

  • Yes, PFI on the Medicare reason code to identify the reason for the rejection.
    • Tell the health professional or practice staff the reason for the rejection
    • Procedure ends here
  • No, the claim may have been health fund rejected or failed to transmit. Go to Step 8

8

Key control line + Read more ...

  • key control line FCEI, Fund Minor ID, Claim ID, DOL=ddmmyy
  • select [Enter]

The system returns details of the status of the claim.

Example:

FCEI, ABC12345, 00001%, DOL=311012

SMITH HEATH FUND

A CLAIM ID DOL - REF CLM SC STATUS RSN OPER REF

00001% - 311012 - 1 REJECTED - ABCD 311012 5555X 428

STATUS – OPER - REFERENCE

01 REJECTED - OPER - 020712 5555X

02 FUND REJ - OPER - 020712 5XXXX 064

03 HUB 2GET - OPER - 020712 5XXXX 064

04 IN PEND - OPER - 020712 5XXXX 326

05 D/E COMP - OPER - 010712 5555X 212

06 FUND ASS - OPER - 010712 FXXXX 000

Go to Step 9 to confirm the status.

9

Confirm claim status + Read more ...

  • If the claims displays PAID-SD or PAID-LD, tell the health professional or practice staff that the claim is paid:
    • PAID-SD - Paid same day
    • PAID-LD - Paid latter day
    • Procedure ends here
  • If the claim displays FUND REJ, this status is applied to either:
    • claims if all services are rejected by Medicare, or
    • AG/SC claims if all services are rejected by the private health insurer. Go to Table 3 to view the details in SAINT

10

Direct for further assistance + Read more…

Direct the health professional or practice staff to the relevant private health insurer or billing agent for help.

Find ECLIPSE claim details in SAINT

Table 3: This table describes how to find ECLIPSE claim details in SAINT. Identify ECLIPSE claims by using the Minor ID or Registration number and date of lodgement.

Step

Action

1

Confirm how the claim was received + Read more ...

Confirm with the caller the claim was either:

  • transmitted via the ECLIPSE channel
  • submitted under the Simplified Billing Agreement or Scheme Arrangement

To identify the provider, see Perform telephone security check for Medicare health professionals.

Note: Simplified Billing Arrangements do not include ECLIPSE Patient Claims. Refer enquiries about ECLIPSE Patient Claims to the eBusiness Service Centre.

2

Get details from the health professional/practice staff + Read more ...

Get the following details from the health professional or practice staff to identify the claim:

  • minor ID
  • date of lodgement
  • patient Medicare number

3

Log on to Saint and search the transaction + Read more ...

  • Go to Search Analyse Investigate Network Tool (SAINT):
    • Process page > Table 2 to log on, then
    • Process page > Table 4, Steps 3 to 9 to search for a transaction
    • Select the transmission ID hyperlink to view

4

View the PROC STAT + Read more ...

To view the status of an Inpatient Medical claim (IMC) go to the Message Details page:

  • Select the Summary heading, 3 green buttons display:
    • Outbound Msg
    • Proc Stat, and
    • Inbound Msg
  • Select PROC STAT. From this screen, staff can see the EXT_STAT_CDE to determine the status of the claim:
    • COMPLETE /SUCCESS, the claim assessment is complete and reports are available
    • MEDICARE_REJECTED, the claim has been rejected by Medicare and 4 digit reason code displays. Select Outbound Msg and [+] to view the status code and text
    • HEALTH_FUND_REJECTED the claim is rejected by the health fund and the 4 digit reason code displays. Select Outbound Msg and [+] to view the status code and text
  • Tell the health professional or practice staff the claim status

Procedure ends here.
