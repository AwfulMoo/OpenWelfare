Amend Responsible Officer (RO) or Organisation Maintenance Officer (OMO) record in the Healthcare Identifiers (HI) Service 012-03090040

This page contains processes for a Service Officer to amend the Responsible Officer (RO) or Organisation Maintenance Officer (OMO) details in the Healthcare Identifiers (HI) Service. See RO and OMO roles for more information on RO and OMO responsibilities.

Initial steps to amend RO or OMO record in the eHealth Program (EHP) database

Table 1

Step

Action

1

Third party + Read more ...

Is the Service Officer acting on behalf of another person known to the HI Service (RO or OMO), that is, in response to a telephone request?

  • Yes, complete Handle ‘Act for’ call in Healthcare Identifiers (HI) Service, go to Step 2
  • No, go to Step 2

2

Search for Individual + Read more ...

From the Search for Individual screen, select Amend in the Action column of the relevant record.

3

Amend Public Officer/OMR screen + Read more ...

From the Amend Public Officer/OMR screen, select Amend in the Action column of the relevant record.

4

Select relevant procedure + Read more ...

From the Amend Public Officer/Organisation Maintenance Role, select the relevant tab and make the changes requested.

Go to the relevant amend procedure:

  • Add or amend name details in the EHP
  • Add EOI details in the EHP
  • Amend personal details in the EHP
  • Add or amend address details in the EHP
  • Add or amend electronic communication details in the EHP
  • Amend linked organisation details in the EHP
  • Amend certificate information in the EHP

5

Validation errors + Read more ...

Note: this step follows on from an amend procedure.

Select Submit.

Are there validation errors?

  • Yes, correct errors as prompted, then select Save and go to Step 6
  • No, go to Step 6

6

Finalise + Read more ...

Check that the details on the Confirm Amend Public Officer/Organisation Maintenance Role screen are correct. If the details are:

  • Incorrect or additional updates are required, select Edit to return to the Amend Public Officer/Organisation Maintenance Role screen. See below processes and select the relevant update required
  • Correct, select Save. A confirmation message displays

Add or amend RO or OMO name details in the eHealth Program (EHP) database

Table 2

Step

Action

1

Navigate to Name Details tab + Read more ...

Select the Name Details tab of the Amend Public Officer/Organisation Maintenance Role screen.

The Name Details tab is displayed listing current details.

2

Add, amend or delete name details + Read more ...

To:

  • add a new name, select Create New Name. All fields are cleared to allow data entry. Use the processing date as the start date
  • amend existing name details (not provided by the Australian Health Practitioner Regulation Agency (Ahpra))
    • Select Amend in the Action column of the row containing the name to be amended and end date the existing record with the processing date
    • Select amend to save
    • Select Create New Name - use the processing date as the start date
  • delete an existing name (not provided by Ahpra):
    • Select Delete in the Action column of the row containing the name to be deleted.
    • This action will delete the selected record. To continue, select OK to delete the name. See Step 5 of the Initial steps to amend RO or OMO record in the EHP table

Note: a name should only be truly amended and not end-dated if it is a clear typing error by a Service Officer. If there is a requirement for a new name set to be added, the old name set should be end dated.

3

Enter details + Read more ...

Enter the information to be added into the relevant fields. The following fields are mandatory:

  • Family Name
  • Name Usage

Note:

  • Name Usage Start Date (defaults to today’s date)
  • Start Date Accuracy Indicator (defaults to Day, month and year are accurate)

4

Select Add + Read more ...

Note: where the name set supplied (including prefix) by the applicant is different to the name set recorded in EHP, the name set supplied is added as the Preferred Name. The Name Usage field is entered as:

  • Registered Name (Legal Name) if the name supplied is a verified name, for example, confirmed by either 100 points of evidence of identity (EOI) or matches their known customer name
  • Other Name (Alias) if the name supplied is different to their 'Registered Name (Legal Name)', for example anglicised names

See Step 5 of the Initial steps to amend RO or OMO record in the EHP table.

Add or amend RO or OMO EOI details in the eHealth Program (EHP) database

Table 3:

Note: evidence of identity (EOI) details already recorded in the HI Service for healthcare providers requested by Ahpra cannot be amended or deleted. New EOI details can be added to the record in this case. If the RO or OMO is not an Ahpra registrant their EOI details can be deleted and re-created.

Step

Action

1

EOI Details tab + Read more ...

Select the EOI Details tab of the Amend Public Officer/Organisation Maintenance Officer screen.

2

Create New EOI + Read more ...

From the EOI Details tab, select Create New EOI.

Key the information to be added into the relevant EOI Documents field. The EOI Document(s) field is mandatory.

Enter in EOI where the OMO is known customer or has provided at least 100 points of EOI – otherwise the option the Service Officer should select is ‘EOI not performed by HI Service.

As required – the EOI type field should be updated as required (for example, No EOI supplied, EOI supplied, known Services Australia customer etc.)

3

Enter Evidence of Identity (EOI) document details + Read more ...

Select Add for the EOI document to be added to the Recorded Evidence of Identity table.

Enter details of all the EOI provided with the application and select Add.

Note: once the Add button has been selected, the EOI can be amended or deleted if incorrect at this point only (prior to saving) by using the Amend and Delete in the Action column. Once the action has been submitted and saved, EOI cannot be amended or deleted.

Return to Step 5 of the Initial steps to amend RO or OMO record in the EHP table.

Amend RO or OMO personal details in the eHealth Program (EHP) database

Table 4

Step

Action

1

Personal Details tab + Read more ...

Select the Personal Details tab of the Amend Public Officer/Organisation Maintenance Officer screen.

Note: Australian Health Practitioner Regulation Agency (Ahpra) records may not be amended. Contact Tier 3 if the date of birth appears to be incorrect on an Ahpra record.

2

Personal Details tab + Read more ...

On the Personal Details tab, enter the information to be corrected into the relevant fields (overtype existing details where necessary). The following fields are mandatory:

  • Date of Birth
  • Date of Birth Accuracy Indicator (defaults to Day, month and year are accurate)
  • Sex

Note: the above fields require evidence of the change, such as a birth certificate. See Gender Guidelines Implementation.

3

Determine additional actions + Read more ...

If the RO or OMO:

  • is to be registered as a Healthcare Provider Individual - Individual (HPI-I), go to Step 4
  • is identified as a potential replica record, go to Step 5
  • needs a date of death recorded, go to Step 6

4

RO or OMO practising an Ahpra profession + Read more ...

Is the RO or OMO practising a profession covered by the Ahpra?

  • Yes, stop processing the application and contact Tier 3 for further advice. The RO or OMO should already have a HPI-I in the system, and records may need to be merged
  • No, select Enable as Provider. The Provider Details tab is made available, with the Amend HPI-I workflow active

Complete Amend Healthcare Provider Identifier – Individual (HPI-I) record in Healthcare Identifiers (HI) Service

Go to Step 7.

5

Potential Replica + Read more ...

If the RO or OMO is identified as a potential replica:

  • Select Record Potential Replica
  • Complete Identify and resolve replica Individual Healthcare Identifier (IHI) in Healthcare Identifiers (HI) Service

Go to Step 7.

6

Date of death + Read more ...

If the RO or OMO needs a date of death recorded:

  • Select Retire
  • Update details in the enabled Date of Death field

Go to Step 7.

7

Continue procedure + Read more ...

Return to Step 5 of the Initial steps to amend RO or OMO record in the EHP table.

Add or amend RO or OMO address details in the eHealth Program (EHP) database

Table 5

Step

Action

1

Address details + Read more ...

Select the Address Details tab of the Amend Public Officer/Organisation Maintenance Officer screen.

2

Add new, amend or delete existing address + Read more ...

To:

  • create a new address, go to Step 3
  • amend existing address (not provided by the Australian Health Practitioner Regulation Agency (Ahpra)):
    • select Amend in the Action column of the row containing the name to be amended and end date the existing record with the processing date
    • select amend to save
    • go to Step 3
  • delete an old address:
    • select Amend in the Action column of the row containing the name to be amended and end date the existing record with the processing date
    • select amend to save
    • return to Step 5 of the Initial steps to amend RO or OMO record in the EHP table

3

Address details + Read more ...

  • Select Address Details tab
  • The Unstructured Address Indicator tick box will be automatically selected. See Address formatting in Healthcare Identifiers
  • Enter the address in the Address Search box, starting with the number of the address. The rapid addressing tool will predict the address and display it as the number and street name starts to be entered. If the address that appears is the correct address, select it
  • The following information will pre-populate into the relevant fields:
    • Address Lines
    • Suburb/Town/Locality
    • State
    • Postcode
  • Select Address Purpose
  • Address Start Date (defaults to today’s date)
  • Start Date Accuracy Indicator (defaults to Day, month and year are accurate)
  • Select Add

4

Preferred address indicator + Read more ...

Select preferred address indicator for the new address keyed.

Select Add.

Return to Step 5 of the Initial steps to amend RO or OMO record in the EHP table.
