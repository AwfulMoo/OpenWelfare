Process RTS Correspondence

Step

Action

Identify the RTS mail + Read more ...

Identify the RTS mail type.

If the RTS mail relates to:

  • Medicare Public and Lifetime Health Cover (LHC) Letter, go to Step 4
  • Continence Aids Payments Scheme (CAPS), reassign the work item in PaNDA to Miscellaneous (MISC_CAPS_RTS_STMT)
  • Debt recovery letters, go to Step 2
  • Child Dental Benefits Schedule (CDBS)
    • See the Setting a CDBS Return Mail Flag indicator table and complete the Steps, then
    • Go to Step 4
  • Provider registration, LSPN/diagnostic imaging equipment, bulk bill, simplified billing, and 90 Day Cheque cancellation letter to provider:
    • Mark the work item as Complete in PaNDA
  • Provider pathology, reassign the work item in PaNDA to Pathology (PROV-PATHCOL-RENEW)
  • Australian Immunisation Register (AIR), go to Returned mail for AIR
  • Australian Organ Donor Register (AODR), go to Action AODR returned mail
  • Aged Care Services, reassign the work item in PaNDA to Aged Care (AC_Enquiry)
  • Veterans Affairs Processing (VAP) or Department of Veterans Affairs (DVA)
    • Mark the work item as Complete in PaNDA

Save work item + Read more ...

Save the Work Item:

  • Right mouse click on the document
  • Select Save As
  • Go to a chosen or temporary folder
  • Rename the File name as the customer ‘reference number’ or ‘name’
  • Select PDF (*.pdf) from the Save as type dropdown
  • Select Save
  • Go to Step 3

3

Send the work item to business area + Read more ...

Open Microsoft Office Outlook.

Send the work item to the relevant business area:

  • Select New Email
  • Attach the work item
  • Send to the relevant business area:
    • Medicare debt recovery, for debt recovery letters
    • Medicare Compensation Recovery, HIC.Compensation.Recovery, for RTS Medicare Compensation Recovery cheques
  • Select Official as the email classification
  • Select Send
  • Add a note to the PaNDA file to confirm the RTS mail is sent to the relevant business area, stating: 'RTS mail sent to <business area> on DD/MM/YYYY, NFA'
  • Change the status of the PaNDA file to Complete

Procedure ends here.

4

Locate customer in CDMS + Read more ...

  • Log on to CDMS
  • Select Amend Consumer Details
  • The Amendment Search screen displays
  • Conduct a Basic Search with the Medicare card number if on the returned mail
  • If no matches conduct an Address Search

Has the customer been identified?

  • Yes, go to Step 5
  • No, go to Step 6

5

Check the customer's address in CDMS + Read more ...

Is the CDMS address different to the claimant's address shown in the return mail?

  • Yes, go to Step 6
  • No, go to Step 7

Add a note in PaNDA file + Read more ...

    • RTS Mail - No action taken
  • Go to Step 8

Set the return mail indicator in CDMS + Read more ...

In CDMS, set a return mail indicator on the claimant's record.

8

Finalise the work item in PaNDA + Read more ...

  • If a Medicare card number is present, add to the Work Item Details - Medicare number section
  • Update the status of the work item to complete
  • For help, see Work Optimiser for staff
  • Procedure ends here

Table 2

Step

Action

1

Identify RTS Cheque + Read more ...

Identify the RTS cheque type.

If the RTS cheque relates to:

  • Medicare Public, go to Step 2
  • Medicare Compensation Recovery RTS cheque, go to Step 2 in Table 1
  • Other, escalate to the Program Support Officer for assessment
  • If the details need to be forwarded via email to the relevant area, see Step 2 in Table 1

2

Search cheque details in mainframe + Read more ...

Open and Log on to the mainframe.

Search for the cheque details:

  • key BBCI,NB?R,cheque number in the control line. Replace ? with the relevant state code the cheque was sent to. See Cheque processing in Medicare for a list of state codes
  • select Enter

Do the cheque details show?

  • Yes, go to Step 4
  • No, the 1-NOT FND error shows. Go to Step 3

3

Search cheque details - other state codes + Read more ...

Search for the cheque details using the other state codes:

  • Key BBCI,NB?R,cheque number in the control line. Replace ? with the other state codes. See Cheque processing in Medicare for a list of state codes
  • Select Enter

The claimant's Medicare number is the number beside CARD NO on the BBCI screen. A digit must be added to the end of the Medicare number as the BBCI screen only displays 9 digits. Patient name can be found on cheque statement.

Do the cheque details show?

  • Yes, go to Step 4
  • No, the 1-NOT FND error shows. Go to Step 10

4

Identify if the cheque is covered under 90 Day Pay Doctor Cheque Scheme + Read more ...

Health professionals eligible under the 90 day scheme are:

  • GPs registered for the 90 Day Pay Doctor Cheque Scheme
  • Specialists (including radiologists), consultant physicians, pathologists and GPs for claims they submitted electronically

Allied Health practitioners, Dentists, Optometrists, Midwives and Nurses are not eligible for the 90 Day Scheme.

All electronically transmitted claims are identified with a CLID line directly above the service line.

5

How to identify cheques under the 90 Day Pay Doctor Cheque Scheme + Read more ...

Access the Medicare claims history:

  • Key NHSI, card number, patient's IRN or first initial in the control line
  • Press Enter

Identify the date of service for the payment

Do not use modifiers when searching history for CLID line.

Is the CLID line directly above the service line?

  • Yes, go to Step 6
  • No, go to Step 7

6

CLID line directly above the service line + Read more ...

Determine if the claim was submitted electronically by the provider:

  • Press [Shift] + [F2] to view Claim Type
  • The CT (Claim Type) column will display the transmission type PCO - Patient claim online and easy claim

Is the claim showing PCO?

  • Yes, go to Step 8
  • No, go to Step 9

7

CLID line is not above the service line + Read more ...

If the claim is not an electronic claim submitted by the provider, but the provider is a GP, the Service Officer must access Provider Directory System (PDS) to check if the GP is registered for the original 90 day scheme.

  • Access and log into PDS
  • From the menu, select Provider Location then View
  • Key the first six digits of the provider number in the Provider stem field
  • Key the first alpha of the provider number in the Provider Location field
  • Select Search

On the provider location detail screen, the bottom right view will display if the provider is registered for 90 Day Pay Doctor Cheque Scheme (Locn Reg for 90d Cheque).

Is the provider registered for the 90 Day Pay Doctor Cheque Scheme (Locn Reg for 90d Cheque)?

  • Yes, go to Step 8
  • No, go to Step 9

8

  • select Tab to go to the next field

  • select Enter

  • select Enter

Go to Step 11.

  • Select Tab to go to the next field
  • Key RTS CHQ and universal log on or operator number (P number)

Confirm cheque status + Read more ...
