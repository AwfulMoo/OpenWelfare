How to identify an Add child to CCS activity + Read more ...

CCS activities are allocated to via Work Optimiser.

  • Open the work item in Process Direct
  • To to the Claim Application screen
  • Select to open Claim Overview information

The activity will contain:

  • only information about the child to be added, and
  • the customers declaration

The claim does not include:

  • Income Estimate,
  • Activity Test Details

The activity will include the keyword ADDCH.

Is the activity an Add child to CCS claim?

  • Yes, see Table 2, Step 1
  • No, and it is a full claim for CCS, see Processing Child Care Subsidy (CCS) claims

1

Lodgement of claim or documentation + Read more ...

Before submitting a shortened claim, all required documents must be uploaded.

The only evidence that may be requested is a:

  • Commonwealth statutory declaration:
    • stating why it is unreasonable for the child to be left unsupervised, and
    • confirming a suitable adult is not available to provide the necessary care
  • National Disability Insurance Scheme (NDIS) participants number, or
  • Evidence to support children with permanent or non-permanent conditions or undergoing assessment (non NDIS participant). The child has, or is undergoing an assessment for, one or more conditions:
    • listed in Schedule 1 of the Minister’s Rules, as diagnosed by a medical practitioner
    • listed in Schedule 2 of the Minister’s Rules, as diagnosed by a psychologist
    • referred to in Part 2 of Schedule 3 to the Disability Care Load Assessment (Child) Determination 2010 by a medical practitioner

Service Centre Service Officers

If the customer is lodging evidence for a submitted claim:

  • Scan the document onto the customer’s record
  • Access the customer’s record in Process Direct
  • Select Transactions
  • Select the Claim ID hyperlink. The Claim Application page displays
  • Select Tasks
  • Update the status of documents required to Provided
  • Select Save
  • Select Status. If all required documents have been provided and the Claim Status is ‘On hold’, select to update. Selecting In Process from the dropdown menu. Select the Save icon

Note: if the Service Officer changes the claim status the work item will be assigned to them. Return the claim to the pool if not trained to process the claim.

  • Go to the work items list
  • Select
  • Check the box for the relevant work item
  • Select Unassign to return the claim to the pool
  • Check the Tasks status within the claim if the customer:
    • gets advice that documents are missing, and
    • states they have already been sent
  • If the document is still showing as Required, select Documents icon to view documents:
    • scanned at a service centre, or
    • uploaded by the customer
  • If the documents have been lodged, update the status to Provided as above
  • If not displaying via Documents, request the customer upload the documents again

Take a verbal claim using Assisted Customer Claim if a customer:

  • contacts to add a child to their CCS, and
  • is unable to lodge using the digital channels

2

Locate Claim + Read more ...

Locate work item in Process Direct.

Select the work item to go to the Claim Application screen.

When a claim is submitted online or via the Assisted Customer Claim (ACC), a CCS Social Application (SOA) new claim work item is generated on the customer’s record. The work item contains all the claim information provided by the customer online or entered into the ACC.

Note: Check if the customer has submitted more than one CCS claim:

  • Go to the Claim Application screen
  • Select Transactions icon

If more than one claim is made, process them in the order they were submitted. This ensures the latest data is applied. Each claim triggers a notice to the customer if there is a change to eligibility or entitlement. A notice is not issued if there is no change between claims.

3

Claim Application screen + Read more ...

The icons at the top of the page display the customer's information held on record..

Before an online or assisted customer claim is lodged, update all personal information.

If the system identifies a change of environment is required for the customer or the child (due to a past or current change of address) an error message will display. I Initiate an interstate transfer via an error message.

4

Review/update Relationships + Read more ...

A claim can only be lodged where a customer is already FTB current for the child. The child will already be linked to the customer.

The following changes cannot be done with a claim:

  • change of circumstance updates, and
  • relationships updates

Step

Action

Start processing + Read more ...

After reviewing the claim, select Process to start the new claim.

2

Claim Processing Screen + Read more ...

Icons display in the same order as the Claim Overview information.

Select an icon to display tables that contain provisional and/or confirmed data. The information automatically loads from the online claim or ACC. If the information did not load correctly, an error, warning or message will display.

The following icons display:

  • About Me
  • My Child Care
  • My Finances
  • My Residence
  • All

The claim will only update:

  • child education details
  • child immunisation details
  • children in Claim
  • exceptional Circumstances (if a child age or school status exemption is requested)

Action errors, warnings and/or messages displayed and validate the evidence provided.

When validating evidence, check:

  • the relevant table(s) is checked and updated
  • correct outcome displays on the Entitlement screen
  • To avoid duplication, select to check the current information in the claim. Tables not updated within the claim present as ‘No Data’
  • Use to switch between the customer and child records to view/update the child claim information Select Stimulate at any time to lock in updated data and identify errors or warnings to action

3

Requesting documents + Read more ...

If more information is required to assess the claim, request documents from the customer. For example, evidence to support an age/school status exemption

The claim is to be made Not Effective if:

  • a CCS claim has come off hold after a request for information has already been sent, and
  • the required documents have not been provided, the claim is to be made Not Effective. Go to Step 5

Can the claim be processed without further documentation/evidence?

  • Yes, go to Step 6
  • No, go to Step 4

4

Request further documentation from the customer + Read more ...

To request documents, issue a Q888 - Request for information - CCS and Additional CCS letter

  • Record Notes to indicate the information requested
  • Put the combined claim or standalone application on hold for 22 days (14 days plus 8 days postage)

Procedure ends here.

5

Claim NEF because request for information has previously been sent but documents have not been provided + Read more ...

  • Select Status
  • Select
  • Select Not Effective from Possible Status drop down menu

Make 2 outbound attempts to advise the customer of the decision. For more information, see Making an Unfavourable decision (CLK).

Record details on a DOC with:

  • CCS NEF: s85AA to 85GB FA Act 1999
  • Claim ID:
  • Claim date: [insert date Claim completed by the customer]
  • Children: [insert names of child/ren for which claim is made NEF]
  • Reason: required documents/information not provided therefore no decision can be made about the customer's entitlement. It is not possible for a customer to ask for a review of a decision about their entitlement. The claim is taken not to have been made. The customer can ask for a review of the decision which made the claim not effective. If the claim is still found to be not effective, a new claim will need to be lodged. A new claim is made not effective if requested documents were lodged late and this decision cannot be reviewed

Procedure ends here.

6

Errors + Read more ...

Errors, warnings and messages will continue to display after selecting Simulate until all issues have been resolved.

Select the error message to be taken to the relevant screen. Action all data items or evidence validation to finalise the claim.

Digital assistant (DA) (Roxy): Select the Help icon at top right to launch VA. Drag the error message into the VA for help.

For help with errors and using the DA, see Process Direct navigation, common screens and functions.

Action:

  • data validations
  • assess evidence
  • Select Stimulate to load the data. The Assess button will display
  • Select Assess
  • Correct any errors, warnings or messages displayed
  • Assessment Results screen will display

Have all errors been resolved and/or Assessment Results is displaying expected outcome?

  • Yes, go to Step 7
  • No,
    • check SharePoint Current Issues for escalation instructions
    • Put the claim on hold
    • Investigate the error
    • Until all errors have been resolved and/or the Assessment Results is displaying the expected outcome, procedure ends here
