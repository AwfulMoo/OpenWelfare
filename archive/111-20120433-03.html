Complete a quality check

Table 1

Step

Action

1

Work item allocated + Read more ...

The Quality and Support Tool (QST) allocates work items for checking to the checker.

  • Access the OPRA home page
  • Select QST
  • Select Open at the bottom of the screen
  • Once the tool is open, select Enable content
  • Do not select ‘Edit Workbook’ (this will lock all other users out of the tool)
  • The tool will open on the QC work page
  • Select Give Me Work
  • Open the item in PaNDA by selecting Open Work Item in Panda

Service Officers can also open the item in PaNDA by selecting Open Work Item In Panda.

2

Log on to required systems for your program + Read more ...

For example, CDMS, PDS, AIR system, PIP system etc.

3

Perform quality check + Read more ...

Check the:

  • information on the source documents is complete
  • information on the form matches the information on the online system
  • associated documents such as letters are uploaded to the correct system

Record the outcome in the Quality Control Status field in the QST.

Select either:

  • Error Free - auto-populates the error code
  • Contains Errors - select an error code from the list
  • Add Comments where needed. Note: when providing additional feedback using the comments box, do not add any personal or identifying information

4

Record an outcome + Read more ...

  • Select Record Outcome to email the result to the original processing Service Officer and their Team Leader
  • The QST creates a draft email in Microsoft Outlook

Does the work item contain an error?

  • Yes, go to Step 5
  • No, Send an email advising quality checking identified no errors, to:
    • the original processing Service Officer
    • their Team Leader
      Note: use the correct security marking (Official) for the email

Once the email is sent, QST returns to the homepage.

Select Give Me Work.

5

Action critical/non-critical error + Read more ...

Is the error a critical error?

  • Yes, Correct critical error(s) immediately and provide feedback to:
    • the original processing Service Officer
    • their Team Leader
  • No, it contains only non-critical error(s). Provide feedback on the non-critical error and instructions to fix the non-critical error to:
    • the original processing Service Officer
    • their Team Leader

Note: use the correct security marking (Official) for the email.

Once the email is sent, QST returns to the homepage.

Select Give Me Work.

Service Officer - submit work for a pre-check

Table 2

Step

Action

1

Load an item into the QST for pre-check + Read more ...

  • Access the OPRA home page
  • Select QST
  • Select Open at the bottom of the screen
  • Once the tool is open, select Enable content
  • Do not select ‘Edit Workbook’ (this will lock all other users out of the tool)
  • Select the Create Pre-Check option from the ribbon at the top of the homepage to manually submit work

The Pre-check page displays. Fill in all required fields on the page.

  • Document ID(s) are not always available. This information is not mandatory
  • Once the Work Item ID is entered, the Submit Pre-Check button is enabled
    Note: Work Item ID and Document IDs are found on the Work Item Details screen in PaNDA. The entire URL of a document within FileNet can be copied and pasted directly into the Document ID field. Only the Document ID will remain in the field once pasted
  • Select Submit Pre-Check

2

Receive feedback and finalise work + Read more ...

Service Officer will receive feedback via an email for each item of work submitted.

  • If the Quality Control Status is 'Error Free', no further action is required
  • If the Quality Control Status is 'Contains Errors', the Service Officer reviews the errors and the Quality Checker's feedback. Correct the work as required and finalise.

Quality Checker - perform a pre-check

Table 3

Step

Action

1

Access the QST for pre-check + Read more ...

  • Access the OPRA home page
  • Select QST
  • Select Open at the bottom of the screen
  • Once the tool is open, select Enable content
  • Do not select ‘Edit Workbook’ (this will lock all other users out of the tool)
  • Tool will open on the QC work page

2

Check for specific agents + Read more ...

To specify an agent(s) processed work for pre-check:

  • Enter the Service Officer’s universal logon into the Logon field. Note: any alphabetical characters need to be in capital
  • In Check Type, select Pre Check
  • Once the details for the Service Officer (SO) are entered select Add Lock. This locks all work completed by the Service officer, and only their work will be delegated to the Quality Checker
  • The SO’s name appears in the list of ‘Agents locked to you’ The SO (agent) remains locked to the Quality Checker until they are removed or the QST reset overnight

3

Remove agent(s) from the list of locked agents + Read more ...

  • To remove the list of locked agents, select the name(s) of the Service Officer (agent) or Select all
  • Once the names are highlighted, select Cancel all

4

Receive pre-check for quality checking + Read more ...

  • Select Give Me Pre-Check Work for Logon ID

5

Complete a quality check + Read more ...

Assessment

  • QST presents a work item to the Quality Checker with relevant information displayed
  • Open the item in PaNDA by selecting Open Work Item In Panda

Quality Checking

  • Once all information has been checked, select the appropriate Quality Control Status check box based on the review of the documentation
  • If Error Free is selected, an explanation of 'Error-free' and the relevant code are populated below the Comments field
  • If Contains Errors is selected, the Explanation field contains a dropdown list. It populates the 'Error Code' and 'Critical/Non-critical' classification once an explanation reason is selected. Quality Checkers can enter up to 9 separate error descriptions

Record an outcome

  • See Step 4 in Table 1
