Notification types

Notification

Description

<User ID> has requested work multiple times in the last 5 minutes.

A direct report has selected Ready 3 times in less than 5 minutes.

<User ID> has requested additional work with 15 or more actionable work items in their inbox.

A direct report has selected Ready with 15 or more open work items assigned to their worklist

<User ID> has handed off more than 10 work items within the past 12 hours.

A direct report has unassigned or reassigned more than 10 work items in the past 12 hours

<User ID> has not finalised work item that has had in-excess of 10 handoff actions recorded.

A direct report has unassigned or reassigned a work item that already has 10 or more handoffs recorded.

<User ID> has finalised a work item on first touchpoint.

A direct report completed a work item without handoff.

<User ID> has finalised work item with more than 5 touchpoints recorded.

A direct report completed a work item that has more than 5 touchpoints recorded.

<User ID> has finalised more than <X> work items during their regular scheduled hours.

A direct report completed more than X number of work items in a day.

The number of work items can be set/changed using the Customise Notification option.

Worklist search criteria

This table outlines the worklist search criteria that appears as the default setting for the following Saved Searches:

  • My Work search is available to all staff
  • My Team Work is available to Team Leaders and Analysts
  • All Work is available to Analysts

    Item

    Description

    Default display

    Work Item ID

    This field can be used to search for a specific work item ID.

    Multiple work items can be entered to refine the search results.
    • My Work
    • My Team Work
    • All Work

    Work Type

    This field can be used to search for a collection of work items of the same type.

    Search can search using a work type description, for example, CCS claims, or the work type ID if known.
    • My Work
    • My Team Work
    • All Work

    Work Item Attributes

    This field can be used to search for work items with specific attributes.

    Users can refine the results by entering ‘||’ into the field. For example, ‘PROCESS_TYPE || SHC’
    • All Work

    Source Status

    Users can restrict the search results to display work items by a selected status in the source system.
    • All Work

    Work Item Status

    Users can restrict search results to work items with a specific status in WLM. For example, OPEN, or Hold Retained (HTU)
    • My Work
    • My Team Work
    • All Work

    Source System

    Used to filter work items that have originated in a specific source system, using a drop down menu.
    • All Work

    Allocation Indicator

    Users can filter the results based on work items that are allocated, unallocated, or both.
    • All Work

    Employee Responsible

    Staff logons can be entered to search for work that is assigned to them.

    Multiple logons can be entered to search for work assigned to groups of staff at one time.
    • My Team Work
    • All Work

    Priority From/Priority To

    These fields are used to search for work items within a numerical priority range.
    • All Work

    Age in Days From/Age in Days to

    Used to search for work items based on the number of days since their creation in WLM.
    • All Work

    Read

    Used to filter the search results between ‘Read’ and ‘Unread’ work items, using a drop down menu.

    Work items update to ‘Read’ when the Employee Responsible has navigated to Work Item Details.
    • My Work
