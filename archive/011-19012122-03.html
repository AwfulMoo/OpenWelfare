Process an electronic Stoma or ParaQuad claim + Read more ...

Before processing the claim, add claim details into the Stoma and ParaQuad Claim Spreadsheet located at R:\NAT\HLTHSERDEL\AGECALLIEDH\STOMA_and_Paraquad\Ostomy Program\Claims Transmission Details

  • Log on to PBS Claims Processing System (via eBusiness system)
  • Select Query Claim Period
  • Key the association’s approval ID into the Approved Supplier Number (ASN) field
  • Key the relevant claim period into the Claim Period Number (CPN) field
  • Select Search
  • Select the radio button beside the claim search results then select Open Claim Period

The PBS Online Claims Processing System automatically flags these claims with the Hold Payment indicator and sets the Payment Due Date to the day following the claim upload. The PBS Payment ID’s will not populate and the payment status will be ‘Withheld’.

Untick the Hold Payment box on the Query Claim Period screen. Do not tick the Hold Payment box again

Once the Hold Payment indicator is unticked, the system will calculate the actual Payment Due Date using the Claim Package Received Date (the date the claim is received via HPOS) and allocate PBS Payment ID’s during the overnight payment batch job.

Check the Claim Package Received Date when unticking the Hold Payment indicator, and prioritise processing accordingly.

After the overnight payment batch process, the Claim Period Status will be Generate Statement and it will remain as this until the due date is reached which is 15 days from the date the claim is received.

2

Quality assurance checking + Read more ...

During the automated claims assessment process, the PBS Online Claims Processing System automatically allocates reason codes for the items that do not meet requirements. The reason codes are located at the bottom window pane of the Ostomy Item Details screen.

See Resources for a link to the Stoma & ParaQuad Reason Codes and fix instructions to assist with quality assurance.

Check the first and last items and a minimum of 4 or maximum of 10 items (randomly selected) in between.

View the items in the system:

  • On the Query Claim Period screen, select View prescriptions
  • Select Open prescriptions to edit the claim data if required. The following fields can be edited:
    • Serial number
    • Patient identifier
    • Date of Supply
    • Item code
    • Quantity

Note: quality assurance must be completed on one day (not across multiple days).

If any items identified in quality assurance have reject reason codes allocated to them, check the supporting paperwork to confirm the electronic data is correct. For example, if rejection reason code 298 - 'Date of supply was invalid' is allocated to an item, check the paperwork to see if it has been included. If it has, amend the item in the system and resubmit so that it is paid. If it hasn’t then leave rejected.

If adjustments are made to the claim, they are automatically picked up during payment processing that night and will have a new PBS Payment ID allocated, which appears on the association’s statement. If adjustments are done over multiple days, a new payment ID is allocated for each day. To make reading statements as easy as possible, it is preferred that all adjustments are done on the same day to ensure minimal payment IDs are allocated.

Step

Commence processing + Read more ...

The PBS Online Claims Processing System automatically flags these claims with the Hold Payment indicator. Processing staff identify claims submitted via HPOS through an automated report emailed daily to the Stoma and ParaQuad processing team (Aged Care Allied Health Branch) and Operational Policy and Legacy Programs Section, (Pharmaceutical Benefits Branch).

Before processing the claim, add the claim details into the Stoma and ParaQuad Claim Spreadsheet located at R:\NAT\HLTHSERDEL\AGECALLIEDH\STOMA_and_Paraquad\Ostomy Program\Claims Transmission Details

To retrieve the claim:

  • Log on to PBS Claims Processing System (via eBusiness system)
  • Select Query Claim Period
  • Key in the association’s approval ID into the Approved Supplier Number (ASN) field
  • In the Channel field, select CTS
  • Key the relevant claim period into the Claim Period Number (CPN) field
  • Select Search
  • Select the radio button beside the claim search results and select Open Claim Period

Note: claims submitted using version 1.1 should only be retrieved once the paperwork arrives and all forms are signed and complete.

The system automatically sets the Payment Due Date to the day after the claim upload. The PBS Payment ID’s do not populate and the payment status is ‘Withheld’.

Untick the Hold Payment checkbox on the Query Claim Period screen. Do not tick the Hold Payment box again.

Once the Hold Payment indicator is unticked, the system will calculate the actual Payment Due Date using the Claim Package Received Date (the date the claim is received via HPOS) and allocate PBS Payment ID’s during the overnight payment batch job.

Check the Claim Package Received Date when unticking the Hold Payment indicator, and prioritise processing accordingly. Note: if claim paperwork is late to arrive it needs to be quality assured and approved by the delegate quickly, before payment is released.

After the overnight payment batch process, the Claim Period Status will be Generate Statement and it will remain as this until the due date is reached:

  • 15 days for version 1.1
  • 7 days for version 2.0

Once the due date is reached, payment will be released and the statement will be generated. The status will then change to Complete.

  • Reason code 434 – claim paperwork is inaccurate or incomplete. This is a reject reason code

To add the manual reason code/s:

  • in the reason code table, select Add
  • the Reason Code Search pop up window displays
  • enter the reason code number (346 or 434) into the Reason Code field, or search for by selecting Operator in the Reason Search Code drop down box

If adjustments are made to the claim during quality assurance they are automatically picked up during payment processing that night and will have a new PBS Payment ID allocated, which appears on the association’s statement. If adjustments are done over multiple days, a new payment ID is allocated for each day. To make reading statements as easy as possible, it is preferred that all adjustments are done on the same day to ensure minimal payment IDs are allocated.

Quality assurance checking of version 2.0 claims + Read more ...

Check a minimum of 4 or maximum of 10 items (randomly selected) allocated with a reason code for exceeding the maximum schedule amount – these include reason codes 144, 158, 246 and 106 (for ParaQuad items only).

Search for these items in the system:

  • Go to Query Claim Period screen
  • Key in the association’s approval number into the Approved Supplier Number (ASN) field
  • Key the claim period number into the Claim Period Number (CPN) field
  • Key the reason code number into the Reason Code field
  • Select Search
  • The search results display in the bottom window pane
  • Randomly select a minimum of 4 and maximum of 10 items

View each item:

  • Select the radio button then View Prescription
  • On the Ostomy Item Details screen, check the Quantity and Price fields are within acceptable amounts
  • If any quantities appear excessive, and appear to be an error (e.g. 1000 instead of 10) contact the association to check they have requested the correct amount
  • Reject or amend the item and include Comments to support a possible phone enquiry. Include user ID and date

If adjustments are made to the claim during quality assurance they are automatically picked up during Payment Processing that night and will have a new PBS Payment ID allocated, which appears on the association’s statement. If adjustments are done over multiple days, a new payment ID will be allocated for each day. To make reading statements as easy as possible, it is preferred that all adjustments are done on the same day to ensure minimal payment IDs are allocated.

Note: quality assurance must be completed on one day (not across multiple days) and financial delegate approval must be obtained within 24hours of completing quality assurance.

Process a manual Stoma or ParaQuad claim

Table 5

Step

Action

1

Process a manual Stoma or ParaQuad claim + Read more ...

Once the manual claim has been registered into the PBS Claims Processing System, add the items into the claim:

  • Go to the Query Claim Period screen
  • Key in the association’s approval ID and the claim period number created
  • Select Add Transactions to add the items
  • Enter the claim reference number into the Claim Reference field
  • Leave the Payment Category as General
  • Enter the serial number range into the Start Serial Number and End Serial Number fields
  • Select New
  • The claim details display
  • Select Claim and then Key Prescriptions to enter the item details

2

Key the ostomy item details + Read more ...

Key the ostomy item details:

  • Serial number
  • Patient identifier (this is the SAS entitlement number)
  • Date of supply
  • Item code (this can be searched for using the Item Name hyperlink search function)
  • Quantity

Key in any optional fields included on the paperwork such as:

  • Medicare number, DVA, Reciprocal Health Care Agreement or Passport number
  • Additional supply information
  • Patient name and date of birth

Note: the Item Code field is case sensitive and requires a leading 0 (zero). Please ensure that Caps Lock is on, otherwise it will be rejected. If searching for the item using the Item Code hyperlink, it must be a valid item code for the link to operate.

Enter only valid characters into the system. Using invalid characters will result in Reason Code 186 – 'Invalid data in the following field'. The Service Officer will need to re-key the information.

In the Given Name and Family Name fields only A-Z characters, and the following special characters are allowed:

  • – (dash)
  • ‘ (apostrophe/single quote)
  • ( ) left and right brackets
  • . (full stop)
  • <a blank space>

Select Submit for system assessment of the information entered. Associated reason codes display at the bottom of the page. See Resources for a link to the Stoma & ParaQuad Reason Codes and fix instructions for more information.

There are two reason codes which can only be generated manually by operators:

  • Reason code 346 – the ostomy claim form was not signed by and/or dated by patient. This is a warning reason code
  • Reason code 434 – claim paperwork is inaccurate or incomplete. This is a reject reason code

To add the manual reason code/s, select Add in the reason code table. The Reason Code Search pop up window will display. The reason code number (i.e. 346 or 434) can be entered directly into the Reason Code field, or they can be searched for by selecting Operator in the Reason Search Code drop down box.

To enter more items in the claim, select New and repeat the above.

The claim status will remain at QAI In Progress until all items have been keyed. Keying of items in the same claim can be done over a number of days. Manual claims can be processed to completion right up to the payment due date.

Once all items have been keyed, the claim is ready for financial delegate approval.

Note: it is preferable that the delegate check is performed before Complete is selected.

3

Financial delegate + Read more ...

Prepare paperwork for financial delegate approval. See Table 6 for instructions on how to prepare a Stoma or ParaQuad claim for the financial delegate. Delegate approval needs to be completed before the due date is reached (as set by the date of lodgement).

For more informationsee Financial delegation checking of a Stoma or ParaQuad claim.

When the delegate has approved payment of the claim, save all financial delegation documents in the Delegation folder under the appropriate association.

If the Delegate confirms adjustments are required, the Service Officer must complete them on the same day. This ensures they will be automatically picked up during payment processing that night. It cannot be adjusted the next day because the system will allocate payment information and pricing that night.

4

Complete claim + Read more ...

Once all items have been keyed, select Complete.

A warning pop-up message will display:

  • ‘602 – this will close the claim transaction. Are you sure you want to continue?’
  • Select Yes

The claim status changes to QAI Complete. The claim is now ready for financial delegate approval.

Once the payment due date has been reached (as set by the date of lodgement), the system automatically picks up the claim, processes payment and generates the payment statements.

Once payment is made, the Claim Period Status changes to Complete.

5

Finalising the claim + Read more ...

  • Complete the Stoma and ParaQuad Claims spreadsheet. See Table 7 for instructions on how to complete Stoma and ParaQuad claims spreadsheet
  • Archive the claim. See Table 8 for instructions on how to archive Stoma or ParaQuad claims

How to prepare a Stoma or ParaQuad claim for the financial delegate

Table 6

Step

Action

1

Financial delegate + Read more ...

Claims for an EL1 financial delegate must be less than $1,000,000. Claims greater than this amount must be approved by a National Manager.

Present the financial delegate with:

  • the quality assurance work undertaken on the claim
  • a copy of the Payment Summary screen showing the expected payment amount and date
  • a completed Stoma & ParaQuad Financial Delegate Approval Sheet. See Resources for a copy of the approval sheet.

Please ensure the delegate is aware of a 24 hour approval required for claims submitted using version 2.0.

2

Prepare the Stoma & ParaQuad Financial Delegate Approval Sheet + Read more ...

Locate the Stoma and ParaQuad Financial Delegate Approval Sheet on Resources or in the National shared drive in the Stoma folder.

Complete the following fields on the approval sheet:

  • Association name and approval number
  • Claim period numbers
  • Software version used (version 1.1 or version 2.0)
  • Date of lodgement
  • Expected payment amount
  • Service Officer name and user ID
  • A list of the items checked in quality assurance processing

3

Add details of the quality assurance processing + Read more ...

Provide the financial delegate with the quality assurance processing undertaken on the claim. This is required to support spending approval.

For version 1.1 claims and manual claims:

  • Scanned copies of the claim supply forms randomly selected for quality assurance and the supporting screen shots of the item details in the system. The first and last items should be included in the sample
  • All information such as patient details and item details should correspond on the supply forms and screen shots
  • If any amendments were made to an item, include a brief overview on the approval sheet.

For version 2.0 claims:
Note: delegate approval is required within 24 hrs after completing quality assurance

  • A list of the items randomly selected for quality assurance which were allocated with reason codes relevant for exceeding the maximum schedule amount. These include reason codes 144, 158, 246 and 106 (for ParaQuad items only)
  • Include the serial numbers checked and the respective reason code allocated to the items on the list
  • If amendments were made to item(s) following discussion with the association, include a brief overview. Screen shots are not required for this

4

Seek financial delegate approval + Read more ...

Email the package to the financial delegate to seek spending approval on the claim.

For version 2.0 claims, monitor to ensure the delegate provides their approval within 24hrs after quality assurance has been completed.

The delegate may request further information about the claim.

See Financial delegation checking of a Stoma or ParaQuad claim for more information.

Register a bulk adjustment for a Stoma or ParaQuad claim

Table 9: A bulk adjustment can only be processed once it has been identified that an association has been overpaid or underpaid (through post-payment integrity checking or via an enquiry from the association). Operational Policy & Legacy Programs Section, Pharmaceutical Benefits Branch must be consulted prior to any bulk adjustments being raised.

Step

Action

1

Register bulk adjustment + Read more ...

  • Go to PBS Claims Processing System (via eBusiness)
  • Select Bulk Adjustments
  • Select Register Bulk Adjustments
  • Key in the association’s approval ID in the Approved Supplier Number field and select Validate
  • Key in the relevant claim period in the Claim Period Number field
  • Key in the date of receipt of the claim as the Reference Number
  • Key in the start date of the Bulk Adjustment into the Bulk Adjustment Start Date field. This should be discussed first with the association
  • Select General from the Payment Category drop down list
  • Key in the amount into the Bulk Adjustment Amount field
  • Select if the bulk adjustment is a Debit or Credit from the drop down list
    • Debit - to recover money from the association due to an overpayment
    • Credit - to provide money to the association due to an underpayment
  • Key in the percentage for each claim – this could be 100% from the next claim or be split over a few claims. This must be agreed with the association
  • Key in any comments supporting the bulk adjustment into the Comments field. Comments could include brief details about why the adjustment is required and details of the discussions the association about the adjustment
  • Select Register from the Action Bulk Adjustment drop down list
  • Select Save

An information box displays, advising the bulk adjustment has saved successfully.

The bulk adjustment must be approved by a line manager/team leader. It cannot be approved by the registering user.

The staff member’s user ID will automatically be logged in the bulk adjustments for audit trail purposes.

2

Print and send Adjustment notice + Read more ...

After the bulk adjustment is confirmed in the system, manually generate and send a statement outlining the adjustment to the association:

  • Select the Adjustment Notice Debit hyperlink. The notice displays.
  • Print and send to the association
