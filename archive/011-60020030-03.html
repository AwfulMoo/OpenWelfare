Assess the claim

Table 1

Step

Action

1

Assess application for refund + Read more ...

Check claim form for :

  • Claimant name and address
  • If marked, what type of claim the claimant is seeking (not mandatory)
  • Check for Claimant's signature in Declaration

Note: if these are not complete reject and return to Claimant. If Claimant is under the age of 16 reject and return to claimant.

2

Assess Pharmacy documentation + Read more ...

Check PRFS/Receipts for :

  • Patients name and details
  • Entitlement or Medicare number
  • Pharmacy approval number
  • Date of dispensing
  • Cost of medication
  • Pharmacist signature

or

Check Tax Invoice/ Patient history for:

  • Patients name and details
  • Entitlement or Medicare number
  • Pharmacy approval number
  • Date of dispensing
  • Cost of medication
  • Pharmacist signature
  • Endorsed by pharmacist ‘Paid in Full’

We do not accept till/register receipts or carbon copy receipts. These must be rejected.

If missing any of this information, reject and return to claimant.

When the Safety Net data is:

  • received and keyed within the following 14 days, the refund is processed
  • not received within the 14 days, the refund is returned to the claimant with a Patient Refund Rejection letter
  • received and not keyed within the following 14 days, the refund should be pended for another 14 days in WLM (PaNDA)

3

Calculate refundable amounts + Read more ...

Over the threshold refunds:

  • Check all documents for duplicate entries or private scripts (these are not to be included in the calculation)
  • Combine total amount for all items
  • Subtract the threshold amount from combined total

This becomes our refundable amount.

Note: date SN/CN was issued may need to be amended. See Table 2

4

Determine Script quantity + Read more ...

For over the Threshold with variable amounts:-

  • Working in descending order from most recent to oldest
  • Subtract each script from the refundable amount
  • Repeat this process until the refundable amount is reached or exceeded

For concession to Threshold refunds where scripts are the same price.

  • Divide the refundable amount by the cost of each script, which leaves the quantity of scripts to refund
  • Working in descending chronological order
  • Subtract that amount of scripts from the refundable amount
  • Repeat this process until the refundable amount is reached or exceeded

Note: if there are several scripts on the same day, refund the most inexpensive script that does reach or exceeds the refundable amount.

Change Date of Safety net reached

Table 2

Step

Action

1

Ascertain date safety net was reached + Read more ...

Check all documents for duplicate entries or Non-PBS/private scripts (outside public hospital dispensing).

2

Calculate threshold + Read more ...

Over the threshold refunds:

  • Check all documents for duplicate entries, private scripts or SN20DR with $0.00 (these are not to be included in the calculation)
  • Combine total amount for all items
  • Subtract the threshold amount from combined total

This is the amount that we need to subtract from the PRFs to find the date the threshold was met.

3

Determine date is met + Read more ...

  • Working in descending order from most recent to oldest
  • Subtract each script from the total amount
  • Repeat this process until only the threshold amount remains

Note: if there are several scripts on the same day, utilise the most inexpensive script that reaches or does not exceed the threshold amount.

Example: the threshold is $244.80 with 36 scripts at $6.80 that would be date threshold was met. However if the scripts are $5.80 then the threshold would be met at 42 scripts totalling $423.60.

Anything within $6.80 of threshold, between $238.01 and $244.80 will be the date required.

or

The threshold is $1457.10 then the threshold date will be the closest script to the threshold without exceeding the threshold. This can be between $1414.61 and $1457.10.

Note: from 1 July 2022 the Safety Net thresholds have been lowered and are not retrospectively applied. Refunds are not payable for items supplied before 1 July that exceed the new threshold.

4

Change date in mainframe + Read more ...

  • Enter fastpath RNFA
  • Enter the SN/CN entitlement number in Entitlement field at bottom of page
  • Press Enter
  • Change the date of issue to the correct date the safety net was met
  • Press F5 to update
  • Escape/Exit out of this screen and Create a new claim to start processing

Create a new claim

Table 3

Step

Action

1

Check for duplicate claims + Read more ...

  • Key RPRW PATIENT REFUND - EFT PROCESSING MENU and press [ENTER]
  • Choose 3 or RPRE PATIENT REFUND - CLAIM ENQUIRIES screen to ensure there is no duplicate claim
  • Key claimant Medicare Card Number in the Claimant Medicare Number field (if claimant’s Medicare card is omitted from the form, use claimant’s surname, given name and locality)
  • Amend from date to the 1 January of the current or previous calendar year
  • Check all names listed on the claim to ensure the refund has not been paid under another claimant
  • If a duplicate claim does not exist, key fastpath RPRI PB PATIENT REFUND - IDENTIFY CLAIM and press [Enter]

If any of the items in the claim are over 2 years old, email PBS Helpdesk and request confirmation whether a previous refund was processed for the items. This is because data is only visible on RPRE for the current 2 year period.

Note: if a duplicate claim has been keyed, confirm all attached documentation has been entered as part of the previous claim. If all information is keyed, reject the claim and return with a Z1680 PBS item refund is not payable letter.

If a previous claim was keyed and any receipts in the current claim were omitted, key a new claim for the unpaid receipts.

2

Key claim details + Read more ...

Key the following claim details:

  • Date of lodgement
  • Number of items

3

Identify claimant + Read more ...

Identify the claimant with 1 of the following:

  • Medicare card number (if known)
  • claimant's name and address

For all claims where the claimant is not a POA/Administrator/Executor for the patient, go to Step 5.

Note: Power of Attorney (POA)/Administrator/Executor claims.

POA and Administrator claims can be processed until the expiration date of the orders, if one is set by the party, until it is cancelled by the individual or the individual's representative. This is until the individual dies or if the individual becomes incompetent or incapacitated, unless the POA was set to be durable. Otherwise, the claimant would need to have the Executor documentation.

Service Officers processing a PBS refund signed by a POA/Administrator/Executor (where no supporting POA/Administrator/Executor documentation is provided in the claim) should refer to CDMS to confirm that the POA and banking details match information recorded in CDMS. Service Officers who do not have access to CDMS should:

  • add a processing note in the WLM stating 'POA/Administrator/Executor, no access to CDMS'
  • unassign the work item in the WLM with Reason Proficiency Raised
  • select Submit (this will allocate the work to an experienced Service Officer in the WLM/)

If POA/Administrator/Executor details and any banking details provided with the claim are in CDMS, and the claimant is listed as a POA for the patient, the claim can be processed. Go to Step 5.

Otherwise, these details will need to be updated before the claim can be processed.

4

Update details for POA/Administrator/Executor + Read more ...

If the claimant has not provided sufficient documentation to demonstrate that they are the POA/Administrator/Executor, return the documentation with a completed Z1682 We need more information to refund your PBS claim letter.

Note: POA documents must be original or a certified copy of an original.

If sufficient POA/Administrator/Executor documents have been provided in the refund, and the POA or Executor details need to be updated in CDMS, this can only be done by Service Officers with the established or experienced skill tag.

Service Officers not trained in the process to update details for POA/Administrator/Executor, should unassign the work item with the reason Proficiency Raised so the trained Service Officers can update the CDMS record.

Any updates to POA, Administrator or Executor details should be made in accordance with the following procedures:

Power of Attorney, Guardianship, Administrative Orders and Authorised Representative for Medicare

Medicare claims for deceased persons

Where the POA/Administrator/Executor has provided bank account details to pay the claim, Service Officers with the established or experienced skill tag can add new banking details in CDMS. These updates should be made in accordance with the following procedure:

Amend customer details in the Consumer Directory Maintenance System (CDMS) for Medicare

Service Officers should add a note in PaNDA stating that a new POA was recorded in CDMS or POA/banking details have been updated.

Updating banking details for Executors

When an executor claims on behalf of a deceased patient, and provides new bank account details to pay the claim, these can be added in CDMS. This can only be done by PBS Service Officers with the established or experienced skill tag, and a clear note must be added in CDMS.

The Service Officer should also input an end date for the bank account of 6 months in the future.

If the executor advises that the bank account in CDMS can be used, but it has been end dated in CDMS, Service Officers with the established or experienced skill tag can amend the end date to 6 months in the future.

5

Check Medicare number + Read more ...

When the claimant’s Medicare number keyed is:

  • valid,
    • go to Step 6
  • invalid:
    • a warning message displays
    • correct the error if necessary or delete the Medicare number keyed
    • key the claimant's name and address in the appropriate fields to initiate a name search

6

Check address + Read more ...

When the required claimant is displayed, check the address listed on the claim form against the address displayed on the screen. If the:

  • address matches
    • select the required claiming
    • press [Enter]
    • go to Step 7
  • address is different
    • key the correct address on the screen
    • key [F6] to update the address

7

Check bank details + Read more ...

The claimant's account details appear on the Mainframe screen when stored in CDMS.

Claims are rejected if:

  • the banking details provided with their application does not match CDMS information (if PB132 Form pre Nov 2019 is submitted)
  • no banking details are recorded in CDMS

Send the claimant the Z1682 We need more information to refund your PBS claim letter requesting they update their bank account details with Medicare.

Note: if a claim was submitted by a patient's POA/Administrator/Executor, and the banking details were provided, the bank account details can be updated as per Step 4.

If an error returns and the banking details appear to be correct:

  • the EFT source must be P
  • press [F5] to confirm
  • press [F6] to continue

Service Officers should add a note in PaNDA, stating that banking details have been updated due to error occurring. Otherwise, leave the EFT sources C, press F5 to confirm and F6 to continue.
