OPDL

Debt List screen

Shows debts that belong to a customer and includes all debts that are:

  • undetermined
  • determined
  • protected, and
  • finalised

If there is more than 1 debt record, a list will display in the following order:

  • Undetermined and AoS protected. If there is more than 1 undetermined and/or AoS protected debt, the list will be from the earliest to the latest date registered
  • Determined. If there is more than 1 determined debt, it will list from the earliest to the latest date raised
  • Finalised. If there is more than 1 finalised debt, it will list from the earliest to the latest date finalised

Use the Select View field to view:

  • the order in which payments will be applied if there is more than 1 debt outstanding, key 'ILY'
  • parked debts, key ‘PRK’
  • deleted debts, key ‘DEL’

OPGI

General Information screen

This screen is a display screen that:

  • provides a quick reference or summary of each debt
  • is used as a look-up point for general enquiries
  • provides details if a debt is Interest Charge eligible
  • cannot be changed, except for the Debt ID field. Note: customers can have more than 1 Debt ID

The OPGI is a debt specific screen, so there will be an OPGI screen for each debt.

OPMM

DMIS Main Menu screen

This is the main menu for DMIS. It is a reference point to the main functions and secondary menus within DMIS. For each function, both the short title and the screen title show.

The OPMM screen has a modifiable Debt ID field. This field is another option for identifying the record request.

If a Customer Reference Number (CRN) has more than 1 debt selected, the user will stay on the OPMM screen.

To view multiple debts:

  • select the required debt (if the Debt ID is known), or
  • navigate to the Debt List (OPDL) screen

There are 4 menus, the first being OPMM. Use:

  • OPDM to update debt specific details
  • OPCM to update customer-specific (rather than debt-specific) details for recovery arrangements

OPAC

Additional Component screen

Although this screen is no longer used, it is still viewable in DMIS and shows:

  • Interest Subsidy (Supplement Loans only). Shows the Financial Supplement Loan (FSL) Interest Subsidy amounts for FSL debts. Staff can add and update the Interest Subsidy. Calculate this manually:
    • the Student Financial Supplement Scheme (SFSS) closed from 1 January 2004. Recovery of all outstanding FSLs has been the responsibility of the Australian Taxation Office (ATO) since December 2005
  • Administrative Charge. Shows the administrative charge amount (if it has been applied) and the applicable date (both future and past). Changes are possible when either the administrative charge applicable date is present:
    • there has been no application of Administrative charges since 30 June 1994
  • Judgment Interest and Costs. Fields to change and show any judgement interest amount added to the debt:
    • from December 2007, raise Judgement Interest debts as separate Public Governance, Performance and Accountability Act (PGPA) debts

OPAD

Add Debt screen

Use this screen to add new debt records manually where an undetermined debt record is not auto-generated in:

  • the host payment
  • compensation
  • Assurance of Support (AoS), or
  • review system

The system allocates a Debt ID when it creates the debt record. The minimum details needed to create a debt record are:

  • the authority
  • date of event, and
  • debt benefit type

The OPAD screen is blank and unprotected when first accessed. After a debt is created via this screen, use the Modify/Assess Debt (OPMA) screen to make changes.

OPRF

Recovery Fee screen

Record the Recovery Fee details on this screen and include:

  • the amount of the debt to which to apply the Recovery Fee
  • calculated Recovery Fee amount
  • decision to apply Recovery Fee (yes/no)
  • Reason field for applying or not applying the Recovery Fee
  • date the first request letter is sent
  • date the first request letter is returned
  • date of second contact

The Recovery Fee also shows on the below screens:

  • Modify/Assess Debt (OPMA)
  • Add Debt (OPAD)
  • General Information (OPGI), and
  • Account Payable (OPAOD)

If using the OPDA screen to change the debt amount, recalculate the Recovery Fee also. This is so that it is still 10% of the proportion of the debt.

Note: only apply Recovery Fees when the debt is due to the income from personal exertion of the customer.

OPAF

Assessment Finalisation screen

Used to finish any DMIS assessment that contains a determination. It also shows errors or warnings resulting from the data input. The acceptable input values are:

  • 'Y', activity finished
  • 'Q', activity finished and sent for Quality Online (QOL) checking
  • 'N', activity not finished, do not proceed with the assessment

When making a determination the activity will flow to the OPAF screen. The user can go to a different screen without finalising the activity:

  • key the screen name in the NXT field
  • press [F16] (Shift+F4)

OPAOD

Account Payable screen

Used to issue Account Payable, or send a manual advice of debt.

Once a debt is determined, details on this screen cannot be changed.

To:

  • reissue the advice, see Viewing or reissuing a letter or electronic message
  • suppress the Account Payable, go to the Key Dates (OPKD) screen

OPCA

Component Amount screen

This screen auto-populates and is used to breakdown components of a debt by financial years, mainly for tax adjustment purposes. No further action is needed if the system estimate is accepted as the calculated amount.

Manually update OPCA if a debt is:

  • manually added, or
  • the calculated amount differs from the system estimate

The dates used in the financial year breakdown are based on the whole entitlement periods. For example, the entitlements that would have been paid in the given financial year. The sum of component amounts must be equal to the debt total amount.

General debt raising information has:

  • a list of common errors for tax processing in relation to debts, and
  • action(s) for how avoid these problems and stop the creation of Manual Follow-ups (MFUs)

OPKD

Key Dates screen

Records and show key events in the life of a debt. A key event is:

  • raising of a debt
  • sending any 1 of the following:
    • a manual Account Payable
    • Account Payable/advice of debt
    • a debt adjust letter for Family Tax Benefit (FTB) and Child Care Benefit - formal (CCF) re-reconciliation debts
    • a reminder letter
    • a first reminder letter (debts raised prior to 10 December 2007)
    • a second reminder letter (debts raised prior to 10 December 2007)
    • a cash arrangement confirmation letter
    • a payment not received letter
    • a tax garnishee warning letter
  • customer has an end date for an address and Services Australia sends a:
    • first or second reminder, or
    • a tax garnishee warning letter
  • referring a customer to the Australian Taxation Office (ATO) for possible garnishee action
  • a debt is due for auto write-off
  • a debt is due for possible referral to External Collection Agent (ECA)
  • a debt is excluded for referral to ECA
  • a debt is due for auto waiver
  • reversing a waiver

The system auto-records and shows key events as they occur. Staff can record the date a manual advice of debt issues, if no previous advice has been sent.

Note: process CCS debts in Process Direct, not in DMIS.

OPMA

Modify/Assess Debt screen

This screen relates to a specific debt and is used to change details on an existing debt record. Updates can be made to undetermined and most determined debt records. Staff cannot update the OPMA screen if:

  • a temporary write-off is in place
  • the debt is finalised
  • a protected Assurance of Support (AoS) debt
  • a compensation debt
  • the customer in the header is not the primary debtor
  • an activity with provisional updates to the debt is on the Activity List (AL) screen
  • there is a reconciliation debt. Staff can update the below fields:
    • Date of Notification
    • Date of Event, and
    • Was debt solely due to system or clerical error?

OPML

Multiple Liability screen

This screen relates to a specific debt. Users can:

  • record
  • view, and
  • change the level of liability for customers who are jointly and severally responsible for the same debt

Jointly and severally means that each person listed against this debt has joint responsibility for the amount. That means that if 1 member pays $1 towards the debt, then the joint debt liability reduces by $1.

For example, the joint liability will reduce to $900 if 3 people are jointly liable for a $1,000 debt and 1 person pays $100. It is possible for only 1 person to pay off all of a liability. This can be without the contribution of the other liable parties.

Use the third party withholdings function on the Withholding Instruction (OPWH) screen when:

  • a debt is on a duplicate record
  • for third party withholdings

OPSD

Statement of Account screen

The Statement of Account:

  • is an auto-Batch Laser Advice letter that gives debtors a statement about their debt(s)
  • contains details about:
    • payments received
    • new debts raised, and
    • any change to debt balances for all debts

Generate statements on request from a customer. Update the period of the statement manually. Use the OPSD screen to:

  • manually produce a statement
  • display the dates covered of statements previously sent

Statements of Account are no longer auto-issued. Staff can issue:

  • manual statements from the OPSD screen for any period in which the customer had outstanding accounts
  • a Statement of Account for any period up to 5 years (60 calendar months)

OPAS

Debt Action Summary screen

This screen:

  • shows a list of actions on the debt record since it was raised
  • is a view only screen, except for the Debt ID field. This is the only updateable field

Use the [F23] and [F24] to scroll through debts on a customer's record where more than 1 is selected via the Debt List (OPDL) screen.

OPDA

Debt Amount screen

Used to record any recalculations of the debt amount. Recalculations of the debt amount cannot be recorded if the debt is:

  • undetermined
  • determined written-off
  • parked
  • held
  • finalised
  • a family assistance reconciliation debt
  • a Family Tax Benefit (FTB) non-lodger debt, or
  • the Activity List (AL) screen has an activity with provisional updates to the debt

There is no limit to the number of recalculations of the debt amount. The system keeps a record of each recalculation. Multiple recalculations show in the Date Changed field. They range from the most recent date to the earliest date. The below fields are protected:

  • Estimated Amount
  • Date raised
  • Original Calculated Amount
  • Date registered

The original calculated amount will not be shown as a recalculated amount. The Reason Changed and Date Changed fields become protected once coded. No changes will be possible for that occurrence.

The Activity Management Reference 'AMR' shows for all adjustments. This is the number of the activity that resulted in the debt adjustment.

The Source AMR field shows for FTB non-lodger and reconciliation debts where an adjustment occurs. This field shows the source ID of the most recent re-reconciliation result that caused the change in the debt amount.

OPDD

Delete Debt screen

From the March 2002 release, the OPDD screen is protected. The manual deletion of debts is not available.

OPDEL

Deleted Debt List screen

Lists deleted debts. If a deleted debt is selected from this screen, the Action Summary (OPAS) screen will show. Staff cannot go to any other screen for details specific to the deleted debt.

OPDM

DMIS Debt Update Menu screen

OPDM is the secondary menu that lists functions for updating specific debt details on DMIS. The screen includes a Debt ID field that may be input. This field is another option for identifying the record required.

OPPR

Process Direct Repayment Status Screen

Allows users to check the repayment source for their Child Care Subsidy debts, in DMIS.

RVR

Review Registration screen

Used to:

  • add manual reviews
  • delete existing manual reviews
  • display any existing manual reviews

Manual reviews:

  • add or delete 1 at a time
  • apply to undetermined and determined debts
  • must have at least 1 debt to be coded
  • can be deleted. Key ‘D’ in the selection field
  • auto-delete if:
    • the review date is reached, and
    • an activity is set up

See How to code and action a manual review.

OPPI

Penalty Interest screen

From 4 December 1999, penalty interest is not added to new debts. The OPPI was used to:

  • cease application of penalty interest to a debt
  • display a history of penalty interest notices sent and the resulting action
  • modify data that was provisionally (for example, had not been determined) updated via this screen and the OPPIA screen

This screen will be protected if:

  • there is no existing record of penalty interest
  • penalty interest has ceased
  • the debt is protected, over recovered or finalised
  • the customer in the header is not the primary debtor
  • an activity containing provisional updates to the debt is on the Activity List (AL) screen
  • a Debt ID has not been input

Each occurrence of this screen is protected and available to view only.

OPPIA

Add Penalty Interest screen

From 4 December 1999, penalty interest is not added to new debts. If penalty interest is currently applied, it will continue to accrue.

OPRH

Repayment History screen

Displays all repayments and adjustments to repayments made against a debt including:

  • cash
  • withholdings
  • Payment Card payments
  • any other monies received
  • plus and minus adjustments

OPRH is mainly a display screen, updates are not allowed. However, a positive or negative adjustment is possible by selecting the payment record. The information displayed on this screen originates from the following sources:

  • ESSentials. Including amounts received by:
    • Australia Post using the Payment Card
    • BPAY
  • the payment systems
  • the OPAR screen
  • when a direct debit instruction is sent to ESSentials from the OPVD screen

Payment details not passed via ESSentials or the host systems will not show on the OPRH screen.

OPQD

Debt Quarantine Details screen

Use to:

  • view quarantine details
  • end current quarantine

Requires DMISAO access.

OPQDA

Add Debt Quarantine screen

Use to apply a new quarantine debt status. Requires DMISAO access.

OPQS

Quarantine Summary screen

Viewing a summary of debts previously or currently quarantined.

OPWAA

Add Waiver screen

Used to record details of the waiver of debt amounts. Waivers can be recorded against determined and undetermined debts. Most debts less than $50 are auto-waived, immediately after raising.

OPWA

Waiver screen

Use to:

  • display details of previous waivers including automatically applied waivers (debts <$50)
  • modify or delete data that is provisional (for example, not determined) added via the OPWAA screen
  • reverse an applied waiver

This is a scrolling screen, allows up to 99 pages. The most recent screen appears first (for example, page 1 of 99).

OPWOA

Add Write-off screen

Used to write-off recovery action for selected debts for specified periods to:

  • record a new write-off
  • override an existing write-off

OPWO

Write-off screen

Use to:

  • display details of a current write-off
  • restart recovery of a debt using the restart recovery indicator
  • modify or delete data that is provisional (for example, not determined) and added via the OPWOA screen
  • update an 'end date' of a current write-off

The data on this screen relates to a specific debt or all determined debts on the record. It depends on the reason code.

OPXD

Extra Details screen

Used to record additional details that are common to all customer debts. These common details show in the following fields:

  • Financial Information Form:
    • Date of last Form
  • Request for Payment Card Issue:
    • Issue Card Indicator
    • Date of previous request(s)
  • Post BillPay/Bpay Customer Reference number. Note: this is similar to a CRN as it replaces the alpha character with 2 numbers. For example, 123 456 789 02

Customers use payment cards for ongoing payments. There is a barcode on the card that identifies the correct customer record. Card requests are allowed once every 7 days. This is due to the card production schedule. This screen is protected if:

  • there is no determined debt for the customer
  • a write-off is in place
  • all debts for the customer are over recovered
  • multiple Payment Cards have been issued to the customer. The 4 most recent issue dates will display on the screen

OPAL

Payment Arrangement List screen

Shows all recovery arrangements and current status:

  • view
  • add
  • reset
  • change or
  • cancel

This includes:

  • regular cash arrangements
  • irregular cash arrangements
  • discounts
  • garnishees
  • direct debits
  • other agency recovery
  • withholding arrangements, and
  • automatic withholdings (which trigger on this screen)

OPAR

Adjust Repayment screen

Most payment details are recorded on DMIS. This is via the interface with ESSentials (for cash including Payment Card) or the host systems (for withholdings).

Use OPAR to add repayments not previously recorded against a debt. A repayment may be:

  • cash/ cheque/ money order
  • Payment Card remittances
  • from ESSentials
  • withholdings from the host systems

The following staff have access to add a repayment:

  • Debt Management Branch staff
  • Service Leaders
  • some Centrelink International Services (CIS) staff

All staff have access to update the OPAR withholding adjustments for Department of Veterans' Affairs (DVA) transfer debts (debt reason 'DVA').

Only positive repayment adjustments are available from this screen.

OPAY

Agency Recovery screen

Use to record details of Services Australia requests to other government departments. The requests relate to withholdings from their benefits against customers who have a debt with Services Australia. An agency recovery cannot be added if:

  • the customer has no debts or only undetermined debts
  • there is another broken recovery arrangement or
  • a determined written-off debt is in place

OPCM

DMIS Customer Recovery Menu screen

OPCM is the secondary menu. This lists functions for updating recovery arrangements, and other details relating to customers. All screens accessed from this menu contain information that applies to all debts for the customer.

OPCVA

Add Civil Action screen

Use to add details of new civil action taken through the court system to recover a person's debts:

  • only 1 civil action can apply to any 1 debt at a time. However, a customer may have more than 1 debt under civil action for different reasons
  • if civil action is recorded without a specific Debt ID, civil action applies to all outstanding debts
  • if you record a Debt ID, the civil action will only apply to that debt
  • once civil action is recorded, the debt status will change to 'Determined - Civil Action'

OPCV

Display Civil Action screen

Use to update details of an existing civil action. This is a scrollable screen. Access historical data by paging down from the current occurrence. This screen is protected if there is no current civil action for the customer.

OPDI

Discount screen

Use to:

  • record discounts that have been accepted by a customer. Discounts may be accepted for any authority type except:
    • Other Government Department (OGD) and
    • Public Governance, Performance and Accountability (PGPA)
  • display any discounts that have been accepted
  • modify any discounts that have been accepted
  • cease any current discount arrangement

A discount arrangement can only be recorded on a record where at least 1 debt meets all of the following criteria:

  • the CRN outstanding balance is greater than $0
  • the status is Determined (DET), or Determined Collection agent (DMC)
  • the authority is not OGD
  • the debt benefit type is not Australian Taxation Office (ATO) savings bonus (SVT) or GST Assistance Scheme (SVG)
  • the debt reason is not compensation charge (Insurer) CMM or CMB (Failed to comply with Comp Prelim Notice S1184G)
  • the status is not Assurance of Support (AoS) protected
  • no current or broken discount is in place

Once a discount has been added:

  • all fields are protected, except the:
    • Comments
    • Payment Due date
    • Discount amount paid
    • Cease date fields
  • the following information is added to the OPDI:
    • CRN Balance Outstanding at Offer
    • Logon ID
    • Office of Logon ID

An activity is set up if the discount amount is paid. It prompts the user to add a waiver with an amount waived equal to the agreed discount amount.

OPGD

Garnishee Details screen

Use to:

  • record details of garnishee notices issued in respect of debts
  • modify any current garnishee notice details
  • cease any current garnishee notice details
  • display garnishee notice details which have been put in place to recover debt(s) (both current and ceased)
  • modify data that is still provisional (for example, not determined)
  • detail 10% garnishee of a wage or salary
  • record any number of current garnishee notices may exist to recover the debt(s)

If more than 1 debt is recorded, a garnishee notice must be issued to recover the customer’s debts. The OPGD screen must be completed for the customer's debts. Complete the Garnishee Debt ID field when issuing a garnishee notice for a specific debt.

Multiple garnishee notices will be current where garnishee notice details are added and:

  • existing garnishee details are in place, and
  • the existing garnishee details do not have a cease date

Updates to the garnishee notice details fields can occur until the data is no longer provisional.

OPIC

Irregular Cash Payments screen

Use to:

  • record details of negotiated irregular cash agreements
  • display irregular cash agreements to recover debt(s) (both current and cancelled)
  • cease a current irregular cash agreement

Six irregular cash instalments can be recorded on the OPIC screen at a time. Cease the previous agreement before adding a new agreement.

An irregular cash agreement cannot be coded if the customer has:

  • no determined debts
  • only undetermined debts
  • a debt write-off in place, or
  • a broken recovery arrangement

OPMC

Collection agent screen

Use to record and show details of referrals, and re-referrals to the External Collection Agent (ECA). This screen also shows returns/recalls from the ECA. Once a debt is referred to the ECA, the following fields on the OPMC screen are auto-updated:

  • Collection agent
  • Recovery Period End Date
  • Date Referred

Once a debt has been recalled or returned, the screen is protected.

The OPMC screen is a scrolling screen. A referral for each debt displays on a separate screen. The data relates to a specific debt, the header information is:

  • Debt ID field must be input
  • Debt Status field displays the status for the Debt ID that has been input

OPRC

Regular Cash Payments screen

Use to:

  • record details of negotiated regular cash agreements
  • display regular cash agreements to recover debt(s) (both current and cancelled)
  • cease a current regular cash agreement

There can only be 1 regular cash agreement at a time. Cease the previous agreement before adding a new one. It is not possible to create a regular cash agreement if the customer has either:

  • no debts
  • written-off debts
  • parked
  • held debts
  • only undetermined debts only
  • a broken recovery arrangement and determined debts exist

OPRS

Repayment Schedule screen

Is an arrangement specific screen. A schedule of future instalments show for:

  • regular cash
  • direct debits
  • non-percentage regular garnishee arrangements

The following arrangement types do not have a Payment Arrangement Schedule screen:

  • withholdings
  • agency recovery
  • irregular cash arrangements
  • percentage
  • one-off garnishee arrangement
  • discount arrangements

Go to the OPRS screen via the OPAL screen.

Note: if users go to OPRS via the Nxt field, they are taken to the OPAL screen. A warning message tells the user they must code 'P' against a current arrangement to view the repayment schedule.

Scheduled payments are limited so that payments already paid, do not display.

Users are able to mark payments as 'payment missed'. This shows the customer has been in contact and advised that they cannot afford to make a payment. All regular cash, non-percentage garnishee and direct debit payment arrangement types can have 'missed payments'.

Payments are marked as 'Due' until the payment is paid. Sometimes arrangements are broken. If the customer wishes to continue the arrangement, you may mark the 'due' payments as 'missed'. This will restore the arrangement and cancel the broken arrangement on the Activity List (AL) screen.

If there is more than 1 arrangement in place, a message displays advising there is another current arrangement. The payment details of that arrangement are not included on this payment schedule. To view the schedule for the other arrangement, go to OPAL and select the arrangement to go to OPRS.

OPVD

Direct Debit screen

Use to:

  • add a direct debit as a recovery arrangement
  • modify current direct debits
  • display direct debits, both current and ceased
  • cease a current or broken direct debit
  • modify provisional updates. If selected from the Activity List (AL) screen

OPVD contains details about the deduction including authority dates and customer's bank accounts details. A new screen appears for each direct debit arrangement.

OPWH

Withholding Instruction screen

A withholding instruction recovers the debt from relevant host system payments. Withholding instructions are auto-added when a customer has an outstanding debt and current payments.

The instruction will:

  • pass to the system that withholds the funds
  • calculate according to the rules used for determining withholdings within the host system. Then pass back to DMIS, unless a non-standard rate exists
  • withhold all amounts paid, except $1, if the amount paid is less than the specified amount
  • set a future review date of 3 months when non-standard
  • apply to the payment record after the first Entitlement Period End Day (EPED) following the start date. Then will deduct from the payday that follows
  • set 1 withholding instruction at a time per payment type
  • be current until the Cease Date field is not empty
  • make withholdings against all of a customer's debts as per current hierarchy rules, if the withholding Debt ID is empty

The instruction can apply to third parties with their permission. This person may have no liability for the debt but has agreed to have withholdings made.

Use OPWH to:

  • record a new instruction to the payment systems to make withholdings from the payments to repay a debt
  • cease any current withholding instruction
  • display instructions for withholdings to recover debt(s) (both current and ceased)
  • modify or delete data that is still provisional (that is, not determined). If the activity is not currently open, select it from the Activity List (AL) screen
  • extend the future review date for a non-standard withholding instruction

Access the OPWH screen from the OPAL screen if:

  • there is no auto-withholding instruction
  • the customer is receiving more than 1 payment and a higher rate of recovery is needed
  • ceasing a previous withholding instruction
  • a third party withholding instruction is awaiting coding
  • a non-standard rate withholding instruction is awaiting coding or review

OPOL

Overpay List screen

Lists and shows over advances for a particular Approved Service Provider. This screen is display-only and shows:

  • each over advance
  • CRN
  • total liability, and
  • outstanding balance

To view an over advance:

  • key 'S' in the Select field
  • to open the Overpay Repayments & Adjustments (OPOH) screen

OPOH

Overpay Repayments & Adjustments screen

Lists and displays all repayments for a particular over advance for Approved Service Providers. If an over advance has been over recovered, adjustments are made to correct the over recovery. Do this via the Adjust Repayment History (OPOAH) screen.

OPOAH

Adjust Repayment History screen

Use to adjust repayment amounts. This will occur mainly where an over advance has become over recovered, for Approved Service Providers.

To adjust a repayment:

  • navigate to the Overpay Repayments & Adjustments (OPOH) screen
  • select the repayment being adjusted to open the OPOAH screen, and
  • key the new repayment amount

OPRTD

Debt Transfer Details screen

  • Use to complete the following actions: Adjust a debt repayment, see Adjusting and adding debt repayments
  • Transfer a debt repayment, see Transferring debt repayments
  • Refund a debt repayment, see Refunding debt repayments
