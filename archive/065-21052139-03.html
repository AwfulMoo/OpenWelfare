3

Series 2 letters - all care types + Read more ...

Prior to issuing a Series 2 letter confirm that:

  • care recipient is still in care, refer to Aged Care means assessment - Preliminary checks for process if care recipient has departed care
  • a means assessment is not able to be completed automatically, refer to Aged Care Means Assessments for information on when a form is required
  • an Aged Care Calculation of your cost of care form has not been lodged
  • 3 business days have passed since entry into care was received in Customer First
  • 21 days have passed plus an allowance for the letter to be received since 2A was sent
  • for residential care, 2B is not able to be issued until 2A has
  • 5 business days have passed since fees have set and 2C letter has failed to issue

Confirm date of entry into care by checking:

  • Aged Care Staff Portal (ACSP) – Care recipient summary screen
  • Aged Care Management Payment System (ACMPS) - Assessments & Events tab > Entry Details

4

Series 2 letter - ACMPS manual letter generation + Read more ...

If care recipient has elected not to disclose their means for home care and 2A letter has not yet been issued, go to Step 5.

To generate a letter, go to ACMPS in DHS Fees & Charges business role, go to the care recipient record and:

  • Note the Care Recipient ID and relevant date of entry
  • From the ACMPS side menu select:
    • Documents, and
    • Manual Letter Generation
  • On the Manual Letter Generation screen:
    • enter care recipient’s ID
    • select the letter needed from drop down list
    • select Care Commencement Date
    • confirm BDF and ITF/MTF have populated,
    • confirm Location Address details, and
    • select Generate Letter
  • A message displays stating that letter has triggered successfully
  • Communication history tab will add letter code straight away
  • Place RCA/HOM or RCA/NCL on hold for:
    • 2A (initial request) - 21 days plus an allowance for the letter to be received from the date the letter is sent
  • Record a care recipient note in the ACSP. See Using the Aged Care Staff Portal to manage care recipient notes (Care Recipient in Context (CRiC)
  • The creation of manual Series 2 letters must be escalated to ICT via MySupport
  • In Customer First, add Fast Note - select Auto Text, use Aged care > assessment > Progress of Aged Care Assessment

Procedure ends here.

5

Manual homecare 2C letter required for elected MND + Read more ...

See Centrelink - Aged care letter templates  sub-site for letter templates.

  • Create letter for care recipient (including nominee if applicable) and service provider All dates added to letters should be in format NN Month YYYY (e.g. 1 January 2021)
  • All text to be in Arial font
  • Any coloured font should be changed to black

Details required to be completed in letter:

  • Care recipient ID and service provider NAPS ID
  • Name and Address:
    • Care recipient/nominee - Add name and address of care recipient/nominee from either Customer First for Centrelink care recipients, or ACMPS for DVA care recipients
    • Service Provider – add name and address from ACMPS
    • Use lower case except for SUBURB and STATE
  • Care Recipient or Nominee Name:
    • Select salutation and add surname
  • Nominee letter only:
    • Add the care recipient name in the letter text
  • Date of commencing care
  • Amount of basic daily fee and income-tested care fee
  • Interpreter and nominee paragraphs as required

Once letter fields completed, see Table 3.

6

HC1, RC1 or RC39 letter - ACMPS manual letter generation + Read more ...

Gather information required for letter generation:

  • Note Care Recipient ID for care recipient
  • Go to Means Testing Data tab, go to Means Tested Assessments
  • Note the Assessment ID for relevant Means Test Assessment
  • Refer to Aged Care Management Payment System (ACMPS) for assistance

To generate a letter, go to ACMPS in DHS Fees & Charges business role, go to the care recipient record and:

  • Note the care recipient Care Recipient ID
  • Go to Means Testing Data tab and look at Means Tested Assessments to find the Assessment ID for the relevant pre entry assessment
  • From the ACMPS side menu select:
    • Documents, and
    • Manual Letter Generation
  • On the Manual Letter Generation screen:
    • enter care recipient’s ID
    • select Care Type from drop down list
    • select Letter Code from drop down list
  • Primary Details for the Letter will display:
    • select the relevant Assessment ID in MTA ID drop down list
    • confirm BDF and ITF/MTF have populated
    • confirm income and asset details that are presented are correct
  • Select Generate Letter
  • A message displays stating that letter has triggered successfully
  • Communication history tab will add letter code straight away
  • Select the Notes tab and add a note advising pre entry letter has been generated including letter recipient/s and date of issue

Was a letter able to be generated?

  • Yes, procedure ends here
  • No, manual letter from template required, see Table 2

2

RFI - means assessment including/change of circumstances + Read more ...

Complete the highlighted fields in the letter template:

  • Your Reference:
    • Add care recipient customer reference number (CRN) and Service Officer logon
  • Name and Address:
    • Add name and address of care recipient/nominee from Customer First
    • Use lower caps except for SUBURB and STATE
  • Care Recipient or Nominee Name:
    • Select salutation and add surname
  • Nominee letter only:
    • Add the care recipient name in the letter text
  • Additional information:
    • Post admission add (with DD/MM/YY being the start date of care for means assessment): As you have already started receiving care, make sure the documents you provide show your income and/or assets as at <DD/MM/YY>
    • Complete a bullet point list of the additional information
    • Service Officers must use the approved text for each item. See the RFI document list on the Resources page
  • Pre admission/post admission:
    • Delete text as needed
  • Letter date:
    • No update needed. Pre-populates to today’s date

Note: do not add free text to the letter template. The additional information requested section lists all requested information/documents. If additional text needed is not listed on the Resources page, seek Technical Peer Support (TPS) approval before sending the letter. See Service Officer role when seeking Technical Peer Support (TPS) in Smart Centres.

Once letter fields completed, see Table 3.

3

RFI - financial hardship claim + Read more ...

Complete highlighted fields in the letter template:

  • Your reference:
    • Add the care recipient customer reference number (CRN) and Service Officer logon
  • Name and address:
    • Add name and address of care recipient/nominee
  • Letter date:
    • No update needed. Pre-populates to today’s date
  • Care recipient or nominee name:
    • Add title, first name and surname
  • Nominee letter only:
    • Add the care recipient name in the letter text
  • Type of care:
    • Delete care type as needed. Leave the care type of the hardship application
  • Application date:
    • Add the hardship application lodgement date
  • Income and assets amounts ($):
    • Add the income threshold amount (15% of the current maximum single rate of Aged Pension)
    • Add the assets threshold (additional financial resources amount)
    • For the rates, see the Financial hardship current and historical thresholds - assets and income table in Aged Care Financial Hardship Assistance - Assessment
  • Fee types:
    • Delete fee type as needed. Leave the fee type/s being assessed
  • Essential expenses information required:
    • Separate expense lists for home care/respite and residential care. Delete text as needed
    • Complete a bullet point list of information required
    • Services Officers must use the approved text for each item. See RFI document list on the Resources page
    • Note: to check when evidence is required, see the List of allowable essential expenses table in Aged Care Financial Hardship Assistance - Assessment
  • Other reasons for more information:
    • Delete text as needed
    • Care recipient must test eligibility for ISP (including claiming ISP under hardship provisions), or
    • Complete a means assessment

Note: do not add free text to the letter template. The letter lists all essential expenses information. If additional text needed is not listed, seek Technical Peer Support (TPS) approval before sending the letter. See Service Officer role when seeking Technical Peer Support (TPS) in Smart Centres.

Once letter fields completed, see Table 3.

6

ACMPS Initial fee letter - home care + Read more ...

Go to ACMPS and in care recipient’s record:

  • Go to Means Testing Data tab, go to Means Tested Assessments
  • Locate the Assessment ID that contains the date of entry as the date of effect
  • Fees and Charges tab should be correct based on this income amount
  • Income data as at date of entry into care will be added to income summary statement

Complete the following details in the letter:

  • Care Recipient ID and service provider NAPS ID
  • Name and address of letter recipient
  • Print Date
  • Nominee letter only:
    • Add the care recipient name in the letter text
  • Date to which income and financial assets can be backdated
    • Entry date of date of effect of means assessment
  • Fees payable and effective date
  • Fortnightly income
    • Go to Total Amount Details
    • Code TOTALINCOME amount
  • Income Summary Statement
    • Go to Total Amount Details
    • Code ISPINCOME and NONISPINCOME

Once letter fields completed, see Table 3.

7

ACMPS Initial fee letter - residential care + Read more ...

Once a means assessment has been completed for post entry residential care, an initial letter is issued from ACMPS that contains an income and asset summary. This letter is not a fee advice. Only manually issue it at time of entry into care or if details have been corrected from date of entry and another letter has been requested.

Use the Aged Care Fee Calculator to calculate the correct fees using the assessable income and assets as at date of entry into care. Assessment of income and assets recorded will be required to confirm letter code that is required.

Go to ACMPS and in care recipient’s record:

  • Go to Means Testing Data tab, go to Means Tested Assessments
  • Locate the latest Assessment ID that contains the date of entry as the date of effect
  • Fees set in ACSP Fees and charges should be correct based on income/assets amount
  • Income and asset data as at date of entry into care will be added to income and asset summary statements

Details required to be completed in letter:

  • Care Recipient ID and service provider NAPS ID
  • Name and address of letter recipient
  • Print Date
    • Use format NN Month YYYY (e.g. 1 January 2021)
  • Nominee letter only:
    • Add the care recipient name in the letter text
  • Rate of BDF as at date of entry into care
    • Use the Aged Care Fee Calculator for BDF rates
  • Minimum amount of net assets (if applicable)
    • Refer to Schedule of Fees and Charges for Residential and Home Care, link on Resources page
  • Net asset amount (if applicable)
    • Care recipients total assets prior to capping being applied
    • In MTA, go to Total Amount Details tab
    • Code TOTALASSETS amount
  • Assessable Assets amount
    • Total assets after home value has been capped
    • In MTA, go to Asset Details
    • Add assets coded from date of entry into care
    • Do not include CAPPEDVALUE field
  • Net fortnightly income amount
    • Go to Total Amount Details
    • Code TOTALINCOME amount
  • Assets Summary Statement
    • Add details using breakdown in Asset Details tab
    • Your principal home field: full value of home
    • HCPA = Other Assets
  • Income Summary Statement
    • Go to Total Amount Details
    • Code ISPINCOME and NONISPINCOME

Once letter fields completed, see Table 3

8

Manual letters including fee change, refunds and departure + Read more ...

Creation and issue of a manual fee change or departure letter is restricted to staff with the MANADJ, QR or RCADEA skill tag. Any other staff should escalate failed letters to ICT and if manual letter required:

  • All fees must be correct and displaying in ACMPS or ACSP
  • Correction to resident status, see Aged care reviews - quarterly reviews
  • Email request for manual letter to ACPB using email template on Resources
  • Include letter codes and ICT response in request
  • Procedure ends here

For MANADJ, QR, RCADEA skilled staff, proceed to create manual letter. Letter template required will depend on resident status and current fees of care recipient. Information required includes:

  • Care Recipient ID and service provider NAPS ID
  • Name and address of letter recipient
  • Print Date
    • Use format NN Month YYYY (e.g. 1 January 2021)
  • Date of effect for fee change/departure
  • Current fee amounts
  • Any refund that has been calculated

Once letter fields completed, see Table 3.

9

Annual or lifetime cap letter + Read more ...

Creation and issue of annual and lifetime cap letter is restricted to staff with the MANADJ or QR skill tag. Any other staff:

  • Create a Fast Note - select Auto Text, use Aged Care > Enquiry > Aged Care Manual Adjustment Request
  • Include details of request that has been made for a manual letter to be issued
  • Procedure ends here

Skilled staff can complete manual annual or lifetime cap reached letters after calculation and confirmation that cap amounts have been reached. Information required includes:

  • Care Recipient ID and service provider NAPS ID
  • Name and address of letter recipient
  • Print Date
    • Use format NN Month YYYY (e.g. 1 January 2021)
  • Date of cap being reached
  • Annual or Lifetime cap threshold amount as at date of event
  • Date that $0 MTCF/ITCF applies
    • Cap reach date plus one day
  • Anniversary date - fees will recommence (annual cap only)

Once letter fields completed, see Table 3.

Saving and uploading a manual letter

Table 3

Step

Action

1

Save letter as PDF file + Read more ...

Once all required fields in the letter are completed:

  • Check that font colour is black and is set as Arial
  • Check that information in header or footer has been updated if required
  • Select File
  • Select Save As
  • Select Desktop for file destination
  • Change the file name to care recipient CRN
  • Change the Save as type: field to PDF
  • Select Save

2

Print letter and send to recipient/s + Read more ...

  • Print the completed letter/s
  • Mail to:
    • care recipient
    • nominee (if required) and
    • service provider
  • Protect the letter recipient’s privacy. If using a window envelope, only display the:
    • return address
    • name
    • address of letter recipient

Refer to aged care letter codes and descriptions table on the Resources page for information on which system issues the different letter types. Upload any letter codes generated or issued from ACMPS into ACMPS. Upload all other letters to Customer First, including RFIs and hardship letters.

If uploading this letter to:

  • ACMPS, go to Step 3, or
  • Customer First go to Step 4

3

Upload letter into ACMPS + Read more ...

In ACMPS Fees and Charges role:

  • Select Documents from the side menu
  • Select Manual Letter Upload from the dropdown menu
  • Upload a PDF version of the manual letter
  • Ensure the correct letter code is selected including care types, do not use OTHER
  • Complete all remaining fields in the screen
  • Review details prior to completing as a letter upload is not able to be deleted

Go to Step 5.

4

Upload letter into Customer First + Read more ...

To upload a document to Customer First, see Attaching electronic documents to a Centrelink customer's record using Document Upload in Customer First.

5

Delete letter file from desktop + Read more ...

  • Minimise all open windows to access the desktop
  • Right click on letter file
  • Select delete

6

Record Fast Note + Read more ...

Record or update existing Customer First Fast Note with details of manual letter issued. Include:

  • Letters that have been issued
  • Recipients of letters
  • Source of request and why letter not issued automatically

Complete any related work items.
