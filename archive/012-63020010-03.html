Trouble shooting + Read more ...

Unable to find prescription

If prescription has no Authority ID number, search for work by keying the prescriber number

  • If unable to locate work with authority ID number or prescriber number, try the Medicare number. Note: the use of Medicare number to name is incorrect and the file will need to be renamed
  • If still unable to locate, Service Officer must escalate this to their Team Leader

Search under other state’s WRIT Work Type options

  • Once prescription is located, Service Officers must recategorise it to the correct work type that matches their current state location. See the How to recategorise work and rename a reference ID table

Search in Authority Mailed Escalation

  • If work cannot be found, Service Officers must escalate this to their Team Leader so it can be re-scanned into WLM (naming at the MFD)
  • If missing work is found in WLM at a later date, it is dealt with as duplicate work

Poor quality image or missing documents

Service Officers are to refer poor quality image or missing documents to their Team Leader for rescanning.

Prescription details displayed in the PDF do not match the hard copy

Service Officers are to refer prescription details displayed in the PDF that do not match the hard copy to their Team Leader. See the Troubleshooting HPOS form uploads table for further information.

Assessment result + Read more ...

Select Submit. The system will assess the request as approved or rejected.

If a Service Officer determines the written authority request meets all relevant criteria and should be approved and the Online PBS Authorities system has rejected the request the Service Officer may:

  • select Approved from the Assessment Result Override drop down box, select reason for override or use the keyboard shortcut Ctrl+Shift+O and submit
  • escalate to PSO or PA for clarification

Rejections for ‘prescribed quantity and/or repeats is greater than allowed’ occurs due to the limits set to prevent prescribers obtaining more than a certain amount when using the Online PBS Authorities system. Operators can override these rejections in accordance with approval rules.

If the Written Authority is:

  • rejected as 'too soon' Reason Code 94 and 95, or
  • has an X or D in the dose field
  • go to Step 12

Note: if recording another Authority for the same patient the date of prescribing will default to today’s date. Use the New Authority Same Prescriber function and change date of prescribing.

12

Rejected as 'too soon' + Read more ...

Written Authority applications with an X or D in the dose field will need to be assessed manually for 'too soon'.

The system will return Information Reason Code 132 to advise that 'MANUAL HISTORY CHECK REQUIRED - a patient history check could not be completed’.

If the Written Authority application is rejecting because it is 'too soon' reason code 94 and 95, or needs to be assessed manually, Service Officers must to go into Search Authorities and:

  • select Display Patient History
  • use the returned history to calculate the 'date the supply will last to' in order to complete the rejection letter
    • To reject an application use the reject button to record the rejection or use the keyboard shortcut Ctrl+Shift+R
    • Select a reject reason from the drop down box

The system uses 12 months of history to calculate 'too soon' and takes into account if there has been any overrides applied and the dispensing history. Therefore, when manually assessing, be aware of the differences and use your usual method of assessment.

For manual assessing of ‘too soon’:

  • Review authority approval history
  • Use 6 months date range from date of processing e.g. if processing on 16/5/22, date range would be 16/11/21 – 16/5/22. See Step 9 in the Requesting a telephone Authority prescription table
  • Use the Too Soon calculator (new) to calculate the date the ‘supply will last’ to

13

Saving an Application + Read more ...

  • Select Resume against the authority which is saved. This opens the authority at the point that it was saved
  • Amend any previously entered information by selecting the Back button
  • Complete authority as usual

How to endorse and post a written authority approval request

Table 9: This table describes how Service Officers endorse and post a written authority approval request.

Step

Action

1

Written authority request results + Read more ...

For written authority requests with the assessment result of:

  • Approved, go to Step 2
  • Rejected, go to Step 3
  • Approved with changes, go to Step 4

2

Approved requests + Read more ...

For written authority requests with an assessment result of Approved, endorse the request:

  • circle or write quantity and repeats in red pen
  • annotate PBS item code if CAR approved, and
  • stamp both original and duplicate copies of the Authority prescription with:
    • 'Approved by PBS-Services Aust' and any other applicable stamp, for example, TPX, SPX, PLC or HSD, or
    • annotate with OPA approval number if stamp is not used

Posting the written authority request

  • Remove the third copy and place in a secure orange bin for shredding
  • Check who is to receive the application – the prescriber or the patient
  • Generate an envelope for the correct recipient
  • Confirm name and address generated on the envelope matches the recipient details on the prescription
  • Confirm prescriber details on any attached letter to be sent with the prescription matches the prescription and the envelope
  • Place the original and the duplicate of the prescription, together with any attached letter, in the envelope
  • Ensure the Authority prescription is folded so no information can be read through the envelope
  • Place the envelope in the correct mailroom tray

Note:

  • when the Authority prescription must be returned to the patient, return to the address on the Authority prescription
  • when addressing hospital prescriptions, including the unit title on the address will improve delivery outcomes. For further information on addressing correspondence, see Quality assurance for external mail in Medicare

Go to Step 5.

3

Rejected requests + Read more ...

For written authority requests with assessment result of Rejected, do not endorse the prescription.

Posting the written authority request

  • Do not remove the third copy
  • Print Rejection letter (Unapproved authority prescriptions). The Resources page contains a link to this letter
  • Staple letter to prescription
  • Use window envelopes where available, folded with the prescriber address visible through the window and return to approved prescriber
    • otherwise, an envelope must be generated
    • confirm prescriber’s name and address on the envelope matches the prescription

Note: always return Rejection letters to the approved prescriber indicated on the written authority request.

Go to Step 5.

4

Approved with changes + Read more ...

For written authority requests that have had their quantity and/or repeats changed from what was originally written on the prescriptions, an assessment result of 'Approved with changes', endorse the request by doing the following:

  • endorse the quantity and repeats being approved in red pen
  • annotate PBS item code if CAR approved, and
  • stamp both original and duplicate copies of the Authority prescription with:
    • 'Approved by PBS-Services Aust, or
    • annotate with OPA approval number if stamp is not used

If applicable: PBS - NOTE decreased QTY/RPTS Approved

Any other applicable stamp, for example, TPX, SPX, PLC or HSD.

Posting the written authority request

  • Remove third copy and place in a secure orange bin for shredding
  • Print Approved with Changes (Approved authority prescription with changes). See the Resources page for a link to these letters
  • Staple any letters to authority prescription
  • confirm prescriber’s name and address on the envelope matches the prescription

Note: always return Approved with changes to the approved prescriber indicated on the written authority request.

Go to Step 5.

5

Complete work item + Read more ...

On completion of the work item in the source system:

  • Select Complete in the Status field
  • A Complete reason is only required when:
    • the work item has been completed in the source system (appears to already be completed i.e. a duplicate work item)
  • select Save

A message will display at the bottom of the screen advising that the work item has been successfully saved.

Note:

  • it may take up to 2 minutes for the work item to disappear from the Worklist
  • refer to Team Leader or PSO if required

Procedure ends here.

5

File has multiple patient prescriptions + Read more ...

If there are multiple patient prescriptions in one file:

  • Save the file:
    • Right click the file and select Save As
    • Save the file to the computer
  • Extract the additional prescriptions from the PDF file and save the new files to the computer:
    • In Acrobat DC, select Tools > Organize Pages > Extract
    • Highlight the pages to be extracted
    • If the pages are to be extracted as separate documents, tick the Extract Pages As Separate Files box before selecting Extract
    • Save the item/s. These item/s can be found in the saved folder which has the same name and numerics to differentiate, e.g. 2 or as pages from <original title>
    • Locate the saved file (the file will have the same title and a numeric, e.g. 2)
  • Create separate files in WLM for each person. To create new work items in WLM:
    • Select WLM Document Upload under the Health Services menu
    • Select Create New Work Item from Upload Type field
    • Select Browse and select a file from the pop up window
    • Change the Date of Lodgement (defaults to today’s date)
    • Select Program from the drop down menu
    • Select Work Type from the drop menu
    • Add Primary Reference Number (i.e. Medicare number or Authority ID number)
    • Select Upload to submit the new work item into the WLM allocation pool
  • Delete files from the computer
  • On the original file, put a processing note to indicate the other documents in the file has been extracted into new files.
