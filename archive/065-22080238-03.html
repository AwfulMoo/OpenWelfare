Step

Action

Going to the Hardship summary screen + Read more ...

3

Using the Hardship summary screen + Read more ...

Note:

  • The No hardship registered for this care recipient information message will display for care recipients who have never been granted financial hardship assistance
  • The Hardship registration is available for residential care recipients only information message will display when Service Officers are viewing hardship records for home care recipients/services in the ACSP

Use the Hardship summary screen to view a care recipient’s current and historical financial hardship assistance records.

The Hardship summary screen will display the following information:

  • Care Recipient banner - displays the care recipient’s name, IDs, Date of Birth, Record type and Type of care status
  • Current and future hardship - displays in a table format
    • Hardship type
    • Start date
    • End date
    • Maximum hardship reduction
    • Care recipient contribution amount
    • Fee reduction amount - a dash will only display for RAD/DAP Accommodation Payment - Post 1 July 2014 hardship assistance records. This field will populate for all remaining hardship assistance types
    • Created date
    • Lasted updated
    • Submitted by
    • Status - select the hyperlinked Status and the View hardship screen will display. Go to Step 4
  • Historical hardship - displays in a table format
    • Hardship type
    • Start date
    • End date
    • Maximum hardship reduction
    • Care recipient contribution amount
    • Fee reduction amount - a dash will display for RAD/DAP Accommodation Payment - Post 1 July 2014 hardship assistance records
    • Created date
    • Lasted updated
    • Submitted by
    • Status - select the hyperlinked Status and the View hardship screen will display. Go to Step 4

Service Officers can:

  • type in a keyword in the Search bar to search for all records that contain the keyword
  • select the number of records to display from the Show records drop down list (bottom left corner of the screen)
  • go to a different page by selecting either the:
    • First, Previous, Next or Last options, or
    • the relevant page number

Procedure ends here.

4

View hardship + Read more ...

Note:

  • Some historical View hardship records may not contain the Care recipient contribution field
  • The rate table displays for the Basic Daily Fee hardship type to reflect the Consumer Price Index (CPI) changes, which occurs biannually on 20 March and 20 September
  • For Basic Daily Fee hardship records that have a future end date and include a CPI period, ACSP will automatically update the fee reduction amount due to the CPI change

To View hardship on the Hardship summary screen:

  • select the relevant hyperlinked Status record
  • the View hardship screen will display
  • a summary of the individual hardship record will display
    • Hardship status

    • Submitted by
    • Hardship type
    • Start date
    • End date
    • Maximum hardship reduction
    • Care recipient contribution - this field will display for all hardship assistance record types except Concessional Supplement Bond – Pre March 2008 or Concessional Supplement Charge – Pre March 2008 hardship assistance record types or where Yes has been selected in the Maximum hardship reduction field
    • Fee reduction amount - a dash will display for RAD/DAP Accommodation Payment - Post 1 July 2014, Concessional Supplement Bond – Pre March 2008 or Concessional Supplement Charge – Pre March 2008 hardship assistance record types. This field will display a dollar ($) value for all remaining hardship types except where Yes has been selected in the Maximum hardship reduction field
    • Reason
    • Additional notes
    • The rate table displaying the Start date, End date, Maximum Hardship, Care recipient contribution and Fee reduction amount

Legend to display the explanations for the various financial hardship assistance types

Managing residential care financial hardship assistance in Aged Care Staff Portal - (Care Recipient in Context (CRiC))

Table 2: this table outlines information for managing residential care financial hardship assistance in the Aged Care Staff Portal (ACSP)

Step

Action

1

Choose Menu option + Read more ...

Note:

  • Staff with the following security roles will be able to select Register hardship and/or the hyperlinked Action status:
    • Fees and charges
    • Aged Care National Office
    • Aged Care Business Administrator
    • Aged Care IT Administrator
  • For home care recipients/services, see Aged care financial hardship assistance - assessment for managing financial hardship assistance in the Aged Care Management Payment System (ACMPS)
  • The Hardship registration is available for residential care recipients only information message will display when Service Officers are viewing hardship for home care recipients/services in ACSP

To action residential care hardship assistance - choose from the following menu options:

  • Register hardship
    • Using the screen - go to Step 2
  • Update hardship
    • Using the screen - go to Step 3
  • Registered/Updated hardship receipt
    • Viewing the screen - go to Step 4
  • Delete hardship
    • Using the screen - go to Step 5

2

Register hardship + Read more ...

Note:

  • Service Officers must check and assess the care recipient’s eligibility for hardship before registering hardship
    • See Aged care financial hardship assistance - assessment for the process
  • The Hardship registration is available for residential care recipients only information message will display when Service Officers are registering hardship for home care recipients/services in ACSP
  • Service Officers are unable to register (create) Individual Hardship records:
    • existing Individual Hardship records can be updated for the End date and Additional notes only
    • Any other changes require the Individual Hardship record to be deleted, go to Step 5, and the correct information to be created in a new Basic Daily Fee hardship, see this step
    • Individual Hardship records will be replaced by Basic Daily Fee hardship records
  • If the following error message displays, this means the care recipient contribution value entered by the Service Officer exceeds the maximum basic daily fee in effect for the start date of the hardship period record and must be corrected:
    Error: there was a problem with your submission. The following error(s) occurred:
    • Errors occurred while creating a Hardship
    • Care recipient contribution exceeded the maximum daily fee liability of $<value calculated as at the start date of the record and displays in this message>
  • The Fee reduction amount* field will not display for Service Officers when registering RAD/DAP Accommodation Payment - Post 1 July 2014 hardship assistance records
  • Service Officers can select Legend to display the explanations for the various financial hardship assistance record types
  • All fields are optional unless marked with an asterisk

To Register hardship on the Hardship summary screen:

  • Select Register hardship
  • The Register hardship screen will display
  • Complete the below fields
    • Hardship type* - select the drop down menu for the appropriate hardship type category
    • Start date* - key the date (DD.MM.YYYY), or select from the calendar (select the calendar icon to display the calendar)
    • End date* - key the date (DD.MM.YYYY), or select from the calendar (select the calendar icon to display the calendar)
    • Maximum hardship reduction* - select either Yes or No. This field will display when the hardship category type is either Basic daily fee, Means Tested Care Fee – Post July 2014, Income Tested Fee – Pre July 2014, DAC/RAC Accommodation Contribution – Pre July 2014 or RAD/DAP Accommodation Payment – Post 1 July 2014
    • Care recipient contribution – key the amount when this field displays for the hardship type category Accommodation Bond – Pre July 2014, Accommodation Charge – Pre July 2014 or Bond hardship and/or when No has been previously selected for Maximum hardship reduction
    • Fee reduction amount* - key the amount when this field displays for the hardship type category Accommodation Bond – Pre July 2014, Accommodation Charge – Pre July 2014 or Bond hardship and/or when No has been selected in the Maximum hardship field. Where the hardship type category is Basic Daily Fee and No has been selected in the Maximum hardship field, the Fee reduction amount is auto-populated after the Service Officer selects Submit
    • Reason - select the drop down menu for the appropriate reason
    • Additional note* - key additional notes. This field has a 254 character limit

Does the Service Officer want to proceed?

  • Yes,
    • Select Submit
    • Go to Step 4
  • No,
    • Select Cancel
    • A message will display asking to select Yes or No to cancel the process
    • Select Yes
    • The Hardship summary screen will display
    • Procedure ends here

3

Update hardship + Read more ...

Note:

  • Service Officers must check and assess the care recipient’s ongoing eligibility for hardship before updating hardship
    • See Aged care financial hardship assistance - overview for the process
  • If the following error message displays, this means the care recipient contribution value entered by the Service Officer exceeds the maximum basic daily fee in effect for the start date of the hardship period record and must be corrected:

Error: there was a problem with your submission. The following error(s) occurred:

  • Errors occurred while creating a Hardship
    • Care recipient contribution exceeded the maximum daily fee liability of $<value calculated as at the start date of the record and displays in this message>
  • The Fee reduction amount* field will not display for Service Officers when updating RAD/DAP Accommodation Payment - Post 1 July 2014 hardship assistance records
  • The rate table displays for the Basic Daily Fee hardship type to reflect the Consumer Price Index (CPI) changes, which occurs biannually on 20 March and 20 September.
  • For Basic Daily Fee hardship records that have a future end date and include a CPI period, ACSP will automatically update the fee reduction amount due to the CPI change
  • Service Officers can select Legend to display the explanations for the various financial hardship assistance types
  • The Hardship type field is greyed out
  • All fields are optional unless marked with an asterisk

To Update hardship on the Hardship summary screen:

  • Select the relevant hyperlinked Status record
  • The View hardship screen will display
  • Select Update
  • The Update hardship screen will display
  • Key the updates
    • Start date* - key the date (DD.MM.YYYY), or select from the calendar (select the calendar icon to display the calendar)
    • End date* - key the date (DD.MM.YYYY), or select from the calendar (select the calendar icon to display the calendar)
    • Maximum hardship reduction* - select either Yes or No. This field will display when the hardship category type is either Basic daily fee, Means Tested Care Fee – Post July 2014, Income Tested Fee – Pre July 2014, DAC/RAC Accommodation Contribution – Pre July 2014 or RAD/DAP Accommodation Payment – Post 1 July 2014
    • Care recipient contribution – key the amount when this field displays for the hardship type category Accommodation Bond – Pre July 2014, Accommodation Charge – Pre July 2014 or Individual hardship Bond hardship and/or when No has been previously selected for Maximum hardship reduction
    • Fee reduction amount* - key the amount when this field displays for the hardship type category Accommodation Bond – Pre July 2014, Accommodation Charge – Pre July 2014 or Individual hardship Bond hardship and/or when No has been selected in the Maximum hardship field. Where the hardship type category is Basic Daily Fee and No has been selected in the Maximum hardship field, the Fee reduction amount is auto-populated after the Service Officer selects Submit
    • Reason - select the drop down menu for the appropriate reason
    • Additional note* - key additional notes. This field has a 254 character limit
    • The rate table displaying the Start date, End date, Maximum Hardship, Care recipient contribution and Fee reduction amount.

Does the Service Officer want to proceed?

  • Yes,
    • Select Submit
    • Go to Step 4
  • No,
    • Select Cancel
    • A message will display asking to select Yes or No to cancel the process
    • Select Yes
    • The View hardship screen will display

Procedure ends here

Viewing the Registered/Updated hardship receipt screen + Read more ...

Note:

  • The rate table displays for Basic Daily Fee hardship types to reflect the Consumer Price Index (CPI) changes, which occurs biannually on 20 March and 20 September

The Registered/Updated hardship receipt screen will display:

  • Hardship submission date
  • Hardship type
  • Start date
  • End date
  • Maximum hardship reduction
  • Care recipient contribution
    • Fee reduction amount - a dash will display for RAD/DAP Accommodation Payment - Post 1 July 2014 hardship assistance records
  • Reason
  • Additional notes
    • The rate table displaying the Start date, End date, Maximum Hardship, Care recipient contribution and Fee reduction amount.

Service Officers must select Go to hardship summary to return to the Hardship summary screen.

Procedure ends here.

5

Delete hardship + Read more ...

Note:

  • See the Reviews and reassessments of hardship assistance - respite and residential care table and the Cease or revoke hardship assistance table on the for information on when to delete a hardship record
  • Service Officers can select Legend to display the explanations for the various financial hardship assistance record types
  • All fields are optional unless marked with an asterisk

To Delete hardship on the Hardship summary screen:

  • Select the relevant hyperlinked Status record
  • The View hardship screen will display
  • Select Delete
  • The Delete Hardship screen will display
  • Complete the:
    • Reason* - select the drop down menu for the appropriate reason
    • Additional note* - key additional notes. This field has a 254 character limit
  • Select Submit
  • The Delete? You are about to delete this hardship. Are you sure you want to delete? pop up message will display

Does the Service Officer want to complete the Delete hardship process?

  • Yes,
    • Select Yes
    • The hardship record has been deleted
    • The Hardship summary screen will display
    • Procedure ends here
  • No,
    • Select No
    • The hardship record has not been deleted
    • The Delete hardship screen will display
