Specified Personnel and customer authentication

Step

Action

1

Authentication of Specified Personnel at an Agent site + Read more ...

Authentication is required for every call before the enquiry can be progressed.

Use Office Locator under Teams at (site name), then click on Confidentiality Agreement to verify the information provided by Specified Personnel. They must confirm:

  • Agent site location and Office Locator code
  • that they are Services Australia Specified Personnel at an Agent site
  • their full name (first and last name)

The Resources page contains a link to Office Locator.

Can the person contacting be verified as Specified Personnel?

  • Yes, go to Step 3
  • No, go to Step 2

2

Further authentication of Specified Personnel is required + Read more ...

Where the person contacting cannot be verified using information already asked, confirm details that can be verified in Office Locator such as:

  • opening hours
  • contact phone numbers
  • email address
  • street/postal address

If there are still concerns about verifying the person, phone the Specified Personnel back on the Agent sites' registered phone number. This Is found in the Contact Details for that site in Office Locator.

Can the person contacting now be verified as Specified Personnel?

  • Yes, go to Step 3
  • No, procedure ends here

3

Authenticate the customer + Read more ...

Ask the Specified Personnel to ask the customer for information that will help in completing Proof of Record Ownership (PORO):

  • Authenticate the customer to verify PORO
    • Note: name, date of birth and Customer Reference Number (CRN) alone are not sufficient. These details may be known to a third party. For example the customer’s partner
    • See Authenticating a Centrelink customer for further questions that can be used

The Agent Specified Personnel will:

  • ask the customer for the information, and
  • relay the responses back to the Service Officer

Service Officers should not ask the Agent Specified Personnel to hand the phone to the customer.

Has the customer been authenticated?

  • Yes, go to Step 4
  • No, procedure ends here

4

Specified Personnel are requesting assistance with a complex enquiry + Read more ...

Support Agent Specified Personnel with enquiries.

If a complex enquiry is received Service Officers should contact the appropriate Technical Peer Support (TPS) queue. The Service Officer may engage in a 3 way conversation with the Agent and TPS to support required assistance.

The Silver Service lines are a priority service for Specified Agent Personnel. They should not be handed off to TPS.

Note: apply first contact resolution in the first instance. Follow the relevant operational blueprint based on the customer's enquiry.

An Agent Specified Personnel may call with a customer:

  • who is registered for self service, and the matter could be resolved using that channel
  • who has a general enquiry. For examples see the Background page

In these instances:

  • ask the Agent Specified Personnel for details about why they are using the Silver Service for their enquiry
  • educate agents and customers about the available self service options. Help them to use the self service channels (if this applies to enquiry)
  • Send any feedback about agents via Team Leaders to the Smart Centre North Branch Portfolio Operations Team mailbox

5

Specified Personnel verbally provides the customer's identity documents for certification or a linking code + Read more ...

Identity documents can only be accepted verbally from Specified Personnel if:

  • the caller is identified as Specified Personnel at an Agent site (refer to 'Authenticating Specified Personnel'), and
  • the identity documents provided are in the customers current name or the customers former name if that name is already recorded on their record, and
  • the customer is providing an approved photo identification document, and
  • the Agent is advising they have verified the customer's identity by completing a manual-visual check of the photo identification to the customer

Note: if the customer does not have 1 document from each category (Commencement of Identity (CoI), Primary Use in Community (PUiC) and Secondary Use in Community (SUiC)), use the Alternative identity process.

Where the customer is providing identity documents, Specified Personnel must:

  • confirm the customer is providing photo identification
  • verify the customer's identity by completing a manual visual check of the customer's appearance against their photo identification
  • confirm the provided identity documents are in the customer's current name or former name if that name is already recorded on the customer's record

See Coding identity documents for a detailed process.

Where the customer requires a myGov linking code, Specified Personnel must provide:

  • The nominated member service the customer wants to link to their myGov account. One linking code can be used for all Services Australia member services (Centrelink, Medicare and/or Child Support)
  • Acceptable photo identity document (see Fast track registration Identity Confirmation)

Using the myGov linking application, complete a fast track registration, see Using the myGov Staff Linking Application to issue the linking code.

6

Finalise the details + Read more ...

Update the customer's record by using the:

  • Identity Confirmation Dashboard in Process Direct
  • fast track option for issuing a linking code

Under Who Contacted:

  • select CAG-Centrelink Agent
  • include the name and ID number of the Specified Personnel
  • include the ID number in the DOC. Get this verbally from the Specified Personnel
