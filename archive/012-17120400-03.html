This page contains information on how an Assessing Officer (AO) is to complete assessment of a Medical Indemnity Online Claiming (MIOC) refund application.

Self Service

Assessing MIOC Refund applications

Table 1: this table explains how an Assessing Officer (AO) completes an assessment of a MIOC refund application for all schemes.

Step

Action

1

Open an application for assessment + Read more ...

  • Open the Medical Indemnity Online Claiming (MIOC) system
  • From the Home screen > select Approvals from the top task bar

Note: Assessing Officers can also refine the list of applications using the filters (example: Insurer, application type, scheme etc.) above the list of current applications.

When a claim is submitted in the MIOC system, the system will auto-generate an application number which is Services Australia’s unique claim reference number.

A refund application submitted via MIOC will display under the heading ‘ARN’ in the format ARN[Application number]-[Insurer identifier number and letter]-[Scheme identifier letter]-[Insurer refund identifier number]R. For example, ARN6000-1U-H-1R, where R indicates a Refund Application. The Insurer identifier number is the number of the payment application relevant to the claim, i.e. 1 for first application, 2 for second application, and so on. The Scheme Identifier letter represents the scheme that the claim is being made under:

H for HCCS

R for ROCS

I for IBNR

IH for IBNR/HCCS

RH for ROCS/HCCS

  • Select a Refund application with a recommended status of Quality Assurance
  • Select the Refund radio button under Recommended action
  • select View for the application required

The screen displays a summary of the application as well as all information entered by the applicant, including case notes. Any warnings associated with the application will also be displayed here. An internal user can undertake the following actions:

  • Download PDF of the application
  • Withdraw the application on the insurers behalf if needed
  • Recommend action to the delegate (i.e. Approve recovery, Request further info from the insurer or Deem not payable) once an assessment has been undertaken

2

Make a Note that the application is being assessed + Read more ...

MIOC will allow multiple users to work in an application at the same time. A case note needs to be created to let other users know the application is being assessed.

  • Select Notes from the drop down menu at the top of the screen This will take the user to the Notes section in the application
  • Select Add new [note]
  • Select Other from the drop down Subject menu
  • Enter “<initials> is assessing this application” in the Note text box
  • Select Save
  • Add the claim ARN and the Assessing Officer’s initials under the ‘Assessed’ column in the ‘MIOC Work on Hand’ document located in the shared drive at:
    G:\med_indemn\Claims by ARN\Work on Hand

Note: if another user has already made a note to this effect, select Approvals from the task bar at the top of the screen and select another application to assess. Repeat the process at steps 1 and 2 above.

3

Review any case notes the applicant has included in the application + Read more ...

The applicant may have added some notes to their application that the user may need to be aware of.

  • Select Notes from the drop down menu at the top of the screen. This will take the user to the Notes section in the application
  • If notes have been added, select the collapsible section beside the Subject of the note to view the note
  • Select Supporting documents from the drop down menu at the top of the screen. This will take the user to the Supporting documents section in the application
  • Select the collapsible section beside the listed Document to open the note field and check for any notes that may have been added

4

Create a sub folder in the drive for assessment documents + Read more ...

  • Open the ‘Claims by ARN’ folder located in the shared drive at:
    G:\med_indemn\Claims by ARN
  • Select the folder with the ARN number range that the ARN application number falls within
  • Select the sub-folder for the ARN. For example ARN6000U
  • Create a new sub folder, within the relevant claim/ARN sub folder, using the format ARN[Application number]-[Insurer identifier number and letter]-[Scheme identifier letter]-[Insurer refund identifier number]R. For example, ARN6000-1U-H-1R

5

Open and save a MIOC Refund Assessment worksheet + Read more ...

  • Open the ‘MIOC Refund Assessment Worksheet’ template located in the shared drive at:
    G:\med_indemn\Claims by ARN\Assessment\MIOC assessment\
  • Save the document as ‘Refund Assessment Worksheet_ARN[Application number]-[Insurer identifier number and letter]-[Scheme identifier letter]-[Insurer refund identifier number]R. For example, ‘Refund Assessment Worksheet_ARN6000-1U-H-1R’ in the application folder, created at Step 4

6

Complete the MIOC Refund Assessment worksheet + Read more ...

The Assessment worksheet is broken down into the following Sections to correspond with the MIOC sections that the refund is being recovered against:

  • Settlement and Plaintiff Costs
  • Defence Costs
  • Refund Amounts

At the top of the worksheet, complete the sections for ARN, Insured person’s name (taken from the Insured person’s details section in MIOC) and Claim ID (taken from the Claim details section in MIOC).

Note: for a refund application lodged against a migrated application, Assessing Officers will need to check the Insured Person’s name in HPE Content Manager (TRIM).

Assessing Officers can copy and paste information from MIOC into the document.

Follow the instructions and prompts noted on the Assessment worksheet to complete the assessment process and determine an assessment outcome.

Complete all relevant questions of the worksheet, selecting the Answer column checkbox that corresponds with the information provided in the MIOC application. Check that the evidence provided supports the information stated in the MIOC application.

All supporting documents are uploaded by the applicant in MIOC.

  • Select Supporting documents from the drop down menu at the top of the screen. This will take you to the Supporting documents section in the application
  • Select Download beside the document you want to view and Open

The applicant may have supplied more than 1 relevant document as evidence of the information provided, or the same document for different categories of evidence.

Question 1: Has a reason been provided for this recovery?

  • Answer the question by checking the applicable checkbox. If ‘Yes’, locate the supporting evidence and note the document Filename and Page number/s in the Supporting evidence location column of the worksheet. Note any relevant findings or discrepancies in the Comments column on the worksheet

Question 2: Are settlement and/or plaintiff costs being refunded in this application?

Answer the question by checking the applicable checkbox. If ‘Yes’ provide details of the refund amount/s in the Comments column on the worksheet. If ‘No’ go to question 4

Question 3: Is there evidence to support the amount being refunded?

If ‘Yes’ at question 2: Answer the question by checking the applicable checkbox. If ‘Yes’, locate the supporting evidence and note the document Filename and Page number/s in the Supporting evidence location column of the worksheet. If ‘No’ add any relevant findings or discrepancies in the Comments column on the worksheet

Question 4: Are defence costs being refunded?

  • Answer by checking the applicable checkbox. If ‘Yes’ provide details of the refund amount/s in the Comments column on the worksheet. If ‘No’ go to question 6.

Question 5: Is there evidence to support the amount being refunded?

If ‘Yes’ at question 4: Answer the question by checking the applicable checkbox. If ‘Yes’, locate the supporting evidence and note the document Filename and Page number/s in the Supporting evidence location column of the worksheet. If ‘No’ add any relevant findings or discrepancies in the Comments column on the worksheet

Question 6: Is the total amount being refunded in this application correct?

Answer by checking the applicable checkbox. If ‘Yes’, provide details of the total amount and the refunded amount in the Comments column on the assessment worksheet under Refund Amounts (Noting HCCS amounts refunded are 50% of the total refund and exclusive of GST)

Perform a reconciliation of the costs being claimed:

  • Open the ‘MIOC Expense Reconciliation template’ located in the shared drive at:
    G:\med_indemn\Claims by ARN\Assessment\MIOC assessment
  • Save the document as ‘MIOC Expense Rec_ARN[Application number]-[Insurer identifier number and letter]-[Scheme identifier letter]-[Insurer refund identifier number]R’, for example ‘MIOC Expense Rec_ARN6000-1U-H-1R’, in the relevant application folder
  • Copy and paste the refund information listed under Judgment/settlement and plaintiff costs (under Cost type, Total amount, Amount claimed and Apportionment) or Defence Costs (under Invoice date, Invoice number, Amount claimed and Cost shared) from MIOC into the expense reconciliation spreadsheet
  • Check that each cost being claimed as a refund is supported by the invoice or evidence provided and that the amount calculated at Total amount to refund in this application in MIOC reconciles with the MIOC Expense Reconciliation spreadsheet
  • When all invoices have been checked, copy the expense reconciliation information from the spreadsheet to the end of the assessment worksheet under MIOC Expense Reconciliation. Select the checkbox to indicate the invoices being refunded have been checked against the application in which the relevant costs were claimed
  • In the assessment worksheet, make a Recommendation by selecting the checkbox for either Approve Refund or Request Further Info and note any issues, discrepancies or comments in the Additional Notes section

Create a case note in MIOC in the Notes section, detailing any issues, discrepancies or comments from the assessment, particularly if it could result in the application being deemed not payable or where further information is required.

7

Creating case notes where issues are found during assessment + Read more ...

If there is information about your assessment that the Approver (delegate) needs to be aware of, create a case note in MIOC in the Notes section.

This must be done for every application where you recommend Deem not payable or Request further info.

The note must be detailed enough to enable the Approver (delegate) to understand the reasons for your recommendation.

Assessing Officers can also add a note if the recommendation is Approve for refund but this is not mandatory.

To create a case note:

  • Select Notes from the drop down menu at the top of the screen. This will take you to the Notes section in the application
  • Select Add new [note]
  • Select the relevant section, from the drop down Subject menu (example: where a defence cost document is not provided or incorrect, select Defence costs)
  • Enter the relevant details in the Note text box (noting there are character number and symbol limitations, for example % is not accepted)
  • Select Save

To finalise the assessment and make a recommendation to the delegate, see Medical Indemnity (MI) Schemes finalising assessment.

Assessing Refund applications

Table 1: this table explains how an Assessing Officer (AO) completes an assessment of a refund application for all schemes.

Step

Action

1

Registration of a refund application + Read more ...

Before an MI refund can be assessed, the Service Officer will need to register the refund application and complete the following actions:

  • Receive MI claim and update the Claims on Hand spreadsheet (see the Resources page)
  • compile the supporting documentation file, and
  • folio relevant pages

Place the paper file in the Assessment drawer of the Claims on Hand cabinet (for handover to an Assessing Officer).

2

Assessment of a refund application + Read more ...

The Assessing Officer will continue the process to finalise the assessment of the refund application.

Read through the application and supporting documentation.

Confirm whether the applicant is requesting a recovery of the total paid to date or a partial amount by checking the comments on the refund application form, invoices provided and applicant's covering letter.

If the amount of the refund requested cannot be determined by the documentation, refer to HP Records Manager and use the claim identifier to confirm payments made to date.

Compare the total payments made to date figure with the total refund amount quoted in the refund application form to determine if the applicant is seeking a refund of the total amount paid by Services Australia to date.

Once the refund amount has been determined, confirm that sufficient supporting documentation has been provided to support the refund.

3

Update HP Records Manager + Read more ...

Update HP Records Manager with the refund details.

Open HP Records Manager via Start > All Apps > HP Records Manager:

  • Ensure the Search By field displays 'Record Number'
  • In the Equal To field, search for the claim’s Application Reference Number (ARN) number
  • Right-click on the application file relating to the refund
  • Scroll down and left-click on 'Properties'

Note: if this is a partial recovery relating to a specific application, select the relevant application, otherwise select the most recent application.

In the General tab, update the:

  • Application Status field to 'Recovery in Progress'
  • Status updated field, key today’s date

In the Financial Recoveries tab, include the date the refund application was received in the Date Recovery Notified field.

Note: recoveries are offset against the next available payment to the applicant.

4

Create an Expense Reconciliation spreadsheet + Read more ...

As with MI payment applications, the supporting invoices provided with the refund application must reconcile with the amount requested in the refund application. The type of refund identified when reviewing the refund determines how the requested refund amount is reconciled.

If it is determined that the refund is for the:

  • whole amount 'paid to date', a reconciliation of all invoices provided and paid under all payment applications relating to the claim must be completed
  • partial recovery, the invoices provided must be reconciled with the refund application to support the request

The expense reconciliation spreadsheet will be used to determine if the refund application contains any apportionment of liability. This should be identified when reviewing the application and supporting invoices. See the Resources page for FAQ about apportionment.

Select the appropriate MI Scheme Expense Reconciliation Spreadsheet. Review the information and detail provided within each invoice. If the refund is for a partial recovery, ensure the invoices provided match any invoices that have been paid under a previous application.

Key invoices in the Expense Reconciliation Spreadsheet ensuring the pre-GST invoice amounts are keyed into the yellow column of the spreadsheet.

Compare the total amount on the 'Expense Reconciliation Spreadsheet' with the refund application.

If the figures do not match, include notes on the 'Expense Reconciliation Spreadsheet' detailing where discrepancies have been identified.

Note: where financial discrepancies exist, the Assessing Officer must refer to the Section 38 request procedures to seek further information from the applicant. See Medical Indemnity (MI) Schemes finalising assessment.

5

Complete the debt recovery worksheet + Read more ...

When the supporting invoices have been reconciled, complete a debt recovery worksheet.

Depending on which MI Scheme the refund application relates to, open the appropriate Debt Recovery worksheet. For example, HCCS Debt Recovery Worksheet.

Complete the following fields on the worksheet:

  • ARN, key the claim number allocated from HP Records Manager. If the refund relates to:
    • invoices in a specific application, use the full HP Records Manager reference. For example, ARN1234-6X-I
    • various invoices across a number of applications, complete a Debt Recovery Worksheet for each ARN number
  • Claim Reference Number, key the unique claim identifier that the applicant has allocated the member on the refund application form
  • Date Recovery Commenced, date Services Australia was notified, as indicated by the date stamp
  • Dr’s Name, key the name of the member involved in the claim
  • Member Number, key the membership number of the member
  • Specialist Field, key the specialisation of the member at the time of the incident

Complete Section 1 titled 'Breakdown of the indemnity scheme payment amount subject to recovery' by keying the relevant refund amounts under the A/C codes and key in the relevant folio numbers. See the Resources page for Vendor codes and general ledger accounts codes.

The ESSentials document number is to be completed once the amount has been recovered.

Complete Section 2 of the Debt Recovery worksheet by keying the reason for recovery as indicated on the refund application form.

Save the worksheet in the relevant ARN folder in the shared drive, titled [Scheme] Refund Worksheet [ARN]. For example, HCCS Refund Worksheet ARN 1234-1A-H.

6

Create a file note + Read more ...

Open the file note that already exists for the MI application (for example, ARN 1234-2X-R).

Save a new copy as 'ARN 1234-1X-R-Refund'.

Select Design tab > Watermark icon > Custom Watermark > Text Watermark. A pop up box will appear, select:

  • Language: English (Australian)
  • Text: key REFUND
  • Font: Arial
  • Size: Auto
  • Colour: Green (use drop down options)
  • Semi-transparent: should be selected
  • Layout: Diagonal

Select Apply > Close for the watermark to display on the background of the file note.

7

Key content for file note + Read more ...

In the File Note > Background section > Application number > key ‘This is a refund application’.

Key the refund amount at ‘Amount sought’ as a negative dollar figure in red font. For example ‘-$100.00’.

Review the ‘Summary’ section of the file note to ensure that it is correct. This section is a summary of the incident(s) that gave rise to the claim.

Update the ‘Assessment of Claim’ section to include any details relevant to the assessment of the refund, such as:

  • details of the reason(s) for the refund
  • any discrepancies or issues found
  • email/Section 38 requests made and responses received

Where it is deemed that all necessary information is available to undertake the recovery, key the sentence ‘Assessment of the refund has shown that the required criteria has been met and the necessary supporting documentation and financials have been provided.’

If additional information is required to complete the recovery, see Finalising assessment of a Medical Indemnity (MI) payment application.

In the section ‘Claims Recommendation to Delegate’, include the following statement:

  • ’I recommend you approve the refund of $[insert amount], consisting of settlement cost [insert amount], plaintiff costs [insert amount], and defence costs [insert amount] as all legislative requirements have been met.’

Once the file note is complete:

  • check and proofread all documentation completed during assessment of the refund application
  • save and print the file note and Debt Recovery worksheet
  • complete a covering sheet ‘Refund Summary’, and
  • update HP Records Manager and Claims on Hand (amending any details as required)

Place the file in the QA cupboard.

8

Update Section 83 potential breach spreadsheet + Read more ...

To update the section 83 spreadsheet, go to the share drive and open the ‘Section 83 potential breach of the Constitution Register Complex Payment Section for MI and MPIS’ for the applicable financial year.

Key the details about the refund into the Section 83 potential breach spreadsheet. A new spreadsheet is created for each financial year.

9

Quality checks and obtaining delegate approval + Read more ...

Once the Assessing Officer (AO) has assessed the refund application, a second Quality Assurance (QA) check is required from another AO.

The second AO is required to conduct:

  • an assessment of the refund application to determine of the correct assessment decision has been made
  • a final proofread to ensure formatting, spelling, grammar and financials are correct. If unsure refer to the Services Australia Writing Style Guide on the Resources page.

In the Debt Recovery Worksheet, complete the following:

  • Q.A. checked by field, key AO name
  • complete the Refund form checklist

For the file note:

  • proofread the document completed to ensure the assessment of the refund application is correct
  • leave the EL1 recommendations signature and date block blank for the EL1 to manually sign and date

Note: if a different Team Leader reviews the claim, they can cross out the typed name, handwrite their name and sign the file note if they support the AO recommendation

If during the QA, the second AO finds errors or issues in relation to the refund application assessment these need to be discussed with the first AO and if necessary the EL1 Assistant Director. Once agreement has been reached all relevant documentation must be amended as required.

Once the second AO is sure that the refund application has been assessed correctly and the documentation is correct:

  • Update Claims on Hand
  • Place the file in the EL1's cupboard for EL1 Assistant Director to conduct a final Quality Check (QC) before the claim is handed to the delegate for their consideration

The EL1 checks the entire claim including the refund application, the recovery amount, HP Records Manager, the debt recovery worksheet, the file note and the recommendation. If the EL1 agrees with the AO recommendation, the file is then handed over to the EL2 delegate for approval.

If any errors are identified at this point the claim is then handed back to the first AO. Once the first AO has amended any errors, the claim undergoes QC again before progressing to the EL2 delegate for approval.

Once completed, the EL1 will progress the claim to the EL2 (the delegate), Director of the Medical Indemnity team for approval. If the Director agrees with the AO recommendation they will sign the debt recovery worksheet on the file. The application is then ready for recovery of the amount as outlined in the pay run process.

If the delegate does not agree with the recommendation, the delegate will discuss with the EL1 for resolution of the refund application.

See Processing pay run for Medical Indemnity (MI) payment application.

Withdrawing MI applications

Table 2: this table describes how Registering Officers (RO) and Assessing Officers (AO) are to withdraw Medical Indemnity (MI) claims for all MI schemes.

Step

Action

1

Withdrawing an MI application + Read more ...

The RO will need to register the withdrawal by compiling the following actions:

  • date stamp correspondence from insurer advising they are withdrawing the MI application
  • update the Claims on Hand spreadsheet (see the Resources page)
  • compile the supporting documentation file, and
  • folio relevant pages
  • Hand over file to AO

2

Action request for withdrawal + Read more ...

To action the withdrawal the AO needs to update HP Records Manager:

  • Open HP Records Manager via Start > All Programs > HP Records Manager
  • Ensure the Search By field displays 'Record Number'. In the Equal To field, search for the claim’s ARN number
  • When the file is displayed in the search box, right click on the file and select Properties from the drop down box
  • In the Notes tab:
    • write a brief explanation why the claim has been withdrawn
    • key initials and logon ID
    • select OK
  • In the General tab, update the following fields:
  • Application status: withdrawn
  • Status Updated: today's date
  • Select OK, to update TRIM with the information keyed
  • Place the paper file in the Assessment drawer of the Claims on Hand cabinet (for handover to an AO)

3

Create a file note + Read more ...

The AO will continue the process from here to finalise the assessment of the withdrawal.

Open the file note that already exists for the application (for example, ARN 1234-2X-R) and select Design tab > Watermark icon > Custom Watermark > Text Watermark. A pop up box will appear, select:

  • Language: English (Australian)
  • Text: key WITHDRAWN
  • Font: Arial
  • Size: Auto
  • Colour: Red (use drop down options)
  • Semi-transparent: should be ticked
  • Layout: Diagonal

Select Apply > Close for the watermark to display on the background of the file note.

4

Key content for file note + Read more ...

Key the following content in the sections below:

  • Assessment of Claim, ‘[Insurer] has advised they wish to formally withdraw the application for payment (folio xx).’ Note: the folio reference is the application/letter/email received regarding the withdrawal of the application
  • Claims Recommendation to Delegate, ‘I recommend you sign the attached letter formally acknowledging withdrawal of the claim.’

Save the file note in the relevant folder, titled ‘FILE NOTE – ARN [reference no] – Withdrawn’.

5

Create letter acknowledging MI application has been withdrawn + Read more ...

In the shared drive, open the relevant MI Scheme ‘Withdrawal Letter’ template.

Update the information in the following sections of the letter:

  • ‘Date’, key the pay run date for the relevant month in which the withdrawal application/letter/email was received
  • ‘Our Reference', key the correct ARN number for the application
  • 'Your Reference’, key the Insurers claim reference number (see application form)
  • Insurer’s address, key the correct address of the relevant insurer
  • Body of the letter, key the:
    • date the application was originally received
    • Dr Name. For example, 'Dr [First Name] [Surname]
    • date the withdrawal letter was received

Save the letter as Withdrawal_Letter_ARN####-#A-H in the relevant ARN folder.

Once the file note is complete:

  • check and proofread all documentation completed during assessment of the withdrawal
  • save and print the file note and Withdrawal letter
  • complete a covering sheet ‘WITHDRAWAL’, and
  • update HP Records Manager and Claims on Hand (amending any details as required)

Place the file in the QA cupboard.

6

Quality checks and obtaining delegate approval + Read more ...

Once the AO has assessed the refund application, a second Quality Assurance (QA) check is required from another AO.

The second AO is required to conduct:

  • an assessment of the withdrawal
  • a final proofread of the final note and letter to insurer to ensure formatting, spelling, grammar and financials are correct. Refer to the Services Australia Writing Style Guide on the Resources page.

If during the QA the second AO finds errors or issues in relation to the withdrawal these need to be discussed with the first AO and if necessary the EL1 Assistant Director. Once agreement has been reached all relevant documentation must be amended as required.

Once the second AO is sure that the withdrawal has been assessed correctly and the documentation is correct:

  • update Claims on Hand
  • place the file in the EL1's cupboard for EL1 Assistant Director to conduct a final Quality Check (QC) before the claim is handed to the delegate for their consideration

The EL1 checks the entire claim including the withdrawal advice received from the insurer, HP Records Manager, the file note and the recommendation and withdrawal letter. If the EL1 agrees with the AO recommendation, the file is then handed over to the EL2 delegate for approval.

If any errors are identified at this point the claim is then handed back to the first AO. Once the first AO has amended any errors, the claim undergoes the QC again before progressing to the EL2 delegate for approval.

Note: if a different Team Leader reviews the claim, they can cross out the typed name, handwrite their name and sign the file note if they support the AO recommendation.

Once completed, the EL1 will progress the claim to the EL2 (the delegate), Director of the Medical Indemnity team for approval. If the Director agrees with the AO recommendation they will sign the withdrawal letter.

If the delegate does not agree with the recommendation, the delegate will discuss with the EL1 for resolution of the refund application.

See Monthly Medical Indemnity (MI) administrative process.
