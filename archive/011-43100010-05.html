Medicare Two-way PHEB screen messages

Table 1: This table describes the messages and fields on the PHEB screen. Search the list using [Ctrl + F].

Message

Description

E-FUND BND REQ

A valid private health insurer brand ID must be entered in the FUND BRAND ID field. If the fund brand ID cannot be determined, press [F12] to cancel and return the claim with an appropriate letter.

E-INV FUND BND

The fund brand ID is not valid. If the fund brand ID cannot be determined, press [F12] to cancel and return the claim with an appropriate letter.

E-MEM REQ

The private health insurer membership number has not been keyed in the MEMBERSHIP NO field. Key the private health insurer membership number and continue processing.

E-MEM FORMAT

The private health insurer membership number keyed does not match the membership number validation rules for the private health insurer. Check keying, correct any errors, and continue processing.

If the membership number has been keyed correctly, press [F12] - CANCEL. The claim must be either rejected and returned to the claimant or sent to the private health insurer manually.

E-CARD REQ

The Medicare card number has not been keyed in the CARDNUMBER field. Check keying, correct any errors and continue processing.

E-NUMERIC REQ

The Medicare card number keyed is not numeric. Check keying, correct any errors and continue processing.

E-ISS NO REQ

The Medicare card number keyed is valid but the issue number has not been keyed. Check keying, correct any errors and continue processing.

E-INV ISS NO

The Medicare card number keyed is valid but the issue number is superseded. Check keying, correct any errors and continue processing.

E-PATIENT REQ

The patient's name has not been keyed in the PATIENT field. Key the first five characters of the patient's name or the sub-numerate, and continue processing.

E-INV PATIENT

The patient name keyed does not match a person enrolled on the card number. Key the sub-numerate or patient name details. If the patient cannot be identified, return claim with appropriate letter.

E-PATIENT SEL

There are two people on this card number with the same first name. Check the claim form and determine the sub-numerate for the patient. Overkey the patient's first name with the appropriate sub-numerate.

E-INV ACT

The PERMANENT ADDRESS field has invalid characters. Valid change of address characters are:

  • P - permanent
  • T - temporary
  • ? - not indicated

E-ACT REQ

The PERMANENT ADDRESS field indicator needs to be keyed. Valid change of address characters are:

  • P - permanent
  • T - temporary
  • ? - not indicated

E-COMP REQ

A compensation case indicator has not been keyed in the COMPENSATION CLAIM field. Valid values are:

  • Y – services are related
  • N - services are not related
  • ? - not indicated

E-INV COMP

The compensation case indicator in the COMPENSATION CLAIM field is invalid. Valid values are:

  • Y – services are related
  • N - services are not related
  • ? - not indicated

E-HOSP FRM REQ

The hospital admission date has not been keyed in the HOSP FROM field. Key the date from the claim form. If there is no hospital admission date shown on the claim form and an account/receipt is attached to the claim form, the date keyed should be equal to the date of service of the first in-hospital service. If no account/receipt is attached, forward documents to relevant private health insurer.

E-INV DATE

An invalid date has been keyed in HOSP FROM/TO field. The date keyed must be prior to or equal to today's date and in the format DDMMYY.

E-TO PRIOR

The date keyed in the TO field is less than the date keyed in the FROM field. Check keying, correct any errors and continue processing.

E-GT 2 YRS

The date in the HOSP FROM field is greater than two years from the date of lodgement (DOL) on the PHEB screen. Check keying, correct errors and continue processing.

E-HOSP TO REQ

The hospital discharge date has not been keyed in the TO field. Key the date from the claim form in the format DDMMYY. If the date has not been supplied, key a question mark '?'.

E-TODAY REQ

The today claim indicator must be keyed in the TODAY CLM field. Valid values are:

  • Y - this will return all the services with dates of service within the FROM and TO date fields that have been processed today
  • N - this will return all the services within the FROM and TO date fields

E-INV TODAY

The today claim indicator keyed in the TODAY CLM field is invalid. Valid values are Y and N. Y will return all the services with dates of service within the FROM and TO date that have been processed today. N will return all the services within the FROM and TO date fields.

E-HOSP NME REQ

The name of the hospital must be keyed in the HOSP NME field. Check the claim form; key the name of the hospital supplied. If the name has not been supplied, key a question mark '?'.

E-NAME REQ

The claimant's family or first name is required in the highlighted fields. Key name details in the appropriate fields and continue processing.

E-ADDR REQ

The first line of the claimant's address has not been keyed. Key the first line of the claimant's address, for example, 55 Example Pl.

E-LOCL REQ

The locality (suburb) has not been keyed. Key the claimant's locality (suburb) in address line 3 and continue processing.

E-PC REQ

The claimant's postcode has not been keyed. Key the claimant's postcode in the postcode field and continue processing.

E-INV PC

The P/C field has been left blank. Key the required details and continue processing.

E-NO SELECT

No item lines have been selected from the list of in-hospital services. Key 'S' (select) against the item lines required and continue processing.

E-INV SELECT

A value other than S or '/' has been keyed in the S (select) field. Check keying, correct any errors and continue processing. For PHEB, key 'S' against the items required and continue processing.

E-NOT ECLIPSE

The ECLIPSE fund brand ID keyed cannot be found. Check keying, correct any errors and continue processing.

E-2WAY NOT

The fund brand ID keyed does not have a current electronic two way contract. This claim must be manually forwarded to the private health insurer.

E-NO SVC FND

No in-patient services have been found for the patient using the dates keyed in the HOSP FROM and TO fields. Check keying, correct any errors and continue processing.

E-TOO MANY SVC

More than 100 service lines have been identified. Refine the search criteria to limit the number of service lines and continue processing.

W-DUP SELN

The service line(s) highlighted yellow has been transmitted to another private health insurer brand or has been transmitted to the same private health insurer brand and either it has been rejected or the transmission has failed. This service line(s) can be selected and retransmitted.

E-DOB/HOSP FR

The admission date entered is before the patient's date of birth. Verify date of admission, amend and re-enter.

W-PATH SELN

Pathology items outside HOSP FROM/TO can be selected.

Medicare Two-way patient claims field and message definitions

Table 2: This table describes the messages and fields on the Patient Claim screen.

Message

Description

6-FUND STM

Displays on all patient claims screens prior to P TO PAY for all in hospital claims providing the in-hospital service is payable.

FUND STM

After the 6-FUND STM message displays, the cursor is automatically positioned under the highlighted FUND STM field. To complete entry, key 'Y' or 'N' in the FUND STM field:

  • Y - gap payment information is to be forwarded to the private health insurer
  • N - no reason to forward gap information to the private health insurer

8-ADDR CHK

When the processing screen is first accessed, the cursor is positioned on the ADDRESS CHECK field and 8-ADDR CHK displays. Claim to be processed in the usual manner.

Note: FUND STM message is suppressed from the patient claims screen at this stage.

2-FUND PAT

Displays for a claim for services for more than one patient and both consist of in-hospital items.

Note: claims which cover services for more than one patient do not need to be split when only one patient claims in-hospital items:

  • the office copy receipt and the customer copy cover all patients within the claim
  • the gap information receipt shows only data for the patient with in-hospital services
