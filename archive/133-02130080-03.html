How to reset a password in PRODA

Table 1

Step

Action

1

Help with self service + Read more ...

The first option is to help the user reset their password online.

Has the user been able to reset their password?

  • Yes, procedure ends here
  • No, go to Step 2

2

Access user's PRODA account + Read more ...

Access the PRODA system

Select Inquiry type related to the inquiry.

Search for PRODA account.

For Inquiry type select Professional.

Search for PRODA account and conduct security check.

3

Reset password + Read more ...

On the Manage Account screen, select Reset Password button in the Security section of the screen.

Note: if the button cannot be accessed, check the account status is Active. If the account is not active, check the account history for the reason. If after investigation the account can be made active, see Reactivate cancelled or suspended accounts table in Managing account status and displaying records for PRODA accounts.

Confirmation message:

'You are about to reset the account password, do you want to continue?'

Select either:

  • Yes, PRODA system resets the password and sends an email to the user's email address. Note: a reset password is always emailed to the user
  • No, the Manage Account screen displays

Procedure ends here.

How to change email and phone numbers in PRODA

Table 3

Step

Action

1

Help with self service + Read more ...

The first option is to help the user update their details online.

Has the user been able to update contact details?

  • Yes, procedure ends here
  • No, go to Step 2

2

Confirm access to email/mobile + Read more ...

Confirm that the user has access to the email/mobile which is being changed, so they can retrieve the activation code.

Email: an email containing the activation code will be sent to the new email address.

Mobile phone: a text containing the activation code will be sent to the new mobile number listed.

Explain to the user that the activation code is needed to save the requested change of details.

The activation code is valid for 30 minutes only.

Does the user have access to the email/mobile which is being changed?

  • Yes, go to Step 3
  • No, advise unable to continue and to please call when they have access to retrieve the activation code. Procedure ends here

3

Access user's PRODA account + Read more ...

Access the PRODA system

For Inquiry type select Professional.

Search for PRODA account and conduct security check.

4

Access Manage contact details screen + Read more ...

On the Manage Account screen, select the Manage contact details button in the Contact details section of the screen.

Note: if the Manage contact details button cannot be accessed, check the account status is Active. If the account is not active, check the account history for the reason. If after investigation the account can be made active, see Reactivate cancelled or suspended accounts table in Managing account status and displaying records for PRODA accounts.

Result: the Manage contact details screen displays. Go to Step 5.

5

Enter new contact details + Read more ...

Confirm the old details with the user before updating these details.

Enter the user's new contact details.

  • Email address and mobile number must be entered a second time. Cut and paste cannot be done.
  • Mobile numbers must start with '04'.

When completed, select the Validate button.

Note: mobile phone numbers can be deleted by selecting the Validate button.

  • If validated, the Confirm new contact details section will be displayed on screen
  • If invalid, the screen will redisplay with an error message. The Resources page contains a table of relevant error messages. Re-enter details

6

Send activation code + Read more ...

In the Confirm new contact details section of screen, select the Send code button for email/mobile.

The Send code button is disabled.

The PRODA system sends an activation code to the user's device/s.

Note: the Send code button will need to be selected for all options. A different code will be sent to each device.

7

User retrieves activation code from device/s + Read more ...

Enter activation code provided by the user:

  • Email: enter email activation code into Email code enter code box
  • Mobile: enter mobile phone activation code into mobile phone code enter code box

When completed, the Submit button is active.

Select the Submit button.

Does the activation code match?

  • Yes, the system will save the updated phone number and/or email address. Procedure ends here
  • No, go to Step 8

8

Resend activation code when entered incorrectly + Read more ...

When the activation code is entered incorrectly, an error message is displayed on the Manage contact details screen. See Resources page for relevant error messages.

The same code can be entered 5 times. After the 5th attempt the Send code button is enabled. Select the Send code button again.

Note:

  • Each code is valid for 30 minutes
  • A new code cannot be sent to a mobile phone number within 5 minutes of sending the previous code

If code times out or reached maximum number

If a total of 5 codes are sent out but each code times out or is entered incorrectly 5 times the Send code button will be disabled for 6 hours. There is no override for an account that has been disabled for 6 hours by the system.

Advise the caller they will require a call back and ask for the following details:

  • caller's name
  • best contact number
  • best time for call back
  • description of error (account disable for 6 hours due to 5 x 5 code attempts)

Send email to Tier 2: PRODA for call back.

Procedure ends here.

How to link identifiers for Healthcare providers and administrators in PRODA

Table 5

Step

Action

1

Help with self service + Read more ...

The first option is to help the user link their identifiers online.

Has the user been able to link their identifiers?

  • Yes, procedure ends here
  • No, go to Step 2

2

Access user's PRODA accoun + Read more ...

Access the PRODA system

Select Inquiry type related to the inquiry.

Search for PRODA account.

For Inquiry type select Professional.

Search for PRODA account and conduct security check.

3

List all identifiers linked to account + Read more ...

On the Manage account screen, the Manage services button will display for all users with an identity status of Known and an account status of Active.

Select the Manage services button.

On the Manage services screen, select the Healthcare providers and administrators tile.

The Healthcare providers and administrators screen will display:

  • linked identifiers for idenfiers already linked to this account
  • identifiers requiring assistance (flagged identifiers)

If more than 10 identifiers are linked, use scrollbar on the right hand side of the table.

4

Add an Identifier + Read more ...

Under the Add Identifiers section of the screen:

  • Select down arrow under Type of identifier
  • Select appropriate type as supplied by the user. See Resources page for a full list of identifiers
  • Enter identifier number into the Identifier text box
  • Select Search button

Note: an identifier can only be linked to 1 account at a time.

Is identifier matched?

  • Yes, a success message is displayed at the top of the screen and the identifier is added to the Linked Identifiers table. Procedure ends here
  • No, go to Step 5

5

Identifier not matched - new fields/buttons + Read more ...

The Manage Identifiers screen is redisplayed with an error banner.

See Resources page for a complete list of errors.

New fields/buttons will be displayed under the Search button.

  • Column Headings
    • Type of Identifierl lists the type of identifier selected by Service Officer
    • Identifier shows identifier text entered by Service Officer
    • Comment will be blank except for identifiers entered by health professionals that have been flagged for investigation
  • Operator comment text box
  • Reject and Approve buttons

6

Investigate identifier not matching + Read more ...

Investigate if the identifier has been keyed in incorrectly and belongs to this user.

If an error message (excluding PIA0095) is displayed, escalate to Tier 2. Procedure ends here.

Has the identifier been keyed in correctly?

  • Yes, go to Step 7.
  • No, add comment into the Operator comments text: 'Keyed incorrectly
    • Select Reject
    • Amend the identifier number in the Identifier text box
    • Select the Search button
    • Go to Step 8

7

Identifier entered correctly + Read more ...

Does the identifier belong to the user?

  • Yes, add comment into the Operator comments text: 'On investigation this identifier does belong to account holder'. Select the Approve button. Procedure ends here
  • No, go to Step 8

8

Search result for amended identifier + Read more ...

  • If approved, the identifier is linked to the PRODA record and access is granted to the applicable services. The flag is removed from this identifier. Procedure ends here
  • If the identifier does not belong to the health professional, go to Step 9

9

Identifier does not belong to the health professional + Read more ...

  • Add comment into the Operator comments text: 'On investigation this identifier does not belong to account holder'.
  • Select the Reject button.

The identifier is not linked to the PRODA record and interaction is logged in account history. The flag is removed from this identifier.

Advise the caller they will require a callback and ask for the following details:

  • caller's name
  • best contact number
  • best time for call back
  • description of error (on investigation this identifier does not belong to account holder)

Send email to Tier 2 PRODA for callback,

Procedure ends here.

Under Linked Identifiers find the identifier the user has requested to be removed.

Select Remove.

Is the Identifier type Medicare Provider number or Medicare Provider Stem?

  • Yes, the Remove button will remove all linked provider numbers from the account. Go to Step 4
  • No, the Remove button will remove just the identifier in the selected row. Go to Step 4

4

Result of removing identifier + Read more ...

A confirmation message displays:

'You are about to remove an identifier from the account, do you want to continue?'

Select either:

  • Yes,
    • The screen will re-display with the identifier(s) removed. Note: identifiers are removed from the PRODA account only. Removal does not affect the source record, e.g. PDS/EHP (not cancelled/deactivated/removed). Identifiers can be re-linked at any time
    • Removed identifiers will be written to the history table
  • No, the screen remains unchanged

Procedure ends here.
