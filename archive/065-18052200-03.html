Register claims and cancel registration

Table 1: this table describes how to register a claim or cancel registration. The cancel claim registration is applicable to both manual and online claims.

Step

Action

1

Register claim or cancel claim registration + Read more ...

Note: the claim registration process is automatically done as part of the claim finalisation process for:

  • online claims via the Aged Care Provider Portal (ACPP)
  • manual claims processed in the Aged Care Staff Portal (ACSP)

The claim registration process is only required for manual claims processed in the Aged Care Management Payment System (ACMPS).

Does the manual claim need to be registered in ACMPS?

  • Yes, go to Step 2
  • No, the claim has already been registered and a cancel claim registration is required. Go to Step 4

2

Claim registration search for manual claims + Read more ...

Service Officers must check if the claim has already been registered in ACMPS.

  • Log on to ACMPS
  • On the Home screen, select Claim Approval from the side menu. The Service screen will display
  • Select Claim Registration Search. The Claim Registration Search screen will display

To narrow the search results, key the following fields:

  • Service ID - can be keyed manually, or selected from search results (to search, select the Search icon to open the search dialog box)
  • Claim Month - can be keyed manually, or selected from the drop down option
  • Claim Year - while defaults to the current year, the year can be entered manually
  • Select Search

The search results will display:

  • Receipt Date - the date when the claims were received
  • Return Date - the date when claims were returned for clarification
  • Return Indicator - if the check box is ticked, this indicates the claims were returned for further clarification
  • User ID - this displays internal users’ IDs if the claim has been registered in ACMPS
  • Date/Time Registered - this displays internal users’ IDs if the claim has been registered in ACMPS

Is the claim already registered?

  • Yes, procedure ends here
  • No, go to Step 3

3

Register claim + Read more ...

  • Log on to ACMPS
  • On the Home screen, select 'Claim Approval' from the side menu. The Service screen will display
  • Select 'Claim Registration'. The Claim Registration screen will display
  • Key the following fields:
    • Claim Year - can be keyed manually. By default, the current year will display
    • Claim Month - select the relevant month from the drop down list. By default, the current month will display
    • Service ID – can be keyed manually
    • Receipt Date – key the Date of lodgement from PaNDA
  • Select Register Claims

The claim registration process is complete.

Procedure ends here.

4

Cancel claim registration + Read more ...

The cancel claim registration process is used when duplicated claims have been created in error. The step applies to manual and online claims.

  • Log on to ACMPS
  • On the Home screen, select 'Claim Approval' from the side menu. The Service screen will display
  • Select 'Claim Registration Search'. The Claim Registration screen will display
  • To narrow the search results, key the following fields to search registered claims:
    • Service ID
    • Claim Month
    • Claim Year
  • Select Search
  • Select the relevant claim month to be cancelled. Note: multiple claims can be selected
  • Select Cancel. The claim has been cancelled
  • Go to Step 3 to register the claim

Note:

  • The search fields displayed are not mandatory
  • The search can be performed using single or multiple parameters

Creating events

Table 2: this table describes how to process home care events in ACMPS. Flexible care events are created in the Aged Care Staff Portal (ACSP). Note: home care events can also be processed in the ACSP. See Aged Care Staff Portal (ACSP) – Service in Context for these steps.

Step

Action

1

Choose menu options + Read more ...

An entry event must be created before creating any other claim event for a care recipient.

Note:

  • An Aged Care Entry Record (ACER) must be accompanied with the entry event
  • If the event fails the validations, an error message will display and the status is updated as Rejected
  • Selecting the Cancel Event icon will update the status to Deleted
  • If the event is outside the 70 day or the 2 year limit, Service Officers must be satisfied with the request and relevant supporting evidence from the aged care service prior to making any updates
  • An entry event is an exception and can be created outside the 70 day limit
  • For the 70 day limit
    • This is only applicable to departed care recipients whose departure date from the home care service occurred on or after 1 September 2021
    • All other events in relation to the 70 day limit will not be in Held status. That is, the event will have a Processed status prior to the claim being finalised and an Event Claimed status after the claim has been approved
  • See Table 9 Approve/reject events outside the 70 day limit in ACMPS for the investigation process
  • A manual adjustment is required as a result of Service Officers creating an event before 1 January 2020
    • See Table 10 Approve/reject events outside the 2 year limit in ACMPS for the process

  • The New Event screen displays

Table 4: this table describes how to finalise claims. Before a claim can be calculated, there are some basic pre-conditions that need to be met. If these are not met, then the claim cannot proceed to the calculation and payment processes. Note: a claim must be registered before it can be finalised.

Step

Action

1

Finalising the claims + Read more ...

Note: for home care claims, an error message will display if there are missing invoice amounts. Service Officers must enter the missing amounts or set all the care recipients’ invoice amounts to zero dollars ($0). See Step 11 in the Using the Aged Care Staff Portal for home care purposes -(Service in Context) table.

Service Officers can finalise the claim once all the missing amounts have been entered or all amounts have been set to zero dollars ($0).

  • Select Search, the search results will display
  • Select either Finalise Claim or Finalise in Background.
    Note: while both options serves the same purpose, Service Officers can use the Finalise in Background option for services with large claim amounts to reduce time out errors. Clearing errors if the Error Status contains errors + Read more ...

If the Error Status of the retrospective claim contains errors, Service Officers need to select:

  • Undo Decision Assessment option to undo the retrospective claim calculation
  • Decision Assessment option to ensure the retrospective claim calculation
  • Save if the error has been resolved

If the error is still present, Service Officers need to:

  • repeat the Undo Decision Assessment option
  • select Eligibility Determination option
  • Save if the error has been resolved
  • repeat the Undo Decision Assessment option
  • select Entitlement Determination option
  • repeat the Undo Decision Assessment option
  • select Entitlement Calculation option
  • Save

If the error persists, Service Officers will need to ensure the escalation process is actioned appropriately. See a Team Leader for the escalation process.
