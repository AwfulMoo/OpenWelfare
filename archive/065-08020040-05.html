Manual Follow-Up (MFU) warnings and errors

Message code

System to generate

Message

Reason

CAO action

W011PH

PLS

Refund Amount is more than outstanding loan balance

Payment is greater than the loan Balance (Costs + Interest + Principal).

MFU also created

Investigate the customer’s record.

Decide:

  • how much of the receipt will be applied to Principal, Interest and Cost
  • if a refund is applicable

Once amounts confirmed update the loan.

  • Log into ESSentials (SAP R1P)
  • Key transaction ‘YFR_PLS’
  • Enter document number and financial year document was created
    Note: the fiscal year is usually the current financial year unless receipt was in previous financial year
  • Select Repost and Refund
  • Enter the amounts against Principal, Interest and Cost, and if any funds refunded via RBA Portal then enter these funds against Refund
  • Remove the error code
  • Select Post

Was there an amount to be refunded?

  • No, procedure ends here
  • Yes, process the refund through the RBA portal for GEP or email the RBA for BPAY

W012PH

ESS

Duplicate Essentials Document Number

This is a data integrity check to stop updating a customer record twice with the same details

Investigate ESS and the customer’s record.

This may not happen because Essentials Document numbers generate for each transaction.

No MFU creates but will be included in the PLS error notification reports sent to pls.retire.

Staff must check ESS and the customer’s record to make sure the customer’s record has not been updated twice.

W013PH

PLS

PLS loan repayment prior to last date on the HEAS Details (PLS) screen

Occurs when the Date of Payment from Essentials is before the last update. MFU is created

Investigate the customer’s record.

Determine, how much of the receipt will be applied to Principal, Interest and Cost.

Determine if refund and interest adjustment is applicable,

Once amounts confirmed update the loan.

  • Log into ESSentials (SAP R1P)
  • Key transaction ‘YFR_PLS’
  • Enter document number and financial year document was created
    Note: the fiscal year is usually the current financial year unless receipt was in previous financial year
  • Select Repost and Refund
  • Enter the amounts against Principal, Interest and Cost, and if any funds refunded via RBA Portal then enter these funds against Refund
  • Remove the error code
  • Select Post

Was there an amount to be refunded?

  • No, procedure ends here
  • Yes, process the refund through RBA portal for GEP or email the RBA for BPAY

W015PH

PLS

PLS loan - payment reversal received, investigate trust account

Negative payment amount is received. MFU is created

Decide if the positive receipt is:

  • Applied to Loan as Principal, Interest and Cost then:
    • decide how much of the negative receipt will be applied to Principal, Interest and Cost to re-instate the loan balance. Find the original positive receipt to work out how this was originally applied
    • once amounts confirmed update loan with the correction. Reverse the original receipt at the date of event the balance was reduced
    • log into ESSentials (SAP R1P)
    • key transaction ‘YFR_PLS’
    • enter document number and financial year the document was created for the negative receipt
      Note: the fiscal year is usually current financial year unless receipt was in previous financial year
    • select Repost and Refund
    • enter the negative amounts against Principal, Interest and Cost
    • remove the error code
    • select Post
    • select Save
  • The same amount of funds where applied to the refund component for the original receipt or the full amount of the positive receipt, or the positive receipt has not been applied to Principal, Loan or interest:
    • Log into ESSentials (SAP R1P)
    • key transaction ‘YFR_PLS’
    • enter document number and financial year document was created
      Note: usually current financial year unless receipt was in previous financial year
    • select Reset Table
    • remove the error code
    • update Comments to include the positive receipt which matches the negative receipt
    • update Status to Manually Confirmed
    • select Save

Was there an amount to be refunded?

  • No, procedure ends here
  • Yes, process the refund through the RBA portal for GEP or email the RBA for BPAY
