|                                   | rejection, the ABSTUDY Schooling  |
|                                   | A claim may reject automatically  |

|                                   | attending primary school and is   |
|                                   | under 14 years of age) or may     |
|                                   | need to be rejected manually. See |
|                                   | Codes for ABSTUDY cancellation,   |

|                                   |                                   |

|                                   | Select the started ABSTUDY new    |

|                                   | record.                           |
|                                   |                                   |
|                                   | Go to the ABSTUDY Task Selector   |

|                                   |                                   |
|                                   | Preselected mandatory screens     |
|                                   | are:                              |
|                                   |                                   |
|                                   | Accommodation (AC) - event date   |
|                                   | must align with the first day of  |
|                                   | entitlement in the grant period.  |
|                                   | There is no entitlement to Rent   |
|                                   | Assistance (RA) for ABSTUDY       |
|                                   | Schooling A. This date also       |
|                                   | cannot be coded prior to the      |
|                                   | earliest date recorded on the     |
|                                   | Address History (ADH) screen. The |
|                                   | ADH screen may need to be updated |
|                                   | before the AC screen can be       |
|                                   | coded. The ADH screen for the     |
|                                   | student must also reflect the     |
|                                   | applicant's ADH screen from the   |
|                                   | date of grant. When entering      |
|                                   | multiple events on ADH screen,    |
|                                   | only 1 AC screen event date can   |
|                                   | be input which must align with    |
|                                   | the date of grant. Note: if the   |
|                                   | AC screen is already coded with   |
|                                   | the following details from an     |
|                                   | earlier event date, do not code   |
|                                   | any further details. If not, code |
|                                   | the AC screen as follows:         |
|                                   |                                   |
|                                   | -   Address Type - HOM            |
|                                   | -   Event Date - must align with  |
|                                   |     the first date of entitlement |
|                                   |     in the grant period           |
|                                   | -   Home Ownership - Non          |
|                                   |     homeowner (NHO)               |
|                                   | -   Type of Rent - No rent paid   |
|                                   |     (NRP)                         |
|                                   | -   Sharer Status - Lives with    |
|                                   |     Parents (LWP)                 |
|                                   |                                   |
|                                   | Tax File Number Details (TFN) -   |
|                                   | If available it should be         |
|                                   | recorded, however a Tax File      |
|                                   | Number (TFN) is not required      |
|                                   | whilst the student is under 16    |
|                                   | years and exemption reason of     |
|                                   | under 16 years of age (U16) needs |
|                                   | to be coded. Note: for Schooling  |
|                                   | A claims lodged before the        |
|                                   | closing date but after the        |
|                                   | student has turned 16, a          |
|                                   | workaround is required to allow   |
|                                   | the U16 TFN exemption to be       |
|                                   | entered. If the student has no    |
|                                   | TFN, the date of receipt (DOR)    |
|                                   | will need to be input as 1/1 in   |
|                                   | the year for which assistance is  |
|                                   | being claimed, rather than the    |
|                                   | actual DOR the claim was lodged.  |
|                                   | This will allow the U16 exemption |
|                                   | to be coded, for example, student |
|                                   | turns 16 on 10/8/2017 but claim   |
|                                   | is lodged on 28/8/2017, the DOR   |
|                                   | will need to be input as 1/1/2017 |
|                                   |                                   |
|                                   | Payment Destination Details       |
|                                   | (PASB) - This is the payment      |
|                                   | details for the applicant         |
|                                   |                                   |
|                                   | Proof of Identity (POI) screen -  |
|                                   | Not required whilst the student   |
|                                   | is under 16 years. Note: this is  |
|                                   | not required for the applicant    |
|                                   | either                            |
|                                   |                                   |
|                                   | Links Summary (LS) screen - Check |
|                                   | for any existing or started (STA) |
|                                   | links. All Schooling A claims     |
|                                   | require an STA link to the        |
|                                   | applicant:                        |
|                                   |                                   |
|                                   | -   If a current STA link already |
|                                   |     exists linking the student to |
|                                   |     the applicant then no further |
|                                   |     action is required on the LS  |
|                                   |     screen                        |
|                                   | -   If a current incorrect STA    |
|                                   |     link already exists, this     |
|                                   |     will need to be end dated on  |
|                                   |     the date prior to the new     |
|                                   |     link starting on the LS       |
|                                   |     screen                        |
|                                   | -   If an incomplete STA link     |
|                                   |     exists then select it by      |
|                                   |     entering 'S' in the box next  |
|                                   |     to the link                   |
|                                   | -   If no STA link exists then    |
|                                   |     enter 'Y' in the box next to  |
|                                   |     the add new link to create a  |
|                                   |     new one                       |
|                                   | -   Selecting an incomplete link  |
|                                   |     or starting a new link will   |
|                                   |     direct the user to the Other  |
|                                   |     Customer Links (OCL) screen.  |
|                                   |     On this screen, code the      |
|                                   |     following:                    |
|                                   |     Link Type - STA               |
|                                   |     From SVC RSN - ABT            |
|                                   |     To - ABA                      |
|                                   |     Start Date - the date ABSTUDY |
|                                   |     Schooling A is granted from,  |
|                                   |     refer to Calculating the      |
|                                   |     start date for ABSTUDY        |
|                                   |                                   |
|                                   | Once the OCL screen has been      |
|                                   | coded and the applicant has been  |
|                                   | selected from the Link Person     |
|                                   | (LP) screen, the user will be     |
|                                   | redirected to the applicant's     |
|                                   | record to complete the STA link   |
|                                   | activity.                         |
