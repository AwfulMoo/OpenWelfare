Example of FTB reconciliation after death of customer and child

This table contains an example of FTB reconciliation processing when a
customer and child have died on the same day.

+-----------------------------------+-----------------------------------+
| Item                              | Example                           |
+-----------------------------------+-----------------------------------+
| 1                                 | Parent and child die on same day  |
|                                   |                                   |
|                                   | Jane and child die in a car       |
|                                   | accident on 1 January 2019.       |
|                                   | Jane's partner notifies Services  |
|                                   | Australia on 10 January 2019. The |
|                                   | child record is updated as        |
|                                   | deceased and a payment is         |
|                                   | generated on Jane's record.       |
|                                   |                                   |
|                                   | At reconciliation Manual          |
|                                   | Intervention is required as the   |
|                                   | system reads that Jane was not    |
|                                   | eligible during the 98 day        |
|                                   | bereavement period.               |
|                                   |                                   |
|                                   | Reconciliation is calculated      |
|                                   | manually using amount received as |
|                                   | per Payment Summary (PS) screen   |
|                                   | and including FTB supplement      |
|                                   | entitlement.                      |
|                                   |                                   |
|                                   | To request an FRMA manual         |
|                                   | adjustment:                       |
|                                   |                                   |
|                                   | -   Refer the customer along with |
|                                   |     the manually calculated       |
|                                   |     results to a Service Support  |
|                                   |     Officer (SSO), Technical Peer |
|                                   |     Support (TPS) or Access       |
|                                   |     Support Centre (ASC). They    |
|                                   |     will refer the issue to the   |
|                                   |     Escalation Referral Helpdesk  |
|                                   |     for a reconciliation manual   |
|                                   |     adjustment (FRMA) to correct  |
|                                   |     the result.                   |
+-----------------------------------+-----------------------------------+
