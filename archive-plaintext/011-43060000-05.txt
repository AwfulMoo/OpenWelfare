Medicare patient history

Table 1: details displayed on Medicare claims history for Simplified
Billing claims.

+-----------------------------------+-----------------------------------+
| No.                               | Simplified Billing details        |
|                                   | displayed                         |
+-----------------------------------+-----------------------------------+
| 1                                 | SBILL                             |
|                                   |                                   |
|                                   | Displayed in ITEM field.          |
+-----------------------------------+-----------------------------------+
| 2                                 | Minor ID (Minor ID or             |
|                                   | Registration number)              |
|                                   |                                   |
|                                   | Displayed in PROVIDER field.      |
+-----------------------------------+-----------------------------------+
| 3                                 | Type of claim                     |
|                                   |                                   |
|                                   | Type of claim:                    |
|                                   |                                   |
|                                   | -   F = fund claim                |
|                                   | -   A = billing agent             |
|                                   |                                   |
|                                   | Displayed in P (Payee Code)       |
|                                   | field.                            |
+-----------------------------------+-----------------------------------+
| 4                                 | Simplified Billing claims are     |
|                                   | allocated a payee code:           |
|                                   |                                   |
|                                   | -   8 = paid service              |
|                                   | -   F = rejected service          |
+-----------------------------------+-----------------------------------+

Manual claims

Table 2: status values that Simplified Billing manual claims may have.

+-----------------------------------+-----------------------------------+
| FCEI Status                       | Description                       |
+-----------------------------------+-----------------------------------+
| BACKGRND                          | Background assessing              |
|                                   |                                   |
|                                   | The claim is currently being      |
|                                   | assessed in background (FBGA).    |
+-----------------------------------+-----------------------------------+
| IN PEND                           | Claim pended                      |
|                                   |                                   |
|                                   | After initial assessment in       |
|                                   | background, the claim could not   |
|                                   | be automatically system paid. The |
|                                   | claim requires Service Officer    |
|                                   | intervention via pend processing  |
|                                   | (FAPI).                           |
+-----------------------------------+-----------------------------------+
| PAID-SD                           | Claim paid same day               |
|                                   |                                   |
|                                   | Claim has been completed          |
|                                   | successfully and reached payment  |
|                                   | status today.                     |
+-----------------------------------+-----------------------------------+
| PAID-LD                           | Claim paid latter day             |
|                                   |                                   |
|                                   | Claim has been completed          |
|                                   | successfully and reached payment  |
|                                   | status more than a day ago.       |
+-----------------------------------+-----------------------------------+

RMDS reports

Table 3: data validation (RMDS) reports available for Simplified
Billing.

+-----------------------------------+-----------------------------------+
| No.                               | Simplified billing details        |
|                                   | displayed                         |
+-----------------------------------+-----------------------------------+
| 1                                 | MCA NSB107 SIMPBILL EFT PYMT RPT  |
|                                   |                                   |
|                                   | -   EFT payment summary details   |
|                                   |     reported by state             |
|                                   | -   the report is produced Daily, |
|                                   |     if EFT payments exist         |
+-----------------------------------+-----------------------------------+
| 2                                 | MCA NSB301 SIMPBILL DLY PEND RPT  |
|                                   |                                   |
|                                   | -   lists ALL manual claims with  |
|                                   |     a current status of PEND,     |
|                                   |     reported by state             |
|                                   | -   the report is produced daily  |
|                                   |     and is used to identify       |
|                                   |     pended claims for FAPI        |
|                                   |     processing that are still     |
|                                   |     unprocessed by the state      |
|                                   |     Service Officers              |
+-----------------------------------+-----------------------------------+
| 3                                 | MCA EDI105 NPHEDI REPORT          |
|                                   |                                   |
|                                   | Checked daily to ensure           |
|                                   | Simplified Billing claims have    |
|                                   | been received successfully.       |
+-----------------------------------+-----------------------------------+
| 4                                 | MCA EDI101 SB EDI UNFORMATTED     |
|                                   | MCARE                             |
|                                   |                                   |
|                                   | Check the unformatted data if a   |
|                                   | claim failed transmission on EDI  |
|                                   | 105 Report.                       |
+-----------------------------------+-----------------------------------+
| 5                                 | MCA NSB 302 SB LDA AGE ANALYSIS   |
|                                   | RPT                               |
|                                   |                                   |
|                                   | -   Shows all claims that have    |
|                                   |     not been finalised/updated.   |
|                                   |     These claims have a status of |
|                                   |     ADJ ENTR, ADJ COMP, or ADJUST |
|                                   | -   The report is produced daily  |
+-----------------------------------+-----------------------------------+
| 6                                 | MCA NSB 304 SB LDA OVERPAYMENT    |
|                                   | REPORT                            |
|                                   |                                   |
|                                   | -   shows all claims that         |
|                                   |     resulted in an overpayment    |
|                                   | -   the report is produced once a |
|                                   |     week on Saturday evenings     |
+-----------------------------------+-----------------------------------+
| 7                                 | MCA NSB 305 SB LDA SUPPRESSED RPT |
|                                   |                                   |
|                                   | Shows all services suppressed in  |
|                                   | claims. This report is produced   |
|                                   | on the first day of each month.   |
+-----------------------------------+-----------------------------------+
| 8                                 | MCA NSB306 SB/ECLIPSE UNASSESSED  |
|                                   | CLM RPT                           |
|                                   |                                   |
|                                   | -   shows any claims which have   |
|                                   |     not been initially assessed   |
|                                   |     after 24 hours                |
|                                   | -   specifically claims which     |
|                                   |     have had a current FCEI       |
|                                   |     status of D/E COMP or         |
|                                   |     BACKGRND for more than one    |
|                                   |     day                           |
|                                   | -   this report is produced daily |
|                                   |     and is used to identify       |
|                                   |     problem claims                |
|                                   | -   this RMDS report is currently |
|                                   |     managed by the Simplified     |
|                                   |     Billing program area          |
+-----------------------------------+-----------------------------------+
