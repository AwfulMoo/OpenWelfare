| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Confirm participant or authorised |
|                                   | payment nominee identity + Read   |
|                                   | more ...                          |
|                                   |                                   |
|                                   | []                                |
|                                   |                                   |
|                                   | Confirm the participant's or      |
|                                   | authorised payment nominee's      |
|                                   | identification before proceeding  |
|                                   | with any card activation          |
|                                   | workflows. It is essential that   |
|                                   | this step is carried out          |
|                                   | thoroughly and carefully to       |
|                                   | reduce the risk of issuing funds  |
|                                   | to third parties in error.        |
|                                   |                                   |
|                                   | If issuing a temporary card to    |
|                                   | the participant’s payment         |
|                                   | nominee, request the nominee      |
|                                   | provide the recorded password     |
|                                   | stored on the Nominee             |
|                                   | Relationship Summary screen       |
|                                   | (NORS) to confirm their           |
|                                   | relationship in addition to       |
|                                   | confirming their identity.        |
|                                   |                                   |
|                                   | Issuing a temporary card to a     |
|                                   | third party in error can result   |
|                                   | in hardship for the participant   |
|                                   | and financial loss to Services    |
|                                   | Australia.                        |
|                                   |                                   |
|                                   | If in any doubt it is not the     |
|                                   | participant and/or they are being |
|                                   | taken advantage of, do not issue  |
|                                   | the temporary card and consult a  |
|                                   | social worker, or the Welfare     |
|                                   | Quarantining Programme Management |
|                                   | team for further assistance.      |
|                                   |                                   |
|                                   | Extra checks to ensure a          |
|                                   | temporary card should be issued   |
|                                   | to a CDC participant              |
|                                   |                                   |
|                                   | Check:                            |
|                                   |                                   |
|                                   | -   the person’s ID               |
|                                   | -   any other supporting ID       |
|                                   |     checks                        |
|                                   | -   any Display on Access (DOA)   |
|                                   |     DOCs or other DOCs, which     |
|                                   |     indicate that there are       |
|                                   |     financial vulnerabilities or  |
|                                   |     potential fraud occurrences   |
|                                   |     against the participant and   |
|                                   |     ensure the temporary card is  |
|                                   |     being issued to the correct   |
|                                   |     participant                   |
|                                   | -   if necessary make contact     |
|                                   |     with someone in the local     |
|                                   |     community to confirm that the |
|                                   |     participant has travelled     |
|                                   |     into town                     |
|                                   | -   where the card was last used  |
|                                   |                                   |
|                                   | Note: issuing a card to a third   |
|                                   | party will provide access to the  |
|                                   | participant's quarantined fund.   |
|                                   | This could result in hardship. If |
|                                   | a service officer has concerns    |
|                                   | someone is fraudulently trying to |
|                                   | access another participant’s      |
|                                   | account they should refer to the  |
|                                   | department’s fraud processes. See |
|                                   | Report Suspected Fraud and        |
|                                   | Corruption.                       |
|                                   |                                   |
|                                   | To issue a current CDC            |
|                                   | participant or payment nominee    |
|                                   | with a temporary card, go to Step |

has presented at a service centre or a remote servicing team.

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Run the Issue Temporary card      |
|                                   | workflow. + Read more ...         |
|                                   |                                   |
|                                   | In Customer First Workspace, go   |
|                                   | to Cashless Debit Card and select |
|                                   | Issue Temporary Card.             |
|                                   |                                   |
|                                   | An authentication screen will     |
|                                   | display:                          |
|                                   |                                   |
|                                   | -   If the participant is the CDC |
|                                   |     cardholder, the ‘customer     |
|                                   |     authentication’ screen will   |
|                                   |     display                       |
|                                   | -   If the payment nominee is the |
|                                   |     CDC cardholder, the ‘payment  |
|                                   |     nominee authentication’       |
|                                   |     screen will display.          |
|                                   |                                   |
|                                   | Who is the cardholder of the      |
|                                   | temporary card?                   |
|                                   |                                   |
|                                   | -   The participant, go to Step 2 |
|                                   | -   The participant's payment     |
|                                   |     nominee, go to Step 3         |
+-----------------------------------+-----------------------------------+
| 2                                 | Authenticating the participant +  |
|                                   | Read more ...                     |
|                                   |                                   |
|                                   | If the participant has a DOA DOC  |
|                                   | present, confirm all DOAs have    |
|                                   | been read and acknowledge any     |
|                                   | potential financial               |
|                                   | vulnerabilities by marking the    |
|                                   | tick box.                         |
|                                   |                                   |

|                                   |                                   |
