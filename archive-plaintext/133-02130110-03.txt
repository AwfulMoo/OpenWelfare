This page contains how Service Officers investigate flagged identifiers
and the actions required by Tier 3 staff to manage flagged identifiers.

How Service Officers action identifiers listed on the HW080 form

Table 1

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Check user's account status +     |
|                                   | Read more ...                     |
|                                   |                                   |
|                                   | Is the user's account in Known    |
|                                   | status?                           |
|                                   |                                   |
|                                   | -   Yes, go to Step 2             |

|                                   |     processed if the status is    |
|                                   |     Known. Make sure to finalise  |
|                                   |     the identity verification     |
|                                   |     process for their PRODA       |
|                                   |     account, before adding the    |
|                                   |     identifiers                   |
+-----------------------------------+-----------------------------------+
| 2                                 | Account status = Known + Read     |
|                                   | more ...                          |
|                                   |                                   |
|                                   | -   Select Manage services        |
|                                   | -   Select Healthcare providers   |
|                                   |     and administrators tile under |
|                                   |     Available services            |
|                                   | -   Select the Type of identifier |
|                                   |     from the dropdown list under  |
|                                   |     Add identifiers               |
|                                   | -   Enter the identifier number   |
|                                   |     into the Identifier field     |
|                                   | -   Select Search                 |
|                                   |                                   |
|                                   | Repeat for all identifiers        |
|                                   | provided on the HW080 form.       |
|                                   |                                   |
|                                   | Did all identifiers link?         |
|                                   |                                   |
|                                   | -   Yes, see the Verify identity  |
|                                   |     verification table, Step 4 in |
|                                   |     Processing the Manual         |
|                                   |     identity verification for     |
|                                   |     Provider Digital Access       |
|                                   |     (HW080) form                  |
|                                   | -   No, go to Step 3              |
+-----------------------------------+-----------------------------------+
| 3                                 | Check Manage names + Read more    |
|                                   | ...                               |
|                                   |                                   |
|                                   | Check if the Medicare record in a |
|                                   | name that exists in PRODA Manage  |
|                                   | names. Note: If the Medicare      |
|                                   | record is not in a name linked to |
|                                   | their PRODA account the user will |
|                                   | need to provide a change of name. |
|                                   |                                   |
|                                   | Is a change of name required?     |
|                                   |                                   |
|                                   | -   Yes, see email templates on   |
|                                   |     the Resources page of         |
|                                   |     Processing the Manual         |
|                                   |     identity verification for     |
|                                   |     Provider Digital Access       |
|                                   |     (HW080) form. Send email      |
|                                   |     'Change of name required to   |
|                                   |     enable linking to Medicare    |
|                                   |     records'                      |

+-----------------------------------+-----------------------------------+
| 4                                 | Escalate to Tier 2 + Read more    |
|                                   | ...                               |
|                                   |                                   |
|                                   | Escalate to Tier 2 with the       |
|                                   | following information:            |
|                                   |                                   |
|                                   | -   flagged identifier            |
|                                   | -   PRODA account username and/or |
|                                   |     Registration Authority (RA)   |
|                                   |     number                        |
|                                   | -   name of user                  |
|                                   | -   type of identifier (see Step  |
|                                   |     5 in the How Service Officers |
|                                   |     investigate and action        |
|                                   |     flagged identifiers table for |
|                                   |     a list)                       |
|                                   | -   identifier number             |
|                                   |                                   |
|                                   | If the Service Officer has        |
|                                   | conducted an investigation in the |
|                                   | source system and has identified  |
|                                   | why the identifier is not         |
|                                   | linking, give a description.      |
|                                   |                                   |
|                                   | See Help for staff Provider       |
|                                   | Digital Access (PRODA).           |

|                                   | The identifier is linked to the   |
|                                   | PRODA record and the relevant     |
|                                   | access is granted to the user.    |

How Tier 3 PRODA Support manage flagged identifiers

Table 3

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Access PRODA staff portal + Read  |
|                                   | more ...                          |
|                                   |                                   |
|                                   | PRODA Support staff are to        |
|                                   | investigate flagged identifiers   |
|                                   | on a daily basis.                 |
|                                   |                                   |
|                                   | Select Flagged Identifiers        |
|                                   | button.                           |
+-----------------------------------+-----------------------------------+
| 2                                 | Action flagged identifiers' +     |
|                                   | Read more ...                     |
|                                   |                                   |
|                                   | All 'flagged identifiers' need to |
|                                   | be actioned:                      |
|                                   |                                   |
|                                   | Reject identifiers that:          |
|                                   |                                   |
|                                   | do not belong to the user's       |
|                                   | account                           |
|                                   |                                   |
|                                   | -   include comments -            |
|                                   |     'identifier does not belong   |
|                                   |     to the user'                  |
|                                   |                                   |
|                                   | have been input incorrectly by    |
|                                   | the user                          |
|                                   |                                   |
|                                   | -   advise the user to try        |
|                                   |     entering the identifier again |
|                                   | -   include comments -            |
|                                   |     'identifier input             |
|                                   |     incorrectly'                  |
|                                   |                                   |
|                                   | Has the provider had a name       |
|                                   | change?                           |
|                                   |                                   |
|                                   | Yes:                              |
|                                   |                                   |
|                                   | -   Contact the provider and step |
|                                   |     them through the 'Change of   |
|                                   |     name' process and linking the |
|                                   |     identifier to the account     |
|                                   |     again                         |
|                                   | -   Reject identifier and include |
|                                   |     comments - 'Change of name    |
|                                   |     required                      |
|                                   |                                   |
|                                   | No, go to Step 3                  |
+-----------------------------------+-----------------------------------+
| 3                                 | Escalate + Read more ...          |
|                                   |                                   |
|                                   | Escalate to the relevant source   |
|                                   | system for the identifier type    |
|                                   | if:                               |
|                                   |                                   |
|                                   | -   name spelt incorrectly in the |
|                                   |     source system                 |
|                                   | -   no/incorrect date of birth    |
|                                   |     recorded in the source system |
|                                   | -   no/incorrect gender recorded  |
|                                   |     in the source system          |
