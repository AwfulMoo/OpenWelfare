Table 2

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Check customer's employment       |
|                                   | status + Read more ...            |
|                                   |                                   |
|                                   | If the customer is self-employed: |
|                                   |                                   |
|                                   | issue a:                          |
|                                   |                                   |
|                                   | -   Business Details (MOD F)      |
|                                   | -   Real Estate Details (MOD R)   |
|                                   |     (only if the business owns    |
|                                   |     real estate)                  |
|                                   |                                   |
|                                   | record all details of contact on  |
|                                   | a Note or DOC                     |
|                                   |                                   |
|                                   | When the customer returns the     |
|                                   | form/s, continue with this        |
|                                   | procedure.                        |
+-----------------------------------+-----------------------------------+
| 2                                 | For customers who are considered  |
|                                   | to be self-employed + Read more   |
|                                   | ...                               |
|                                   |                                   |
|                                   | [] If the customer's contract for |
|                                   | services is part of their         |
|                                   | business, treat the contract      |
|                                   | income as business income.        |
|                                   |                                   |
|                                   | If the customer's contract for    |
|                                   | services is a one-off occurrence  |
|                                   | and income is received as a lump  |
|                                   | sum, the Net contract income is   |
|                                   | treated as a non-remunerative     |
|                                   | lump sum, see Treatment of lump   |
|                                   | sums.                             |
|                                   |                                   |
|                                   | If the customer receives          |
|                                   | commission income, the net        |
|                                   | commissions are treated as        |
|                                   | business income.                  |
|                                   |                                   |
|                                   | Commission or contract payments   |
|                                   | received by self-employed         |
|                                   | customers may be reduced by the   |
|                                   | costs of obtaining the income as  |
|                                   | for businesses. To assess         |
|                                   | allowable deductions, see         |
|                                   | Business deductions.              |
|                                   |                                   |
|                                   | For commission workers considered |
|                                   | to be self-employed, go to Step   |
|                                   | 5.                                |
|                                   |                                   |
|                                   | For contract workers considered   |
|                                   | to be self-employed, go to Step   |
|                                   | 6.                                |
+-----------------------------------+-----------------------------------+
| 3                                 | For customers considered to be    |
|                                   | employees + Read more ...         |
|                                   |                                   |
|                                   | If contract income is received by |
|                                   | employees periodically, as a wage |
|                                   | or retainer, or as lump sum for a |
|                                   | period of 52 weeks or less, treat |
|                                   | the commission income as          |
|                                   | employment income:                |
|                                   |                                   |
|                                   | -   Record the income on the      |
|                                   |     Employment Income Paid        |
|                                   |     Details (EAPP) screen using   |
|                                   |     the Employment Type code      |
|                                   |     'Earnings from Employment’    |
|                                   |     (EAR)                         |
|                                   | -   Use the frequency ‘Income for |
|                                   |     one period’ (IOP) or ‘Long    |
|                                   |     Period’ (LOP) to record the   |
|                                   |     income using the date paid as |
|                                   |     the event date                |
|                                   | -   The income is assessed from   |
|                                   |     the EPSD of the period in     |
|                                   |     which the payment was paid,   |
|                                   |     for a number of days equal to |
|                                   |     the period that the payment   |
|                                   |     represents or for the length  |
|                                   |     of the entitlement period     |
|                                   |     (whichever is longer). This   |
|                                   |     is the known as the           |
|                                   |     assessment period             |
|                                   | -   See Recording and correcting  |
|                                   |     employment income details for |
|                                   |     assistance with coding the    |
|                                   |     EAPP screen                   |
|                                   |                                   |
|                                   | If contract income is received by |
|                                   | employees as a lump sum for a     |
|                                   | period greater than 52 weeks:     |
|                                   |                                   |
|                                   | Record the remunerative lump sum  |
|                                   | on the Employment Income Paid     |
|                                   | Details (EAPP) screen under a     |
|                                   | separate employer entry. For the  |
|                                   | new employer entry:               |
|                                   |                                   |
|                                   | -   Use the Employment Type code  |
|                                   |     'Earnings from Employment’    |
|                                   |     (EAR)                         |
|                                   | -   Enter the employer’s name and |
|                                   |     add ‘lump sum’ at the end,    |
|                                   |     for example ‘Services         |
|                                   |     Australia lump sum’. If       |
|                                   |     repeating for multiple lump   |
|                                   |     sums, add each lump sum using |
|                                   |     a new employer entry and add  |
|                                   |     the date the lump sum was     |
|                                   |     received to create a unique   |
|                                   |     employer name                 |
|                                   |                                   |
|                                   | Use the frequency code ‘Daily’    |
|                                   | (DAY) to record the income using  |
|                                   | the Entitlement Period Start Date |
|                                   | (EPSD) of the period in which the |
|                                   | payment was paid as the event     |
|                                   | date                              |
|                                   |                                   |
|                                   | -   Use a combination of the      |
|                                   |     Payment Calendar and the      |
|                                   |     customer’s Payment Delivery   |
|                                   |     (RAD) and Payment Summary     |
|                                   |     (PS) screens to determine the |
|                                   |     EPSD of the period if         |
|                                   |     required                      |
|                                   |                                   |
|                                   | The income is assessed from the   |
|                                   | EPSD of the period in which the   |
|                                   | payment was paid, for a number of |
|                                   | days equal to the period that the |
|                                   | payment represents. This is the   |
|                                   | known as the assessment period    |
|                                   |                                   |
|                                   | Divide the amount received by the |
|                                   | number of days in the period that |
|                                   | the payment represents. Record    |
|                                   | this figure in the Amount field   |
|                                   |                                   |
|                                   | -   For example, a lump sum of    |
|                                   |     $5,000 for a period of 1,000  |
|                                   |     days is a daily amount of $5  |
|                                   |                                   |
|                                   | The income must be ceased from    |
|                                   | the assessment period end date    |
|                                   | plus 1 day. Use the Perpetual     |
|                                   | Centrelink Calendar if required   |
|                                   | to determine the end date of the  |
|                                   | assessment                        |
|                                   |                                   |
|                                   | -   If this date has not yet      |
|                                   |     passed, record a manual       |
|                                   |     review in Customer            |
|                                   |     First/Customer Record         |
|                                   |     including details of the      |
|                                   |     income received, the          |
|                                   |     assessment period and the     |
|                                   |     date from which the           |
|                                   |     assessment is to be ceased.   |
|                                   |     See How to code and action a  |
|                                   |     manual review. Use review     |
|                                   |     reason ‘Earnings Review’      |
|                                   |     (INC) and add the keyword     |
|                                   |     OVER52WK                      |
|                                   | -   If this date has passed at    |
|                                   |     the time the income is being  |
|                                   |     recorded, record and cease    |
|                                   |     the income in the same        |
|                                   |     activity, no review is        |
|                                   |     required                      |
|                                   |                                   |
|                                   | To cease the income, record an    |
|                                   | amount of zero using the          |
|                                   | assessment period end date plus 1 |
|                                   | day as the event date. See        |
|                                   | Recording and correcting          |
|                                   | employment income details for     |
|                                   | assistance with coding the EAPP   |
|                                   | screen                            |
|                                   |                                   |
|                                   | The table on the Resources page   |
|                                   | explains how to assess commission |
|                                   | income for employees after 7      |
|                                   | December 2020.                    |
|                                   |                                   |
|                                   | No deductions can be made for     |
|                                   | expenses.                         |
|                                   |                                   |
|                                   | Note: when an employee, such as a |
|                                   | taxi driver, has takings          |
|                                   | including Goods and Services Tax  |
|                                   | (GST), the amount of GST is not   |
|                                   | considered income if the amount   |
|                                   | is remitted to the Australian     |
|                                   | Taxation Office (ATO). This is    |
|                                   | similar to the treatment of       |
|                                   | business income where GST is      |
|                                   | forwarded to the ATO.             |

|                                   | Where the income forms part of    |
|                                   | the person’s regular business     |
|                                   | income, assess the income as part |
|                                   | of the normal self-employment     |
|                                   | assessment. See Assessment of     |
|                                   | income and assets from business   |
|                                   | structures for Centrelink         |
|                                   | payments.                         |
|                                   |                                   |
|                                   | Where the income is a one-off     |
|                                   | occurrence record the income on   |
|                                   | the Other Income (OIN) screen and |
|                                   | assess for a period of 52 weeks.  |
|                                   |                                   |
|                                   | The date of event (DOV) is the    |
|                                   | date a change of circumstance or  |
|                                   | notifiable event actually         |
|                                   | occurred.                         |
|                                   |                                   |
|                                   | The Other Income Summary (OINS)   |
|                                   | screen will only assess a single  |
|                                   | amount of income from each        |
|                                   | particular employer or other      |
|                                   | income source. Each new date and  |
|                                   | amount recorded overrides the     |
|                                   | previously recorded amount. As    |
|                                   | lump sums can be apportioned up   |
|                                   | to 52 weeks, each time an         |
|                                   | individual lump sum is received,  |
|                                   | it must be added as a new         |
|                                   | employer with a unique Employer   |
|                                   | Name/Description to remain on     |
|                                   | OINS for the full relevant        |
|                                   | period.                           |
|                                   |                                   |
|                                   | Check to ensure any old income    |
|                                   | coded previously on OIN using an  |
|                                   | ongoing assessment code, for      |
|                                   | example, MTH, ANN, etc. have been |
|                                   | removed at the end of the         |
|                                   | assessment period.                |
|                                   |                                   |
|                                   | Code the following fields on the  |
|                                   | OIN screen:                       |
|                                   |                                   |
|                                   | -   Type: key 'ROC' (Royalty or   |
|                                   |     Commission)                   |
|                                   | -   Description: key the source   |
|                                   |     of the commission business or |
|                                   |     royalty income - free text.   |
|                                   |     If the customer receives lump |
|                                   |     sums from a number of         |
|                                   |     contracts with the same       |
|                                   |     company within a 12 month     |
|                                   |     period, a unique Description  |
|                                   |     needs to be used for each     |
|                                   |     instance of contract income.  |
|                                   |     For example, code as Porter's |
|                                   |     Homes 12/09/16, Porter's      |
|                                   |     Homes 26/09/16, Porter's      |
|                                   |     Homes 10/10/16, etc., using   |
|                                   |     the date of event of the      |
|                                   |     payment                       |
