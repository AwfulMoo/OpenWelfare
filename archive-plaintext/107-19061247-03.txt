|                                   |                                   |
|                                   | If a negative amount has been     |
|                                   | identified in the Debt            |

|                                   | This displays all debts that      |
|                                   | affect this period for the        |
|                                   | customer                          |
|                                   |                                   |
|                                   | check the:                        |
|                                   |                                   |
|                                   | -   DOC list                      |
|                                   | -   Modify/Assess Debt (OPMA)     |

|                                   | -   Account Payable (OPAOD)       |
|                                   |     screen to determine the       |
|                                   |     reason for each relevant debt |
+-----------------------------------+-----------------------------------+
| 2                                 | Note the Debt IDs + Read more ... |
|                                   |                                   |
|                                   | Note the relevant:                |
|                                   |                                   |
|                                   | -   Debt IDs                      |
|                                   | -   benefit type                  |
|                                   | -   period and                    |
|                                   | -   amount for each relevant debt |
|                                   |                                   |
|                                   | If:                               |
|                                   |                                   |
|                                   | -   the benefit type is 'ABY'     |
|                                   |     (ABSTUDY) and a third party   |

|                                   |     (MFU) tax activities for      |

|                                   |     Follow-up (MFU) tax           |
|                                   |     activities for third party    |
|                                   |     debts other than ABSTUDY      |
|                                   | -   the amount of the debt is     |
|                                   |     higher than the amount the    |
|                                   |     customer received, go to Step |
|                                   |     6                             |
|                                   | -   another case, go to Step 3    |

| 3                                 | Go to the Payment Summary (PS)    |
|                                   | screen + Read more ...            |

|                                   | In the following PS screen        |

|                                   |                                   |

|                                   |     Date (TXYTD) screen           |

|                                   |     Scroll through the list of    |
|                                   |     payments to locate the period |

|                                   |                                   |
|                                   | Was a payment received for the    |
|                                   | period noted from the TXYTD       |
|                                   | screen?                           |
|                                   |                                   |
|                                   | -   Yes, go to Step 5             |
|                                   | -   No, go to Step 4              |
+-----------------------------------+-----------------------------------+

|                                   | period noted from the TXYTD       |
|                                   | screen + Read more ...            |
|                                   |                                   |
|                                   | Go to the Debt List (OPDL) screen |
|                                   | and select the specific debt:     |
|                                   |                                   |
|                                   | -   go to the OPMA screen         |
|                                   | -   note the office code of the   |
|                                   |     team who raised the debt      |
|                                   |                                   |
|                                   | Go to Step 6.                     |
+-----------------------------------+-----------------------------------+
| 5                                 | Check benefit type + Read more    |
|                                   | ...                               |
|                                   |                                   |
|                                   | If the debt:                      |

|                                   | is recovered, go to Step 6        |
|                                   |                                   |

|                                   | is recovered and the Benefit Type |

|                                   | -   select the relevant debt from |

|                                   | -   go to the OPRH screen         |

|                                   |                                   |
|                                   | If the debt is not recovered and  |
|                                   | the Benefit Type is incorrect:    |
|                                   |                                   |
|                                   | -   go to OPCA                    |
|                                   | -   amend the Benefit Type        |
|                                   | -   complete the debt activity    |
|                                   |                                   |
|                                   | See Safeguarding customer privacy |
|                                   | when raising and recovering       |

|                                   | -   end of financial year         |
|                                   |     processing, the payment       |
|                                   |     summary will become available |
|                                   |     for request by the customer   |

|                                   |     correspondence channel        |

|                                   |                                   |

Determining what action to follow after debt has been reviewed

Table 2

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Multiple debts raised + Read more |
|                                   | ...                               |
|                                   |                                   |
|                                   | Have multiple debts been raised   |
|                                   | for the same period?              |
|                                   |                                   |
|                                   | -   Yes, see Processing Manual    |
|                                   |     Follow-up (MFU) tax           |
|                                   |     activities for multiple debts |
|                                   |     raised for the same period    |
|                                   | -   No, go to Step 2              |
+-----------------------------------+-----------------------------------+
| 2                                 | For single debts in the period +  |
|                                   | Read more ...                     |
|                                   |                                   |
|                                   | Go to the Debt List (OPDL) screen |
|                                   | and select the specific debt.     |
|                                   |                                   |
|                                   | Has the debt been raised for a    |
|                                   | Compensation reason (CMA, CMC or  |
|                                   | CMM codes)?                       |
|                                   |                                   |
|                                   | -   Yes, see Processing Manual    |
|                                   |     Follow-up (MFU) tax           |
|                                   |     activities for compensation   |
|                                   |     debts                         |
|                                   | -   No, go to Step 3              |
+-----------------------------------+-----------------------------------+
| 3                                 | Arrears paid for a debt period +  |
|                                   | Read more ...                     |
|                                   |                                   |
|                                   | Have arrears been paid for all or |
|                                   | part of a debt period?            |
|                                   |                                   |
|                                   | -   Yes, see Processing Manual    |
|                                   |     Follow-up (MFU) tax           |
|                                   |     activities for debts raised   |
|                                   |     for arrears payments          |
|                                   | -   No, go to Step 4              |
+-----------------------------------+-----------------------------------+
| 4                                 | Check each remaining debt + Read  |
|                                   | more ...                          |
|                                   |                                   |
|                                   | For each remaining debt, check    |
|                                   | the amount on the following       |
|                                   | screens:                          |
|                                   |                                   |
|                                   | -   Modify/Assess Debt (OPMA)     |
|                                   | -   Component Amount (OPCA)       |
|                                   |                                   |
|                                   | Compare these amounts against the |
|                                   | total payments received for the   |
|                                   | period of the debt.               |
|                                   |                                   |
|                                   | This can be done using the        |
|                                   | Payment Summary (PS) screen or PS |
|                                   | Tool.                             |
|                                   |                                   |
|                                   | If the income support payments    |
|                                   | changed during the debt period,   |
|                                   | calculate the total payments      |
|                                   | received for the new payment.     |
|                                   |                                   |
|                                   | Do not include any penalty        |
|                                   | reduction amounts. The customer   |
|                                   | has not received these funds.     |
|                                   | ADEX (Debt Explanation) does not  |
|                                   | recognise penalty reduction       |
|                                   | amounts and calculates based on   |
|                                   | the gross figures.                |
|                                   |                                   |
|                                   | Does the total amount of the debt |
|                                   | exceed the total amount of all    |
|                                   | relevant payments received during |
|                                   | the debt period?                  |
|                                   |                                   |
|                                   | -   Yes, go to Step 5             |
|                                   | -   No, go to Step 6              |
+-----------------------------------+-----------------------------------+
| 5                                 | Debt total exceeded + Read more   |
|                                   | ...                               |
|                                   |                                   |
|                                   | If the total of the debt exceeds  |
|                                   | the total of all relevant         |
|                                   | payments received during the debt |
|                                   | period:                           |
|                                   |                                   |
|                                   | the MFU tax activity needs to be  |
|                                   | investigated by the business area |
|                                   | responsible for the debt          |
|                                   |                                   |
|                                   | the details on the OPCA screen    |
|                                   | need to be corrected:             |
|                                   |                                   |
|                                   | -   complete the TAXMFU Referral  |
|                                   |     Fast Note                     |
|                                   | -   attach to the MFU tax         |
|                                   |     activity                      |
|                                   | -   categorise the MFU tax        |
|                                   |     activity to the business area |
|                                   |     responsible for the raising   |
|                                   |     of the debt                   |
|                                   |                                   |
|                                   | When the debt component details   |
|                                   | are corrected, the system will    |
|                                   | recalculate the payment summary.  |
|                                   |                                   |
|                                   | Note: see Safeguarding customer   |
|                                   | privacy when raising and          |
|                                   | recovering debts, before issuing  |
|                                   | any debt notices. Take the        |
|                                   | appropriate actions to protect    |
|                                   | the customer's privacy.           |
|                                   |                                   |
|                                   | If the MFU tax activity was due   |
|                                   | to:                               |
|                                   |                                   |
|                                   | -   end of financial year         |
|                                   |     processing, the payment       |
|                                   |     summary will become available |
|                                   |     for request by the customer   |
|                                   | -   retrospective changes, the    |
|                                   |     amended payment summary will  |
|                                   |     automatically issue via the   |
|                                   |     customers preferred           |
|                                   |     correspondence channel        |
|                                   |                                   |
|                                   | Procedure ends here.              |
+-----------------------------------+-----------------------------------+
| 6                                 | Possible cause of error + Read    |
|                                   | more ...                          |
|                                   |                                   |
|                                   | If the amount on the:             |
|                                   |                                   |
|                                   | -   OPMA screen does not exceed   |
|                                   |     the total payments received,  |
|                                   |     but                           |
|                                   | -   the total of debts for the    |
|                                   |     customer on TXYTD Debt        |
|                                   |     Summary exceeds Payments on   |
|                                   |     the Calculation Summary       |
|                                   | -   the error is most likely an   |
|                                   |     incorrect breakdown of        |
|                                   |     components on the OPCA screen |
|                                   |                                   |
|                                   | See Processing Manual Follow-up   |
|                                   | (MFU) tax activities for          |
|                                   | incorrect component amount debts. |
