This page contains the procedures for processing items payable to
service providers that require Prior Financial Authorisation (PARS
Approval).

On this Page:

Items that require a PARS Approval

Check for Prior Financial Authorisation (PARS Approval)

Table 1: This table describes how Service Officers can check if Prior
Financial Authorisation has been granted by DVA using the Prior Approval
& Referral System (PARS) and processing.

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Check the service for previous    |
|                                   | payment + Read more ...           |
|                                   |                                   |
|                                   | Check the veteran's history       |
|                                   | 'VHOI' to ensure there is no      |
|                                   | previous payment of the same item |
|                                   | within a few days of the date of  |
|                                   | service.                          |
|                                   |                                   |
|                                   | Has the service been previously   |
|                                   | paid?                             |
|                                   |                                   |
|                                   | -   Yes, reject the item with RSN |
|                                   |     code S160                     |
|                                   | -   No, go to the PARS 'VEPI'     |
|                                   |     screen, go to Step 2          |
+-----------------------------------+-----------------------------------+
| 2                                 | Check for a Prior Approval Record |
|                                   | (PARS) + Read more ...            |
|                                   |                                   |
|                                   | Press [PF2] to display Veterans   |
|                                   | PARS screen (VEPI)                |
|                                   |                                   |
|                                   | Check for a relevant Prior        |
|                                   | Approval Record, ensuring the:    |
|                                   |                                   |
|                                   | -   date of service matches the   |
|                                   |     Request Date or is within the |
|                                   |     Effective Dates               |
|                                   | -   item in the approval matches  |
|                                   |     the item claimed              |
|                                   | -   service provider listed in    |
|                                   |     the approval matches the      |
|                                   |     provider making the claim or  |
|                                   |     is a provider practicing at   |
|                                   |     the same location             |
|                                   |                                   |
|                                   | Note: PARS for MT items must have |
|                                   | an Effective and End date period  |
|                                   | under 3 months unless a date is   |
|                                   | specified in the comments, or the |
|                                   | request date or effective date    |
|                                   | matches the Date of Service.      |
+-----------------------------------+-----------------------------------+
| 3                                 | Check low cost items for previous |
|                                   | payment within 3 month period +   |
|                                   | Read more ...                     |
|                                   |                                   |
|                                   | Low cost items PDT, HRT, GDX and  |
|                                   | 24hr Ambulatory blood pressure    |
|                                   | monitors have PARS approvals with |
|                                   | an effective period of 3 months.  |
|                                   |                                   |
|                                   | Have the services been previously |
|                                   | paid when the date of service     |
|                                   | falls within the 3 month period?  |
|                                   |                                   |
|                                   | -   Yes, reject with RSN S 160    |
|                                   | -   No, pay with PI 18            |
|                                   |                                   |
|                                   | Note: if service is not Low cost  |
|                                   | item PDT, HRT, GDX or 24hr        |
|                                   | Ambulatory Blood Pressure item,   |
|                                   | go to Step 4.                     |
+-----------------------------------+-----------------------------------+
| 4                                 | When there is a matching PARS +   |
|                                   | Read more ...                     |
|                                   |                                   |
|                                   | When there is a matching          |
|                                   | Approval, key 'S' in the SEL      |
|                                   | field, press [Enter] and check    |
|                                   | the:                              |
|                                   |                                   |
|                                   | -   date of services matches the  |
|                                   |     date specified in the         |
|                                   |     comments                      |
|                                   | -   amount approved and amount    |
|                                   |     charged are the same          |
|                                   | -   decision on the PARS is       |
|                                   |     approved                      |
|                                   |                                   |
|                                   | Press PF2 to return to claim.     |
+-----------------------------------+-----------------------------------+
| 5                                 | Action PARS screen + Read more    |
|                                   | ...                               |
|                                   |                                   |
|                                   | When the PARS 'VEPI' screen       |
|                                   | shows:                            |
|                                   |                                   |
|                                   | -   no approval matching the      |
|                                   |     claim, reject the item using  |
|                                   |     RSN code S279                 |
|                                   | -   a matching record that has    |
|                                   |     been rejected (Decision field |
|                                   |     is R), reject the item using  |
|                                   |     RSN code S289                 |
|                                   | -   a matching record that has    |
|                                   |     been approved (Decision field |
|                                   |     is A), pay the item with PI18 |
+-----------------------------------+-----------------------------------+

Items where charge is different to amount approved

Process items where the charge is different to the amount approved

Table 2: This table describes how to process a service where the Prior
Financial Authorisation (PARS) amount charged does not match the amount
approved.

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Charge is more than PARS          |
|                                   | approval - manual claim + Read    |
|                                   | more ...                          |
|                                   |                                   |
|                                   | When the charge is more than PARS |
|                                   | approval (for example, approval   |
|                                   | is for $100 and the provider is   |
|                                   | claiming $125):                   |
|                                   |                                   |
|                                   | -   insert a new line and key:    |
|                                   | -   the item number               |
|                                   | -   the amount approved           |
|                                   | -   processing indicator 18       |
|                                   | -   press [F9] to accept the      |
|                                   |     amount claimed                |
|                                   | -   insert a second line and key  |
|                                   |     the:                          |
|                                   | -   item number                   |
|                                   | -   residual charge               |
|                                   | -   reject the second line with   |
|                                   |     reason code 283               |
+-----------------------------------+-----------------------------------+
| 2                                 | Charge is more than PARS          |
|                                   | approval - online claim + Read    |
|                                   | more ...                          |
|                                   |                                   |
|                                   | When the charge is more than PARS |
|                                   | approval (for example, approval   |
|                                   | is for $100 and the provider is   |
|                                   | claiming $125):                   |
|                                   |                                   |
|                                   | -   Key Processing Indicator (PI) |
|                                   |     58 and enter the approved     |
|                                   |     amount in the FEE field       |
|                                   | -   Key RSN 288 and press ENTER   |
|                                   |     to pay the claim              |
+-----------------------------------+-----------------------------------+
| 3                                 | Charge is less than PARS          |
|                                   | approval - manual or online       |
|                                   | claim + Read more ...             |
|                                   |                                   |
|                                   | When the charge is less than PARS |
|                                   | approval (for example, approval   |
|                                   | is for $795 and the provider is   |
|                                   | claiming $325):                   |
|                                   |                                   |
|                                   | -   key processing indicator 18   |
|                                   | -   press [F9] to accept the      |
|                                   |     amount claimed on the invoice |
+-----------------------------------+-----------------------------------+
