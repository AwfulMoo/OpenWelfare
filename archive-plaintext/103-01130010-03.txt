Actioning a request to update existing RDS deduction

Table 1

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Is the customer currently paying  |
|                                   | rent to a State Housing Authority |
|                                   | (SHA) via the Rent Deduction      |
|                                   | Scheme? + Read more ...           |
|                                   |                                   |
|                                   | Go to the Payment Instruction     |
|                                   | Summary (PINS) screen to check if |
|                                   | the customer has an existing Rent |
|                                   | Deduction Scheme (RDS) deduction  |
|                                   | in place.                         |
|                                   |                                   |
|                                   | -   Yes, go to Step 2             |
|                                   | -   No, conduct Initial           |
|                                   |     Assessment interview.         |
|                                   |     Procedure ends here           |
+-----------------------------------+-----------------------------------+
| 2                                 | Discuss with customer priority    |
|                                   | needs and rent payment options to |
|                                   | their SHA + Read more ...         |
|                                   |                                   |
|                                   | Discuss with the customer how     |
|                                   | they would like to continue to    |
|                                   | pay their rent and all of their   |
|                                   | other priority needs.             |
|                                   |                                   |
|                                   | Advise the customer rent can be   |
|                                   | paid from their income managed    |
|                                   | funds, their discretionary funds  |
|                                   | or a combination of both if       |
|                                   | required.                         |
|                                   |                                   |
|                                   | There may be situations when the  |
|                                   | customer has insufficient         |
|                                   | discretionary or income managed   |
|                                   | funds alone to cover their full   |
|                                   | rent payment. In these cases it's |
|                                   | up to the customer to determine   |
|                                   | how all of their priority needs   |
|                                   | will be met and what proportion   |
|                                   | of their rent should be paid from |
|                                   | Income Management and what        |
|                                   | proportion should be paid from    |
|                                   | their discretionary funds.        |
|                                   |                                   |
|                                   | It is essential that customer's   |
|                                   | continue to pay their full rent   |
|                                   | to their SHA.                     |
|                                   |                                   |
|                                   | -   Customer wishes to pay rent   |
|                                   |     in full from discretionary    |
|                                   |     funds, go to Step 3           |
|                                   | -   Customer wishes to pay rent   |
|                                   |     in full from Income           |
|                                   |     Management funds, or a        |
|                                   |     combination of both Income    |
|                                   |     Management and discretionary  |
|                                   |     funds, go to Step 4           |
+-----------------------------------+-----------------------------------+
| 3                                 | Customer wishes to pay full       |
|                                   | amount of rent payment from their |
|                                   | discretionary funds + Read more   |
|                                   | ...                               |
|                                   |                                   |
|                                   | No action is required, leave the  |
|                                   | RDS deduction in place.           |
|                                   |                                   |
|                                   | Ensure the customer understands   |
|                                   | that the deduction will continue  |
|                                   | to come out of their              |
|                                   | discretionary funds which means   |
|                                   | less discretionary funds will be  |
|                                   | paid to them.                     |
|                                   |                                   |
|                                   | Record contact and customer       |
|                                   | decision to leave the allocation  |
|                                   | as a Rent Deduction Scheme        |
|                                   | deduction in a DOC. Procedure     |
|                                   | ends here.                        |
+-----------------------------------+-----------------------------------+
| 4                                 | Customer wishes to pay rent       |
|                                   | payment in full from their Income |
|                                   | Management funds, or via a        |
|                                   | combination of income managed and |
|                                   | discretionary funds + Read more   |
|                                   | ...                               |
|                                   |                                   |
|                                   | Discuss with the customer which   |
|                                   | funds they would like to use to   |
|                                   | pay their rent payment.           |
|                                   |                                   |
|                                   | Rent paid from Income Management  |
|                                   | funds only                        |
|                                   |                                   |
|                                   | Review existing expense           |
|                                   | allocations. It may be necessary  |
|                                   | to modify existing expenses to    |
|                                   | ensure sufficient funds are       |
|                                   | available to enable rent to be    |
|                                   | paid in full from Income          |
|                                   | Management funds.                 |
|                                   |                                   |
|                                   | Any adjustments to priority needs |
|                                   | expenses should be made in        |
|                                   | consultation with the customer.   |
|                                   |                                   |
|                                   | Rent from Income Management and   |
|                                   | discretionary funds               |
|                                   |                                   |
|                                   | Discuss with the customer what    |
|                                   | amount will be paid from Income   |
|                                   | Management funds and what amount  |
|                                   | will be paid from discretionary   |
|                                   | funds:                            |
|                                   |                                   |
|                                   | -   Income Management - review    |
|                                   |     existing Income Management    |
|                                   |     expenses and unallocated      |
|                                   |     funds. The customer may       |
|                                   |     choose to modify existing     |
|                                   |     Income Management expenses to |
|                                   |     enable rent to be paid        |
|                                   | -   Discretionary funds - ensure  |
|                                   |     sufficient funds will be      |
|                                   |     available to cover the        |
|                                   |     modified RDS deduction and    |
|                                   |     any other Centrepay or        |
|                                   |     voluntary deductions          |
+-----------------------------------+-----------------------------------+

Coding new expenses for SHA rent payment

| Step                              | Action                            |

| 1                                 | Coding new expense for SHA rent   |
|                                   | payment from Income Management    |
|                                   | funds, or a combination of income |
|                                   | managed and discretionary funds + |
|                                   | Read more ...                     |
|                                   |                                   |
|                                   | -   Navigate to the customer's    |
|                                   |     Expense Management summary    |
|                                   | -   Under New expenses select Add |
|                                   |     regular expense from the drop |
|                                   |     down selection                |
|                                   | -   Select Start                  |
+-----------------------------------+-----------------------------------+
| 2                                 | Organisation Search screen + Read |
|                                   | more ...                          |
|                                   |                                   |
|                                   | Note: Housing Tasmania is an      |
|                                   | uncontracted TPO, and a record    |
|                                   | may not be available. In this     |
|                                   | case, a New TPO required: field   |
|                                   | will appear with a tick defaulted |
|                                   | in the box. See Coding Income     |
|                                   | Management expenses to complete   |
|                                   | the process for payments being    |
|                                   | made to Housing Tasmania.         |
|                                   |                                   |
|                                   | -   TPO CRN: enter the State or   |
|                                   |     Territory Housing Authority   |
|                                   |     details                       |
|                                   | -   Select Find                   |
|                                   | -   Expense Category: field,      |
|                                   |     select the appropriate option |
|                                   | -   Select Continue               |
|                                   | -   Select the correct SHA from   |
|                                   |     the Expense Management TPO    |
|                                   |     list. Note: the SHA may be    |
|                                   |     listed a number of times so   |
|                                   |     ensure the record with the    |
|                                   |     correct service reason is     |
|                                   |     selected                      |
+-----------------------------------+-----------------------------------+
| 3                                 | Coding a regular expense via      |
|                                   | direct credit to a contracted     |
|                                   | TPO + Read more ...               |
|                                   |                                   |
|                                   | When a SHA with an Income         |
|                                   | Management contract has been      |
|                                   | selected from the TPO Inputs      |
|                                   | section, complete the Expense     |
|                                   | Inputs screen:                    |
|                                   |                                   |
|                                   | -   TPO Name: this will default   |
|                                   |     with information entered in   |
|                                   |     the Expense Management TPO    |
|                                   |     List                          |
|                                   | -   TPO CRN: this will default    |
|                                   |     with information entered in   |
|                                   |     the Expense Management TPO    |
|                                   |     List                          |
|                                   | -   Account Reference: input      |
|                                   |     customer details              |
|                                   | -   Expense Category: this will   |
|                                   |     default with information      |
|                                   |     entered in the Expense        |
|                                   |     Management TPO List           |
|                                   | -   Amount: enter amount to be    |
|                                   |     paid                          |
|                                   | -   Target amount: enter amount   |
|                                   |     only where the customer is    |
|                                   |     required to pay a fixed total |
|                                   |     to the TPO                    |
|                                   | -   Start Date: this will default |
|                                   |     to the customer's next EPED   |
|                                   |     or EPED +1 depending on the   |
|                                   |     customer's payment type.      |
|                                   |     Income Management holiday     |
|                                   |     processing coding is to apply |
|                                   |     where the default date is     |
|                                   |     incorrect due to holiday      |
|                                   |     processing                    |
|                                   | -   End Date: Leave blank unless  |
|                                   |     the customer nominates a      |
|                                   |     specific end date. Note: if   |
|                                   |     the payment is for a consumer |
|                                   |     lease, recommend to the       |
|                                   |     customer they include an end  |
|                                   |     date to avoid overpayment     |
|                                   | -   Frequency: Select 1 Weekly, 2 |
|                                   |     Weekly or 4 Weekly            |
|                                   |                                   |
|                                   | Note: ensure that the IM          |
|                                   | deduction is equal to either the  |
|                                   | full rent amount required or the  |
|                                   | total amount that will be paid    |
|                                   | from Income Management with the   |
|                                   | remainder to then be paid using   |
|                                   | the Rent Deduction Scheme.        |
|                                   |                                   |
|                                   | Select Continue.                  |
+-----------------------------------+-----------------------------------+
| 4                                 | Finalise activity + Read more ... |
|                                   |                                   |
|                                   | When expense allocation is        |
|                                   | completed finalise assessment.    |
|                                   |                                   |
|                                   | -   Review the Expense Management |
|                                   |     Summary screen to ensure      |
|                                   |     details are correct           |
|                                   | -   Select Finalise               |

| 5                                 | End the RDS deduction + Read more |
|                                   | ...                               |
|                                   |                                   |
|                                   | From the Payment Instruction      |
|                                   | Direct Deduction (PIDD) screen on |
|                                   | the customers record key in:      |
|                                   |                                   |
|                                   | -   New deduction amount of       |
|                                   |     '0.00' (or a single zero) in  |
|                                   |     the Instruction Amount P.F.   |
|                                   |     $: field                      |
|                                   | -   Date of effect in the Start   |
|                                   |     Date: field. (The RDS date of |
|                                   |     effect can be in the future)  |
|                                   | -   Reason for termination in the |
|                                   |     Change Reason: field: (A      |
|                                   |     change reason code must be    |
|                                   |     given for a termination; use  |

|                                   |     fields                        |

|                                   |     use Action of 'D' (delete) to |
|                                   |     terminate an instruction)     |
|                                   | -   Press [Enter]. The Payment    |
|                                   |     Instruction Summary (PINS)    |
|                                   |     screen will display again     |
