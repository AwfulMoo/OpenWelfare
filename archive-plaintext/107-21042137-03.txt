Mail Procedure

Table 1

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Mail duties + Read more ...       |
|                                   |                                   |
|                                   | Delivery of mail is between 10:00 |
|                                   | am and 11:00 am daily. 2 staff    |
|                                   | are rostered to:                  |
|                                   |                                   |
|                                   | -   distribute mail for other     |
|                                   |     teams via the Internal Mail   |
|                                   |     System                        |
|                                   | -   sight, count and verify all   |
|                                   |     cheques and remittances (both |
|                                   |     staff)                        |
|                                   | -   scan remittance advices and   |
|                                   |     email to Medicare Recovery    |
|                                   |     for the banking officer       |
|                                   | -   complete and sign the mail    |
|                                   |     register (both staff)         |
|                                   | -   physically hand all cheques   |
|                                   |     received and mail registered  |
|                                   |     to the Banking Officer        |
|                                   | -   refer all other mail to the   |
|                                   |     Operational Team Leader       |
|                                   |                                   |
|                                   | Is there any return to sender     |
|                                   | (RTS), mail?                      |
|                                   |                                   |
|                                   | -   Yes, go to Step 2             |
|                                   | -   No, procedure ends here       |
+-----------------------------------+-----------------------------------+
| 2                                 | Return to sender mail + Read more |
|                                   | ...                               |
|                                   |                                   |
|                                   | Mail may be returned to sender    |
|                                   | (RTS) when:                       |
|                                   |                                   |
|                                   | -   the address is incorrect      |
|                                   | -   the customer is no longer at  |
|                                   |     the address or with the       |
|                                   |     practice                      |
|                                   | -   the debtor is incorrect       |
|                                   | -   there is not enough detail in |
|                                   |     the address                   |
|                                   |                                   |
|                                   | Before returning to sender,       |
|                                   | forward all mail to the correct   |
|                                   | location/address. Use different   |
|                                   | options, for example:             |
|                                   |                                   |
|                                   | -   The Consumer Directory        |
|                                   |     Maintenance System (CDMS),    |
|                                   |     for updated records           |
|                                   | -   Contact the customer by phone |
|                                   | -   Search using Google, White    |
|                                   |     and Yellow pages              |
|                                   | -   Check information on Services |
|                                   |     Australia program records     |
|                                   | -   Request release of            |
|                                   |     information from Centrelink   |
|                                   |     records. See Requests for     |
|                                   |     Centrelink Information table  |
|                                   |     in Active recovery of         |
|                                   |     outstanding Medicare debt.    |
|                                   |                                   |
|                                   | Update the address and phone in   |
|                                   | the customer's record. In SAP,    |
|                                   | run Transaction FBL5N - Customer  |
|                                   | Line Item Display.                |
|                                   |                                   |
|                                   | Can the debt now be followed up?  |
|                                   |                                   |
|                                   | Yes, re-send the reminder letters |
|                                   | and relevant documents:           |
|                                   |                                   |
|                                   | -   verify the original invoice   |
|                                   |     issued and any attachments    |
|                                   | -   change the B line date in the |
|                                   |     SAP record                    |
|                                   | -   change the Dunning date to    |
|                                   |     restart the dunning process.  |
|                                   |     See Table 2 in Dunning for    |
|                                   |     Medicare debt                 |
|                                   |                                   |
|                                   | No:                               |
|                                   |                                   |
|                                   | -   Update the Assignment field   |
|                                   |     to 'NCD' (No Contact Details) |
|                                   | -   Review each NCD customer      |
|                                   |     record in 3 months            |
|                                   | -   If all searches are complete  |
|                                   |     and contact or address        |
|                                   |     details cannot be located,    |
|                                   |     consider a write off. Discuss |
|                                   |     this option with the delegate |

Medicare mainframe - duplicate statements

Table 3

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Logging onto mainframe + Read     |
|                                   | more ...                          |
|                                   |                                   |
|                                   | Note: where a step requires an    |
|                                   | [Enter] keypress, use the number  |
|                                   | keypad only.                      |
|                                   |                                   |
|                                   | Open mainframe - Medicare Branch  |
|                                   | Session A - NO SSO                |
|                                   |                                   |
|                                   | -   Key 'CESN'. Press [Enter].    |
|                                   |     The Login screen shows        |
|                                   | -   Operator: key P Number -      |
|                                   |     P00000 - without the Prefix   |
|                                   |     (P)                           |
|                                   | -   Password: key the 6 digit     |
|                                   |     code generated by the RSA     |
|                                   |     token. [Enter]                |
|                                   | -   Key 'OPOI'. Press [Enter].    |
|                                   |     The mainframe action page     |
|                                   |     shows                         |
|                                   | -   Tab down to the line AMAG SHQ |
|                                   |     FINANCE. Key 'S' and [Enter]  |
|                                   | -   Basic commands can now be     |
|                                   |     entered. See Resources for a  |
|                                   |     list of Mainframe search      |
|                                   |     commands                      |
+-----------------------------------+-----------------------------------+
| 2                                 | Printing a duplicate statement in |
|                                   | mainframe + Read more ...         |
|                                   |                                   |
|                                   | Open mainframe - Medicare Branch  |
|                                   | Session A - NO SSO:               |
|                                   |                                   |
|                                   | Key 'DCEI', the Provider Number   |
|                                   | ('000000XX') and Claim ID         |
|                                   | ('X0000@') to find the date of    |
|                                   | lodgement (DOL)                   |
|                                   |                                   |
|                                   | DOL is in format DDMMYY           |
|                                   |                                   |
|                                   | Replace DCEI,ProviderNumber on    |
|                                   | the Command Line with             |
|                                   |                                   |
|                                   | -   'DPSI'                        |
|                                   | -   Provider Number ('000000XX')  |
|                                   | -   Claim ID ('X0000@')           |
|                                   | -   Date of Lodgement             |
|                                   |     ('dol=DDMMYY')                |
|                                   | -   for example in                |
|                                   |     'DPSI,P                       |
|                                   | roviderNumber,ClaimID,Dol=DDMMYY' |
|                                   |     format                        |
|                                   |                                   |
|                                   | Select the Print icon at the top  |
|                                   | left of the screen                |
|                                   |                                   |
|                                   | Check if the provider or patient  |
|                                   | has received payment              |
|                                   |                                   |
|                                   | Key 'BRNA', [Enter]. (Use the     |
|                                   | [Enter] key on the number keypad  |
|                                   | only)                             |
|                                   |                                   |
|                                   | Payee Type:                       |
|                                   |                                   |
|                                   | -   'N' - Patient claims          |
|                                   | -   'P' - Provider                |
|                                   | -   'G' - Third Party             |
|                                   | -   'F' - Health Funds (SB)       |
|                                   | -   'B' - Billings Agents (SB -   |
|                                   |     usually Hospitals)            |
|                                   |                                   |
|                                   | For a:                            |
|                                   |                                   |
|                                   | Patient Claim:                    |
|                                   |                                   |
|                                   | -   Payee ID line = Medicare      |
|                                   |     Number                        |
|                                   | -   Key in a date range           |
|                                   | -   [Enter] (on the numeric       |
|                                   |     keypad)                       |
|                                   |                                   |
|                                   | Provider Claim:                   |
|                                   |                                   |
|                                   | -   Payee ID line = Provider      |
|                                   |     Number                        |
|                                   | -   Key in a date range           |
|                                   | -   [Enter] (on the numeric       |
|                                   |     keypad)                       |
