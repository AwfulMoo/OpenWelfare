Known customer (organisation)

Determine if the applicant is a known customer (organisation)

Table 1: This table describes the process Service Officers follow when
determining whether the applicant is a known customer (organisation).

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Is the organisation being         |
|                                   | registered as a pharmacy? + Read  |
|                                   | more ...                          |
|                                   |                                   |
|                                   | -   Yes, go to Step 2             |
|                                   | -   No, process as any other      |
|                                   |     organisation registration     |
+-----------------------------------+-----------------------------------+
| 2                                 | Public Key Infrastructure (PKI)   |
|                                   | registration number or approval   |
|                                   | number provided + Read more ...   |
|                                   |                                   |
|                                   | Has the applicant provided a PKI  |
|                                   | registration number or a pharmacy |
|                                   | approval number on the            |
|                                   | application form for the          |
|                                   | pharmacy?                         |
|                                   |                                   |
|                                   | -   Yes, this is sufficient       |
|                                   |     evidence of business identity |
|                                   |     for the purposes of the HI    |
|                                   |     Service                       |
|                                   | -   No, the pharmacy cannot show  |
|                                   |     that they have previously     |
|                                   |     provided evidence of business |
|                                   |     identity. The pharmacy must   |
|                                   |     provide acceptable business   |
|                                   |     identity for HI Service       |
|                                   |     registration                  |
+-----------------------------------+-----------------------------------+

Known customer (individual)

Determine if the applicant is a known customer (individual)

Table 2: This table describes how to determine whether the applicant is
a known customer (individual).

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Check Responsible Officer (RO)    |
|                                   | status + Read more ...            |
|                                   |                                   |
|                                   | To check that the applicant, RO   |
|                                   | is:                               |
|                                   |                                   |
|                                   | -   the Duly Authorised Officer   |
|                                   |     (DAO) for the pharmacy, go to |
|                                   |     Step 2                        |
|                                   | -   the Approved Person for the   |
|                                   |     pharmacy approval number, go  |
|                                   |     to Step 3                     |
+-----------------------------------+-----------------------------------+
| 2                                 | Check applicant is DAO for        |
|                                   | pharmacy + Read more ...          |
|                                   |                                   |
|                                   | To check that the applicant RO is |
|                                   | the Duly Authorised Officer (DAO) |
|                                   | for the pharmacy location         |
|                                   | certificate:                      |
|                                   |                                   |
|                                   | -   log on to the Certificate     |
|                                   |     Information Management System |
|                                   |     (CIMS)                        |
|                                   | -   select the Search             |
|                                   |     Registrations menu option     |
|                                   |     (from the left hand side menu |
|                                   |     selections)                   |
|                                   | -   key the Registration          |
|                                   |     Authority (RA) number         |
|                                   |                                   |
|                                   | The DAO name is displayed.        |
|                                   |                                   |
|                                   | Go to Step 5.                     |
+-----------------------------------+-----------------------------------+
| 3                                 | Check applicant RO is the         |
|                                   | Approved Person + Read more ...   |
|                                   |                                   |
|                                   | To use Mainframe to check that    |
|                                   | the applicant RO is the Approved  |
|                                   | Person for the pharmacy approval  |
|                                   | number supplied in the            |
|                                   | application:                      |
|                                   |                                   |
|                                   | -   log onto Mainframe (if        |
|                                   |     required, request access via  |
|                                   |     your Team Coach or Manager)   |
|                                   | -   enter control line ‘rpba’     |
|                                   | -   enter the Pharmacy Approval   |
|                                   |     number                        |
|                                   | -   press [F9] to confirm the     |
|                                   |     Approved Person(s) and attach |
|                                   |     a screen dump to application  |
|                                   |                                   |
|                                   | Note: if a business name or an    |
|                                   | individual’s name on the Approved |
|                                   | Person(s) screen is different     |
|                                   | from the applicant, press F9      |
|                                   | again to go to the Authorised     |
|                                   | Person(s) screen. If the          |
|                                   | applicant appears on this screen  |
|                                   | with either (Director) or         |
|                                   | (Trustee) after their name, they  |
|                                   | are an approved person for the    |
|                                   | pharmacy approval number. Attach  |
|                                   | a screen dump of this to the      |
|                                   | application.                      |
+-----------------------------------+-----------------------------------+
| 4                                 | Check status of applicant + Read  |
|                                   | more ...                          |
|                                   |                                   |
|                                   | Is the applicant the Duly         |
|                                   | Authorised Officer (DAO) for the  |
|                                   | pharmacy PKI certificate, the     |
|                                   | Approved Person or an authorised  |
|                                   | person who is Trustee/Director    |
|                                   | for the pharmacy?                 |
|                                   |                                   |
|                                   | -   Yes, the RO application can   |
|                                   |     be processed under the known  |
|                                   |     customer model                |
|                                   | -   No, the applicant is required |
|                                   |     to supply full EOI for the    |
|                                   |     purposes of HI Service        |
|                                   |     registration                  |
|                                   |                                   |
|                                   | Attach either the CIMS screen     |
|                                   | shot or screen dump from the      |
|                                   | Mainframe search to the           |
|                                   | application form for filing and   |
|                                   | archiving purposes.               |
+-----------------------------------+-----------------------------------+
