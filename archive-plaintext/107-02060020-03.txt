+-----------------------------------+-----------------------------------+
| OPDL                              | Debt List screen                  |
|                                   |                                   |
|                                   | Shows debts that belong to a      |
|                                   | customer and includes all debts   |
|                                   | that are:                         |
|                                   |                                   |
|                                   | -   undetermined                  |
|                                   | -   determined                    |
|                                   | -   protected, and                |
|                                   | -   finalised                     |
|                                   |                                   |
|                                   | If there is more than 1 debt      |
|                                   | record, a list will display in    |
|                                   | the following order:              |
|                                   |                                   |
|                                   | -   Undetermined and AoS          |
|                                   |     protected. If there is more   |
|                                   |     than 1 undetermined and/or    |
|                                   |     AoS protected debt, the list  |
|                                   |     will be from the earliest to  |
|                                   |     the latest date registered    |
|                                   | -   Determined. If there is more  |
|                                   |     than 1 determined debt, it    |
|                                   |     will list from the earliest   |
|                                   |     to the latest date raised     |
|                                   | -   Finalised. If there is more   |
|                                   |     than 1 finalised debt, it     |
|                                   |     will list from the earliest   |
|                                   |     to the latest date finalised  |
|                                   |                                   |
|                                   | Use the Select View field to      |
|                                   | view:                             |
|                                   |                                   |
|                                   | -   the order in which payments   |
|                                   |     will be applied if there is   |
|                                   |     more than 1 debt outstanding, |
|                                   |     key 'ILY'                     |
|                                   | -   parked debts, key ‘PRK’       |
|                                   | -   deleted debts, key ‘DEL’      |
+-----------------------------------+-----------------------------------+
| OPGI                              | General Information screen        |
|                                   |                                   |
|                                   | This screen is a display screen   |
|                                   | that:                             |
|                                   |                                   |
|                                   | -   provides a quick reference or |
|                                   |     summary of each debt          |
|                                   | -   is used as a look-up point    |
|                                   |     for general enquiries         |
|                                   | -   provides details if a debt is |
|                                   |     Interest Charge eligible      |
|                                   | -   cannot be changed, except for |
|                                   |     the Debt ID field. Note:      |
|                                   |     customers can have more than  |
|                                   |     1 Debt ID                     |
|                                   |                                   |
|                                   | The OPGI is a debt specific       |
|                                   | screen, so there will be an OPGI  |
|                                   | screen for each debt.             |

|                                   | menus within DMIS. For each       |

|                                   |                                   |
|                                   | The OPMM screen has a modifiable  |
|                                   | Debt ID field. This field is      |
|                                   | another option for identifying    |
|                                   | the record request.               |
|                                   |                                   |
|                                   | If a Customer Reference Number    |
|                                   | (CRN) has more than 1 debt        |
|                                   | selected, the user will stay on   |
|                                   | the OPMM screen.                  |
|                                   |                                   |
|                                   | To view multiple debts:           |
|                                   |                                   |
|                                   | -   select the required debt (if  |
|                                   |     the Debt ID is known), or     |
|                                   | -   navigate to the Debt List     |
|                                   |     (OPDL) screen                 |
|                                   |                                   |
|                                   | There are 4 menus, the first      |
|                                   | being OPMM. Use:                  |
|                                   |                                   |
|                                   | -   OPDM to update debt specific  |
|                                   |     details                       |
|                                   | -   OPCM to update                |
|                                   |     customer-specific (rather     |
|                                   |     than debt-specific) details   |
|                                   |     for recovery arrangements     |
+-----------------------------------+-----------------------------------+

|                                   | administrative charge amount (if  |

|                                   | applicable date (both future and  |

|                                   | either the administrative charge  |

|                                   | -   there has been no application |
|                                   |     of Administrative charges     |

|                                   | Judgment Interest and Costs.      |
|                                   | Fields to change and show any     |
|                                   | judgement interest amount added   |

+-----------------------------------+-----------------------------------+
| OPAD                              | Add Debt screen                   |
|                                   |                                   |
|                                   | Use this screen to add new debt   |
|                                   | records manually where an         |
|                                   | undetermined debt record is not   |
|                                   | auto-generated in:                |
|                                   |                                   |
|                                   | -   the host payment              |
|                                   | -   compensation                  |
|                                   | -   Assurance of Support (AoS),   |
|                                   |     or                            |
|                                   | -   review system                 |
|                                   |                                   |
|                                   | The system allocates a Debt ID    |
|                                   | when it creates the debt record.  |
|                                   | The minimum details needed to     |
|                                   | create a debt record are:         |
|                                   |                                   |
|                                   | -   the authority                 |
|                                   | -   date of event, and            |
|                                   | -   debt benefit type             |
|                                   |                                   |
|                                   | The OPAD screen is blank and      |
|                                   | unprotected when first accessed.  |
|                                   | After a debt is created via this  |
|                                   | screen, use the Modify/Assess     |
|                                   | Debt (OPMA) screen to make        |
|                                   | changes.                          |

|                                   |                                   |
|                                   | Record the Recovery Fee details   |
|                                   | on this screen and include:       |
|                                   |                                   |
|                                   | -   the amount of the debt to     |
|                                   |     which to apply the Recovery   |
|                                   |     Fee                           |
|                                   | -   calculated Recovery Fee       |
|                                   |     amount                        |
|                                   | -   decision to apply Recovery    |
|                                   |     Fee (yes/no)                  |
|                                   | -   Reason field for applying or  |
|                                   |     not applying the Recovery Fee |
|                                   | -   date the first request letter |
|                                   |     is sent                       |
|                                   | -   date the first request letter |
|                                   |     is returned                   |
|                                   | -   date of second contact        |
|                                   |                                   |
|                                   | The Recovery Fee also shows on    |
|                                   | the below screens:                |
|                                   |                                   |
|                                   | -   Modify/Assess Debt (OPMA)     |
|                                   | -   Add Debt (OPAD)               |
|                                   | -   General Information (OPGI),   |
|                                   |     and                           |
|                                   | -   Account Payable (OPAOD)       |
|                                   |                                   |
|                                   | If using the OPDA screen to       |
|                                   | change the debt amount,           |
|                                   | recalculate the Recovery Fee      |
|                                   | also. This is so that it is still |
|                                   | 10% of the proportion of the      |
|                                   | debt.                             |
|                                   |                                   |
|                                   | Note: only apply Recovery Fees    |
|                                   | when the debt is due to the       |

|                                   |                                   |
|                                   | Used to finish any DMIS           |
|                                   | assessment that contains a        |
|                                   | determination. It also shows      |
|                                   | errors or warnings resulting from |
|                                   | the data input. The acceptable    |
|                                   | input values are:                 |
|                                   |                                   |
|                                   | -   'Y', activity finished        |
|                                   | -   'Q', activity finished and    |
|                                   |     sent for Quality Online (QOL) |
|                                   |     checking                      |
|                                   | -   'N', activity not finished,   |

|                                   |     assessment                    |
|                                   |                                   |
|                                   | When making a determination the   |
|                                   | activity will flow to the OPAF    |
|                                   | screen. The user can go to a      |
|                                   | different screen without          |
|                                   | finalising the activity:          |
|                                   |                                   |
|                                   | -   key the screen name in the    |
|                                   |     NXT field                     |
|                                   | -   press [F16] (Shift+F4)        |
+-----------------------------------+-----------------------------------+
| OPAOD                             | Account Payable screen            |
|                                   |                                   |
|                                   | Used to issue Account Payable, or |
|                                   | send a manual advice of debt.     |
|                                   |                                   |
|                                   | Once a debt is determined,        |
|                                   | details on this screen cannot be  |
|                                   | changed.                          |
|                                   |                                   |
|                                   | To:                               |
|                                   |                                   |
|                                   | -   reissue the advice, see       |
|                                   |     Viewing or reissuing a letter |
|                                   |     or electronic message         |
|                                   | -   suppress the Account Payable, |
|                                   |     go to the Key Dates (OPKD)    |
|                                   |     screen                        |
+-----------------------------------+-----------------------------------+
| OPCA                              | Component Amount screen           |
|                                   |                                   |
|                                   | This screen auto-populates and is |
|                                   | used to breakdown components of a |
|                                   | debt by financial years, mainly   |
|                                   | for tax adjustment purposes. No   |
|                                   | further action is needed if the   |
|                                   | system estimate is accepted as    |
|                                   | the calculated amount.            |
|                                   |                                   |
|                                   | Manually update OPCA if a debt    |
|                                   | is:                               |
|                                   |                                   |
|                                   | -   manually added, or            |
|                                   | -   the calculated amount differs |
|                                   |     from the system estimate      |
|                                   |                                   |
|                                   | The dates used in the financial   |
|                                   | year breakdown are based on the   |
|                                   | whole entitlement periods. For    |
|                                   | example, the entitlements that    |
|                                   | would have been paid in the given |
|                                   | financial year. The sum of        |
|                                   | component amounts must be equal   |
|                                   | to the debt total amount.         |
|                                   |                                   |
|                                   | General debt raising information  |
|                                   | has:                              |
|                                   |                                   |
|                                   | -   a list of common errors for   |
|                                   |     tax processing in relation to |
|                                   |     debts, and                    |
|                                   | -   action(s) for how avoid these |
|                                   |     problems and stop the         |
|                                   |     creation of Manual Follow-ups |
|                                   |     (MFUs)                        |
+-----------------------------------+-----------------------------------+

|                                   | the life of a debt. A key event   |
|                                   | is:                               |
|                                   |                                   |
|                                   | raising of a debt                 |
|                                   |                                   |
|                                   | sending any 1 of the following:   |
|                                   |                                   |
|                                   | -   a manual Account Payable      |
|                                   | -   Account Payable/advice of     |
|                                   |     debt                          |
|                                   | -   a debt adjust letter for      |
|                                   |     Family Tax Benefit (FTB) and  |
|                                   |     Child Care Benefit - formal   |
|                                   |     (CCF) re-reconciliation debts |
|                                   | -   a reminder letter             |
|                                   | -   a first reminder letter       |
|                                   |     (debts raised prior to 10     |
|                                   |     December 2007)                |
|                                   | -   a second reminder letter      |
|                                   |     (debts raised prior to 10     |
|                                   |     December 2007)                |
|                                   | -   a cash arrangement            |
|                                   |     confirmation letter           |
|                                   | -   a payment not received letter |
|                                   | -   a tax garnishee warning       |
|                                   |     letter                        |
|                                   |                                   |
|                                   | customer has an end date for an   |
|                                   | address and Services Australia    |
|                                   | sends a:                          |
|                                   |                                   |
|                                   | -   first or second reminder, or  |
|                                   | -   a tax garnishee warning       |
|                                   |     letter                        |
|                                   |                                   |
|                                   | referring a customer to the       |
|                                   | Australian Taxation Office (ATO)  |
|                                   | for possible garnishee action     |
|                                   |                                   |
|                                   | a debt is due for auto write-off  |
|                                   |                                   |
|                                   | a debt is due for possible        |
|                                   | referral to External Collection   |
|                                   | Agent (ECA)                       |
|                                   |                                   |
|                                   | a debt is excluded for referral   |
|                                   | to ECA                            |
|                                   |                                   |
|                                   | a debt is due for auto waiver     |
|                                   |                                   |
|                                   | reversing a waiver                |
|                                   |                                   |
|                                   | The system auto-records and shows |
|                                   | key events as they occur. Staff   |
|                                   | can record the date a manual      |
|                                   | advice of debt issues, if no      |
|                                   | previous advice has been sent.    |
|                                   |                                   |
|                                   | Note: process CCS debts in        |
|                                   | Process Direct, not in DMIS.      |

| OPMA                              | Modify/Assess Debt screen         |
|                                   |                                   |
|                                   | This screen relates to a specific |
|                                   | debt and is used to change        |
|                                   | details on an existing debt       |
|                                   | record. Updates can be made to    |
|                                   | undetermined and most determined  |
|                                   | debt records. Staff cannot update |
|                                   | the OPMA screen if:               |
|                                   |                                   |
|                                   | a temporary write-off is in place |
|                                   |                                   |
|                                   | the debt is finalised             |
|                                   |                                   |
|                                   | a protected Assurance of Support  |
|                                   | (AoS) debt                        |
|                                   |                                   |
|                                   | a compensation debt               |
|                                   |                                   |
|                                   | the customer in the header is not |
|                                   | the primary debtor                |
|                                   |                                   |
|                                   | an activity with provisional      |
|                                   | updates to the debt is on the     |
|                                   | Activity List (AL) screen         |
|                                   |                                   |
|                                   | there is a reconciliation debt.   |
|                                   | Staff can update the below        |
|                                   | fields:                           |
|                                   |                                   |
|                                   | -   Date of Notification          |
|                                   | -   Date of Event, and            |
|                                   | -   Was debt solely due to system |
|                                   |     or clerical error?            |
+-----------------------------------+-----------------------------------+
| OPML                              | Multiple Liability screen         |
|                                   |                                   |
|                                   | This screen relates to a specific |
|                                   | debt. Users can:                  |
|                                   |                                   |
|                                   | -   record                        |
|                                   | -   view, and                     |
|                                   | -   change the level of liability |
|                                   |     for customers who are jointly |
|                                   |     and severally responsible for |
|                                   |     the same debt                 |
|                                   |                                   |
|                                   | Jointly and severally means that  |
|                                   | each person listed against this   |
|                                   | debt has joint responsibility for |
|                                   | the amount. That means that if 1  |
|                                   | member pays $1 towards the debt,  |
|                                   | then the joint debt liability     |
|                                   | reduces by $1.                    |
|                                   |                                   |
|                                   | For example, the joint liability  |
|                                   | will reduce to $900 if 3 people   |
|                                   | are jointly liable for a $1,000   |
|                                   | debt and 1 person pays $100. It   |
|                                   | is possible for only 1 person to  |
|                                   | pay off all of a liability. This  |
|                                   | can be without the contribution   |
|                                   | of the other liable parties.      |
|                                   |                                   |
|                                   | Use the third party withholdings  |
|                                   | function on the Withholding       |
|                                   | Instruction (OPWH) screen when:   |
|                                   |                                   |
|                                   | -   a debt is on a duplicate      |
|                                   |     record                        |
|                                   | -   for third party withholdings  |
+-----------------------------------+-----------------------------------+

|                                   |                                   |
|                                   | The Statement of Account:         |
|                                   |                                   |
|                                   | is an auto-Batch Laser Advice     |

|                                   |     for all debts                 |
|                                   |                                   |
|                                   | Generate statements on request    |
|                                   | from a customer. Update the       |
|                                   | period of the statement manually. |
|                                   | Use the OPSD screen to:           |
|                                   |                                   |
|                                   | -   manually produce a statement  |
|                                   | -   display the dates covered of  |
|                                   |     statements previously sent    |
|                                   |                                   |

|                                   | longer auto-issued. Staff can     |
|                                   | issue:                            |
|                                   |                                   |
|                                   | -   manual statements from the    |
|                                   |     OPSD screen for any period in |
|                                   |     which the customer had        |
|                                   |     outstanding accounts          |
|                                   | -   a Statement of Account for    |
|                                   |     any period up to 5 years (60  |
|                                   |     calendar months)              |

+-----------------------------------+-----------------------------------+
| OPDA                              | Debt Amount screen                |
|                                   |                                   |
|                                   | Used to record any recalculations |
|                                   | of the debt amount.               |
|                                   | Recalculations of the debt amount |
|                                   | cannot be recorded if the debt    |
|                                   | is:                               |
|                                   |                                   |
|                                   | -   undetermined                  |
|                                   | -   determined written-off        |
|                                   | -   parked                        |
|                                   | -   held                          |
|                                   | -   finalised                     |
|                                   | -   a family assistance           |
|                                   |     reconciliation debt           |
|                                   | -   a Family Tax Benefit (FTB)    |
|                                   |     non-lodger debt, or           |
|                                   | -   the Activity List (AL) screen |
|                                   |     has an activity with          |
|                                   |     provisional updates to the    |
|                                   |     debt                          |
|                                   |                                   |
|                                   | There is no limit to the number   |
|                                   | of recalculations of the debt     |
|                                   | amount. The system keeps a record |
|                                   | of each recalculation. Multiple   |
|                                   | recalculations show in the Date   |
|                                   | Changed field. They range from    |
|                                   | the most recent date to the       |
|                                   | earliest date. The below fields   |
|                                   | are protected:                    |
|                                   |                                   |
|                                   | -   Estimated Amount              |
|                                   | -   Date raised                   |
|                                   | -   Original Calculated Amount    |
|                                   | -   Date registered               |
|                                   |                                   |
|                                   | The original calculated amount    |
|                                   | will not be shown as a            |
|                                   | recalculated amount. The Reason   |
|                                   | Changed and Date Changed fields   |
|                                   | become protected once coded. No   |
|                                   | changes will be possible for that |
|                                   | occurrence.                       |
|                                   |                                   |
|                                   | The Activity Management Reference |
|                                   | 'AMR' shows for all adjustments.  |
|                                   | This is the number of the         |
|                                   | activity that resulted in the     |
|                                   | debt adjustment.                  |
|                                   |                                   |
|                                   | The Source AMR field shows for    |
|                                   | FTB non-lodger and reconciliation |
|                                   | debts where an adjustment occurs. |
|                                   | This field shows the source ID of |
|                                   | the most recent re-reconciliation |
|                                   | result that caused the change in  |
|                                   | the debt amount.                  |

|                                   |                                   |

|                                   | debt and current payments.        |
|                                   |                                   |
|                                   | The instruction will:             |
|                                   |                                   |
|                                   | -   pass to the system that       |
|                                   |     withholds the funds           |
|                                   | -   calculate according to the    |
|                                   |     rules used for determining    |
|                                   |     withholdings within the host  |
|                                   |     system. Then pass back to     |
|                                   |     DMIS, unless a non-standard   |
|                                   |     rate exists                   |
|                                   | -   withhold all amounts paid,    |
|                                   |     except $1, if the amount paid |
|                                   |     is less than the specified    |
|                                   |     amount                        |
|                                   | -   set a future review date of 3 |
|                                   |     months when non-standard      |
|                                   | -   apply to the payment record   |
|                                   |     after the first Entitlement   |
|                                   |     Period End Day (EPED)         |
|                                   |     following the start date.     |
|                                   |     Then will deduct from the     |
|                                   |     payday that follows           |
|                                   | -   set 1 withholding instruction |
|                                   |     at a time per payment type    |
|                                   | -   be current until the Cease    |
|                                   |     Date field is not empty       |
|                                   | -   make withholdings against all |
|                                   |     of a customer's debts as per  |
|                                   |     current hierarchy rules, if   |
|                                   |     the withholding Debt ID is    |
|                                   |     empty                         |
|                                   |                                   |
|                                   | The instruction can apply to      |
|                                   | third parties with their          |
|                                   | permission. This person may have  |
|                                   | no liability for the debt but has |
|                                   | agreed to have withholdings made. |
|                                   |                                   |
|                                   | Use OPWH to:                      |
|                                   |                                   |
|                                   | -   record a new instruction to   |
|                                   |     the payment systems to make   |
|                                   |     withholdings from the         |
|                                   |     payments to repay a debt      |
|                                   | -   cease any current withholding |
|                                   |     instruction                   |
|                                   | -   display instructions for      |
|                                   |     withholdings to recover       |
|                                   |     debt(s) (both current and     |
|                                   |     ceased)                       |
|                                   | -   modify or delete data that is |
|                                   |     still provisional (that is,   |
|                                   |     not determined). If the       |
|                                   |     activity is not currently     |
|                                   |     open, select it from the      |
|                                   |     Activity List (AL) screen     |
|                                   | -   extend the future review date |
|                                   |     for a non-standard            |
|                                   |     withholding instruction       |
|                                   |                                   |
|                                   | Access the OPWH screen from the   |
|                                   | OPAL screen if:                   |
|                                   |                                   |
|                                   | -   there is no auto-withholding  |
|                                   |     instruction                   |
|                                   | -   the customer is receiving     |
|                                   |     more than 1 payment and a     |
|                                   |     higher rate of recovery is    |
|                                   |     needed                        |
|                                   | -   ceasing a previous            |
|                                   |     withholding instruction       |
|                                   | -   a third party withholding     |
|                                   |     instruction is awaiting       |
|                                   |     coding                        |
|                                   | -   a non-standard rate           |
|                                   |     withholding instruction is    |
|                                   |     awaiting coding or review     |
