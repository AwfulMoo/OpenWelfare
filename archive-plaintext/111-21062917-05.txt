Scenarios

Table 1 This table describes different scenarios and what warning
messages appear with multiple users. In the following scenarios Service
Officers will be referred to as User 1, User 2 and User 3

+-----------------------------------+-----------------------------------+
| Scenario                          | Outcome                           |
+-----------------------------------+-----------------------------------+
| Two users access the same CRN     |                                   |
|                                   |                                   |
| -   User 1 is actioning a         |                                   |
|     transaction in a customer’s   |                                   |
|     record                        |                                   |
| -   User 2 accesses the same      |                                   |
|     customer record               |                                   |
| -   User 1 does not receive any   |                                   |
|     messaging                     |                                   |
| -   User 2 will receive a warning |                                   |
|     message 'User: <USERID> is    |                                   |
|     currently in transaction      |                                   |
|     ID<XXXXXXXXXX>'               |                                   |
| -   User 2 is able to make        |                                   |
|     updates to the record as has  |                                   |
|     only accessed the same record |                                   |
|     and not the same transaction  |                                   |
| -   All actions such as Assess,   |                                   |
|     Add, Edit and Delete are      |                                   |
|     enabled                       |                                   |
+-----------------------------------+-----------------------------------+
| Multiple users are accessing the  |                                   |
| same CRN with one user accessing  |                                   |
| one transaction                   |                                   |
|                                   |                                   |
| -   User 1 accesses CRN via       |                                   |
|     Customer Summary tile         |                                   |
| -   User 2 access Transaction ID  |                                   |
|     001 via Inbox tile            |                                   |
| -   User 3 accesses CRN via       |                                   |
|     Customer Summary tile         |                                   |
| -   User 1 does not receive any   |                                   |
|     messaging                     |                                   |
| -   User 2 will receive a warning |                                   |
|     message 'User: <USERID> is    |                                   |
|     currently in this CRN'        |                                   |
| -   User 2 is able to continue to |                                   |
|     action the transaction they   |                                   |
|     are working on                |                                   |
| -   User 3 will be presented with |                                   |
|     the messages that User 1      |                                   |
|     'User: <USERID> is currently  |                                   |
|     in this CRN' and User 2       |                                   |
|     'User: <USERID> is currently  |                                   |
|     in transaction ID<001>'       |                                   |
| -   User 3 is able to make        |                                   |
|     updates to the customer       |                                   |
|     record except to Transaction  |                                   |
|     ID 001                        |                                   |
+-----------------------------------+-----------------------------------+
| Multiple users access different   |                                   |
| transactions on the same CRN      |                                   |
|                                   |                                   |
| -   User 1 accesses Transaction   |                                   |
|     ID 001 via Inbox tile         |                                   |
| -   User 2 access Transaction ID  |                                   |
|     002 via Inbox tile            |                                   |
| -   User 3 accesses Transaction   |                                   |
|     ID 001 via Inbox tile         |                                   |
| -   User 1 does not receive any   |                                   |
|     messaging                     |                                   |
| -   User 2 receives a message     |                                   |
|     'User: <USERID> is currently  |                                   |
|     in transaction ID<001>'       |                                   |
| -   User 2 is able to make        |                                   |
|     updates within their          |                                   |
|     transaction ID 002            |                                   |
| -   User 3 will be presented with |                                   |
|     the messages that             |                                   |
|     User 1 - 'User: <USERID> is   |                                   |
|     currently in this             |                                   |
|     transaction'                  |                                   |
|     User 2 - 'User: <USERID> is   |                                   |
|     currently in transaction      |                                   |
|     ID<002>' and 'User: <USERID>  |                                   |
|     is currently in this CRN'     |                                   |
| -   User 3 is unable to make any  |                                   |
|     updates in Transaction ID 001 |                                   |
| -   All actions such as Assess,   |                                   |
|     Add, Edit and Delete are      |                                   |
|     disabled                      |                                   |
+-----------------------------------+-----------------------------------+
| Multiple users are accessing the  |                                   |
| same transaction with one user    |                                   |
| accessing the same CRN            |                                   |
|                                   |                                   |
| -   User 1 accesses Transaction   |                                   |
|     ID 001 via Inbox tile         |                                   |
| -   User 2 accesses CRN via       |                                   |
|     Customer Summary tile         |                                   |
| -   User 3 accesses the same      |                                   |
|     transaction ID 001 via the    |                                   |
|     Inbox tile and removes        |                                   |
|     transaction lock from User 1  |                                   |
| -   User 1 does not receive any   |                                   |
|     messaging                     |                                   |
| -   User 2 receives a message     |                                   |
|     that User 1 is 'User:         |                                   |
|     <USERID> is currently in      |                                   |
|     transaction ID<001>'          |                                   |
| -   User 2 is able to make        |                                   |
|     updates to the customer’s     |                                   |
|     record as they are not in the |                                   |
|     same transaction as User 1    |                                   |
| -   User 3 will receive a message |                                   |
|     that User 1 'User: <USERID>   |                                   |
|     is currently in this          |                                   |
|     transaction' and User 2 is    |                                   |
|     'User: <USERID> is currently  |                                   |
|     in this CRN'                  |                                   |
| -   User 3 will not be able to    |                                   |
|     make any updates to the       |                                   |
|     transaction, as User 1 is     |                                   |
|     already in the transaction    |                                   |
| -   User 3 is able to remove the  |                                   |
|     transaction lock from User 1  |                                   |
|     only in urgent circumstances  |                                   |
+-----------------------------------+-----------------------------------+
