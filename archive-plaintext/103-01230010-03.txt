| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Issue card workflow + Read more   |
|                                   | ...                               |
|                                   |                                   |
|                                   | [                                 |
|                                   | \\INTERNAL.DEPT.LOCAL\Shared\NAT\ |
|                                   | SERDELEXCEL\WORKPRODIMP\Operation |
|                                   | Blueprint Migration\RDT Release   |
|                                   | Icons\32w\icon-face-to-face.png]  |
|                                   |                                   |
|                                   | The Service Officer is presented  |
|                                   | with an authentication screen. If |
|                                   | the customer has Display on       |
|                                   | Access (DOA) DOCs the Service     |
|                                   | Officer must confirm that they    |
|                                   | have read the DOC.                |
|                                   |                                   |
|                                   | The Service Officer must also     |
|                                   | confirm if the customer's record  |
|                                   | is password protected before they |
|                                   | can continue.                     |
|                                   |                                   |
|                                   | Once confirmed, the Service       |
|                                   | Officer must indicate how they    |
|                                   | authenticated the customer. The   |
|                                   | Service Officer will not be able  |
|                                   | to continue through the workflow  |
|                                   | until the Customer has been fully |
|                                   | authenticated.                    |
|                                   |                                   |
|                                   | If a customer has a nominee, the  |
|                                   | Service Officer will be presented |
|                                   | with a variation of the           |
|                                   | authentication screen.            |
|                                   |                                   |
|                                   | All sections will be greyed out   |
|                                   | except for the section allowing   |
|                                   | access to DOA DOCs and the        |
|                                   | customer's password. Once this is |
|                                   | completed the Service Officer     |
|                                   | must then confirm the nominees    |
|                                   | name and password before the      |
|                                   | workflow will allow the Service   |
|                                   | Officer to continue.              |
|                                   |                                   |
|                                   | If a nominee cannot recall their  |
|                                   | password refer to Appointing      |
|                                   | Nominees.                         |
|                                   |                                   |
|                                   | Once authentication is complete,  |
|                                   | if the customer has a payment     |
|                                   | nominee they will be presented    |
|                                   | with an 'assign card' screen.     |
|                                   |                                   |
|                                   | An information message will       |
|                                   | display stating:                  |
|                                   |                                   |
|                                   | 'BasicsCard can only be assigned  |
|                                   | to the customer where the payment |
|                                   | nominee has given their           |
|                                   | permission. Where the customer is |
|                                   | assigned the BasicsCard both the  |
|                                   | payment nominee and the customer  |
|                                   | must be authenticated'.           |
|                                   |                                   |
|                                   | The 'assign card' screen will     |
|                                   | inform the Service Officer who    |
|                                   | the BasicsCard is currently       |
|                                   | assigned to, and allows the       |
|                                   | payment nominee to assign the     |
|                                   | BasicsCard to the principal       |
|                                   | (customer) if they wish to.       |
|                                   |                                   |
|                                   | The Service Officer will be       |
|                                   | presented with the question       |
|                                   | 'assign BasicsCard to'. The       |
|                                   | options presented are 'nominee'   |
|                                   | or 'customer'.                    |
|                                   |                                   |
|                                   | If the nominee selects to have    |
|                                   | the BasicsCard assigned to the    |
|                                   | customer, they will be presented  |
|                                   | with the question 'does the       |
|                                   | nominee consent to assigning the  |
|                                   | BasicsCard to the customer'.      |
|                                   |                                   |
|                                   | -   If 'no' is selected then the  |
|                                   |     system will not allow the     |
|                                   |     Service Officer to go any     |
|                                   |     further and the Service       |
|                                   |     Officer must navigate back    |
|                                   |     and select 'nominee'          |
|                                   | -   If 'yes' is selected then the |
|                                   |     'continue' button will become |
|                                   |     available and allow the       |
|                                   |     Service Officer to proceed    |
|                                   |                                   |
|                                   | Note: if the customer does not    |
|                                   | have a payment nominee the assign |
|                                   | card section will not appear.     |
|                                   |                                   |
|                                   | Note: a customer who has exited   |
|                                   | Income Management, may request a  |
|                                   | BasicsCard within 60 days of      |
|                                   | Income Management ending to       |
|                                   | access residual funds in their IM |
|                                   | account.                          |
