Process an EBPRP claim form

Table 2: This table describes how the EBPRP Processing Team process an
EBPRP claim form.

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | EBPRP + Read more ...             |
|                                   |                                   |
|                                   | -   Log onto the EBPRP claims     |
|                                   |     system                        |
|                                   | -   Open customer first           |
|                                   | -   Medicare Portal               |
|                                   | -   Source off = select- (IBPR)   |

|                                   |     Breast Prosthesis Reimbur     |
|                                   | -   Select Submit                 |

| 2                                 | Medicare and Reference Number +   |
|                                   | Read more ...                     |
|                                   |                                   |
|                                   | -   Select the Claims tab         |
|                                   | -   Enter the claimant's Medicare |
|                                   |     number and reference number   |
|                                   | -   Select the Validate button    |
|                                   |                                   |
|                                   | The EBPRP claim screen will open  |
|                                   | to the applicable claimant’s      |
|                                   | details screen                    |
|                                   |                                   |
|                                   | Note: if a reference number is    |
|                                   | not entered, a return message     |
|                                   | will be displayed. Go to Step 3   |
+-----------------------------------+-----------------------------------+
| 3                                 | Recheck details + Read more ...   |
|                                   |                                   |

|                                   |     Medicare number and reference |

|                                   |     Maintenance System (CDMS)     |
|                                   | -   Enter the details again on    |
|                                   |     the EBPR screen               |
|                                   |                                   |
|                                   | Note: a return message will       |
|                                   | appear on the screen noting the   |
|                                   | error ‘No matching Medicare card  |
|                                   | number was found. There are no    |
|                                   | bank details in CDMS for this     |
|                                   | claimant'. A return message will  |
|                                   | also appear if the gender of the  |
|                                   | claimant is not female            |
|                                   |                                   |
|                                   | -   Recheck the screen to ensure  |
|                                   |     the Medicare reference number |
|                                   |     for the individual has been   |
|                                   |     entered and that the correct  |
|                                   |     claimants reference number is |
|                                   |     entered                       |
+-----------------------------------+-----------------------------------+
| 4                                 | Check Medicare and reference      |
|                                   | number details + Read more ...    |
|                                   |                                   |
|                                   | Confirm that the Medicare card    |
|                                   | number and the reference number   |
|                                   | details auto loaded onto the EBPR |
|                                   | screen are exactly the same as    |
|                                   | the details on the claim form.    |
|                                   |                                   |
|                                   | If they are not the same, go to   |
|                                   | Step 3 and recheck the claimant's |
|                                   | details.                          |
|                                   |                                   |
|                                   | Note: do not continue to process  |
|                                   | the claim if the Medicare and     |
|                                   | reference number do not match.    |
|                                   | The process ends here. Consult a  |
|                                   | Team Leader for further details.  |
|                                   |                                   |
|                                   | Attempt to contact the customer   |
|                                   | by phone                          |
|                                   |                                   |
|                                   | If contact is made:               |
|                                   |                                   |
|                                   | -   Read outbound call disclaimer |
|                                   |     to the customer               |
|                                   | -   Conduct an outbound call      |
|                                   |     security check                |
|                                   | -   Include the outcome of phone  |
|                                   |     call in comments              |
|                                   | -   For example, ‘EBPRP Claim DOL |
|                                   |     01/01/2001 (L/R/B) contacted  |
|                                   |     customer via phone confirmed  |
|                                   |     <reason>’                     |
|                                   |                                   |
|                                   | If contact cannot be made by      |
|                                   | phone, or when a customer refuses |
|                                   | to complete a security check a    |
|                                   | letter will need to be sent:      |
|                                   |                                   |
|                                   | -   Enter comment in CDMS         |
|                                   | -   For example, ‘EBPRP Claim DOL |
|                                   |     01/01/2001 (L/R/B) unable to  |
|                                   |     contact via phone. Letter     |
|                                   |     sent <reason>’                |
|                                   |                                   |
|                                   | Enter comment in PaNDA with       |
|                                   | reason/outcome                    |
|                                   |                                   |
|                                   | Close work item                   |
|                                   |                                   |
|                                   | Ensure Medicare card number is    |
|                                   | correct                           |
|                                   |                                   |
|                                   | Ensure naming convention is       |
|                                   | correct SurnameFirstname          |
|                                   |                                   |
|                                   | The process ends here.            |
+-----------------------------------+-----------------------------------+
| 5                                 | Complete details + Read more ...  |
|                                   |                                   |
|                                   | Complete the following fields of  |
|                                   | information:                      |
|                                   |                                   |
|                                   | -   Date of lodgement (DOL)       |
|                                   | -   Choose a Left or Right or     |
|                                   |     Both side purchase            |
|                                   | -   Date of purchase (when the    |
|                                   |     prosthesis was paid in full,  |
|                                   |     not ordered)                  |
|                                   |     Note: generally this is the   |
|                                   |     date on the EFT slip if it    |
|                                   |     differs from the invoice date |
|                                   | -   Purchase amount of external   |
|                                   |     breast prosthesis             |
|                                   | -   If discount given, use EBPRP  |
|                                   |     discount calculator. See the  |
|                                   |     Resources page for a link     |
|                                   |     Note: discounts from a store  |
|                                   |     are different to financial    |
|                                   |     assistance, PHI, DVA          |
|                                   | -   If the purchase is from       |
|                                   |     overseas, forward to EBPRP    |
|                                   |     Program Support to convert to |
|                                   |     Australian Dollars            |
|                                   | -   Any monies paid by            |
|                                   | -   Private Health Insurer (PHI)  |
|                                   | -   Department of Veteran’s       |
|                                   |     Affairs, or                   |
|                                   | -   Other financial assistance    |
|                                   |     (if applicable)               |
|                                   |     Note: if this question has    |
|                                   |     not been answered or answered |
|                                   |     as No, then the claim can be  |
|                                   |     processed)                    |
|                                   | -   The customer must have paid   |
|                                   |     for the prosthesis in full.   |
|                                   |                                   |
|                                   | Note: gift cards and store        |
|                                   | credits are considered the same   |
|                                   | as cash payments.                 |

| 6                                 | Confirm details + Read more ...   |
|                                   |                                   |
|                                   | Check all the details entered     |
|                                   | onto the EBPRP screen are the     |
|                                   | same as the details on the claim  |
|                                   | form                              |
|                                   |                                   |
|                                   | Ensure that all appropriate boxes |
|                                   | are filled before moving to the   |
|                                   | next step                         |
|                                   |                                   |
|                                   | -   The Date of Lodgement DOL     |
|                                   |     matches the date in PaNDA     |
|                                   | -   The date of purchase is when  |
|                                   |     the prosthesis was paid in    |
|                                   |     full                          |

|                                   |                                   |
|                                   | When all the details have been    |

|                                   | claims page select the Pay claim  |
|                                   | button at the bottom of the page. |
|                                   |                                   |
|                                   | This will enter the claim into    |
|                                   | the EBPRP system for payment.     |
|                                   |                                   |
|                                   | The system will show the claim is |
|                                   | unpaid as payment is pending or   |
|                                   | rejected if claim cannot be paid  |
|                                   | and a letter is required to be    |
|                                   | sent.                             |
+-----------------------------------+-----------------------------------+
| 8                                 | EBPRP Reference number + Read     |
|                                   | more ...                          |
|                                   |                                   |
|                                   | Note in PaNDA:                    |
|                                   |                                   |
|                                   | EBPRP reference number in         |
|                                   | comments (for example,            |
|                                   | EBPR0123911)                      |
|                                   |                                   |
|                                   | If National Office is contacted   |
|                                   | for approval to process a         |
|                                   | discretionary claim, leave a      |

|                                   |     process claim (for example,   |

|                                   |     discretionary request         |
|                                   |     approved or EBPRP claim DOL   |
|                                   |     11/8/2020 processed, National |
|                                   |     Office approval to process    |
|                                   |     claim (for example, when      |
|                                   |     there is a discrepancy with   |
|                                   |     the invoice/claim form)       |
|                                   |                                   |
|                                   | Private health fund details have  |
|                                   | been checked (for example, PHI    |
|                                   | checked)                          |
|                                   |                                   |

|                                   | EBPRP0123456 – PHI checked        |

|                                   |                                   |

|                                   | button.                           |

|                                   | Reserve Bank of Australia (RBA).  |

claimant that a claim has been rejected.

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Assess incorrect information +    |
|                                   | Read more ...                     |
|                                   |                                   |

|                                   | incorrect in the claimant’s       |

Incomplete EBPRP claims

Table 4: This table describes how the EBPRP Processing Team process an
incomplete EBPRP claim form.

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Identify Missing information +    |
|                                   | Read more ...                     |
|                                   |                                   |
|                                   | Assess the form and supporting    |
|                                   | documents to confirm what         |
|                                   | information is missing.           |
|                                   |                                   |
|                                   | Note: do not contact the claimant |
|                                   | if Question 10 ‘Have you received |
|                                   | a refund or financial assistance  |
|                                   | from a private health fund for    |
|                                   | the prosthesis(es) you are        |
|                                   | claiming?’ has not been answered. |
|                                   | In this instance the program will |
|                                   | make the assumption the answer is |
|                                   | ‘No’.                             |
+-----------------------------------+-----------------------------------+
| 2                                 | Contact claimant + Read more ...  |
|                                   |                                   |
|                                   | Contact the claimant using the    |
|                                   | phone number provided on the      |
|                                   | claim form to request the missing |
|                                   | information.                      |
|                                   |                                   |
|                                   | Note: if the customer can be      |
|                                   | uniquely identified using the     |
|                                   | information already provided,     |
|                                   | proceed with the processing of    |
|                                   | the claim.                        |
|                                   |                                   |
|                                   | Questions:                        |
|                                   |                                   |
|                                   | -   Medicare number and reference |
|                                   |     at Question 1                 |
|                                   | -   Side Left/Right/Both at       |
|                                   |     Question 3                    |
|                                   | -   No bank details recorded in   |
|                                   |     Consumer Directory            |
|                                   |     Maintenance System (CDMS) or  |
|                                   |     on claim form at Question 8   |
|                                   | -   Private Health Insurer (PHI)  |
|                                   |     – if ticked yes but no dollar |
|                                   |     amount entered at Question 10 |
|                                   |                                   |
|                                   | Only one attempt should be made   |
|                                   | to contact the claimant by phone. |
|                                   |                                   |
|                                   | Note: contact the supplier to     |
|                                   | confirm any details on the        |
|                                   | claimant’s invoice if needed. The |
|                                   | claimant's permission is not      |
|                                   | required. Quote the               |
|                                   | invoice/reference number.         |
+-----------------------------------+-----------------------------------+
| 3                                 | Confirmation of Identity + Read   |
|                                   | more ...                          |
|                                   |                                   |
|                                   | When the claimant has been        |
|                                   | contacted by phone, use the       |
|                                   | Medicare security check process   |
|                                   | to confirm the customer’s         |
|                                   | identity                          |
+-----------------------------------+-----------------------------------+
| 4                                 | Notation of contact attempt +     |
|                                   | Read more ...                     |
|                                   |                                   |
|                                   | Note in PaNDA and CDMS that       |
|                                   | contact with the claimant was     |
|                                   | attempted. Include the outcome of |
|                                   | contact.                          |
+-----------------------------------+-----------------------------------+
| 5                                 | Complete claim form + Read more   |
|                                   | ...                               |
|                                   |                                   |
|                                   | If successful in contacting the   |
|                                   | claimant, and the claimant is     |
|                                   | able to supply the details over   |
|                                   | the phone:                        |
|                                   |                                   |
|                                   | -   Process the claim for payment |
|                                   | -   Record the appropriate        |
|                                   |     comment for the claim         |
|                                   |     processing in both PaNDA and  |
|                                   |     CDMS                          |
|                                   |                                   |
|                                   | Note: if the claimant has not     |
|                                   | signed the claim form, it must be |
|                                   | returned to the claimant for      |
|                                   | signature before continuing to    |
|                                   | process the claim.                |
+-----------------------------------+-----------------------------------+
| 6                                 | Pending documentation + Read more |
|                                   | ...                               |
|                                   |                                   |
|                                   | If successful in contacting the   |
|                                   | claimant, and the required        |
|                                   | documents must be manually re     |
|                                   | sent:                             |
|                                   |                                   |
|                                   | -   Pend the work to yourself in  |
|                                   |     PaNDA until the required      |
|                                   |     documentation is received     |
|                                   | -   Leave a comment to explain    |
|                                   |     why the work item has been    |
|                                   |     Pended in both PaNDA and CDMS |
|                                   | -   The claim should not be held  |
|                                   |     for longer than 48 hours      |
|                                   | -   When documents received,      |
|                                   |     process the claim             |
|                                   | -   Upload all documents to PaNDA |
|                                   |     as a part of the claim ID     |
|                                   |     number                        |
|                                   | -   Follow the EBPRP letter       |
|                                   |     checking process              |
|                                   | -   Leave comment as to what has  |
|                                   |     been done                     |
|                                   | -   Complete work item in PaNDA   |
+-----------------------------------+-----------------------------------+
| 7                                 | Unsuccessful contact attempt +    |
|                                   | Read more ...                     |
|                                   |                                   |
|                                   | If unsuccessful in contacting the |
|                                   | claimant via the phone:           |
|                                   |                                   |
|                                   | -   Prepare the EBPRP more        |
|                                   |     information letter from the   |
|                                   |     Letters and electronic        |
|                                   |     messaging sub site            |
|                                   |                                   |
|                                   | Note: only Service Officers       |
