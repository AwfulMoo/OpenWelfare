| 4                                 | Recording a legal name change for |
|                                   | a customer (or a customer aged    |
|                                   | under 18) + Read more ...         |
|                                   |                                   |
|                                   | A legal name cannot be changed    |
|                                   | without evidence. The customer    |
|                                   | must provide one of the following |
|                                   | supporting documents issued by    |
|                                   | BDM before a changing a legal     |

|                                   |     the new name                  |
|                                   |                                   |
|                                   | Note: if the customer provides    |
|                                   | alternative evidence of a name    |
|                                   | change, or the name change        |

|                                   | overseas, email the Customer      |
|                                   | Details Helpdesk for advice.      |
|                                   |                                   |
|                                   | Supporting documents must be      |
|                                   | original and unaltered, and       |
|                                   | presented in person. If the       |
|                                   | customer contacts the Smart       |

|                                   | person due to COVID-19            |
|                                   | restrictions, see Table 2.        |

|                                   | the CPDS screen.                  |
|                                   |                                   |
|                                   | When supporting evidence is       |
|                                   | supplied, record the legal name   |
|                                   | change details as follows:        |
|                                   |                                   |
|                                   | Customer First                    |
|                                   |                                   |
|                                   | Go to the Customer Personal       |
|                                   | Details Summary (CPDS) screen     |
|                                   |                                   |
|                                   | Select Add new person details and |
|                                   | update the following fields (as   |
|                                   | appropriate):                     |
|                                   |                                   |
|                                   | -   Name Type (LGL)               |
|                                   | -   Event Date                    |
|                                   | -   Surname                       |
|                                   | -   Given Name/s                  |
|                                   | -   Sex                           |
|                                   | -   Date of Birth                 |
|                                   | -   Enter the appropriate Source  |
|                                   |     and Date of Receipt and press |
|                                   |     Continue                      |
|                                   | -   Select activity from Activity |
|                                   |     List (AL) screen and go to    |
|                                   |     Activity Results (AR) screen. |
|                                   |     Record in the Notes: field    |
|                                   |     details regarding name change |
|                                   | -   DOC action taken. Procedure   |
|                                   |     ends here                     |
|                                   |                                   |
|                                   | Note:                             |
|                                   |                                   |
|                                   | -   If the name change is the     |
|                                   |     result of a relationship      |
|                                   |     change, check the Marital     |
|                                   |     Status (MS) screen and update |
|                                   |     it in the same activity as    |
|                                   |     the name change, if not       |
|                                   |     previously updated            |
|                                   | -   Event date: this can be the   |
|                                   |     date the change occurred,     |
|                                   |     i.e. marriage date, or the    |
|                                   |     date the change is advised    |
|                                   | -   Unless correcting an error,   |
|                                   |     record previous names as Also |
|                                   |     Known As (AKA), Maiden Name   |
|                                   |     (MDN), Tribal Name or         |
|                                   |     Aboriginal or Torres Strait   |
|                                   |     Islander Traditional Name     |
|                                   |     (TRI), Name At Birth (NAB) or |
|                                   |     Preferred Name (PRF), as      |
|                                   |     appropriate. For further      |
|                                   |     information, see Aliases and  |
|                                   |     other names (legal,           |
|                                   |     preferred, sex and gender     |
|                                   |     diverse customers, cultural   |
|                                   |     considerations etc.)          |
|                                   | -   Names containing hyphens or   |
|                                   |     apostrophes should not        |
|                                   |     contain spaces, for example   |
|                                   |     when 'Anne-Maree' is coded as |
|                                   |     'Anne - Maree' when searching |
|                                   |     for a customer's record it    |
|                                   |     will not be found             |
|                                   | -   Diacritics or accent signs in |
|                                   |     the customerâ€™s name cannot be |
|                                   |     coded. Code the name without  |
|                                   |     diacritics, and include a     |
|                                   |     Display on Access (DOA) DOC   |
|                                   |     to note the differences in    |
|                                   |     spelling                      |
