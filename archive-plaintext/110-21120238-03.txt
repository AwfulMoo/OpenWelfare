Actions to prevent fraud

This table highlights key areas of concern about fraud and emergency
payments. Staff must continue to follow processes, including fraud
processes, in the Operational Blueprint for the relevant payment.

| 3                                 | Customer does not have a record + |
|                                   | Read more ...                     |
|                                   |                                   |
|                                   | Some fraudulent claims occur      |
|                                   | because of fabricated records     |
|                                   | being created, highlighting the   |
|                                   | need for care to be taken when    |
|                                   | establishing a person's identity. |
|                                   |                                   |
|                                   | Staff must make sure they are     |
|                                   | satisfied that the person making  |
|                                   | the claim or on the phone is who  |
|                                   | they say they are, before         |
|                                   | creating a new record or issuing  |
|                                   | a linking code.                   |
|                                   |                                   |
|                                   | Proof of Record Ownership cannot  |
|                                   | be completed on a new CRN. It is  |
|                                   | more critical that customers      |
|                                   | provide verifiable identity       |
|                                   | details when a CRN is created.    |
|                                   | Take extra care to make sure that |
|                                   | identity requirements are met     |
|                                   | when creating a new record.       |
|                                   |                                   |
|                                   | Some key indicators from recent   |
|                                   | fraud activity to be aware of     |
|                                   | when creating a new record        |
|                                   | include:                          |
|                                   |                                   |
|                                   | The customer's phone number does  |
|                                   | not show and says 'anonymous'     |
|                                   |                                   |
|                                   | The customer provides a different |
|                                   | phone number for their record     |
|                                   | creation saying they are using    |
|                                   | someone else's phone for the      |
|                                   | contact. Check SMS/Email Search   |
|                                   | for links to other customers      |
|                                   |                                   |
|                                   | Customers who say they are        |
|                                   | Australian born but do not have a |
|                                   | Customer Reference Number (CRN):  |
|                                   |                                   |
|                                   | -   ask where their parents were  |
|                                   |     born                          |
|                                   | -   if in Australia, then may     |
|                                   |     have a child record for       |
|                                   |     Family Tax Benefit (FTB)      |
|                                   |     purposes                      |
|                                   |                                   |
|                                   | The customer:                     |
|                                   |                                   |
|                                   | -   struggles to answer simple    |
|                                   |     PoRO questions such as Date   |
|                                   |     of Birth (DOB), partners or   |
|                                   |     children’s name or DOB - do   |
|                                   |     not continue with the call    |
|                                   | -   provides:                     |
|                                   |     - proof of age or photo ID    |
|                                   |     card and bank cards as their  |
|                                   |     only identity documents, or   |
|                                   |     - their Drivers Licence fails |
|                                   |     DVS and the customer advises  |
|                                   |     it recently expired or        |
|                                   |     license was suspended         |
|                                   | -   will say they have a friend   |
|                                   |     or partner with them who also |
|                                   |     needs to claim and passes the |
|                                   |     phone to them                 |
|                                   | -   seeks the alternative         |
|                                   |     identity process and cannot   |
|                                   |     provide at least one          |
|                                   |     verifiable document           |
|                                   |                                   |
|                                   | Note: it may be reasonable for an |
|                                   | individual to have one of these   |
|                                   | indicators. However, the more     |
|                                   | indicators a person has, the      |
|                                   | greater the identity fraud risk.  |
|                                   |                                   |
|                                   | For more details, see:            |
|                                   |                                   |
|                                   | -   AGDRP, NZ DRP, DRA and NZ     |
|                                   |     DRA - Claiming emergency      |
|                                   |     payments                      |
|                                   | -   Pandemic Leave Disaster       |
|                                   |     Payment - Pandemic Leave      |
|                                   |     Disaster Payment - Assisted   |
|                                   |     Customer Claims               |

| 4                                 | The customer has trouble          |
|                                   | answering PORO questions + Read   |
|                                   | more ...                          |
|                                   |                                   |
|                                   | Never give the customer details   |
|                                   | for them to confirm. Scammers     |
|                                   | collect these details so they can |
|                                   | impersonate customers and access  |
|                                   | their record.                     |
|                                   |                                   |

|                                   | Ownership by asking multiple PoRO |
|                                   | questions in addition to CRN,     |
|                                   | name, date of birth and password: |
|                                   |                                   |
|                                   | -   Name and date of birth of     |
|                                   |     parents, partner, children    |
|                                   | -   When they travelled overseas  |
|                                   |     last (check the RSIM screen)  |
|                                   | -   Email and telephone number    |
|                                   | -   Last known payment with       |
|                                   |     Centrelink                    |
|                                   |                                   |
|                                   | Confirm that the customer does    |
|                                   | not have an existing record, see  |
|                                   | Searching for a customer on the   |
|                                   | system                            |
|                                   |                                   |
|                                   | -   Search with minimal fields,   |
|                                   |     such as the surname and date  |
|                                   |     of birth only                 |
|                                   | -   Check spelling, and ask if    |
|                                   |     they have gone by any other   |
|                                   |     names                         |
|                                   | -   Check if the customer's       |
|                                   |     mobile phone number has been  |
|                                   |     recorded on another record by |
|                                   |     conducting a SMS/Email Search |
|                                   |                                   |
|                                   | Staff with Online Search Facility |
|                                   | access can perform additional     |
|                                   | searches for the customer         |
|                                   |                                   |
|                                   | -   Customers born in Australia   |
|                                   |     should have a Medicare (HIC)  |
|                                   |     record on OLS                 |
|                                   |                                   |

|                                   | Identity Confirmation Dashboard.  |

|                                   | -   shooter or firearm licence    |
|                                   |                                   |

|                                   | emergency payments. Staff must    |
|                                   | make sure they follow the correct |
|                                   | processing Operational Blueprint  |

|                                   | payment when creating a new       |
|                                   | record and issuing a linking code |
|                                   |                                   |
|                                   | If the customer has trouble       |
|                                   | answering PoRO questions, or      |
|                                   | there are doubts:                 |

|                                   |     back as per authenticating a  |
|                                   |     Centrelink customer process   |

|                                   | -   tell them to attend a service |
|                                   |     centre with photo             |
|                                   |     identification                |
|                                   | -   do not update the record or   |
|                                   |     release any information if    |
|                                   |     you doubt you are speaking to |
|                                   |     the customer or their         |
|                                   |     authorised representative     |
|                                   |                                   |
|                                   | See the Resources page for a link |

|                                   |                                   |
|                                   | If there are concerns about the   |
|                                   | customer's identity, stolen or    |

|                                   |                                   |
