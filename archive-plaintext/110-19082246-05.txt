Considerations when actioning account-based and market-linked handoffs

+-----------------------------------+-----------------------------------+
| Item                              | Description                       |
+-----------------------------------+-----------------------------------+
| 1                                 | Category 9 commutation checks     |
|                                   |                                   |
|                                   | Category 9 income stream reviews  |
|                                   | may have a commutation handoff    |
|                                   | keyword. As commutations are not  |
|                                   | a factor for these income         |
|                                   | streams, determine if:            |
|                                   |                                   |
|                                   | -   a significant decrease in     |
|                                   |     account balance has occurred  |
|                                   |     and requires investigation    |
|                                   | -   the customer's payment type   |
|                                   |     will be impacted by this      |
|                                   |     decrease in account balance   |
|                                   |                                   |
|                                   | If uncertain whether the          |
|                                   | commutations require              |
|                                   | investigation, refer to Technical |
|                                   | Peer Support (TPS). Record any    |
|                                   | decision in Notes before          |
|                                   | finishing the activity.           |
+-----------------------------------+-----------------------------------+
| 2                                 | System limitations for            |
|                                   | commutations                      |
|                                   |                                   |
|                                   | The system only displays total    |
|                                   | commutations made from 6 December |
|                                   | 2004. Commutations recorded       |
|                                   | before 6 December 2004 would have |
|                                   | been adjusted in the Original     |
|                                   | Purchase Price.                   |
+-----------------------------------+-----------------------------------+
| 3                                 | 1 July balance coding before 2007 |
|                                   |                                   |
|                                   | When recoding account based       |
|                                   | income streams (AIS) products     |
|                                   | with commenced before 1 July      |
|                                   | 2007, date of events (DOV) before |
|                                   | 1 July 2007 will have no 1 July   |
|                                   | account balance. As a workaround, |
|                                   | use the first available current   |
|                                   | account balance in the applicable |
|                                   | financial year as the 1 July      |
|                                   | balance.                          |
+-----------------------------------+-----------------------------------+
| 4                                 | Purchase and commencement dates   |
|                                   | for account-based and             |
|                                   | market-linked income streams      |
|                                   |                                   |
|                                   | The system uses the commencement  |
|                                   | date for all income stream types, |
|                                   | rather than the purchase date.    |
|                                   | For account based and market      |
|                                   | linked income streams, the        |
|                                   | commencement date of the product  |
|                                   | must be identical to the purchase |
|                                   | date to ensure an accurate        |
|                                   | assessment occurs.                |
|                                   |                                   |
|                                   | As purchase dates and             |
|                                   | commencement dates can differ,    |
|                                   | rectification is not required     |
|                                   | based on the provider data        |
|                                   | supplied as part of the August    |
|                                   | and February review process. If   |
|                                   | the commencement date appears to  |
|                                   | have been coded incorrectly,      |
|                                   | refer to previous schedules or    |
|                                   | request a new schedule to verify  |
|                                   | if needed.                        |
+-----------------------------------+-----------------------------------+
| 5                                 | Relevant number - reversionary    |
|                                   | beneficiary passes away before    |
|                                   | primary beneficiary               |
|                                   |                                   |
|                                   | If the primary beneficiary        |
|                                   | nominated a binding reversionary  |
|                                   | beneficiary and the nominated     |
|                                   | reversionary beneficiary passes   |
|                                   | away, the relevant number may     |
|                                   | change back to the primary        |
|                                   | beneficiary’s life expectancy at  |
|                                   | the time of purchase. This would  |
|                                   | only occur if the reversionary    |
|                                   | beneficiary had a higher life     |
|                                   | expectancy (as this is the new    |
|                                   | expected lifespan of the          |
|                                   | product). It is not mandatory for |
|                                   | the relevant number to change in  |
|                                   | such cases, it is up to the       |
|                                   | provider and is based on the      |
|                                   | super fund’s trust deed.          |
|                                   |                                   |
|                                   | If the provider data appears to   |
|                                   | be using the relevant number      |
|                                   | based on the deceased             |
|                                   | reversionary beneficiary’s life   |
|                                   | expectancy, it can be accepted.   |
+-----------------------------------+-----------------------------------+
| 6                                 | Category 9 fields not checked as  |
|                                   | part of automation process        |
|                                   |                                   |
|                                   | The system does not check the     |
|                                   | original purchase price or        |
|                                   | relevant number for category 9    |
|                                   | income stream products, as they   |
|                                   | are not relevant for the          |
|                                   | assessment of the product.        |
+-----------------------------------+-----------------------------------+
| 7                                 | Start Date Original Commuted      |
|                                   | Product - market-linked income    |
|                                   | streams                           |
|                                   |                                   |
|                                   | Some market-linked income streams |
|                                   | with a purchase date on or after  |
|                                   | 20 September 2007 can retain the  |
|                                   | asset-test exemption (ATE) of the |
|                                   | original product.                 |
|                                   |                                   |
|                                   | For more information, see Adding  |
|                                   | or updating a market-linked       |
|                                   | income stream.                    |
|                                   |                                   |
|                                   | Contact the FINS helpdesk before  |
|                                   | coding this field. Normal         |
|                                   | escalation procedures apply.      |

QMA details

Activities submitted to QMA must not have the following fields marked as
an error for category 9 (deemed) Account-based income streams when
coding, as they do not affect assessment or future product matching
under AISR processes:

-   Incorrect Relevant Number
-   Incorrect 1 July balance
-   Incorrect Original Purchase Price
-   Incorrect Gross Annual Payment

Regular processes apply to income streams coded outside of AISR
activities.
