|                                   |     scripts from the refundable   |

|                                   | -   Repeat this process until the |
|                                   |     refundable amount is reached  |
|                                   |     or exceeded                   |
|                                   |                                   |
|                                   | Note: if there are several        |
|                                   | scripts on the same day, refund   |
|                                   | the most inexpensive script that  |
|                                   | does reach or exceeds the         |
|                                   | refundable amount.                |

| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Ascertain date safety net was     |
|                                   | reached + Read more ...           |
|                                   |                                   |
|                                   | Check all documents for duplicate |
|                                   | entries or Non-PBS/private        |
|                                   | scripts (outside public hospital  |
|                                   | dispensing).                      |
+-----------------------------------+-----------------------------------+
| 2                                 | Calculate threshold + Read more   |
|                                   | ...                               |
|                                   |                                   |
|                                   | Over the threshold refunds:       |
|                                   |                                   |
|                                   | -   Check all documents for       |
|                                   |     duplicate entries, private    |
|                                   |     scripts or SN20DR with $0.00  |
|                                   |     (these are not to be included |
|                                   |     in the calculation)           |
|                                   | -   Combine total amount for all  |
|                                   |     items                         |
|                                   | -   Subtract the threshold amount |
|                                   |     from combined total           |
|                                   |                                   |
|                                   | This is the amount that we need   |
|                                   | to subtract from the PRFs to find |
|                                   | the date the threshold was met.   |
+-----------------------------------+-----------------------------------+
| 3                                 | Determine date is met + Read more |
|                                   | ...                               |
|                                   |                                   |
|                                   | -   Working in descending order   |
|                                   |     from most recent to oldest    |
|                                   | -   Subtract each script from the |
|                                   |     total amount                  |
|                                   | -   Repeat this process until     |
|                                   |     only the threshold amount     |
|                                   |     remains                       |
|                                   |                                   |
|                                   | Note: if there are several        |
|                                   | scripts on the same day, utilise  |
|                                   | the most inexpensive script that  |
|                                   | reaches or does not exceed the    |
|                                   | threshold amount.                 |
|                                   |                                   |
|                                   | Example: the threshold is $244.80 |
|                                   | with 36 scripts at $6.80 that     |
|                                   | would be date threshold was met.  |
|                                   | However if the scripts are $5.80  |
|                                   | then the threshold would be met   |
|                                   | at 42 scripts totalling $423.60.  |
|                                   |                                   |
|                                   | Anything within $6.80 of          |
|                                   | threshold, between $238.01 and    |
|                                   | $244.80 will be the date          |
|                                   | required.                         |
|                                   |                                   |
|                                   | or                                |
|                                   |                                   |
|                                   | The threshold is $1457.10 then    |
|                                   | the threshold date will be the    |
|                                   | closest script to the threshold   |
|                                   | without exceeding the threshold.  |
|                                   | This can be between $1414.61 and  |
|                                   | $1457.10.                         |
|                                   |                                   |
|                                   | Note: from 1 July 2022 the Safety |
|                                   | Net thresholds have been lowered  |
|                                   | and are not retrospectively       |
|                                   | applied. Refunds are not payable  |
|                                   | for items supplied before 1 July  |
|                                   | that exceed the new threshold.    |
+-----------------------------------+-----------------------------------+
| 4                                 | Change date in mainframe + Read   |
|                                   | more ...                          |

|                                   | -   Change the date of issue to   |
