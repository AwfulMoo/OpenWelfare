System reference match type codes

Table 1: This table includes the match keys and scores used for identity
matching assessments. The highest score from each match category is used
for the assessment. For example, if the customer has two potential
matches under the 'Personal information' category, the system will use
the higher score of the potential matches, rather than adding them
together.

|                       | Sex.                  |                       |
+-----------------------+-----------------------+-----------------------+

|                       | and First Name. Used  |                       |
|                       | where other features  |                       |
|                       | (such as sex or DOB)  |                       |
|                       | may not be present in |                       |
|                       | the data.             |                       |
+-----------------------+-----------------------+-----------------------+
| Personal information  | Matched on Maiden     | 20                    |
|                       | Name and First Name.  |                       |
|                       | Used where no other   |                       |
|                       | features (such as sex |                       |
|                       | on DOB) are present   |                       |
|                       | in the data.          |                       |
+-----------------------+-----------------------+-----------------------+
| Personal information  | Surname, First Name   | 20                    |
|                       | and Middle Name       |                       |
|                       | swapped, DOB, Sex.    |                       |
