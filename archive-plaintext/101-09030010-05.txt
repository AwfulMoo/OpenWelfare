FAQs about the Organisation review workflow

Table 1

+-----------------------------------+-----------------------------------+
| Item                              | Description                       |
+-----------------------------------+-----------------------------------+
| 1                                 | Question: Why can't text be added |
|                                   | to the notes section on the set   |
|                                   | up page?                          |
|                                   |                                   |
|                                   | Answer:                           |
|                                   |                                   |
|                                   | If there is a future date,        |
|                                   | details cannot be entered. To     |
|                                   | work around this:                 |
|                                   |                                   |
|                                   | -   Make the date today's date,   |
|                                   |     enter the notes and then      |
|                                   |     change the date back to when  |
|                                   |     the review will be done       |
|                                   | -   Save and Exit. Do not         |
|                                   |     continue onto the other tabs  |
|                                   |     until they need to be         |
|                                   |     completed                     |
+-----------------------------------+-----------------------------------+
| 2                                 | Question: Can documents be        |
|                                   | uploaded outside of the workflow? |
|                                   |                                   |
|                                   | Answer:                           |
|                                   |                                   |
|                                   | Yes. For the procedure to add     |
|                                   | documents after completing the    |
|                                   | workflow, see Digital Information |
|                                   | Management and Locating digital   |
|                                   | images. Select No where it says   |
|                                   | Hand Off For Further Action and   |
|                                   | classify the documents as TPOREV. |
+-----------------------------------+-----------------------------------+
| 3                                 | Question: If a review has been    |
|                                   | selected as random or targeted,   |
|                                   | can this be changed to ad-hoc?    |
|                                   |                                   |
|                                   | Answer:                           |
|                                   |                                   |
|                                   | No. TPOs selected as random or    |
|                                   | targeted cannot have the review   |
|                                   | reason changed, if another review |
|                                   | was added in error, a request     |
|                                   | will need to be sent to the       |
|                                   | administrator to remove one of    |
|                                   | the activities.                   |
+-----------------------------------+-----------------------------------+
| 4                                 | Question: How to tell if a step   |
|                                   | has been missed in the review     |
|                                   |                                   |
|                                   | Answer:                           |
|                                   |                                   |
|                                   | The review cannot be completed if |
|                                   | a step has been missed. A red     |
|                                   | asterisk on one of the tab panels |
|                                   | will indicate that a missed       |
|                                   | question.                         |
+-----------------------------------+-----------------------------------+
| 5                                 | Question: How to view uploaded    |
|                                   | documents after a review has been |
|                                   | completed?                        |
|                                   |                                   |
|                                   | Answer:                           |
|                                   |                                   |
|                                   | Uploaded documents can be viewed  |
|                                   | in 2 ways, via:                   |
|                                   |                                   |
|                                   | Organisation Compliance Review    |
|                                   | Report on the business record:    |
|                                   |                                   |
|                                   | -   select the twisty Completed   |
|                                   | -   select View                   |
|                                   | -   select View attachments       |
|                                   | -   Note: an error may appear the |
|                                   |     first time, re-try and the    |
|                                   |     attached document should open |
|                                   |                                   |
|                                   | Document Tools in Customer First  |
+-----------------------------------+-----------------------------------+
| 6                                 | Question: There is a future       |
|                                   | review due to non-compliance, can |
|                                   | this review be finalised before   |
|                                   | the due date?                     |
|                                   |                                   |
|                                   | Answer:                           |
|                                   |                                   |
|                                   | Yes. The review date can be       |
|                                   | changed when the review is        |
|                                   | updated. The date can be changed  |
|                                   | to the date the review is being   |
|                                   | finalised. There is no need to    |
|                                   | wait until the due date.          |
+-----------------------------------+-----------------------------------+
| 7                                 | Question: If the reviewer         |
|                                   | changes, can the set up panel be  |
|                                   | changed to reflect this?          |
|                                   |                                   |
|                                   | Answer:                           |
|                                   |                                   |
|                                   | The setup panel does not need to  |
|                                   | change if the reviewer changes.   |
|                                   | The person who selects the        |
|                                   | workflow will become the          |
|                                   | reviewer. Name and Logon ID will  |
|                                   | be displayed. If the reviewer     |
|                                   | does change, the review may need  |
|                                   | to be allocated to the current    |
|                                   | reviewer in workload manager.     |
+-----------------------------------+-----------------------------------+
| 8                                 | Question: If the incorrect review |
|                                   | reason has been selected, can     |
|                                   | this be changed after the         |
|                                   | workflow has commenced?           |
|                                   |                                   |
|                                   | Answer:                           |
|                                   |                                   |
|                                   | No, the review reason cannot      |
|                                   | change once entered. Any errors   |
|                                   | made in the workflow can be       |
|                                   | deleted. Send a request via email |
|                                   | to Program Assurance team. Use    |
|                                   | the referral template on the      |
|                                   | Third Party Programmes Team       |
|                                   | shared drive.                     |

Details of review methods

Table 3

+-----------------------------------+-----------------------------------+
| Type of review                    | Review method                     |
|                                   |                                   |
|                                   | Note: the information below does  |
|                                   | not apply to Centrepay            |
+-----------------------------------+-----------------------------------+
| Targeted reviews (TAR)            | Desk review                       |
|                                   |                                   |
| Targeted Assurance Checks (TAC)   | TARs address potential or actual  |
|                                   | behaviours by individual          |
|                                   | businesses that represent a risk  |
|                                   | to programs and have been         |
|                                   | identified via the various        |
|                                   | sources of information that       |
|                                   | program owners use for assurance  |
|                                   | purposes. Data is used to make    |
|                                   | inferences aboout the likelihood  |
|                                   | of individual businesses engaging |
|                                   | in potentially non-compliant      |
|                                   | behaviour. Business category,     |
|                                   | time since last review and        |
|                                   | transaction/enquiry/deduction     |
|                                   | volume are secondary risk         |
|                                   | factors.                          |
|                                   |                                   |
|                                   | Targeted reviews must be          |
|                                   | completed via desk review.        |
|                                   |                                   |
|                                   | TAR reviews will be loaded as     |
|                                   | ad-hoc sampling for the financial |
|                                   | year, but can also be added       |
|                                   | manually by typing M3RV in Next   |
|                                   | and then selecting the 'add       |
|                                   | review' button. For the type of   |
|                                   | review, select TAR.               |
|                                   |                                   |
|                                   | Centrepay has Targeted Assurance  |
|                                   | Checks - TACs.                    |
+-----------------------------------+-----------------------------------+
| Education, Support and Training   | Desk review                       |
| (EST)                             |                                   |
|                                   | EST is a critical component of    |
|                                   | program assurance under the       |
|                                   | ‘Assist to Comply’ model.         |
|                                   | Businesses participating in       |
|                                   | programs generally have a strong  |
|                                   | willingness to comply. Education, |
|                                   | support and training are critical |
|                                   | to ensuring they are aware of     |
|                                   | their obligations and can meet    |
|                                   | them.                             |
|                                   |                                   |
|                                   | An EST activity may be conducted  |
|                                   | on the basis that:                |
|                                   |                                   |
|                                   | -   the business will not be      |
|                                   |     reviewed any further in the   |
|                                   |     relevant financial year,      |
|                                   |     and/or                        |
|                                   | -   the business Service Reason   |
|                                   |     or main business/service      |
|                                   |     category is high risk and the |
|                                   |     business is not being         |
|                                   |     reviewed in the relevant      |
|                                   |     financial year                |
|                                   | -   there has been past           |
|                                   |     non-compliance and the        |
|                                   |     business needs more support   |
|                                   |     to meet their obligations     |
|                                   |                                   |
|                                   | EST based activities are          |
|                                   | completed over the phone.         |
|                                   |                                   |
|                                   | A CCeS training presentation has  |
|                                   | been developed to help the        |
|                                   | reviewer which can also be        |
|                                   | emailed to the TPO.               |
|                                   |                                   |
|                                   | EST activities can be completed   |
|                                   | in the TPO Program Assurance      |
|                                   | Workflow. EST reviews will be     |
|                                   | added by ad-hoc sampling and can  |
|                                   | be added manually by typing M3RV  |
|                                   | in Next and then selecting the    |
|                                   | 'add review' button. For the type |
|                                   | of review, select EST.            |
|                                   |                                   |
|                                   | The outcome of an EST review can  |
|                                   | be:                               |
|                                   |                                   |
|                                   | -   Assurance Met - no issues     |
|                                   |     found during the EST          |
|                                   | -   Assurance not met - GND       |
|                                   |     review is required (informal  |
|                                   |     review required)              |
|                                   | -   Assurance not met -TAR review |
|                                   |     is required (issues found     |
|                                   |     during the EST that require a |
|                                   |     formal review                 |
|                                   |                                   |
|                                   | Note: this does not apply to      |
|                                   | Centrepay.                        |
+-----------------------------------+-----------------------------------+
| Post Approval Discussions (PAD)   | Desk review                       |
|                                   |                                   |
|                                   | PADs are conducted with all new   |
|                                   | businesses approved to            |
|                                   | participate in programs, except   |
|                                   | BCD merchants who do not sell     |
|                                   | excluded goods. PADs promote      |
|                                   | messages around key areas of      |
|                                   | program risk and proactively      |
|                                   | encourage compliance.             |
|                                   |                                   |
|                                   | PADs are to be conducted 30 days  |
|                                   | after the TPO's Business          |
|                                   | Application has been approved.    |
|                                   |                                   |
|                                   | PADs are automatically created    |
|                                   | once the business has been        |
|                                   | approved to participate in a      |
|                                   | program (except for TPOs who have |
|                                   | been rolled out nationally).      |
|                                   |                                   |
|                                   | PADs can be added manually if     |
|                                   | needed (via the M3RV screen and   |
|                                   | then selecting the 'add review'   |
|                                   | button. For the type of review,   |
|                                   | select PAD).                      |
|                                   |                                   |
|                                   | A list of themes and prompts will |
|                                   | display the reviewer can select   |
|                                   | if the list of prompts have been  |
|                                   | discussed.                        |
|                                   |                                   |
|                                   | The answers to the list of        |
|                                   | prompts is either Discussed/Not   |
|                                   | Discussed.                        |
|                                   |                                   |
|                                   | Where the answer is 'N/A' or not  |
|                                   | Discussed, it should be noted in  |
|                                   | the comments section of the       |
|                                   | workflow.                         |
|                                   |                                   |
|                                   | The outcome of a PAD can be:      |
|                                   |                                   |
|                                   | -   Assurance Met                 |
|                                   | -   Assurance not Met - a GND may |
|                                   |     be needed (informal review    |
|                                   |     required)                     |
|                                   | -   Assurance not Met - which may |
|                                   |     require EST (education based  |
|                                   |     approach required)            |
|                                   | -   Assurance not Met - more      |
|                                   |     serious issues are found      |
|                                   |     which requires a Targeted     |
|                                   |     review                        |
|                                   |                                   |
|                                   | Any EST, GND or TAR review        |
|                                   | created as a result from a PAD    |
|                                   | will be linked to the initial     |
|                                   | review.                           |
|                                   |                                   |
|                                   | Reviewers are to conduct PADs     |
|                                   | with merchants who sell excluded  |
|                                   | goods and any new Income          |
|                                   | Management TPOs newly approved.   |
|                                   |                                   |
|                                   | Reviewers are to conduct PADs     |
|                                   | with newly approved CCeS and EVoR |
|                                   | TPOs.                             |
|                                   |                                   |
|                                   | If the merchant has been          |
|                                   | nationally rolled out, a reviewer |
|                                   | should send the BasicsCard        |
|                                   | template - BasicsCard for New     |
|                                   | Merchant. See the Resources page  |
|                                   | for the template.                 |
|                                   |                                   |
|                                   | Centrepay has Post Approval       |
|                                   | Checks which fall due 30 days     |
|                                   | after the business has been       |
|                                   | approved.                         |
+-----------------------------------+-----------------------------------+
| Rent Deduction Scheme (RDS)       | Desk review                       |
| Reviews                           |                                   |
|                                   | RDS reviews can be completed      |
|                                   | within the workflow. These        |
|                                   | reviews can be added manually by  |
|                                   | entering M3RV in Next and then    |
|                                   | selecting the 'add review'        |
|                                   | button. In the type of review,    |
|                                   | will be able to select RDS.       |
|                                   |                                   |
|                                   | This type of review is            |
|                                   | educational and support based.    |
|                                   | There are no breaches, and if any |
|                                   | issues or gaps are identified,    |
|                                   | the reviewer should support and   |
|                                   | help the state housing authority  |
|                                   | where appropriate.                |
|                                   |                                   |
|                                   | The response to questions is      |
|                                   | either Yes/No and a comments      |
|                                   | section for each question is      |
|                                   | available to provide an           |
|                                   | explanation of what was           |
|                                   | discussed.                        |
|                                   |                                   |
|                                   | The outcome of this review will   |
|                                   | always be 'completed'.            |
|                                   |                                   |
|                                   | The reviewer is required to       |
|                                   | conduct reviews with their State  |
|                                   | Housing Authorities once every    |
|                                   | financial year.                   |
|                                   |                                   |
|                                   | Note: Q825 for RDS does not need  |
|                                   | to be issued. RDS is conducted in |
|                                   | conjunction with a CCeS review.   |
|                                   | When the Q825 is sent for the     |
|                                   | CCeS review, advise the state     |
|                                   | housing authority that the RDS    |
|                                   | review will also be conducted.    |
|                                   |                                   |
|                                   | Note: VIC should need to review   |
|                                   | TAS as well as the VIC state      |
|                                   | housing authority and NSW need to |
|                                   | review ACT as well as the NSW     |
|                                   | state housing authority.          |
+-----------------------------------+-----------------------------------+
| Random Sampling (RND)             | Desk review                       |
|                                   |                                   |
| Random Assurance Check (RAC)      | Random sample reviews are         |
|                                   | conducted to provide an indicator |
|                                   | of program health. The Program    |
|                                   | Assurance team will advise if a   |
|                                   | random sample has been conducted  |
|                                   | for the relevant financial year.  |
|                                   |                                   |
|                                   | RND reviews will be loaded as     |
|                                   | ad-hoc sampling for the financial |
|                                   | year, but can also be added       |
|                                   | manually by typing M3RV in Next   |
|                                   | and then selecting on the 'add    |
|                                   | review' button. For the type of   |
|                                   | review, select RND.               |
|                                   |                                   |
|                                   | Randomly selected CCeS and        |
|                                   | BasicsCard businesses that have   |
|                                   | low or nil transactions and/or    |
|                                   | are low risk will have Baseline   |
|                                   | Check questions only.             |
|                                   |                                   |
|                                   | Income Management reviews will be |
|                                   | conducted in association with     |
|                                   | BasicsCard if the Income          |
|                                   | Management business is in the     |
|                                   | Northern Territory.               |
|                                   |                                   |
|                                   | Centrepay has Random Assurance    |
|                                   | Checks (RACs).                    |
+-----------------------------------+-----------------------------------+
| General Activity (GND)            | Desk review                       |
|                                   |                                   |
|                                   | The conversation or any           |
|                                   | correspondence sent can be        |
|                                   | recorded here if there is concern |
|                                   | regarding a media article (for    |
|                                   | example, a referral from ASIC) if |
|                                   | there:                            |
|                                   |                                   |
|                                   | -   is no direct impact to the    |
|                                   |     agency's program, but         |
|                                   | -   may be concern there could be |
|                                   |     damage to the agency's        |
|                                   |     reputation                    |
|                                   |                                   |
|                                   | If any support outside of the     |
|                                   | education process was provided to |
|                                   | the business, this can also be    |
|                                   | recorded here.                    |
|                                   |                                   |
|                                   | On an ad-hoc basis, the Program   |
|                                   | Assurance team will load a GND    |
|                                   | with instructions to proceed with |
|                                   | a Targeted review, outlining what |
|                                   | needs to be checked.              |
|                                   |                                   |
|                                   | If a General (GND) activity is    |
|                                   | created (by typing M3RV in Next   |
|                                   | and then selecting on the 'add    |
|                                   | review' button), for the type of  |
|                                   | review, select GND.               |
|                                   |                                   |
|                                   | The options available to select   |
|                                   | include:                          |
|                                   |                                   |
|                                   | -   Ad-hoc                        |
|                                   | -   Media                         |
|                                   | -   Data Integrity                |
|                                   | -   Regulatory agency referral    |
|                                   | -   Complaint                     |
|                                   | -   TPO initiated                 |
|                                   | -   Services Australia initiated  |
|                                   |                                   |
|                                   | In the set up section of the      |
|                                   | activity, provide details on the  |
|                                   | option selected and why the GND   |
|                                   | activity is being run (limit of   |
|                                   | 250 characters).                  |
|                                   |                                   |
|                                   | The outcome of these reviews      |
|                                   | could be either:                  |
|                                   |                                   |
|                                   | -   Assurance Met - no issues     |
|                                   |     found                         |
|                                   | -   Assurance Not Met - which     |
|                                   |     requires education,           |
|                                   |     support/training              |
|                                   | -   Assurance Not Met - which     |
|                                   |     requires a full review -      |
|                                   |     Targeted review               |
|                                   | -   Event Recorded - no contact   |
|                                   |     was made with the business    |
|                                   |     and a GND was recorded due to |
|                                   |     an event that does not        |
|                                   |     directly impact the program   |
|                                   |                                   |
|                                   | Any EST or TAR review created as  |
|                                   | a result from a GND will be       |
|                                   | linked to the initial review.     |
|                                   |                                   |
|                                   | There is a 500 character limit on |
|                                   | the completion page. If more      |
|                                   | information is required, there is |
|                                   | the option to upload documents.   |
|                                   |                                   |
|                                   | Assurance met as opposed to event |
|                                   | recorded:                         |
|                                   |                                   |
|                                   | If Assurance is Met, it means     |
|                                   | there had to be a level of        |
|                                   | discussion with the TPO to assure |
|                                   | the agency that there are no      |
|                                   | issues with the program.          |
|                                   |                                   |
|                                   | Event Recorded is where a         |
|                                   | discussion with the TPO is not    |
|                                   | required, or the impact of the    |
|                                   | referral/media article or         |
|                                   | complaint is not related to the   |
|                                   | Deduction and Confirmation        |
|                                   | Services Branch. The agency       |
|                                   | acknowledges the referral/media   |
|                                   | article or complaint by           |
|                                   | documenting that it has no impact |
|                                   | on the program at hand and        |
|                                   | selecting the 'event recorded'    |
|                                   | outcome in the workflow.          |
|                                   |                                   |
|                                   | It is a requirement to check GND  |
|                                   | activities before starting a      |
|                                   | review as well as checking        |
|                                   | previous history of               |
|                                   | non-compliance. DOC this has been |
|                                   | checked.                          |
|                                   |                                   |
|                                   | Assurance not met means a breach  |
|                                   | was identified. Depending on the  |
|                                   | nature of the breach and how      |
|                                   | serious it is, the outcome could  |
|                                   | either lead to an education and   |
|                                   | support based activity or a full  |
|                                   | Targeted review                   |
|                                   |                                   |
|                                   | The above does not apply to       |
|                                   | Centrepay. Centrepay has General  |
|                                   | Activities (GNCs) which will be   |
|                                   | deemed as the re-review for MI    |
|                                   | purposes.                         |

Requesting access from ICT Security Portal (ISP)

Request access to the Organisation (Compliance) Review workflow through
the ICT Security Portal:

-   Select Work Dashboard > Make a Process Request>Resources from drop
    down box then select continue
-   Under Make a Process Request select Request Resource
-   Under Search Criteria: Mainframe Applications (select + next to
    Mainframe Applications for > Income Security. Then select Search
-   Under the Search Results section, a list of resources will display.
    Scroll down to CM3UPD - Account Managers completing a Third Party
    Organisation Compliance Reviews
-   Select this to highlight it, then select the right arrow to move it
    to the Resources to Request section. In the Reason section at the
    top of the page, type why access is needed
-   Select Submit
