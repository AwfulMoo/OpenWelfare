Adding or removing CCeS characteristics

Table 5: This table describes the process for staff to add or remove
CCeS characteristics from a CCeS business.

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Assessing business request + Read |
|                                   | more ...                          |
|                                   |                                   |
|                                   | All requests by a CCeS business   |
|                                   | to add or remove characteristics  |
|                                   | to its profile must be in         |
|                                   | writing.                          |
|                                   |                                   |
|                                   | If characteristics are being      |
|                                   | added the business must detail    |
|                                   | the additional information        |
|                                   | required and why it is needed.    |
|                                   |                                   |
|                                   | A CCeS business should only be    |
|                                   | given access to the minimum       |
|                                   | amount of information required to |
|                                   | enable it to confirm entitlement  |
|                                   | to a concession, rebate or        |
|                                   | service.                          |
+-----------------------------------+-----------------------------------+
| 2                                 | Access the business record + Read |
|                                   | more ...                          |
|                                   |                                   |
|                                   | Code 'B' into Env: field, 'ISS'   |
|                                   | in the System: field and enter    |
|                                   | the business record using its     |
|                                   | Customer Reference Number (CRN).  |
|                                   | Conduct a search for the business |
|                                   | if its CRN is unknown.            |
+-----------------------------------+-----------------------------------+
| 3                                 | Access Org Characteristic         |
|                                   | Profile + Read more ...           |
|                                   |                                   |
|                                   | -   On the OSAS screen select ‘S’ |
|                                   |     against the CCS agreement     |
|                                   | -   On the OSAB screen select ‘Y’ |
|                                   |     against View Org              |
|                                   |     Characteristic Profile and    |
|                                   |     hit Enter                     |
|                                   | -   Select ‘S’ against the        |
|                                   |     appropriate Characteristic    |
|                                   |     Enquiry Group; Centrelink,    |
|                                   |     DVA or Superannuation         |
+-----------------------------------+-----------------------------------+
| 4                                 | Org Characteristic Market         |
|                                   | Sector + Read more ...            |
|                                   |                                   |
|                                   | -   As you are not creating a new |
|                                   |     Market Sector on the OCMS     |
|                                   |     screen hit Enter to take you  |
|                                   |     to the OCCD screen            |
+-----------------------------------+-----------------------------------+
| 5                                 | Org Characteristic Details + Read |
|                                   | more ...                          |
|                                   |                                   |
|                                   | -   If you are adding new         |
|                                   |     characteristics, on the OCCD  |
|                                   |     screen select ‘S’ against all |
|                                   |     existing characteristics the  |
|                                   |     business already has, as      |
|                                   |     indicated by a ‘Y’, as well   |
|                                   |     as selecting ‘S’ any new ones |
|                                   |     to be added.                  |
|                                   | -   If you are removing any old   |
|                                   |     characteristics, select all   |
|                                   |     existing characteristics that |
|                                   |     are to remain. Do not select  |
|                                   |     any characteristics that are  |
|                                   |     to be removed.                |
|                                   | -   Code Source ‘int’, DOR ‘t’,   |
|                                   |     and Action ‘C’. Finalise and  |
|                                   |     Confirm ‘Y’                   |
|                                   |                                   |
|                                   | Note: there are multiple pages of |
|                                   | characteristics on the CCS        |
|                                   | record. When updating             |
|                                   | characteristic profiles make sure |
|                                   | you have checked all pages.       |
+-----------------------------------+-----------------------------------+
| 6                                 | Org Work Unit Summary + Read more |
|                                   | ...                               |
|                                   |                                   |
|                                   | -   Within the same activity      |
|                                   |     navigate to the OWUS screen   |
|                                   |     and select ‘S’ the CCS Work   |
|                                   |     Unit                          |
|                                   | -   On the OWUD screen hit Enter  |
|                                   | -   Select ‘S’ against the        |
|                                   |     appropriate Characteristic    |
|                                   |     Enquiry Group; Centrelink,    |
|                                   |     DVA or Superannuation         |
|                                   | -   On the OCCD screen select ‘S’ |
|                                   |     against all existing          |
|                                   |     characteristics the business  |
|                                   |     already has, as indicated by  |
|                                   |     a ‘Y’, as well as any new     |
|                                   |     ones just added.              |
|                                   | -   Code Action ‘C’ and Confirm   |
|                                   |     ‘Y’                           |
|                                   | -   To complete code in the Nxt   |
|                                   |     field ‘AR’                    |
+-----------------------------------+-----------------------------------+
| 7                                 | Notify the business + Read more   |
|                                   | ...                               |
|                                   |                                   |
|                                   | -   The business can be notified  |
|                                   |     by email that the request to  |
|                                   |     add or remove characteristics |
|                                   |     is complete                   |
|                                   |                                   |
|                                   | DOC the record                    |
