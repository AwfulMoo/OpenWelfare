+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | When handover is required + Read  |
|                                   | more ...                          |
|                                   |                                   |
|                                   | Once a claim has been coded and   |
|                                   | Assess has been selected in       |
|                                   | Process Direct an ISIS activity   |
|                                   | is created and is visible on the  |
|                                   | Activity List (AL) screen in      |
|                                   | Customer First or Customer        |
|                                   | Record. This is linked to the SAP |
|                                   | claim activity in Process Direct. |
|                                   |                                   |
|                                   | Sometimes a claim activity may    |
|                                   | need to be processed in Customer  |
|                                   | First/Customer Record and the     |
|                                   | Handover to CF/CR function may be |
|                                   | required. Generally, this is to   |
|                                   | apply an approved workaround for  |
|                                   | a known issue/error or gaps in    |
|                                   | Process Direct.                   |
|                                   |                                   |
|                                   | Roxy Digital Assistant must be    |
|                                   | used before completing the        |
|                                   | Handover function from Process    |
|                                   | Direct to Customer First/Customer |
|                                   | Record where an error has         |
|                                   | occurred. This allows underlying  |
|                                   | issues to be investigated and     |
|                                   | addressed. Handover to CF/CR      |
|                                   | should only be used when          |
|                                   | directed, for example, by an      |
|                                   | approved workaround for a current |
|                                   | issue or when instructed by ICT.  |
|                                   |                                   |
|                                   | -   To complete a handover from   |
|                                   |     Process Direct to Customer    |
|                                   |     First/Customer Record, go to  |
|                                   |     Step 2                        |
|                                   | -   To complete a handover in     |
|                                   |     Customer Record/Customer      |
|                                   |     First back to Process Direct, |
|                                   |     go to Step 3                  |
|                                   | -   To index a claim in Customer  |
|                                   |     Record or Customer First      |
|                                   |     where there is an existing    |
|                                   |     SAP claim activity in Process |
|                                   |     Direct, go to Step 4          |

|                                   | Customer First/Customer Record +  |
|                                   | Read more ...                     |
|                                   |                                   |

|                                   | Customer First/Customer record    |

|                                   | gaps):                            |
|                                   |                                   |
|                                   | Use Roxy Digital Assistant to     |
|                                   | raise an incident                 |
|                                   |                                   |
|                                   | Select [more options] > Handover  |
|                                   | to CF/CR                          |
|                                   |                                   |
|                                   | On the Change Status screen,      |
|                                   | select the applicable Reason:     |
|                                   |                                   |
|                                   | -   ACC - No Access to PD. This   |
|                                   |     reason should only be used by |
|                                   |     staff who have selected the   |
|                                   |     activity in Customer          |
|                                   |     First/Customer record and     |
|                                   |     have not been trained in      |
|                                   |     Process Direct                |

|                                   |     error. This reason should be  |
|                                   |     selected when the Service     |

|                                   |     not yet been released in      |
|                                   |     Process Direct                |

|                                   |     released in Process Direct    |

|                                   |                                   |
|                                   | In Customer First/Customer        |

|                                   | If a claim activity is selected   |

|                                   | process in Process Direct:        |

|                                   | In Customer First/Customer        |

|                                   | screen, key 'C' against the       |

|                                   |                                   |
|                                   | On the Activity Cancel (AYC)      |
|                                   | screen, select 'C' to cancel all  |
|                                   | provisional changes               |
|                                   |                                   |
|                                   | Enter the reason to hand the      |
|                                   | activity back to Process Direct   |
|                                   | and cancel the started ISIS       |
|                                   | activity. Key '?' if unsure:      |
|                                   |                                   |
|                                   | -   NLR - No Longer Required      |
|                                   |     Cancels both Process Direct   |
|                                   |     and Customer Record/Customer  |
|                                   |     First claims                  |

|                                   |     when the claim is no longer   |
|                                   |     required. This option will    |

|                                   |     remove the claim from the     |
|                                   |     Customer's Online View        |
|                                   | -   PPD - Process in PD           |
|                                   |     Cancels the Customer          |
|                                   |     Record/Customer First claim   |
|                                   |     Returns the claim to Process  |
|                                   |     Direct to be processed        |

|                                   |     Cancels the Customer          |
|                                   |     Record/Customer First claim   |

|                                   |     Record/Customer First         |
|                                   |     All provisional data must be  |
|                                   |     re-coded                      |
|                                   | -   TKL - Task Locking            |
|                                   |     Cancels the Customer          |
|                                   |     Record/Customer First claim   |
|                                   |     All provisional data must be  |
|                                   |     re-coded                      |
|                                   |                                   |
|                                   | Upon selecting the cancellation   |
|                                   | reason, press [Enter]. The ISIS   |

|                                   | now be processed in Process       |
|                                   | Direct                            |
|                                   |                                   |
|                                   | Re-code in Process Direct:        |
|                                   |                                   |
|                                   | -   provisional data that was     |
|                                   |     coded in the claim activity   |
|                                   | -   coding completed in Customer  |
|                                   |     First/Customer Record before  |
|                                   |     the claim was cancelled       |
|                                   |                                   |

Record and transaction locking

Table 4

| Item                              | Description                       |
+-----------------------------------+-----------------------------------+
| 1                                 | Record locking + Read more ...    |
|                                   |                                   |
|                                   | Record locking occurs if another  |
|                                   | Service Officer is in a           |
|                                   | customer’s record. A Record       |
|                                   | Access Warning Message with the   |
|                                   | user log on ID will appear at the |
|                                   | top left hand side of the screen. |
|                                   |                                   |
|                                   | -   This is only a warning        |
|                                   |     message, Service Officers can |
|                                   |     still view and update the     |
|                                   |     customer record               |
|                                   | -   This allows multiple users    |
|                                   |     (such as Service Officers,    |
|                                   |     the system via automation, or |
|                                   |     the customer through online   |
|                                   |     services) to access and       |
|                                   |     update a record at the same   |
|                                   |     time                          |
+-----------------------------------+-----------------------------------+
| 2                                 | Transaction locking + Read more   |
|                                   | ...                               |
|                                   |                                   |
|                                   | When transaction locking occurs a |
|                                   | Service Officer is not able to    |
|                                   | update or assess the same         |
|                                   | transaction another Service       |
|                                   | Officer is currently processing.  |
|                                   | The transaction will be locked by |
|                                   | the first Service Officer.        |
|                                   |                                   |
|                                   | If a Service Officer accesses a   |
|                                   | locked transaction:               |
|                                   |                                   |
|                                   | -   A Transaction Error Message   |
|                                   |     with the user log on ID will  |
|                                   |     appear at the top left hand   |
|                                   |     side of the screen. The rest  |
|                                   |     of the record can still be    |
|                                   |     accessed and viewed           |
|                                   | -   The transaction cannot be     |
|                                   |     updated or finalised.         |
|                                   |     Functions like Assess, Add,   |
|                                   |     Edit and Delete will be       |
|                                   |     disabled                      |
