Examples of action by a Debt Management Officer (DMO)

+-----------------------------------+-----------------------------------+
| <Category title>                  | <Description>                     |
+-----------------------------------+-----------------------------------+
| Adjusting debt repayments using   | An Authorised Review Officer      |
| the Debt Transfer Details         | (ARO) advises a DMO to implement  |
|                                   | a decision to waive a finalised   |
| (OPRTD) screen.                   | debt and refund receipted         |
|                                   | repayments. The waiver cannot be  |
| Example                           | actioned unless the debt is       |
|                                   | Determined.                       |
|                                   |                                   |
|                                   | -   The DMO will use Table 5 on   |
|                                   |     the Process page of Refunding |
|                                   |     debt repayments               |
|                                   | -   The DMO will start the refund |
|                                   |     to reduce a repayment down by |
|                                   |     $0.01 using the OPRTD screen  |
|                                   | -   This will return the debt to  |
|                                   |     Determined and allow the      |
|                                   |     waiver to be actioned         |
|                                   | -   After the waiver is actioned, |
|                                   |     the DMO will continue the     |
|                                   |     refund using the OPRTD screen |
|                                   |     for the remaining over        |
|                                   |     recovered amount of $144.99   |
|                                   | -   The OPRH screen will now      |
|                                   |     display two negative          |
|                                   |     transactions, one for -$0.01c |
|                                   |     and one for -$144.99          |
|                                   | -   The DMO will complete the     |
|                                   |     refund to pay the full amount |
|                                   |     of $145 using the PYAJ screen |
+-----------------------------------+-----------------------------------+
| Adjusting debt repayments using   | An Authorised Review Officer      |
| the Adjust Repayment History      | (ARO) advises a DMO to implement  |
| (OPAH) screen.                    | a decision to waive a finalised   |
|                                   | debt and transfer or refund       |
| Example 1                         | receipted repayments. The waiver  |
|                                   | cannot be actioned unless the     |
|                                   | debt is Determined:               |
|                                   |                                   |
|                                   | -   The DMO will adjust a         |
|                                   |     repayment on the finalised    |
|                                   |     debt down by $0.01 using the  |
|                                   |     OPAH screen. ($145.00         |
|                                   |     repayment to $144.99)         |
|                                   | -   This will return the debt to  |
|                                   |     Determined and allow the      |
|                                   |     waiver to be actioned         |
|                                   | -   After the waiver is actioned, |
|                                   |     the DMO will reverse the      |
|                                   |     $0.01 adjustment using the    |
|                                   |     OPAH screen                   |
|                                   | -   The DMO will now be able to   |
|                                   |     transfer or refund the        |
|                                   |     correct amount                |
+-----------------------------------+-----------------------------------+
| Adjusting debt repayments using   | Family Tax Benefit (FTB) arrears  |
| the Adjust Repayment History      | incorrectly generates. This pays  |
| (OPAH) screen.                    | $2,500 to the customer’s bank     |
|                                   | account and allocates $2,500 to   |
| Example 2                         | the customer’s debt.              |
|                                   |                                   |
|                                   | ICT process a Stop Payment to     |
|                                   | block the $2,500 from paying to   |
|                                   | the bank account. This does not   |
|                                   | stop the $2,500 from allocating   |
|                                   | to the debt.                      |
|                                   |                                   |
|                                   | The DMO will adjust the incorrect |
|                                   | FTB arrears repayment to $0.00    |
|                                   | using the OPAH screen.            |
+-----------------------------------+-----------------------------------+
| Adding debt repayments using the  | Services Australia has an         |
| Add Repayment (OPAR) screen.      | agreement with the Department of  |
|                                   | Veterans' Affairs (DVA). DVA may  |
| Example 1                         | withhold from the customer’s DVA  |
|                                   | payment to repay a Centrelink     |
|                                   | debt.                             |
|                                   |                                   |
|                                   | This repayment will credit        |
|                                   | automatically. Alternatively, a   |
|                                   | DMO will manually credit the      |
|                                   | repayment via the Add Repayment   |
|                                   | (OPAR) screen. Workforce          |
|                                   | Strategy, Analytics & Reporting   |
|                                   | (WSAR) will allocate this Work    |
|                                   | Item, along with additional       |
|                                   | instruction.                      |
+-----------------------------------+-----------------------------------+
| Adding debt repayments using the  | A DMO incorrectly refunds an over |
| Add Repayment (OPAR) screen.      | recovered amount and intends to   |
| Example 2                         | reissue the refund correctly. ICT |
|                                   | will stop the refund and adjust   |
|                                   | the PS screen.                    |
|                                   |                                   |
|                                   | An Operational Leader with the    |
|                                   | required access will add the over |
|                                   | recovered repayment(s) to the     |
|                                   | same debt via the Add Repayment   |
|                                   | (OPAR) screen. This action will   |
|                                   | return the debt balance back to   |
|                                   | over recovered. The DMO will then |
|                                   | repeat the refund process, coding |
|                                   | Debt Transfer Details (OPRTD)     |
|                                   | screen and Payment adjust (PYAJ)  |
|                                   | screen again.                     |
