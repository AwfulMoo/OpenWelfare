Action Australian Organ Donor Register (AODR) emails

Table 1

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Assess enquiry + Read more ...    |
|                                   |                                   |
|                                   | Open PaNDA, when receiving an     |
|                                   | email enquiry to assess if the    |
|                                   | request is:                       |
|                                   |                                   |
|                                   | -   the first email from a        |
|                                   |     sender, and                   |
|                                   | -   a template response is        |
|                                   |     required, or                  |
|                                   | -   a phone call                  |
|                                   |                                   |
|                                   | A non-standard response includes  |
|                                   | where amendments are required,    |
|                                   | and the use of multiple templates |
|                                   | in a reply.                       |
|                                   |                                   |
|                                   | -   If a template response is     |
|                                   |     required, go to Step 3        |
|                                   | -   If a phone call is possible   |
|                                   |     and more appropriate, go to   |
|                                   |     Table 2                       |
|                                   | -   If a phone call is            |
|                                   |     unsuccessful or unable to be  |
|                                   |     actioned, go to Step 2        |
+-----------------------------------+-----------------------------------+
| 2                                 | Escalate a non-standard reply to  |
|                                   | a PSO via the PSO referral form + |
|                                   | Read more ...                     |
|                                   |                                   |
|                                   | Enter comments in the PaNDA work  |
|                                   | item and save the status as       |
|                                   | ‘pended’ with the appropriate     |
|                                   | Pend Reason, for example: Program |
|                                   | Area Escalation.                  |
|                                   |                                   |
|                                   | Once a response is received, go   |
|                                   | to Step 3                         |
+-----------------------------------+-----------------------------------+
| 3                                 | Select Reply to email + Read more |
|                                   | ...                               |
|                                   |                                   |
|                                   | Go to the General Response folder |
|                                   | in the mailbox folders, select an |
|                                   | appropriate response from the     |
|                                   | templates available.              |
|                                   |                                   |
|                                   | Copy the relevant template or     |
|                                   | approved non-standard response    |
|                                   | and paste the text into the start |
|                                   | of the reply email.               |
|                                   |                                   |
|                                   | -   Remove any personal details   |
|                                   |     from both the body and        |
|                                   |     subject line of the email     |
|                                   | -   Adjust all required details,  |
|                                   |     such as: Name of the person   |
|                                   |     who sent the enquiry          |
|                                   |                                   |
|                                   | Note: remove any additional text  |
|                                   | that does not apply to the        |
|                                   | enquiry. This is indicated by the |
|                                   | ***asterisk*** surrounding the    |
|                                   | text from the template.           |
|                                   |                                   |
|                                   | See the Resources page for AODR   |
|                                   | PSO contact details for           |
|                                   | escalation, if you are unsure     |
|                                   | which template to use.            |
+-----------------------------------+-----------------------------------+
| 4                                 | Complete checks before sending +  |
|                                   | Read more ...                     |
|                                   |                                   |
|                                   | -   Complete a spellcheck located |
|                                   |     on the Review tool bar        |
|                                   | -   All emails are to be font -   |
|                                   |     Arial 11                      |
|                                   | -   Ensure the ‘From’ is the      |
|                                   |     public email box and not your |
|                                   |     personal email address (refer |
|                                   |     to your Team Leader or PSO    |
|                                   |     for assistance)               |
|                                   | -   Address the email to the name |
|                                   |     as per the email (if the      |
|                                   |     email is signed 'Regards      |
|                                   |     Toby' and CDMS is 'Tobias',   |
|                                   |     use 'Toby')                   |
|                                   | -   If the email is from a        |
|                                   |     Medical Professional, their   |
|                                   |     Title should be added         |
|                                   | -   All email templates include   |
|                                   |     an approved AODR signature -  |
|                                   |     ‘Regards, Australian Organ    |
|                                   |     Donor Register’. Do not       |
|                                   |     include your name, details or |
|                                   |     personal signature block      |
|                                   | -   Check the email address       |
|                                   |     matches the sender prior to   |
|                                   |     responding                    |
|                                   | -   Use the Privacy template      |
|                                   |     where details cannot be       |
|                                   |     matched to the email address  |
|                                   | -   All manual fields within the  |
|                                   |     email body are amended, such  |
|                                   |     as date of email received     |
|                                   | -   Check if ‘Date of last        |
|                                   |     contact’ and comments are     |
|                                   |     required in CDMS and action   |
|                                   |     if required                   |
+-----------------------------------+-----------------------------------+
| 5                                 | Finalise + Read more ...          |
|                                   |                                   |
|                                   | -   Send the reply using email    |
|                                   |     classification ‘Unclassified’ |
|                                   | -   Place the sent email into the |
|                                   |     sent folder in the mailbox.   |
|                                   |     Original emails should be     |
|                                   |     kept in the inbox.            |
|                                   | -   Enter relevant CDMS inbound   |
|                                   |     entry and relevant comments   |
|                                   |     including email response sent |
|                                   | -   Enter comments in the PANDA   |
|                                   |     work item and finalise the    |
|                                   |     work item in PaNDA            |
|                                   | -   See Processing rules for      |
|                                   |     Australian Organ Donor        |
|                                   |     Register (AODR)               |
|                                   |                                   |
|                                   | Note: CDMS comments must be       |
|                                   | prefixed with ‘AODR’              |
+-----------------------------------+-----------------------------------+

Action response to an email enquiry via phone

Table 2

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Phone contact + Read more ...     |
|                                   |                                   |
|                                   | Contact the customer via their    |
|                                   | phone contact information in the  |
|                                   | email or CDMS                     |
+-----------------------------------+-----------------------------------+
| 2                                 | Update CDMS + Read more ...       |
|                                   |                                   |
|                                   | Add a comment to email enquiry    |
|                                   | document in PaNDA stating, ‘Reply |
|                                   | has been sent/PSO advised a call  |
|                                   | is to be made/contacted via phone |
|                                   | DDMMYYYY. To edit an original     |
|                                   | email: Not required. Dealing with |
|                                   | email by phone call and adding    |
|                                   | comments on document on PaNDA.    |
|                                   |                                   |
|                                   | -   Action any required CDMS      |
|                                   |     entries including the inbound |
|                                   |     email contact                 |
|                                   | -   Enter CDMS comments including |
|                                   |     outbound phone call reply and |
|                                   |     relevant information          |
|                                   | -   Go to Step 3                  |
|                                   |                                   |
|                                   | Note: CDMS comments must be       |
|                                   | prefixed with ‘AODR’              |
+-----------------------------------+-----------------------------------+
| 3                                 | Record information in Comments    |
|                                   | field + Read more ...             |
|                                   |                                   |
|                                   | Information entered into the      |
|                                   | Comments field is visible to      |
|                                   | Authorised Medical Personnel      |
|                                   | (AMP) who have access to the AODR |
|                                   | On-Line Enquiry System (PRODA).   |
|                                   | As a result, SO need to ensure    |
|                                   | that when an organ donor          |
|                                   | registrant supplies information   |
|                                   | to the AODR, only appropriate     |
|                                   | details are entered into the      |
|                                   | Comments field.                   |
|                                   |                                   |
|                                   | Comments may:                     |
|                                   |                                   |
|                                   | -   be obtained through Freedom   |
|                                   |     of Information (FOI) requests |
|                                   | -   must be entered via the AODR  |
|                                   |     comments tab or Medicare      |
|                                   |     Personal Comments tab where   |
|                                   |     no AODR registration exists   |
|                                   |                                   |
|                                   | Once a date of contact and or     |
|                                   | decision date has been updated in |
|                                   | an AODR registration, it cannot   |
|                                   | be reversed or backdated. In this |
|                                   | circumstance a comment can be     |
|                                   | placed to note the                |
|                                   | original/correct date for this    |
|                                   | purpose only (no processing       |
|                                   | comments for normal business):    |
|                                   |                                   |
|                                   | Example: ‘AODR – the correct Most |
|                                   | Recent Donation Decision date and |
|                                   | the Date of last contact, the     |
|                                   | form processed on DDMMYYYY is     |
|                                   | actually processed on this date   |
|                                   | DDMMYYYY- unable to correct.’     |
|                                   |                                   |
|                                   | Note: Entering this comment may   |
|                                   | be important in the event that an |
|                                   | AMP calls or completes an online  |
|                                   | register, check to confirm the    |
|                                   | consumer’s last decision and      |
|                                   | decision date with the register.  |
|                                   |                                   |
|                                   | All comments in relation to AODR  |
|                                   | must be entered with the prefix   |
|                                   | of ‘AODR’ as per policy and       |
|                                   | Medicare agreements to            |
|                                   | distinguish comments from a       |
|                                   | Medicare or other comment.        |
|                                   |                                   |
|                                   | See Processing rules for          |
|                                   | Australian Organ Donor Register   |
|                                   | (AODR)                            |
|                                   |                                   |
|                                   | Got to Step 4                     |
+-----------------------------------+-----------------------------------+
| 4                                 | Finalise + Read more ...          |
|                                   |                                   |
|                                   | -   Check all actions are         |
|                                   |     completed                     |
|                                   | -   Enter comments in the PaNDA   |
|                                   |     work item including action    |
|                                   |     taken                         |
|                                   | -   Finalise the work item in     |
|                                   |     PaNDA                         |
|                                   | -   Leave original email in the   |
|                                   |     Inbox                         |
|                                   | -   Process ends here             |
+-----------------------------------+-----------------------------------+
