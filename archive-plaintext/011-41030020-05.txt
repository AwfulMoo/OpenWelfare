This page contains Assessing and Benefits contact details, a description
of the modifiers for the Patient History 'NHSI' screen on Mainframe for
the Child Dental Benefits Schedule (CDBS), how to handle requests for
Medicare card details from Child CDBS patients and dental providers and
links to the Medicare stationery order form page and Child Dental
Benefits Schedule page. Links on this page may be unavailable to
external audiences.

Assessing and Benefits

CDBS dental modifier in Medicare patient history

Table 1: This table describes the modifier for the Patient History
'NHSI' screen on Mainframe for the Child Dental Benefits Schedule
(CDBS).

+-----------------------------------+-----------------------------------+
| No.                               | NHSI screen modifier and          |
|                                   | description                       |
+-----------------------------------+-----------------------------------+
| 1                                 | NHSI, card number, patient,       |
|                                   | DENT/ALL                          |
|                                   |                                   |
|                                   | Filters and displays all CDBS     |
|                                   | items which have been processed.  |
+-----------------------------------+-----------------------------------+

Handling requests for Medicare card details from CDBS patients and dental providers

Table 2: This table describes how to handle requests for Medicare card
details from the Child Dental Benefits Schedule (CDBS) patients and
dental providers.

+-----------------------------------+-----------------------------------+
| No.                               | Requestor and action              |
+-----------------------------------+-----------------------------------+
| 1                                 | Patient or patient’s              |
|                                   | parent/guardian                   |
|                                   |                                   |
|                                   | A Medicare card number can be     |
|                                   | released to a person identified   |
|                                   | as the CDBS patient or patient’s  |
|                                   | parent/guardian.                  |
|                                   |                                   |
|                                   | Perform a Public security check   |
|                                   | on the person.                    |
|                                   |                                   |
|                                   | Note: if the person is the        |
|                                   | claimant but is not the patient   |
|                                   | or patient's parent/guardian,     |
|                                   | they must also provide documents  |
|                                   | to show the child is in their     |
|                                   | care, for example, court orders.  |
+-----------------------------------+-----------------------------------+
| 2                                 | Dental provider                   |
|                                   |                                   |
|                                   | Medicare card numbers can be      |
|                                   | released over the telephone to    |
|                                   | dental providers when both of the |
|                                   | following conditions are met:     |
|                                   |                                   |
|                                   | -   the Medicare card number must |
|                                   |     be for a patient who is       |
|                                   |     receiving services from the   |
|                                   |     provider                      |
|                                   | -   the caller passes a provider  |
|                                   |     security check                |
+-----------------------------------+-----------------------------------+
