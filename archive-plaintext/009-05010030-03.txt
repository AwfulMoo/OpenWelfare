For Carer Payment /Carer Allowance New Claims Smart Centre Processing
staff only

This page contains information to help Service Officers assess and
record claims for Carer Allowance (CA) for 2 disabled children when
neither attracts CA in their own right.

Assessing CA claims

Coding CA claims

Assessing CA claims

Table 1: This table describes the process to help staff assess claims
for CA for 2 disabled children when neither attracts CA in their own
right.

| Step                              | Action                            |

| 1                                 | CA medical report + Read more ... |
|                                   |                                   |

|                                   | (SA333TDR/SA426) form been        |

|                                   |                                   |

|                                   | -   No, processing cannot         |
|                                   |     continue until all the forms  |
|                                   |     are provided. Procedure ends  |
|                                   |     here                          |
+-----------------------------------+-----------------------------------+
| 2                                 | Recognised disability + Read more |
|                                   | ...                               |

|                                   | []                                |

|                                   |                                   |

|                                   | -   No, go to Step 4              |
+-----------------------------------+-----------------------------------+
| 3                                 | Medically qualified to attract    |
|                                   | full CA + Read more ...           |
|                                   |                                   |

Coding CA claims

Table 2: This table describes the process of coding paper claims for CA
for 2 disabled children when neither attracts CA in their own right.

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Select activity + Read more ...   |
|                                   |                                   |
|                                   | []                                |
|                                   |                                   |
|                                   | Select the activity from the      |
|                                   | Activity List (AL) screen (if the |
|                                   | review has become due) or the     |
|                                   | Future Activity List (FAL) screen |
|                                   | if it has not yet become due:     |
|                                   |                                   |
|                                   | -   in the Carer Allowance Task   |
|                                   |     Selector (CATS) screen,       |
|                                   |     select the Care Receiver      |
|                                   |     Summary (CRS) screen and      |
|                                   |     press [Enter]                 |
|                                   | -   select the relevant care      |
|                                   |     receiver to view the Care     |
|                                   |     Receiver Task Selector (CETS) |
|                                   |     screen. Select the Disability |
|                                   |     Assessment Tool to view the   |
|                                   |     Child Disability Assessment   |
|                                   |     Tool Summary (CDAS) screen    |
|                                   | -   select the current Event Date |
|                                   |     to display the Child Medical  |
|                                   |     Details Screen (CDMD). This   |
|                                   |     screen has already been       |
|                                   |     completed before starting     |
|                                   |     this procedure                |
|                                   | -   press [Enter] to view the     |
|                                   |     Treating Health Professional  |
|                                   |     Assessment (CDTT) screen      |
|                                   | -   Press [Enter] to view the     |
|                                   |     Carer Assessment (CDTC)       |
|                                   |     screen. Press [Enter]         |
+-----------------------------------+-----------------------------------+
| 2                                 | Complete Carer Assessment (CDTC)  |
|                                   | screen + Read more ...            |
|                                   |                                   |
|                                   | []                                |
|                                   |                                   |
|                                   | Code the relevant fields:         |
|                                   |                                   |
|                                   | Care Rcvr, CRN, Birth Date. If    |
|                                   | the care receiver is:             |
|                                   |                                   |
|                                   | -   not a customer in their own   |
|                                   |     right, the CDTC screen will   |
|                                   |     display these fields from     |
|                                   |     within the carer's record     |
|                                   | -   is a customer in their own    |
|                                   |     right, these details will be  |
|                                   |     displayed in the CDTC screen  |
|                                   |     header only                   |
|                                   |                                   |
|                                   | Age Milestone is a protected      |
|                                   | field calculated by comparing the |
|                                   | child's date of birth with the    |
|                                   | date signed                       |
|                                   |                                   |
|                                   | Adjusted in some circumstances,   |
|                                   | the DCLAD (2010) score is         |
|                                   | adjusted to the next milestone    |
|                                   | age. If this 'cusp adjustment'    |
|                                   | occurs, the screen will display   |
|                                   | Adjusted                          |
|                                   |                                   |
|                                   | Score is protected and represents |
|                                   | the system calculated Child       |
|                                   | Disability Assessment score       |
|                                   | associated with the assessment    |
|                                   | provided by the THP               |
|                                   |                                   |
|                                   | Initial DOR is protected and      |
|                                   | displays the date of receipt of   |
|                                   | the first activity used to enter  |
|                                   | details of the DCLAD (2010)       |
|                                   | assessment and is defaulted from  |
|                                   | the Initial DOR field on the CDMD |
|                                   | screen                            |
|                                   |                                   |
|                                   | Effect Date is protected and      |
|                                   | shows the date from which the     |
|                                   | medical information takes effect. |
|                                   | Itis defaulted from the CDMD      |
|                                   | screen. If carer assessment       |
|                                   | details are keyed first, the date |
|                                   | in the Initial DOR field will     |
|                                   | default to the Effect Date field  |
|                                   |                                   |
|                                   | Date signed is to record the date |
|                                   | the carer signed the assessment   |
|                                   | or if it was not dated, the date  |
|                                   | of receipt                        |
|                                   |                                   |
|                                   | Date Returned the date the carer  |
|                                   | returned the carer assessment     |
|                                   |                                   |
|                                   | Questions 1 - 10 are used to      |
|                                   | input the responses received from |
|                                   | the carer. Use field help ('?')   |
|                                   | for relevant entries. Questions   |
|                                   | 1-8 are mandatory. For Questions  |
|                                   | 9 and 10, the carer can mark as   |
|                                   | many statements as they want to   |
|                                   | describe the child's behaviour.   |
|                                   | If none of the statements are     |
|                                   | relevant to this child, Questions |
|                                   | 9 and 10 do not need to be        |
|                                   | answered                          |
|                                   |                                   |
|                                   | Source, DOR and Action in the     |
|                                   | footer. The Action field is       |
|                                   | optional and will accept 'I' for  |
|                                   | Insert or 'C' for Correct on this |
|                                   | screen                            |
|                                   |                                   |
|                                   | Press [Enter]. The CDTC screen    |
|                                   | will redisplay with the score for |
|                                   | each individual question and the  |
|                                   | total carer score.                |
+-----------------------------------+-----------------------------------+
| 3                                 | Screen selection + Read more ...  |
|                                   |                                   |
|                                   | []                                |
|                                   |                                   |
|                                   | If the screen displays as:        |
|                                   |                                   |
|                                   | -   Care Load Validation (CLVL),  |
|                                   |     go to Step 4                  |
|                                   | -   Child Disability Assessment   |
|                                   |     Tool Summary (CDAS), go to    |
|                                   |     Step 5                        |
+-----------------------------------+-----------------------------------+
| 4                                 | CLVL screen + Read more ...       |
|                                   |                                   |
|                                   | []                                |
|                                   |                                   |
|                                   | If the CLVL screen is displayed,  |
|                                   | validation needs to occur.        |
|                                   |                                   |
|                                   | For care receivers under 3 years  |
|                                   | of age, a workaround is required. |
|                                   |                                   |
|                                   | The DCLAD questions from 'Domain  |
|                                   | B - Behaviour 2' are questions 9  |
|                                   | to 20 on the PDF version of the   |
|                                   | Care Needs Assessment (CNA).      |
|                                   | These questions are not           |
|                                   | applicable to care receivers      |
|                                   | under 3 years of age.             |
|                                   |                                   |
|                                   | If validation is triggered for a  |
|                                   | care receiver under three years   |
|                                   | of age, the following questions,  |
|                                   | from 'Domain B - Behaviour 2'     |
|                                   | will incorrectly display on the   |
|                                   | CLVL screen:                      |
|                                   |                                   |
|                                   | -   Q11 - The child wanders,      |
|                                   |     escapes or runs away from     |
|                                   |     home, school or community     |
|                                   |     settings                      |
|                                   | -   Q13 - The child becomes       |
|                                   |     physically aggressive and     |
|                                   |     harms others by hitting,      |
|                                   |     pushing, kicking, biting or   |
|                                   |     throwing objects              |
|                                   | -   Q18 - The child behaves in    |
|                                   |     ways that result in injury to |
|                                   |     himself/herself. This may     |
|                                   |     include head-banging,         |
|                                   |     hand-biting or other          |
|                                   |     self-harming behaviours       |
|                                   |                                   |
|                                   | Where validation is triggered for |
|                                   | a care receiver under three years |
|                                   | of age the value 'a' should be    |
|                                   | recorded in the CNA field against |
|                                   | the above questions.              |
|                                   |                                   |
|                                   | Document the validation entries   |
|                                   | made to 1 or all of these         |
|                                   | questions and record that they    |
|                                   | are a workaround to address       |
|                                   | incorrect functionality of the    |
|                                   | validation process.               |
|                                   |                                   |
|                                   | This means there are              |
|                                   | discrepancies between the         |
|                                   | responses provided in the THP     |
|                                   | report and CNA form, and a        |
|                                   | validation is required. Fields on |
|                                   | this screen not requiring         |
|                                   | validation will be protected.     |
|                                   | Fields requiring validation will  |
|                                   | be unprotected and will display   |
|                                   | the THP and CNA responses that    |
|                                   | need to be checked:               |
|                                   |                                   |
|                                   | -   use field help (?) in the CNA |
|                                   |     or THP action field beside    |
|                                   |     the validation question for   |
|                                   |     help on what to do to         |
|                                   |     validate the assessment. If   |
|                                   |     necessary, this will be       |
|                                   |     contacting the carer and/or   |
|                                   |     THP, and/or a second THP to   |
|                                   |     check their responses         |
|                                   | -   update each response via the  |
|                                   |     CNA or THP fields on the CLVL |
|                                   |     screen. These updated         |
|                                   |     responses will flow through   |
|                                   |     to the appropriate CDAT       |
|                                   |     screens without the user      |
|                                   |     having to navigate away from  |
|                                   |     CLVL                          |
|                                   | -   once responses have been      |
|                                   |     validated with the THP and/or |
|                                   |     carer and updated in the CNA  |
|                                   |     and/or THP fields, the        |
|                                   |     Validated? field should be    |
|                                   |     completed. Use field help (?) |
|                                   |     for applicable codes          |
|                                   | -   once the Validation screen is |
|                                   |     completed, press [Enter] to   |
|                                   |     display the new scores        |
|                                   |     attributed after validation   |
|                                   |     on the Child Disability       |
|                                   |     Assessment Tool Summary       |
|                                   |     (CDAS) screen                 |
|                                   | -   the Care Receiver Assessment  |
|                                   |     Results (CJAR) screen should  |
|                                   |     then display with the         |
|                                   |     assessment result             |
|                                   |                                   |
|                                   | Validation occurs in certain      |
|                                   | circumstances which are clearly   |
|                                   | outlined in the Social Security   |
|                                   | Guide, 3.6.12.320, Validation of  |
|                                   | the Disability Care Load          |
|                                   | Assessment (Child) - CP (child).  |
