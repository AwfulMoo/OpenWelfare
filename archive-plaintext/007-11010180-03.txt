Table 1: This table contains information about the automatic process of
FTB re-reconciliation when family circumstances are updated.

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Initial reconciliation            |
|                                   | completed + Read more ...         |
|                                   |                                   |
|                                   | After initial reconciliation of   |
|                                   | FTB has completed,                |
|                                   | re-reconciliation will be         |
|                                   | automatically triggered if the    |
|                                   | details used to calculate FTB     |
|                                   | entitlement for the relevant      |
|                                   | financial year are updated.       |
|                                   |                                   |
|                                   | Re-reconciliation automatically   |
|                                   | compares the customer's           |
|                                   | entitlement based on the new      |
|                                   | information with the earlier      |
|                                   | reconciliation result.            |
|                                   |                                   |
|                                   | Re-reconciliation will occur if,  |
|                                   | for example:                      |
|                                   |                                   |
|                                   | -   Centrelink receives new       |
|                                   |     actual adjusted taxable       |
|                                   |     income details from the       |
|                                   |     Australian Taxation Office    |
|                                   |     (ATO). The ATO link ends      |
|                                   |     after three years unless the  |
|                                   |     customer has an outstanding   |
|                                   |     FTB non-lodger debt. If       |
|                                   |     actual income details are     |
|                                   |     required for a year for which |
|                                   |     the link has ended, for       |
|                                   |     example, after 30 June 2015   |
|                                   |     for 2011-12, the customer     |
|                                   |     will need to provide their    |
|                                   |     Notice of Assessment (NOA)    |
|                                   |     and tax return (if            |
|                                   |     available). See Mutual        |
|                                   |     Customer Identification (MCI) |
|                                   |     process for family assistance |
|                                   |     customers                     |
|                                   | -   The customer revises their    |
|                                   |     estimate of taxable income    |
|                                   |     for the relevant financial    |
|                                   |     year                          |
|                                   | -   The customer advises their    |
|                                   |     FTB deferred child's income   |
|                                   |     was below/ not below the      |
|                                   |     child income limit and this   |
|                                   |     is updated on the Child       |
|                                   |     Student/Income Circumstances  |
|                                   |     (CHSI) screen                 |
|                                   | -   The customer advises of a     |
|                                   |     return to work after caring   |
|                                   |     for a child during the        |
|                                   |     relevant financial year. See  |
|                                   |     Family Tax Benefit (FTB) Part |
|                                   |     B quarantine and the          |
|                                   |     reconciliation process        |
|                                   | -   A non-lodger exemption reason |
|                                   |     is coded for the current      |
|                                   |     partner or ex-partner by the  |
|                                   |     end of the lodgement year     |
|                                   | -   A customer and/or their       |
|                                   |     partner lodges a tax return   |
|                                   |     having previously advised     |
|                                   |     they were not required to     |
|                                   |     lodge a tax return, or having |
|                                   |     been treated as exempt from   |
|                                   |     lodging                       |
|                                   | -   A circumstance change with a  |
|                                   |     date of effect in the         |
|                                   |     relevant financial year is    |
|                                   |     recorded (for example, child  |
|                                   |     left care)                    |
|                                   | -   A retrospective change        |
|                                   |     (increase or decrease) to a   |
|                                   |     child support assessment for  |
|                                   |     a private collect customer is |
|                                   |     received from Child Support   |
|                                   | -   There is a retrospective      |
|                                   |     change (increase or decrease) |
|                                   |     to a child support assessment |
|                                   |     for a child support collect   |
|                                   |     customer and the Maintenance  |
|                                   |     Income Credit (MIC) offset    |
|                                   |     amount changes                |
|                                   | -   There is a retrospective      |
|                                   |     change in a child's           |
|                                   |     immunisation or health check  |
|                                   |     status which results in a     |
|                                   |     reassessment of the FTB Part  |
|                                   |     A rate reduction              |
|                                   | -   For periods after 1 July      |
|                                   |     2018, there is a              |
|                                   |     retrospective change to a     |
|                                   |     customer's Income Support     |
|                                   |     Payment status, this could    |
|                                   |     include removal of ISP        |
|                                   |     eligibility or retrospective  |
|                                   |     calculation of a nil rate     |
|                                   |     period due to employment      |
|                                   |     income. Both scenarios would  |
|                                   |     result in the customer being  |
|                                   |     subject to FTB Part A income  |
|                                   |     test                          |
|                                   |                                   |
|                                   | Customer disagrees with actual    |
|                                   | ATO income? A customer may advise |
|                                   | that their taxation Notice of     |
|                                   | Assessment (NOA) shows a          |
|                                   | different amount from that        |
|                                   | transferred by the ATO and shown  |
|                                   | on their customer record. This    |
|                                   | could be because the NOA only     |
|                                   | shows the customer's taxable      |
|                                   | income component.                 |
|                                   |                                   |
|                                   | Assessable income for Centrelink  |
|                                   | is the customer's adjusted        |
|                                   | taxable income (ATI), which       |
|                                   | includes other components such as |
|                                   | exempt reportable fringe          |
|                                   | benefits, other reportable fringe |
|                                   | benefits, reportable              |
|                                   | superannuation contributions and  |
|                                   | total net investment losses (for  |
|                                   | example) which are not displayed  |
|                                   | on a customer's tax NOA from the  |
|                                   | ATO.                              |
|                                   |                                   |
|                                   | Wrong actual income? If the       |
|                                   | taxable income on a customer's    |
|                                   | NOA shows a different taxable     |
|                                   | income amount from that           |
|                                   | transferred by the ATO and shown  |
|                                   | on the customer's record (for     |
|                                   | example, no link with the ATO,    |
|                                   | wrong Tax File Number recorded),  |
|                                   | there may be a mismatch with the  |
|                                   | Mutual Customer Identification    |
|                                   | (MCI) link. See Mutual Customer   |
|                                   | Identification (MCI) process for  |
|                                   | family assistance customers.      |
|                                   |                                   |
|                                   | The amounts could also differ as  |
|                                   | a result of the customer          |
|                                   | accessing superannuation funds    |
|                                   | under the First Home Super Saver  |
|                                   | (FHSS) Scheme. For these          |
|                                   | customers, income details         |
|                                   | transferred by the ATO will       |
|                                   | automatically have amounts        |
|                                   | released under the FHSS Scheme    |
|                                   | excluded, as these do not form    |
|                                   | part of a customer's adjusted     |
|                                   | taxable income (ATI). However,    |
|                                   | the taxable income amount on      |
|                                   | their NOA maybe higher than the   |
|                                   | amount reported by the ATO via    |
|                                   | the data link on our system as    |
|                                   | the ATO does regard a portion of  |
|                                   | the released funds as taxable.    |
|                                   |                                   |
|                                   | Zero entitlement? If the          |
|                                   | re-reconciliation result is that  |
|                                   | the customer had nil entitlement  |
|                                   | to FTB Part A and/or FTB Part B   |
|                                   | as actual income precluded        |
|                                   | entitlement and it is the second  |
|                                   | of two consecutive zero           |
|                                   | entitlement years from the        |
|                                   | 2009-10 financial year, the       |
|                                   | customer may be subject to the    |
|                                   | non-payment of FTB by             |
|                                   | instalments.                      |

FTB manual re-reconciliation

re-reconciliation when income circumstances are updated for a single
income support payment (ISP) customer.

| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Single income support payment     |
|                                   | (ISP) customer lodges their tax   |
|                                   | return and adjusted taxable       |
|                                   | income is above the FTB Part A    |
|                                   | supplement income limit + Read    |
|                                   | more ...                          |
|                                   |                                   |
|                                   | 2016-17 financial year            |
|                                   | re-reconciliation activities      |
|                                   | require manual checking where:    |
|                                   |                                   |
|                                   | -   initial reconciliation was    |
|                                   |     completed in bulk             |
|                                   |     reconciliation processing and |
|                                   |     the Part A supplement was     |
|                                   |     included                      |
|                                   | -   the customer has lodged a tax |
|                                   |     return, and                   |
|                                   | -   their adjusted taxable income |
|                                   |     in re-reconciliation is above |
|                                   |     the Part A supplement income  |
|                                   |     limit, resulting in           |
|                                   |     overpayment of Part A         |
|                                   |     supplement                    |
|                                   |                                   |
|                                   | These activities will be          |
|                                   | identified by a '#ISP80K' keyword |
|                                   | and will be mapped to             |
|                                   | appropriately trained staff.      |
|                                   |                                   |
|                                   | Note: The FTB/CCF Manual          |
|                                   | Intervention (FAMIV) screen will  |
|                                   | display a Manual Intervention     |
|                                   | Reason 'Single ISP cust has       |
|                                   | failed FTB(A) Supp income test'.  |
|                                   | Activities displaying this reason |
|                                   | do not need to be processed by    |
|                                   | staff in the Manual Intervention  |
|                                   | team (MIV), unless another Manual |
|                                   | Intervention Reason is also       |
|                                   | displayed on the FAMIV screen.    |
|                                   |                                   |
|                                   | If a customer contacts and has an |
|                                   | activity started on the Activity  |
|                                   | List (AL) screen with keyword     |
|                                   | '#ISP80K', and the activity       |
|                                   | requires urgent completion,       |
|                                   | families trained staff may        |
|                                   | complete the assessment following |
|                                   | the steps below.                  |
|                                   |                                   |
|                                   | Non-families trained staff,       |
|                                   | transfer the customer to the      |
|                                   | Families queue.                   |
|                                   |                                   |
|                                   | Families trained staff, go to     |
|                                   | Step 2.                           |
