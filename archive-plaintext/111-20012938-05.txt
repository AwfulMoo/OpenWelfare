|                                   | of circumstance transaction, Task |

|                                   | Selectors will contain shortcuts  |
|                                   | to screens that are most commonly |

|                                   | relevant work item.               |
|                                   |                                   |
|                                   | When a customerâ€™s record is       |
|                                   | accessed through Customer         |
|                                   | Summary, the Task Selector        |
|                                   | (START) screen will display. This |
|                                   | screen contains a list of all     |
|                                   | workflows and tasks that can be   |
|                                   | actioned on a customerâ€™s record.  |
|                                   |                                   |
|                                   | Go back to the Task Selector at   |
|                                   | any time by keying START into the |
|                                   | Super Key.                        |
|                                   |                                   |
|                                   | For processing advice and support |
|                                   | on a Task Selector workflow refer |
|                                   | to the relevant payment-related   |
|                                   | resources and Operational         |
|                                   | Blueprint procedure.              |
+-----------------------------------+-----------------------------------+
| Active MSP: Yes/No                | Active Managed Service Plan (MSP) |
|                                   | header                            |
|                                   |                                   |
|                                   | -   If a customer has an active   |

|                                   |     Active MSP hyperlink          |

|                                   | depending on the screen selected. |
|                                   | Some screens display more than 1  |
|                                   | related table and are known as    |
|                                   | summary screens.                  |
|                                   |                                   |
|                                   | Tables may contain:               |
|                                   |                                   |
|                                   | -   Provisional data - details    |
|                                   |     provided by the customer in   |
|                                   |     their claim or coded by a     |
|                                   |     Service Officer. This can be  |
|                                   |     edited or deleted.            |
|                                   | -   Confirmed data - information  |

|                                   |     determine if it is correct or |
|                                   |     should be deleted from the    |
|                                   |     activity                      |
|                                   | -   Provisional Deleted data      |
|                                   |     -confirmed data that has been |
|                                   |     deleted                       |

|                                   | Financial Institution details.    |
|                                   |                                   |
|                                   | The Child Lines are the           |
|                                   | individual entries underneath     |
|                                   | this heading, for example, bank   |
|                                   | accounts including bank balances. |
|                                   |                                   |
|                                   | To access the Child Lines, use    |
|                                   | the chevron at the left of the    |
|                                   | parent line.                      |

|                                   | also present any Task Selectors   |
|                                   | associated with the work item.    |
|                                   | Once the SWE screen no longer     |

|                                   | item is ready to be assessed.     |
|                                   |                                   |
|                                   | Warnings and errors               |

|                                   | -   Errors display in red and     |

|                                   | -   Warnings display in yellow    |
|                                   |     and may not require action    |
|                                   |     prior to finalising the work  |
|                                   |     item                          |
|                                   |                                   |
|                                   | Use the Message Log to select the |
|                                   | error/warning to action or        |
|                                   | manually go to the screen that    |
|                                   | requires action/update.           |
+-----------------------------------+-----------------------------------+
| Finalise activity                 | When finalising an activity,      |
|                                   | select Assess to go to the        |

|                                   | The ELD screen is similar to the  |
|                                   | Assessment Results (AR) screen in |
|                                   | Customer First.                   |
|                                   |                                   |

Process status definitions

Table 7: This table lists some of the process statuses that may display
on a work item in Process Direct.

Note: not all business processes and Customer First SAP native processes
may use all of the listed statuses. See relevant procedures for payment
specific information.

+-----------------------------------+-----------------------------------+
| Status                            | Description                       |
+-----------------------------------+-----------------------------------+
| Draft                             | The customer has started an       |
|                                   | online claim or update, but has   |
|                                   | yet to submit the claim or        |
|                                   | update. The submission date will  |
|                                   | display as 00:00:0000.            |
|                                   |                                   |
|                                   | If a customer contacts about a    |
|                                   | submission with a draft status,   |
|                                   | they should be advised to return  |
|                                   | to their online account and       |
|                                   | submit the claim or update.       |
+-----------------------------------+-----------------------------------+
| In Process                        | An online update or claim has     |
|                                   | been submitted by the customer    |
|                                   | and is ready to assess.           |
+-----------------------------------+-----------------------------------+
| On Hold                           | A work item has been held for     |
|                                   | action at a later date. This      |
|                                   | status is manually updated. It    |
|                                   | can be used when requesting more  |
|                                   | information from a customer or    |
|                                   | third party referrals.            |
|                                   |                                   |
|                                   | New claims (NCL), non-new claims  |
|                                   | (NNCL), also known as Change of   |
|                                   | Circumstance (CoC) transactions,  |
|                                   | if placed on hold for:            |
|                                   |                                   |
|                                   | -   the reason System             |
|                                   |     Investigation/Issue, the hold |
|                                   |     date defaults to 28 days.     |
|                                   |     This date cannot be changed   |
|                                   | -   for all other reasons, the    |
|                                   |     maximum hold period is 120    |
|                                   |     days. Staff must select a     |
|                                   |     time frame less than 120 days |
+-----------------------------------+-----------------------------------+
| Completed                         | A work item has been assessed and |
|                                   | finalised. This status will       |
|                                   | display for claims that have been |
|                                   | granted or rejected.              |
+-----------------------------------+-----------------------------------+
| Not Required                      | A work item does not need         |
|                                   | assessment.                       |
|                                   |                                   |
|                                   | For example, after a claim        |
|                                   | rejection a customer lodges       |
|                                   | another claim. If reassessing the |
|                                   | original claim is appropriate,    |
|                                   | staff can manually update the     |
|                                   | status of the second claim to Not |
|                                   | Required.                         |
+-----------------------------------+-----------------------------------+
| Withdrawn                         | A work item, for example, an      |
|                                   | online claim, has been withdrawn  |
|                                   | by the customer through their     |
|                                   | online account. Staff can also    |
|                                   | update this status manually.      |
+-----------------------------------+-----------------------------------+
| Submission Failed                 | A system error occurred between   |
|                                   | submission of an online or        |
|                                   | assisted customer claim and the   |
|                                   | claim generation.                 |
|                                   |                                   |
|                                   | This may be a result of           |
|                                   | conflicting activities on the     |
|                                   | customer or partner record, or    |
|                                   | record locking at the time of     |
|                                   | claim submission. Review the      |
|                                   | records to check if this is the   |
|                                   | case.                             |
|                                   |                                   |
|                                   | Once there are no activities      |
|                                   | preventing the claim activity     |
|                                   | from creating, update the claim   |
|                                   | status from Submission Failed to  |
|                                   | In Process. Then update as        |
|                                   | required.                         |
