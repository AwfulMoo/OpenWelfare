This page contains the process to follow when processing payments for
the Disaster Health Care Assistance Scheme (DHCAS).

Payment process for the DHCAS

Payment process for the Disaster Health Care Assistance Scheme (DHCAS)

This table describes the payment run process processing staff follow for
the Disaster Health Care Assistance Scheme (DHCAS) claims.

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Complete the Payment Run          |
|                                   | spreadsheet + Read more ...       |
|                                   |                                   |
|                                   | -   Open a new Payment Run        |
|                                   |     Template on the G shared      |
|                                   |     drive in the                  |
|                                   |     :\SITTeam\SPECIAL             |
|                                   |     ASSISTANCE\4. Payment Run     |
|                                   |     Spreadsheets                  |
|                                   | -   Save as file                  |
|                                   |     ‘P                            |
|                                   | AYMENTRUNTEMPLATE.DHCAS.YYYYMMDD’ |
|                                   |     on the G shared drive in the  |
|                                   |     :\SITTeam\SPECIAL             |
|                                   |     ASSISTANCE\4. Payment Run     |
|                                   |     Spreadsheets\                 |
|                                   |     For example,                  |
|                                   |                                   |
|                                   | PAYMENTRUNTEMPLATE.DHCAS.20180131 |
|                                   |                                   |
|                                   | The following information must be |
|                                   | entered for each claim:           |
|                                   |                                   |
|                                   | -   vendor number as shown on the |
|                                   |     Payment Advice claim document |
|                                   |     in the Vendor field           |
|                                   | -   date of issue, using format   |
|                                   |     dd/mm/yyyy, from claim        |
|                                   |     payment advice in the Enter   |
|                                   |     Invoice Date field            |
|                                   | -   claimant details and the      |
|                                   |     claim number from the claim   |
|                                   |     payment advice in the Header  |
|                                   |     Text field                    |
|                                   | -   same claim number details as  |
|                                   |     entered in header text in the |
|                                   |     Reference field               |
|                                   | -   applicable GL code for the    |
|                                   |     adverse event in the GL       |
|                                   |     Account field                 |
|                                   | -   applicable Cost Centre for    |
|                                   |     the adverse event in the Cost |
|                                   |     Centre field                  |
|                                   | -   key ‘P5’ in the Tax Code      |
|                                   |     field                         |
|                                   | -   leave Order, Quantity and     |
|                                   |     Quantity Unit blank and not   |
|                                   |     required (as advised by       |
|                                   |     Finance)                      |
|                                   | -   amount as shown on claim      |
|                                   |     payment advice in Amount      |
|                                   |     field                         |
|                                   | -   same date as entered in the   |
|                                   |     Invoice Date field, using     |
|                                   |     format dd/mm/yyyy in the      |
|                                   |     Bline Date field              |
|                                   | -   the adverse event, claimant   |
|                                   |     name and claim number in the  |
|                                   |     Line Item Text field          |
|                                   |                                   |
|                                   | Repeat until all claims are       |
|                                   | included on the Payment Run.      |
+-----------------------------------+-----------------------------------+
| 2                                 | Complete quality check of payment |
|                                   | run – by Team Leader (TL5) only + |
|                                   | Read more ...                     |
|                                   |                                   |
|                                   | All details entered in the        |
|                                   | Payment Run spreadsheet must be   |
|                                   | Quality Checked by the Team       |
|                                   | Leader 5 (TL5).                   |
|                                   |                                   |
|                                   | -   Open the Payment Run          |
|                                   |     spreadsheet located on the G  |
|                                   |     drive in the                  |
|                                   |     :\SITTeam\SPECIAL             |
|                                   |     ASSISTANCE\4. Payment Run     |
|                                   |     Spreadsheets                  |
|                                   | -   Open DHCAS Vendor List        |
|                                   |     located on the G drive in the |
|                                   |     :\SITTeam\SPECIAL             |
|                                   |     ASSISTANCE\References\Vendor  |
|                                   |     List                          |
|                                   | -   Ensure the information on the |
|                                   |     claim Payment Advice has been |
|                                   |     recorded correctly on the     |
|                                   |     Payment Run spreadsheet       |
|                                   |     (payment advices are saved on |
|                                   |     each claimant’s file in       |
|                                   |     PaNDA)                        |
|                                   | -   Check Vendor details against  |
|                                   |     current DHCAS Vendor List to  |
|                                   |     ensure they match. If no      |
|                                   |     match found, open Vendor List |
|                                   |     Errors to be Reported         |
|                                   |     spreadsheet located on the G  |
|                                   |     drive in the                  |
|                                   |     :\SITTeam\SPECIAL             |
|                                   |     ASSISTANCE\References\Vendor  |
|                                   |     List                          |
|                                   | -   Any errors detected must be   |
|                                   |     returned to the Service       |
|                                   |     Officer who processed the     |
|                                   |     claim for correction          |
+-----------------------------------+-----------------------------------+
| 3                                 | Delegate approval – by Team       |
|                                   | Leader (TL5) only + Read more ... |
|                                   |                                   |
|                                   | Forward notification email to     |
|                                   | appropriate delegates.            |
|                                   |                                   |
|                                   | The delegation amounts per claim  |
|                                   | are:                              |
|                                   |                                   |
|                                   | -   up to $2000 – TL6             |
|                                   | -   up to $5000 – EL1             |
|                                   | -   up to $10 000 – EL2           |
|                                   | -   over $10 000 – National       |
|                                   |     Office                        |
|                                   |                                   |
|                                   | Include in the email:             |
|                                   |                                   |
|                                   | -   a summary of the claims to be |
|                                   |     approved at each delegation   |
|                                   |     level                         |
|                                   | -   a copy of the completed       |
|                                   |     Payment Run spreadsheet for   |
|                                   |     approval                      |
+-----------------------------------+-----------------------------------+
| 4                                 | Delegate approval process – by    |
|                                   | delegate only + Read more ...     |
|                                   |                                   |
|                                   | -   Open each ‘pended’ claim in   |
|                                   |     PaNDA that has been added to  |
|                                   |     the Payment Run at their      |
|                                   |     delegate level                |
|                                   | -   Confirm dollar amount is      |
|                                   |     within approval level         |
|                                   |                                   |
|                                   | Delegation approved?              |
|                                   |                                   |
|                                   | -   Yes, notate ‘($value)         |
|                                   |     approved to pay under (insert |
|                                   |     level) delegation.’ in PaNDA, |
|                                   |     go to Step 5                  |
|                                   | -   No, send email response to    |
|                                   |     the Team Leader highlighting  |
|                                   |     the reason for rejection,     |
|                                   |     procedure ends here           |
+-----------------------------------+-----------------------------------+
| 5                                 | Delegate sends response email to  |
|                                   | TL5 with outcome + Read more ...  |
|                                   |                                   |
|                                   | -   All delegates must send a     |
|                                   |     response email back to TL5    |
|                                   |     advising that approval        |
|                                   |     process is complete           |
|                                   | -   Service Support Management    |
|                                   |     (SSM) delegate must include a |
|                                   |     note approving release of     |
|                                   |     Payment Run to Finance        |
+-----------------------------------+-----------------------------------+
| 6                                 | Print payment advice + Read more  |
|                                   | ...                               |
|                                   |                                   |
|                                   | -   Print each payment advice for |
|                                   |     each claim that has been      |
|                                   |     recorded on the Payment Run   |
|                                   |     spreadsheet                   |
|                                   | -   Send payment advice to the    |
|                                   |     claimant via post             |
+-----------------------------------+-----------------------------------+
| 7                                 | Send Payment Run to Finance +     |
|                                   | Read more ...                     |
|                                   |                                   |
|                                   | Email the Payment Run spreadsheet |
|                                   | to Accounting Payable Team and cc |
|                                   | in Cash Management team -         |
|                                   | Medicare and National Office      |
|                                   | Programme Management.             |
+-----------------------------------+-----------------------------------+
| 8                                 | Confirm release of payment + Read |
|                                   | more ...                          |
|                                   |                                   |
|                                   | When the payment run has been     |
|                                   | released to the RBA, Medicare     |
|                                   | Provider Branch will be cc’d in   |
|                                   | to the email from Finance         |
|                                   | (accounting operations)           |
|                                   |                                   |
|                                   | For Team Leader 5 (TL5) only      |
|                                   |                                   |
|                                   | -   Cross check all payments on   |
|                                   |     the Claimant Payment Advices  |
|                                   |     (printed in the QC process),  |
|                                   |     match with the Payment        |
|                                   |     Summary details attached to   |
|                                   |     the email response from       |
|                                   |     Finance                       |
|                                   | -   File Payment run confirmation |
|                                   |     email in MPS Boutiques        |
|                                   |     mailbox, sub folder DHCAS,    |
|                                   |     Payment Runs                  |
|                                   | -   Payment Statements are        |
|                                   |     despatched and a second copy  |
|                                   |     is left with payment run      |
|                                   |     documents filed with Team     |
|                                   |     Leader (TL5)                  |
+-----------------------------------+-----------------------------------+
