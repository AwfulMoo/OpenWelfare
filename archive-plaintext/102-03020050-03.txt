Commencing the ICoA transfer

Table 1: This table describes the process to commence a scheduled
Inter-environment Change of Address activity.

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Regular checks + Read more ...    |
|                                   |                                   |
|                                   | Regular checks for Scheduled      |
|                                   | (SCH) transfers need to be        |
|                                   | performed in both the gaining and |
|                                   | losing environments.              |
|                                   |                                   |
|                                   | Before checking SCH transfers     |
|                                   | clear out of the customer's       |
|                                   | record.                           |
|                                   |                                   |
|                                   | Note: transfers can be one of     |
|                                   | four status types.                |
|                                   |                                   |
|                                   | To check for scheduled (SCH)      |
|                                   | transfers of:                     |
|                                   |                                   |
|                                   | -   customer records, go to Step  |
|                                   |     2                             |
|                                   | -   child records, see Step 1 of  |
|                                   |     the ICoA transfer for child   |
|                                   |     records table                 |
+-----------------------------------+-----------------------------------+
| 2                                 | Gaining environment + Read more   |
|                                   | ...                               |
|                                   |                                   |
|                                   | Check the gaining environment for |
|                                   | scheduled (SCH) transfers of      |
|                                   | customer records:                 |
|                                   |                                   |
|                                   | In the gaining environment, go to |
|                                   | the Scheduled Transfers (ITSC)    |
|                                   | screen. The default display is a  |
|                                   | list of all scheduled (SCH)       |
|                                   | transfers for the user's region   |
|                                   |                                   |
|                                   | Refine the search criteria to     |
|                                   | locate customer transfers-in by   |
|                                   | coding:                           |
|                                   |                                   |
|                                   | -   the Service Centre code of    |
|                                   |     the gaining office in the     |
|                                   |     RGN: field                    |
|                                   | -   in the Status: field 'SCH'    |
|                                   | -   'Y‘ in the Trans In: field    |
|                                   |     and ‘N‘ in the Trans Out:     |
|                                   |     field                         |
|                                   | -   the user’s log on ID in the   |
|                                   |     User ID: field                |
|                                   | -   Code ‘C‘ in the C/K field to  |
|                                   |     locate customer transfers     |
|                                   |                                   |
|                                   | Select the relevant transfer      |
|                                   | using the 'L' option. The ICOA    |
|                                   | Log Information (ITLG) screen     |
|                                   | will be displayed.                |
+-----------------------------------+-----------------------------------+
| 3                                 | On the ICOA Log Information       |
|                                   | (ITLG) screen + Read more ...     |
|                                   |                                   |
|                                   | -   Check the Edit status: and    |
|                                   |     Current Edits: fields         |
|                                   | -   If 'OK' is showing in the     |
|                                   |     Edit status: field on the     |
|                                   |     ITLG screen and '0' is        |
|                                   |     showing in the Current Edits: |
|                                   |     field, then there is no       |
|                                   |     situation in the gaining      |
|                                   |     environment preventing the    |
|                                   |     transfer. Go to Step 5        |
|                                   | -   If the ITLG screen shows a    |
|                                   |     warning or an error, go to    |
|                                   |     Step 4                        |
|                                   | -   If the record has not         |
|                                   |     transferred within 2 days of  |
|                                   |     the last edit being cleared,  |
|                                   |     go to Step 5                  |
+-----------------------------------+-----------------------------------+
| 4                                 | Still in the ICOA Log Information |
|                                   | (ITLG) screen + Read more ...     |
|                                   |                                   |
|                                   | If the Edit status: field value   |
|                                   | on the ITLG screen is 'Warning',  |
|                                   | and the Current Edits: field      |
|                                   | value is one or more, then there  |
|                                   | is no situation in the gaining    |
|                                   | environment preventing the        |
|                                   | transfer:                         |
|                                   |                                   |
|                                   | -   Warnings may impact on        |
|                                   |     benefit processing so go to   |
|                                   |     the Edit List (ITED) screen   |

|                                   |     the transfer. See Step 1 of   |
|                                   |     the Edits required in a ICoA  |
|                                   |     transfer table                |

|                                   |     6                             |
|                                   | -   If the record has not         |
|                                   |     transferred within 2 days of  |

Edits required scheduled ICoA activities

Edits required in an ICoA transfer

Table 3: This table describes the process to deal with edits required in
a scheduled Inter-environment Change of Address activity.

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Determining when the transfer     |
|                                   | will be completed (COM) + Read    |
|                                   | more ...                          |
|                                   |                                   |
|                                   | Are there any edits preventing a  |
|                                   | transfer?                         |
|                                   |                                   |
|                                   | -   Yes, go to Step 3             |
|                                   | -   No, go to Step 2              |
+-----------------------------------+-----------------------------------+
| 2                                 | There are no edits preventing     |
|                                   | transfer + Read more ...          |
|                                   |                                   |
|                                   | -   Checking for completion of    |
|                                   |     the transfer can start after  |
|                                   |     2 hours. ICoA processing      |
|                                   |     copies the customer's record  |
|                                   |     from the losing environment   |
|                                   |     and reproduces it in the      |
|                                   |     gaining environment           |
|                                   | -   The transfer of the record    |
|                                   |     and the deletion in the       |
|                                   |     losing environment can occur  |
|                                   |     on the same day               |
|                                   |                                   |
|                                   | Procedure ends here.              |
+-----------------------------------+-----------------------------------+
| 3                                 | There are edits preventing a      |
|                                   | transfer + Read more ...          |
|                                   |                                   |
|                                   | Edits must be cleared before the  |
|                                   | transfer will complete (COM). See |
|                                   | Inter-environment Change of       |
|                                   | Address (ICoA) facility edits for |
|                                   | assistance with identifying and   |
|                                   | clearing edits. Some edits will   |
|                                   | be automatically cleared (for     |
|                                   | example, internal edits such as   |
|                                   | the locking of the customer       |
|                                   | record by another user), while    |
|                                   | others will require action (for   |
|                                   | example, a system specific edit   |
|                                   | such as a started (STA) activity) |
|                                   |                                   |
|                                   | If the ITLG screen displays an    |
|                                   | Edit status: field value of OK    |
|                                   | and the Current Edits: field      |
|                                   | value is 0 but has not            |
|                                   | transferred within 2 days of the  |
|                                   | last edit being cleared, see      |
|                                   | Inter-environment Change of       |
|                                   | Address (ICoA) facility problem   |
|                                   | escalation process                |
|                                   |                                   |
|                                   | The processing triggers are       |
|                                   | updated with information that     |
|                                   | indicates when the transfer       |
|                                   | should be retried by the batch    |
|                                   | system. Generally speaking there  |
|                                   | are 2 periods of time that are    |
|                                   | used:                             |
|                                   |                                   |
|                                   | -   if there is an ICoA internal  |
|                                   |     edit preventing transfer,     |
|                                   |     then the transfer will be     |
|                                   |     retried in about 2 hours      |
|                                   | -   if there is a system specific |
|                                   |     edit preventing transfer,     |
|                                   |     then the transfer will be     |
|                                   |     retried on the following day  |
|                                   |     (that is, there will be a     |
|                                   |     delay of one day after the    |
|                                   |     last edit is cleared)         |
+-----------------------------------+-----------------------------------+
