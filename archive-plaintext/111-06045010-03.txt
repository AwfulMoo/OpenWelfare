This page contains information about accessing and navigating customer
details and case information for payments delivered using Customer First
SAP native processes.

Viewing customer details and cases

Customer Details screen, Super Case and Benefit Case tabs

This table contains information about viewing and navigating the
Customer Details screen and the Super Case and Benefit Case tabs.

+-----------------------------------+-----------------------------------+
| Item                              | Description                       |
+-----------------------------------+-----------------------------------+
| 1                                 | Access + Read more ...            |
|                                   |                                   |
|                                   | To access the Customer Details    |
|                                   | screen:                           |
|                                   |                                   |
|                                   | -   enter a Customer Reference    |
|                                   |     Number (CRN) or Business      |
|                                   |     Partner Number (BP) in the    |
|                                   |     CRN/BP field in the Customer  |
|                                   |     First header, or              |
|                                   | -   select the CRN/BP hyperlink   |
|                                   |     once the customer is in       |
|                                   |     context.                      |
|                                   |                                   |
|                                   | To navigate to customer's Social  |
|                                   | Applications (claims in progress  |
|                                   | or finalised):                    |
|                                   |                                   |
|                                   | -   select the Cases tab on the   |
|                                   |     Customer Details screen       |
|                                   | -   select the Super Case for     |
|                                   |     business partner hyperlink    |
|                                   |                                   |
|                                   | To navigate to the customer's     |
|                                   | Social Service Plans and          |
|                                   | information related to current    |
|                                   | and past payments and services:   |
|                                   |                                   |
|                                   | -   select the Cases tab on the   |
|                                   |     Customer Details screen       |
|                                   | -   select the Benefit Case       |
|                                   |     hyperlink                     |
+-----------------------------------+-----------------------------------+
| 2                                 | Customer Details screen - upper   |
|                                   | section + Read more ...           |
|                                   |                                   |
|                                   | Customer tab contains information |
|                                   | in assignment blocks and          |
|                                   | hyperlinks to more information    |
|                                   | about particular details.         |
|                                   | Information on this tab is        |
|                                   | derived from the legacy systems.  |
|                                   | There are hyperlinks to the       |
|                                   | screen where the information is   |
|                                   | recorded:                         |
|                                   |                                   |
|                                   | -   Customer Summary for customer |
|                                   |     details, relationships,       |
|                                   |     contacts and accommodation    |
|                                   |     details, and other general    |
|                                   |     information                   |
|                                   | -   Payments and Services for     |
|                                   |     payment information (not      |
|                                   |     Customer First SAP native     |
|                                   |     payments)                     |
|                                   | -   Incomplete/outstanding        |
|                                   |     activities for current Income |
|                                   |     Security Integrated System    |
|                                   |     (ISIS) Integrated Activity    |
|                                   |     Management (IAM) activities   |
|                                   |     on the Activities List (AL)   |
|                                   |     screen and future activities  |
|                                   |     on the Future Activities List |
|                                   |     (FAL) screen                  |
|                                   | -   Last 10 interactions with     |
|                                   |     Centrelink including          |
|                                   |     DOCuments and letters, SMS    |
|                                   |     and emails to the customer    |
|                                   | -   Customer Feedback for last 12 |
|                                   |     months of interactions        |
|                                   |     recorded in the Customer      |
|                                   |     Feedback Tool                 |
|                                   |                                   |
|                                   | Bank Details Edit provides the    |
|                                   | ability to change a customer’s    |
|                                   | payment destination if the        |
|                                   | payment is delivered by Customer  |
|                                   | First SAP native processes and    |
|                                   | does not apply to legacy systems  |
|                                   | and payments                      |
|                                   |                                   |
|                                   | Circumstances Data tab contains   |
|                                   | customer information in a variety |
|                                   | of assignment blocks and          |
|                                   | hyperlinks to more details about  |
|                                   | particular items. This tab only   |
|                                   | displays customer information     |
|                                   | stored in departmental Software   |
|                                   | Application Program (SAP) systems |
|                                   | and not information stored in     |
|                                   | legacy systems, such as ISIS.     |
|                                   | Assignment blocks relevant to the |
|                                   | customer's circumstances will     |
|                                   | display. Note: if staff find      |
|                                   | differences between the           |
|                                   | information in the SAP systems    |
|                                   | and the legacy systems, this      |
|                                   | should be reported via mySupport  |
|                                   | with screen shots of the          |
|                                   | information in both systems       |
|                                   |                                   |
|                                   | Tax File Number tab provides the  |
|                                   | ability to update the Tax File    |
|                                   | Number. For privacy reasons, the  |
|                                   | actual Tax File Number will not   |
|                                   | be displayed once recorded        |
|                                   |                                   |
|                                   | Relationships tab provides        |
|                                   | information about the customer’s  |
|                                   | relationships, including marital  |
|                                   | status and children               |
+-----------------------------------+-----------------------------------+
| 3                                 | Customer Details screen - lower   |
|                                   | section + Read more ...           |
|                                   |                                   |
|                                   | Cases tab provides access to the  |
|                                   | customer’s Super Cases and        |
|                                   | Benefit Cases:                    |
|                                   |                                   |
|                                   | -   Select the top entry Super    |
|                                   |     Case for business partner to  |
|                                   |     access to Social              |
|                                   |     Applications.                 |
|                                   | -   Select other entries (Benefit |
|                                   |     Cases) to access information  |
|                                   |     and assessments which are     |
|                                   |     Benefit Case specific. Once a |
|                                   |     Social Application is         |
|                                   |     approved, a Benefit Case is   |
|                                   |     created. The list of Benefit  |
|                                   |     Cases displays the history of |
|                                   |     benefit types the customer    |
|                                   |     has previously claimed.       |
|                                   |                                   |
|                                   | Communication History tab         |
|                                   | provides a list of communications |
|                                   | to the customer (letter, SMS)     |
|                                   |                                   |
|                                   | Inactive Addresses tab provides a |
|                                   | list of the customer’s previous   |
|                                   | addresses                         |
|                                   |                                   |
|                                   | Homelessness tab provides a list  |
|                                   | of the customer’s current and     |
|                                   | previous homelessness indicators. |
|                                   | A new entry can be made using the |
|                                   | Add New icon                      |
|                                   |                                   |
|                                   | Activities provides a list of     |
|                                   | Customer First SAP native         |
|                                   | transactions, including scans     |
|                                   | (Document Management System) and  |
|                                   | online claims                     |
|                                   |                                   |
|                                   | Marketing Attributes tab is not   |
|                                   | currently in use                  |
|                                   |                                   |
|                                   | Appointments tab provides future  |
|                                   | appointments and the              |
|                                   | functionality to complete,        |
|                                   | reschedule or cancel appointments |
|                                   | using the Actions icons           |
|                                   |                                   |
|                                   | History currently provides        |
|                                   | Customer First SAP native         |
|                                   | transaction information           |
|                                   |                                   |
|                                   | Interaction History tab can be    |
|                                   | selected to view a list of the    |
|                                   | customer’s interactions (tasks)   |
+-----------------------------------+-----------------------------------+
| 4                                 | Super Case tabs + Read more ...   |
|                                   |                                   |
|                                   | -   Case Details tab includes the |
|                                   |     reference date which is the   |
|                                   |     date the Super Case was       |
|                                   |     created                       |
|                                   | -   Primary Person tab displays   |
|                                   |     basic customer information,   |
|                                   |     such as names, gender, birth  |
|                                   |     date and marital status       |
|                                   | -   Notes tab provides all notes  |
|                                   |     associated to the Super Case  |
|                                   |     and also the ability to add   |
|                                   |     notes                         |
|                                   | -   Record tab provides access to |
|                                   |     all current and past claims   |
|                                   |     (made after September 2015)   |
|                                   |     and Benefit Cases             |
|                                   | -   Transactions tab also         |
|                                   |     provides access to past and   |
|                                   |     current claim activities, the |
|                                   |     same as available on the      |
|                                   |     Record tab                    |
|                                   | -   Activities tab is not         |
|                                   |     currently in use              |
|                                   | -   Related Cases tab provides    |
|                                   |     access to information about   |
|                                   |     reviews and other future      |
|                                   |     Benefit Case actions          |
|                                   | -   Parties involved tab is not   |
|                                   |     currently in use              |
|                                   | -   Attachments tab is not        |
|                                   |     currently in use              |
+-----------------------------------+-----------------------------------+
| 5                                 | Benefit Case tabs + Read more ... |
|                                   |                                   |
|                                   | -   Case Details tab includes the |
|                                   |     reference date, which is the  |
|                                   |     date the Benefit Case was     |
|                                   |     created                       |
|                                   | -   Primary Person tab displays   |
|                                   |     basic customer information,   |
|                                   |     such as names, gender, birth  |
|                                   |     date and marital status       |
|                                   | -   Case Notes tab displays all   |
|                                   |     notes associated to the       |
|                                   |     Benefit Case and also the     |
|                                   |     ability to add notes          |
|                                   | -   SSP Notes tab provides all    |
|                                   |     notes associated to Social    |
|                                   |     Service Plans (SSPs)          |
|                                   | -   Social Services tab provides  |
|                                   |     access to all Benefit Case    |
|                                   |     SSPs                          |
|                                   | -   Activities tab is not         |
|                                   |     currently in use              |
|                                   | -   Related Cases tab provides    |
|                                   |     access to benefit specific    |
|                                   |     activities such as reviews    |
|                                   | -   Parties involved tab can      |
|                                   |     identify parties involved in  |
|                                   |     the Benefit Case such as case |
|                                   |     officers                      |
|                                   | -   Attachments tab is not        |
|                                   |     currently in use              |
