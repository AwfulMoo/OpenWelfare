Assessing a job seeker's eligibility for variable reporting

Table 2

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Assess eligibility for variable   |
|                                   | reporting + Read more ...         |
|                                   |                                   |
|                                   | Variable reporting is automated   |
|                                   | in most cases. This means the     |
|                                   | system will look at the job       |
|                                   | seeker's record and if the        |
|                                   | customer's mutual obligation      |
|                                   | requirements or other             |
|                                   | circumstances indicate they       |
|                                   | should be on variable reporting,  |
|                                   | the system will do this.          |
|                                   |                                   |
|                                   | Check the Activity and Exemption  |
|                                   | Summary (AEX) screen.             |
|                                   |                                   |
|                                   | A job seeker can be put on        |
|                                   | variable reporting if they meet   |
|                                   | one or more of the following      |
|                                   | categories:                       |
|                                   |                                   |
|                                   | -   granted with the Disability   |
|                                   |     Support Pension (DSP)         |
|                                   |     exemption                     |
|                                   | -   participating in the          |
|                                   |     Community Development Program |
|                                   |     Community Participation (RCP) |
|                                   |     activity                      |
|                                   | -   participating in the Adult    |
|                                   |     Migrant English Program       |
|                                   |     (AMEP)                        |
|                                   | -   granted with the 'REF'        |
|                                   |     (Refugee in first 13 weeks in |
|                                   |     Australia) exemption          |
|                                   | -   granted with the 'R6M'        |
|                                   |     (Refugee first 6 months)      |
|                                   |     exemption                     |
|                                   | -   granted with the Temp Incap   |
|                                   |     Serious Illness exemption     |
|                                   |                                   |
|                                   | And the job seeker:               |
|                                   |                                   |
|                                   | -   is not indebted to the        |
|                                   |     Commonwealth under the Act    |
|                                   |     (except if the debt is a      |
|                                   |     third party debt or incurred  |
|                                   |     through no fault of the job   |
|                                   |     seeker, or the debt is        |
|                                   |     another government department |
|                                   |     debt or any CDEP debts        |
|                                   |     incurred before 20 September  |
|                                   |     1996). Check the General      |
|                                   |     Information (OPGI) screen for |
|                                   |     each debt, There are no       |
|                                   |     outstanding compliance        |
|                                   |     actions pending (check the    |
|                                   |     Participation Compliance      |
|                                   |     Summary (BIIS) for an urgent  |
|                                   |     issue)                        |
|                                   |                                   |
|                                   | Is there sufficient information   |
|                                   | on the job seeker's record to     |
|                                   | determine their eligibility for   |
|                                   | variable reporting?               |
|                                   |                                   |
|                                   | -   Yes, go to Step 2             |
|                                   | -   No, explain to the customer   |
|                                   |     why they do not satisfy the   |
|                                   |     variable reporting criteria.  |
|                                   |     Record reason on a DOC.       |
|                                   |     Procedure ends here           |
+-----------------------------------+-----------------------------------+
| 2                                 | Check job seeker circumstances +  |
|                                   | Read more ...                     |
|                                   |                                   |
|                                   | Does the job seeker satisfy the   |
|                                   | variable reporting criteria?      |
|                                   |                                   |
|                                   | -   Yes, go to Step 3             |
|                                   | -   No, explain to the job seeker |
|                                   |     why they do not satisfy the   |
|                                   |     variable reporting criteria.  |
|                                   |     Record reason on a DOC.       |
|                                   |     Procedure ends here           |
+-----------------------------------+-----------------------------------+
| 3                                 | Update job seeker record + Read   |
|                                   | more ...                          |
|                                   |                                   |
|                                   | To update the reporting frequency |
|                                   | in Customer First, go to the      |
|                                   | Reporting Summary hyperlink,      |
|                                   | either:                           |
|                                   |                                   |
|                                   | -   via the Employment Income and |
|                                   |     Reporting menu, or            |
|                                   | -   search Reporting Summary in   |
|                                   |     Workspace                     |
|                                   |                                   |
|                                   | Under 'Update options available   |
|                                   | for the customer', select 'Change |
|                                   | to Variable reporting'.           |
|                                   |                                   |
|                                   | If the workflow is unavailable,   |
|                                   | update the Reporting Regime       |
|                                   | Details (RPRD) screen manually:   |
|                                   |                                   |
|                                   | -   select the most recent Start  |
|                                   |     Date hyperlink of the         |
|                                   |     Current/Future Reports table, |
|                                   |     or from the Reporting Regime  |
|                                   |     Summary (RPRS) screen         |
|                                   | -   select the frequency of       |
|                                   |     reporting in the Frequency:   |
|                                   |     field and select a reason in  |
|                                   |     the Reason for requirement:   |
|                                   |     field. Press [Enter]          |
|                                   | -   if appropriate to manually    |
|                                   |     place the job seeker on 4     |
|                                   |     weekly reporting frequency as |
|                                   |     they do not currently have    |
|                                   |     access to reporting           |
|                                   |     facilities (including self    |
|                                   |     service) that would allow     |
|                                   |     them to report every 2 weeks, |
|                                   |     the job seeker may also be    |
|                                   |     exempt from the personal      |
|                                   |     lodgement requirement during  |
|                                   |     the relevant period. In       |
|                                   |     addition, if the job seeker   |
|                                   |     has been reporting using self |
|                                   |     service, update the RPRD      |
|                                   |     screen to allow the S19 form  |
|                                   |     to be sent to the job seeker. |
|                                   |     It can be posted or faxed to  |
|                                   |     Services Australia on the     |
|                                   |     reporting due date. To do     |
|                                   |     this, change the Report Form  |
|                                   |     Type: field to 'S19           |
|                                   |     application for payment', and |
|                                   |     in the Reason: field, use     |
|                                   |     'CSA override' reason. Press  |
|                                   |     [Enter]                       |
|                                   | -   the RPRD screen will be       |
|                                   |     update and redisplay. For     |
|                                   |     more information, see         |
|                                   |     Reporting screens             |
|                                   | -   record the decision on a DOC  |
+-----------------------------------+-----------------------------------+
| 4                                 | Finalise activity + Read more ... |
|                                   |                                   |
|                                   | Complete the activity on the      |
|                                   | Assessment Results (AR) screen.   |
|                                   | The system will change the        |
|                                   | frequency of reporting for the    |
|                                   | job seeker and issue an advice    |
|                                   | and a new Reporting Statement     |
|                                   | with the new dates.               |
|                                   |                                   |
|                                   | The only time the frequency of    |
|                                   | reporting cannot be changed is if |
|                                   | the change is being made on the   |
|                                   | job seeker's Entitlement Period   |
|                                   | End Day (EPED). If so record      |
|                                   | details in an open DOC and place  |
|                                   | on hold for the following day:    |
|                                   |                                   |
|                                   | -   Advise the job seeker if they |
|                                   |     have provided their mobile    |
|                                   |     phone number or email address |
|                                   |     previously they will be sent  |
|                                   |     a reminder the day before     |
|                                   |     they are due to report        |
|                                   | -   Update the reporting          |
|                                   |     frequency the following day   |
|                                   |     (if appropriate) and close    |
|                                   |     DOC                           |

Checking and updating variable reporting requirements

Table 3:

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Check job seeker eligibility for  |
|                                   | variable reporting + Read more    |
|                                   | ...                               |
|                                   |                                   |
|                                   | Is the job seeker eligible for    |
|                                   | variable reporting?               |
|                                   |                                   |
|                                   | -   Yes, this procedure does not  |
|                                   |     apply                         |
|                                   | -   No, the system may            |
|                                   |     automatically revert to       |
|                                   |     fortnightly reporting in      |
|                                   |     certain circumstances.        |
|                                   |     Otherwise, the job seeker     |
|                                   |     will need to be put back to   |
|                                   |     reporting every 2 weeks. Go   |
|                                   |     to Step 2                     |
+-----------------------------------+-----------------------------------+
| 2                                 | Update job seeker record + Read   |
|                                   | more ...                          |
|                                   |                                   |
|                                   | To update the reporting frequency |
|                                   | in Customer First, go to the      |
|                                   | Reporting Summary hyperlink,      |
|                                   | either:                           |
|                                   |                                   |
|                                   | -   via the Employment Income and |
|                                   |     Reporting menu, or            |
|                                   | -   search Reporting Summary in   |
|                                   |     Workspace                     |
|                                   |                                   |
|                                   | Under 'Update options available   |
|                                   | for the customer', select 'Place  |
|                                   | on two weekly reporting'.         |
|                                   |                                   |
|                                   | If the workflow is unavailable,   |
|                                   | update the Reporting Regime       |
|                                   | Details (RPRD) screen manually:   |
|                                   |                                   |
|                                   | -   select the most recent Start  |
|                                   |     Date hyperlink of the         |
|                                   |     Current/Future Reports table, |
|                                   |     or from the Reporting Regime  |
|                                   |     Summary (RPRS) screen         |
|                                   | -   select the frequency of       |
|                                   |     reporting in the Frequency:   |
|                                   |     field and select a reason in  |
|                                   |     the Reason for requirement:   |
|                                   |     field. Press [Enter]. The     |
|                                   |     screen will be updated and    |
|                                   |     re-displayed                  |
|                                   | -   complete the activity on the  |
|                                   |     Assessment Results (AR)       |
|                                   |     screen                        |
|                                   | -   ensure that the job seeker is |
|                                   |     advised of their next report  |
|                                   |     due date                      |
|                                   |                                   |
|                                   | Note: the following business      |
|                                   | rules apply when compliance       |
|                                   | action is generated.              |
|                                   |                                   |
|                                   | -   if less than 5 days to the    |
|                                   |     Entitlement Period End Date   |
|                                   |     (EPED) the two weekly         |
|                                   |     lodgement date should not be  |
|                                   |     coded until after this date.  |
|                                   |     Note: contact will still be   |
|                                   |     required with the             |
|                                   |     Participation Solutions Team  |
|                                   |     (PST) to release payment,     |
|                                   |     however, reporting will only  |
|                                   |     be required from the          |
|                                   |     following fortnight. This     |
|                                   |     ensures that the job seeker   |
|                                   |     is aware they are now         |
|                                   |     required to report every 2    |
|                                   |     weeks and receive             |
|                                   |     notification of their changed |
|                                   |     reporting requirements prior  |
|                                   |     to their next EPED            |
|                                   | -   if more than 5 days to the    |
|                                   |     EPED the customer should be   |
|                                   |     reverted to two weekly        |
|                                   |     reporting immediately         |
+-----------------------------------+-----------------------------------+
| 3                                 | Check record successfully         |
|                                   | updated + Read more ...           |
|                                   |                                   |
|                                   | The system should change the      |
|                                   | frequency of reporting for the    |
|                                   | job seeker and they will be       |
|                                   | notified of their new reporting   |
|                                   | requirements via:                 |
|                                   |                                   |
|                                   | -   a Short Message Service (SMS) |
|                                   |     alert if they receive their   |
|                                   |     letters or advices            |
|                                   |     electronically via Online     |
|                                   |     Letters (OLL)                 |
|                                   | -   an alert/prompt when they     |
|                                   |     enter one of the self service |
|                                   |     channels if they use one of   |
|                                   |     the self service options      |
|                                   | -   or issue an advice and a new  |
|                                   |     Reporting Statement (SU19)    |
|                                   |     with the new dates            |
|                                   |                                   |
|                                   | Has the frequency of reporting    |
|                                   | updated successfully?             |
|                                   |                                   |
|                                   | -   Yes, record details on a DOC. |
|                                   |     Procedure ends here           |
|                                   | -   No, go to Step 4              |
+-----------------------------------+-----------------------------------+
| 4                                 | Check if further action           |
|                                   | required + Read more ...          |
|                                   |                                   |
|                                   | -   If the job seeker has         |
|                                   |     contacted on their            |
|                                   |     Entitlement Period End Day    |
|                                   |     (EPED) the frequency of       |
|                                   |     reporting may be unable to be |
|                                   |     changed. If so, set up a      |
|                                   |     Review DOC so that removal    |
|                                   |     from variable reporting can   |
|                                   |     be actioned the following     |
|                                   |     day, or when the variable     |
|                                   |     report is processed. Go to    |
|                                   |     Step 5                        |
|                                   | -   If the system is              |
|                                   |     automatically putting the job |
|                                   |     seeker straight back on       |
|                                   |     variable reporting due to     |
|                                   |     other activities or exemption |
|                                   |     from the mutual obligation    |
|                                   |     requirements coded on the     |
|                                   |     system, override this by      |
|                                   |     entering a code in the Reason |
|                                   |     for requirement: field. See   |
|                                   |     Field Help ('?') for          |
|                                   |     assistance                    |
+-----------------------------------+-----------------------------------+
| 5                                 | Update day after EPED + Read more |
|                                   | ...                               |
|                                   |                                   |
|                                   | The day after the EPED, action    |
|                                   | the review DOC and remove the job |
|                                   | seeker from variable reporting.   |
|                                   |                                   |
|                                   | Go to the Reporting Summary       |
|                                   | hyperlink, either:                |
|                                   |                                   |
|                                   | -   via the Employment Income and |
|                                   |     Reporting menu, or            |
|                                   | -   search Reporting Summary in   |
|                                   |     Workspace                     |
|                                   |                                   |
|                                   | Under 'Update options available   |
|                                   | for the customer', select 'Place  |
|                                   | on two weekly reporting'.         |
|                                   |                                   |
|                                   | If the workflow is unavailable,   |
|                                   | update the Reporting Regime       |
|                                   | Details (RPRD) screen manually:   |
|                                   |                                   |
|                                   | -   select the most recent Start  |
|                                   |     Date hyperlink of the         |
|                                   |     Current/Future Reports table, |
|                                   |     or from the Reporting Regime  |
|                                   |     Summary (RPRS) screen         |
|                                   | -   select the frequency of       |
|                                   |     reporting in the Frequency:   |
|                                   |     field and select a reason in  |
|                                   |     the Reason for requirement:   |
|                                   |     field. Press [Enter]. The     |
|                                   |     screen will be updated and    |
|                                   |     re-displayed                  |
|                                   | -   complete the activity on the  |
|                                   |     Assessment Results (AR)       |
|                                   |     screen                        |
|                                   | -   ensure the job seeker is      |
|                                   |     advised of their next report  |
|                                   |     due date                      |
|                                   | -   record details on a DOC       |
