Remove Entitlement End Date AODR entitlement in CDMS

Table 1: This table describes the rules associated with removing
Entitlement End Date and End Date Reason where the member of the public
has an AODR entitlement recorded in the CDMS.

Note: if a Service Officer attempts to amend an AODR entitlement and
there is an Entitlement End Date and End Date Reason present, the system
returns error message 01236/01237 - ENTITLEMENT END DATE.

+-----------------------------------+-----------------------------------+
| End Date Reason                   | Rule                              |
+-----------------------------------+-----------------------------------+
| All Reasons                       | Must be actioned via signed forms |
|                                   | and/or PSO or Policy request      |
|                                   | only.                             |
+-----------------------------------+-----------------------------------+
| Age                               | The date of birth recorded on the |
|                                   | consumer record must be updated   |
|                                   | to equal 16 years before the      |
|                                   | Entitlement End Date and End Date |
|                                   | Reason of 'Age' can be removed.   |
+-----------------------------------+-----------------------------------+
| Deceased                          | The Date of Death and Death       |
|                                   | Notification Source must be       |
|                                   | removed from the consumer record  |
|                                   | before the Entitlement End Date   |
|                                   | and End Date Reason of 'Deceased' |
|                                   | can be removed.                   |
+-----------------------------------+-----------------------------------+
| Permanent departure overseas      | The address must be amended to an |
|                                   | Australian (or production control |
|                                   | if COVID-19 related) address on   |
|                                   | the consumer record before the    |
|                                   | Entitlement End Date and End Date |
|                                   | Reason of 'Permanent Departure    |
|                                   | Overseas' can be removed.         |
+-----------------------------------+-----------------------------------+
| POD (Potential Organ Donor)       | The Entitlement End Date and End  |
| Request                           | Date Reason of 'POD Request' must |
|                                   | be removed to continue            |
|                                   | processing.                       |
+-----------------------------------+-----------------------------------+
| Fact of Death                     | The Date of Death and Death       |
|                                   | Notification Source must be       |
|                                   | removed from the consumer record  |
|                                   | before the Entitlement End Date   |
|                                   | and End Date Reason of 'Fact of   |
|                                   | Death' can be removed.            |
+-----------------------------------+-----------------------------------+
| Duplicate                         | The Entitlement End Date and End  |
|                                   | Date Reason of 'Duplicate' must   |
|                                   | be removed to continue            |
|                                   | processing.                       |
+-----------------------------------+-----------------------------------+
| Admin                             | The Entitlement End Date and End  |
|                                   | Date Reason must not be removed   |
|                                   | and the registration cannot be    |
|                                   | updated.                          |
+-----------------------------------+-----------------------------------+
