Waiving a small debt

This table describes the process to action a small debt waiver in the
Debt Management and Information System (DMIS). To reverse a waiver for
debts managed in Process Direct, see Reversal of the waiver of a debt in
Process Direct table in Raising Child Care Subsidy (CCS) debts in
Process Direct.

+-----------------------------------+-----------------------------------+
| Item                              | Description                       |
+-----------------------------------+-----------------------------------+
| 1                                 | Debt recovery script + Read more  |
|                                   | ...                               |
|                                   |                                   |
|                                   | The Debt Action script, Debt      |
|                                   | Recovery script or the Debt       |
|                                   | Repayment guided procedure (Smart |
|                                   | Centres and Service Centres) must |
|                                   | be used when actioning a debt.    |
|                                   |                                   |
|                                   | Use a Fast Note to create a DOC,  |
|                                   | Callback or ACTDOC.               |
|                                   |                                   |
|                                   | Where a manual process is         |
|                                   | included in the following table,  |
|                                   | these are only to be followed     |
|                                   | when the script, guided procedure |
|                                   | or Fast Note are unavailable      |
+-----------------------------------+-----------------------------------+
| 2                                 | Family Tax Benefit (FTB) debts    |
|                                   | before reconciliation + Read more |
|                                   | ...                               |
|                                   |                                   |
|                                   | Only waive FTB debts of less than |
|                                   | $50 if the debt is a              |
|                                   | qualification debt.               |
|                                   |                                   |
|                                   | See the Resources page for a list |
|                                   | of examples.                      |
|                                   |                                   |
|                                   | Waive these debts on the          |
|                                   | Assessment Circumstances (ASC)    |
|                                   | screen using the code 'LTF' (less |
|                                   | than fifty).                      |
|                                   |                                   |
|                                   | When actioning a new debt (NDB)   |
|                                   | activity and:                     |
|                                   |                                   |
|                                   | -   the debt is a qualification   |
|                                   |     debt, waive the debt with a   |
|                                   |     reason code 'LTF' (less than  |
|                                   |     fifty)                        |
|                                   | -   the debt is not a             |
|                                   |     qualification debt, finalise  |
|                                   |     the debt as 'no debt' (FND)   |
|                                   |     with a reason code of 'FAM'.  |
|                                   |     This will include the amount  |
|                                   |     in reconciliation             |
|                                   |                                   |
|                                   | Record the details on a DOC as    |
|                                   | per the minimum DOC standards for |
|                                   | debt waivers.                     |
+-----------------------------------+-----------------------------------+
| 3                                 | Manually raising and waiving a    |
|                                   | debt + Read more ...              |
|                                   |                                   |
|                                   | -   Use the Debt Action script    |
|                                   |                                   |
|                                   | or                                |
|                                   |                                   |
|                                   | go to the Debt List (OPDL) to     |
|                                   | action a started debt activity,   |
|                                   | or                                |
|                                   |                                   |
|                                   | go to the Add Debt (OPAD) screen  |
|                                   | and complete the following fields |
|                                   | to raise a new debt:              |
|                                   |                                   |
|                                   | -   Benefit Type field, use field |
|                                   |     help (?) to find the          |
|                                   |     appropriate codes if required |
|                                   | -   Reason field                  |
|                                   | -   Section of Act field          |
|                                   | -   Date of Event field, if       |
|                                   |     necessary                     |
|                                   | -   Start Date field, if          |
|                                   |     necessary                     |
|                                   | -   End Date field, if necessary  |
|                                   | -   Calculated Amount $ field     |
|                                   | -   press [Enter]                 |
|                                   | -   press [Enter] again to        |
|                                   |     confirm the details           |
|                                   |                                   |
|                                   | Complete the Component Amount     |
|                                   | (OPCA) screen, use field help (?) |
|                                   | to find the appropriate codes if  |
|                                   | required.                         |
|                                   |                                   |
|                                   | -   press [Enter]                 |
|                                   | -   press [Enter] again to        |
|                                   |     confirm the component details |
|                                   |                                   |
|                                   | The Debt Management and           |
|                                   | Information System (DMIS) will    |
|                                   | then automatically go to the Add  |
|                                   | Waiver (OPWAA) screen where the   |
|                                   | debt can be waived. Use the       |
|                                   | reason code of 'LTF' (less than   |
|                                   | fifty)                            |
|                                   |                                   |
|                                   | On the Assessment Finalisation    |
|                                   | (OPAF) screen, key 'Y' in the     |
|                                   | Finished? field. Press [Enter] to |
|                                   | finalise the activity             |
|                                   |                                   |
|                                   | Note: where a small debt has been |
|                                   | manually raised and waived, a DOC |
|                                   | is required as per the minimum    |
|                                   | DOC standards for debt waivers.   |
+-----------------------------------+-----------------------------------+
| 4                                 | Waive a determined debt after     |
|                                   | review + Read more ...            |
|                                   |                                   |
|                                   | []                                |
|                                   |                                   |
|                                   | -   Use the Debt Action script    |
|                                   |                                   |
|                                   | or                                |
|                                   |                                   |
|                                   | go to the Debt List (OPDL) screen |
|                                   | and select the debt that is to be |
|                                   | actioned                          |
|                                   |                                   |
|                                   | go to the OPDA and enter the new  |
|                                   | debt amount                       |
|                                   |                                   |
|                                   | -   enter the Reason Changed as   |
|                                   |     REV                           |
|                                   | -   enter the Calculated Amount   |
|                                   | -   press [Enter]                 |
|                                   |                                   |
|                                   | Update the Component Amount       |
|                                   | (OPCA) screen                     |
|                                   |                                   |
|                                   | -   press [Enter]                 |
|                                   | -   press [Enter] again to        |
|                                   |     confirm the component details |
|                                   |                                   |
|                                   | DMIS will then automatically flow |
|                                   | to the Add Waiver (OPWAA) screen  |
|                                   | where the debt can be waived with |
|                                   | a reason code of 'LTF' (less than |
|                                   | fifty)                            |
|                                   |                                   |
|                                   | On the Assessment Finalisation    |
|                                   | (OPAF) screen, key 'Y' in the     |
|                                   | Finished? field. Press [Enter] to |
|                                   | finalise the activity             |
+-----------------------------------+-----------------------------------+
| 5                                 | Manually waiving a debt + Read    |
|                                   | more ...                          |
|                                   |                                   |
|                                   | If a DMN/NDB activity exists on   |
|                                   | the Activity List (AL) screen:    |
|                                   |                                   |
|                                   | -   'S'elect it                   |
|                                   | -   'S'elect the Debt ID to be    |
|                                   |     waived from the Debt List     |
|                                   |     (OPDL) screen                 |
|                                   | -   Press [Enter]                 |
|                                   |                                   |
|                                   | Go to the Add Waiver (OPWAA)      |
|                                   | screen and complete:              |
|                                   |                                   |
|                                   | -   Debt Amount Waived $: field,  |
|                                   |     code the amount to be waived  |
|                                   | -   Interest Charge Amount Waived |
|                                   |     $: field, cost the amount of  |
|                                   |     interest to be waived         |
|                                   | -   Reason Waived: field, use     |
|                                   |     field help ('?') to locate    |
|                                   |     the correct code              |
|                                   | -   Comments: field, code a brief |
|                                   |     explanation of why the debt   |
|                                   |     is being waived               |
|                                   | -   Press [Enter]                 |
|                                   | -   Finalise activity on the      |
|                                   |     Assessment Finalisation       |
|                                   |     (OPAF) screen                 |
|                                   |                                   |
|                                   | Note: it is not possible to waive |
|                                   | a debt and an interest charge     |
|                                   | within the same activity.         |
|                                   | Complete the update in 2 separate |
|                                   | activities.                       |
+-----------------------------------+-----------------------------------+
| 6                                 | Waive a finalised fully recovered |
|                                   | debt + Read more ...              |
|                                   |                                   |
|                                   | For DMIS to allow a waiver, the   |
|                                   | debt status must show as DET      |
|                                   | (determined). This is done by     |
|                                   | adjusting the repayment history.  |
|                                   |                                   |
|                                   | Over-recovered debts              |
|                                   |                                   |
|                                   | Do not tell the customer if a     |
|                                   | refund will be paid. The          |
|                                   | Compliance and Debt Operations    |
|                                   | Branch will decide this. They may |
|                                   | pay a refund or transfer the      |
|                                   | amount to other outstanding       |
|                                   | debts.                            |
|                                   |                                   |
|                                   | Non Debt staff                    |
|                                   |                                   |
|                                   | -   Record the details on a DOC   |
|                                   |     using Fast Note located in    |
|                                   |     Customer First. The Fast Note |
|                                   |     will automatically refer to   |
|                                   |     Compliance and Debt           |
|                                   |     Operations (NS7), requesting  |
|                                   |     the repayment history be      |
|                                   |     adjusted to allow the waiver  |
|                                   |     to be applied:                |
|                                   | -   Create a Fast Note, select    |
|                                   |     Auto Text. Use Debt           |
|                                   |     Recovery > Debt Recovery      |
|                                   |     referrals > OPRH Adj Req for  |
|                                   |     Debt Waiver                   |
|                                   | -   Select confirm                |
|                                   | -   Record relevant information   |
|                                   |     as required                   |
|                                   |                                   |
|                                   | Debt staff                        |
|                                   |                                   |
|                                   | -   Select the debt to waive from |
|                                   |     the Debt List (OPDL) screen   |
|                                   | -   Go to the Repayment History   |
|                                   |     (OPRH) screen. Adjust the     |
|                                   |     last payment by $0.01 cent.   |
|                                   |     Note: reverse this before     |
|                                   |     finalising the debt           |
|                                   |                                   |
|                                   | Once a waiver is complete, DMIS   |
|                                   | generates an over-recovered       |
|                                   | activity for the repaid amount.   |
|                                   | These activities automatically    |
|                                   | map to Compliance and Debt        |
|                                   | Operations to follow up whether a |
|                                   | refund is due to the customer.    |
