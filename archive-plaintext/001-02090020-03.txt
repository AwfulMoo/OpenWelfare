|                                   | The Customer Information field on |

|                                   | Summary (AEX) screen shows if a   |
|                                   | customer is Centrelink or         |
|                                   | provider managed. Note:           |
|                                   | volunteers must be registered as  |
|                                   | volunteer job seekers before a    |
|                                   | referral can be made.             |
|                                   |                                   |
|                                   | In Process Direct:                |
|                                   |                                   |
|                                   | -   select Customer Summary from  |
|                                   |     the Process Direct Landing    |
|                                   |     page                          |

|                                   |     'AEX' in the Super Key,       |

|                                   | -   the Activity and Exemptions   |
|                                   |     Summary (AEX) will display    |
|                                   |                                   |
|                                   | For:                              |
|                                   |                                   |
|                                   | -   Centrelink managed job        |
|                                   |     seekers, go to Step 2         |
|                                   | -   Provider managed job seekers, |

|                                   | Read more ...                     |
|                                   |                                   |
|                                   | The Service Officer must check    |
|                                   | the following screens in Process  |
|                                   | Direct before starting the        |
|                                   | referral process. This assists    |
|                                   | with determining eligibility:     |
|                                   |                                   |
|                                   | Citizenship and Visa sub class    |
|                                   |                                   |
|                                   | -   key 'RSLEG' in the Super Key, |

|                                   |     screen will display           |
|                                   |                                   |
|                                   | Permanent visa details            |
|                                   |                                   |
|                                   | -   key 'RSRD' in the Super Key,  |
|                                   |     select Enter                  |
|                                   | -   the Residence Results Display |
|                                   |     screen will display           |
|                                   |                                   |
|                                   | End date of the Newly Arrived     |
|                                   | Resident's Waiting Period (NARWP) |
|                                   |                                   |
|                                   | -   key 'RSPAP' in the Super Key, |
|                                   |     select Enter                  |
|                                   | -   the Period of Australian      |
|                                   |     Presence screen will display  |
|                                   |                                   |
|                                   | Job seekers with mutual           |
|                                   | obligation requirements:          |
|                                   |                                   |
|                                   | -   advise these job seekers of   |
|                                   |     their obligation to           |
|                                   |     participate in SEE, if        |
|                                   |     assessed as requiring         |
|                                   |     training at the Pre-Training  |
|                                   |     Assessment                    |
|                                   | -   use the Create SEE Referral   |
|                                   |     workflow in Process Direct to |
|                                   |     refer them to SEE and arrange |
|                                   |     a Pre-Training Assessment     |
|                                   |     appointment with the SEE      |
|                                   |     provider while the job seeker |
|                                   |     is present                    |
|                                   |                                   |
|                                   | See Step 1 in Table 2.            |

| 3                                 | Provider managed job seekers +    |
|                                   | Read more ...                     |
|                                   |                                   |
|                                   | Customer is eligible for SEE.     |
|                                   |                                   |
|                                   | -   Tell the job seeker they are  |
|                                   |     eligible to be referred to    |
|                                   |     SEE. If the job seeker agrees |
|                                   |     to the referral, contact the  |
|                                   |     job seeker's Employment       |
|                                   |     Services Provider             |
|                                   | -   Discuss any impact on the     |
|                                   |     requirements already in the   |
|                                   |     Job Plan, if a referral is    |
|                                   |     made to SEE. This discussion  |
|                                   |     can be with any staff member  |
|                                   |     of the provider, not just the |
|                                   |     job seeker's Case Manager     |
|                                   | -   If the provider has           |
|                                   |     reasonable grounds to oppose  |
|                                   |     a referral, tell the job      |

|                                   |     the referral                  |
|                                   |                                   |
|                                   | If the provider agrees, continue  |
|                                   | with the referral:                |
|                                   |                                   |
|                                   | -   Obtain the jobseeker's        |
|                                   |     consent to pass on their      |
|                                   |     personal details to the SEE   |
|                                   |     provider                      |
|                                   | -   Use the Create SEE Referral   |
|                                   |     workflow in Process Direct to |
|                                   |     refer the job seeker to SEE   |
|                                   |                                   |
|                                   | See Step 1 in Table 2.            |

|                                   | Read more ...                     |
|                                   |                                   |
|                                   | In Process Direct:                |
|                                   |                                   |
|                                   | key START in the Super Key        |
|                                   |                                   |
|                                   | the START screen will display     |
|                                   |                                   |
|                                   | from the Task Selector, select    |
|                                   | Create SEE Referral               |
|                                   |                                   |
|                                   | -   Note: Create SEE Referral     |
|                                   |     appears on the Referral       |
|                                   |     Summary (RRSUM) screen but is |
|                                   |     not a functional referral     |
|                                   |     option - only refer using the |
|                                   |     Create SEE Referral action    |
|                                   |     from the Task Selector        |
|                                   |                                   |
|                                   | the Referral effective date will  |
|                                   | pre-populate with today’s date.   |
|                                   | If the date is incorrect, select  |
|                                   | the calendar to enter the correct |
|                                   | date                              |
|                                   |                                   |
|                                   | from the Select referral type     |
|                                   | field, select LIT - Skills for    |
|                                   | Education and Employment          |
|                                   |                                   |
|                                   | select More Info to view more     |
|                                   | details about SEE to help discuss |
|                                   | the referral with the customer    |
|                                   |                                   |
|                                   | select Next                       |
|                                   |                                   |
|                                   | Note: If the customer is not      |
|                                   | eligible or has a current SEE     |
|                                   | referral in place, an error will  |
|                                   | display. To view the existing     |
|                                   | referral key 'RRSUM' in the Super |
|                                   | Key.                              |
|                                   |                                   |
|                                   | The Skills for Education and      |
|                                   | Employment Eligibility screen     |
|                                   | will display:                     |
|                                   |                                   |
|                                   | The following fields will         |
|                                   | pre-populate based on the         |
|                                   | customers circumstances:          |
|                                   |                                   |
|                                   | -   Customer is aged 15 to less   |
|                                   |     than Aged Pension age         |
|                                   | -   Customer is registered with   |
|                                   |     Centrelink and in receipt of  |
|                                   |     any payment or already        |
|                                   |     registered as a volunteer job |
|                                   |     seeker                        |
|                                   |                                   |
|                                   | Update the following fields by    |
|                                   | selecting Yes or No:              |
|                                   |                                   |
|                                   | -   Customer is suitable for      |
|                                   |     training without any barriers |
|                                   |     to participation              |
|                                   | -   Customer is either an         |
|                                   |     Australian citizen, permanent |
|                                   |     resident or has working       |
|                                   |     rights in Australia           |
|                                   |                                   |
|                                   | A message will display advising   |
|                                   | if the customer is eligible or    |
|                                   | not eligible for SEE              |
|                                   |                                   |
|                                   | If the job seeker is:             |
|                                   |                                   |
|                                   | -   eligible, select Next then go |
|                                   |     to Step 2                     |
|                                   | -   not eligible, go to Step 3    |
+-----------------------------------+-----------------------------------+
| 2                                 | Job seeker is eligible for SEE +  |
|                                   | Read more ...                     |
|                                   |                                   |
|                                   | The Service Provider screen will  |
|                                   | display all SEE service providers |
|                                   | and distance education providers  |
|                                   | in the job seeker's region. The   |
|                                   | Resources page contains a list of |
|                                   | distance education providers.     |
|                                   |                                   |
|                                   | Is a suitable provider or         |
|                                   | distance educator available?      |
|                                   |                                   |
|                                   | -   Yes, go to Step 5             |
|                                   | -   No, go to Step 4              |
+-----------------------------------+-----------------------------------+
| 3                                 | Job seeker is not eligible for    |
|                                   | SEE + Read more ...               |
|                                   |                                   |
|                                   | A job seeker is not eligible when |
|                                   | they:                             |
|                                   |                                   |
|                                   | are in New Enterprise Incentive   |
|                                   | Scheme (NEIS)/or Self Employment  |
|                                   | Assistance                        |
|                                   |                                   |
|                                   | are in Adult Migrant Education    |
|                                   | Program (AMEP)                    |
|                                   |                                   |
|                                   | have recently participated in SEE |
|                                   | and the last program exit date is |
|                                   | within 12 weeks of the next       |
|                                   | referral date:                    |
|                                   |                                   |
|                                   | -   When it's a new provider, the |
|                                   |     workflow completes and        |
|                                   |     creates a DOC advising the    |
|                                   |     date when the job seeker can  |
|                                   |     be referred back to SEE       |
|                                   | -   Job seekers wanting to go     |
|                                   |     back to their previous SEE    |
|                                   |     provider can contact the      |
|                                   |     provider themselves to        |
|                                   |     re-start SEE                  |
|                                   | -   When a Service Officer        |
|                                   |     attempts to code another      |
|                                   |     referral to the same provider |
|                                   |     within the 12 week period     |
|                                   |     will receive the following    |
|                                   |     error ''E409RR - Not          |
|                                   |     Eligible, can be referred to  |
|                                   |     SEE after <Due Date>'. These  |
|                                   |     job seekers are not eligible  |
|                                   |     for a new referral by         |
|                                   |     Services Australia before the |
|                                   |     date recorded against the     |
|                                   |     previous finalisation         |
|                                   |                                   |
|                                   | have recently successfully        |
|                                   | completed SEE. This is shown by   |
|                                   | the latest SEE status reason of   |
|                                   | Lang/Lit/Num Successful on the    |
|                                   | Referral Summary screen.          |
|                                   | Customers who have completed the  |
|                                   | program will be eligible again    |
|                                   | after a 12 week break             |
|                                   |                                   |
|                                   | for other reasons, the workflow   |
|                                   | completes and creates a DOC with  |
|                                   | the text Customer is not eligible |
|                                   | for SEE                           |
|                                   |                                   |
|                                   | Job seeker has changed address:   |
|                                   |                                   |
|                                   | The job seeker can:               |
|                                   |                                   |
|                                   | -   contact their previous        |
|                                   |     provider to advise they have  |
|                                   |     relocated. They should        |
|                                   |     request their record be       |
|                                   |     updated in the provider's     |
|                                   |     system with Transfer to new   |
|                                   |     provider. Once updated, a     |
|                                   |     referral to the new provider  |
|                                   |     is possible                   |
|                                   | -   wait until 12 weeks from      |
|                                   |     their last exit date, then    |
|                                   |     seek a new referral, or       |
|                                   | -   resume with the previous SEE  |
|                                   |     provider by contacting the    |
|                                   |     provider directly             |
|                                   |                                   |
|                                   | Job seeker wants to change SEE    |
|                                   | providers:                        |
|                                   |                                   |
|                                   | -   wait until 12 weeks from      |
|                                   |     their last exit date, then    |
|                                   |     seek a new referral           |
|                                   |                                   |
|                                   | Finalise the transaction:         |
|                                   |                                   |
|                                   | -   Select Next                   |
|                                   | -   The Referral Summary (RRSUM)  |
|                                   |     screen will display a         |
|                                   |     provisional line and list the |
|                                   |     referral ineligibility reason |
|                                   |     under the Status reason       |
|                                   |     column                        |
|                                   | -   Select Next                   |
|                                   | -   The Errors (SWE) screen will  |
|                                   |     display                       |
|                                   | -   Finalise the transaction by   |
|                                   |     selecting Assess again, then  |
|                                   |     select Finish. Record details |
|                                   |     of the update on the Finalise |
|                                   |     screen                        |
|                                   | -   Select Finalise to complete   |
|                                   |     the transaction               |
|                                   |                                   |
|                                   | Procedure ends here.              |
