Gathering information for the purposes of debt recovery

Table 2

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Consider garnishee options + Read |
|                                   | more ...                          |
|                                   |                                   |
|                                   | The system will not allow the     |
|                                   | cancellation of a No Arrangement  |
|                                   | (NOARR) activity until a payment  |
|                                   | arrangement is in place.          |
|                                   |                                   |
|                                   | If customer contact has been      |
|                                   | unsuccessful, consider if the     |
|                                   | debt can be recovered by          |
|                                   | garnishee.                        |
|                                   |                                   |
|                                   | Does the debt meet the criteria   |
|                                   | for a garnishee?                  |
|                                   |                                   |
|                                   | Yes, refer to the garnishee team. |
|                                   | Add keyword to the NOARR debt     |
|                                   | activity:                         |
|                                   |                                   |
|                                   | -   RCVGRNWG for wage garnishees, |
|                                   |     or                            |
|                                   | -   RCVGRNBK for bank garnishees  |
|                                   | -   Procedure ends here           |
|                                   |                                   |
|                                   | No, go to Step 2                  |
+-----------------------------------+-----------------------------------+
| 2                                 | Garnishee criteria not met + Read |
|                                   | more ...                          |
|                                   |                                   |
|                                   | Review the case to determine if   |
|                                   | it is suitable for other recovery |
|                                   | or non-recovery actions.          |
|                                   |                                   |
|                                   | Is the case suitable for other    |
|                                   | recovery or non-recovery?         |
|                                   |                                   |
|                                   | Yes, consider options:            |
|                                   |                                   |
|                                   | -   Temporary write off of        |
|                                   |     Centrelink debts, or          |
|                                   | -   Referring debts to an         |
|                                   |     External Collection Agent     |
|                                   |     (ECA)                         |
|                                   | -   Procedure ends here           |
|                                   |                                   |
|                                   | No, go to Step 3                  |
+-----------------------------------+-----------------------------------+
| 3                                 | Recovery options exhausted + Read |
|                                   | more ...                          |
|                                   |                                   |
|                                   | If recovery options have been     |
|                                   | exhausted, run the Fast Note:     |
|                                   |                                   |
|                                   | -   Select Auto Text. Use Debt    |
|                                   |     Recovery > Unsuccessful       |
|                                   |     Engagement > Recovery Options |
|                                   |     Exhausted                     |
|                                   | -   DOC the decision and include  |
|                                   |     the exhausted recovery        |
|                                   |     options                       |
|                                   | -   Link the NOARR activity to    |
|                                   |     the Fast Note - select 'Link  |
|                                   |     to existing activity' in the  |
|                                   |     Document Completion: field    |
|                                   | -   Complete the Fast Note        |
|                                   |                                   |
|                                   | Retain the work items and exit    |
|                                   | the customer's record:            |
|                                   |                                   |
|                                   | -   Retain the work item in the   |
|                                   |     user inbox until the Priority |
|                                   |     shows as '0' - this generally |
|                                   |     takes less than 20 minutes    |
|                                   |     once the user has exited the  |
|                                   |     customer record               |
|                                   | -   When the Priority resets to   |
|                                   |     '0', unassign the work item.  |
|                                   |     Use the reason 'Service       |
|                                   |     Support Referral'             |
|                                   | -   This prevents the activity    |
|                                   |     from re-allocation            |
