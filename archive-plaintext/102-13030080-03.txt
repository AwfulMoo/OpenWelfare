This page contains information to assist staff when determining if
multiple Customer Reference Numbers (CRNs) are for the same customer.

On this Page:

Same customer assessment - initial steps

Table1: This table describes information to assist staff to determine if
multiple Customer Reference Numbers (CRNs) are for the same customer.

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Do not update records + Read more |
|                                   | ...                               |
|                                   |                                   |
|                                   | Records with the same name and    |
|                                   | same date of birth do not         |
|                                   | necessarily belong to the same    |
|                                   | person.                           |
|                                   |                                   |
|                                   | If a Service Officer locates 2 or |
|                                   | more records potentially          |
|                                   | belonging to the same customer,   |
|                                   | they must be referred to the Data |
|                                   | Quality Unit (DQU) before         |
|                                   | completing any updates on the     |
|                                   | record(s). Use Fast Note - select |
|                                   | Auto Text, use DATA QUALITY >     |
|                                   | UPDATES > select required         |
|                                   | template. DQU will determine the  |
|                                   | primary CRN. If urgent assistance |
|                                   | is required contact the DQU       |
|                                   | helpdesk. See Resources for a     |
|                                   | link to the DQU Homepage.         |
|                                   |                                   |
|                                   | When different customers have the |
|                                   | same or similar details and the   |
|                                   | wrong customer record is updated  |
|                                   | the records become intertwined.   |
|                                   | No updates are to occur on        |
|                                   | intertwined records and, even if  |
|                                   | the incorrect data has been       |
|                                   | deleted, the CRNs must be         |
|                                   | referred to DQU using the         |
|                                   | Intertwined Referral Form. See    |
|                                   | Resources for a link.             |
+-----------------------------------+-----------------------------------+
| 2                                 | Follow up action required + Read  |
|                                   | more ...                          |
|                                   |                                   |
|                                   | The following records are not     |
|                                   | multiple CRNs:                    |
|                                   |                                   |
|                                   | Unauthenticated Online Entity     |
|                                   | (UOE)                             |
|                                   |                                   |
|                                   | Organisation (ORG)                |
|                                   |                                   |
|                                   | Duplicate CRN (DUP)               |
|                                   |                                   |
|                                   | Registered Care Provider (RCP)    |
|                                   |                                   |
|                                   | -   RCP records have a status of  |
|                                   |     RCP on the Benefit Status     |
|                                   |     line. No other information    |
|                                   |     should appear on the Benefit  |
|                                   |     Status line. If there is      |
|                                   |     additional data (for example, |
|                                   |     payment history) on the RCP   |
|                                   |     record, send a Multiple CRN   |
|                                   |     Referral Fast Note to the     |
|                                   |     DQU. Use Fast Note - select   |
|                                   |     Auto Text, use DATA QUALITY > |
|                                   |     UPDATES > select required     |
|                                   |     template                      |
|                                   |                                   |
|                                   | Are any of the records a UOE,     |
|                                   | ORG, DUP or RCP?                  |
|                                   |                                   |
|                                   | Yes,                              |
|                                   |                                   |
|                                   | -   If the customer has one CRN   |
|                                   |     and one UOE/ORG/DUP/RCP, a    |
|                                   |     same customer determination   |
|                                   |     is not required. Procedure    |
|                                   |     ends here                     |
|                                   | -   If there are two or more      |
|                                   |     records that are not          |
|                                   |     UOE/ORG/DUP/RCP types, go to  |
|                                   |     Step 1 in the Check Details   |
|                                   |     table                         |
|                                   |                                   |
|                                   | No, go to Step 1 in the Check     |
|                                   | Details table                     |

Same customer assessment - check details

Table 2: This table assists Service Officers to compare details for
records including scanned images, identity documents, etc., to determine
if multiple Customer Reference Numbers (CRN) are for the same customer.
Information on parent(s)/guardian(s)' record(s) may provide more
information.

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Customer contact + Read more ...  |
|                                   |                                   |
|                                   | Customer contact may be necessary |
|                                   | to establish if the records       |
|                                   | belong to the same person. Data   |
|                                   | Quality Unit (DQU) may request    |
|                                   | the Service Delivery Network      |
|                                   | (SDN) conduct a same customer     |
|                                   | assessment if there is minimal or |
|                                   | conflicting data recorded.        |
|                                   |                                   |
|                                   | If customer contact is required:  |
|                                   |                                   |
|                                   | -   Proof of record ownership     |
|                                   |     (PORO) must be established to |
|                                   |     avoid inadvertent disclosure  |
|                                   |     of personal information and   |
|                                   |     breaches of privacy. Service  |
|                                   |     Officers must never provide   |
|                                   |     customer information for the  |
|                                   |     individual to confirm. Refer  |
|                                   |     to Authenticating a           |
|                                   |     Centrelink customer for       |
|                                   |     assistance                    |
|                                   | -   Unless the CRN is a newly     |
|                                   |     created record with a CLI/MLC |
|                                   |     activity, customer contact    |
|                                   |     would only be appropriate if  |
|                                   |     the customer is claiming or   |
|                                   |     in receipt of a Centrelink    |
|                                   |     entitlement                   |
|                                   |                                   |
|                                   | The SDN are to record a DOC       |
|                                   | relating to each customer         |
|                                   | contact. Be mindful not to DOC    |
|                                   | details that could breach another |
|                                   | customer's privacy.               |
+-----------------------------------+-----------------------------------+

|                                   |                                   |
|                                   | Screens to check include (but are |
|                                   | not limited to):                  |
|                                   |                                   |

|                                   | (CPDS):                           |

|                                   |     maiden name, previous married |

|                                   |     Summary (ES) screen in        |
|                                   |     Customer First                |
|                                   | -   Take care when comparing      |

|                                   | similar names (for example,       |
|                                   | Alexander and Alexandra). Gender  |
|                                   | code may assist in determining a  |
|                                   | multiple birth as opposed to a    |
|                                   | multiple CRN.                     |

|                                   |     income (PIT/PAR), Assurance   |
|                                   |     of Support (AOE/AOC) and      |

|                                   |     see if there are additional   |
|                                   |     possible multiple CRNs to     |

Same customer assessment - determination

Table 3: This table describes the process to follow to finalise a same
customer assessment.

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Same customer determination +     |
|                                   | Read more ...                     |
|                                   |                                   |
|                                   | If discrepancies are found,       |
|                                   | consider if the records are for   |
|                                   | different people and/or are       |
|                                   | intertwined.                      |
|                                   |                                   |
|                                   | Do all the records belong to the  |
|                                   | same customer?                    |
|                                   |                                   |
|                                   | Yes,                              |
|                                   |                                   |
|                                   | -   Service Delivery Network      |
|                                   |     (SDN) staff, go to Step 2     |
|                                   | -   Data Quality Unit (DQU) staff |
|                                   |     only, go to Identifying a     |
|                                   |     customer's primary Centrelink |
|                                   |     Customer Reference Number     |
|                                   |     (CRN). Procedure ends here    |
|                                   |                                   |
|                                   | No, go to Step 4                  |
+-----------------------------------+-----------------------------------+
| 2                                 | CLI/MLC Activity + Read more ...  |
|                                   |                                   |
|                                   | Go to Activity List (AL)          |
|                                   |                                   |
|                                   | Is there a CLI/MLC activity       |
|                                   | present?                          |
|                                   |                                   |
|                                   | -   Yes, go to Actioning a Data   |
|                                   |     Quality Unit (DQU) request on |
|                                   |     a multiple Customer Reference |
|                                   |     Number (CRN). Procedure ends  |
|                                   |     here                          |
|                                   | -   No, go to Step 3              |
+-----------------------------------+-----------------------------------+
| 3                                 | Same customers + Read more ...    |
|                                   |                                   |
|                                   | SDN Service Officers, refer the   |
|                                   | CRNs to DQU using Fast Note -     |
|                                   | select Auto Text, use DATA        |
|                                   | QUALITY > UPDATES > select        |
|                                   | required template.                |
|                                   |                                   |
|                                   | Procedure ends here.              |
+-----------------------------------+-----------------------------------+
| 4                                 | Intertwined records + Read more   |
|                                   | ...                               |
|                                   |                                   |
|                                   | Are the records intertwined?      |
|                                   |                                   |
|                                   | Yes, refer to the DQU Intertwined |
|                                   | Team via the Intertwined Referral |
|                                   | Form. See Resources for a link.   |
|                                   |                                   |
|                                   | No,                               |
|                                   |                                   |
|                                   | -   SDN staff, procedure ends     |
|                                   |     here                          |
|                                   | -   DQU staff only, go to Step 5  |
+-----------------------------------+-----------------------------------+
| 5                                 | Different customers + Read more   |
|                                   | ...                               |
|                                   |                                   |
|                                   | DQU Service Officers only:        |
|                                   |                                   |
|                                   | If a Multiple CRN Referral Fast   |
|                                   | Note has been received annotate   |
|                                   | the referral with the outcome     |
|                                   | ensuring there is no breach of    |
|                                   | privacy.                          |
|                                   |                                   |
|                                   | For twins/multiple birth          |
|                                   | customers, record a Display on    |
|                                   | Access (DOA) DOC confirming the   |
|                                   | customer is a twin. If a customer |
|                                   | has twin children ensure a DOA    |
|                                   | DOC is recorded in the relevant   |
|                                   | parent(s)/guardian(s) record(s)   |
|                                   |                                   |
|                                   | For different people who share    |
|                                   | the same or similar details,      |
|                                   | record a DOA DOC advising the SDN |
|                                   | that additional proof of identity |
|                                   | questions are required to ensure  |
|                                   | updates are made for the correct  |
|                                   | customer.                         |
