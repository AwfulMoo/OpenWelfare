|                                   | comments for normal business):    |
|                                   |                                   |

|                                   |                                   |

|                                   | decision date with the register.  |

|                                   |                                   |

|                                   | example scenarios and how to      |
|                                   | action these.                     |

discrepancies relating to new registrations where no Donor Only
registration exists.

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Identified major discrepancy +    |
|                                   | Read more ...                     |
|                                   |                                   |
|                                   | This is where a major discrepancy |
|                                   | in personal details has been      |
|                                   | identified between an organ donor |
|                                   | registration form and either:     |
|                                   |                                   |
|                                   | -   the linked organ donor        |
|                                   |     entitlement, or               |
|                                   | -   consumer record               |
|                                   |                                   |
|                                   | Where the discrepancy is the      |
|                                   | surname:                          |
|                                   |                                   |
|                                   | -   refer to Confirm previous     |
|                                   |     name, see Link Donor Only     |
|                                   |     entitlement to consumer       |
|                                   |     record                        |
|                                   |                                   |
|                                   | If the discrepancy still exists   |
|                                   | or is not resolved, or is a DOB   |
|                                   | major discrepancy, go to Step 3.  |
|                                   |                                   |
|                                   | Note: if the Service Officer is   |
|                                   | proficient in the Medicare Public |
|                                   | processing team, follow the       |
|                                   | current Medicare protocols and    |
|                                   | policies.                         |

|                                   | more ...                          |
|                                   |                                   |

Major discrepancy for AODR TAS processing staff

Table 3: describes how AODR process all major discrepancies relating to
new registrations where no Donor Only registration exists.

+-----------------------------------+-----------------------------------+
| Step                              | Action                            |
+-----------------------------------+-----------------------------------+
| 1                                 | Medicare entitlement match + Read |
|                                   | more ...                          |
|                                   |                                   |
|                                   | If:                               |
|                                   |                                   |
|                                   | the Medicare number is provided   |
|                                   | on the registration form, but all |
|                                   | personal details have not been    |
|                                   | provided on the form:             |
|                                   |                                   |
|                                   | -   use the personal details on   |
|                                   |     the Medicare entitlement for  |
|                                   |     the AODR entitlement          |
|                                   |     registration                  |
|                                   |                                   |
|                                   | one or more of the personal       |
|                                   | details provided on the form      |
|                                   | varies from those in the Consumer |
|                                   | Directory Maintenance System      |
|                                   | (CDMS), go to Step 2              |
+-----------------------------------+-----------------------------------+
| 2                                 | Personal details + Read more ...  |
|                                   |                                   |
|                                   | Check for any comments previously |
|                                   | entered about any changes in the  |
|                                   | name and/or details mentioning    |
|                                   | signed documentation. Update the  |
|                                   | comments and sighted              |
|                                   | documentation or escalate to the  |
|                                   | AODR PSO Team if unsure.          |
|                                   |                                   |
|                                   | This will prevent staff from      |
|                                   | sending an AODR letter for a      |
|                                   | prior processing error.           |
|                                   |                                   |
|                                   | If the:                           |
|                                   |                                   |
|                                   | -   first name varies, go to Step |
|                                   |     3                             |
|                                   | -   date of birth varies, go to   |
|                                   |     Step 4                        |
|                                   | -   gender varies, go to Step 5   |
|                                   | -   address varies, go to Step 6  |
|                                   | -   group and personal name vary, |
|                                   |     go to Step 7                  |
|                                   | -   the family name varies, go to |
|                                   |     Step 8                        |
+-----------------------------------+-----------------------------------+
| 3                                 | First name + Read more ...        |
|                                   |                                   |
|                                   | Different spelling including      |
|                                   | abbreviations to the name in CDMS |
|                                   | and other scenarios should be     |
|                                   | taken as an exact match and the   |
|                                   | registration processed and linked |
|                                   | to the Medicare record in         |
|                                   | accordance with the Minor         |
|                                   | discrepancies in Medicare         |
|                                   | Enrolment File procedures. Record |
|                                   | comment in the AODR entitlement   |
|                                   | comment field 'AODR from received |
|                                   | first name also known as <list    |
|                                   | name>                             |
|                                   |                                   |
|                                   | Note: for Medicare purposes a     |
|                                   | person can be known by a name     |
|                                   | other than their legal name.      |
|                                   | Where a person presents Medicare  |
|                                   | with documentation that confirms  |
|                                   | their 'known in the community'    |
|                                   | name, this name will be changed   |
|                                   | at 'Group' level only. The        |
|                                   | person's legal name is still      |
|                                   | recorded at 'Personal' level the  |
|                                   | AODR card will issue with the     |
|                                   | name as it appears in 'Personal'  |
|                                   | level of CDMS. If the person      |
|                                   | rings and asks for the name to be |
|                                   | changed refer to Change legal     |
|                                   | name on Medicare record.          |
+-----------------------------------+-----------------------------------+
| 4                                 | Date of Birth + Read more ...     |
|                                   |                                   |
|                                   | Date of birth variations should   |
|                                   | be taken to be an exact match,    |
|                                   | where:                            |
|                                   |                                   |
|                                   | All other details match that is,  |
|                                   | first name, last name, gender and |
|                                   | address:                          |
|                                   |                                   |
|                                   | -   process the registration as   |
|                                   |     an AODR entitlement linked to |
|                                   |     Medicare using the date of    |
|                                   |     birth recorded in Medicare,   |
|                                   |     and the date of birth         |
|                                   |     variation is considered to be |
|                                   |     a 'minor discrepancy'. See    |
|                                   |     also: Amend a person's        |
|                                   |     Medicare enrolment details    |
|                                   |                                   |
|                                   | Where the discrepancy is not      |
|                                   | considered a 'Minor discrepancy': |
|                                   |                                   |
|                                   | -   send the appropriate AODR     |
|                                   |     letter ‘Date of birth details |
|                                   |     differ to CDMS’ and finalise  |
|                                   |     Note: where there are 2       |
|                                   |     people on the one Medicare    |
|                                   |     card:                         |
|                                   | -   with the same name, and       |
|                                   | -   it is obvious that the dates  |
|                                   |     of birth have been switched,  |
|                                   |     attempt one call only. This   |
|                                   |     is to seek clarification from |
|                                   |     the individual prior to       |
