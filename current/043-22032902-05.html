Apprenticeships Data Management System (ADMS)

Information for employers on gaining access to ADMS

Logging into ADMS

  • Select the ADMS hyperlink
  • On the ADMS landing page, select the Services Australia logon hyperlink
  • When prompted, select the text field and key 'Services Australia'. Select Services Australia from the menu of options, and press Continue
  • Select the Process Claim Applications button. The BAC/CAC or PWS forms page displays

Note: the ADMS system will time out after a period of inactivity

System access for authorised users

Table 1

Item

Description

1

Requesting eSAM and Apprentice Data Management System (ADMS) access

Access to ADMS is administered and approved by DEWR for Services Australia Service Officers who require access to view and verify evidence for AWS claims.

Complete eSAM registration before requesting ADMS access.

If already registered for eSAM, for example, if using Access Apprenticeships, see Item 3.

Access to ADMS is suspended after 40 days of inactivity. To request reactivation of ADMS access, see Item 3.

Access to eSAM is removed after 90 days of inactivity. If eSAM access is removed, re-register for eSAM and request ADMS reactivation.

To register for eSAM:

  • go to the eSAM Portal
  • select VANguard SSO, then Sign in to go to the Authentication Service screen
  • key 'Services Australia' into the search field when prompted to select 'Which organisation do you represent' and select Services Australia from the options presented (do not select Services Australia – Protected)
  • select Continue to go to the User Security Declaration Acceptance screen
  • after reading the User Security Declaration, tick the I Accept box
  • select Submit

2

eSAM Home page

On the eSAM home page, add a primary phone number:

  • select My details
  • in the Primary phone number box, enter Team Leader’s full desk phone number, including area code
  • select Submit

A green banner stating User Updated successfully will display.

Note: if staff do not record a primary phone number in eSAM, they will not be granted access to ADMS.

The My details section displays the ESS User ID. Take note of this as it must be provided when requesting access to ADMS.

Use of personal information:

After registering in eSAM, personal details in My details (for example name and phone number) are disclosed to DEWR.

DEWR will collect this information to verify identity to determine whether access can be provided to ADMS.

DEWR may use the information to manage and administer ADMS, protect against unauthorised access, verify security problems and ensure compliance with the relevant policies.

3

Request ADMS access

Email Apprenticeship Programmes advising:

  • eSAM registration is completed and ADMS access is required, or
  • ADMS access has been suspended and reactivation is required

Include in the email the ESS User ID and the date of eSAM registration when requesting access to ADMS. The ESS User ID is located on the My Details page in eSAM.

Staff will receive a confirmation email when ADMS access is granted or reactivated.

The ESS User ID is used to log into ADMS.

ADMS claim status

Table 2

Status

Definition

Ready

Claim application has been generated and is ready to be actioned by the employer

Draft

Employer has commenced updating the claim application, but has not yet submitted

Submitted

Claim application has been submitted by employer for assessment

Approved

Claim application approved

Rejected

Claim application rejected

Returned

Claim application has been returned to employer by a Service Officer for more information

Verified

Claim application marked eligible (either through automation or by a Service Officer), pending completion of the DEWR delegation approval process. Once this process completes, the status is updated to Approved

Assessment

Claim application has been submitted and requires manual assessment by a Service Officer
