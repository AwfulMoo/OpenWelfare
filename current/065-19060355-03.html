Reviewing the application

Table 1

Step

Action

1

Assessment of the Scheme application + Read more ...

Prior to commencing processing the Scheme application, determine if:

  • the applicant meets the eligibility requirements, or if the application is to be rejected
  • all required documents have been provided
  • a real estate valuation has been returned or one was completed in the last 12 months for the Australian real estate offered as security for the loan

See Assessing Home Equity Access Scheme applications for information on how to conduct the assessment. If another officer has already conducted the assessment, a Scheme claim streaming Note will display on the customer’s record. To check the notes in Process Direct select the Notes icon.

Has the Scheme application been assessed and is ready to process?

  • Yes, go to Step 2
  • No, the Scheme application cannot be processed until the assessment is complete. Go to Assessing Home Equity Access Scheme applications

2

Check payment frequency + Read more ...

Check the customer’s record to see if they receive their Scheme qualifying payment (if any) as fortnightly or weekly payment.

Does the customer receive their qualifying payment as a weekly payment?

  • Yes:
    • loan payments cannot be granted if the qualifying payment is paid weekly
    • Discuss with the customer if they want to change their pension to fortnightly payments. See Assessing, coding and reviewing weekly payments for assistance in making decisions to cease weekly payments, with consideration to whether changing to fortnightly payments will have unintended results or put the customer under financial stress
  • No, go to Step 3

3

Review Work Item + Read more ...

Review the claim information in Process Direct, in readiness to begin processing the Scheme application, including (but not limited to):

  • check the customer and application information on the Transaction Summary (TS) screen
  • check the information entered in the claim by the customer
    • for claims submitted via the online channel or ACC this is available in the Claim Summary
    • for paper claims this is available in the SA310 Home Equity Access Scheme partnered application, or SA496 Home Equity Access Scheme single application in the Documents section
  • check the claim Status (if the application is on hold for any reason, it may not be ready to process)
  • confirm that all customer and staff Tasks have been completed
  • review the information in the Documents provided by the customer
  • read the Notes on the record

Also check the Customer Summary in Customer First for customer specific information and warnings that may impact the application process. The Customer Summary alerts staff to any issues before processing the claim or contacting the customer. This includes checking:

  • if the customer is income managed. See Identifying and assisting income managed customers for assistance
  • the Customer Incident Management System (CIMS)
  • if an active Managed Service Plan (MSP) exists which may include service channel restrictions. See Customer aggression - Managed Service Plan (MSP) for more information

Note: combined partner online claims are 1 claim work item. The steps staff complete for the customer must also be followed within the same work item, for the partner.

Once ready to commence processing the application, see Table 2.

Processing the Scheme applications in Process Direct

Table 2

Step

Action

1

Commence processing + Read more ...

Select Process to start processing the application. The SAP Warning and Errors (SWE) screen will display with a Message Log and Task Selector.

Note: any coding updates made from this point will be provisionally held in the application for assessment. To exit the application at any time:

  • press Back until the Transaction Summary (TS) screen displays, or
  • enter TS to the Super Key field to return to the Transaction Summary screen, or
  • select Home to return to the Process Direct landing page

See Process Direct navigation, common screens and functions for information on how to navigate and update details for the customer and partner.

Go to Step 2

2

Message Log + Read more ...

The Message Log on the SWE screen displays any errors with existing coding. Error codes starting with E, X or S must be fixed before the claim can be progressed.

Warnings and information messages (codes starting with a W or I) may also display but do not necessarily require action to be taken. Review any warnings and messages and determine what action may be required before proceeding with the application.

To view more information about a message, select it or enter the message code to the Roxy Digital Assistant (DA). To launch the DA, select the Question Mark icon at top right of the screen. The DA will populate details of any error presented on the screen it is launched from. See Process Direct navigation, common screens and functions for more information about the DA.

Go to Step 3.

3

Select tasks + Read more ...

The Task Selector on the SWE screen lists the common screens that may need reviewing and updating as part of the claim. For information about each of the listed screens see Task Selector screens for Scheme in Process Direct.

Select the tasks to be reviewed, noting the following should be selected at a minimum:

  • PNA – Pensions Assessment Date
  • PLS – Pension Loans Scheme
  • REBS – Real Estate/Business Identity Details
  • PAS – Payment Destination Details
  • TFN – Tax File Number Authorisation

For customers applying for an advance payment as part of their claim, also select the HEAS Advance (HEASADV) screen.

For loan-only claims, also select the residence tasks:

  • RSIMME – Immigration Enquiry
  • RSLEG – Residence Details
  • CRES – Country of Residence
  • RSIM – Immigration Movements

Screens with a should also be selected, as the flag indicates that a provisional coding update has been applied to the screen within the application.

Once the Task Selector has been updated, select Next to start navigating through the selected screens. Screens can also be accessed directly via the Super Key field.

Go to Step 4

4

Code application details + Read more ...

Review each of the selected screens to verify they are coded correctly. Make updates to include any new information the customer has provided in their Scheme application.

Note: if the Scheme application was lodged online or via Assisted Customer Claim (ACC), most information provided by the customer will pre-populate to the applicable screen. Check the claim information to ensure the details have populated correctly and make updates as required. If the application was lodged via the SA310 or the SA496 paper form, or the Social Application (SOA) was manually created, all new information must be manually coded.

The following coding is to be applied where relevant:

  • For grant outcomes:
    • Set the Effect Date on the HEAS Details (PLS) screen to the Entitlement Period Start Date (EPSD) of the current entitlement period, so that loan arrears are not generated. The loan start date is only backdated if the customer specifically requests it
    • For advance-only claims, (that is, if the customer has not applied for fortnightly loan payments) record advance details on the HEAS Advance (HEASADV) screen
    • For loan-only claims, the date on the Pension Assessment Date (PNA) screen must match the Effect Date on the PLS screen
    • Where the secured real estate has both assessable and non-assessable components, the components must be coded separately on the Real Estate/Business Summary (REBS) screen to ensure allowable deductions under the pension assets test are apportioned
  • For rejection outcomes:
    • To manually reject the application, code the rejection reason on the PLS screen. Do not reject the Scheme via the Benefit Action (BA) screen
    • If the customer has claimed an advance payment only (without a fortnightly loan payment), the advance payment rejection reason must be recorded via the (Advance Claim Rejected) HEAS Advance (HEASADV) screen. Based on this coding the system will set the Reject/Ceased reason code on the PLS screen to ARJ
    • To identify if a manual rejection reason should be applied, see Assessing Home Equity Access Scheme applications

For help coding specific screens, see Resources for guidance.

When navigation of the selected screens is completed, the application will return to the SWE screen and the Assess button will be enabled. Individual screens can be viewed and updated again if needed by selecting Next, or by navigating directly to a screen using the Super Key field.

Once all coding is complete, go to Step 5.

5

Assess application + Read more ...

To initiate the system assessment for the application, select Assess.

Selecting Assess will send a transaction to the legacy system to create a claim activity and upload the provisional claim information to the customer’s record. For reference purposes only, the activity type used in the legacy system for applications is as follows:

  • PEN/LON if the customer is already current on a Scheme qualifying payment
  • AGE/NCL for loan-only applications where the applicant is of Age Pension age
  • DSP/NCL or CAR/NCL for loan-only applications where the applicant is under Age Pension age. The pension type is based on the Scheme qualifying payment the applicant has chosen to test their eligibility against

If errors are identified in the legacy claim activity, these will be returned to Process Direct and displayed in the Message Log on the SWE screen for action. Review error messages and take action within the application in Process Direct to resolve them.

Once complete, select Assess again to confirm the errors have been resolved. Note: each time the Assess button is selected, the provisional claim activity created in the legacy system is cancelled and new activity created.

Have all errors been resolved?

  • Yes, go to Step 6
  • No, if all coding is correct and an error continues to persist, raise an incident via the Roxy Digital Assistant. The application will be automatically placed on hold pending resolution of the issue. Process ends here pending ICT advice.

6

Entitlement (ELD) screen + Read more ...

If there are no further errors, the Entitlements (ELD) screen will display with the provisional claim outcome. A row will display for the Scheme with a benefit status of CUR (grant outcome) or REJ (rejection outcome).

Is the provisional outcome a grant?

  • Yes, go to Step 8
  • No, it's a rejection, go to Step 7

7

Rejection outcome + Read more ...

Where the application results in a rejection outcome, the ELD screen displays the PLS benefit status as PLS/REJ with a corresponding reason for rejection. The benefit status of the qualifying payment displays in a separate row (for example, AGE/CUR).

Review the provisional outcome to check that the rejection outcome and rejection reason are correct. See Home Equity Access Scheme cancellation and rejection codes for a full list of rejection reasons.

Is the rejection outcome (including the reason) correct?

  • Yes, go to Step 11
  • No, go to Step 9

8

Grant outcome + Read more ...

Where the application results in a grant outcome, the ELD screen displays the benefit status as PLS/CUR. The benefit status of the qualifying payment displays in a separate row (for example, AGE/CUR). The calculated rate of payment displays alongside the qualifying payment (including for loan--only claims).

Check the outcome, effect dates and payment information for correctness, including (if applicable) the fortnightly loan rate (LON component) the advance payment (LONA component) and the pension components (for example, BASIC, PSBAS, PSREM, PSMIN, CES). Payment and rate information can be accessed via the Super Key field, relevant screens include:

  • Customer Daily Rate Component (RAC)
  • Daily Rates Summary (RATS)
  • Pension Rate Calculator (PRC)
  • Factors Affecting Rate (PFAR)
  • Payment Summary (PS)
  • Payment Details (PD)
  • Payment Details Component (PDC)

See Home Equity Access Scheme for information about the fortnightly rate of the Scheme loan payments, including how the Scheme rate is impacted by the qualifying payment rate.

For loan-only applications that result in a grant outcome, double check the following:

  • HEAS Only Claim indicator on the HEAS Details (PLS) screen is 'Y'
  • date on the Pension Assessment Date (PNA) screen matches the Effect Date on the PLS screen
  • the LON component (for fortnightly loan payments) and LONA component (for advanced loan payments) are the only rate components. No pension components should be created for loan-only applications

Is the grant outcome, rate and Effect Date correct?

  • Yes, go to Step 10
  • No, go to Step 9

9

Incorrect or unexpected outcome + Read more ...

If the provisional outcome on the ELD screen is unexpected or incorrect, select Back and check the claim coding is correct. Use the Roxy Digital Assistant (DA) for assistance to resolve errors and edits. Make any necessary coding updates, then Assess the claim again to review the revised outcome.

For help determining correct assessment outcomes see:

  • Assessing Home Equity Access Scheme applications
  • Coding Home Equity Access Scheme rejections

A SAP refresh may be required if:

  • data from ISIS displays incorrectly or not at all in Process Direct
  • data has been updated in the customer’s record in legacy after the application was submitted

For instructions on how to conduct a SAP refresh, see Process Direct navigation, common screens and functions.

If all coding is confirmed to be correct and an incorrect or unexpected outcome continues to occur, raise a new incident via the Roxy DA. The application will be automatically placed on hold pending resolution of the issue. Procedure ends here pending ICT advice.

10

Project loan balance + Read more ...

The Home Equity Access Scheme grant and projection calculator is used to project the Maximum Loan Amount (MLA), loan balance and accrued interest. The purpose of projecting the balance is to provide the customer with an estimate of future balance and repayment obligations to ensure they can make an informed decision before agreeing to proceed with the loan. The projection is a point-in-time estimate only based on the information available at the time of grant.

Access the Home Equity Access Scheme grant and projection calculator and enter the customer’s details to project the loan balance. Save a copy of the projection as a PDF document and upload it to the customer’s record using classification code UNS005.

Go to Step 11.

11

Contact third parties + Read more ...

Where the real estate offered by the customer as loan security:

  • is the asset of an individual private trust, verify the details of the trustee(s) against the trust. If the details are unable to be verified or there is doubt that the appropriate person(s) signed Part D of the SA310/SA496, contact must be made with the person(s) prior to granting the loan
  • is the asset of a corporate trust or company, conduct a trust and company search to verify the details of the director and secretary. If the details are unable to be verified or there is doubt that the appropriate person(s) signed Parts E-F of the SA310/SA496, contact must be made with the person(s) prior to granting the loan
  • has any other registered proprietors such as third party co-owners of the property (not including partners), contact must be made with the person(s) to seek verbal assurance that they willingly signed the SA310/SA496 Part C declaration and understand a loan is going to be taken out on the property

Where contact is required, make genuine attempts to contact the relevant person(s) to advise the following:

  • A loan is going be taken out against the property and a caveat or charge will be lodged on the title
  • The loan and caveat/charge may present limitations in taking out future loans on the property or selling it until the loan is repaid
  • Current and future projection of the Maximum Loan Amount (MLA)
  • The loan balance will accrue compound interest each fortnight
  • Where a guarantee declaration is provided under Parts D-F of the SA310/SA496, the trust or company undertakes to pay the PLS loan and agrees for a charge to be placed on the real estate

Have the relevant third parties been successfully contacted?

  • Yes:
    • update the Progress of Claim (POC) Note to record the details of the discussion
    • go to Step 12 unless the person requests to withdraw the written declaration they provided in the SA310 Part C-F. Ask for the withdrawal to be provided in writing and add an allow for the notice to be received. Place the application on hold. Process ends here pending due date for return of information
  • No:
    • where genuine contact attempts have been unsuccessful, update the Progress of Claim (POC) Note to record the details of each unsuccessful contact attempt
    • go to Step 12

12

Contact customer + Read more ...

Before finalising the application, the Decision Maker (DM) must contact the customer to discuss the outcome.

Rejection decision

Before rejecting the application, the DM must contact the customer to provide an explanation of the decision and an opportunity for the customer to provide further information. Action as follows:

  • Make two genuine attempts to contact the applicant
  • Advise the applicant of the rejection outcome and explain their review and appeal rights
  • Record the outcome of each contact (including unsuccessful contacts) in the Progress of Claim (POC) Note

For more information see Advising verbally of an unfavourable decision.

Grant decision

Where the application can be granted, the DM must contact the customer to conduct a pre-grant interview. Home Equity Access Scheme loans must not be granted until the interview has been conducted. Action as follows:

  • Make genuine attempts to contact the customer
  • If the customer does not have a valid contact number or nominee, issue a written request via Q888 letter to the customer asking them to contact the agency in relation to their application. If contact is not made within 14 days, the application is to be rejected FRC (failure to respond to correspondence).
  • Record details of unsuccessful contact attempts in the Progress of Claim (POC) Note
  • Upon successful contact with the customer, proceed to conduct the interview. See Resources for guidance to conduct the interview including mandatory topics
  • Upon successful completion of the interview, run the HEAS application interview Fast Note, located at Fast Note - select Auto Text, use Older Australians > Home Equity Access Scheme > HEAS application interview
  • Add any relevant points of discussion outside of the Fast Note content to ensure the conversation is well documented

If applicable (for both grant and rejection outcomes), make any final coding updates to the Scheme application based on the discussion with the customer.

Once the customer contact requirements have been completed, go to Step 13 to finalise the application.

13

Before Limiting Date (BLD) + Read more ...

If a row displays on the ELD screen with a status of BLD, manual adjustment of the rate for the applicable benefit and period is required before the application can be finalised.

A BLD status generally occurs if a provisional coding update has resulted in a reassessment from an Effect Date that is before the pension Limiting Date (for example, a change to historical residence information).

Does a BLD line display on the ELD screen?

  • Yes, action the manual rate adjustment:
    • Using the Super Key field, go to the Daily Rate Component (RAC) screen
    • The rows requiring manual adjustment will state BLD in the Man Adj Reason field
    • Review the details for each row and enter the manual adjustment in the $ Man Adj field. In most cases the adjustment will be $0 but this must be assessed on an individual basis
    • Go to Step 14
  • No, go to Step 14

14

To finalise the application, select Finish.

A pop-up window will display with pre-populated comments relevant to the determination outcome, including:

  • Outcome of claim (grant or reject and reason)
  • Claim ID
  • Payment type
  • Payment start date
  • Date of grant
  • Date of lodgement
  • Customer contact date (if applicable)
  • Act reference
  • Payment rate/arrears payments/advance payment

Review the details and update the information as needed, then add the following information. Remove any details not relevant to the customer:

  • Security property address:
  • Legal Description:
  • Valuation of security:
  • Deductions:
  • Nominated Amount:
  • Maximum Loan Amount:
  • Customer advised Maximum Loan Amount:
  • Fortnightly loan amount:
  • Advance payment amount
  • Customer <and partner> has confirmed their understanding of the terms and conditions and has agreed to proceed with the application
  • The secured real estate is co-owned with a third party / the asset of a trust or company. <XXXXXXXXXX> has agreed in writing, on the application form, to allow this property to be used as security to enable receipt of payments under the Scheme
  • CAO details:

The comments will be automatically recorded as a Note on the customer’s record upon completion of the application.

Select Finalise to complete the application. A dialog box will display and confirm when the application has been successfully finalised.

Upon finalisation of the application, a letter will be automatically generated and sent to the customer to advise of the outcome (grant or rejection). The letter will display on the History Summary (HS) screen the day after completing the activity. Note: the system does not generate a rejection letter if the rejection reason is DEA (death of customer).

Was the application granted?

  • Yes:
    • Create a Display on Access (DOA) DOC by using Fast Note - select Auto Text, use Older Australians > Home Equity Access Scheme > Home Equity Access Scheme DOA DOC. Review and update the text to suit the customer’s circumstances
    • Set up a Manual Follow Up (MFU) for the annual review of the real estate used as loan security unless the property is exempt under the assets test for the customer’s ISP. For instructions see Home Equity Access Scheme reviews
    • Submit a request to register a caveat or charge on the real estate asset(s) accepted as loan security. For instructions see Table 4
    • If arrears are issued and exceed the pensions system limit amount, set up an MFU to reissue the payment in instalments after the payment rejects
  • No, it was rejected, procedure ends here

Processing the Scheme claims in the legacy system

Table 3: how to process the Scheme applications in Customer First. This is only required in contingency situations.

Step

Action

1

Contingency processing + Read more ...

The Scheme claims are processed in Process Direct CPTv5. Processing in legacy is available as a contingency option, to be applied only if:

  • an agency instruction has been issued to process the Scheme applications in legacy, or
  • endorsement to use legacy for an individual application is provided by the Business Process and Change Branch (Older Australians)

Requests to process the Scheme claims in legacy are to be referred to the Older Australians Business Processing team. It must be demonstrated that all other avenues to process the application in Process Direct have been explored and are not suitable. Contingency processing is a last resort when no other options are available.

System errors and edits are generally not a valid reason to invoke the contingency process. If the Scheme application is affected by an error or edit, raise an incident via the Roxy Digital Assistant (DA). If the ICT resolution advice is to process the claim in legacy, seek approval from Older Australians Business Processing before proceeding.

Has proper approval been provided to process the Scheme application in legacy?

2

Access legacy system + Read more ...

Is the application for a Scheme-only loan?

  • Yes, go to Step 3
  • No, go to Step 4

3

Loan-only claims + Read more ...

The Scheme is assessed as a rate component of the Home Equity Access Scheme qualifying payment, therefore to process the Scheme application, a new claim (NCL) activity must be indexed for the qualifying payment.

  • If the customer is of Age Pension age, the qualifying payment type is Age Pension (AGE)
  • If the customer is under Age Pension age, the qualifying payment type is either Disability Support Pension (DSP) or Carer Payment (CAR), depending on which payment the customer nominated in their Scheme application

To index the pension claim, go to the Name Search (IN) screen in the Pension System (PEN) and code the following fields:

  • ServRsn: AGE, DSP or CAR (enter applicable payment type)
  • Act: NCL
  • Srce: OLC for applications lodged online or via ACC, otherwise INT
  • DOR: date the Scheme claim was submitted
  • Press [Enter] to commence the claim

Go to Step 5.

4

Pension recipients + Read more ...

For customers who already receive a Scheme qualifying payment, the Scheme application is processed in legacy as a PEN/LON activity. To create a PEN/LON activity, go to the HEAS Details (PLS) screen and code the loan information, ensuring the source and date are coded as:

  • Srce: SA310/SA496
  • DOR: enter the date the Scheme application was submitted

For help coding all other fields on the PLS screens, see Resources.

Go to Step 5.

5

Screen selection + Read more ...

For loan-only claims, also select the residence tasks:

  • Residence Task Selector (RETS)

Go to Step 6.

6

Code application details + Read more ...

Review each of the selected screens to verify they are coded correctly. Make updates to include any new information the customer has provided in their Scheme application. No information from the Scheme application is pre-populated when processing in legacy, so all new information must be manually coded.

The following coding is to be applied where relevant:

For grant outcomes:

  • Set the Effect Date on the HEAS Details (PLS) screen to the Entitlement Period Start Date (EPSD) of the current entitlement period, so that loan arrears are not generated. The loan start date is only backdated if the customer specifically requests it
  • For advance-only claims (that is, where the customer has not applied for fortnightly loan payments) the advance details must be recorded on the HEAS Advance (HEAA) screen
  • For loan-only claims:
    • the HEAS Only Claim indicator must be set to 'Y' on the PLS screen
    • the date on the Pension Assessment Date (PNA) screen must match the Effect Date on the PLS screen
  • Where the secured real estate has both assessable and non-assessable components, the components must be coded separately on the Real Estate/Business Summary (REBS) screen to ensure allowable deductions under the pension assets test are apportioned

For rejection outcomes:

  • For loan-only claims, the HEAS Only Claim indicator must be set to 'Y' on the PLS screen
  • To manually reject an application, code the manual rejection reason on the PLS screen. Do not reject the claim via the Benefit Action (BA) screen
  • If the customer has claimed an advance payment only (without a fortnightly loan payment), the advance rejection reason must be recorded via the HEAS Advance (HEAA) screen. Based on this coding the system will set the Reject/Cease reason code on the PLS screen to ARJ (Advance Claim Rejected)

  • To identify if a manual rejection reason should be applied, see Assessing Home Equity Access Scheme applications

See Resources for help coding specific screens.

Once all coding is complete, navigate to the Assessments Results (AR) screen.

Go to Step 7.

    • the provisional status will be CUR (current) or REJ (reject) in the PHLS.LOAN.STS.CODE field
    • for rejection outcomes, a provisional rejection reason code will display in the PHLS.LOAN.RSN.CODE field

Is the provisional outcome a grant?

  • Yes, go to Step 10
  • No, it's a rejection, go to Step 9

Grant outcome + Read more ...

Relevant screens to view the payment and rate information include:

12

Project loan balance + Read more ...

The Home Equity Access Scheme grant and project calculator is used to project the Maximum Loan Amount (MLA), loan balance and accrued interest. The purpose of projecting the balance is to provide the customer with an estimate of future balance and repayment obligations to ensure they can make an informed decision before agreeing to proceed with the loan. The projection is a point-in-time estimate only based on the information available at the time of grant.

Access the grant and projection calculator and enter the customer’s details to project the loan balance. Save a copy of the projection as a PDF document and upload it to the customer’s record using classification code UNS005.

Go to Step 13.

13

Contact third parties + Read more ...

Where the real estate offered by the customer as loan security:

  • is the asset of an individual private trust, verify the details of the trustee(s) against the trust. If the details are unable to be verified or there is doubt that the appropriate person(s) signed Part D of the SA310/SA496, contact must be made with the person(s) prior to granting the loan
  • is the asset of a corporate trust or company, conduct a trust and company search to verify the details of the director and secretary. If the details are unable to be verified or there is doubt that the appropriate person(s) signed Parts E-F of the SA310/SA496, contact must be made with the person(s) prior to granting the loan
  • has any other registered proprietors such as third party co-owners of the property (not including partners), contact must be made with the person(s) to seek verbal assurance that they willingly signed the SA310/SA496 Part C declaration and understand a loan is going to be taken out on the property

Where contact is required, make genuine attempts to contact the relevant person(s) to advise the following:

  • A loan is going be taken out against the property and a caveat or charge will be lodged on the title
  • The loan and caveat/charge may present limitations in taking out future loans on the property or selling it until the loan is repaid
  • Current and future projection of the Maximum Loan Amount (MLA)
  • The loan balance will accrue compound interest each fortnight
  • Where a guarantee declaration is provided under Parts D-F of the SA310/SA496, the trust or company undertakes to pay the PLS loan and agrees for a charge to be placed on the real estate

Have the relevant third parties been successfully contacted?

Ask for the withdrawal to be provided in writing and an allowance for the letter to be received. Place the application on hold. Process ends here pending due date for return of information
  • No, where genuine contact attempts have been unsuccessful, update the Progress of Claim (POC) Note to record the details of each unsuccessful contact attempt. Go to Step 14

Contact customer + Read more ...

Before finalising the application, the Decision Maker (DM) must contact the customer to discuss the outcome.

Rejection decision:

Before rejecting the application, the DM must contact the customer to provide an explanation of the decision and an opportunity for the customer to provide further information. Action as follows:

  • Advise the applicant of the rejection outcome and explain their review and appeal rights
  • Record the outcome of each contact (including unsuccessful contacts) in the Progress of Claim DOC via the Document List (DL) screen

Grant decision:

Where the application can be granted, the DM must contact the customer to conduct a pre-grant interview. The Scheme loans must not be granted until the interview has been conducted. Action as follows:

  • If the customer does not have a valid contact number or nominee, issue a written request via Q888 letter to the customer asking them to contact the agency in relation to their Scheme application. If contact is not made within 14 days, the application is to be rejected FRC (failure to respond to correspondence).
  • Record details of unsuccessful contact attempts in the Progress of Claim DOC
  • Upon successful contact with the customer, proceed to conduct the interview. See Resources for guidance to conduct the interview
  • Upon successful completion of the interview, use Fast Note - select Auto Text, use Older Australians > Home Equity Access Scheme > HEAS application interview
  • Add any relevant points of discussion outside of the Fast Note content to ensure the conversation is well documented

Before Limiting Date (BLD) + Read more ...

If a row displays on the AR screen with a status of BLD (for example, AGE/BLD), manual adjustment of the rate for the applicable benefit and period is required before the activity can be finalised.

A BLD status generally occurs if a provisional coding update has resulted in a reassessment from an Effect Date that is before the pension Limiting Date (for example, a change to historical residence information).

Does a BLD line display on the AR screen?

  • Yes, action the manual rate adjustment:
    • Go to the Daily Rate Component (RATS) screen by entering 'T' next to the row with the BLD status
    • Go to the RAC screen by entering 'R' next to the BLD row
    • Review the details for each row and enter the manual adjustment in the $ Man Adj field. In most cases the adjustment will be $0 but this must be assessed on an individual basis
    • Go to Step 16
  • No, go to Step 16

  • Enter 'Y' to the Finished field and press [Enter]
