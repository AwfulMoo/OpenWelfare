Manual action required when system cannot finalise a Disability Support Pension (DSP) new claim

If a Job Capacity Assessment (JCA) and Disability Medical Assessments (DMA) report recommends medical eligibility, the system tries to accept the JCA report and grant the DSP new claim once a DMA report has been accepted.

If all non-medical eligibility is not coded before accepting the DMA, the system may not finalise the claim or the customer may be incorrectly granted.

If the system cannot finalise the DSP new claim activity, it matures it to Due and adds a keyword to explain why the claim could not be finalised. It may generate a work item for processing staff to manually finalise the DSP claim.

When the issue is resolved, the system will not try again to finalise the claim. If must be finalised manually.

Keywords to generate a work item to finalise DSP new claim

Table 2

Keyword

Activity note

Action

DSPDMAER

DMA data unable to populate customer record.

The data from the DMA report cannot be populated to the MC screen because there is an error. Contact the GCD Contract Management team for assistance.

DSPERRJC

Job Capacity Assessment (JCA) link failed, accept report and action record.

The system cannot auto-accept the JCA report. The Delegate needs to manually accept the JCA report. The system will update the MC and WC screens with the JCA data only. The Delegate will then need to manually finalise the claim.

If the JCA Report cannot be accepted because it is still locked due to DMA Referral, but DMA has been accepted, see Table 4, Step 2 on the Process page,

DSPPOSRQ

DMA report advises 20 pts, Program of Support (POS) assessment required, action as per Operational Blueprint.

DMA Report varies JCA recommendations and a POS assessment is required before claim can be finalised. Seek a POS assessment as per Operational Blueprint. Once POS assessment is received, manually finalise the claim.

DSPDMAOP

Open DMA referral exists on the customer's record and DSP claim is unable to be finalised.

The system cannot accept the DMA report as there is an open DMA referral on the customer's record or required triggers have not yet run to invoke auto JCA acceptance, auto medical cluster screen updates or the auto finalisation process. Claims are unable to be finalised until the DMA report is accepted.

To action a DMA referral, see Disability Medical Assessment (DMA) referrals.

The keyword is only added to the record if the user attempts to manually finalise the claim. Auto claim finalisation would never be invoked if there is an open DMA or if the triggers had not yet run.

DSPDMACV

No DMA completed within the last 2 years (no current and valid DMA).

The keyword occurs when a Delegate is attempting to manually grant a DSP claim and the claim cannot be granted as there is no current and valid DMA on the customer's record.

For assistance, lodge an online enquiry via the Level 2 Policy Helpdesk.

DSPCITW

JCA report is accepted with manifest eligible code and CITW value in the report is not AUS.

JCA indicates customer meets manifest medical eligibility but CITW is outside Australia.

SO needs to determine if CITW occurred in AUS, update PDI screen and manually finalise the DSP claim.

See Granting Disability Support Pension (DSP) (manifest).
