Record a new COA application

Steps for recording a new COA application

Table 1: This table describes the process steps to record a new COA application.

Step

Action

1

Start point - Record a new COA application + Read more ...

Start point: COA Maintenance window

2

Record the application details + Read more ...

  • In the Applicant group box, select the applicant
  • The Lodgement Date in the COA Details group box will default to the current date and can be changed by overtyping. (This should be the date the application was received by CSP). This auto populates the Expected Finalisation Date to 35 days after the lodgement date
  • The Application status drop down list defaults to None. Until the case is allocated to a decision maker, the status will remain as None

To add the reasons and sub reasons:

  • click on the button to open the Applicable Reasons Selection List window
  • select a reason and then select the associated sub reason (if Reason 8 or RICA application), and click on the Add to List button. This will add the reason to the Reasons list

If more reasons are required, repeat the above for each reason.

To remove a reason, highlight the reason in the Reasons list and click the Remove from list button.

Note: the Selected Reasons field can be updated at any time until the COA is finalised.

Click the OK button to return to the COA Maintenance window.

3

Save the application details + Read more ...

Click on the icon to save the record. A confirmation message will appear advising the applicant and lodgement date cannot be changed once saved.

If No is selected you will be returned to the COA Maintenance window, and the applicant/application details can be changed as described in Step 2.

If Yes is selected, the details will be saved and the following will occur:

  • the Application # field will display a unique number
  • a COA Application in Progress (COAINPRO) intray is generated with a due date 35 days later
  • a COA Case Assessment Required (COAASREQ) intray is generated with a due date of 48 hours, and
  • a COA Awaiting SCO Allocation (COAWTSCO) intray is generated (if a decision maker is yet to be selected) with a due date of 48 hours

If at the time of saving, the other customer has a current COA application for the case, which has not been finalised, the This is a cross-application tick box is automatically ticked (on the new application).

4

Further applications received + Read more ...

If at some time before finalisation, the respondent also lodges a COA application for the same case, the Cross-Application exists for this COA tick box is automatically ticked (on the original application). If this occurs, the lodgement date of that application will automatically be recorded in the Cross Application Lodgement Date field (on the original application).

If the COA applicant has a COA in progress for another case/s, the for Applicant tick box will automatically be ticked.

If the COA respondent has a COA in progress for another case/s, the for Respondent tick box will automatically be ticked.

5

Record case complexity + Read more ...

During the initial review of the application the complexity of the case will be determined.

Select the complexity of the case from the Case Assessment drop down list.

  • Click the icon to open the Notes window and create a note recording the case assessment information
  • Click the icon to save the record

Upon saving the COA Case Assessment Required intray is deleted and a COA Initial Contact Required (COAINCON) intray is generated.

6

The application is complete, eligible and a reason is established + Read more ...

If the application is complete, eligible and a reason is established, set the Application Status to App to proceed.

This will auto generate the COA 5-1 Invitation to respond letter to the respondent and the COA 5-2 Acknowledgement of COA application letter to the applicant.

7

Assign the case to a decision maker + Read more ...

To assign the case to a decision maker, in the Decision Maker Details group box:

  • select a location from the COA Site drop down list. This customises the options in the Officer drop down list to provide a list of officers in the selected location
  • select the officer from the Officer drop down list
  • select the Level of the Officer from the Level drop down list (select External if allocating to an External SCO)
  • the Allocation Date field will auto populate with the date you are entering the decision maker details

The COA Awaiting SCO Allocation intray is deleted.

8

Route the intray + Read more ...

  • Click on the icon to exit the COA Maintenance window
  • Route the COA Initial Contact Required intray to the selected decision maker

Record customer contact for COA applications

Steps for recording customer contact for COA applications

Table 2: This table describes the process steps for the COA decision maker to record customer contact for COA applications.

Step

Action

1

Start point - Record customer contact for COA applications + Read more ...

Start point: COA Maintenance window

2

Select the customer type and the type of contact established + Read more ...

Click the Customer Contact tab of the COA Maintenance window.

Select the customer type and the type of contact established from the Action drop down list in the Customer Contact group box. Select:

  • Payee/Payer contact for initial and subsequent conversations
  • Issue CAL - Payee/Payer to generate CAL letter
  • Payee/Payer outcome contact for contact discussing outcome of application
  • Payee/Payer post discussion to discuss decision after NTD/letters have issued

3

Record the contact details + Read more ...

Click the Record button to open the Customer Contact dialogue box.

  • The Date and Time fields are auto populated
  • Select an option from the Phone Number drop down list. This includes options for outbound and inbound calls
  • Select an outcome from the Outcome drop down list
  • If contact is unsuccessful, select a reason for the outcome from the Reason drop down list
  • Click the OK button to save this information and return to the COA Maintenance window's Customer Contact tab

Note: once saved, the information is displayed in summary form in the COA Contact Summary group box in the Customer Contact tab.

This box will provide a summary only of contact between the decision maker and the customers. The contact recorded by decision makers will progress the status of COA intrays on the case. All documentation of conversations and actions taken on the case are to be recorded in the COA Maintenance window notepad.

4

Record if further information is to be provided + Read more ...

If the customer is to provide further information, in the Pending Information from group box:

  • click on the tick box pertaining to the relevant customer
  • directly underneath this field, select the awaited information from the Information requested drop down list
  • the Due Date field defaults to 7 days from the current date and generates a COA Pending Information intray, and
  • when received, key the date the information was received in the Received Date field. This will delete the COA Pending Information intray

Note: only 1 pending info action can be recorded for each party at any time.

5

Record received response details + Read more ...

In the Response group box, update the relevant section depending on whether it relates to the Payee Response or Payer Response.

  • When a copy of an application and a response booklet (CAR) is sent to the customer, in the CAR sent date field, key in the date this was sent
  • When a CAR (response) is received from the respondent key the date it was received by CSP in the CAR Received Date field and select the method of communication from the Response Method drop down list
  • When a CAR is actioned (copy sent to the applicant), key in the date in the CAR actioned Date field

When the CAR received and actioned dates are entered a COA 6-1 Advise response received letter will generate to the applicant advising we have received a response.

Note: if the response method selected is verbal no letter will generate.

6

Document and save the details + Read more ...

  • the icon to open the Notes window to create a note recording the details of the customer contact
  • Click on the icon to save the record
  • Click on the icon to exit the COA Maintenance window

Finalise a COA application prior to setting application status

Steps for finalising a COA application prior to setting status to application to proceed

Table 3: This table describes the process steps to finalise a COA application when the decision is made prior to setting the Application Status to App to Proceed.

Step

Action

1

Start point - Finalise COA application prior to setting status to Application to Proceed + Read more ...

Note: generally, finalising a COA application should be processed from the Finalisation window. However, you are only able to access the Finalisation window in the first instance if the Application Status on the COA Maintenance window is set to App to Proceed.

Subsequent access is allowed if there were previously recorded outcomes on the Finalisation window, that is, variations, refusals, withdrawals or agreement outcomes.

Start point: COA Maintenance window

2

Select the appropriate outcome + Read more ...

Select an outcome from the Application Status drop down list in the Application Details tab.

If the Status selection is:

  • Incomplete - go to Step 3
  • Inel-Agreement - Inel-Crt Order, Inel-No Assess or Inel-Other, go to Step 4
  • Poss Refusal - go to Step 5

When the applicant decides to withdraw the application before you speak to the respondent, go to Step 6.

When the applicant decides to withdraw the application after you speak to the respondent, the withdrawal must be processed from the Finalisation window. See the Steps for finalising a COA application after setting the status to application to proceed table.

3

COA application incomplete + Read more ...

When Incomplete is selected:

  • click on the enabled button to open the Incomplete Reasons dialogue box
  • highlight the appropriate reason(s) and click OK
  • click the SCS Information button to open the SCS Information dialogue box. Refer to Specialist Client Services (SCS) Information Cuba Process Help for instructions
  • click the icon to save the record. The icon will then be disabled. In the event the decision is referred for an amendment, the Save icon will be enabled again

Saving will generate a COA 7-1 Incomplete COA Application letter to the applicant. Go to Step 7.

4

COA application ineligible + Read more ...

When Inel-Agreement, Inel-Crt Order, Inel-No Assess or Inel-Other is selected:

  • click the SCS Information button to open the SCS Information dialogue box. See Specialist Client Services (SCS) Information Cuba Process Help for instructions
  • click the icon to save the record. The icon will then be disabled. In the event the decision is referred for an amendment, the Save icon will be enabled again
  • click OK on the Confirmation dialogue box

Saving will generate a COA 3-1 Ineligible application letter to the applicant. Go to Step 7.

5

COA application possible refusal + Read more ...

When Poss Refusal is selected:

  • click the icon to save the record. The icon will then be disabled. In the event the decision is referred for an amendment, the Save icon will be enabled again
  • click on the Finalisation button to open the Finalisation window
  • type in the NTD Return date in the NTD Return date textbox
  • click the icon to save the record. Go to Step 7

6

Applicant withdrawal prior to respondent contact + Read more ...

  • select W'drawn-Estimat or W'draw-Other as appropriate
  • click the SCS Information button to open the SCS Information dialogue box. See Specialist Client Services (SCS) Information Cuba Process Help for instructions
  • click the icon to save the record. The icon will then be disabled. In the event the decision is referred for an amendment, the Save icon will be enabled again

Saving will generate a COA 6A-1 COA Application has been withdrawn letter to the applicant.

Note: W'drawn-Estimat must not be used if the applicant is RICA and the payee is in receipt of Centrelink benefits.

7

Document the decision + Read more ...

  • Click the icon to open the Notes window to create a note recording the details of the customer contact
  • Click the icon to save the record
  • Click on the icon to exit the COA Maintenance window

Finalise a COA application after setting application status

Steps for finalising a COA application after setting the status to application to proceed

Table 4: This table describes the process steps to finalise a COA application after setting the Application Status to App to Proceed.

Step

Action

1

Start point - Finalising a COA application after setting the status to application to proceed + Read more ...

Note: generally, finalising a COA application should be processed from the Finalisation window. However, you are only able to access the Finalisation window in the first instance if the Application Status on the COA Maintenance window is set to App to Proceed.

Subsequent access is allowed if there were previously recorded outcomes on the Finalisation window, that is, variations, refusals, withdrawals or agreement outcomes.

Start point: Finalisation window

2

Add additional successful reasons (if required) + Read more ...

If the Notice of Decision (NTD) includes reasons that are not highlighted in the Highlight Successful Reasons group box:

  • close the Finalisation window to return to the COA Maintenance window
  • add additional reasons according to Step 2 in Record a new COA application, and
  • return to the Finalisation window

3

Record the Notice of Decision date + Read more ...

In the NTD Return Date textbox, type the date the SCO returned the NTD to the CSP office.

Note: the date must be later than the application Lodgement Date recorded on the COA Maintenance window and earlier than or equal to the current date. Unless the application is finalised on the same date the NTD is returned, the NTD Return Date must be different to the Outcome Date.

4

Select the appropriate outcome + Read more ...

Select an outcome from the Outcome drop down list.

If the Outcome selected is:

  • Agree Accepted - go to Step 5
  • Refusal-Before or Refusal-After - go to Step 6
  • Var-Increase or Var-Decrease - go to Step 7
  • W'drawn-Other or W’drawn-Estimat, go to Step 8

5

Agreement Accepted + Read more ...

When Agree Accepted is selected:

  • type the date the COA was finalised in Cuba in to the Outcome Date and Agreement Date fields. The Agreement Date must be greater than the application lodgement date recorded on the COA Maintenance window and cannot be later than the current date
  • click the SCS Information button to open the SCS Information dialogue box. See Specialist Client Services (SCS) Information Cuba Process Help for instructions
  • click the icon to save the record. The icon will then be disabled. In the event the decision is referred for an amendment, the Save icon will be enabled again

Saving will generate a COA10-1 Acceptance of an agreement after request for a Change of Assessment letter to both parties. Go to Step 9.

6

Refusal decision made + Read more ...

When Refusal-Before or Refusal-After is selected:

  • in the Outcome Date textbox, type in the date of the outcomes. The outcome date must be the date the application is being progressed (the current date)
  • select a reason from the Outcome Reasons drop down list
  • click the SCS Information button to open the SCS Information dialogue box. See Specialist Client Services (SCS) Information Cuba Process Help for instructions
  • click the icon to save the record. The icon will then be disabled. In the event the decision is referred for an amendment, the Save icon will be enabled again

Saving will generate a COA9-1 Refusal decision made and NTD attached letter to both customers. Go to Step 9.

7

Variation decision made + Read more ...

When Var-Increase or Var-Decrease is selected:

  • in the Outcome Date textbox, type in the date of the outcomes. The outcome date must be the date the application is being progressed (the current date)
  • from the Parental Agreement drop down list select whether the parents agreed or whether it was a SCO decision (this should be documented in the NTD)
  • click the SCS Information button to open the SCS Information dialogue box. See Specialist Client Services (SCS) Information Cuba Process Help for instructions
  • click the icon to save the record. The icon will then be disabled. In the event the decision is referred for an amendment, the Save icon will be enabled again

Saving will generate a COA8-1 Advice of COA decision letter to both customers.

If a change to the assessment needs to be processed in the Formula Modification window (which is accessed through the Case/Registration window) the formula modification must be applied to the originating case and any reverse case.

  • a COA Decision Variation intray is generated with a due date of 14 days

If the Outcome selected is Var-Increase due to Reason 8 (payer income), a COA Reason 8 Inc intray is generated. The Customer Management Approach (CMA) for Child Support should be followed in this instance. Go to Step 9.

8

Application withdrawn after respondent contact + Read more ...

When W'drawn-Other or W’drawn-Estimat is selected:

  • in the Outcome Date textbox, type in the date of the outcome. The outcome date must be the date the application is being progressed (the current date)
  • click the SCS Information button to open the SCS Information dialogue box. See Specialist Client Services (SCS) Information Cuba Process Help for instructions
  • click the icon to save the record. The icon will then be disabled. In the event the decision is referred for an amendment, the Save icon will be enabled again

If the applicant is the payee or payer, a COA 6A-1 COA application has been withdrawn letter will be generated.(See Step 6 of Steps for finalising a COA application prior to setting status to application to proceed table for processing a withdrawal where the respondent has not been contacted).

If the applicant is RICA, a COA 17-1 Acceptance of joint election to withdraw from RICA process letter needs to be manually generated and issued to both customers from the Letter Details window.

9

Document the decision + Read more ...

  • Click the icon to open the Notes window to create a note recording the details of the customer contact
  • Click the icon to save the record
  • Click on the icon to exit the COA Maintenance window

Refer a COA application to be considered for amendment

Steps for referring a COA application to be considered for amendment

Table 5: This table describes the process steps to refer a COA application to be considered for amendment.

Step

Action

1

Start point - Refer a COA application to be considered for amendment + Read more ...

Start point: Finalisation window

2

Process the Finalisation + Read more ...

In the Referred For Amendment group box:

  • select the decision maker from the drop down list
  • the Referred Date defaults to the current date and the Expected Reply Date to 14 days from current date

Based on Decision selected, the following apply:

  • If the decision selected is Amend No Change or No Amend No Chg; the Outcome group box is not enabled. Go to Step 4
  • If the decision selected is Revoke, Revoke and Change, or Amend and Change; the Outcome group box is enabled and you must record a new outcome. In practice, the outcome could be the same as the original outcome. However, the outcome can only be a Var-Increase or Var-Decrease. Go to Step 5

3

Record the date the decision was made + Read more ...

Type in the date the SCO made the new decision in the Decision Date textbox.

4

Record the processing date + Read more ...

Type in the date the outcome is being processed (current date) in the Outcome Date textbox.

5

Save the finalised record + Read more ...

  • Click the icon to save the decision details of the referral for amendment
  • Click the icon to exit the Finalisation window

Create a new SCO record

Steps for creating a new SCO record

Table 6: This table describes the process steps to create a new SCO record.

Step

Action

1

Start point - Create a new SCO record + Read more ...

Start point: Senior Case Officer Maintenance window

2

Create a new SCO record + Read more ...

To create a new SCO or CSO5 Senior Service Officer decision maker record, in the Senior Case Officer Find dialogue box:

  • select a location from the COA Site drop down list
  • click the Find button
  • if an existing SCO is displayed, refresh the window by clicking the icon. This clears all the fields

3

Record the officer's details + Read more ...

In the Officer group box:

  • select a location from the COA Site drop down list
  • type the name of the SCO/Senior Service Officer in the Name field
  • type the start date of the contract in the Start Date field, and
  • type the end date of the contract in the Start Date field (if end date is unknown leave blank)

4

Record the officer's address details + Read more ...

In the Address group box:

  • type the street address in Line 1 and Line 2
  • type the suburb in the Suburb field
  • select the State from the State drop down list
  • type the postcode in the Postcode field. Go to Step 5

If you are unsure of the postcode, click on the icon to open the Suburb to Postcode Find dialogue box.

  • Type the known information in the relevant fields
  • Click the Find button. This reveals the Suburb to Postcode List window
  • Select the appropriate record from the Suburb to Postcode List group box
  • Click the icon to append the address record to the SCO record and return to the Senior Case Officer Maintenance window

5

Record the officer's contact details + Read more ...

In the Phones and Email Address group boxes, type in the phone numbers and email address of the SCO.

6

Save the new record + Read more ...

Click on the icon to save the record. The new record will now appear in the Senior Case Officers group box.

7

Document any further information + Read more ...

To add a note, select the record from the Senior Case Officers group box, then click the icon to open the Notes window.

Locate a SCO record

Steps for locating a SCO record

Table 8: This table describes to process steps to locate a SCO record in Cuba.

Step

Action

1

Start point - Locate a SCO record + Read more ...

Start point: Senior Case Officer Maintenance window

2

Search options to locate the record + Read more ...

To search the latest end date of a SCO's record:

  • in the Senior Case Officer Find dialogue box type the date you require in the Officer With Schedule Period End Date After field
  • from the drop down list select the COA site

Delete a SCO record

Steps for deleting SCO record

Table 9: This table describes the process steps to delete a SCO record from Cuba.

Step

Action

1

Start point - Update a SCO record + Read more ...

Start point: Senior Case Officer Maintenance window

2

Locate the SCO record + Read more ...

To locate the record to be updated, in the Senior Case Officer Find dialogue box:

  • select the officer's location from the COA Site drop down list

Locate SCO report

Steps for locating a SCO report

Table 10: This table describes the process steps to locate a SCO report.

Step

Action

1

Start point - Locate a SCO report + Read more ...

Start Point: Senior Case Officer Reports Window Help

2

Conduct search + Read more ...

To search by:

Senior Case Officer

  • select a location from the COA Site drop down list
  • select a venue from the Venue drop down list. This customises the Senior Case Officer drop down list (option disabled)
  • select a SCO from the Senior Case Officer drop down list. This enables the Find button
  • go to Step 3.

Case Number

  • type in a case number in the Case Number field. This disables all other fields
  • go to Step 3

User Generated reports

  • select the User Generated radio button from the Report Status group box. This disables all other fields
  • go to Step 3

Possible refusals:

  • select the Possible Refusal radio button from the Report Status group box. This only enables the COA Site field. It is optional to select a location
  • go to Step 3

3

Select the report + Read more ...

  • Click the Find button. This reveals the Senior Case Officer Reports window with the search results
  • Highlight the required record and click the icon to print the report
  • Click on the icon to exit the Senior Case Officer Reports window

4

Generate a manual report + Read more ...

If there is no SCO Report available, a report can be manually generated.

  • From the customer's COA Maintenance window, select File
  • Click on Request SCO report. This will manually generate a SCO report overnight, which will be available the following day via the find function as detailed from Step 1
