Registering a new hospital claim

Table 1: This table describes the steps required to register a new hospital claim.

Step

Action

1

Log in CICS Production + Read more ...

Key ‘VRDI’ and press [Enter].

The claim registration screen displays.

2

Enter registration information + Read more ...

Enter the required information in the following fields:

  • Payee provider (only when shown on the claim form)
  • Service provider
  • Claim ID
  • DOL
  • Claim date
  • Vouchers quoted
  • Claim type (H – for in-patient private hospital claim: R – for public hospital claim for example, prosthesis and private non-contracted hospital)

Press [Enter].

See the Resources page for more information about the field descriptions.

Details of the servicing provider, and payee provider if applicable, display.

‘U TO UPDTE’ message will also display.

3

Check for errors + Read more ...

Check the details displayed to ensure the correct provider numbers have been keyed. If errors exist:

  • correct the errors
  • press [Enter]
  • over-key VRDC with VRDU and press [Enter] to update and start entering the claim details

4

Access hospital data entry screen + Read more ...

Over-key VRDC with ‘VRDU’ and press [Enter].

The Hospital data entry screen is displayed. The type of data entry screen displayed depends on the type of hospital claim being processed.

Hospital date entry (VKHC) screen

Table 2: This table describes how a Service Officer enters data on the VKHC or H screen.

Step

Action

1

Manually register the claim + Read more ...

Complete Manual claim registration procedure. See table Register a new hospital claim.

2

Display Hospital data entry (VKHC) screen + Read more ...

Take notice of the type of hospital claim being processed:

  • 'R' for non-contracted hospital where approval is present or where restrictions prevent payment of a private hospital claim according to the General Private Hospital Processing Rules
  • 'H' for private hospital

Key ‘H’ or ‘R’ in CLAIM TYPE field on either the:

  • Data Entry Menu (VDEI) screen, or
  • Combined Claim Registration and Data Entry Menu (VRDC) screen

Note: when keying ‘R’ as a claim type, the Data Entry screen VKRC displays instead of VKHC.

3

Enter claim details + Read more ...

Enter details from the claim documentation in the following fields and select [Enter].

  • File #
  • Name
  • DOS
  • To
  • No
  • Item
  • Amount
  • A/C No
  • RR
  • R
  • AD
  • STB
  • TT
  • PG

Not all fields are mandatory. Where mandatory information is unavailable, key‘?’ in the required field to PEND the claim for further assessing.

Not all the fields above will be displayed when using ‘R’ as a claim type.

See also Claims for Veterans' Affairs Processing (VAP).

The Resources page contains a list of field names and the information that can be used in each field of the hospital data entry screens.

4

Close claim + Read more ...

Select [PF3] to close claim once data entry is complete.

Identifying, accessing and resolving claims

Table 3: This table describes how to identify, access and resolve claims using the Pend Resolution Basic screen.

Step

Action

1

Identify claim + Read more ...

Identify a claim with the status of ASS/ERR.

For more information see Pend Resolution in Claims for Veterans' Affairs Processing (VAP)

2

Enter details + Read more ...

Key either:

  • ‘VAPI’, PROVIDER NUMBER,CLAIM ID
  • ‘VQOI’ CLAIM ID
  • ‘VQRF’, CLAIM ID

Press [Enter].

Note: where a payee provider is involved, the provider number keyed in the control line is that of the payee and not the service provider.

Where there is more than one claim with the same Claim ID, the system returns the message ‘More lines’. Select the correct claim, by keying ‘S’ in the S field and select [Enter].

The Pend processing screen is displayed.

3

Check for errors + Read more ...

The system highlights any errors in the claim and displays return messages that can assist in resolving these errors. Place the cursor on the message and select [PF1] for instructions on resolving the error.

When no errors can be found the system returns message ‘NO MORE ERRORS OK TO PAY’.

4

Pay the claim + Read more ...

Over-key VAPC with ‘VPPI’ and press [Enter] then when ‘No Error’ message is displayed key ‘P’ and press [Enter] to pay the claim.

Adding a voucher

Table 4: This table describes the procedure for adding a voucher.

Step

Action

1

Adding a voucher + Read more ...

When working out the amount that needs to be entered on a split accommodation line either:

  • use the Department of Veterans’ Affairs (DVA) Hospital Contracts Enquiry HCVS/VAMM system to obtain the rate that applies when a private hospital is involved
  • check the claim form and invoice for daily bed charges

2

Access Pend Resolution Insert screen + Read more ...

From the Pend Resolution Basic screen:

  • over-key the VAPC with ‘VAPI’
  • add the modifier A to the end of the control line (for example, VAPI, 123456A, C7890, DOL=101017, A
  • press [Enter]

The message ‘INSERT’ displays and the Pend Resolution Insert screen is available.

Note: when a voucher needs to be inserted and you are not currently on the Pend Resolution Basic screen, the full VAPI control line may be used.

3

Allocate new voucher + Read more ...

Enter the service details for the voucher and select [Enter].

The system allocates the new voucher a position in the claim and the message ‘INS VCH OK’ is displayed.

4

Enter details + Read more ...

Enter the following details:

  • file number in the FILE NO field
  • first name in the PATIENT field
  • item number in the ITEM field
  • accommodation From Date in the DOS field
  • payee code 6 in the P field
  • charge in the AMOUNT field

Press [Enter]

The system displays the message 'INS VCH OK'.

5

Display messages + Read more ...

Press [Enter].

The system displays the messages:

  • ‘SWAP SCREEN’ – PF11 to swap to calculation screen
  • ‘1-INV BULK’ – correct the NO field
  • ‘1-FLD REQ’ – insert missing information on Swap screen
  • ‘1-LST DT’ – the ‘date to’ is required

6

Calculate amount payable + Read more ...

Enter the following details:

  • accommodation To Date
  • number of days
  • amount
  • any other adjustments required and press [Enter]

The system calculates the amount payable.

Step

Action

1

2

Enter details + Read more ...

Enter the following details:

  • ‘CERT’ in the ITEM field
  • commencement date of acute care period in the DOS field

Press [Enter].

3

System message displayed + Read more ...

Press [Enter].

The system displays the message ‘1-LST DT’ and highlights the TO field on the CERT line.

Enter end date details + Read more ...

Enter the end date of the acute care period in the TO field of the CERT line and press [Enter].

The message ‘NO ERRORS OK TO PAY’ displays.

Adding a theatre fee line

Step

Action

1

Action Pend Resolution Insert screen + Read more ...

Select [PF11-CALC].

5

Calculate amount payable + Read more ...

Enter the following details:

  • Medicare Benefit Schedule (MBS) procedure item
  • theatre type and procedure group in the appropriate fields

6

Continue payment processing + Read more ...

Select [PF11-CALC] to continue payment processing.

Accessing information on the VAMM screen

Table 11: This table describes the procedure that Service Officers must complete to access information on the Veterans’ Affairs maintenance Menu (VAMM) system. VAMM is used to display private hospital contracted rates, for example, accommodation/theatre band and package items.

Step

Action

1

Access Veterans' Affairs Processing menu + Read more ...

Key ‘VAMM’ and press [Enter].

The Veterans' Affairs Processing menu displays the options available.

Note: the option displayed depends on the Service Officer's security access level.

2

Contracted private hospital menu + Read more ...

Select the contracted private hospital menu.

Key ‘1’ in the SEL field.

Select [Enter].

3

Display details + Read more ...

To display contract details key ‘1’ in the SEL field.

Enter the provider number, including any leading 0s, without the alpha and check digit.

Note: the PROVIDER LOCATION field defaults to 0. If the provider has a practice location other than 0, the practice location must be entered in the PROVIDER LOCATION field.

Select [Enter].

The details of the contracted private hospital are displayed in order of most recent. The period each contract is valid for is displayed in the START and END DATE fields.
