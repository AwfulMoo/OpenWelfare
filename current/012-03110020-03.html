    Process amalgamated seed organisation registration form – questions 1 to 11

    Process RO details of the amalgamated seed organisation registration form

    Table 14: This table describes how a Service Officer processes a completed amalgamated seed organisation registration form in the eHealth Program (EHP) system, specifically question 1 to 11 which relate to Responsible Officer (RO) details.

    Step

    Action

    1

    PART A - Healthcare Identifiers Service Registration + Read more ...

    Question 1 – Does this organisation employ an identified healthcare provider?

    • No is ticked or crossed, the organisation is not eligible for a HPI-O at this time. Contact the applicant. As per legislation any organisation wanting to participate in the HI Service is required to employ a healthcare provider
    • Yes is ticked or crossed, proceed to question 2

    2

    Question 2 - Are you a known customer to us? + Read more ...

    • No is ticked or crossed, proceed with application at Question 4. No action by Service Officer
    • Yes is ticked or crossed, known customer number is entered in box below, go to Question 3

    Note: if tick boxes not completed, refer to Question 3 and process according to information supplied.

    3

    Question 3 - Type of Known Customer. + Read more ...

    If a Known Customer Identifier is provided:

    • the individual does not need to provide evidence of identity (EOI) (see Evidence of identity processing for an individual in the Healthcare Identifiers (HI) Service), it can be verified as per the following:
      • Public Key Infrastructure (PKI) certificates - verified using Certificate Issuance Management System (CIMS)
      • Medicare provider number - verified using Provider Directory Service (PDS)
      • Existing RO number - verified using EHP
      • Existing Contracted Services Provider (CSP) officer number - verified EHP
      • Duly Authorised Officer of a site PKI - verified using CIMS

    To identify the correct rating for the type of EOI supplied, see Evidence of identity for Healthcare Identifiers (HI) Service.

    4

    Question 4 - Responsible Officer (RO), prefix, Family Name, Given name, and suffix. RO details must be provided + Read more ...

    A prefix must be entered for the purposes of the business registration letter.

    • If prefix is not complete, attempt to derive the title prefix of Dr, Ms, Mrs, Mr where appropriate using names, or other searching techniques, for example, if the doctor is listed on the practice website or by their existing Australian Health Practitioner Regulation Agency (Ahpra) record
    • If a prefix cannot be derived using other methods, phone to confirm

    Note: where the name set supplied by the applicant (including prefix) is different to the name set recorded in EHP, the name set supplied is added as the 'Preferred Name' and the Name Usage field is entered as:

    'Registered Name (Legal Name)' - if the name supplied is a verified name (that is, confirmed by either required EOI or matches their known customer name)

    'Other Names (Alias)' - if the name supplied is different to their 'Registered Name (Legal Name)', that is, anglicised names.

    5

    Question 5 - RO date of birth + Read more ...

    Usually required to obtain an exact match, should make a follow up call if this is not completed. Service Officers can attempt to find this through internal sources, such as PDS if possible.

    6

    Question 6 - RO Gender + Read more ...

    • select Male or Female

    If this question is not completed and a match is found on the system, no follow up call is required.

    Service Officers may also derive this information from the prefix listed in Question 3.

    7

    Question 7 - Business address + Read more ...

    This must be provided. This is the address that the registration letter is mailed to and must be ticked as the 'preferred address' when this record is entered into EHP.

    8

    Question 8 - Daytime Phone Number, Email + Read more ...

    One of 'Daytime Phone Number' or 'Email' must be provided

    9

    Question 9 – Email + Read more ...

    One of ‘Daytime Phone Number’ or ‘Email’ must be provided

    10

    Question 10- Electronic Access to the HI Service + Read more ...

    • This question is not mandatory. Ignore if not completed
    • HI Operations Tier 3 team process all new PKI requests until further notice. However Tier 2 team can add HI Service permissions to an applicant’s existing PKI Individual Certificate
    • If a PKI certificate registration number is entered here, it should be the same as Question 3 confirming the applicant is a known customer

    11

    Question 11 – RA number + Read more ...

    If the applicant ticks ‘yes’ in question 10 the RA number should be in this box.

    Question 14 - Type of Known Customer + Read more ...

      • Public Key Infrastructure (PKI) certificates - verified using Certificate Issuance Management System (CIMS)
      • Medicare provider number - verified using Provider Directory Service (PDS)
      • Existing RO number - verified using EHP
      • Existing Contracted Services Provider (CSP) officer number - verified using EHP
      • Duly Authorised Officer of a site PKI - verified using CIMs

    To identify the correct rating for the type of EOI supplied, see Evidence of identity for Healthcare Identifiers (HI) Service.

    4

    Question 15 - OMO prefix, Family Name, Given name, and suffix + Read more ...

    • If this is not completed, attempt to derive the title prefix of Dr, Ms, Mrs, Mr where appropriate using names, or other searching techniques. For example, if the doctor is listed on the practice website or by their existing Australian Health Practitioner Regulation Agency (Ahpra record
    • If this cannot be derived using other methods, phone to confirm

    Note: where the name set supplied (including prefix) by the applicant is different to the name set recorded in EHP, the name set supplied is added as the 'Preferred Name' and the Name Usage field is entered as:

    • Registered Name (Legal Name) - if the name supplied is a verified name (such as confirmed by either required EOI or matches their known customer name)
    • Other Names (Alias) – if the name supplied is different to their 'Registered Name (Legal Name)', that is, anglicised names

    Question 16 - OMO date of birth + Read more ...

    • If this question is not completed and a match is found on the system, no follow up call is required. The Service Officer may also be able to derive this information from the prefix listed in Question 15

    Question 23 - Question 24 - Organisation name, Trading name + Read more ...

    Note: a sole trader is not required to provide business evidence of identity (EOI) or authorisation that they are authorised to act on behalf of themselves.

    If the entity type for the Australian Business Number (ABN) supplied is listed as 'Individual/Sole Trader' and this is the same name as the Responsible Officer (RO) in the application, this is sufficient to prove that they meet the sole trader criteria. This must be verified using the ABN website.

    2

    Question 25 – Seed Organisation’s Australian Business Number (ABN) or Australian Company Number (ACN) + Read more ...

    ABN or ACN should be completed by applicant here. If it is left blank contact the applicant. There are also EOI evidence required, for more information see Apply for a Public Key Infrastructure (PKI) certificate

    If an application is supported by an ‘InfoTrack’ document sent in by the applicant providing Australian Securities and Investments Commission (ASIC) evidence that the ABN is linked to the organisation name, a Service Officer still must always go to the Australian Business Register website (see Resources for a link) to validate that the ABN or ACN that is provided is correct and current for the organisation being created.

    3

    Question 26 – Confirmation of Legal Business Entity + Read more ...

    If the:

    • organisation has an existing site certificate the number is entered here. To identify the correct rating for the type of EOI supplied, see Evidence of identity for Healthcare Identifiers (HI) Service
    • tick box indicates that 'a document establishing legal business entity is attached', or no boxes are selected the organisation needs to provide evidence of business identity

    4

    Question 27 – Question 29 - Organisation type + Read more ...

    • the organisation type reference guide is available on the Services Australia website and gives the applicant a guide on how to complete these questions
    • one organisation type is required (and an organisation service type), such as General Practice. If this question is not answered, attempt to derive the service type from the business name of what type of organisation is being registered. Examples:
      • Milburn Family Practice where Dr John Milburn is the RO. The Service Officer could determine that the organisation type is a 'General Practice - General Medical Practitioner Service'
      • Trading Name Black Point Pharmacy. The Service Officer could determine that the organisation type is 'Retail Pharmacy – Pharmacy, retail, operation'
    • the Service Start Date in EHP must be entered as the date the application is being processed
    • the Service Officer may use a public website to determine the type of organisation

    5

    Question 30 - Question 31 - Business address, Postal address + Read more ...

    At least 1 must be provided from the applicant.

    Postal Address

    • if the Postal address is completed by the applicant, this must be ticked as preferred address in the EHP system, as the preferred address is used for mailing purposes

    Business Address

    • for Healthcare Provider Directory (HPD) entries the business address is selected in the EHP system. This allows the location of the organisation to be identified

    6

    Question 32 - Daytime phone number, Fax number, Email + Read more ...

    • at least 1 must be provided
    • preferred contact tick box not mandatory. If 1 is not ticked, select daytime phone number

    Question 33 - Seed Organisation PKI details + Read more ...

    This question is not mandatory. Ignore if not completed.

    8

    Question 34 - Question 35 - Do you want the organisation displayed in the Healthcare Provider Directory? + Read more ...

    This question is not mandatory. Ignore if not completed.

    If:

    • No is ticked or crossed, however the applicant has ticked or crossed particular fields they consent to publish (excluding the mandatory fields) key the details as requested
    • Yes is ticked or crossed, the organisation name and business address are the 2 default display items. If any other display items are ticked or crossed these must also be keyed
    • ’Yes’ or ‘No’ questions are not completed, however the applicant has ticked or crossed particular fields they consent to publish (excluding the mandatory fields) key the details as requested

    Question 36 - Declaration + Read more ...

    The declaration must be signed by the applicant. Check the signature where possible to ensure it is signed by the applicant - if signature EOI is supplied, this can be used to verify the applicant's signature. Where possible, and signature is readable, check that it is the correct name.

    If date is not completed, the date the email was received may be used.
