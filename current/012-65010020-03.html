Sorting and receipting AHS claims

Table 1

Step

Action

1

Sorting Aboriginal Health Services (AHS) claims + Read more ...

Claims are received by email or post and saved as a PDF. The PDF is received and managed by Services Australia’s scanning provider before being saved through Processing and National Demand Allocation (PaNDA).

Any related correspondence from the pharmacy should be uploaded into PaNDA to accompany the claim paperwork, and saved into the relevant PaNDA work item. This includes emails sent from the Aboriginal Health Services (AHS) mailbox.

Claims are sorted in PaNDA by:

  • pharmacy approval number
  • AHS name
  • date of supply
  • claim reference number

2

Receipting claims + Read more ...

Ensure claims received are complete by checking the following:

  • correct version of the Aboriginal Health Services (AHS) pharmaceutical supplies request and claim form (PB042). If claim is made on an electronic order form ensure the form:
    • Is a version approved by the agency, a list of approved form numbers is located in the QLD shared drive in the AHS folder
    • Is accompanied by a signed and completed AHS pharmaceutical supplies request and claim form (PB042) as a coversheet
      Note: if an incorrect or unapproved form has been provided an email should be sent to Operational Policy and Legacy Programmes, Pharmaceutical Benefits and Digital Health Branch with a copy of the claim form. Continue to process the claim if required details are included
  • Claim signed and dated by the pharmacy and by an AHS qualified health professional. Both electronic and manual signatures are permitted
    Note: An individual, if employed as a pharmacist and an AHS qualified health professional, can switch roles and sign the declaration on the claim form as either the AHS qualified health professional or as the pharmacy representative, if acting in that capacity. However, while this action is permitted, they must never sign as both the AHS qualified health professional and pharmacy representative on the same claim. Service Officers must reject these claims
  • Claim is legible
  • Pharmacy and AHS details have been completed
  • Item codes have been provided
  • Column 'Quantity Supplied' has been completed
  • Column 'Subset Quantity Supplied' for an individual has been completed and the total number of items supplied has been added
  • If the column 'Subset Quantity Supplied' for an individual has been completed, the date of supply is 1 January 2018 or later
  • The Claim Reference Number has been provided. This is a compulsory field at Registration stage in the Consumer Directory Maintenance System (CDMS)

3

Incomplete claims + Read more ...

Incomplete claims should be returned to the approved supplier with an explanation. Use the Aboriginal Health Services (AHS) standard email template 'Email - more information is needed to process your section 100 Remote Area Aboriginal Health Services claim'.

Claims for payment of the Remote Area Aboriginal Health Services (RAAHS) patient specific medicine supply fee for pharmaceutical benefits provided before 1 January 2018 will not be accepted. These are to be returned using the AHS standard email template 'Email - return of your section 100 Remote Area Aboriginal Health Services claim'.

The claim will still need to be registered and then abandoned in the AHS claims system.

4

Access to the group mail box + Read more ...

Access to the mail box is required. Instances where access has not been provided, escalate to Established in PaNDA including a Processing Note with the reason for rejection.

Registering AHS claims

Table 2

Step

Action

1

Accessing the Aboriginal Health Services (AHS) Claims Processing system + Read more ...

Access the AHS Claims Processing system through the PBS Claim Processing system:

  • launch Internet Explorer and open the Services Australia intranet (SAI) home page
  • select Apps on the top right hand side of the intranet screen
  • select Medicare Portal from the menu displayed
  • select the Source Office code relevant to your work area
  • select PBS Claims Processing System from the eBusiness System drop down box
  • click on the Submit button

The PBS Claim Processing System homepage will include a link for Aboriginal Health Services Claims to enable claims processing.

2

Registering Aboriginal Health Services (AHS) S100 Claims onto system + Read more ...

From the Aboriginal Health Services Claim Menu, select Register a new AHS claim.

Key in the details of the claim:

  • AHS Registration Number
  • Approved Supplier Number
  • Date of supply

Date of supply for hospital pharmacies is the last date of the month claimed.

ABN details and ABN Branch Number will pre-populate.

  • key Claim Reference Number provided by the pharmacy

When a valid AHS registration number has been keyed a drop down box of AHS communities will be available:

  • select Main AHS Centre – No Community or relevant AHS community
  • AHS claim number - field will populate when the claim has been registered
  • Current Status – field will populate when the claim has been registered
  • Status Date - field will pre populate
  • select Register to register the claim
  • rekey the Claim Package Receipt Date if it is different from the default date

If the claim is incomplete it should be abandoned. A processing note must be entered in the Claims Comments box confirming the reason the claim is incomplete.

Processing AHS claims

Table 3

Step

Action

1

Query Aboriginal Health Services (AHS) Claims + Read more ...

From the Aboriginal Health Services Claims Menu:

  • Select Item Details
  • Click 'OK' on warning message

2

Enter Item Code Details + Read more ...

  • Key PBS Item Code. This will bring up the Item Name and Form and Strength details. When the item code is entered, if the description of the item name does not match that on the claim form, the item must be rejected using Reason Code R645
    Note: no item code substitution is allowed
  • Repatriation (DVA) items are not eligible therefore need to be rejected
  • Note: this refers to BIO equivalent items
    When an item code is entered however is equivalent for the purposes of substitution go to the Pharmaceutical Benefits Scheme (PBS) website to locate substituted PBS Item Code. Confirmation of whether the item is equivalent for substitution will be under the ‘Notes’ on the PBS Website
  • When the approved Supplier has noted an incorrect or incomplete PBS Item Code on the claim form, the item should be rejected. As the CDMS system won’t accept invalid or incomplete item code numbers, we should omitted it by placing an X next to the line of that particular item and make a notation on the claim stating that the supplier has provided incorrect/incomplete PBS item code
  • When an item is rejected and has been claimed as a subset quantity supplied for an individual, it must not be included in the total number of items supplied for the individual
    Note: please ensure you deduct PSMSF from total amount claimed for all rejected items

Note: to amend the claim because of an error with the item code or quantity, save the work item in PaNDA to the desktop as an editable PDF. Make amendments and save. Upload the document to PaNDA and add to the existing work item. Add a Processing Note in the Claims Comments box to indicate action taken.

3

Enter Brand/Total Supplies Details + Read more ...

  • Key the Brand. Leave blank when the brand is not provided. If item details disappear when keying leave brand details blank
  • When keying in the Brand and the item name details disappear remove the brand details and leave blank
  • Key Total Supplies (check Quantity Supplied is a multiple of the PBS maximum)
  • Select Submit

When the system returns an error code:

  • Click on the code to expand reason
  • Assess, and pay or reject the item

To override a rejection:

  • Select the line item and select Amend Item button
  • Tick Override Reject box and select Submit
  • When required, reverse this action by unticking the Override Reject box and select Submit

4

Verify and Complete Claim Details + Read more ...

Verify details in the item summary. Items Paid and Total Items should match:

  • Select Complete Claim this sets the claim status to complete. A warning message is displayed to confirm
  • Select 'OK' in the pop up screen. Once the claim is closed it cannot be amended

Note: when the system is slow, do not select Complete Claim as this may pay the claim twice.

5

Generate Statement + Read more ...

  • Select Generate Statement, this should open the AHS Claim Summary Statement
  • Verify that there is only 1 Payment ID and duplication of item codes has not occurred
  • When duplication has occurred, more than 1 payment has been generated and the claim will need to be abandoned and re-entered
  • Save As the AHS Claim Summary Statement in the Processed folder on the G: drive named as Pharmacy Approval Number followed by AHS claim number (in the current Journal folder). This is located in \\INTERNAL.DEPT.LOCAL\Shared\OLD\BRIS-Turbot\callcen\AHS\AHS Claim Summary Statements\Processed
  • Keep AHS Claim Summary Statement open ready for Access Database entry

Note: when the system is slow, do not select Generate Statement again as this may pay the claim twice.

6

How to access the Access Database + Read more ...

Access to the AHS Journal Access Database is through the group drive:
G:\callcen\AHS\PAYMENTS\Masters\AHS_Journal.

7

Recording in Access Database + Read more ...

Add the following information to the open, current AHS Journal number:

  • Select Open the Input Data Form
  • Key the Pharmacy Approval number
  • Check that the relevant Pharmacy Name is populated
  • Key the AHS Registration Number
  • Check that the relevant AHS Name is populated
  • Key the AHS claim number
  • Key the date claim was received
  • Key that the relevant Date Processed is populated (defaults to current date)
  • Key the Date of Supply
  • Key the Bulk Supply Value (that is, total dispensed price figure without GST from AHS Claim Summary Statement)
  • Select Submit

Extra steps from 1 January 2018 to process claims for the RAAHS patient specific medicine supply fee:

  • Key the Individual Supply Quantity if applicable (that is, total number of items supplied for an individual from the completed claim form; default value is zero)
  • Check that the Patient Specific Medicine Supply Fee is populated
  • Check that the Individual Supply Value is populated
  • Select Submit

The Resources page contains the Aboriginal Health Services (AHS) Access Database task card.

8

Update work item in PaNDA + Read more ...

  • Key in Journal Number as Case ID
  • Key in AHS Number in AHS Claim field

Complete the work item in PaNDA.

Requesting quality control + Read more ...

  • 5% of all AHS claims in a Journal must be quality checked prior to payment
  • Prior to payment the AHS Processing team to send an email to the Program Support Officer advising the Journal Number and the number of claims to be quality checked
  • PSO completes the quality checking and returns an email to AHS Claims Processing team advising of quality checking outcome

2

Verify details match + Read more ...

The number of claims entered on the Access Database has to match with the number of Statements saved in the AHS Claim Summary Statements folder in the drive \INTERNAL.DEPT.LOCAL\Shared\QLD\BRIS-143-Turbot\callcen\AHS\AHS Claim Summary Statements

  • In the Access Database select appropriate Journal number by double clicking Journal Data
  • Under the heading Journal Number drop list down select appropriate Journal
  • Check every AHS claim number displayed in the Access Database against respective AHS Claim Summary Statements
  • Verify Pharmacy Approval Number, Individual Supply Quantity and Bulk Supply Value (that is, total dispensed price figure without GST)
  • Check the AHS Claim Summary Statement for duplication of items or payment
  • Notify the AHS Claims Processing team of any errors that require amendment and that quality checking is completed

3

Generate Journal + Read more ...

When generating a Journal the current Journal needs to be closed and a new Journal number allocated:

  • Open Access database
  • Check the Journal Number to be paid is correct (default is current journal)
  • Select Generate Journal. An Excel document (for example, YYYYMMDD RAAHS Journal PBS XXX, where XXX is the Journal Number) will be created in the Payments folder on the G drive

4

Allocate new Journal number + Read more ...

Once the current Journal has been closed (by generating a journal), the nominated Service Officer must manually allocate the next Journal number:

  • Open the Access database
  • Double click on Current Journal and add the new Journal number
  • Save and close the Access database
  • Re-open the Access database and check the new Journal Number has been added
  • Create a new folder in the G drive with the new Journal number

5

Payment authorisation + Read more ...

A delegate approved under Schedule 1.34 (currently EL2 or EL1 in Medicare Providers Branch) must approve all AHS requests for payment.

After quality checking has been confirmed, forward approved quality checking email including Journal name and payment amount to the relevant delegates below:

Is the amount over $1 million?

  • Yes, National Manager approval must be sought
  • No, EL1 approval is sufficient

Follow the instructions given in the AHS payment authorisation recipient list on the Resources page to seek relevant approval.

6

Distributing the Journal + Read more ...

Once the Journal has been approved, email it to the Cash Management Section using the standard email template and following the instructions given in the AHS payment authorisation recipient list on the Resources page.

Actioning an AHS pharmaceutical supplies request

Table 7

Step

Action

1

Submitting requests for PBS medicines + Read more ...

The AHS must submit requests for PBS medicines to the approved supplier on an Aboriginal Health Services (AHS) pharmaceutical supplies request form (PB042). A link to this form can be found on the Resources page.

The following details must be provided on the order form:

  • AHS registration number
  • AHS name and address, and
  • The name and signature of the AHS qualified health professional who is employed or contracted by the AHS community and is qualified to supply all PBS medicines covered by these arrangements

Approved AHS can develop their own electronic order form to replace the requested items section of the Aboriginal Health Services (AHS) pharmaceutical supplies request and claim form (PB042). Only approved electronic order forms are accepted. Check if an order form is approved by the agency, by emailing the Pharmaceutical Benefits Branch.

Electronic order forms must be accompanied by the Aboriginal Health Services (AHS) pharmaceutical supplies request and claim form (PB042) as a coversheet, with all details and declarations and completed.

2

Medicines not available under AHS arrangements + Read more ...

The following medicines are not available under AHS arrangements:

  • Extemporaneously prepared items
  • Highly specialised drugs
  • any medicine supplied under Section 100 Special Arrangement, e.g. IVF or Growth Hormone
  • emergency drug (prescriber bag) supplies
  • Repatriation Pharmaceutical Benefit Scheme medicines
  • Schedule 8 medicines as defined by the drugs and poisons legislation in the relevant state or territory

Note: Schedule 8 or extemporaneously prepared medicine must be prescribed on an approved prescription form and dispensed under standard PBS arrangements.

3

Approved supplier actions + Read more ...

  • The approved supplier will supply the requested PBS medicines
  • The approved supplier can then claim reimbursement by completing and submitting the Aboriginal Health Services (AHS) pharmaceutical supplies request and claim form (PB042)
  • The Aboriginal Health Services (AHS) pharmaceutical supplies request and claim form (PB042) can be submitted by email as PDF to Aboriginal Health Services or by post to the AHS Claims Processing Team

AHS claim enquiries

Action

1

Security information + Read more ...

If an enquiry is received be email, Service Officer must confirm that the email address is the valid email address provided by the Authorised Person. If unsure, contact the Pharmaceutical Benefits Branch.

The following information needs to be provided by the approved supplier before a Service Officer can access AHS claim information:

  • Pharmacy approval number/approved supplier number (ASN)
  • Claim reference number; or

Once the above information has been provided ask the approved supplier what the nature of their enquiry is.

Note: query on payment is only disclosed to an authorised person listed under the approved supplier.

  • Select Medicare Portal from the menu displayed
  • Select the Source Office code relevant to your work area
  • Select PBS Claims Processing System from the eBusiness System drop down box
  • Click on the Submit button

Query an AHS claim + Read more ...

Current and previous claims can be viewed:

  • Search for claim data by keying the AHS registration number, approved supplier number or the AHS claim number

The AHS search query fields table on the Resources page outlines the fields available for a Service Officer to perform a search on the Query AHS screen.

4

Query an AHS payment + Read more ...

  • Select Query AHS claim
  • Key relevant details, as detailed in Step 3 above

The following information will be presented:

  • The date finance processed and released the payment to the RBA (Reserve Bank of Australia) will display in the Date Posted field
  • The amount paid for the claim is also displayed on this screen
