This page contains information on the cancellation, rejection and not effective reasons for Child Care Subsidy (CCS) and Additional Child Care Subsidy (ACCS).

Explanation of initiation codes used on this page

Table 1: this table describes the legend for the subsequent tables.

Item

Description

1

Initiation

In the tables below, cancellation, rejection and not effective codes are either:

  • Manual - the code can be recorded manually on the Benefit Action (BA) screen
  • Auto - System generated codes that cannot be keyed manually

Explanation of reason code fields used on this page

Table 2: this table describes the codes / acronyms used for CCS and ACCS assessments.

Item

Description

1

Valid Action codes

  • REJ - Reject
    • An assessment can be automatically or manually rejected when a customer does not meet the eligibility requirements
  • CAN - Cancel
    • An existing assessment can be automatically or manually cancelled when a customer is no longer assessed as meeting eligibility requirements
  • NEF - Not effective
    • Where the only information missing from a customer’s claim lodgement is a TFN or bank account details, the system should automatically assess the claim as not effective
    • When it is determined that the most recent claim information lodged by a customer is duplicate, the claim is to be manually assessed as not effective. The assessment process to follow for each CCS/ACCS claim types can be found in the relevant process tab of the appropriate Operational Blueprint claim assessment files.
