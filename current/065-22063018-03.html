Using the Refunds for manual approval screen + Read more ...

Use the Refunds for manual approval screen to search for care recipients who are eligible for refunds requiring manual approval or rejection.

The Refunds for manual approval screen will only display a maximum number of 200 record. Service Officers can filter by a combination of the review type and groups based on the first letter of the care recipient’s family name. The review type can be an ad hoc review, a departure review, a QR or an Annual and Lifetime Cap (ALC) review.

The Refunds for manual approval screen only includes refunds that are awaiting decision:

  • all QR refunds above the current threshold amount from the production run
  • all QR refunds with an accommodation contribution fee of any value
    • See the Aged care reviews – quarterly reviews for the current QR threshold amount
  • all refunds from an ad hoc review
  • all refunds from a departure review
  • all refunds from ALC reviews. Note: ALC reviews are automatic reviews generated when a care recipient in the post 1 July 2014 fee scheme reaches their Annual or Lifetime cap

Once a refund has been manually approved or rejected, Service Officers will need to use the Reviews and refunds search screen to search for the specific refund record or view the results within the care recipient’s reviews and refunds page. The Refunds for manual approval screen displays refunds based on the combination of the review types and family name filters. Due to the volume of records, the Refunds for manual approval screen does not display all refunds in a single view

The Refunds for manual approval screen displays the following table:

  • Approval ID
  • Review type - displays either Quarterly Review, Ad-hoc Departure or ALC
  • Care recipient ID
  • Given name
  • Family name
  • Service ID
  • Review date
  • Review period
  • Adjustment amount
  • Status - Service Officers can select the hyperlinked Awaiting decision status and the Refund approval details screen will display. Go to Step 4

Service Officers can also

  • type in a keyword in the Search bar to search for all records that contain the keyword. Search results display where the keyword is used anywhere on the record, even if it appears in the middle of another word
  • filter the table by a combination:
    • Review type, and
    • Family name
  • go to a different page by selecting either the:
    • First, Previous, Next or Last options, or
    • the relevant page number
  • select Download CSV to download the current filter view of the Refunds for manual approval table in CSV format

Procedure ends here.

Using the Refund approval details screen + Read more ...

Note:

  • The Refund approval details screen will display different fee assessment scheme banner depending on the care recipient’s means assessment scheme
  • The Assessment scheme - Pre 1 July 2014 banner will display for care recipients who were receiving permanent residential care before 1 July 2014
  • The Assessment scheme - Post 1 July 2014 banner will display for care recipients who were receiving permanent residential care either
    • on or after 1 July 2014
    • before 1 July 2014 and who have chosen to be assessed under the post 1 July 2014 means assessment scheme
  • As part of the claim process, the Payment Assessment Calculation Engine (PACE) checks if the care recipient has reached either their Annual or Lifetime Cap (ALC). If either cap is reached, an ‘auto-reconciliation’ will occur and these are known as ALC reviews
  • ALC reviews will:
    • display for care recipients with the Assessment scheme - Post 1 July 2014 banner when either their Annual or Lifetime cap has been reached
    • set the care recipient’s fees to zero and identifies a refund ready for approval or rejection

Using the Refund approval details screen

The Refund approval details screen displays the following:

  • Care Recipient banner - displays the care recipient’s name, IDs, Date of Birth, Record type and Type of care status
  • Refund details section
    • Approval ID
    • Review type - displays either Quarterly Review, Ad-hoc, Departure or ALC
    • Review date
    • Last entry date
    • Review period
    • Review notes
  • The Assessment scheme - Pre 1 July 2014 or Assessment scheme - Post 1 July 2014 banner - displays the service’s details in a table
    • Service NAPS ID
    • Service name
    • Fee type
    • Total previous fee
    • Total new fee
    • Adjustment amount
  • Adjustment amount total - Awaiting decision
  • Decision notes - displays No records found information message for manual refunds awaiting approval or rejection

Before manually approving or reject refunds, Service Officers can use the Fee reconciliation details screen to determine if the refund amount is correct or not.

Using the Reviews and refunds search screen + Read more ...

Note:

  • Service Offices can use the Reviews and refunds search screen to:
    • search using different criteria for reviews and/or refunds
    • search by care recipient ID to return all reviews and/or refunds
  • A review can be an ad hoc review, departure review, a QR or an ALC review.
    • Approve or reject refunds including those from a QR that are less than the threshold amount before the bulk approval
  • The In progress status will not be displayed for ALC reviews
  • All fields are optional unless marked with an asterisk

In the Reviews and refunds search screen, Service Officers can search by either:

  • Search by Review and refund details - default option
    • Care type - default option
    • Review type - defaults to Select all. Select the drop down menu for either Ad-hoc, Departure, Quarterly or ALC
    • Status - defaults to Select all. Select the drop down menu for either Awaiting decision, Completed or Submitted
    • Outcome - defaults to Select all. Select the drop down menu for either Entitled to refund, In progress, Not entitled to refund, Reconciled - no change, Refund paid or Refund rejected
    • Review date from
    • Review date to
    • Adjustment amount minimum - defaults to $0.00. Service Officers can key another minimum amount
    • Adjustment amount maximum - key the maximum amount if applicable
    • Fee type - defaults to Select all. Select the drop down menu for either Accommodation Contribution, ITF - Income Tested Fee or MTF - Means tested care fee
  • Search by Care recipient ID - select the drop down menu to display this search option
    • Care recipient ID* - this field is mandatory if searching by Care recipient ID and all refunds/reviews results will display in relation to that specific care recipient

Select Search and the Review and refund search results screen will display.

The Process quarterly review - Exclude care recipients screen + Read more ...

Note:

  • There is no limit to the maximum number of care recipients that can be excluded from a QR process
  • A large number of exclusions may result in performance issues
  • ACSP will automatically check the care recipient IDs in the Exclude care recipients section
  • A validation error message will display if Programme Officers have keyed incorrect and/or duplicate care recipient IDs
  • Service Officers must resolve the validation error message before continuing with the QR process
  • They can do this by
    • selecting Remove error(s) to clear any duplicated care recipient IDs (these are care recipients with the Already excluded status)
    • key in the correct care recipient IDs (these are care recipients with the Invalid status and/or care recipient IDs who have exceeded the character limit)

The Process quarterly review - Exclude care recipients screen can be used to exclude specific care recipients from the QR process.
