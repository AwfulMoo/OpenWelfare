Table 1: This table describes how Service Officers create OMO record for a network organisation in the HI Service.

Step

Action

1

Log on to the system + Read more ...

Log on to eBusiness Home Page.

The Search for Individual screen displays.

Note: when creating a network organisation the following fields must be completed in the inquirer details area of the search screen:

  • Inquirer Type - OMO (the applicant)
  • Family Name - OMO’s family name
  • Given Name - OMO’s given name

A completed Application to register a Network Organisation form must be provided. No additional documentation is required.

2

Is linking required + Read more ...

Does the Healthcare Provider Identifier - Organisation (HPI-O) require an OMO to be linked?

  • Yes, go to Step 3
  • No, create HPI-O record for a network organisation, see Step 1 in the Create HPI-O record for a network organisation in the HI Service table. Note: the address supplied at Question 5 of this application should not be added to the OMO's HI record if they are not going to be linked to the newly created networked organisation and they are an applicant only. Procedure ends here

3

Search for existing eHealth Program individual + Read more ...

Searching for records in the Healthcare Identifiers Service using the eHealth Program (EHP).

Does a record exist for an existing eHealth Program (EHP) individual? Note: this may be an existing RO, OMO or Healthcare Provider Identifier – Individual (HPI-I) record.

  • Yes, the OMO details display at the bottom of the screen. See Amend Responsible Officer (RO) or Organisation Maintenance Officer (OMO) record in the Healthcare Identifiers (HI) Service for more details
  • No, select Create OMR. See Create Organisation Maintenance Officer (OMO) record in Healthcare Identifiers (HI) Service for more detail

Record the OMO’s number.

When complete create HPI-O record for a network organisation, see Step 1 in the Create HPI-O record for a network organisation in the HI Service table.

Table 5: This table describes how Service Officers create organisation service details for a network organisation in the HI Service.

Step

Action

1

Access organisation service details on the system + Read more ...

Select the Organisation Service Details tab.

2

Organisation service type details + Read more ...

From the drop down menu, select:

  • Organisation Type provided for the network organisation
  • Organisation Service Type provided for the network organisation

Note: when selecting an organisation service type, ensure that descriptions at all levels are appropriate for the type of service the organisation provides.

For example, when the organisation indicates it provides a baby clinic service, but it is not a pharmacy, it is not appropriate to select Retail Pharmacy > Community Pharmacy > Baby clinic service, as the service type at level 1 and 2 (pharmacy) is not suitable. In this case, the correct selection would be Other Healthcare services nec > Healthcare service nec.

Contact Tier 3 for advice when unsure which organisation type to select.

3

Organisation service type start date + Read more ...

Is an Organisation Service Type Start date provided?

  • Yes,
    • Key date provided
    • Select Day, month and year are accurate in the Accuracy field
  • No,
    • Key processing date
    • Select Day, month and year are estimated in the Accuracy field
    • Leave the following fields blank:
      • Organisation Service Type End Date
      • Accuracy

Note: when only a year is provided:

  • Key '01/01/YEAR' provided
  • Select Day and month are unknown, year is accurate in the Accuracy field

Check for validation errors + Read more ...

  • No, go to Step 5

Table 10: This table describes how Service Officers create Public Key Infrastructure (PKI) certificates for a network organisation in the HI Service.

Step

Action

1

Access Certificates in the system + Read more ...

Select the Certificates tab.

The Certificates tab of the Create HPI-O Networked Organisation screen displays with the following 4 buttons:

  • Link Existing Certificate - Search for an existing certificate using the organisation's existing certificate RA number. Authorised Service Officers may link an existing PKI certificate. Refer to Subject Matter Expert (SME)/team coach

Table 12: This table describes how Service Officers create a HI Healthcare Provider Directory (HPD) entry for an organisation in the HI Service.

Step

Action

1

Search for organisation created + Read more ...

From the Healthcare Identifier eHealth program (EHP), search for the organisation just created:

  • complete this procedure when the registration form includes a request to publish network organisation details in the HI HPD
  • when the 'yes' or 'no' section is not completed and the applicant has marked fields they consent to publish (excluding the mandatory fields), call the organisation to clarify if the HPD entry should be completed

2

Create new entry + Read more ...

From the Amend HPI-O Network Organisation screen, select the HI PDS Entries tab.

Select Create New Entry.

3

Key organisation HI PDS details + Read more ...

Key the information provided into the relevant Organisation HI PDS Entry Details fields. The following fields are mandatory:

  • Name
  • Address
  • Priority

Note:

  • Always tick the Display in HI PDS box as a Provider Directory Service (PDS) entry will not be published when this box is left blank
  • Select the business address for display unless the applicant has requested otherwise (process according to the applicant’s request)
  • When the organisation type is Other Healthcare services nec > Healthcare services nec, add the healthcare service the organisation provides in the additional comments box (for example, baby clinic)
  • When more than 1 PDS entry is required, a priority order needs to be selected

4

Add additional HI HPD entries if required + Read more ...

Select Add.

Note:

  • To add additional HI HPD entries, select Create New Entry and repeat Steps 4 and 5
  • When there is more than 1 entry required for any drop down menu, select the + (plus) button beside the appropriate drop down menu. This enables more than 1 option for that field

5

Check for validation errors + Read more ...

The HI HPD details are displayed in the Recorded Entries table.

Select Submit.

Are there validation errors?

  • Yes, correct errors as prompted. When the errors are fixed, go to Step 6
  • No, go to Step 6

6

Correct errors where required + Read more ...

The Confirm Amend HPI-O Network Details screen displays.

When the details are:

  • incorrect, select Edit to return to the Amend HPI-O Network Organisation screen. Go to Step 2
  • correct, select Save. A confirmation message is displayed

A letter is automatically generated and addressed to the Responsible Officer (RO) of the seed organisation at the top of the organisation hierarchy notifying of the network creation. The letter is addressed to the RO and sent to their preferred address.
