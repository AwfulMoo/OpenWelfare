Mandatory detailed search in CDMS

Table 1: how to perform a detailed search for a customer where no existing or previous Medicare number is available. This is mandatory before creating a new Medicare registration.

Step

Action

1

Search for customer in CDMS + Read more ...

On the CDMS Home page:

  • select Inquire on Consumer Details
  • select the Detailed Search tab

2

Enter search details + Read more ...

Enter search details as follows:

  • Select All Programs from the Program drop-down menu (mandatory)
  • Key Family Name (mandatory)
  • Select All from the Gender field (mandatory)
  • Key Age or Age Range (mandatory)
  • Leave the Postcode or Postcode range field as default
  • Select Personal tab

If information message 00005 returns, follow the instructions in the message to refine the search or adjust search criteria by including:

  • the first initial, or
  • for newborn enrolments:
    • check other parent surname, or
    • key 'baby of' in the First Name field

3

Define search result + Read more ...

Services Officers must do extra checks to confirm no other consumer records have been previously created in CDMS. For example:

  • Does the name have any hyphens or spaces? When searching for a name with a hyphen, do the search without the hyphen, e.g. Mary-Ann and Mary Ann
  • Confirm the spelling of the name/s
  • Did the customer have another surname as a child?
  • Check for other variations of first name e.g. if a customer’s name is Oliver also search Ollie
  • Has the customer ever used or been known by any other name?
  • Family and given names may have been transposed in the past
  • Consider ethnic name practices
  • Check other parent's surname

Possible matches are displayed under Matched Consumers.

Is a possible match been found after all checks?

  • Yes, go to Step 4
  • No, see Step 1 in Table 7 to do an address search

Note: make sure the correct record has been identified by confirming details such as legal name and date of birth before proceeding with the enrolment.

4

Possible matches found + Read more ...

Select the consumer:

  • select radio button next to the consumer
  • select Personal tab

The Personal Details Inquiry screen displays.

If more than one record is found, see Duplicate PIN and Consumer IDs in Medicare.

5

Swap to Amend screen + Read more ...

  • Select Personal
  • Scroll to bottom and select Edit
  • A message will display asking if the user wants to swap from Inquiry to Amend:
    • select Yes
    • amend Personal Details will display
    • make the updates as required (VALIDATE, VALIDATE, UPDATE)

See Step 1 in Table 2 for Document Verification System (DVS) process if applicable.

Register a new customer

Table 2: how to create a new Medicare registration where no registration for the customer already exists. Service Officers must do a detailed search in CDMS before creating a new registration.

Step

Action

1

Request/new enrolment + Read more ...

  • Request received by PaNDA, go to Step 2
  • Request processed in a service centre:
    • complete a public security check. If Service Officer is satisfied with supporting documents, Document Verification System (DVS) is not required. Go to Step 5

2

Request via PaNDA + Read more ...

Does the application include customer Document Verification System (DVS) consent?

  • Yes, go to Step 4
  • No, go to Step 3

3

Obtain verbal consent + Read more ...

Attempt to contact the customer.

If contact is successful:

  • Perform a public security check
  • Read the following to the customer:
    • 'You confirm that you are authorised to provided the personal details presented and you consent to your information being checked with the document issuer or official record holder via third party systems for the purposes of confirming your identity .'

Has verbal consent been given?

  • Yes, go to Step 4
  • No, add comments in Personal - verbal consent not obtained, document not verified through DVS. Procedure ends here

If contact is unsuccessful:

  • issue letter Z0137 Medicare enrolment application - more information required

4

Access DVS tool + Read more ...

Does the Service Officer have access to the Document Verification System (DVS) tool?

  • Yes, access DVS tool and attempt to verify documents
  • No, see Escalation process for Medicare Eligibility

Is verification successful?

  • Yes, go to Step 5
  • No, go to Step 14

5

New customer registration + Read more ...

  • Select Register New Consumer on the CDMS Home page
  • The Registration Type screen displays
  • Select New Consumer
  • The Search for Consumer (Register) screen displays

6

Enter customer's details + Read more ...

Service Officers must complete the mandatory detailed search before registering a new customer.

  • Select the Immigration Data Search (tick will be displayed)
  • Key details for the customer in the following fields:
    • Family Name (mandatory) - add onlyname here
    • First name
    • Second name (if applicable)
    • Gender (mandatory)
    • Date of Birth (mandatory)
    • Postcode or Postcode range (leave field as default, for example 0001-9999)
  • Select Search

Warning message 01875 will display if the person registering is over 12 months old. Select Yes to continue.

Search result displays:

  • matched consumers
  • exact or possible immigration clients on the Immigration Clients tab

On the Immigration Clients tab, where the:

  • Etm (Entitlement) is a dash (-) and Client Status is Lodged, relationship documents may be required
  • Client Status is Refused further documentation is required

Note: see Operational Blueprint processes for the visa type for more details.

Follow the instructions in all warning messages.

7

Determine possible matches to customer search + Read more ...

If no records found:

  • A message displays indicating that no records were found
  • Select OK to acknowledge the message
  • Go to Step 5

If possible matches found:

  • Consumer details display a possible match of a customer
  • Select the person by using the radio button and select Add to list
  • The Registration Type screen displays

Asn, Curr Etm and Curr Etm Start Date fields will be blank if there is no entitlement recorded.

See Table 5 to Register customer's Medicare entitlement.

8

Record customer's details + Read more ...

  • Select New Consumer

The Record New Consumer screen displays. The personal details are filled with any search details previously entered. Complete the following fields:

  • Title
  • Only Name Indicator (select if customer is registering with only one name)
  • Second Name
  • Gender (mandatory)
  • Multiple Birth Indicator (select if customer is part of a multiple birth)
  • Aboriginal or Torres Strait Islander Status (mandatory)

Note: a customer’s identity can be verified by information provided via VEVO or the Immigration Datalink. As part of the enrolment process, if:

  • the datalink or VEVO is established, Service Officers are not required to use Document Verification System (DVS)
  • there is no datalink or VEVO is not established, Service Officers must verify the identity documents in DVS before processing the application

9

Address or contact details + Read more ...

  • Select Address tab
  • Select Address Source from the drop down box
  • Key locality or postcode
  • Select the locality from drop down box
  • Start to key the number and street name on the Address line
  • Select the address from the drop down box

Note:

  • Only add a residential address if different from the postal address
  • There is no need to enter both a mailing and a residential address in CDMS. If the customer has only provided a mailing (postal) address such as a PO Box, this is entered in CDMS as the mailing address. See Change of address for Medicare enrolments for more details

10

Record customer's contact details + Read more ...

  • Select Contact details tab
  • Key the customer's contact details that are available and any comments, key phone numbers without any spacing
  • Select an option from the drop down menu in the Source field for any relevant contact details keyed

Note: do not record contact details for persons under 15 years.

11

Record customer's bank details + Read more ...

  • Select Banking details tab
  • Key:
    • BSB number
    • account number
    • account name (key 'and', do not use ampersand '&')
  • Select Notification Source from drop down box

Note: Account start date field defaults to today's date and cannot be changed.

See Store, confirm and end date bank account details for Medicare EFT payments for procedures. Do not record bank details for persons under 14 years.

12

Record comments + Read more ...

  • If Document Verification System (DVS):
    • was used, record comments in Personal tab in CDMS and result in DVS database:
      'DVS consent acquired [insert date]. [Insert document type and document reference text/document number], VRN: [insert VRN number]'. For example PaNDA work item ID: 1235467891. DVS consent acquired 10/12/2020. passport, reference number EF1234567, VRN: 12345
    • was not used, (processed in a service centre), record comments in Personal tab in CDMS:
      ‘Processed in service centre, DVS not required’
  • Scroll down and select Validate
  • Action any warning messages and Validate
  • Select either:
    • Change to amend the details if required
    • Update to confirm the details displayed. A message displays confirming that a consumer record has been created
  • Select OK to acknowledge the message
  • The Registration Type screen is displayed with the customer's details and customer ID
  • If the details are correct, select Update

13

Family registration + Read more ...

If the application has:

  • more than one customer, go to Step 1 and repeat the process
  • only one customer, see Step 1 in Table 5

14

Document verification unsuccessful + Read more ...

  • F2F Service Officers - see Escalation process for Medicare Eligibility
  • HSDD Service Officers - call HSDD PSO

Was PSO document verification successful?

  • Yes, complete application. Go to Step 5
  • No, Service Officer attempts to contact customer

Was customer contact successful?

  • Yes, customer provides other identification document. Go to Step 3
  • No, customer contact is unsuccessful and/or second attempt at verification fails:
    • return original document to the customer with Medicare Enrolment application more information required (DVS interim only) letter

      Assess additional identity documents + Read more ...

      Customers may come to a service centre with the issued letter and additional identity documents.

      Step

      Action

      1

      Medicare entitlement details + Read more ...

      • Select Entitlement on the Registration Type screen

      • Select the entitlement details for the customer:
        • Entitlement Type (mandatory)

      Record customer’s eligibility documents + Read more ...

      • Select Documents tab
      • Select or key the details of each of the customer’s eligibility documents:
        • Eligibility Document (mandatory)
        • Residency Document (if applicable)
        • Reference Text (mandatory) (number or text of the document)
        • Reference Date (mandatory)
      • Select Add
      • The New Medicare Entitlement Details screen will display
      • Select Document tab
      • Repeat above to enter all documents provided separately

      See Table 1 on the Resources page for document details to be recorded in CDMS.

      3

      Record comments for customer details + Read more ...

      • Select Comments tab
      • Key any comments relevant to the enrolment in the text box

      4

      Add Power of Attorney details + Read more ...

      To add new or additional POA details, see Power of Attorney, Guardianship, Administrative Orders and Authorised Representative for Medicare.

      5

      My Health Record + Read more ...

      Processing if the My Health Record question is not on the form

      If the My Health Record question is not asked on the form, select Question is not on form or not answered.

      Processing for applicant

      Process according to the applicant’s or additional person’s My Health Record choice:

      • Consumer does NOT want a My Health Record
      • Consumer DOES want a My Health Record

      If the applicant did not answer the question or sign the form, select Question is not on form or not answered.

      Do not select Consumer does not want a My Health Record if the applicant has not signed the form.

      Processing for additional persons

      If there are additional persons on the form check the age of the additional person before processing. Process the My Health Record choice if the additional person is:

      • under 14 and the applicant has signed the My Health Record form, or
      • 14 years and over and has signed their own signature box

      Do not select Question is not on form or not answered if these conditions are met.

      Only select Question is not on form or not answered if the additional person is:

      • under 14 and the applicant has not signed the My Health Record form
      • 14 years and over and has not signed their own signature box

      Select Validate

      6

      Immigration Datalink Search + Read more ...

      Where the entitlement type returns:

      • an Immigration Datalink match (except for New Zealand citizens):
        • select the radio button against the record to be linked to the Department of Home Affairs (this should match the ‘Application ID’ available on the Home Affairs letter where one is provided)
        • select Link button
        • the Confirm New Medicare Entitlement Details screen is displayed
        • go to Step 8
      • dash (-) or non-alpha, and client status shows:
        • Lodged – this indicates the applicant does not have permission to work, go to Step 7
        • Refused – appeal document and work right check required, see the Appeals to a visa refusal table
        • Withdrawn – procedure ends here

      Record relationship document details + Read more ...

      • Key in type of relationship document in the Reference text field
      • Select:
        • Validate
        • Return
        • Validate

        • Change to amend the details as required
        • Update to confirm the details displayed. A message displays confirming that a consumer record has been created
      • Select OK to acknowledge the message

      Address search in CDMS

      Table 7

      Step

      Action

      1

      Address search in CDMS + Read more ...

      On the CDMS Home page, select Inquire on Consumer Details select Address search.

      2

      Enter address search criteria + Read more ...

      • Program field (default is All Programs)
      • Country field (default is Australia)
      • Key Locality or postcode - select from the drop down box
      • Key Address line - select from the drop down box

      3

      Action any address matches + Read more ...

      Select either Consumers or Groups tab.

      If the address:

      • matches an address in the Postal Address File (PAF):
        • the Address Usage - Consumer or Group details screen displays showing the address details
        • confirm name, address and date of birth, then go to Step 4
      • matches an address with correction:
        • the Address Matching screen displays a message indicating that the address matches with correction, select OK to acknowledge the message
        • select Accept to accept the address correction
        • select Change to change the address
        • select Continue to continue with the address keyed
      • does not match an address:
        • the Address Matching screen displays a message indicating the address is not found
        • select OK to acknowledge the message

      4

      Customer Group Details displayed + Read more ...

      Select the customer or group details using the radio button.

      Select 1 of the following as required:

      • Personal - Personal details of the group contact
      • Entitlement - Medicare entitlement details of the group contact
      • Group - Group or card details of a group

      Result: the system displays the information requested.

      Note: to return to the Inquiry or amendment Search screen, select Search Page.

      If no matches return, see Step 2 in Table 2.

      Action search results

      Table 8

      Step

      Action

      1

      Determine search results + Read more ...

      What are the search results?

      • Only one matched customer, go to Step 2
      • Only one match customer with no Immigration Datalink, go to Step 3
      • One or more immigration data matches, go to Step 4
      • More than one matched customer, go to Step 5
      • No records found, go to Step 6

      2

      Only one matched customer + Read more ...

      Note: this applies to all search types.

      The relevant search screen displays, showing the details of the customer. Confirm Medicare number with customer and proceed with inquiry or amendment.

      Procedure ends here.

      3

      Only one matched customer with no Immigration Datalink + Read more ...

      Note: this applies to all search types.

      • Confirm matched customer is correct and proceed with amendment or inquiry
      • Eligibility documents and a Visa Entitlement Verification Online (VEVO) check done by a Service Officer must be provided where a person has now applied for permanent residency or a protection visa, see Applicants for permanent residency eligibility for Medicare

      Procedure ends here.

      4

      One or more immigration data matches + Read more ...

      Note: this applies to register new customer and/or immigration searches.

      • Check all exact and possible immigration data matches and confirm the correct match. Where there is no existing customer record, select New Consumer and enrol as per immigration data and Medicare Enrolment application form
      • If a customer record already exists, update the entitlement type on CDMS with the correct immigration data match

      Note: eligibility documents and a Visa Entitlement Verification Online (VEVO) check done by a Service Officer must be provided where a person has now applied for permanent residency or protection visa, see Applicants for permanent residency eligibility for Medicare

      Procedure ends here.

      5

      More than one matched customer + Read more ...

      Note: this applies to all search types.

      • Narrow the search parameters by including the first initial of the first name of the customer
      • If the search still returns too many possibilities, include first name (and possibly second name, if relevant)
      • Select the correct customer and proceed with amendment or inquiry

      Note: if more than one exact customer record is returned, further investigation is required as it may be a duplicate record. Contact Medicare Customer Data Integrity Unit for follow-up.

      6

      No records found + Read more ...

      Note: this applies to all search types.

      If the customer is:

      • an Australian citizen, go to Step 7
      • a migrant or an applicant for permanent residency, go to Step 8

      7

      No record found for an Australian citizen + Read more ...

      Where an Australian citizen does not have an existing customer record, this may be due to:

      • residing or previously resided overseas
      • newborn child
      • government and defence force personnel
      • prisoners and inmates

      Documentation to prove eligibility is required.

      Procedure ends here.

      8

      No record found for a migrant or an applicant for permanent residency + Read more ...

      Where the applicant is a migrant or an applicant for permanent residency, documentation from the Department of Home Affairs and a Visa Entitlement Verification Online (VEVO) check run by a Service Officer is required.
