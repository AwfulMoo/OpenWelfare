Explanation of codes and terms used

Table 1: This table explains the codes and acronyms used in the following tables.

Item

Description

1

Code + Read more ...

This a 3 letter code used to reject, suspend or cancel.

2

Description + Read more ...

This is the expanded meaning of the code.

3

Valid Action Codes + Read more ...

REJ - rejection

SUS - suspension

CAN - cancellation

RES - restoration

4

Initiation + Read more ...

Man - reasons that can be manually input on the Benefit Action (BA) screen

Auto - automatically occurring (cannot be keyed manually)

Note: on the BA screen use the ? in the cancellation reason field for an online list of codes.

5

Restoration + Read more ...

M - payment can be manually restored via the BA screen

No - payments cannot be restored

If a payment is rejected, this cannot be restored. The claim must be reindexed and processed as a re-grant.
