Reclaiming after being paid for the same event + Read more ...

If a customer has already been paid for the same event, they cannot receive another payment, even if they have a new and valid impact reason.

To check if a customer has been paid for the same event

  • Go to the Payment Summary (PS) screen by keying PS next to Super Key and press [Enter]
  • Display payments for: select FSP
  • Number of months in past: defaults to 3. If the event has been activated for longer than 3 months, adjust this number to capture relevant period
  • Select Execute
  • If a Flexible Support Payment (FSP) payment shows, select to view the Payment Components. The Benefit Qualifier Code will show the event code it applied to,
  • Key 'DL' next to Super Key and press [Enter] to check the customer’s Document List (DL) screen
  • Review any Claim Finalisation Note to determine if the customer has already been granted for the same event
  • If the customer has been paid for the same event, the customer cannot be paid again and their claim must be rejected
  • If approval has
    • been given to reject claims, process the claim. See Table 2
    • not been given, see Step 2 in Table 3 to place the claim on hold

Reclaiming after rejections or withdrawals + Read more ...

Customers are able to reclaim for an event if their previous claim(s) have been rejected or withdrawn.

How to check withdrawn and rejected claims

  • Select the Transactions icon
    • This will show any claims withdrawn or not completed
  • Select All Transactions
    • This will show all completed claims, including claims:
      - granted and paid, and
      - rejected

Take care when assessing the new claim to make sure:

  • any previous reasons for rejection are taken into account
    • this includes concerns about the validity of the claim or the customer’s identity
  • the customer has not changed the circumstances in the new claim to be granted a payment
  • claimants are not issued another payment for the same event if the AGDRP or NZ DRP is to be granted
    • this includes children, and secondary claimants (NZ DRP)

Why has the customer lodged a new claim for the same event?

  • Previous claim(s) was rejected, go to Step 12
  • Previous claim(s) was withdrawn before the claim was processed:
    • Withdrawn once (one withdrawal and 2 claims lodged), go to Step 16
    • Withdrawn 2 times or more (minimum of 2 withdrawals and 3 claim lodgements), go to Step 14
