Handle call from Responsible Officer (RO), Organisation Maintenance Officer (OMO) or Contracted Service Provider (CSP) on behalf of an organisation 012-03020010

This page contains contact details for Tier 3 Service support and information on what details may be provided in response to a phone call from a Responsible Officer (RO), Organisation Maintenance Officer (OMO) or Contracted Service Provider (CSP) officer calling on behalf of an organisation including links to forms. Links on this page may be unavailable to external audiences.

On this Page:

Contact details

For Tier 3 HI Service support see Internal contacts in Healthcare Identifiers (HI) Service

RO or OMO calls with a HPOS query

Table 1: This table describes what information may be provided, or what actions may be taken when a RO or OMO calls on behalf of an organisation with a request using Health Professional Online Services (HPOS).

For more information on how to access the Healthcare Identifiers (HI) Service via HPOS, see HPOS user guide.

If the enquiry is about…

then…

linking a CSP organisation to a healthcare organisation using HPOS

advise the RO or OMO that this function may be performed online.

Advise the individual to:

  • logon to the HI Service via the HPOS website using their individual Public Key Infrastructure (PKI) certificate or Provider Digital Access (PRODA) account
  • from the HI Services available screen, select the My organisation details hyperlink
  • from the Organisation snapshot screen, select the CSP Links tab, and then select the Manage CSP links hyperlink
  • either add or remove the CSP organisation

The CSP Organisation List screen will display.

searching the Healthcare Provider Directory (HPD) for Healthcare Provider Organisation details using HPOS

Menu item: Search HI provider directory service for organisation

advise the RO or OMO that this function can be performed online.

Advise the RO or OMO to:

  • logon to the HI Service via the HPOS website using their individual PKI certificate or PRODA account
  • from the HI Services available screen, select the Search HI provider directory service for organisation hyperlink

User can search with the Healthcare Providers Identifier - Organisation (HPI-O) or demographic details.

Note: the demographic search minimum search criteria includes organisation name and state.

The result will return all details that were consented to be published in the HPD for all organisations which match the search criteria. If the caller does not have a PKI certificate, a PRODA account or is unable to logon, this can be confirmed over the phone - ensuring the information requested and provided back is the same as if using HPOS.

searching the HI Service to verify any organisations HPI-O number using HPOS

Menu item: Search HI for organisation

advise the RO or OMO that this function can be performed online.

Advise the RO or OMO to:

  • logon to the HI Service via the HPOS website using their individual PKI certificate or PRODA account
  • from the HI Services available screen, select the Search HI for organisation hyperlink

User can search with their HPI-O only. The result will return HPI-O number and status only.

If the caller does not have a PKI certificate, a PRODA account or is unable to logon, this can be confirmed over the phone - ensuring the information requested and provided back is the same as if using HPOS.

searching the HPD for Healthcare Provider Individual details using HPOS

Menu item: Search HI provider directory service for individual

advise the RO or OMO that this function can be performed online.

Advise the RO or OMO to:

  • logon to the HI Service via the HPOS website using their individual PKI certificate or PRODA account
  • from the HI Services available screen, select the Search HI provider directory service for individual hyperlink

User can search with their HPI-I or demographic details. Note: the demographic search minimum search criteria includes organisation name and state. The result will return all details that were consented to be published in the HPD for all individuals which match the search criteria. If the caller does not have a PKI certificate, a PRODA account or is unable to logon, this can be confirmed over the phone - ensuring the information requested and provided back is the same as if using HPOS.

searching the HI Service to verify any providers HPI-I using HPOS

Menu item: Search HI for provider individual

advise the RO or OMO that this function can be performed online.

Advise the RO or OMO to:

  • logon to the HI Service via the HPOS website using their individual PKI certificate or PRODA account
  • from the HI Services available screen, select the Search HI for provider individual hyperlink

User can search with HPI-I OR Ahpra Registration ID PLUS limited demographics – result will return HPI-I, status and any other information used in the search criteria.

If the caller does not have a PKI certificate, a PRODA account or is unable to logon, this can be confirmed over the phone - ensuring the information requested and provided back is the same as if using HPOS.

RO or OMO calls with a non-HPOS related request

Table 2: This table describes what information may be provided, or what actions may be taken when a RO or OMO calls on behalf of an organisation with a request not related to Health Professionals Online Service (HPOS).

If the enquiry is about…

then…

confirmation of their own RO or OMO ID number

this can be disclosed over the phone.

confirmation of the organisation HPI-O they are linked to

advise the RO or OMO that this function may be performed online using HPOS or disclosed over the phone.

  • logon to the HI Service via the HPOS website using their individual Public Key Infrastructure (PKI) certificate or Provider Digital Access (PRODA) account
  • from the HI Services available screen, select the My organisation details hyperlink

The organisation's HPI-O will be displayed in the Summary tab.

consent to publish details in the HPD (must be an organisation the RO or OMO is linked to)

advise the RO or OMO that this function may be performed online.

  • logon to the HI Service via the HPOS website using their individual PKI certificate or PRODA account
  • from the HI Services available screen, select the My organisation details hyperlink
  • from the Organisation snapshot screen, select the HPD tab
  • select the Add new PDS entry details hyperlink to create a new HPD entry

The HPD entry will be created.

This can be actioned in response to a phone request. Make sure a file note of the request is made and archived.

amendment of details published in the HPD (must be an organisation the RO or OMO is linked to)

advise the RO or OMO that this function may be performed online.

  • logon to the HI Service via the HPOS website using their individual PKI certificate or PRODA account
  • from the HI Services available screen, select the My organisation details hyperlink
  • from the Organisation snapshot screen, select the HPD tab
  • select either the Amend or Delete hyperlink to either amend or remove the HPD entry

The HPD entry will be amended/deleted.

This can be actioned in response to a phone request. Make sure a file note of the request is made and archived.

amendment of record details pertaining to their own record or the HPI-O record they are linked to

advise the RO or OMO that this function may be performed online.

  • logon to the HI Service via the HPOS website using their individual PKI certificate or PRODA account
  • to amend:
    • organisation details, from the HI Services available screen, select the My organisation details hyperlink
    • individual details, select the My OMR (or RO) details hyperlink

Details such as a change of address and electronic communication details can also be amended via a phone call.

Make sure a file note of the request is made and archived.

amendments to details such as name, date of birth, sex, must have proof of the change

direct the caller to HPOS where they can amend their details or if they don’t have a HPO login direct them to the Services Australia website to obtain the HI Services - application to amend an organisation officer’s personal details form (HW042).

adding of a new OMO record or removing an existing OMO from an organisation (must be an organisation the OMO is linked to)

advise the RO or OMO that this function may be performed online.

  • logon to the HI Service via the HPOS website using their individual PKI certificate or PRODA account
  • from the HI Services available screen, select the My organisation details hyperlink
  • from the Organisation snapshot screen, select the OMR tab, select the Manage maintenance roles hyperlink
  • select add button to create an additional OMO or remove button to delete any linked OMOs

If the caller does not have a PKI certificate, a PRODA account or is unable to logon, direct the caller to the Services Australia’s website to obtain the HI Service - application to replace a responsible officer or add or remove an organisation maintenance officer for an organisation form (HW040).

a  request to link or unlink providers to an organisation (must be an organisation the OMO is linked to)

advise the RO or OMO that this function may be performed online.

  • logon to the HI Service via the HPOS website using their individual PKI certificate or PRODA account
  • from the HI Services available screen, select the My organisation details hyperlink
  • from the Organisation snapshot screen, select the HPI-I Links tab, select the Manage providers hyperlink
  • either add or remove providers

This can be actioned in response to a phone request.

Make sure a file note of the request is made and archived.

a request for a new PKI certificate for an RO or OMO

if an individual needs access to HPOS and doesn’t have a current PKI certificate, they need to register for a PRODA account.

A request to link an existing PKI certificate for an RO or OMO

authorised Subject Matter Experts (SME) may action this request in response to a phone request. Refer to an SME.

a request for a new a National Authentication Service for Health (NASH) PKI Certificate for a Healthcare Provider Organisation

refer caller to the NASH PKI certificates pages on the Services Australia website for more information and application details.

other matters

transfer call to Tier 3 HI Serivce support.

RO or OMO calls on behalf of a CSP organisation

Table 3: This table describes what happens when a RO or OMO calls on behalf of a CSP organisation.

If the enquiry is about…

then…

whether a CSP organisation is linked to the ROs or OMOs organisation

this can be disclosed over the phone.

whether the organisation needs to link their software provider as a CSP

refer the caller to their software vendor. The software vendor will advise the organisation if this is necessary.

other matters

transfer call to Tier 3 HI Service support.

CSP officer calls on behalf of a CSP organisation

Table 4: This table describes what happens when a CSP officer calls on behalf of a CSP organisation.

If the enquiry is about…

then…

linking a CSP organisation to a HPI-O

(Linking in HPOS)

this cannot be performed by a CSP officer. The linking of a CSP organisation to a HPIO must originate from the Healthcare Provider organisation's RO or OMO, and this can be done online using HPOS.

confirmation of their own CSP officer number

this can be disclosed over the phone.

confirmation of the CSP organisation number of the organisation they are linked to

this can be disclosed over the phone.

removing the link between their own CSP officer record and a CSP organisation

direct caller to the Services Australia website to obtain the HI Service - application to add, replace or remove a contracted service provider officer form (HW046).

amendments to details such as name, date of birth, gender or qualifications, they must have proof of the change

direct caller to the Services Australia website to obtain the HI Service - application to amend a contracted service provider officer's details form (HW047).

amendments to details such as phone number or address

this can be performed by the Service Officer.

Make sure a file note of the request is made and archived.

requesting a new PKI certificate for HI purposes

direct caller to the Services Australia website to obtain the HI Service – application to link a PKI certificate form (HW045)

asking about using a NASH PKI certificate to access healthcare identifiers

direct caller to the Services Australia website for National Authentication Service for Health (NASH) information and warm transfer caller to the NASH PKI team if they would like to speak to someone.

linking an existing PKI certificate to a CSP organisation record

this can be performed by authorised Service Officers.

See: Create or link a PKI certificate for a HPI-I or a HPI-O procedure (in Authentication required to access digital health in Healthcare Identifiers (HI) Service).

Note: a file note must be completed.

other matters

transfer call to Tier 3 HI Service support.

RO calls about doing OMO tasks

Table 5: This table describes what happens when an RO calls about doing OMO tasks.

If the enquiry is about…

then…

RO asks if they can do something that is an OMO task

advise the RO:

  • functions and responsibilities of ROs and OMOs are distinct under the legislation
  • they can nominate themselves as a seed OMO via HPOS. If HPOS is not available they can do this verbally over the phone after passing the security check
  • OMO tasks can be completed through HPOS

They can also nominate themselves anytime by completing the HI Service - application to replace a responsible officer or add or remove an organisation maintenance officer for an organisation form (HW040).
