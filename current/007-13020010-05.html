Explanation of status fields on the MCI screen when no action is required

Table 1

Item

Description

1

ATO End + Read more ...

Australian Taxation Office (ATO) have requested that the link be ended with a stop link transaction. An Automatic mismatch letter will be sent.

No action required.

2

ATO Link Fail + Read more ...

ATO has not sent a return transaction to either of the 2 initial establish link transactions so it is assumed the link has failed.

No action required.

3

ATO Pending + Read more ...

ATO unable to give Centrelink a final resolution of either a match or mismatch with the establish link transaction and are doing manual follow up or the customer has confirmed Tax File Number (TFN) with Centrelink after Mismatch 1 or 2 and ATO is conducting a manual follow-up.

No action required.

4

Awaiting 1 + Read more ...

Either the first initial establish link transaction or the first link transaction after the customer has stated, but not confirmed their TFN, has been sent and Centrelink are awaiting the ATO return transaction which will show whether the customer has matched or not.

No action required.

5

Awaiting 2 + Read more ...

The second initial establish link transaction.

No action required.

6

CLK End + Read more ...

Centrelink have requested that the link be ended with a stop link transaction.

No action required.

7

Confirmed 1 + Read more ...

Customer has confirmed TFN details after Mismatch 1 or 2 and Centrelink have either changed details or confirmed that existing details are correct.

No action required.

8

Confirmed 2 + Read more ...

Centrelink have sent a second link transaction after the customer has confirmed their TFN details as the ATO have not responded within 10 days of the first confirm link transaction.

No action required.

9

Match + Read more ...

Customer has been identified as a mutual customer of Centrelink and ATO.

No action required.

Explanation of status fields on the MCI screen when action is required

Table 2

Item

Description

1

CLK Pending + Read more ...

ATO is unable to confirm an initial match as there are Multiple CRNs for the reported TFN. There will be a CRN in the Multiple CRN field for the CLK Pending Status.

Action: Manual follow-up required. See Mutual Customer Identification for family assistance customers with status of Centrelink Pending (CPN).

2

Error + Read more ...

ATO has returned the transaction to Centrelink as there is an error in the transaction for example, Transaction data is corrupt.

Action: No Service Officer intervention required. To be resolved by production support.

3

Mismatch 1 + Read more ...

Customer has not been identified as a mutual customer with the first established link transaction and a letter has been sent to the customer by Centrelink to restate their TFN.

Action: When the customer replies to this letter the TFN details may be the same or different to what Centrelink already have on the record.

  • If they are different, then the user will need to change the customer's TFN details. This will send an automatic establish link transaction
  • If the TFN details are the same, then verification of TFN needs to be requested. Once supplied, type 'SIT' in the Request Action: field on the MCI screen

Note: the Mutual Customer Identification Progress Stage Screen (MCIPS) will display historical ATO information in relation to the progress of a customer match transaction.

4

Mismatch 2 + Read more ...

2nd mismatch - customer has replied to 1st mismatch letter by either supplying a new TFN or restating the existing TFN without confirmation of the TFN and Centrelink has sent a second link transaction to the ATO that has again come back as a mismatch. A further letter is sent to the customer asking them to provide verification of the TFN.

Action: Once the TFN verification has been received:

  • If it is different from the one Centrelink has recorded, change the TFN on the Tax File Number (TFN) screen and code 'SIT' on the Request Action: field on the MCI screen
  • If the TFN is the same as what Centrelink already has, request the customer provides proof of their TFN, such as a Notice of Assessment from the ATO. There is no need to update the TFN screen if the TFN is verified and is the same as the one previously provided, instead code 'SIT' on the Request Action: field on the MCI screen. If the customer provides proof of their TFN and it is different from what is on the system, refer to the first dot point above

If the customer does not comply with a Mismatch 2 request their FTB will be cancelled as the TFN is non-compliant.

Note: the Mutual Customer Identification Progress Stage Screen (MCIPS) will display historical ATO information in relation to the progress of a customer match transaction.
