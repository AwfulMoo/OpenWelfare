This page contains information about performing security checks and releasing Individual Healthcare Identifier (IHI) information. Includes individuals who are not eligible for Medicare or Department of Veterans' Affairs (DVA).

On this page:

Release of IHI information

Security check and release of IHI - Medicare or DVA eligible individuals

Security check and release of IHI - non-Medicare or DVA eligible individuals

Release of IHI information

Table 1: how to determine the correct process for releasing IHI information.

Step

Action

1

Releasing IHI information + Read more ...

An IHI can be given to an individual after they pass a security check.

An individual can ask about:

  • their own IHI
  • IHI for their child under 14 years of age.

Note: both the parent and the child's identity must be confirmed before releasing the child's information.

2

Select the release of IHI process + Read more ...

Is the IHI for an individual (including a child under 14 years old) who is eligible for Medicare or DVA?

  • Yes, see Security check and release of IHI - Medicare or DVA eligible individuals
  • No, see Security check and release of IHI - non-Medicare or DVA eligible individuals

Security check and release of IHI - Medicare or DVA eligible individuals

Table 2: how to perform a security check and release IHI details for individuals who are eligible for Medicare or DVA. Includes children under 14 years old.

Step

Action

1

Perform security check + Read more ...

Perform a security check, see Authenticating a Medicare customer.

Note: if details are requested for a child under 14, both the parent and the child's identity must be confirmed before releasing the child's information.

2

Outcome of security check + Read more ...

Does the information for the individual (and their child under 14 if requested) from the security check match their CDMS record?

  • Yes, go to Step 3
  • No, tell the individual that information cannot be given to them as they did not meet the requirements of the security check. Note: if the individual disputes this or says Medicare records are incorrect:
    • via telephone request, transfer the call to a Team Coach
    • via face-to-face request, refer the customer to the Service Centre Manager

3

Return to CDMS Home + Read more ...

  • Select Inquire on Consumer Details
  • The Inquiry Search screen displays

4

Select Medicare in the Program field + Read more ...

Conduct a basic or detailed search to find the Medicare record.

See Search for Individual Healthcare Identifier (IHI) in Healthcare Identifiers (HI) Service.

Is the request for a child under 14?

  • Yes, go to Step 5
  • No, go to Step 6

5

Record found showing Personal Details screen + Read more ...

Is a record found showing the Personal Details screen?

  • Yes:
    • Select the Group button at the bottom of the Personal Details screen
    • Select the Members Details tab to confirm the child is on the parent's Medicare card
    • Go to Step 6
  • No, refine the search again by doing a basic or detailed search

6

Select Entitlement + Read more ...

Hover over the Summary button and select Entitlement from the drop down list.

The IHI for the selected individual displays in the IHI Details section of the Program Entitlement Summary page.

7

IHI system log requests + Read more ...

All requests for printed IHI system log must be sent in writing to Tier 3 support.

Security check and release of IHI - non-Medicare or DVA eligible individuals

Table 3: how to perform a security check and release IHI details for individuals who are not eligible for Medicare or DVA. Includes children under 14 years old.

Step

Action

1

Confirm name, date of birth and address + Read more ...

Note: if an IHI is requested for a child under 14, repeat Steps 1 and 2 for both the parent and the child.

In CDMS, confirm the below details for the individual (and their child under 14 if requested):

  • Full name
  • Date of birth
  • Address

Do the details match the CDMS record?

  • Yes, go to Step 2
  • No, go to Step 5

Note: if no record is found, go to Step 4.

2

Confirm 2 additional personal items + Read more ...

In CDMS, confirm any 2 of the below items for the individual (and their child under 14 if requested):

  • Phone number (if recorded in CDMS)
  • Email address (if recorded in CDMS)
  • Document of identity supplied to create their IHI - this can be found in the Status details tab of the CDMS record
  • Document ID number found on the document supplied to create their IHI - this can be found in PaNDA. Search PANDA using the below criteria:
    • Program: Miscellaneous
    • Work Type: MISC_HISERVICE_URG – Healthcare Identifiers Service Consumer Reg MS003, or
    • Work type: MISC_HISERVICE_ESC – Healthcare Identifier Tier 3 Escalation
    • Search using the consumer’s name

Did the individual confirm 2 items correctly?

  • Yes, go to Step 3
  • No, go to Step 5

Note: if no record is found, go to Step 4.

If a person knows their IHI, and is calling for another purpose, their IHI can be used as one security check personal item. Verify another item as listed above.

3

Security check successful + Read more ...

The individual can be given their IHI once the security check is passed.

See Search for Individual Healthcare Identifier (IHI) in Healthcare Identifiers (HI) Service.

Does the individual want a reprint of their IHI notification letter?

  • Yes, go to Step 6
  • No, procedure ends here

4

Record not found in CDMS + Read more ...

Ask the individual if they have submitted an MS003 form:

  • Yes, search PaNDA for the application to see the progress or status
  • No, tell them to complete and submit Request or update an Individual Healthcare Identifier form (MS003). Procedure ends here

Note: the MS003 must be completed to request or update an IHI.

5

Security check unsuccessful + Read more ...

If the individual's identity can't be confirmed:

  • tell them that they:
    • did not meet the requirements to pass the security check
    • can get a copy of the original IHI letter sent to the address in the application form (go to Step 6)
  • do not:
    • release any information
    • make any updates to the IHI record

6

Reprint IHI notification letter + Read more ...

If the individual asks for a letter reprint, email My Health Record - Tier 3.

Include:

  • details of the person requesting the letter
  • a return phone number in case details need to be confirmed
