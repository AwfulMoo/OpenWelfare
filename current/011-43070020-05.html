Table 5: This table provides additional information on adjustment screens and their functions.

Screens

Description

FHSI

Register to perform an adjustment

Key the following details into corresponding fields:

  • minor customer ID (Minor ID or Registration number)
  • original Claim ID
  • original DOL
  • adjustment Claim ID (using Claim ID followed by ‘A’ for the first adjustment)
  • adjustment DOL
  • ADJ on ADJ (N/A in the first adjustment)

Note: the ADJ on ADJ field is only used if an adjustment on adjustment is required to be performed,

Then select enter. Details of the private health insurer or billing agent will appear at the bottom of page.

Then enter again.

FHSC

Commence amending claim details

PF3 to confirm adjustment result. The claim result will then be displayed in the FHAC screen.

FHAC

Access the claim

Segments are displayed in original card number order and any assessing errors applicable to the first segment are displayed.

  • Where no assessing errors exist or the Service Officer is satisfied the adjustment result for the segment will be displayed, and [F3 NXT ADJ] will be activated:
  • Each time [F3 NXT ADJ] is pressed, the adjustment details are immediately written to Patient History.
  • Where an adjustment claim includes multiple segments for the same patient the Service Officer can choose the order in which these segments are assessed by using the [F11-SKIP] key.
  • [F11] enables a Service Officer to skip over one or more segments to find the segment to be assessed and updated first.

This allows the Service Officer to control the order in which segments are updated to history to ensure correct assessing rules are applied.

When the last segment in the claim has been assessed the message U TO UPDTE will be displayed.

Key ‘U’ to update the claim and the adjustment is complete. The claim result will then be displayed. The final claim results can be:

  • Underpayment - total result of all segments is greater than original amount paid
  • Overpayment - total result of all segments is less than the original amount paid
  • Statistical - total result of all segments is equal to the original amount paid
  • Suppressed - result of claim is a non-recoverable overpayment.

FHAU

Update the claim

When U to UPDTE is pressed on FHAC screen, the FHAU screen is returned:

The screen displays the final result for all adjusted segments and benefit payable:

  • overpayment
  • underpayment
  • history adjustment (statistical adjustment)
  • suppress adjustment

The total of all overpayment segments is off set against the total of all underpayment segments and the amount payable is the difference between the two.

For example, if 2 segments were adjusted within a claim and one segment resulted in an overpayment of $50.00 and the second segment resulted in an underpayment of $30.00, then total adjustment result will be an overpayment of $20.00.

FHAI

Confirms update of adjustment completed

FHRI

Recall a claim paid today to be reversed

The FHRI transaction allows a claim updated with U TO UPDTE at the FHAC screen to be reversed on the same date of processing.

Control line

To recall the claim for reversing key control line FHRI,MINOR ID OR REGISTRATION NUMBER,CLAIM ID,DOL=DDMMYY

FHRR

R to REVERSE

When recalling the claim to reverse, over key FHRI on the control line with ‘FHRR’.

The control line displayed is FHRR,MINOR ID OR REGISTRATION NUMBER,CLAIM ID,DOL=DDMMYY

Update the reversal (FHRR)

The message REVERSED is displayed and the status of the claim reset to ADJUST.

The claim can either be reassessed or deleted by recalling the claim using the FHAI transaction.
