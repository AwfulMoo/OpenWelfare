Report Management Distribution System (RMDS) + Read more ...

MCO debts are transferred to Medicare Recovery via a Medicare mainframe Report Management Distribution System (RMDS) Report (MCO111).

The Medicare program area will:

  • identify the debt and raise it in the Medicare mainframe
  • issue the overpayment letter to the claimant, requesting repayment of the debt

The Medicare Recovery team will:

  • download the RMDS report from the Medicare mainframe
  • remove any incorrect overpayment information advised by the program area
  • create financial records in SAP
  • upload the remaining data, and
  • process the write-off in SAP

Note: follow-up action is only required where the debt is the result of incorrect processing of in-hospital services. For these cases, raise the debt against the provider.

Go to Step 7.

4

Voluntary acknowledgement from the customer + Read more ...

If a customer advises of an incorrect payment, Medicare Recovery will:

  • create the customer (if required)
  • raise the invoice, and
  • allocate the payment, when it is received

5

Debt Advice Notice (DAN) + Read more ...

The Medicare program area will:

  • identify the debt
  • complete a DAN
  • prepare an overpayment letter for the customer, and
  • send the overpayment letter and the DAN to Medicare Recovery

Medicare Recovery will:

  • create the customer if required
  • raise the invoice
  • send the invoice (due date 30 days) and overpayment letter to customer

6

Follow up of small debts + Read more ...

For customers who have several small debts that total over $100, send a statement:

  • 30 days after the invoice is created
  • with a due date of 14 days from the date of issue

Note: if a customer contacts regarding a Medicare Patient claim overpayment under $100, refer them to the Medicare General Enquiry line. See Contact details on the Resources page.

7

Write-offs + Read more ...

Information in the RMDS report 111 is used to write off some Medicare overpayments.

Public debts under $100 are written off (WOD) without creating a debt record in SAP. For all other debts consider:

  • the total number of debts
  • the total value of all debts owed by the customer, including debts for other programs
  • the probability of successful recovery and;
  • if it is cost effective to recover the debt

Write-offs are not permanent and can be re-raised at any time, for any reason. Do not advise the customer of the write-off.

8

Weekly report by Medicare Program area + Read more ...

Use the RMDS report to identify overpayments for patient claims over $100.

See Active recovery of outstanding Medicare debt for information on sending an initial tax invoice and statement, and phone contact.

Extracting data from the Report Management Distribution System (RMDS) report s (over $100)

Table 2

Step

Action

1

Extracting data + Read more ...

To extract data for debts over $100, open the Excel macro spreadsheet.

Go to the file in the shared folder: FINANCE > Macros > Digitalised signed macros:

  • open the RMDS MCO 101 Excel spreadsheet
  • select Enable Content
  • the spreadsheet will automatically be on the Paste Data Tab. If not, select Paste Data

2

Open the RMDS function from Mainframe + Read more ...

Log into Mainframe:

  • Key the command 'OPOI'
  • Select CODE SOURCE OFFICE NAME by keying ‘S’ in front of AMAG SHQ FINANCE
  • Key 'RMDS' to go to the RMDS log in page
  • Log into RMDS

3

Run the macro + Read more ...

Return to the RMDS MCO 101 Excel spreadsheet:

  • Select Run Macro
  • At the pop up screen, enter operator number
  • Run the macro

4

Verify the amount + Read more ...

Check the amount for each State in the Opp amount column against column P. Are the amounts the same?

  • Yes, save the RMDS MCO 101 spreadsheet. Go to Step 5
  • No, manually verify the amount from the spreadsheet against the RMDS report in mainframe. When correct, go to Step 5

5

Upload customers + Read more ...

Go to the Customer Upload tab in the spreadsheet:

  • Copy everything in the Customer Upload tab into a new Excel spreadsheet and save as a TXT file
  • Upload the TXT file into SAP. See Table 3 in SAP invoices and customer navigation

6

Upload invoices + Read more ...

Go to the Invoice Upload tab in the spreadsheet:

  • copy everything in the Invoice Upload tab into a new Excel spreadsheet
  • remove any overpayment information relating to Compliance or Department of Health and Aged Care (unrelated to Medicare Recovery) then
  • save as a CSV file
  • upload the CSV file into SAP. See Table 3 in SAP invoices and customer navigation

7

Save the File + Read more ...

Go to the file located in the shared folder: FINANCE > Medicare Overpayments > RMDS Reports. Name the file RMDS YYMMDD (For example, RMDS 181231).

Table 3

Step

Action

1

Extracting data + Read more ...

To extract data from the Medicare Overpayment (MCO) and Bulk Bill (BB) reports, run Transaction FBL5N in SAP:

  • Company Code: 'HIC'
  • Select Dynamic selections
  • Reconciliation acct field:
    • MCO report - key '13311'
    • BB report - select [arrow] to choose multiple accounts; key '13318' and '13110'
  • Document type: 'DR'
  • Clearing Date: Clear the date range. Include the dates for the new reporting period
  • Layout: '/DRP_MBS_REP'
  • Select Execute [F8]

2

Export the file to Excel + Read more ...

Go to the List menu:

  • Select Export
  • Select the spreadsheet in the pop up box
  • Save the file to desktop

3

Edit/format the Excel sheet + Read more ...

To edit or format the Excel spreadsheet:

  • Go to cell N2. Copy and paste Formula 1 into cell N2. See the Resources page for the formula
  • Go to cell O2. Copy and paste Formula 2 into cell O2. See the Resources page for the formula
  • Copy the formulas from columns N and O to every row in the spreadsheet
  • Select columns N and O, then copy and Paste Special - Values to remove the formulas
  • Select the whole spreadsheet [Ctrl+A]
  • Go to the Data menu in Excel
  • Select Sort
  • Select column N and O from the list
  • Delete values from cells in Column O that do not contain a Q, N, V, T, S or W
  • Update the blank cells in column O with:
    • the State (if known), or
    • key a 'Z' in each blank cell
  • Delete rows for FAR and invoice reversals by filtering column L (Assignment) and on column I (Clearing Document)
  • Create the following tabs on the spreadsheet:
    • 'BB DRI'
    • 'BB VCR'
    • 'MCO DRI' and
    • 'MCO VRC'
  • Filter by column N for Bulk bill and copy the data on tabs BB DRI and BB VCR
  • Filter by column N for MCO and copy the data on tabs MCO DRI and MCO VCR
  • On tab BB DRI, filter column J to include items with last letter D
  • On tab BB VCR, filter column J to include items with last letter V
  • On tab MCO DRI, filter column J to include items with last letter D
  • On tab MCO VRC, filter column J to include items with last letter V
  • Insert a blank row above every letter change in column O
  • Highlight the new blank row in yellow

4

Email the reports + Read more ...

  • Create email content:
    • Subject Line: Bulk Bill / MCO Payment Report for DD-MM-YY to DD-MM-YY
    • Body: Hello all,
      Please find attached payment report for the period DD-MM-YY to DD-MM-YY. If you have any enquiries please email (insert Medicare Recovery address)
      Thank you
  • Send email to Medicare Provider Services and copy to:
    • Medicare Provider Communications
    • Medicare Provider Services Support
  • Email Medicare Recovery once the data transfer to the Excel spreadsheet is complete
  • Delete the file saved to the desktop
