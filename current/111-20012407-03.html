Notifications + Read more ...

Notifications give line managers real-time information about WLM activity for staff reporting to them. This information can be used to identify:

  • opportunities to provide staff recognition for positive work practices
  • if staff require more support to better manage their workload

The Resources page contains a description of each of the Notification types.

Notifications are accessed using the bell icon at the top right of Work Optimiser:

  • a red counter on the bell icon shows new notifications are available for viewing
  • a black counter shows notifications have been viewed but not closed
  • select X to close a chosen notification
  • select Close All to close all notifications

Select Settings to personalise the types of notifications received:

  • select the team member’s button to choose the staff each notification type will apply to
  • use the Yes/No toggle to turn a notification type subscription on or off

2

Worklist + Read more ...

Team Leaders can view and manage work assigned to themselves and their direct reports, using the Worklist My Team Work search.

Analysts can view and manage work assigned to staff from a programme level, using the Worklist All Work search.

From the Worklist, Team Leaders and Analysts can:

  • find work assigned to themselves, their direct reports, or work related to a customer record
  • save searches
  • redirect assigned work items from their inbox (or direct report's inbox) to another staff member
  • return work items from their inbox (or direct report's inbox) to the National Pool
  • assign work to their inbox for action
  • view documents in PaNDA (Medicare only)

For more on managing work items, see Work Optimiser for staff.

3

Worklist search criteria + Read more ...

Search for work items from the Worklist using 1 or more of the search criteria.

Search results are restricted by access role within Work Optimiser. Users with:

  • Service Officer access only receive information based on work assigned to them
  • Team Leader access receive information based on work assigned to them, and line managed staff
  • Analyst access receive information based on all work in the national pool

The Resources page contains a description of each of the search fields available to each role.

To refine search results, use Filters to enter more information. This will add and remove fields from the search.

Worklist search results + Read more ...

Results from the work item search show in the bottom half of the Worklist screen.

The following information is displayed in columns for each work item.

  • Work Item ID
  • Work Type
  • Work Item Status
  • Source System
  • Employee Responsible
  • Priority
  • Age in days

The Read column also allows staff to identify when work items were last viewed. Work items can display as:

  • Unread – the work item has not been viewed
  • Just Now – the page has not been reloaded since the work item was viewed
  • Today – the work item has been viewed today
  • Yesterday – the work item was viewed yesterday
  • [number of days] Days Ago – the number of days since the work item was last viewed.

Select Columns and Sort Options functions can be used to customise the information displayed in the search results.

Work items can be selected from the results for further action. To select:

  • A single work item, click on the select the table row containing the work item
  • Multiple work items, select the multi-select check box located on each row. To select all work items in the results list, select the checkbox in the header of the results table.

5

Worklist saved searches + Read more ...

To access the Saved Searches list, select the search view (for example, My Work) at top left of the Worklist. A list of saved searches will display.

These saved search views are standard for all Team Leaders and Analysts:

  • All Work: to search through all work items in the National Allocation Pool. For Analysts only
  • Customer Search to find work on a customer or organisation record. Enter the Customer reference Number (CRN), Child Support Reference Number (CSRN) or Medicare Number
  • My Team Work to view and manage work assigned to staff who report to the user
  • My Work to show all work assigned to the user

Staff can save personalised searches combining frequently used search criteria and preferred columns displayed in search results.

To save a customised search:

  • Use Select Filters to select the search criteria
  • Enter the required details into each search criteria
  • Use Select Columns to select the preferred column display
  • Select Go
  • When the results display, access the Saved Searches list
  • Select Save As
  • Enter a name for the saved search, and select the Set as default checkbox if needed
  • Select Confirm

The default view is My Work. This opens when launching the Worklist.

To change the default view to another view:

  • Select My Work, a list of Saved Searches displays
  • Select Manage in the Saved Searches window
  • Select the default search
  • Select Confirm

Staff Profile + Read more ...

This application is in the Work Management menu of Work Optimiser.

It is a snapshot of a staff member’s WLM profile. It can help troubleshoot why a staff member is not receiving work as expected.

Team Leaders can view the profile for staff who report to them.

Analysts can view the profile for all staff.

Enter a staff member’s logon ID in the search field under Staff Selection at top left to view:

  • Observable Org Units: if they are linked to an Org unit in ESSentials that is not their actual team. For example, a user may have access to run reports on behalf of another team
  • Capabilities: skill tags assigned to them in ESSentials, proficiency levels and the start and end dates for each
  • Temporary Work Groups: any groups the staff member is assigned to
  • Queues: they are a member of
  • Customer Relationships: Conflict of Interest, Restricted Access and Child Support Bundle on Allocation relationships between the staff member and customers

Simulate Allocation is intended to provide a list of potential work items staff may receive if they select Ready. There is no simulation available for Quick Ready.

This function is available for Analysts only. Refer to the information as a general guide only. Actual allocation of work can vary as a result of:

  • National workload management strategies
  • Relationship Profiles, and
  • Temporary Work Group allocation to other staff members

If Enable Email Notifications is selected, the staff member will receive an email when a work item is assigned to their inbox. This is useful for staff who:

  • Have a case management relationship with a customer, and
  • Do not undertake processing work daily

Conflict of interest - Centrelink and Child Support + Read more ...

If a staff member is allocated work for a customer they know personally, for example, family member, friend, personal acquaintance or neighbour:

  • They can unassign the work item using the reason Conflict of Interest
  • This creates a Conflict of Interest relationship between the staff member and the customer
  • No further work for this customer will be allocated to the staff member

Note: recording a conflict of interest for WLM does not record inadvertent access to a customer’s information.

View these relationships in the Staff Profile under Actor Relationships.

Team Leaders can add or remove them for staff they directly line manage, if appropriate.

To remove a conflict of interest relationship for a staff member:

  • Select Staff Profile in the Work Management menu
  • Enter the staff member's logon ID in the Staff Selection search
  • Select the staff member's name from the search results
  • The conflict of interest relationships display under Actor Relationships
  • Select X at the end of the row containing the relevant CRN
  • Select Confirm when the prompt displays

Unclassified Items Summary report

This lists work items not automatically classified into a work type. WLM cannot allocate unclassified items to staff for processing, as it cannot determine what the piece of work is. This report can help to identify and resolve issues with WLM business rules.

To view the names of these commands, hover the cursor over the icons at the top right of the screen:

  • Run executes the report and returns results. Note: at least 2 filters must be used for the results to display
  • Refresh the data and re-run with filters applied
  • Choose Columns of data to include in the results
  • Filter to refine the report results. Select at least 2 filters to run the report
  • Sort by various columns in ascending or descending order

Unclassified Items Search function

This function allows users to view the following information for a specific Work Item ID if known. This can help to determine issues with WLM business rules.

  • Source system
  • Attribute type
  • Attribute value
  • Last changed date

Daily Analytics Dashboard + Read more ...

This dashboard gives a real-time snapshot of workloads at a national level. It includes work in progress, work item movement, and staffing profiles.

Filter the report by selecting the option on the left:

  • Work Program
  • Work Type Process Category
  • Select Work Types

The reporting information is split into groups:

Work Item Summary

  • Created Today: total work items created on the current date
  • Completed Today: total work items completed on the current day
  • Actionable: total work items currently available for completion. This includes work allocated and waiting to be allocated. Held items are not included
  • On Hold: total work items with a current held status
  • Overdue: total work items past the due date set by the source system
  • Due within a week: total work items with a due date within the next week

Today’s Work item Movement

  • Unassigned: total times staff unassigned a piece of work on the current date
  • Capability Adjusted: total work items adjusted on the current day
  • Reassigned: total times staff reassigned a work item on the current day
  • Proficiency Raised: total work items on the current day
  • Recategorised: total work items recategorised to another work type, from the current filtered work type
  • Held to User: total work items held to any users Worklist on the current day
  • Activity Complete: total work items unassigned with the reason ‘Activity Complete’ on the current date

Skilled Staff Information

  • Skilled staff: number of staff who currently hold the base skill for the work type. If applying a filter, this is the number of staff who can receive at least 1 work item that matches the criteria
  • Staff Allocated Work: number of staff allocated at least 1 work item by selecting Ready or Quick Ready on the current day. If applying a filter, this is the number of staff who can receive at least 1 work item that matches the criteria
  • Average Completed per Staff Member: average number of work items completed by each staff member allocated work on the current date
  • Completed by Due Date: percentage of work completed on the current day, before the due date has passed. This excludes Work items with no due date

Staff using Temporary Work Groups

  • If not applying filters, this is the total number of staff who are part of at least 1 Temporary Work Group of any type
  • If applying filters, this is the total number of staff who are part of at least 1 Temporary Work Group allocating the selected work type/s
  • The count excludes Temporary Work Groups that use skill tags

Active Temporary Work Groups

  • If not applying filters, this is the total number of Temporary Work Groups that are currently active
  • If applying filters, this is the total number of Temporary Work Groups that allocate the selected work type/s
  • The count excludes Temporary Work Groups that use skill tags

Created vs Completed Work Graph

The graph displays for the current day:

  • In blue, an hourly breakdown of the number of work items created
  • In orange, the number of work items completed

Click on a point in the graph to see the actual number of work items at that time.

The counts change according to filter options. The display updates when the page first loads, when changing filters, or when selecting Clear or Refresh.

Temporary Work Groups + Read more ...

These enable workload managers to apply allocation rules to a user or group of users for a set time.

Temporary Work Groups can be created by Capabilities or Work Type.

A Work Type Group:

  • Will prioritise specific work over other work the user is skilled to receive. Staff will resume receiving work that matches their skill tags in ESSentials when:
    • There is no more work of that work type, or
    • The group volume limit for the work type is reached

A Capabilities based Group:

  • Temporarily adds skills to the staff member’s existing skill tags coded in ESSentials
  • WLM will allocate the highest priority work item matching any of the capabilities, not just those coded in the Temporary Work Group

If a work type has a configured daily limit, this will be overridden when added to a Temporary Work Group.

2

Viewing Temporary Work Groups + Read more ...

Select the Temporary Work Group application from Work Optimiser launchpad. The screen is split into 2 sections:

  • Temporary Group Selection: to search for current Temporary Work groups and see basic information. This includes the valid date range and number of staff in the group
  • Temporary Work Group Details: to view details or update configured details of a Temporary Work Group. Detail displays here after highlighting a group in the Temporary Group Selection pane.

View details of available work in the Available Work Item Summary at the bottom of the Temporary Work Group Details screen.

To view the history of changes to a Temporary Work Group:

  • Search and select the profile from the Temporary Work Group Selection
  • Select Show History at the bottom of the screen
  • The Temporary Work Group History window will display previous changes to:
    • Details history
    • Capabilities history
    • Work types history
    • Staff History

3

Create a Temporary Work Group + Read more ...

Select Add at the bottom of the Temporary Group Selection pane.

In the New Temporary Work Group box, enter these details:

  • Temporary Work Group Description: a name that describes the group and makes it easier to find
  • Valid Date/Time From: the date and time the group should start allocating work to the staff
  • Valid Date/Time To: the date and time the group should stop allocating work to the staff
  • Group Allocation Limit: the total number of work items that can be allocated to this group, the total amount of work items that can be allocated is 5,000,000
  • Select Confirm

Once created, go to Item 4.

Note: Valid Date/Time From and Valid Date/Time To will start and stop based on the time zone of where the creator is located.

4

Managing Temporary Work Groups + Read more ...

After creating the group, the following details must be configured for work to allocate correctly. The same process can be used to change the configuration of an existing Temporary Work Group.

Select the group from the Temporary Work Group Selection list.

Select Edit at bottom right of the Temporary Work Group Details window.

Note: Group Allocation limit applies to the work type, not to each user. To set a limit for each user, add a Temporary Work Group for each staff member.

  • Capabilities: add skills when the group is being created to process work for a particular skill tag
  • Work Types: add work types when creating the group to process a particular work type only
  • Staff: add staff to the group by searching for:
    • Logon or name of the staff member
    • Org unit to add all staff in the Org Unit, or
    • Multiple staff members and Org Units
  • Select Save

Bulk Configure allows users to enter multiple work types or staff to a Temporary Work Group at once. This feature is mainly for WLM Analysts. Up to 500 entries can be added. Entries must be comma separated, or copied from an Excel spreadsheet.

To delete a Temporary Work Group:

  • Locate the Temporary Work Group using the Temporary Work Group Selection search function
  • Select the Temporary Work Group from the search results
  • Select the Delete at the bottom of the Temporary Work Group Selection pane
  • Select Confirm

Work type configuration + Read more ...

Analysts have read only access to this application in the Maintenance menu of Work Optimiser.

Work type configuration is where the WLM business rules are applied to work types. Analysts can view:

  • Details for specific work types, for example, if they are set to Bundle on Allocation or are time zone sensitive
  • Capabilities and delegations needed to process the work type
  • Recategorisation options set by the business owner

Use the Work Type Selection panel to search for work types and see basic information as well as the work type description.

The Work Type Details screen displays the configured details for the work type highlighted in the Work Type Selection panel. It is grouped in headings

  • Details
  • Configuration
  • Delegations
  • Capabilities
  • Work Item Attributes
  • Actor Attributes
  • Recategorisation Options
