Action a notice of judgment or settlement

Table 1: this table follows on from an enquiry case.

Step

Action

1

Evaluate - select document type + Read more ...

  • Select Notice of judgment or settlement as the document type from the Please choose the document type dropdown menu
  • Select the relevant toggle icons
  • Select continue

Note: if the claim requires registration or updating the claim will flow to that process before the Notice of judgment or settlement can be processed. See Search for or update an existing claim or register a new claim.

2

Assign work item + Read more ...

  • A message displays stating Are you sure you want to process a Notice of judgment or settlement? Would you like to continue?
  • Select Yes

3

Capture NOJS details + Read more ...

  • Enter the Date document received
  • If applicable enter the Date the notifiable person advised the injured person of potential debt liability (S22)
  • Select the relevant option from the Was compensation fixed by dropdown list
  • Enter the Date of judgment or settlement
  • If applicable select the Is there a date the NP is required to pay the compensation toggle icon and enter the date into the field generated
  • Enter the Total amount of compensation
  • If applicable select the:
    • Was the compensation based on apportioned liability - toggle icon and enter apportionment percentage into the field generated
    • Does the amount of compensation fixed redeem liability for periodic payments - toggle icon
    • Were past expenses fixed under judgement - toggle icon
    • Do past expenses awarded represent the Medicare recovery - amount toggle icon
    • Enter Amount of past medical expenses awarded and/or Amount of past care expenses awarded - one must be entered if either or both past expenses toggle icon is selected
    • Is any amount payable to Centrelink - toggle icon and enter the amount into the field generated
    • Has the notifiable person indicated Advance payment intent - toggle icon and enter the amount into the field generated
    • Has the notifiable person advised the injured person in writing of AP - intent toggle icon
    • Does the notifiable person authorise payment of any excess amounts from failed advance payment to the injured person - toggle icon
  • Select Continue

Note: based on toggle icons selected additional fields may appear about the individual toggle icons.

4

Assess NOJS details - S23A statement + Read more ...

MCRS will advise if there is a valid Notice of past benefits or if a Notice to claimant is required. If a Notice to claimant is required the workflow goes to the Capture NTC details screen after processing the Notice of judgment or settlement.

If a S23A statement been received:

  • select the Has a S23A Statement been provided - toggle icon
  • if S23A is complete select the Is S23A complete - toggle icon
  • if accepting the S23A select the Do you accept the S23A - toggle icon

Note: if the S23A is received but is incomplete or not accepted and no valid Notice of past benefits a Notice to claimant is issued.

5

Assess NOJS details - Payment details + Read more ...

Merlin payment details display will present if funds have been received and allocated to the WIN in SAP.

Check for payments in Systems, Applications and Products (SAP).

If money has been received that relate to the claim, even if not yet allocated to the WIN:

  • Select the Has any amount been received for this claim in SAP toggle icon
  • If payment meets the AP criteria select the Do payment, document and claim details meet AP criteria toggle icon
    • Enter Advance payment amount
    • Enter Advance payment SAP document # (receipt #)
    • Enter the Date advance payment received
    • Enter the Compensation payer SAP unique account number
  • If payment is not an intended AP or does not meet the AP criteria leave the Do payment, document and claim details meet AP criteria - toggle icon
    • Enter Ad-hoc payment amount
    • Enter Ad-hoc payment SAP document # (receipt #)
    • Enter the Ad-hoc payment date
    • Enter the Compensation payer SAP unique account number
  • Select Continue

Note: depending on the toggle icon selected fields will display as ad-hoc or advance payment.

If no money has been received:

  • Enter the Compensation payer SAP unique account number
  • Select Continue

6

Review NOJS details + Read more ...

  • Review the data entered
  • Select Edit to change any details if required
  • Select Reprocess SBC if available and required - SBC Capture displays. Procedure ends here
    Note: use this function if an amendment pre or post claim finalisation requires processing
  • Select Submit

The next workflow will change depending on the fields entered in previous steps. A message displays advising what action is required, for a:

  • Issue a Notice to claimant, see Notice of past benefits request and notice to claimant. Procedure ends here
  • Issue a Notice of charge (where a valid Notice of past benefits exists, claim settled by judgment and Medicare recovery set by past expenses awarded, or a S23A is accepted), go to Step 7
  • Issue Z2810 notice where an advance payment has failed, go to Step 8
  • Issue Z2809 notice where a Notice of judgement or settlement has failed, go to Step 9
  • Issue a Z2812 notice if the claim is a 33K, go to Step 10
  • Issue a Z2812 notice if the claim is a 24(4), go to Step 11

7

Valid Notice of past benefits exists, claim settled by judgment and Medicare recovery set by past expenses awarded, or a S23A is accepted + Read more ...

If a valid NOPB exists:

  • Message displays
  • Select Yes to proceed to NOC summary, go to Step 12

If claim has settled via judgment and Medicare recovery set by past expenses awarded:

  • Message displays Case settled by judgment and fixed costs awarded. Close claim on fixed costs NOC
  • Select Yes to proceed to NOC summary, go to Step 12
    • If the claim is an AP and the Merlin file does not match the advance payment amount the claim needs action by Compensation Finance. Go to Step 1 in Table 2

If a S23A is accepted:

  • Message displays NOPB not issued and S23A accepted, thus issue the NOC
  • Select Yes to proceed to NOC summary, go to Step 12
    • If the claim is an AP and the Merlin file does not match the advance payment amount the claim needs action by Compensation Finance. Go to Step 1 in Table 2

8

Failure of advance payment + Read more ...

  • Message displays [Issue 2810] This claim is a failed AP. Are you sure you want to request a Notice to Claimant?
  • Select Yes

Claims with:

  • a valid NOPB go to the NOC summary, go to Step 12
  • no valid NOPB go to the see Notice of past benefits request and notice to claimant. Procedure ends here

Note: Z2810 notice is sent to the notifiable person to notify them of the failed AP with a copy sent with the Z2803 letter to the claimant.

9

Non-compliance of Notice of judgment or settlement – 29(3) + Read more ...

  • Message displays [Issue 2810] This claim is a failed AP. [Issue Z2809] Notifiable person has failed to comply. S29(3) applies. Are you sure you want to request a Notice to claimant?
  • Select Yes

Claims with:

  • a valid NOPB go to the NOC summary, go to Step 12
  • no valid NOPB go to the Notice to claimant work step, see Notice of past benefits request and notice to claimant. Procedure ends here

Note: Z2809 notice is sent to the notifiable person to notify them that S29(3) applies with a copy sent with the Z2803 letter to the claimant.

10

Claim is S33K + Read more ...

  • Message displays S33K applies, thus issue NOC
  • Select Yes
  • If the Merlin file does not match the advance payment amount the claim needs action by Compensation Finance. See Step 1 in Table 2
    Note: Z2812 notice is sent to the claimant
  • Go to Step 12

11

Claim is S24(4) + Read more ...

  • Message displays S24(4) applies, thus issue NOC
  • Select Yes

Note: Z2812 notice is sent to the claimant.

12

Notice of charge summary + Read more ...

A table displays with the Notice of charge summary.

Review the summary and totals displayed:

  • If the totals are incorrect select Cancel
  • If the totals are correct select Submit
  • If the amount payable/refundable does not exceed the threshold go to Step 13
  • If the amount payable/refundable exceeds the threshold, the Review and approve ‘To do’ screen will appear, do not select Go. Select Home icon from the left hand side menu

13

Issue correspondence + Read more ...

The correspondence to be issued will vary depending on information previously entered.

  • Select the Notification type from the dropdown list
  • Select the relevant letter recipient
  • Select the Preview hyperlink to view the letter
  • Add a case note to detail action taken
  • Select Submit

Action AP reconciliation failure

For Compensation Finance staff only

Table 2

Step

Action

1

Merlin file does not match the advance payment + Read more ...

When the Merlin file does not match the advance payment details the claim is marked as complex and action by Compensation Finance is needed.

Select the relevant radio button and select OK:

  • Request for override - advance payment is valid but payment in SAP is split, go to Step 2
  • Move NOPB funds back - previous NOPB and advance payment paid. NOPB payment moved to Medicare card to allow refund to be processed, go to Step 3
  • Make correction in SAP - advance payment found in SAP and not yet allocated to the claim, go to Step 4

2

Request for override + Read more ...

  • An email template within MCRS displays
  • Complete the required fields
  • Select Send
  • Add a case note to detail action taken
  • Select Wait for Comp Finance to place the claim on hold for 24 hours. Procedure ends here

3

Move NOPB funds back + Read more ...

  • Message displays Submit request to Move NOPB funds back using SharePoint link
  • Select the hyperlink
  • Complete the required SharePoint form
  • Add a case note to detail action taken
  • Select Wait for Comp Finance to place the claim on hold for 24 hours. Procedure ends here

4

Make correction in SAP + Read more ...

  • Message displays Submit request to Make correction in SAP using SharePoint link
  • Select the hyperlink
  • Complete the required SharePoint form
  • Add a case note to detail action taken
  • Select Wait for Comp Finance to place the claim on hold for 24 hours. Procedure ends here
