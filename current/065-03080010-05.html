This page contains a table of cancellation, suspension and rejection reasons for Age Pension. It also contains a legend to assist in interpreting the table of codes.

Legend

Legend for Age Pension cancellation, suspension and rejection codes

Table 1: This table provides information to assist in interpreting cancellation, suspension and rejection codes for Age Pension.

Item

Description

1

Suspension (SUS) + Read more ...

  • Suspensions can always be restored

2

Cancellation (CAN) + Read more ...

  • Cancelled records can sometimes be manually or automatically restored
  • Before cancelling a customer's payment, care should be taken to establish that the customer has lost entitlement for that payment

3

Rejection (REJ) + Read more ...

  • A new claim activity must be completed to get the customer on payment if the decision to reject was incorrect
  • Customers may withdraw their claims voluntarily or may have lodged an inappropriate or incorrect claim. In all these circumstances, claims are not rejected, they are taken to have never been lodged. See Withdrawal of claims
  • Rejection can never be restored

4

Initiation + Read more ...

  • Manual (M) - Manual reason codes that can be manually input via the Benefit Action (BA) screen
  • Auto (A) - cancellation or suspension reason codes that automatically occur on the system

5

Restoration + Read more ...

  • No – cancelled record cannot be restored
  • M - can be manually restored. Some restorations may require a circumstance screen to be updated before the restoration can be input on the Benefit Action (BA) screen (e.g. updating assets or employment income)
