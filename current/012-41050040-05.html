Error codes

General consultation item error codes

This table describes general consultation scenarios and associated system responses and codes for resolution.

Situation

System response/Resolution code

A general consultation time-based item is claimed prior to the item's start date.
  • error message 2-ITEM COV is returned
  • use reason code 206 to resolve

The practice location from where the hosting practitioner and patient initiate the video consultation must be located in an eligible outer metropolitan area to claim a Telehealth benefit.
  • error message 4-ELG NOT is returned
  • use reason code 639 to resolve message

In all of the scenarios above, where a general practitioner is hosting a video consultation and an additional bulk bill item is being claimed.
  • use reason code 500 to resolve

Reason codes

Reason codes for processing claims

This table describes reason codes applicable for processing claims containing Telehealth Medicare Benefits Schedule (MBS) items.

Reason code

Description

223

Service not payable - specified item not claimed or present

638

Derived fee item and other item cannot be claimed in hospital

639

Provider not in an eligible area to claim this item

640

More than one base and derived item claimed

641

More than one base item claimed

642

Benefit paid for derived and other item claimed

643

Derived item assessed with other item on statement

500

Rejected in association with another item in this claim

Return messages

Return messages for bulk bill and simplified billing

This table describes return messages applicable to Telehealth patient, bulk bill and simplified billing claims.

Return message

Description

Claiming channel

Resolution

1-HOSP DRV

Derived item claimed in-hospital.
  • manual patient claims
  • pended patient claims
  • bulk bill claims
  • simplified billing claims
  • Check the keyed data and correct any errors
  • If keying is correct, reject the highlighted item/s with RSN 638
    		•

    3-MULT ITM

    More than one base specialist consultation item and derived item pair exists in the same group/sub-group.
    • manual patient claims
    • pended patient claims
    • bulk bill claims
  • Check the keyed data and correct any errors
  • If all the keyed data is correct, reject all derived fee and consultation items with RSN 640
  • If additional incentive item/s are claimed, reject the incentive item/s with RSN 500
    		•

    3-NO BASE

    Derived item is claimed without a base specialist consultation item.
    • manual patient claims
    • pended patient claims
    • bulk bill claims
  • Check the keyed data and correct any errors
  • If all the keyed data is correct, reject the item service line with RSN 223
  • If additional incentive item/s are claimed, reject the item service line with RSN 500
    		•

    4-ELG NOT

    The provider performing the Telehealth service servicing location is an ineligible location (inner metropolitan).
    • manual patient claims
    • pended patient claims
    • bulk bill claims
  • Check the keyed data and correct any errors
  • If all the keyed data is correct, reject the item service line with RSN 639
  • If all additional incentive items/s are claimed, reject the item service line with RSN 500
    		•

    6-MULT BSE

    The derived item is claimed with more than one base specialist consultation item (in the same claim).
    • manual patient claims
    • pended patient claims
    • bulk bill claims
  • Check if one derived fee and consultation item pair is on a separate account/voucher and process the account separately
  • If all items are on one account/voucher, reject all item service lines with RSN 641
  • If additional incentive item/s are claimed, reject the incentive item/s with RSN 500
    		•

    8-DRV ITM

    The derived item is paid with a zero dollar benefit or the system generated RSN codes where the base and derived fee item has been processed. Note: warning message 8-DRV ITM will display

    Papers 4, 5 and 7.
    • manual patient claims
    • pended patient claims

    System will automatically generate reason codes - the base item with RSN 643 and the derived fee item with RSN 642
