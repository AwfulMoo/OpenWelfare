This page contains how Service Officers investigate flagged identifiers and the actions required by Tier 3 staff to manage flagged identifiers.

How Service Officers action identifiers listed on the HW080 form

Table 1

Step

Action

1

Check user's account status + Read more ...

Is the user's account in Known status?

  • Yes, go to Step 2
  • No, identifiers can only be processed if the status is Known. Make sure to finalise the identity verification process for their PRODA account, before adding the identifiers

2

Account status = Known + Read more ...

  • Select Manage services
  • Select Healthcare providers and administrators tile under Available services
  • Select the Type of identifier from the dropdown list under Add identifiers
  • Enter the identifier number into the Identifier field
  • Select Search

Repeat for all identifiers provided on the HW080 form.

Did all identifiers link?

  • Yes, see the Verify identity verification table, Step 4 in Processing the Manual identity verification for Provider Digital Access (HW080) form
  • No, go to Step 3

3

Check Manage names + Read more ...

Check if the Medicare record in a name that exists in PRODA Manage names. Note: If the Medicare record is not in a name linked to their PRODA account the user will need to provide a change of name.

Is a change of name required?

  • Yes, see email templates on the Resources page of Processing the Manual identity verification for Provider Digital Access (HW080) form. Send email 'Change of name required to enable linking to Medicare records'
  • No, go to Step 4

4

Escalate to Tier 2 + Read more ...

Escalate to Tier 2 with the following information:

  • flagged identifier
  • PRODA account username and/or Registration Authority (RA) number
  • name of user
  • type of identifier (see Step 5 in the How Service Officers investigate and action flagged identifiers table for a list)
  • identifier number

If the Service Officer has conducted an investigation in the source system and has identified why the identifier is not linking, give a description.

See Help for staff Provider Digital Access (PRODA).

Procedure ends here.

Check if the Medicare record is in a name that exists in PRODA Manage names. Note: If the Medicare record is not in a name linked to their PRODA account the user will need to provide a change of name.

Escalate to Tier 2 + Read more ...

Escalate to Tier 2 with the following information:

See Help for staff Provider Digital Access (PRODA).

If Tier 2 is unable to resolve the issue with linking the user's identifiers, escalate to Tier 3 to conduct a further investigation. Add comments to the users PRODA account 'Unable to identify why identifier is unable to link.'

5

Tier 3 - Conduct investigation + Read more ...

Access to the source system is required to conduct an investigation. Where access is not available, view access may be requested by following the normal process and requesting access through the supervisor.

The identifiers source system for users accessing the department's services are:

  • eHealth Program (EHP)
  • Provider Directory System (PDS)
  • Australian Immunisation Register (AIR)

The following identifiers will need to be investigated in the relevant system:

  • EHP
    • HPI-I Number
    • HPI-O Number
    • RO/OMO Number
    • Ahpra Registration Number
  • PDS
    • Medicare Provider Number
    • Medicare Stem
    • DVA Provider Number
    • DVA Stem
    • PBS Approved Prescriber
    • HECSRS Identifier
    • Medication Review - AACP Accreditation
    • Medication Review - SHPA Accreditation
    • Midwife Unique Identifier
    • Pharmacist - State Registrations
    • Pharmacist - National Registration
    • Ahpra Registration Number
  • AIR
    • AIR Registration

6

Tier 3 - Action flagged identifiers + Read more ...

All flagged identifiers need to be actioned:

Reject identifiers that:

  • do not belong to the user's account
    • include comment - 'identifier does not belong to the user'
    • send email 'The identifier you are trying to link does not belong to you'
    • procedure ends here
  • have been input incorrectly by the user
    • advise the user to try entering the identifier again
    • include comment - 'identifier input incorrectly'

Is change of name required?

  • Yes, send email 'Change of name required to enable linking to Medicare records'. Procedure ends here
  • No, go to Step 8

7

Change of name required + Read more ...

  • Advise the user they can add a change of name:
    • log on to their PRODA account
    • select Profile from the header
    • select Update next to Name in My details section
    • select Add a new name link
    • follow the online process which will include providing identify verification
  • Once they have completed the change of name they can link the identifier to their account:
    • select Services from the header
    • select Link identifiers on the Health Professional Online Services tile
    • follow the online instructions

  • Advise the user if they are unable to do the change of name online, they will need to supply the Manual identity verification for Provider Digital Access (HW080) form and supply the relevant supporting documentation

8

Tier 3 - Approve investigated identifiers + Read more ...

Include the reason for approving the identifier in the comments field and select the Approve button.

If the source system (EHP or PDS) has minor differences in spelling the user's name or data missing from the user's record:

  • Force the approval by selecting Approve and entering the relevant comments.
  • If the source system continues to reject the approved override, Tier 3 will need to escalate to the source systems for their investigation

Once approved

The identifier is linked to the PRODA record and the relevant access is granted to the user.

Action
