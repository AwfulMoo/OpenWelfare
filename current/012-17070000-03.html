Online claiming options and enquiries

Table 1: This table contains information about online claiming options, online veteran identification and escalating technical and registration enquiries.

Item

Description

1

Online claim options + Read more ...

The Claim Type: field on the VQ screens is used to identify how a claim has been transmitted:

  • H - Medical Online
  • A - Allied Health Online
  • O - Optical Online
  • D - Dental Online
  • N - Community Nursing Online
  • I - ECLIPSE In Hospital Claims (IHC)
  • C - Classic online (this claim channel closed on 1 January 2017)
  • W - DVA Webclaim
  • blank - there is no indicator for Manual or EDI

Encourage medical providers to use the latest DVA electronic claiming channels:

  • providers do not need to submit supporting documents
  • lodging an error free claim and receiving payment can be finalised in 72 hours
  • access to Online Veteran Verification (OVV) and Enterprise Veteran Verification (EVV) enables providers to check if a veteran is known to DVA and Medicare
  • remittance advice is immediately available through practice management software, for up to 6 months from the date a service was paid

2

Online veteran verification + Read more ...

Online Veteran Verification (OVV) and Enterprise Veteran Verification (EVV):

  • enables a practice to check if a single veteran (OVV request), up to 1,000 veterans (EVV request) or other eligible person is known to DVA before lodging a claim
  • does not check a patient's eligibility or accepted disabilities - a positive match may still result in the claim being rejected for other reasons
  • can be used to check the veteran's card type

If a provider advises OVV or EVV has not identified the veteran, and searches fail to locate the file number for the veteran details supplied, the claim may be rejected. The provider or Service Officer should contact DVA to confirm the correct veteran details before resubmitting a claim.

3

Escalating technical and registration enquiries + Read more ...

Transfer enquiries from providers and practice managers about:

  • technical issues to the eBusiness Service Centre
  • registration for Medicare online accounts to eBusiness in the appropriate state

For contact details, see ebusiness Service Centre.

Transfer direct enquiries from software vendors about the DVA Logic Pack to the Medicare software vendor helpdesk. For contact details, see Online Support.

Note: eBusiness service centres and software vendor helpdesk can escalate issues to the VAP Helpdesk. For contact details, see Veterans' Affairs Processing (VAP).

VAP Claim ID numbers –web services claiming platform

Table 5: This table describes how to assist providers identify claim IDs.

Step

Action

1

Change in Claim ID generation for online claims + Read more ...

Medical practices can use a web services claiming platform. This is a change to how Claim IDs generate in the Practice Management Software (PMS).

There are still practices using the Client Adaptor (PKI) claiming platform, where the software generates a unique Claim ID.

With the web services platform, Services Australia can issue a Claim ID which is unique to that payee provider on that day. The web services Claim IDs generates in the following format:

Medicare Bulk Bill

1 Prefix:

  • Prefix A-Z indicates out of hospital services (for example, A0001@)
  • Prefix # indicates in-hospital services (for example, #0001@)

VAP

1 Prefix:

  • Prefix A-Z indicates out of hospital and in-hospital non-pathology services (for example, A0001@)
  • Prefix # indicates in-hospital Pathology services (for example, #0001@)

Individual payee providers will get the same Claim ID if multiple payee providers in the same practice submit claims:

  • from the same location or practices
  • with the same lodgement date

For example:

  • Provider A's first Claim ID will be A0001@ followed by A0002@
  • Provider B's first Claim ID will be A0001@ followed by A0002@

The Claim ID will reset/restart the next day at A0001@ or #0001@ for each individual payee provider submitting claims using web services.

2

Identifying the claim ID + Read more ...

When Service Officers receive a call from medical practices:

  • where the PMS has created a claim ID, and
  • it does not match the claim ID recorded on the patient’s history or under the payee provider number:
    • instruct the caller to provide the Claim ID returned from Medicare or VAP

If the caller cannot identify any other Claim ID in the PMS they must contact their software developer. The caller must ask the developer how to identify the claim ID generated by Services Australia in order for the claim to be investigated.

Medical practices can request payment/processing reports using just the payee provider.

If the medical practice is still unable to retrieve their processing/payment reports staff must refer them back to their software developer.

3

Further information + Read more ...

Staff must refer any questions or concerns to the Programme Support Officer (PSO).

Keying service details + Read more ...

Mixed services + Read more ...

If a voucher contains a mixture of services (e.g. consultation and pathology items), the services must be keyed on the data entry screen appropriate for its type.

Key all details relating to 1 type of service on the appropriate screen and press [Enter].

To key details of the other service type, press the appropriate [PF] key.

Note: all errors highlighted by the system must be resolved before swapping to another data entry screen. The system responds by displaying the data entry screen required, with the service line last keyed displayed at the top of the screen.

3

Voucher count + Read more ...

If the voucher contains more than 1 service for a patient, the FILE #, NAME, DOS, REF PR and REF DT fields are only keyed once for each voucher.

If the FILE # and NAME fields are keyed for each service, the system will count each service line as a separate voucher which will affect the Voucher Count which displays at the top of the screen in the VC field.

Exceptions:

On the Pathology Data Entry (VKPC) screen, the date of service must be keyed for every service line. The DUP key [shift-insert] can be used to duplicate the date if it is the same.

On the Specialist Data Entry (VKSC) screen, the date of service must be repeated for each new service type. When 1 item requires a Specialist Referral (SR) and another item requires a Diagnostic Imaging Request (DI), a new voucher needs to be created for each service by keying the date of service again.

More than 16 lines required

An individual data entry screen contains 16 keying lines. When a claim requires more than 16 keying lines, press [Enter] when the last service line on the screen has been keyed. The system responds by providing additional keying lines with the service line last processed shown at the top of the screen.

4

Vouchers with no or incorrect provider number + Read more ...

Further assistance can be provided by a VAP Programme Support Officer (PSO). For contact details see Veterans' Affairs Processing (VAP).

If provider's name and address is on the voucher:

Using VPQI, search for an address that matches the location on the claim.

If the provider’s contact details are on the voucher, call the provider to obtain the provider number needed to process the claim.

Is a match found?

  • Yes, use the location provider number to process the claim
  • No, send the 'Veterans' Affairs Treatment account could not be processed' letter to the provider requesting confirmation of the provider number to be used when processing the claim. See Resources page for a link to the letter

If provider contact details or a provider number cannot be located, dispose of the voucher as secure waste, or file as Complete in PaNDA with no further action required.

5

Closing the data entry screen + Read more ...

When service details have been entered and there are no errors in the claim, press the [PF3-CLOSE] key to close the data entry screen.

The system will not allow a claim to be closed if an error exists.

Depending on which screen was used to access the data entry screen, the Combined Claim Registration and Data Entry Menu (VRDC) or Data Entry Menu (VDEI) screens is displayed.

Not closed VCH count

When the number of vouchers quoted by the provider does not match the number of vouchers counted by the system, the message NOT CLOSED VCH COUNT displays. To resolve this, manually close the claim - see Item 6.

6

Closing the claim - manually or automatic + Read more ...

Allied health professionals and specialist claims often have to be manually closed as a single voucher/account can contain a number of different dates of service and the system counts each date as a separate voucher.

Manually closing a claim

To manually close the claim from the Combined Claim Registration and Data Entry Menu (VRDC) or Data Entry Menu (VDEI) screens, key 'C' (close) in the OPTION field and press [Enter]. The system responds with the message CLM CLOSED.

Details of the Service Officer who processed the claim are displayed in the OPER REF field. The claim is now ready for background assessing.

Automatic closing of a claim

When the number of vouchers quoted by the provider matches the number of vouchers counted by the system, the system can automatically close the claim and displays the message CLM CLOSED.

Details of the Service Officer who processed the claim are displayed in the OPER REF field. The claim is now ready for background assessing.

7

Re-opening a claim + Read more ...

If it is necessary to add or delete service lines or vouchers after a data entry screen is closed, a facility is available on the Data Entry Menu (VDEI) screen to re-open the claim.

Note: this facility is only available for claims that have not yet been background assessed. In this case the message 3 - STATUS is displayed if an attempt is made to reopen the claim. The pend Service Officer will need to insert or delete the service lines or vouchers.

To re-open a claim after data entry screen is closed:

  • Access VDEI screen
  • Key details in the relevant fields to identify the claim
  • Key 'R' in the OPTION field and press [Enter]

The message CLM OPENED displays

It is now possible to key 'K' (key service details) or 'L' (list service details) in the OPTION field depending on the type of processing action required

8

End day procedures + Read more ...

On completion of daily manual claims processing Service Officers are required to check for any unpaid claims.

To display:

  • individual Service Officer claims, key 'VQOF' and press [Enter]
  • all outstanding claims for a source office, code key 'VQRF',(source office code), total ,age and press [Enter]

When there are errors or the claim requires further information, the system will display WAITPEND or IN PEND to indicate the claim is in pend until further action, key 'S' to select in the A (Action) field and press [Enter]

When there are errors or the claim requires further information, the system will automatically open the claim to the voucher that requires assessing:

  • Press [PF1] under return messages for instructions and key information as required
    • Review and apply assessing restrictions by following warning messages
    • Warning messages must be actioned in order, from left to right, so further explanations can be viewed. Place the cursor at the beginning of the return message and press [PF1]
  • Press [PF3] to move to the next error until all vouchers have been resolved
  • When all errors are resolved Mainframe will display NO ERRORS OK TO PAY directly below the control line
    • Overkey VAPC with 'VPPI' and press [Enter]
    • Overkey I with 'P' to pay the claim and press [Enter]
  • The confirmation screen will be displayed with the VQRI line and the message PAID OK to indicate successful payment of claim

If kilometres are missed during keying at the data entry stage, they can be added by inserting a line in the VAPC screen. See Pend resolution tab.

2

4-PRAC CLS - practice has closed + Read more ...

When the claim date is after the practice is closed, the message 4-PRAC CLS is returned on the Combined Claim Registration and Data Entry Menu (VRDC) screen.

If any of the service dates are before the date the practice closed, register a claim using transaction VHCI.

Note: if the service dates are after the date the practice closed, return the paperwork to the provider with a Practice Closed letter. See Resources page for a link to the letter.

  • Key 'VHCI' and press [Enter] to go to the VHCC screen
  • Key the details for the following fields:
    • DOL
    • SERV PROV
    • CLM ID
    • VQ
    • CLM DT (must be on or before the date the practice closed)
    • RSN (must key ‘DTE’ in the RSN field
    • PAYEE PROV (if required)
  • Press [Enter] and 'U' to update and claim header is created
  • Key 'VQRF, [provider number], [claim ID]'
  • Press [Enter] and claim header details are displayed
  • Select the appropriate data entry screen by keying CLAIM TYPE 'G', 'P', 'R', 'S', 'N' or 'H'
  • Press [Enter]

The Data Entry screen is displayed. Continue processing as normal.

3

SWAPSCRN + Read more ...

When service details have been entered on an inappropriate data entry screen, e.g. specialist consultation has been keyed on the allied health data entry screen, the system displays the message SWAP SCRN and highlights the service lines.

  • If all services have been keyed on the incorrect data entry screen:
    • Press the appropriate [PF] key to obtain the correct data entry screen
    • Press [Enter] and conform to the keying requirements for that screen
  • If some services have been keyed on the incorrect data entry screen:
    • Remove them from the screen by dashing them out. It may be necessary to key details in the FILE #, NAME, DOS, REF PR and REF DT fields depending on the line being removed
    • Press [Enter] and correct any errors before pressing the appropriate [PF] key to obtain the data entry screen required to key details of the line that was removed

4

3 - SVC LIM + Read more ...

When service details have been entered and the system identifies there is more than the allowable number of services per voucher, the system displays the message 3 - SVC LIM and highlights the excess services.

The maximum numbers of services that can be keyed per voucher are:

  • 5 services per voucher on the General Data Entry (VKGC) and Specialist Data Entry (VKSC)
  • 14 services per voucher on the Pathology Data Entry (VKPC), Allied Health Data Entry (VKRC) and Time-Based Anaesthetic Data Entry (VKAC) screens

Check to ensure the services relate to the relevant patient, then take the following actions:

  • When the services are for the same patient, key the date of service against the first highlighted service and press [Enter]. It may be necessary to key details in the REF PR and REF TO fields
  • When the services are for a different patient, key details in the FILE #, NAME and DOS fields against the appropriate services and press [Enter]. It may be necessary to key details in the REF PR and REF TO fields

5

3 - STATUS + Read more ...

When an attempt is made to delete a claim containing service details, the message 3 - STATUS displays.

Check the control line details have been keyed correctly.

If the control line has been correctly keyed, all service lines will need to be deleted before the deleting the claim.

Control line modifiers
