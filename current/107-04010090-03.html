For Debt Staff only.

Debts where multiple CRN exist

Step

Action

1

Is the debt on a duplicate record? + Read more ...

  • Yes, check the Duplicate CRN List (DCL) screen for the environments a record is in. For further information, see Inter-environment transfer of debts. Procedure ends here
  • No, go to Step 2

2

Has Data Quality Unit (DQU) taken action + Read more ...

A warning will display when a multiple CRN (MLC) is linked to a customer's primary record. The Multiple and Intertwined Warning (MIW) screen will display with the following message on access:

  • 'This record has been assessed as a multiple and cannot be used for the customer. The customer's primary record is 123 456 789X. Do you wish to proceed? 'N'
  • Service Officers can enter 'Y' to access the record in 'read only' mode. Contact the Data Quality Unit (DQU) to make updates.

Check if DQU (office code NA8) has assessed the case via the Document List (DL) screen. A Display on Access (DOA) DOC will have region code NA8.

Has DQU linked and updated the records?

  • Yes, Display on Access DOCs (DOA) on the records indicate which record is the primary CRN. Go to Step 4
  • No, go to Step 3

3

Refer possible multiple or intertwined CRNs to DQU + Read more ...

Do not make any updates to the records. DQU will need to determine which record is the primary CRN.

Multiple CRN:

Refer Multiple CRNs to DQU using the Multiple CRN Referral Fast Note.

To assist DQU to make a primary determination, note any additional details in the 'Other relevant information:' section of the Fast Note.

Intertwined CRN:

Refer Intertwined CRNs to the DQU using the Intertwined Referral form.

To assist DQU with their investigation, note any additional details on the form.

Process ends here.

4

Check the Debt List (OPDL) screen + Read more ...

Does the customer have debts for:

  • family assistance Reconciliation (FRC)
  • ATO Reconciliation (FRA), or
  • FA Re-reconciliation (FRR), or
  • family assistance non-lodger (NLD) debts
  • Yes, go to Step 5
  • No, go to Step 6

5

Family assistance reconciliation and non-lodger debts + Read more ...

The system will raise Family assistance reconciliation and non-lodger debts. They cannot be manually adjusted on the Debt Management Information System (DMIS).

In these cases a DOA DOC will display on the multiple records stating:

  • the primary record, and
  • details of debts on the secondary records

Create an ACTDOC with a keyword of 'ACTDRU'.

The ACTDOC must be sent to NS7.

Record details in a DOC and include the original Debt ID. See Online Document Recording (ODR) for further details.

Procedure ends here.

6

Not fully recovered debt where multiple CRNs exist + Read more ...

Has a non-recovered debt been raised where a customer has multiple CRNs?

  • Yes, go to Step 7
  • No, procedure ends here

7

Is the Debt on a multiple or intertwined CRN? + Read more ...

  • Yes, go to Step 8
  • No, go to Step 9

8

Recreating a manual debt on the correct CRN + Read more ...

If the incorrect CRN is locked for action, to have the record unlocked:

  • send an email request to Data Quality Unit, or
  • if your request is urgent, contact the DQU Helpdesk and select Option 1

On the incorrect CRN:

  • Go to the Debt Amount (OPDA) screen
  • Reduce the debt to zero by finalising 'no debt'

Manually re-raise the debt on the correct CRN by creating a new debt record manually on the Debt Management and Information System (DMIS).

  • Do not send the auto letter
  • On the Key Dates (OPKD) screen
    • record the date of issue of the original advice, and
    • record the original Debt ID in the Comments: field

Any recovery arrangement (other than withholdings) need to end before manually raising a debt.

9

Repayments and recovery arrangements + Read more ...

Have repayments been made or are there recovery arrangements in place on the incorrect record?

  • Yes, go to Step 10
  • No, record all details in a DOC. Include details of the original Debt ID. See Online Document Recording (ODR) for further details. Procedure ends here

10

Refer the debt to the Debt Management Team + Read more ...

  • See Debt Management for further information
  • Record all details in a DOC including details of the original Debt ID. See Online Document Recording (ODR) for further details
