This page contains information about how to record and finalise a JCA referral.

Staff trained in Process Direct select the Process Direct tab, staff not yet trained select Customer First.

Recording referrals

Table 1

Step

Action

1

Assess type of referral + Read more ...

If the JCA referral is:

  • for a Disability Support Pension (DSP) new claim, go to Step 2
  • part of a DSP Service Update, see the Recording JCA referrals table on the Customer First tab
  • part of a formal review or an Administrative Appeals Tribunal (AAT) review, see Reviews and appeals for DSP rejection or cancellation decisions. Procedure ends here
  • for a Centrelink International Services (CIS) customer (new claim processing, portability assessments and Foreign Pension assessments), see Foreign pension medical assessments. Procedure ends here
  • for ‘no future work capacity’ portability assessments, see Assessment for Disability Support Pension (DSP) customers going overseas under the no future work capacity provisions. Note: this is a CIS only referral. Procedure ends here
  • for a possible new claim or request for an explanation, check Document List (DL)/Notes for a DOC advising an explanation is pending. Referral for a JCA related to an explanation will be actioned by a Subject Matter Expert (SME)
  • for Supported Wage System (SWS) assessment requests received from Department of Jobs and Small Business, see Supported Wage System (SWS) medical assessment process. Procedure ends here

2

Check the MAT recommendation has been completed + Read more ...

Check scanned claim Documents in Process Direct for the DSP Medical Assessment Recommendation (SA479) and/or a DOC with a subject line of 'MAT DSP recommendation'.

Confirm the recommendation supports completion of a JCA referral to determine medical eligibility. MAT must submit a recommendation (SA479) to be available in Documents. See Streaming a new claim for Disability Support Pension (DSP).

Has a MAT been completed recommending a JCA referral?

  • Yes, go to Step 3
  • No, see Streaming a new claim for Disability Support Pension (DSP)

3

Confirm non-medical eligibility has been assessed + Read more ...

A JCA referral must not be requested for a DSP new claim until all non-medical eligibility has been assessed, including income and assets. In some instances, the claim will be automatically granted following the JCA, so it is critical the non-medical eligibility assessment occurs prior to the JCA referral.

Before a JCA referral is actioned, the Service Officer must confirm that non-medical eligibility has been assessed, confirmed and coded on the record (ensure coding has been applied using the Selective Application of Data (SAD) function). The Service Officer must also ensure that any changes to the customer contact details such as address, phone number, email address, etc. are recorded on the customer’s record before a JCA referral is actioned.

Ensure all other referrals have been actioned and finalised such as Complex Assessment Officer (CAO), Compensation, etc.

Has non-medical eligibility been assessed?

  • Yes, go to Step 4
  • No, see Streaming a new claim for Disability Support Pension (DSP)

4

Ensure medical evidence is scanned + Read more ...

Ensure all medical and other relevant information has been scanned and added to the electronic MIFE (eMIFE).

To view eMIFE information:

  • select Documents icon
  • If documents are scanned incorrectly and need to be separated or reclassified, complete the following in Customer First:
    • select Document Tools in Quick Links
    • See Step 7 in Table 1 of Scanning Centrelink medical/sensitive documents using an MFD to scan documents correctly

Has all available medical evidence been scanned correctly?

  • Yes, go to Step 6
  • No, go to Step 5

5

Do not proceed with JCA referral + Read more ...

A JCA referral must not be made until all available medical information and the MAT recommendation report has been scanned and attached to the eMIFE.

See Streaming a new Claim for Disability Support Pension (DSP).

6

Confirm contact details + Read more ...

Check the customer's address and contact details against the claim or review information to ensure they are current and correct. Update details if they have changed prior to starting the referral process.

Check the customer's interpreter requirements are correctly recorded.

7

Job seeker status + Read more ...

Check the XRN field in the header and JSR/ACT on Benefit Status line.

Does the customer have an active jobseeker registration?

  • Yes, go to Step 8
  • No:
    • Ensure customer data (specifically OCD CRES, RSLEG and RSIMME) is coded or has been updated
    • Process Direct attempts to complete the registration when a JCA booking is commenced by Assessment Services
    • Go to Step 8

8

Status of previous JCA or ESAt referrals + Read more ...

Any previous JCA or Employment Services Assessment (ESAt) referrals must have a status of finalised before a new JCA referral can be recorded. Process Direct does not allow a referral to be created if an un-finalised ESAt or JCA report is on the record.

If there is an un-finalised report for an accepted JCA or ESAt, staff with the relevant access can request (manual) early finalisation of the JCA/ESAt report. If staff do not have the required access, Request Finalisation will be greyed out. A Direct Referral to an SSO should be made. See Resources for a link to the Direct Referral to SSO webform.

  • Go to the Transaction Summary (TS) screen
  • Select > DSP Claim Summary
  • Select JCA/ESAt icon to view previous reports
  • Identify report with status of Accepted
  • Use Reference ID hyperlink to enter the referral
  • Select Request Finalisation

In some cases, the referral status on the customer's record will still show as submitted rather than finalised, even though the report has been accepted by the agency and at least 28 days has elapsed since the report was accepted. If the referral cannot be manually finalised using the Request Finalisation button, the case is to be reported to the SSO. See Resources for a link to the Direct Referral to SSO webform.

Has new medical evidence been lodged?

  • Yes, go to Step 9
  • No, see Step 1 in Table 2

9

New medical evidence lodged + Read more ...

Is the new medical evidence indicating potential manifest eligibility?

  • Yes, annotate the DSP claim progress DOC with 'Medical evidence lodged containing new information. Assessment by MAT required'. Create a MAT referral as per the below:
    • Go to the TS screen
    • Select > Referral
    • Referral Type: Medical Assessment Team Referral
    • Referral Reason: Query MAT - New evidence or Trm Ill - SA494/SA495/Other (if SA495 or other evidence indicating terminal illness is lodged). If Trm Ill - SA494/SA495/Other MAT referral is used, add additional text to referral that MAT Recommendation dated DD/MM/YYYY requires review as new medical evidence has been lodged
    • Select Finish
    • The claim status will be set to On hold for reason Awaiting MAT recommendation
    • Procedure ends here
  • No, see Step 1 in Table 2

Check the MAT recommendation has been completed + Read more ...

Confirm that the recommendation supports the completion of a JCA referral to determine medical eligibility.

Note: a recommendation template is not completed in all circumstances. For example, where no medical evidence is lodged with the claim, the MAT will complete a Fast Note and not upload a recommendation template. See Streaming a new claim for DSP.

If MAT have not completed an assessment and one is required:

  • Create an open work item using the Fast Note workflow
  • Select:
    • Category 1 - Disabilities
    • Category 2 - Claims
    • Template - DSP MAT Recommendation Query. Select Option 4: DSP New Claim lodged, No MAT assessment Completed (for example, if MAT assessment but UNSSPOC/SA479 has not been uploaded to the record). Manually add this information into the Fast Note
  • Manually hold the DSP new claim activity for 42 days
  • Procedure ends until the MAT assessment has been completed. See Streaming a new claim for DSP

3

Confirm non-medical eligibility has been assessed + Read more ...

A JCA referral must not be requested for a DSP new claim until all non-medical eligibility has been assessed, including income and assets.

Before a JCA referral is actioned, the Service Officer must:

  • confirm that non-medical eligibility has been assessed, confirmed and coded on the record (ensure coding has been applied using Selective Application of Data (SAD)
  • make sure that any changes to the customer contact details such as address, phone number, email address, etc. are recorded on the customer's record

Make sure all other referrals have been actioned and finalised, e.g. Complex Assessment Officer (CAO), Compensation.

Sometimes the claim will auto grant following the JCA. So it is critical to assess non-medical eligibility before the JCA referral.

8

Status of previous JCA or ESAt referrals + Read more ...

Any previous JCA or Employment Services Assessment (ESAt) referrals must have a status of 'finalised' or 'unable to complete' before a new JCA referral can be recorded. Check the Referral Summary screen in the customer's record. If there is an un-finalised referral, staff with the relevant access can request (manual) early finalisation of the JCA/ESAt report. If there is a NCL (New Claim) activity on the record, it will need to be cancelled to allow the early finalisation, and then re-indexed ensuring original keywords are put back onto the activity.

In some cases, the referral status on the customer record will still show as submitted rather than finalised, even though the report has been accepted by Services Australia and at least 28 days has elapsed since the report was accepted. If the referral cannot be manually finalised using the Request Finalisation button, the case is to be reported to the SSO. See Resources for a link to the Direct Referral to SSO webform.

New medical evidence indicates potential manifest eligibility + Read more ...

Annotate the DSP claim progress DOC with 'Medical evidence lodged containing new information. Assessment by MAT required'.

Create a MAT referral as per the below:

  • Return to the Activity List (AL) screen before launching the Fast Note workflow via Customer First > Workspace > Fast Note:
  • Select:

    • Category 1: Disabilities

  • This will create a Fast Note work item. It will show as the highest work priority for the MAT
  • Procedure ends here

Completing a JCA referral

Table 2

Step

Action

1

Run Fast Note + Read more ...

Run the relevant Fast Note (based on the mode of the JCA recommendation by the MAT):

  • JCA referral required post MAT - F2F
  • JCA referral required post MAT - File assessment
  • JCA referral required post MAT - Phone assessment

Ensure all details in the Fast Note are completed:

  • Date of claim: date the DSP claim was lodged
  • Residency: review and advise if the customer meets 10 years residency or has a Qualifying Residence Exemption (QRE) (this will inform the assessor if the Continuing Inability to Work (CITW) needs to be assessed). Ensure this field contains one word: Yes or No
  • Discipline: review and advise if the primary medical condition, as outlined in the Medical Assessment Team (MAT) recommendation is Physical or Non-Physical in nature. Ensure this field contains one word: Physical or Non-Physical
  • Specialist Referral/Assessment required: Yes/No (if yes, include the type of specialist assessment required)
  • Special Needs: if applicable
  • Add any additional information or special needs in the dialogue boxes if required. For example, interpreter required, potential violent behaviour, gender preference of JCA Assessor

Note: the Fast Note will stream the referral directly to the JCA Admin Team, bypassing the Assessment Services Triage Team. This process will not happen immediately, and may take several hours to move from the Service Officer's region code to the JCA Admin Team. Ensure only these Fast Notes are used.
