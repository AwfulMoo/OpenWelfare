Initial checks before starting quality check

Table 1

Step

Action

1

Allocation of QMA work items + Read more ...

QMOs undertake quality checks on work items allocated to them via Workload Management for work they are skilled to quality check. QMO's must have the appropriate business and technical knowledge in the relevant work type to complete the quality check.

QMOs must Unassign a work item, using Reason: Service Support Referral if allocated QMA work items they are unable to complete due to:

  • not having the appropriate skills to complete the quality check, or
  • received their own QMA work item

Note: QMOs may get a variety of QMA work items (new claim/non-new claim) to quality check. These are based on current demands. Staff should not unassign QMA work items that they are skilled to quality check.

See Work Optimiser.

Work item status

Work that needs quality checking will have a status of QMA. The corresponding activity on the Activity List (AL) screen in Customer First will have a status of SUB until the work item is completed in Process Direct.

If the work item has a status of Completed, it is finalised and not selected for quality checking.

If the work item/claim:

  • has a status of QMA but displays a message Error WS999-CRN: XXX Claim No: XXX Activity has been handed over to CF/CR for processing. Please click to view the Claim Summary, see Step 2 in Table 4
  • has a status of QMA but displays a message 'This work item cannot be processed using Process Direct', see Step 3 in Table 4
  • is completed outside Process Direct and the QMA status needs updating, see Step 4 in Table 4
  • has a status of QMA but the QMA check has already been completed and the transaction finalised, see Step 5 in Table 4

Urgent QMAs

If requested to do an urgent QMA, use Customer Search in Work Optimiser. Select the work item and then select Assign to Me before checking the QMA. See Work Optimiser for Staff.

2

Initial checks + Read more ...

Check the Claim Overview slider, scanned documents and extra information on the customer's record to support actions in the work item.

Ensure relationship details are correct on the Marital Status (MS) screen. Check record linking, for example, partner and child records.

Select Process. If Process is disabled:

  • the QMO may not have the skill tag to check this work item
  • the QMO has the correct skills to check the work item, but the skill tag may not be at the correct level (Experienced)

Check with the Team Leader/Manager about the skill tag or reallocation, if applicable.

Use the Super Key to go to the Entitlements (ELD) screen and any other required assessment screens, for example, RATS/RAC, to check the rate and outcome are correct. This will also provide a view of any manual adjustments applied by the original staff member.

Note: the assessment results displayed at this point reflect the coding completed by the staff member within the work item at the point of sampling for QMA. Check the Notes in Process Direct for explanation of any manual adjustments.

3

Check interface + Read more ...

Check which interface is being used for the quality check: standalone or integrated. See the comparison table on the Resources page for guidance.

If the QMA work item is using the:

  • standalone QMA, see Step 1 in Table 2
  • integrated QMA workflow, see Step 1 in Table 3

Completing quality checks using the standalone QMA

Table 2

Step

Action

1

Start quality check + Read more ...

Use the Super Key to go back to the SAP Warnings and Error (SWE) screen.

Select > QMA Processing

The QMA Summary screen will open as a new window and show all screens that need to be quality checked.

2

Check correctness + Read more ...

Use the fields displayed on the QMA Summary screen as a guide to what the staff member entered.

Verify these against information on Notes and any source documentation in Process Direct.

The Notes in Process Direct will display annotations made by the staff member, which include reasons for decisions and explanation for any manual adjustments made. This information must be taken into consideration when undertaking the quality check.

The Filter by Entity function can be used to limit the information displayed (by entity name).

3

Error codes + Read more ...

Select for an error code that matches the identified error. Select the most relevant code, this must be done for all errors.

No error for the field

If there is no error code available, apart from Q99999 (no error), send an email to QMA QOL tools with a screen shot showing the field in question. Continue with the quality check using the Other errors identified row, if appropriate.

Other errors identified

There is a mandatory Other errors identified row for recording other identified errors. Use this:

  • for errors that cannot be entered at the field level, including where Q99999 is the only option presented
  • to record an error when a field was not updated but should have been
  • to record feedback where claim status changes are required

Select QMO Comment field to enter feedback

Feedback must be provided for every identified error, including Other errors identified field

No line option for an error to be coded against

Use the most appropriate 'catch all' error against the mandatory row. Clearly identify what the error is in the comments field and include references to the appropriate Operational Blueprint files. Using the catch all error does not give the staff member a 'view' of their error. This can cause disputes if feedback is not sufficient.

Autofill remaining fields - Q99999 no error

After all errors have been marked, select Set on the QMA Summary screen to autofill remaining fields with a Q99999 no error entries.

To undo autofill, select Reset. This will remove all Q99999 no errors.

If necessary, change the Q99999 code by typing over the Error Code field.

All fields subject to QMA must have an error code recorded against it. This includes recording Q99999 on fields where no error exists.

Lines with the Green M are deemed high risk by the Business Owners. These must always be checked and a valid error code entered. (Use Q99999 if no error exists).

Validation errors

A QMA validation error will display within the QMA summary if an Error Code is not completed on a mandatory check. Using autofill will prevent this from occurring.

4

Enter feedback + Read more ...

For all errors identified, feedback must explain:

  • the error
  • how to correct it
  • the source reference that supports the result, including Operational Blueprint

Links to Operational Blueprint can be included in the QMO Comment field.

Feedback must be constructive, to the point and non-judgemental.

Effective feedback can avoid disputes. Give enough detail in the feedback so the staff member can understand and learn from their mistake.

See the References page for Quality Checking Feedback Policy

Note: the QMO completes all corrective action identified in the check.

5

Unable to complete the work item/claim now? + Read more ...

The status of the work item may need to be changed to In Process if:

  • relationship details are incorrect, e.g. incorrect date or partner
  • historical data has been incorrectly deleted
  • it cannot process without regenerating or resetting the status of the work item. This includes where a streamline rejection reason has been applied. The work item will need regenerating to remove this coding
  • it needs ICT escalation due to system errors that cannot be resolved
  • a request for more information needs to be sent
  • a referral to a specialist required e.g. CAO
  • the current claim is not required, as previous claim was rejected incorrectly and needs reassessing
  • Assurance of Support – an update is required to the Assurer Record or AoS Case

Can the work item/claim be completed?

  • Yes, go to Step 6
  • No:
    • where the work item needs ICT escalation, go to Step 8
    • all other cases, go to Step 7

6

Launch the finalisation workflow + Read more ...

Select Assess in Process Direct to launch the finalisation workflow. Check the final assessment outcome on the Entitlement (ELD) screen is correct, especially after any corrective action.

Assessment validation error(s)

Assessment validation errors may display on the SWE screen. This can occur, for example, because provisional data in the work item/ claim conflicts with data held in Customer First.

If the QMA summary screen is not correctly completed, an error prompt will display on the SWE Message Log, 'Please complete all ‘Error Code’ fields on the QMA summary screen'. It will prevent the work item from completing. If this occurs, go back to the QMA summary and complete the outstanding fields.

Has a validation error occurred?

  • Yes, go to Step 8
  • No, go to Step 9

7

Reject the QMA + Read more ...

Select Reject in QMA Summary to finalise the QMA check.

This will change the work item status to In Process in Process Direct.

  • Complete all follow up action needed, including Business As Usual (BAU) processes (e.g. request document, complete a referral, regenerate or escalate)
  • If requesting documents, when provided or the work item becomes due, it will be allocated by Workload Management to the next available suitably skilled staff member to finalise
  • Ensure notes are updated to record all actions taken as a result of the QMA check, as the work item will not be re-selected for QMA checking. If the original staff member's Notes do not reflect the QMA outcome, consider if this should be deleted. See Creating, reviewing and deleting documents (Fast Note, Callbacks, ACTDOCs and DOA DOCs)

Do not select the Reject option unless the status needs resetting due to any of the reasons listed in Step 5. This is not the same as a return in Quality On Line (QOL). If errors exist and can be corrected, the Submit option must be used.

Procedure ends here.

8

Resolve errors + Read more ...

Use all available resources to resolve validation errors. For help, see Digital Assistant in Process Direct.

If the error(s) has been resolved, go to Step 9

If the error needs ICT referral:

  • Use catch-all error Q00024 in the QMA. This is a non-proficiency affecting error, only for ICT referrals. An additional ‘Other errors identified’ line may be required to complete this action:
    • If no errors have been identified in the ‘other errors identified’ line, replace Q99999 code with Q00024
    • where errors have been identified in the ‘other errors identified’ field, an additional line will be required. Do not remove any existing error codes that were captured from the QMA quality check
  • Select Reject in QMA to finalise the checking process. The QMA must be rejected before making an ICT referral
  • Refer the work item to ICT, using ‘Been granted a bypass’, and include reference 111-20010856
  • Complete any BAU processes
  • Add Notes to clearly explain the action taken and that the claim has been through QMA. If the original staff member's Notes do not reflect the QMA outcome, consider if this should be deleted. See Creating, reviewing and deleting documents (Fast Note, Callbacks, ACTDOCs and DOA DOCs)

Procedure ends here.

9

Submit the QMA + Read more ...

Select Submit in QMA to finalise the checking process.

A confirmation of the QMA finalisation will display, QMA feedback has been saved, please finalise the work item. Select Yes to finalise the QMA.

Adjustments

If the staff member made an adjustment on the RATS/RAC screen, this must be coded again. For example, to suppress arrears. As the original action was not transferred to Customer First at the time the work item submitted for checking, the details will not hold.

The staff member should have recorded Notes to explain any adjustments. Replicate these Notes if they are correct.

Inhibited letters

Check if there are letters which should be inhibited. Follow normal processes for the work item being checked. Ensure these remain suppressed if required.

Note: the Submit option should be selected for most QMA outcomes. This applies if:

  • no errors exist, or
  • there are errors but corrective action can be taken to finalise the claim/work item in Process Direct

10

Finalise the work item in Process Direct + Read more ...

  • Complete any BAU processes
  • Add Notes for action taken
  • Select Finish
  • Select Finalise

If the original staff member's Notes do not reflect the QMA outcome, consider if this should be deleted. See Creating, reviewing and deleting documents (Fast Note, Callbacks, ACTDOCs and DOA DOCs).

Procedure ends here

Completing quality checks using the Integrated QMA workflow

Table 3

Step

Action

1

Start quality check + Read more ...

The SAP Warnings and Errors (SWE) screen will display and show:

  • any warnings or errors in the Message Log for review
  • Circumstance screens, where staff have completed updates, and
  • in the Task Selector, the QMA Summary and a question-based workflow Task (where applicable)

> QMA processing is disabled under the integrated QMA workflow. See the comparison table on the Resources page.

Select Next to start the quality check.

Is the screen part of a question-based workflow?

  • Yes, go to step 2
  • No, continue with the instructions below

The first Circumstance screen displays.

Check all provisional data for correctness. Verify these against information in the Claim Overview, on Notes and any source documentation in Process Direct. The Notes will display annotations made by the original staff member, which include reasons for decisions and explanation for any manual adjustments made.

If no error exists:

  • select No error. No other updates are required on the screen and the QMA will automatically be updated with code Q99999 no error
  • the QMA action options will update to Change and can be updated, if required
  • select Next to check next Circumstance Screen

If all Circumstance screens have been checked with No Error, go to Step 5

If an error is found, select Error, then go to Step 3

Note: If the screen is not part of a question-based workflow and is missing the No-error/Error buttons, email QMA.QOL.TOOLS with details including screenshots for further investigation.

2

Question-based workflow + Read more ...

Some work types use a question-based workflow to capture the online claim responses. These screens will display FORM on the Super Key. The following are not available in the question based workflow:

  • QMA Action column
  • No-Error and Error buttons

Errors identified within the question-based workflow must be recorded on the QMA Summary, under ‘Other errors identified’ field.

Work types that use this question-based workflow include but not limited to:

  • Pandemic Leave Disaster Payment claim
  • Australian Government Disaster Recovery claim
  • New Zealand Disaster Recovery Payment claim
  • Disaster Recovery Allowance
  • New Zealand Disaster Recovery Allowance

When a question-based workflow presents in the Task Selector on the SWE screen, QMOs must manually select both the QMA Summary and the Question-based workflow tasks in order for the Next button to be enabled.

For Emergency claims only (PLDP/AGDRP/DRA):

QMOs must take note of any errors when first reviewing the question-based workflow but do not complete any corrections at this point. Doing so can change the Claim Decision outcome, which may not reflect the original staff member’s decision.

Record any errors identified on the question based workflow under the ‘Other errors identified field on the QMA Summary.

  • If errors have been identified within the question-based workflow:
    • QMOs will need to complete the QMA Summary then return back to the question-based workflow to undertake any corrective action, then
    • Finalise the claim, where applicable
  • If no errors have been identified within the question-based workflow, continue to finalise the claim, where applicable

Some work items can have Circumstance screen Tasks as well as the Question-based workflow Task. If the SWE screen displays both Circumstance screen Tasks and Question-based workflow task, all tasks must be selected and quality checked.

Select Next to start the quality check.

The first Circumstance screen displays, where applicable.

Check all provisional data for correctness. Verify these against information in the Claim Overview, on Notes and any source documentation in Process Direct. The Notes will display annotations made by the original staff member, which include reasons for decisions and explanation for any manual adjustments made.

If no error exists:

  • select No error. No other updates are required on the screen and the QMA will automatically be updated with code Q99999 no error
  • the QMA action options will update to Change and can be updated, if required
  • select Next to check next Circumstance Screen

If all Circumstance screens have been checked with No Error, go to Step 5

If an error is found, select Error, then go to Step 3.

3

Error codes + Read more ...

Select for an error code that matches the identified error. Select the most relevant code. This must be done for all errors

Select QMO Comment field to enter feedback.

Feedback must be provided for every identified error, including Other errors identified field.

No error for the field

If there is no error code available, apart from Q99999 no error:

  • send an email to QMA QOL tools with a screen shot showing the field in question
  • continue with the quality check
  • if feedback is required, use the Other errors identified row available in the QMA Summary screen

Autofill remaining fields - Q99999 no error

After all errors have been marked, select Set on the QMA Comments screen to autofill remaining fields with a Q99999 no error entries.

To undo autofill, select Reset. This will remove all Q99999 no errors.

If necessary, change the Q99999 code by typing over the Error Code field.

All fields subject to QMA must have an error code recorded against it. This includes recording Q99999 in fields where no error exist.

Lines marked with M are deemed high risk by the Business Owners. These must always be checked and a valid error code entered. (The QMO can use Q99999 if no error exists).

Validation errors

A QMA validation error will display within the QMA summary if an Error Code is not completed on a mandatory check. Using autofill will prevent this from occurring.

4

Enter feedback + Read more ...

Use the Comments box to enter feedback.

For all errors identified, feedback must explain:

  • the error
  • how to correct it
  • the source reference that supports the result, including Operational Blueprint

Operational Blueprint links can be included in the QMO Comment field.

Feedback must be constructive, to the point and non-judgemental.

Effective feedback can avoid disputes. Give enough detail in the feedback so the staff member can understand and learn from their mistake.

See the References page for Quality Checking Feedback Policy.

Select Submit to save the error codes and comments noted.

The QMA action options will update to Change and can be updated if required. When going back to the Circumstance screen, complete the corrective action using Edit. Ensure the correct details are now displayed before going to the Next screen.

Note: the QMO completes all corrective action identified in the check.

5

QMA summary screen + Read more ...

When all the Circumstance screens and question-based workflow selected for QMA have been checked and corrective action completed, where applicable, the QMA Summary screen will be presented.

Note: where a question-based workflow presents in the QMA check, QMOs will be navigated back to the Task Selector after viewing the workflow. The QMA Summary task will be automatically selected. Select Next to complete the QMA check.

The QMA Summary screen will display all the screens checked, including any QMO Feedback noted, and the mandatory Other errors identified row.

Catch all errors

There is a mandatory Other errors identified row for recording other identified errors. Use this:

  • for errors that cannot be entered at the field level, including where Q99999 is the only option presented
  • to record an error when a field was not updated but should have been
  • for errors identified within the question-based workflow
  • to record feedback where claim status changes are required

Use the most appropriate 'catch all' error against the mandatory row. Clearly identify what the error is in the comments field and include references to the appropriate Operational Blueprint file(s). Using the catch all error does not give the staff member a view of their error. This can cause disputes if feedback is not sufficient. For a list of catch all error codes, see Resources.

Select to provide QMO Feedback:

  • Clearly identify what the error is in the QMO Feedback field
  • Select Save

Expand the Other errors identified row to access the Error Code field. Use the most appropriate catch all error, or select Q99999 if there are no other errors.

If there are multiple Other errors identified, select to insert extra lines.

Note: a QMA data validation error will display if an Error Code is not recorded on this mandatory row.

Unable to complete the work item/claim now + Read more ...

The status of the work item may need to be changed to In Process if:

  • relationship details are incorrect, for example, incorrect date or partner
  • historical data has been incorrectly deleted
  • it cannot process without regenerating or resetting the status of the work item. This includes where a streamline rejection reason has been applied. The work item will need regenerating to remove this coding.
  • it needs ICT escalation due to system errors that cannot be resolved

  • a referral to a specialist is required e.g. CAO
  • the current claim is not required, as previous claim was rejected incorrectly and needs reassessing

Can the work item/claim be completed?

  • Yes, select Submit and then Next to complete the QMA check. Go to Step 7
  • No:
    • where the work item needs ICT escalation, go to Step 8
    • all other cases, go to Step 10

7

Review the SWE screen + Read more ...

Select Assess in Process Direct to launch the finalisation workflow. Check that the final assessment outcome on the Entitlement (ELD) screen is correct, particularly if any corrective action was made.

Assessment validation error(s)

Assessment validation errors may display on the SWE screen. This can occur because provisional data in the work item/ claim conflicts with data held in Customer First

If the QMA is not correctly completed, an error prompt will display on the SWE Message Log - Please complete all ‘Error Code’ fields on the QMA summary screen. This will prevent the work item from completing. If this happens, navigate back to the QMA Summary and complete the outstanding fields.

Has a validation error occurred?

  • Yes, go to Step 8
  • No, go to Step 9

8

Resolve errors + Read more ...

Use all available resources to resolve assessment validation errors. For help, see Using Digital Assistant Roxy in Process Direct.

If the error has been resolved, go to Step 9.

If the error needs ICT referral:

  • enter QMA in the Super Key field to go back to the QMA Summary screen
  • use catch all error Q00024 under the expanded Other errors identified row. This is a non-proficiency affecting error and is only for ICT referrals. An additional ‘Other errors identified’ line may be required in order to complete this action:

    • if errors have been identified in the ‘other errors identified’ field, an additional line will be required. Do not remove any existing error codes that were captured from the QMA quality check

Extra processing information

Table 4

Item

Description

1

Regenerated claims + Read more ...

If a claim/work type has already been quality checked in QMA and this transaction is then regenerated, all QMA feedback, error data and coding will be lost. This means the original staff member will not receive any feedback or be aware of any identified errors.

Therefore, for any claims/work types that have been through QMA, regeneration should be the last resort.

If a claim needs regenerating, send all QMA feedback to the staff member via the Staff Feedback Tool.

2

Error WS999-CRN: XXX Claim No: XXX Activity has been handed over to CF/CR for processing. Please click to view the Claim Summary + Read more ...

When a work item/claim has been handed over from Process Direct to Customer First/Customer Record using the Handover function, Process Direct will:

  • display the WS999 error when selecting the work item
  • update the status of the work item to QMA where the activity has been selected for Quality Online (QOL) in Customer First/Customer Record

In this case, the QMA work item will allocate via Workload Management.

QMOs who are:

  • suitably skilled in completing the quality check:
    • locate the activity in the QOL tool and complete the quality checking
    • when the QOL has been completed, the status in Process Direct will update accordingly
    • no further action is required to the work item/claim in Process Direct unless the QOL outcome results in critical errors. In this case, the work item in Process Direct must be reassigned back to the original staff member to undertake the corrective action. See Work Optimiser. If the update/claim is urgent, QMO/QOL checker must take the corrective action. See Quality On Line (QOL)
    • if the QOL check has been completed but the status in Process Direct remains QMA, escalate the issue. Go to Step 3
  • not suitably skilled in completing the quality check in the QOL Tool, seek help from Team Leader/SSO. They will allocate the work item/claim to a suitably skilled QMO/QOL checker

For more information about the Handover function, see Process Direct navigation, common screens and functions.

3

Error 'This work item cannot be processed using Process Direct' + Read more ...

If a QMO is allocated a work item and this error displays when selecting it from the Process Direct inbox, the QMO must:

  • take note of the Customer’s Reference Number (CRN)
  • go to the Customer Summary tile on the Process Direct landing page
  • enter the CRN in the CRN Field, select Go
  • select the customer from the Search results
  • the Circumstance Data screen will display in the customer's record
  • Select to escalate the issue to ICT via Roxy

Note: Roxy is not available within the QMA work item/transaction if this error presents. In this case, escalate the escalation outside the QMA work item/transaction.

ICT will:

  • investigate the issue
  • remove the QMA status from the work item in Process Direct
  • remove the SUB (submitted) activity on the Activity List (AL) screen in Customer First (where applicable), and
  • advise if further action is needed. For example. they may direct staff to re-code the information in Customer First/Customer Record

Procedure ends here.

4

Changing the QMA status when the work item is completed outside Process Direct + Read more ...

When the work item/claim assessment has been completed in Customer First, a QMO must action the status change of the QMA work item:

  • Reject the QMA using the catchall error Q00024 to change the status to In Process
  • In Process Direct:
    • navigate to the Status icon
    • set the work item status from In Process to Complete
    • select the Save button

Note: the QMO must make an ICT referral via Roxy only if the above actions fail. Use the following wording: 'QMA status change attempts via PD failed'.

5

Changing the QMA status when the work item has been completed within Process Direct + Read more ...

When the QMA work item/claim assessment has been finalised in Process Direct, the system automatically updates the status from QMA to Completed. However, in some circumstances it may remain in a QMA status. If this occurs, QMOs must:

  • Select to escalate the issue to ICT via Roxy

If Roxy functionality is unavailable, QMOs must:

  • take note of the Customer’s Reference Number (CRN)
  • go to the Customer Summary tile on the Process Direct landing page
  • enter the CRN in the CRN Field, select Go
  • select the customer from the Search results
  • the Circumstance Data screen will display in the customer's record
  • Select to escalate the issue to ICT via Roxy

ICT will manually change the status from QMA to Completed.

Procedure ends here.
