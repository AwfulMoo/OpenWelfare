Reconciliation history statuses

Table 1: the overarching status of each occurrence under the Reconciliation History tab will be either Completed or Replaced. Further sub-statuses displayed underneath Completed or Replaced are explained below.

Code

Explanation

PRVS

This is the first status written, it shows that a reconciliation is required for this family unit, no summarised data will be present on this record.

AMET

This status means that the adequate income rules have been met and that this family unit is ready for processing, this will normally occur within the next few working days.

ASSD

This means the SSP has been processed and an assessment is pending wash up via the debt system.

WAIT

This is a status set when a previous reconciliation has not completed, however, a new update has been received. This status will wait until the previous result has been completed, then apply the updated information, potentially triggering a new reconciliation instance (re-reconciliation).

ECOM

The PD debt system has completed processing/offsetting and the provisional results have been written to the summarised record.

REPL

This status is seen when a re-reconciliation has occurred and a previous result has been superseded.

CANC

This status occurs when a reconciliation instance is cancelled due to the Australian Taxation Office (ATO) cancelling any or all of the previously confirmed family income. A new reconciliation instance will not occur until the ATO re-submits new income.

COMP

All offsets out of ATO and ISIS have been finalised and reconciliation instance is fully completed, triggering the reconciliation outcome letter to be issued and the online payment balancing page to become available for the relevant year. Where the customer has been impacted by 1DL or 2DL, these instances will show as COMP.

ACCS (Transition to Work) 2018-19 reconciliation debts require further investigation

Table 3: Portions of this process are to be completed by ACCS Processing Team (ACCPT) staff only.

Step

Action

1

Issue identified + Read more ...

An issue was identified where some ACCS (Transition to Work) applications were finalised without the 'Grant' indictor being selected in the 2018/19 financial year. This resulted in customers being granted ACCS (Transition to Work) incorrectly.

An additional issue may also affect these identified cases where ACCS (Transition to Work) standalone applications lodged prior to 13 July 2018 were also incorrectly displaying as closed. See, NNU 10808.

ICT have fixed the issue, but some customers may have an outstanding Child Care Subsidy (CCS) reconciliation debt for the 2018-19 financial year.

Staff can identify affected cases by navigating to the Transition to Work Assessment table using the TTW Super Key in Process Direct.

The Transition to Work Assessment table will display:

  • a Transition to Work line where the Grant indicator is blank, and
  • the line is displaying a Rejection Reason code of ICE

Is the record impacted by this issue?

  • Yes, go to Step 2
  • No:
    • further investigation is not required
    • procedure ends here

2

ACCS TTW incorrectly granted, action required + Read more ...

Staff must not:

  • escalate identified cases to ICT
  • attempt to reassess or correct entitlement to the payment, unless they are in one of the identified ACCS Processing Teams (ACCSPT)

Where a customer contacts enquiring about a reconciliation debt, and is also impacted by this issue:

  • advise the customer the debt will be investigated and someone will contact them soon
  • check that there are no existing ACTDOCs recorded on the customer’s record for this issue
  • send an ACTDOC to ACCSPT via Fast Note. Select Auto Text use Families > Claims > ACCS Claim Escalation Urgent Processing
  • region code FCC
  • service reason CPP
  • copy and paste the following text and include at the bottom of the ACTDOC:
    Urgent processing not required. Customer’s ACCS debt requires investigation for Transition to Work line with a Rejection Reason code of ICE

ACCSPT, go to Step 3.

All other staff, procedure ends here.

3

Retrospective reassessment + Read more ...

For ACCS Processing Team (ACCPT) staff only.

Check the debt via the CCS Reconciliation Dashboard and Debt Management tool to determine the periods where ACCS (Transition to Work) incorrectly paid.

The Transition to Work Assessment table and Child Care Entitlement summary may also be indicative of the period incorrectly paid.

Records affected will require a retrospective reassessment on the application lodged prior to 13 July 2018, to determine whether the debt can potentially be offset:

  • select the Transactions icon in Process Direct
  • select All Transactions
  • locate the original completed ACCS (Transition to Work) claim with a date of 13 July 2018 and copy the Claim ID
  • using the Claim ID, view the original assessment receipt in Customer First. Note: where the activity does not display in Interaction History, check the Activities tab
  • complete the usual claim pre-checks and request additional documents if required to allow reassessment
  • retrospectively assess the application, see Processing Additional Child Care Subsidy (ACCS) (Transition to Work) applications. Note: do not apply user discretion to favourably grant - the reassessment must reflect the correct entitlement
  • use the Date of Receipt (DOR) of 2 July 2018
  • check result on Assessment Results prior to finalisation

4

Record reassessment details, attempt customer contact + Read more ...

For ACCS Processing Team (ACCPT) staff only.

Record the following reassessment Note:

  • Category - Change of Circumstances
  • Summary - ACCS (Transition to Work) REASSESSED
  • Content - Copy, paste and edit the following:
    Transaction/Claim ID:
    ACCS (Transition to Work) REASSESSED - Division 4 s13 CCS Minister’s Rules 2017
    Reason: [Copy and paste one of the following options]:
    • < Customer was originally affected by an issue mentioned in NNU 10808 where the system incorrectly closed all Transition to Work assessments 13/07/2018 without further assessment. An additional issue affected the record where Transition to Work paid incorrectly, as no Grant indicator was recorded on the Transition to Work assessment table. Original application XXXXXXXX Submitted 13/07/2018 has been reassessed based on circumstances lodged at that time >
    • < Customer was affected by an issue where Transition to Work has paid incorrectly, as no Grant indicator was recorded on the Transition to Work assessment table.
      Debt amount: [if applicable]
      ACCS Start date:
      ACCS End date:
      Subsidy Period: [insert new grant weeks]
      JETCCFA deducted weeks:
  • Copy and paste one of the following options:
    • < This reassessment has reduced the amount of CCS debt for the customer by $XX. Customer will need to appeal the residual debt >
    • < While rejection for this assessment is the correct outcome, customer was still incorrectly paid by the system. Customer will need to appeal the reconciliation debt if they do not agree >
    • < This reassessment generated arrears which should fully offset the amount of CCS debt >

Once the reassessment has occurred:

  • annotate and close the ACTDOC to refer to the reassessment Note in Process Direct
  • make 2 genuine attempts to contact to notify of the reassessment outcome and to run the appeal script if necessary
  • where contact cannot be made, annotate the reassessment Note with the attempts

Note: ACCSPT Staff requiring technical assistance in reassessing ACCS (Transition to Work) assessments must refer to the boutique Technical Peer Support (TPS) model.

For ACCSPT actioning appeals affected by this issue, go to Step 5

Otherwise, procedure ends here.

5

ACCSPT actioning appeals affected by this issue + Read more ...

Normal appeal procedures apply and any consideration to debt waiver must meet the criteria outlined in Administrative error waivers.

Where debt waiver meets requirements, staff must ensure that the portion of the debt to be waived is only the CCS/ACCS affected by this issue. For example, exclude non-periods of ACCS where a debt is raised due to income review.
