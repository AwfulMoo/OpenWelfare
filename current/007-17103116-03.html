Recording activity details + Read more ...

Activity details are recorded in the customer's, partner's or ex-partners record on the Activity Test Details (ATD) screen in Process Direct.

  • Open menu
  • select the customer or partner as appropriate, then
  • go to the ATD screen via Super Key field

When updating the hours of a current activity:

  • the confirmed data line is to be end-dated first, and then
  • a new line must be recorded with the start date of when the new hours commenced

Important: complete the above in the same activity to ensure the correct outcome.

  • Where an increase in activity hours is within 28 days of notification, or a future date, go to Step 3
  • Increase in hours is more than 28 days prior to notification, go to Step 4
  • Commences an activity, go to Step 5
  • Decrease in hours go to Step 6
  • Ceases an activity, go to Step 8
  • Changes the end date of a current activity, go to Step 9

  • To code an Activity Test exemption, go to Table 4

3

Increase in hours is within 28 days of notification, or a future date + Read more ...

If a customer advises an increase to their activity hours from a date less than 28 days prior, or a future date (including where they previously recorded the hours incorrectly, e.g. weekly instead of fortnightly), end date the existing entry so the activity can be re-added.

Note: do not vary the hours by editing the existing row. This results in a reassessment and causes incorrect arrears or overpayments.

In the Activity Test Details (ATD) table:

  • Select Edit icon for the activity to be varied
  • End Date: insert the date the day before the customer changed their hours of activity. For example, if the customer notified on the 15 January 2019 their hours changed from 10 January 2019, the end date would be 9 January 2019
  • ATD Date of Receipt this is the date the activity was originally notified. Do not change from the original date
  • Select Save
  • Enter the original DOR in the 'Update Header Data' Receipt Date pop-up window
  • Select Save

If the partner's activity details have also changed:

  • select the partner from menu, and
  • repeat the process to update their activity details

Once the current activity has been end-dated, a new entry is required to record the new hours for the activity (within the same update). Go to Step 5.

4

Increase in hours is more than 28 days prior to notification + Read more ...

Customers may advise:

  • their hours of activity are higher than previously recorded, and
  • the date the hours increased is more than 28 days prior to notification

End date the confirmed data, using the date 29 days before the date of notification

  • Select Edit icon for the activity to be varied
  • End Date: insert the date 29 days before date of notification that the customer changed their hours of activity. For example, if the customer notified on the 15 January 2019 their hours changed from 10 October 2018, the end date would be 17 December 2018 (29th day)
  • Date of Receipt ensure this is the date the activity was originally notified. This should not be changed from the original date
  • Select Save
  • Enter the original DOR in the 'Update Header Data' Receipt Date pop-up window
  • Select Save

Once the current activity has been end-dated, a new entry is required (in the same update). This is to record the new hours for the activity with a manual calculation of the start date 28 days before the date of notification.

  • Select to add a new row
  • Start Date: enter the date 28 days before the date of notification. The system applies the correct Date of Effect and the customer’s entitlement is correctly assessed
  • End Date: this is a mandatory field
    • If the activity is Setting up a business or Unpaid leave (except for unpaid parental leave), staff are to ensure the activity period does not exceed 26 weeks
    • If Other Activity is selected, use the date the Service Officer has determined the activity is to apply, or 31/12/9999
    • For other activities, record an end date where this known (e.g. end date of training course, study), or 31/12/9999
  • ATD Date of Receipt: will default to today’s date. Do not backdate unless advised by an NNU, IT or the Level 2 Policy Helpdesk
  • Activity Type: select the relevant activity from the dropdown list. If the Activity selected is 'Other Activity', go to Step 7
  • Casual Employment will only display if 'Paid Work' activity type is selected. Select Yes or No as appropriate
  • Total Hours (per fortnight): enter the number of hours a customer usually spends engaged in the activity each fortnight. Where the customer (or partner) is in casual or irregular paid work, use the highest number of hours the person expects to work in a fortnight over the next 3 months
  • Select Save

If the partner's activity details have also changed:

  • select the partner from menu, and
  • repeat the process to update their activity details

Note: this process also applies where the increased hours occurred from the same activity start date already recorded. For example, customer indicates they were studying for 50 hours per fortnight from 1 August 2019, not 30 as previously recorded. Use the date 28 days prior to notification as the start date of the increased hours.

Go to Table 5, Step 1.

Logically deleted ATD data

Table 3

Item

Description

1

Logically deleted ATD activities + Read more ...

Previously confirmed activity data can sometimes be logically deleted when new information is provided. When this occurs, the system considers the updated ATD using the latest ATD DOR recorded on the ATD table. This may result in an unfavourable outcome for the customer.

View logically deleted ATD data by:

  • navigating to the ATD table in Process Direct, and
  • select the Show History link (top right of ATD table)

If investigating a reconciliation outcome, to view deleted or corrected activity updates during the CCS relevant year:

  • select the Activity History tab in the CCS Reco Dashboard
  • select the Show Logically Deleted button

This information is helpful for investigating if:

  • ATD updates have impacted the reconciliation outcome, and/or
  • corrections may be required

When DOR or ATD corrections are required, go to Step 2.

2

DOR or ATD corrections + Read more ...

There may be times when the ATD DOR requires correction, or previously confirmed activity data may need to be re-added. These include:

  • customers used the ‘I made a mistake’ option when changing their hours of activity. This caused their previous information to be logically deleted. This results in an unfavourable decision being applied as previously confirmed hours are no longer considered when calculating the Activity Test Result
  • incorrect DOR being applied when activity details have been updated
  • confirmed data being incorrectly deleted, instead of end dated and a new line being added when hours changed
  • system/user applying a later DOR to an activity when the only change was adding an end date
  • system/user applying a later DOR to an activity even though no details had changed

Important: Do not backdate the DOR to give a favourable outcome:

  • where the customer did not notify of the change within 28 days, unless
  • it has been approved through the review and appeal process

To determine what corrective coding (if any) is required, investigate the ATD history by navigating to the ATD table in Process Direct and select Show History:

  • Compare the DOR of the ‘confirmed’ entry against the original (earliest DOR) ‘deleted’ entry
  • Check whether previously deleted lines removed the customer’s confirmed hours incorrectly
  • Review the DORs and the hours advised to see what corrective coding may be required. Several updates may be required where:
    • the customer has advised varying hours of activity, and
    • multiple activities require correction

If action is required, coding should be completed as per the Resources page which contains hints and examples of correcting logically deleted data.

If outcome and entitlement are correct following updates, finalise activity and go to Table 5, Step 1.

    • Evidence of 65% of more care of child provided: YES/NO Source:

  • do not meet the definition of a grandparent

Outcome of Change + Read more ...

Once all required activity test changes and corrective coding has been completed for the customer and partner (if applicable), select the Assess button in the CCS customer’s record.

  • CCS and ACCS (where applicable) eligibility for each child
  • Activity Test Result, particularly in relation to the updates which have been made within the activity
  • Preschool additional hours (if applicable)
  • CCS Percentage

Scroll down to the Planned Payments table to view the provisional outcome of the updates.

  • If no payment outcome is expected, such as when the change will take effect from a future date, there should be nil impact (0.00) or ‘No Data’ will display. If this is the expected outcome, go to Step 3

    • investigate the outcome is correct by selecting the hyperlink and viewing the Provider and Customer Entitlement Overview, then
    • go to Step 2

Note: if a customer’s CCS has already been reconciled, arrears will be used to offset any debt during that year. If an undetermined debt exists, arrears will be offset against the undetermined debt for the same period.

Provider and Customer Entitlement Overview + Read more ...

Where arrears or overpayment is generated, investigate the outcome is correct by selecting the Net Amount hyperlink. This opens the Provider and Customer Entitlement Overview in a new window.

The outcome of the activity will display in the Total of the Provisional activity column.

If arrears or a debt amount is displaying in the Provisional activity, investigate the reason for the calculation:

  • Select the positive or negative amount hyperlink in the Adjustment Amount column
  • Click on Display Weekly Breakdown box
  • Take note of the Adjustment reason to the right (this may display a reason for the change), then click on the Difference hyperlink
  • The information that has changed within the activity will be red

Note: arrears generated due to the ATD update will be offset against any:

  • shell debt for the same period, or
  • reconciliation debt raised

For more information about favourable and unfavourable decisions in relation to Activity Test, see Changes in circumstances and date of effect for Child Care Subsidy (CCS) and Additional Child Care Subsidy (ACCS).

If the results are correct the Provider and Customer Entitlement Overview window can be closed. Go to Step 3.

Important: If the entitlement or payment outcome are not as expected, do not finalise the activity:

  • If the outcome is still not as expected or cannot be explained, follow the usual escalation process. If necessary, place the activity On Hold pending investigation

  • Select the Notes icon
  • Select and record details of the correction made
