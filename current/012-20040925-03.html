Step

Action

Access Online PBS Authorities – Request Authority + Read more ...

In the Medicare - eBusiness Gateway:

Approved prescriber search + Read more ...

  • Key the Approved prescriber number
  • Select [Enter] or Search
  • Ask the Approved prescriber to confirm their full name (if not previously provided) and select [Enter] or Next. Staff only proceed with the authority application if the prescriber provides both their first and last names as part of the security check

If the prescriber states they are known by another name such as their middle name, staff must explain they need to provide the name they have registered with Ahpra. The prescriber must obtain this information from the patient or Find A Patient service in Health Professional Online Services (HPOS)
  • Select [Enter] or Search

  • No, select the correct patient from the list and select [Enter] or Next

Item selection + Read more ...

Search for the item:

  • Key the:
    • name or brand of the item in the Keyword field (include the prefix ‘TAS’ for CAR items), or
    • item code in the Item Code field
  • Select [Enter] or Search
  • One search result will be returned for each item code. To view the brands listed for an item code, select the Item Name, Drug Form and Strength or Brand Name hyperlinks
  • Keyword search results can be further refined by entering applicable data in the Drug Form & Strength field. For example, after searching for morphine, enter 20 amp in the Drug Form & Strength field to view only the 20mg ampoule listings
  • Select the correct item from the list
  • The Prescribed Quantity and Prescribed Repeats will default to the PBS listed quantity and repeats for the item. These can be changed if required
  • The Daily Dose must be keyed. Doses up to 2 decimal places can be entered, e.g. a dose of 1.75ml per day can be entered as 1.75. Doses of less than 1 can also be recorded, for example 0.5 for half a tablet per day
  • If a daily dose cannot be measured key:
    • ‘X’ for items not measured by dosage
    • ‘D’ if directions were given to the patient, or
    • ‘N’ if no directions were given
  • For Highly Specialised Drugs (HSDs) the Hospital Provider Number field will appear. Key the hospital provider number. Note: Hospital Provider Number is not required for Community Access (CA) S100 items or Trastuzumab S100 items
  • After keying the correct quantity, repeats, dose and hospital provider number (if applicable), select [Enter] or Next

Select Reject.

Assessment result + Read more ...

If the assessment is Approved and an approval number is generated:

  • check for and action any Reason Codes as required (see Step 2 in Table 6)
  • read the approval number to the prescriber, and
  • ask them to write the number on the prescription

Go to Step 9 if the Telephone Authority is:

  • rejected as ‘too soon’ with Reason Code 094 or 095
  • approved with Reason Code 132 or Reason Code 805, or
  • has an ‘X’, ‘D’ or ‘N’ in the dose field

If the assessment is Approved and the Service Officer determines the request does not meet the criteria and should be rejected, the Service Officer can:

  • select Rejected from the Assessment Result Override drop down box and select Override, and
  • select the reason from the Override Reason drop down box or select Other and provide the reason in the text box, or
  • escalate to a PSO or PA for clarification

If the assessment result is Authority not required and the item is Authority Required (STREAMLINED) for the listed quantity and repeat(s) as listed in the PBS Schedule, tell the prescriber:

  • about the Streamlined Codes, that they are on the PBS website and the relevant Streamlined Code must be written on the prescription
  • that there is no need to call Services Australia to prescribe the PBS listed quantities and/or repeats

The Service Officer may provide the code after reminding the prescriber to ensure their patient meets the PBS criteria.

If the assessment result is Rejected, action any Reason Codes as required (see Step 2 in Table 6)

If the Service Officer determines the Authority request meets the criteria and should be approved, the Service Officer can:

  • select Approved from the Assessment Result Override drop down box and select Override, and
  • select the reason from the Override Reason drop down box or select Other and provide the reason in the text box, or
  • escalate to a PSO or PA for clarification

Note: rejections for ‘prescribed quantity and/or repeats is greater than allowed’ happen because of the limits set on the quantity and repeats by the Department of Health and Aged Care. Service Officers can override these rejections only if the requested quantity and repeats are in line with the internal approval rules. Check the relevant Dosage chart, PBS website and NDRD to determine if the request is within PBS guidelines.

Rejected as ‘too soon’ or manual assessment required + Read more ...

Applications with an ‘X’, ‘D’ or ‘N’ in the dose field need to be assessed manually for ‘too soon’. The system will return Information Reason Code 132 to advise that ‘MANUAL HISTORY CHECK REQUIRED - patient history check could not be completed’.

Where a previous authority has been rejected as ‘too soon’ and overridden by Services Australia, the system will return Information Reason Code 805. The application will need to be assessed manually for ‘too soon’.

If the telephone Authority application is rejected ‘too soon’ with reason code 094 or 095, or needs to be assessed manually, Service Officers must check patient history:

  • Select Display Patient History
  • Select Item Code to sort the results by Item Code
  • Use the patient history to calculate the ‘date the supply will last to’ and tell the prescriber of the rejection

The assessment results of the patient’s previous authorities are abbreviated in the Patient History pop up. Assessment Results in bold should be in the manual history check:

  • 1 = Approved
  • 2 = Approved with changes
  • 3 = Rejected
  • 4 = Pended/Saved
  • 5 = Previously rejected/now approved
  • 6 = Authority not required
  • D = Duplicate
  • C = Cancelled
  • E = Re-Issued

Note: when assessing ‘Too Soon’, OPA:

  • performs a consistent approach to all ‘Too Soon’ checking
  • starts the assessment from the most recent approval and works back
  • stops assessing once the first instance of ‘Too Soon’ is found
  • uses up to 12 months of history to calculate ‘Too Soon’
  • considers if any previous ‘Too Soon’ rejections have been overridden
  • considers whether the original has been supplied from previous approvals
  • does not take into consideration any prior approvals that may compound the calculation (if the doctor is disputing a rejection, a manual calculation could extend the next approval date further into the future)

OPA does not include prescriptions for non-authority or Streamlined quantities and/or repeats that may appear on the patient’s dispensing history. These supplies may be included when doing a manual calculation

For Service Officer information only (not for release): remember that the way the OPA calculates ‘too soon’ may not return the same result as a manual calculation.

When to perform a manual calculation to determine ‘too soon’

In some cases, a prescriber may dispute the too soon result returned from OPA. Perform a manual calculation using the Display Patient History function before changing the OPA assessment.

  • When assessing manually, calculate the past 6 months of authority approval history
  • Once investigated, if there is a valid reason for an early authority request, the rejection can be overridden
  • Select Approved and an appropriate reason from the Override Reason drop down box. Then OVERRIDE the application request
  • If the application is not approvable, advise the prescriber the history has been double checked and the application is not approvable unless there was an extenuating circumstance or explanation

Some authorities may return a too soon date within the next 24 to 48 hours. Do not automatically override these applications because ‘it's only 1 or 2 days away’. Advise the prescriber the patient has access to 1 or 2 months (depending on the drug) of medication. An application can be made on the displayed date through the prescribers HPOS system or via the PBS authority line.

To approve the replacement authority, key as an afterhours approval in Mainframe to maintain the correct date of lodgement:

  • Key RAUM in fast path (Mainframe)
  • Select '3 Record after hours authority'
  • Press [Enter]
  • Enter Medicare Number
  • Select correct patient
  • Press [Enter]
  • If patient comments appear press [Enter]
  • Select PF6
  • Rekey the new details onto Mainframe
  • Date of lodgement as per the script being replaced
  • Approval number as per the script being replaced or use the time and initials
  • Leave dispatch type as D
  • Enter '1' as Assessment result
  • Enter an Authority Comment to reflect why the authority is being keyed in Mainframe, for example:
    • W151 ASN 12345B
    • Previous authority did not print
  • Press [Enter]
  • Select PF5

If the authority request being replaced has been dispensed, do not change assessment result and refer to PSO/PA as per escalation procedure.

7

Re-Issue a prescription + Read more ...

The re-issue function can be used to generate a replacement authority based on the item, quantity & repeats, restriction and Q&A’s that were given previously.

Note: the date of prescribing on a re-issued authority will be the same as the date of prescribing on the authority being replaced, and cannot be amended. The re-issue function must only be used where the replacement prescription is written on the same date as the original, and only if trained in using the re-issue function.

To re-issue an authority:

  • Select Search Authorities
  • Key the applicable Medicare Number
  • From the search results list, select Re-Issue for the relevant authority
  • Key the new Authority Prescription Number
  • If required, amend the Daily Dose, Prescribed Quantity and/or Prescribed Repeats
  • Select the reason from the Reason For Re-Issue drop down
  • Select Re-Issue
  • Read the approval number to the prescriber for phone, or follow the usual process for written applications

Adding an authority comment after assessment + Read more ...

An authority comment or patient comment can be added after an authority request has been submitted and assessed.

  • Select Search Authorities
  • Key the applicable Medicare Number
  • In the search results list, select View next to the relevant record
  • Key applicable Authority Comments or Patient Comments
  • Select Save Comments
