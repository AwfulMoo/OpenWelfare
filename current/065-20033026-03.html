How to check the business role in ESSentials + Read more ...

Note:

  • Service Officers must have the below role and resource:
    • The AGED CARE - SERVICE OFFICER role (business role) via ESSentials
    • The ac_staff_portal Tivoli Access Manager (TAM) resource via mySupport

Service Officers can check if they have the business role in ESSentials by doing the following:

2

How to request the business role in ESSentials + Read more ...

Note: all fields are optional unless marked with an asterisk.

To request the business role:

  • Select the Add Role button underneath the User Access tab. The Select Roles pop up box will display
  • In the Functional Area field, select Aged Care from the drop down menu
  • Select the Search button. The Search results will display in the Available assignment block
  • Search for the required business role and select the row
  • Select the down arrow to move the selected role to the Selected assignment block
  • Select OK. The Access Request screen will display
  • In the Description* field, key in the business reason for the role request
  • In the Valid From* and Valid To* dates field, adjust the dates if required
  • Select Submit to finalise the role request

The access request requires line manager approval, then role owner approval.

The Service Officer will receive an email once access has been approved, go to Step 3

3

How to check if Service Officers have a P number + Read more ...

Service Officers must have a Tivoli Access Manager (TAM) account to access the ACSP. The Service Officer’s P number is confirmation that they have an existing TAM account.

To check the P number via the ICT Security Portal (ISP):

  • Open ICT Security Portal (ISP)
  • Select Copy or View Access underneath the Request Management menu option
    • The Copy or View Access screen will display
  • Select the magnifying icon next to the User to View field
    • The Search object list screen will display
  • Key the relevant details in the search field. Note: the Search field defaults to First Name
  • Select Search
    • The search results will display
    • The Agency Login ID field displays the Service Officer’s P number

Does the Service Officer have a P number?

  • Yes, go to Step 4
  • No, the Service Officer needs to complete the Medicare Legacy Access form in mySupport for the creation of a Tivoli Access Manager (TAM) account. Once ICT has confirmed completion of the TAM account, go to Step 4

4

How to request the Tivoli Access Manager (TAM) resource + Read more ...

Note: all fields are optional unless marked with an asterisk.

To request the TAM resource:

  • Open mySupport
  • In the Search field, key in ‘Medicare Access Change’. The search results will display
  • Select the Medicare Access Change mySupport form. The Medicare Access Change mySupport form will display
  • In the Requested For* field, ensure the details are correct (while this defaults to the requester’s log on ID, this can change to another log on ID by selecting the drop down menu)
  • In the What TAM (CDMS/PDS etc.) group(s) are to be added?* section, key in the required Tivoli Access Manager (TAM) resources (ac_staff_portal). Note: valid TAM groups must be specified. Requests with invalid group(s) will not be processed
  • In the What additional Mainframe access is required? section, key in ‘Not applicable’
  • In the What TAM group(s) are to be removed? section, key in ‘Not applicable’
  • In the What Mainframe access is to be removed? section, key in ‘Not applicable’
  • In the What is the business case for this access change?* section, key in the business reason:
    • Suggested text:
      “Please add TAM account for my [log on ID] in PRODUCTION Environment and assign TAM group (ac_staff_portal). This role allows me to access the production version of the Aged Care Staff Portal. I require this role as part of my job for aged care purposes. Please do not remove my other existing TAM groups.”
  • Select the Submit button. The Request Details screen will display
  • In the User Define Request Title* section, key in the title ‘Aged Care Staff Portal’
  • In the User Define Request Description section, key in description
  • Select Submit

The access request requires line manager approval, then role owner approval. The Service Officer will receive an email once access has been approved.

The Claim entitlements screen for residential care claims + Read more ...

  • Restricted access is required to perform the Approve and/or Cancel claim functionality in ACSP. Only a SME different to the one who finalised or refinalised the claim can approve or cancel the claim
  • Before approving a claim, a SME must check the following tolerances:
    • If the difference between the total entitlement subsidy amount of the service’s current and the previous claim month is less than 20%
    • If the total subsidy entitlement amount within the service’s current claim is between 1 cent and $5,000,000
    • If the positive or negative adjustment (excluding manual adjustments) within the service’s current claim is less than 20% variance of the previous claim month total entitlement amount
    • If there is no previous or last claim
    • If a respite care claim is more than $5000
    • There has been viability supplement changes
  • If the claim does not meet any of the claim tolerances, the SME must investigate and complete the claim manually or send back to the Service Officer to finalise again
  • If any error messages display, see the Payment Failures and Claim Errors table under Resources of the Aged Care Staff Portal (ACSP) Service in Context
  • These error messages must be resolved before the claim can be refinalised and approve again
  • For banking error messages, see Action an Aged Care GDES report for the process

The Claim entitlements screen displays the following sections:

  • Service details
    • Service ID - Note: if Service Officers selects the Service ID hyperlink, the Service details screen displays for the service in context. See Step 2 in the Using the Aged Care Staff Portal for service details (Service in Context) table for details about the screen
    • Service name
    • Care type
    • State
  • Claim details
    • Finalised date
    • Finalised by
    • Status
    • Claim month
  • Claim IDs
    • Claim ID
    • PACE Claim ID
    • Calculation ID
  • Entitlement details - displays the entitlements for the current and retrospective claim month(s) in a table
    • Entitlement month - select the relevant hyperlink to display the Subsidy and supplement details screen. The Subsidy and supplement details screen contains a list of care recipients’ payment details, entitlements and manual adjustment details for the entitlement month
    • Entitlement group
    • Entitlement type
    • Entitlements
  • Total entitlements - displays the total entitlement for the current and retrospective claim month(s)
  • Notes

Prior to a claim approval or claim deletion, Service Officers must review each claim’s entitlement information.

To Approve a residential care claim:

Using the Register service excluded from receiving basic daily fee supplement screen + Read more ...

Note:

  • This step is for Aged Care Programmes Branch’s Operations Team with the Aged Care Business Administrator business role
  • The No records information message will display when Service Officers key an incorrect NAPS ID, RACS ID or BP ID
  • The Service type must be residential information message will display when Service Officers key a home care or flexible care details
  • Service Officers can key the date (DD.MM.YYYY), or select from the calendar (select the calendar icon to display the calendar)
  • The Cannot be later than the current quarter date information message will display when Service Officers key a start date beyond the current quarter
    • For example - the Oct 2021 Dec 2021 quarter
    • The information message will display for Service Officers key a date of 01/02/2022 in the Exclusion start date field
  • A service cannot have concurrent exclusion periods
    • That is, the Exclusion start date* cannot overlap with another exclusion period
    • The Exclusion period cannot overlap with another period for the same service information message will display
  • To resolve this issue, Service Officers can either:
    • delete the existing exclusion period and register (create) a new exclusion period or
    • update the existing exclusion period to reflect the new updated exclusion period
  • All fields are optional unless marked with an asterisk

To use Register service excluded from receiving basic daily fee supplement screen:

  • Select Register excluded services on the Services excluded from receiving basic daily fee supplement screen
  • The Register service excluded from receiving basic daily fee supplement screen will display
  • Complete the Service details section:
    • Search by* - defaults to NAPS ID. Select RACS ID or BP ID from the drop down menu
    • NAPS ID* - this is a mandatory field if the NAPS ID option has been selected in the Search by* field
    • RACS ID* - this is a mandatory field if the RACS ID option has been selected in the Search by* field
    • BP ID* - this is a mandatory field if the BP ID option has been selected in the Search by* field
  • Select Search
  • Complete the Excluded date range section:
    • Exclusion start date* - the start date cannot be before 01/07/2021
    • Exclusion end date - the exclusion date must be after the start date and cannot overlap with another period for the same service

Service Officers can select either:

  • Submit
    • The ‘SUCCESS: Service record is created’ information message within the Receipt screen will display
    • Select Go to excluded services and the Services excluded from receiving basic daily fee supplement screen will display
    • The excluded service record’s Status is updated to Created
    • procedure ends here
  • Cancel
    • The Services excluded from receiving basic daily fee supplement screen will display

Procedure ends here.
