This page contains information on how to obtain a balance or transaction summary for a customer's BasicsCard.

On this page:

BasicsCard information

Manual process

BasicsCard information

Obtaining a balance or transaction summary for a BasicsCard

Table 1: This table details the processing instructions for workflows to obtain a balance or transaction summary for a customer's BasicsCard.

Step

Action

1

Customer/payment nominee contacts requesting a BasicsCard balance. + Read more ...

If not already done, confirm the customer's identification before proceeding with interview. See Authenticating a Centrelink customer. Note: there is no requirement to sight or photocopy customer identity documentation.

If an income managed customer has a payment nominee the BasicsCard will be issued to the payment nominee on behalf of the customer.

If it is the payment nominee contacting, ensure that the BasicsCard was issued to the payment nominee. Check that the payment nominee arrangement exists and the payment nominee's identity.

2

BasicsCard Balance Enquiry Line + Read more ...

Advise the customer/payment nominee of the BasicsCard Balance Enquiry line (Freecall) in case they wish to use this for future enquiries.

Calls to this number from a home phone are free. Calls from a public phone and mobiles may be timed and charged at a higher rate.

The BasicsCard Balance Enquiry Line (Freecall) is an automated Interactive Voice Response (IVR) line for BasicsCard balances only. Customers will not be able to be transferred to a Service Officer from this facility. Note: exceptions apply, including remote and vulnerable customers. For more information, see Conditions for transferring or referring calls.

A recorded message will advise customers that if they require any other Income Management business they must call the Income Management Line.

See the Resources page for a link to the Income Management Line and BasicsCard Balance Enquiry Line numbers on the Phone us page on the Services Australia website.

3

Processing + Read more ...

To view the BasicsCard status including balance and card number:

  • Via Customer First > Workspace > Income Management and under the BasicsCard heading, select BasicsCard Summary, go to Step 4
  • To access manually, see Step 1 in the Alternative process for a balance or transaction summary for a BasicsCard table
  • To access via the National Index, see Step 4 in the Alternative process for a balance or transaction summary for a BasicsCard table
  • If Customer First and National Index are unavailable use BasicsCard Alternative interface, see Step 6 in the Alternative process for a balance or transaction summary for a BasicsCard table
  • If all other options are unavailable, use the BasicsCard Contingency Support System. See Step 8 in the Alternative process for a balance or transaction summary for a BasicsCard table

4

BasicsCard Summary page + Read more ...

The following will be displayed on the BasicsCard Summary page:

BasicsCard details

  • Current Card Number
  • Name on card
  • Card status
  • Expiry Date
  • Card balance remaining $
  • Daily spend remaining $
  • Allocation Method

Income Management account

  • IM available balance $
  • IM current balance

BasicsCard Regular Expenses

  • Category
  • Next delivery date
  • End date
  • Amount $
  • Frequency

BasicsCard card limits

  • Daily spend limit $
  • Maximum card balance $

Advise customer of card balance, daily spend remaining and card status.

5

Change the daily spend limit or maximum card balance of BasicsCard + Read more ...

Customer/payment nominee can consent for the daily spend limit to be varied to an amount between $20 - $1,500. The maximum card balance can be varied to an amount between $100 - $3,000.

The default limits are $1,500 for daily spend and $3,000 for the maximum card balance.

Does the customer/payment nominee want to change the daily spend limit or maximum card balance of their BasicsCard?

  • Yes, see BasicsCard changing daily spend, maximum card balance limits and allocation method
  • No,
    • and the customer requires further transaction details, go to Step 6
    • and no further details are requested. Procedure ends here

6

Transaction Summary + Read more ...

Access the Transaction Summary screen. This can be located under Workspace > Income Management > BasicsCard > Transaction summary. Note: a transaction summary can be printed from this screen.

The transaction history for successful and/or declined transactions display can be refined by selecting the option of:

  • Today's Transactions
  • Last 30 Days, or
  • by entering the specific dates needed in the Transactions - From Date: and To Date: fields

Transactions that have 'Centrelink' in the Merchant Name: field will indicate transactions between the Income Management account and the BasicsCard account.

  • Positive (+) transactions will indicate funds transferred to the BasicsCard from the Income Management Account
  • Negative (-) transactions will indicate funds recalled from the BasicsCard to the Income Management Account

If necessary, discuss with the customer/payment nominee the transaction history.

If the customer/payment nominee disputes the balance or the transaction summary, see BasicsCard dispute resolution.

If requested by the customer/payment nominee, print a copy of the transaction summary.

7

Change the allocation method for the BasicsCard + Read more ...

Customer/payment nominee can consent and request to change the allocation method for funds transferring to the BasicsCard.

The default method customers will be on is the residual payment method. Customer/payment nominee can opt-out of this method and go onto the partial payment method.

Does the customer/payment nominee want to change allocation method of their BasicsCard?

  • Yes, see BasicsCard changing daily spend, maximum card balance limits and allocation method
  • No, procedure ends here

Alternative process for a balance or transaction summary for a BasicsCard

Action

Manual coding is available + Read more ...

  • Yes, go to Step 3
  • No, no further action is required. Procedure ends here

4

To access using National Index + Read more ...

When Centrelink system is unavailable, limited activities can be done in the National Index System. National Index is located in Environment 'N', in the 'NNS' system, and can only be accessed via Customer Record. Users can access BasicsCard balances but a BasicsCard transaction summary is not available.

To log into the National Index system:

  • Open Customer Record and enter 'NNS' in the system field and 'N' in the environment field

5

Search for customer in National Index + Read more ...

To search for a customer using the National Index:

  • Go to the IN screen with the system code set to 'NNS'
  • Search for the customer (this is the same as if in Centrelink system)
  • Press [Enter]
  • From the Client List (CL) screen 's'elect the customer, the National Index Person Details (PED) screen will display. This screen should provide enough information to authenticate the customer

Access information in National Index

Go to the BasicsCard Balance Enquiry (BCBE) screen and advise customer of the Card Balance, Daily Available Amount Remaining and Card Status.

Note: the BCBE screen can be printed if needed.

Procedure ends here.

6

Access the BasicsCard Alternative Interface web page + Read more ...

The Alternative Interface provides BasicsCard:

  • balance
  • status
  • transaction history
  • status updates, and
  • can be accessed to suspend, activate (reactivate previously suspended cards) and unblock BasicsCards when the BasicsCard number is known

The BasicsCard number must be known to use this interface.

Generally this would only be used when Customer First and the National Index are not available. As the BasicsCard Alternative Interface does not contain any customer information it could not be used to locate a customer's record or to verify a customer's identity. Access and use of this facility is intended for Service Officers in the 24/7 Income Management Line or Level 1 and 2 Helpdesks.

Note: this must be used in conjunction with Income Management Business Continuity Procedures to authenticate customers.

Log into the BasicsCard Alternative Interface

  • Open an Internet Browser session
  • Access the interface's web entry page. See the Resources page for the link to the login page for the BasicsCard Alternative Interface. Note: this page can be added to 'Favourites' for future use
  • To access this system, enter your assigned: 'username' and 'password', then select 'Log in'
  • Usernames are the user's full Services Australia email address, (for example, john.citizen@servicesaustralia.gov.au) in lower case
  • If the password is forgotten, select the 'Forgotten password?' link on the User Login screen. The email address will be the user's full Services Australia email address, (for example, john.citizen@servicesaustralia.gov.au) in lower case
  • If there are any issues accessing this facility, email the BasicsCard and TPO Management Team. See the Resources page for a link to Office Locator

7

Account or Card Search page will display + Read more ...

From the Account or Card Search page:

  • Select the 'Search by Card No' and enter the 16 Digit BasicsCard Number. Press 'Search'
  • The 'Transaction History' screen will display card balance, transactions and status
  • If the customer requires a Transaction History statement, this can be printed from this screen

The Service Officer will not be able to DOC the customer's record in the BasicsCard Alternative Interface. Manually record the details of contact for a DOC to be updated on the customer's record when Centrelink system is available.

Procedure ends here.

8

BasicsCard Contingency Support System (BCSS) + Read more ...

The BasicsCard Contingency Support system allows access to limited functions for BasicsCard when the Centrelink system, National Index and BasicsCard Alternative Interface are not operational.

The BasicsCard Contingency Support System will allow the following functions:

  • display current BasicsCard status
  • obtain BasicsCard balance
  • obtain Income Management account balance
  • suspend, reactivate or unblock a BasicsCard
  • change/reset a BasicsCard Personal Identification Number (PIN)

The BasicsCard Contingency Support System will automatically update details of any transactions completed when systems are restored.

To access the contingency system, use the link in Resources to the login page for the BasicsCard Contingency Support System. Note: add this address to 'Favourites' for future use.

See the Resources page for links to the training presentation available on the BasicsCard and TPO Management team homepage.

Note: BCSS will not open with the Firefox internet browser. Ensure that the BCSS is opened using Internet Explorer (IE).
