Actioning death of a PPS customer

Step

Action

1

Notification of death + Read more ...

When a person tells Services Australia about the death of a customer, the following action is to be undertaken:

  • If the person providing the information is distressed, offer a private interview before proceeding
  • Issue the pamphlet Are you needing help after someone has died? (HD001) using the Mail Forms guided procedure or manually in person
  • Offer the services of a social worker

Does customer's record indicate that they were income managed, as shown by INM on the benefit status line?

  • Yes, follow process to update record for death of an income managed customer, then return to this procedure and go to Step 2
  • No, go to Step 2

2

Confirming the death of the deceased customer + Read more ...

Confirmation can be accepted when customer's full name is provided together with at least 2 of the following:

  • deceased customer's address in full
  • date of birth
  • Customer Reference Number (CRN)

See Confirmation of the death of an adult or child.

Confirm if the person notifying of the death is the executor of the estate. If the informant is not the executor, but knows the executor details, obtain:

  • the name and address of the executor, and
  • the name of the funeral director or hospital

Nominee arrangements and Power of Attorney (POA) stop at time of death. Any correspondence, including the payment summary and possible debt recovery action from a deceased estate, will be sent to the:

  • executor, or
  • administrator of an estate

Services Australia's ability to finalise any outstanding matters will be impacted if executor or administrator details are:

  • incorrectly recorded, or
  • not recorded when details have been provided

Where executor details are not known at the time of death notification, the executor details fields in the Notification of Death guided procedure must be left blank.

Note: executor details can be recorded manually on the Death/Executor (DEA) screen if details become known later.

Can the informant supply enough information to confirm death?

  • Yes, go to Step 3.
  • No,

In the deceased customer's record run the Notification of Death guided procedure and complete as many fields as possible. Include:

  • the informant's name, address and phone number and their relationship to the deceased
  • if the informant is the Executor of the estate. If not, obtain the name and details of the Executor if known
  • obtain the name of the funeral director if known

The guided procedure will not code the details, but will record details on a DOC on the record which is an open work item to be allocated to a Service Officer with the required skill set to confirm the death.

If the guided procedure is not available, create an open work item using Fast Note. Select Auto text, use Generic > Update > Death Notification (Adult) - ACTION REQUIRED.

Confirm the death before taking any action.

If the customer's death cannot be confirmed, PPS should be suspended, reason OTH. The date of event of the suspension is the unconfirmed date of death or date paid to, whichever is the later.

  • Select the suspension option in the Notification of Death guided procedure, or
  • Manually code the suspension for reason 'other' (OTH)
  • Create a DOC on the customer’s record, ensuring that it meets Online Document Recording (ODR) standards for significant decisions and including what action is required to confirm the death
  • Record a manual review for 14 days from suspension to check if death has been confirmed
  • Do not issue a suspension letter

Procedure ends here until death can be confirmed.

Note: it is imperative that the death is confirmed and actioned as a matter of urgency.

3

Action once death is confirmed + Read more ...

Restore payment if suspended and determine if any arrears are payable, taking into consideration the confirmed date of death. The restoration advice must be inhibited, as it is not appropriate to send a letter to a deceased customer.

If the deceased customer was on reporting at time of death, see Death of a single pensioner.

A death action cannot be processed with activities on the Activity List (AL) screen. The Notification of Death guided procedure will provide a prompt if there is an activity on the AL screen. Where possible ensure any activities are completed and run the daily driver (DDR). If there is a 'STA' debt activity on the AL screen, create an open work item using Fast Note with the following information:

  • Select Auto text > Debts > DMB Raising > Debt Raising ACTion DOC Request
  • Select Confirm
  • Complete reason for request

If it is not possible to complete an activity on AL, the Notification of Death guided procedure will not code the death but will generate a DOC on the record which is an open work item to be allocated to a Service Officer with the required skill set to confirm the death.

4

Where it is not possible to complete an activity + Read more ...

If the Service Officer who is allocated the activity cannot complete it, the customer’s record should be suspended reason OTH, pending completion of the activity.

The suspension date of event should be the date of death or date paid to, whichever is the later.

  • Select the suspension option in the Notification of Death guided procedure, or
  • Manually code the suspension for reason 'other' (OTH)
  • Record a DOC on the customer’s record, ensuring that it meets Online Document Recording (ODR) standards for significant decisions and including what action must occur so the death can be actioned
  • Record a manual review for 14 days from suspension to check if outstanding activity has been completed
  • As the customer’s death has been confirmed, no letter must be sent. It is not appropriate to send a letter to a deceased customer
  • Immediately contact the team or Service Officer who needs to complete the outstanding activity
  • When activity is completed, restore payment if suspended. Ensure that no arrears are issued. The restoration advice must be inhibited, as it is not appropriate to send a letter to a deceased customer

Death action is to then be processed immediately.

  • Run the Notification of Death guided procedure (which includes executor details). If the guided procedure is not available, manually code the death action
  • Do not unlink children before coding death action. If the deceased customer was receiving Family Tax Benefit (FTB) or Child Care Subsidy (CCS), the guided procedure will cancel FTB/CCS and allow children to be linked to another person once the daily driver has run
  • Invite the new carer to claim FTB

This final instalment will be paid into the customer's bank account which can then be accessed by the executor of their estate.

5

Record details on a DOC and notify service providers + Read more ...

  • Identify any service providers to be advised of the death
  • Record the details of the bereavement on a DOC
