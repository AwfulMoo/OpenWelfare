This page contains the procedures for identifying and resolving replica HPI-O records in the HI Service.

Note: before completing this procedure, ensure that the policy conditions are met.

On this page:

Identify potential replica HPI-O record

Resolve identified potential replica organisation

Acting on behalf of another person? + Read more ...

Resolve identified potential replica organisation

For Tier 3 staff only

Table 2

Step

Action

1

Perform a search + Read more ...

Perform a search for the identified HPI-O potential replica organisation.

See Search for EHP Healthcare Identifier (IHI) record in Healthcare Identifiers (HI) Service

2

Contact the Responsible Officer (RO) or Organisation Maintenance Officer (OMO) + Read more ...

  • Determine whether the HPI-O is a replica
  • Advise them of the resolution process

3

Amend relevant record + Read more ...

Select Amend in the Action column of the relevant record

4

Organisation details + Read more ...

Select the Organisation Details tab

5

Resolve potential replica + Read more ...

Select Resolve Potential Replica

Potential replica details are displayed.

From the Replica Resolution list, select either:

  • Resolve as not Replica, if the HPI-O record is not a replica
  • Resolve as a Replica, if the HPI-O record is a replica

6

Key details + Read more ...

Key appropriate details in the Comments field. (Example: 'HPI-O record resolved as replica.').

7

Add details + Read more ...

Select Add

If resolving as a replica, the following warning message is returned.

'In resolving a HPI-O record as a replica, you will need to manually end date all links to RO, OMO and HPI-I records as well as either:

  • Restructure attached network organisations then deactivate this HPI-O record, or
  • Deactivate this HPI-O record and all its children

Select OK.

8

Submit details + Read more ...

Select Submit

Are there validation errors?

  • Yes, correct errors as prompted
  • No, the Confirm Amend HPI-O Seed/Network Details screen is displayed

If the details are:

  • incorrect, select Edit to return to the Amend Seed/Network HPI-O screen
  • correct, select Save. A confirmation message is displayed

9

When a replica has been resolved in the HI Service + Read more ...

Verify with the healthcare organisation:

  • which Public Key Infrastructure (PKI) certificates are no longer required to access the HI Service
  • what action needs to be taken on any organisation under the resolved replica organisation's hierarchy. One of the following may need to be completed:
    • Deactivate Healthcare Provider Identifier - Individual (HPI-I) record in Healthcare Identifiers (HI) Service
    • Restructure Healthcare Provider Identifier - Organisation (HPI-O) in Healthcare Identifiers (HI) Service
    • Link or unlink Responsible Officer (RO) or Organisation Maintenance Officer (OMO) to organisation in the Healthcare Identifiers (HI) Service
    • Link or unlink Healthcare Provider Identifier - Individual (HPI-I) to an organisation in Healthcare Identifiers (HI) Service
