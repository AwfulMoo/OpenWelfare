This page contains information on how to assess and process notifications and requests for general practitioners; medical practitioners who are registered or previously registered as a specialist in the specialty of General practice (including FRACGP and FACRRM who have registered or previously registered as a specialist with Ahpra) and medical practitioners included, or previously included on the VR.

On this page:

Assess a general practitioner notification

Process recognition as a specialist general practitioner subject to Section 19AA

Process recognition as a specialist general medical practitioner subject to Section 19AB

Process removal as a general practitioner

Process a request to confirm if a medical practitioner is on the Vocational Register

Escalate GP specialist recognition enquiries

PSO receives escalated GP specialist recognition enquiry

Assess a general practitioner notification

Table 1: This table describes how to assess a general practitioner notification in order to recognise or remove a medical practitioner’s general practitioner (GP) status.

Step

Action

1

Check notification + Read more ...

The notification processed must be either:

  • An Ahpra exception work item:
    • General Practitioner subject to 19AB
    • Record cannot be processed for a GP
  • the only, or most recent, notification recorded on a medical practitioner’s ‘shell’ record

If the notification is an Ahpra exception work item, go to Step 2.

If the notification was present on the medical practitioner’s ‘shell’ record, go to Step 3.

Note:

  • If an application form, requesting recognition (including re-inclusion on the VR) or removal as a general practitioner is received, it cannot be processed. Return the application using the appropriate option in the Recognition as a general practitioner/dual recognition letter (Z2569) template. Do not send these for delegate approval
  • For applications lodged prior to 16 June 2021, also escalate to a PSO who will escalate to a Programme Officer (PO) for data collection purposes
  • If there are any issues or concerns about a notification, contact a PSO for assistance

Confirm medical practitioner details in PDS + Read more ...

Open PDS:

  • Select Provider
  • Select View

  • In the Given name field key the first 3 letters of the first name followed by a %
  • Key year of birth in Year of Birth field
  • Select Profession ‘Medical Practitioner’ in the drop down
  • Select Search
  • A list of medical practitioners with a matching name displays

Is there a PDS match for the medical practitioner?

  • Yes, go to Step 3
  • No, contact a PSO for assistance

Confirm medical practitioner is Registered as a Specialist in General practice + Read more ...

  • Notification
  • Ahpra Information window

Is the medical practitioner Registered as a Specialist in General practice?

  • Yes, go to Step 4
  • No, go to Step 6

4

Check for Specialty Codes + Read more ...

Scroll to the bottom of the medical practitioner stem record to view the specialty codes for the medical practitioner.

Does the medical practitioner have a current 530 or 540 specialty code on their record?

Yes:

  • If a 530 is current on the medical practitioners stem recognition has already been processed. In PaNDA, add a note ‘Duplicate NFA’, then select Complete and select Save
  • If a 540 is current on the medical practitioners stem record:
    • confirm that the date of effect has been applied to the 540. The Resources page contains information about how to determine the date of effect
    • if the date applied to the 540 matches the date of effect, in PaNDA add a note ‘Duplicate NFA’. Select Complete, then select Save
    • if the date applied to the 540 does not match the date of effect, contact a PSO for assistance

No:

  • Use the information on the Resources page to determine the date of effect to be applied
  • Go to Step 5

Note: the date of effect cannot be before 16 June 2021. Legislation prior to changes in effect from 16 June 2021 did not support recognition as a specialist general practitioner.

Add Provider Service Rule (PSR) on all locations without a valid 19AB exemption + Read more ...

Select the medical practitioner’s stem (for example 123456) at the top of the Provider Stem record in PDS:

  • Check all locations
  • If there are no 519 19AB Exemption or Class Exemption flags recorded, add PSR to the location as per below

To add PSR:

Select:

  • Amend corresponding to the relevant provider number
  • Loc-Service Rules tab
  • Add Service Rule

Add Service Rule:

  • if a current 180 or 181 was identified in Step 1 (with a current 500/519):
    • Key 618 in the Service Rule Number field
    • In the Activity Code field, select YES from the drop down menu
    • In the Start Date field, key the date of effect and leave the End Date blank
  • if 110 was identified in Step 1 (with a current 500/519) and the location has business structure and EFT details keyed in full:
    • Key 618 in the Service Rule Number
    • In the Activity Code field, select YES from the drop down menu
    • In the Start Date field, key the date of effect and
    • In the End Date field, key the end of the 500 specialty code (Ten Year Moratorium - Medicare Benefits Not Payable)
  • if 110 was identified in Step 1 (with a current 500/519) and the location does not have business structure and EFT keyed in full:
    • Key 618 in the Service Rule Number
    • In the Activity Code field, select YES from the drop down menu
    • In the Start Date field, key the date of effect and leave the End Date blank
  • Select Add New
  • Select Save

Repeat this process for all required locations which do not have a current 19AB or Class Exemption. This action will prevent Medicare claims being paid at locations where a valid 19AB exemption is not in place.

8

Phone call to medical practitioner to notify new 19AB exemptions required + Read more ...

Check the medical practitioner’s provider number locations - if the medical practitioner has current 19AB exemptions and 3GA placement/s, the medical practitioner is required to reapply for a 19AB exemption as a general practitioner.

Note: where a medical practitioner is recognised as specialist general practitioner, there is no requirement to end 3GA placement/s. Requests to remove a 3GA program placement are sent by the authorising body. See Approved 3GA placements for more information.

If it is unclear which locations have 3GA placements issued, contact a PSO for assistance.

If a contact mobile or telephone number is recorded on the stem, make one attempt to contact the medical practitioner.

The telephone security check for Medicare health professionals must be completed before providing any Medicare eligibility information. If the security check cannot be successfully completed, go to Step 10.

Was the medical practitioner successfully contacted?

  • Yes, go to Step 9
  • No, make a note of this on the work item in PaNDA and go to Step 10

9

Successful contact + Read more ...

If the medical practitioner is successfully contacted, advise the following:

  • An application for a provider number with all supporting documentation is required to be forwarded to Services Australia as soon as possible for locations where access to Medicare is still required
  • 3GA placements can be withdrawn by the consortium. If this occurs and a 19AB exemption as a general practitioner has not been received, access to Medicare benefits at the location will cease

Add a note on the medical practitioner’s stem of the phone call. Select:

  • Edit
  • Notes tab
  • New Note - in Notes Subject key ‘Provider Contacted’, then in Notes Description key the details of the conversation. For example, 'Advised provider to submit application where MBS access is required'
  • Add new
  • Save
