Recall an original bulk bill LDA claim

Table 9: This table describes how Service Officers recall an original bulk bill LDA claim.

Note: the claim ID (to be adjusted) and date of lodgement (DOL) are equal to the original claim.

Step

Action

1

Key control line + Read more ...

Key control line 'DHSI,PROVIDER NO,CLAIM ID,DOL=DDMMYY'.

DHSC screen returns displaying the message 'CARD' indicating the vouchers within the claim are displayed in card number order.

Note:

  • when the DHSI control line is entered in the first instance, adjustment processing is initiated and the date of lodgement used on the control line is the DOL of the original claim
  • key A at the end of the control line to display the claim in alphabetical order - the return message 'ALPHA' is displayed
  • after initiation of adjustment processing, the DOL shown on the control line of the DHSC screen is the new adjustment claim DOL

Table 12: This table describes how Service Officers delete a line on the DHSC screen for bulk bill LDA claims.

Line deletion + Read more ...

  • Key T in the A field
  • Press [Enter]

Insert a line during adjustment

Table 13: This table describes how Service Officers insert a line during a bulk bill adjustment.

Step

Action

1

Activate insert function + Read more ...

Key I over the R in the A field of any restated line.

Note: I must be keyed over the R otherwise the insert function will not be activated.

2

New line insertion + Read more ...

Press [Enter].

The system returns the message 'INSERT', together with a new line that enables new details to be keyed.

Note: an N appears in the A field to indicate a new line.

3

Enter details + Read more ...

Key details in the following fields:

  • ITEM
  • DOS
  • BN ASSG

Note: these fields are highlighted and must be keyed. All usual keying fields in the new line remain unprotected.

The system:

  • repeats the card number and patient name from the restated line and these fields are protected
  • inserts PAYEE Code 6 and zeros in the FEE and BENEFIT fields, these fields are protected
  • leaves the PI, RSN and TX fields blank and not required when inserting a line

4

Finalise + Read more ...

  • key new line details
  • press [Enter]

The system returns:

  • restated line
  • new line
  • any messages that relate to the accuracy of the new details

Note: a new line can also be inserted over an 'N' once a new line has been generated.
