Using Notes

Table 1

Item

Description

1

Information recorded in Notes + Read more ...

Information recorded may include, but is not limited to:

  • the enquiry type or reason for the contact and the service or payment related to the contact
  • the details of who contacted. Calls received from third parties with implied consent, record the name, organisation or company and the contact details
  • additional relevant information that has not been recorded elsewhere on the record
  • relevant information received from a customer or third party that cannot be recorded elsewhere
  • reasons for the manual issue of forms, modules or letters not automatically generated by the system
  • advice given or received on future eligibility that is not recorded elsewhere on the record
  • the full name and contact details of the person who has contacted when the information has come from a third party
  • reviews that have been coded manually and why
  • specific details of updates made to the customer's record where these updates are not from a workflow or other process tool

Any details recorded must be in line with Operational Blueprint instructions.

9

Delete a Note + Read more ...

Information in Notes must only be deleted for the following reasons:

  • judgemental statements or personal opinion about the customer or their situation
  • information not mindful of Freedom of Information (FOI) and Privacy
  • information is not relevant to the customer, activity or decision
  • information is of a sensitive or personal nature
  • abusive or offensive language

If none of the above reasons apply, annotate the Note to correct the information, e.g. if a Note states home address is 1 Fleet Street when it is 2 Fleet Street.

Privacy incident: if there has been a breach of privacy, record a privacy incident. For more information, see the Resources page for a link to the Privacy and Secrecy intranet page.

To request deletion

This process is for deleting Notes generated and viewable in Process Direct only.

Complete and send the email template on the Resources page.

The Operations Design and Transformation team will action the deletion request within 1 business day.

If the deletion request is:

  • accepted, information will be deleted and text left to indicate information has been removed
  • not accepted, an email will be sent to the staff member for follow-up action.

2

Adding a DOC using Fast Note workflow + Read more ...

Access the Fast Note workflow via:

  • Workspace > Documents and Correspondence, or
  • Type Fast Note in the Search field on the Workspace page

If the workflow is not available, see Item 3.

Use the Auto Text option. (This will prefill mandatory fields)

If the auto text template name is:

  • known, before selecting the Auto text button, conduct a search:
    • key all or part of the template name in the Search field then select Search. A list of all templates that meet the search criteria can be displayed by selecting the drop down box in the Template field
    • select the required template
  • unknown, select the Auto text button and then select the appropriate categories from the Text Level 1: drop down boxes

Select Confirm once the template is listed.

Select Continue once the template content is complete.

If the correct Auto text template has been selected, the correct status will be applied to the DOC.

3

Adding a DOC when Fast Note workflow is unavailable + Read more ...

To manually create a standalone DOC:

In Workspace, select Documents & Correspondence > Document list > Add New Document.

Complete all mandatory fields of the DOC.

  • Summary line is automatically generated from the Who, Enquiry Type and Service Reason fields
  • Storage field will default to no source document
  • Storage date field is linked to the Storage field and cannot be prior to the Receipt date
  • If required, use Extra details field to:
    • briefly describe what is in the body of the DOC
    • highlight the need to read some specific information within the DOC
    • record information in addition to what is contained on the Summary line

Finalising the DOC

  • If still on the DOC screen, once fields have been completed, select Complete
  • If no longer in the DOC screen, either:
    • select the DOC from the Documents List (DL) screen, then select Complete, or
    • select the DOC from the Activity List (AL) screen and select Continue, then select Complete

4

Accessing a DOC + Read more ...

To access a DOC from the Document List (DL) screen:

  • Go to DL screen
    • in Workspace select Documents & Correspondence > Document list, or
    • key 'DL' in the Next: field and press [Enter]
  • On DL screen, select the relevant summary line to view the DOC, author ID, storage date, resubmit date, date of last update, storage location, last user ID, channel type, activity ID and text ID

Claim information

Claim information is available in the OLC Claim Progress DOC in Customer First.

Claim information is also available in Customer First for claims processed in Process Direct:

  • Service Officers can easily identify if customers have lodged a claim for one of these payments as there will be an OLC Progress DOC in Customer First
  • To respond to customer enquiries, Service Officers can view in Customer First the Notes recorded in Process Direct by:
    • select CRN/BP hyperlink
    • select Cases tab
    • select Super Case for business partner hyperlink
    • select Notes tab

Accessing DOCs that have not been finalised

The options are:

  • key 'AL' in the Next: field to go to the Activity List (AL) screen and select the started DOC or activity link to action
  • go to Customer Summary and select the link from the Incomplete/Outstanding Activities section. The AL screen will display. Select the DOC or activity link to action
  • select relevant DOC on the Event Summary (ES), Activity History List (HAL), or Historical Document List (HDL) screens

5

Annotating a DOC + Read more ...

A DOC should be annotated if the new circumstances relate directly to the existing content.

The only field that can be amended is the Text field.

All users can annotate another user's DOC. In this case, the date and logon ID of the user will be visible above the annotation within the text of the DOC.

Users cannot annotate their own DOC as the Text field remains open for the original user unless it has been completed or if another user annotates the DOC or subsequent comments are made on another date.

If a DOC has been annotated, the symbol (a) will display at the end of the summary line on the Document List (DL) screen.

To annotate a DOC:

Access the Fast Note workflow

  • Select Annotate existing DOC. A drop down selection will appear. Select the DOC to be annotated
  • To complete an annotation use a template or complete the Text field to add the annotated information

Alternatively, access the DL screen:

  • Select the Annotate button directly beside the summary line of the required DOC
  • Add the required information in the Text box
  • Select Continue or Complete to finalise the update

6

Printing a DOC + Read more ...

To print a single DOC:

  • go to the Document List (DL) screen
  • select the Print button on the subject line for the relevant DOC

To print multiple DOCs:

  • select the Print All Documents button at the top of the screen
  • the Print Documents (PDOC) screen will display
  • select Yes to proceed with printing
  • a message will display confirming the number of documents being printed and the relevant printer

7

Access to delete a DOC + Read more ...

Access to delete DOCs in Customer First should be automatically applied to APS5s and above. When access is required but not available, a request must be made in the ICT Security Portal (ISP) as follows:

  • ISP Category: Mainframe Application
  • Filter by Name: ODR
  • Select: ODRDEL - Online Document Recording Delete
  • Reason: Record clear details of why access is necessary for the role

Approval is by the Partnerships, Design and Assurance Team.

If help is needed with this access, contact the ODR helpdesk.

8

Deleting a DOC + Read more ...

Information in DOCs must only be deleted for the following reasons:

  • judgemental statements or personal opinion about the customer or their situation
  • information not mindful of Freedom of Information (FOI) and privacy
  • information is not relevant to the customer, activity or decision
  • information is of a sensitive or personal nature
  • abusive or offensive language

If none of the above reasons apply, annotate the DOC to correct the information, for example, if a DOC states home address is 1 Fleet Street when it is 2 Fleet Street.

Privacy incident: If there has been a breach of privacy, record a privacy incident. For more information, see the Resources page for a link to the Privacy and Secrecy intranet page.

To delete a DOC:

  • highlight the row of the relevant DOC on the Document List (DL) screen and select the More dropdown box
  • select Delete
  • the DDOC screen will display. Check the correct DOC is selected
  • select the suitable statement from the Reason dropdown box and select Continue. The system will replace information in the Text field
  • choose 'Yes' in the Confirm field and select Continue
  • the DL screen will display. The subject line of the DOC will be updated with date of deletion and logon ID of the Service Officer who actioned the deletion

Creating a Callback/ACTDOC work item

Table 2

Step

Action

1

Check if a Callback or ACTDOC work item is needed + Read more ...

Do not use the Callback ACTDOC guided procedure if there are instructions to use a Fast Note template.

Check the following:

  • Operational Blueprint procedures relevant to the customer's enquiry
  • Operational Blueprint Communications hub for important customer service delivery messages and updates
    • Announcements - daily communications and key messages
    • Messages - policy and procedural changes and reminders
    • OB updates - changes to the role of staff in completing updates on customer records
    • NNU bulletins - system issues
  • Locator page of the business team relating to the enquiry

See the Resources page for links.

Is a Callback or ACTDOC work item required?

  • Yes, go to Step 2
  • No, procedure ends here

2

Create the Callback/ACTDOC work item + Read more ...

Follow instructions in the relevant Operational Blueprint to create the work item using either Fast Note workflow or Callback/ACTDOC guided procedure.

If using Fast Note workflow

  • Access the Fast Note workflow
  • To find the template via search:
    • enter all or part of the template name in the Search field then select Search. A list of all templates meeting the search criteria can be displayed by selecting the drop down box in the Template field
    • select the required template
  • Or to find the template via drop down options:
    • select AUTO Text
    • select the relevant category 1 and category 2 options
    • select the template from the drop down list
  • Select Confirm
  • Complete the template, then select Continue

If using the guided procedure

  • Access the Callback/ACTDOC guided procedure. This will default to the service reason based on the customer's primary payment. Contact details will be generated by information on the customer record
  • Select type Callback or ACTDOC
  • Select appropriate category and item name
  • Select Generate Document Content to generate text
  • Complete details in the document content
  • Select Finish

When to action Callbacks and ACTDOC work items

Table 3

Item

Description

1

Is an Interpreter required? + Read more ...

Does the customer have a language recorded in the interpreter language field on Other Contact Details (OCD) screen?

  • Yes, Before actioning the Callback, contact an On-demand telephone interpreter to complete the Callback:
    • Contacting an on-demand telephone interpreter (Interpreter Connect)
    • Accessing an Indigenous interpreter
  • No, go to Step 2

2

Actioning Callbacks + Read more ...

Callback work items allocated to staff must be completed as soon as possible.

For information on contacting a customer, see Calling a customer or returning a customer's call.

  • After successful contact with the customer, the Service Officer must annotate the Callback work item DOC with the information provided or received
  • If unable to contact the customer, the Service Officer must annotate the Callback work item DOC with:
    • the attempt to contact, and
    • appropriate notes, including any relevant information easily distributed if the customer recontacts

3

Callbacks - attempt to contact the customer + Read more ...

Was the customer contacted?

  • Yes,
    • annotate the Callback work item DOC with the outcomes of the conversation
    • close work item
  • No,
    • annotate the Callback work item DOC with the contact attempt and any relevant information
    • Service Officer can utilise information when the customer contacts
    • close work item
    • if a request or follow up of further information is required, close the work item, do not place on hold. If needed, code a review for follow up

4

Actioning ACTDOC + Read more ...

Action is required when the ACTDOC work item is allocated.

All actions must be recorded in the ACTDOC work item DOC.

Could action be taken?

  • Yes,
    • annotate the ACTDOC work item DOC with the outcomes of the action taken
    • close work item (unless directed otherwise by the relevant operational blueprint)
  • No,
    • make sure all reasons for not actioning the request are recorded on the ACTDOC work item DOC including any follow up required
    • use information when the customer re-contacts to address the ACTDOC
    • if a request or follow up of further information is required, action the ACTDOC as instructed by the relevant Operational Blueprint. If there are no directions, finalise the ACTDOC. Do not place the ACTDOC on hold. If required, code a review for follow up

Displaying a DOA DOC + Read more ...

Staff must open and read each DOA DOC on the customer’s record and follow any included instructions.

If there is only one DOA DOC, the DOC will auto display when the Service Officer enters the record.

If there is more than one, the Document List (DL) screen will display a list under the Display on Access documents heading.

Adding a DOA DOC using the Fast Note workflow + Read more ...

Access the Fast Note workflow in Customer First. If the workflow is not available, see Item 3.

  • Select YES in the Document Completion field
  • Select the Display on Access button
  • In the Reason field, select appropriate reason for generating the DOA DOC from the drop down list
  • If the reason selected is adding a password, see Customer password on a Display on Access DOC for further information
  • Select an Expiry date as required
  • Select Confirm
  • Enter relevant text, then select Continue

See the Resources page for a list of DOA DOC codes, functions and expiry information.

If using a prepopulated Fast Note template:

  • Search and select the appropriate template as per relevant Operational Blueprint instruction
  • Check the 'Display on access' box

The DOA DOC code will prepopulate based on the template. Complete template as per relevant Operational Blueprint instruction.

3

Adding a DOA DOC when Fast Note workflow is unavailable + Read more ...

Manually create a standalone DOC.

  • Leave the Enquiry Type field blank
  • In the DOA DOC field, select appropriate reason for generating the DOA DOC from drop down list
  • Select an Expiry date as required

See the Resources page for a list of DOA DOC codes, functions and expiry information.

4

DOA DOC expiry periods and keywords + Read more ...

The work item ZIAM_DOA_REV Annual Review of DOA Message is created for this reason.

Each DOA DOC reason code has its own set of expiry and review date rules. It is important to use the most appropriate reason code for the message as the rules have been tailored to each.

Refer to the Resources page to identify what these are.

To manually expire or extend a DOA DOC

  • On the Documents List (DL) screen, select the grey box to the left of the relevant DOA DOC. It will then be highlighted
  • Select Expire or Extend from the More drop down box
  • Change the Expiry Date field to the desired date
  • Select Continue
