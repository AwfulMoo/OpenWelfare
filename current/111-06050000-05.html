Access roles for Centrelink (CLK) Workload Management (WLM) and Work Optimiser

Table 1: The following table outlines the security roles needed to access different functions in WLM and Work Optimiser for the Centrelink ICLK) programme. Security roles are requested and approved in ESSentials. For task card support when requesting a security role go to ESSentials > Help Home > User Access > Requestors.

Note: to avoid tiles displaying more than once in Work Optimiser, staff should only have one WLM role.

A separate procedure explains security roles for Customer First.

Security role

Role description

Details

ZCCRM_DHS_WLM

Access to display work items from Work Optimiser in the Customer First Inbox

All staff require this role.

Work Optimiser Centrelink - Service Officer

Enables individual WLM in Customer First functionality.

Access to Work Optimiser to view details relating to own work

WLM functionality for an individual.

Work Optimiser applications include:

  • Work Management menu
    • Worklist
    • My Dashboard
    • Staff Profile (own only)
  • Maintenance menu
    • Temporary Work Groups (read only)
  • Actor Management menu
    • Relationship Management (own only)

This role requires Line Manager approval.

  • Work Management menu

    • Relationship Management

Work item status definitions

Table 4

Status

Code

Description

New

NEW

Newly created and waiting for allocation to a staff member

Open

OPEN

Waiting for allocation to a staff member or to be actioned.

Draft

DRFT

A piece of work created by the customer but not yet completed/lodged.

Defer

DEFER

Allocation for a work item has been deferred due to an emergency event.

Waiting for Update

WAIT

A work item in Customer First or Process Direct has had a status change. The system is waiting for the new status to flow through.

Action Required

ACTI

Follow-up needed before action can be completed. Awaiting allocation to a staff member or to be actioned.

Hold Release

HOLD

Placed on hold and automatically assigned back to the allocation pool for action at a later date.

Hold To User

HTU

Placed on hold and retained in the staff member’s Worklist for action at a later date.

Quality Check

QUOL

Selected for quality checking.

Quality Management System

QMS_PRE

QMS pre-check required.

Submitted

SUBM

In transition between open and completed, e.g. pending or quality checking.

When an activity is submitted for Quality Online Checking (QOL), the related work item will auto assign to the staff member who submitted it. This prevents reallocation to other staff members.

Completed

COMP

Finished.

Cancelled

CANC

Cancelled, or created incorrectly (the work item should never have been created).

Work item unassign/reassign reasons

Table 5

Reason

Description

Capability Adjusted

The capabilities attributed to a work item can be changed if they are incorrect or if extra skills are needed.

Business owners can restrict the ability to do this for a work type in the WLM Business Rules.

Conflict of Interest

Reallocating a work item with this reason will set up a Conflict of Interest relationship between the staff member and the customer. This will prevent the allocation of work between the 2 parties.

Team Leaders can remove a Conflict of Interest relationship from staff they line manage. See Work Optimiser for team leaders and analysts.

Proficiency Raised

Each work item in WLM has a proficiency level attributed to it.

This reason can be used to update a complex work item to a higher proficiency. This allows for reallocation to a suitably skilled staff member.

There are 3 levels of proficiency:

  • Entry
  • Established, or
  • Experienced

Business owners can restrict the ability to do this for a work type in the WLM business rules.

Recategorised

If an allocated work item has been classified incorrectly, it can be recategorised to the correct work type. A work item should only be recategorised when:

  • The initial classification is incorrect, or
  • Recategorisation is part of an approved and documented business process, e.g. triage of JCA referrals or escalation process

Business owners can restrict the ability to do this for a work type in the WLM business rules.

Service Support Referral

A staff member needs technical support to complete the work item. They reallocate (reassign) the work item to a Service Support Officer.

User Unavailable

Selecting this reason returns the work item to the allocation pool unchanged.

This reason is generally used when a staff member is on unplanned leave or is unable to complete work items. Team Leaders can use this reason to reassign work from one staff member to another or back to the allocation pool.

Second Contact Attempt Required

Use this reason when a staff member received an appointment to action via outbound call, but the customer was not available. It should only be used for appointments and only applies to Centrelink.
