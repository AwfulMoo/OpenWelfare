Going to and using the Assessment summary screen + Read more ...

Go to the Assessment summary screen using any of the following:

  • Main menu (Hamburger button)
    • Select the Main menu icon
    • Select ACSP home page
    • Select Care recipient search tile under the Aged Care Staff Portal section
    • The Care recipient search screen displays
    • Search for the care recipient to display the Care recipient summary screen
    • Select Assessment summary under the Care recipient section
    • The Assessment summary screen displays
  • Care recipient menu tab
    • Select the Main menu icon
    • Select the ACSP home page
    • Select the All searches menu tab
    • Select the Care recipient search tile under the All searches menu tab
    • The Care recipient search screen displays
    • Search for the care recipient to display the Care recipient summary screen
    • Hover the mouse pointer over the Care recipient menu tab
    • A pop up window displays
    • Select Assessment summary
    • The Assessment summary screen displays

Using the Assessment summary screen

The Assessment summary screen displays the following:

  • Care Recipient banner displays the care recipient’s:
    • name
    • IDs
    • Date of Birth
    • Record type, and
    • Type of care status
  • Assessment summary table
    • Social Application ID
    • Unique Event ID
    • Effective date
    • End date
    • Form type - displays either NSAF, ACCR or ACCR-old
    • Status - select the hyperlinked Status to view the details of the individual assessment form details. Go to Step 3

Service Officers can:

  • select Transfer NSAF to display the Transfer National Screening and Assessment Form screen. Go to Step 8
  • type in a keyword in the Search bar to search for all records that contain either a single, or multiple keywords
  • select Go to approval details to return to the Approval details screen
  • select Legend for the explanation of the various NSAF codes
  • select the number of records to display from the Show records dropdown menu (bottom left corner of the screen)

Procedure ends here.

Going to and viewing the NSAF details screen + Read more ...

Note:

  • An individual NSAF form consists of multiple NSAF details screens
  • The NSAF details screen defaults to the latest version of the NSAF
  • The screen name may be different depending on the selected Form type record and is dependent on the assessment date
    • For example, the NSAF details screen displays if the selected Form type is NSAF, the ACCR details screen displays if the selected Form type is ACCR

Go to the NSAF details screen:

  • Select the hyperlinked Status on the Assessment summary screen
  • the NSAF details screen displays the following:
    • Part 1 - Client registration
    • Part 2 - Intervention/contact dates
    • Part 6 - Approval as a care recipient
    • Additional information

Service Officers can select on each NSAF details screen:

  • Version dropdown menu to view previous versions of the NSAF
  • Close to return to the Assessment summary screen
  • Back to view the previous NSAF details screen
  • Next to progress to the next NSAF details screen

Procedure ends here.

5

Going to and using the Match National Screening and Assessment Form (NSAF) screen + Read more ...

To go to the Match National Screening and Assessment Form (NSAF) screen, in the Held NSAFs screen:

  • Select the hyperlinked Social Application ID
  • The Match National Screening and Assessment Form (NSAF) screen displays

Note:

  • Do not create a new care recipient record, unless the NSAF does not match an existing care recipient record
  • Service Officers must check the Existing care recipient table for matching/similar care recipient records, or enter a known care recipient’s ID in the Enter care recipient ID* input field
  • If a matching care recipient record exists, Service Officer must attach the NSAF to that care recipient record
  • If Service Officers select an incorrect care recipient record, select a different care recipient record in the Similar care recipients table, or use Remove to remove (delete) the incorrect care recipient ID
  • All fields are optional unless marked with an asterisk

Use the Match National Screening and Assessment Form (NSAF) screen to match a held NSAF assessment to either an existing care recipient record, or create a new care recipient record. This is to attach the NSAF.

The Match National Screening and Assessment Form (NSAF) screen displays are below.

The ‘Please check for existing care recipients in the table below or by entering an ID. If the care recipient record already exists, do not create a new care recipient record.’ information message.

NSAF care recipient details:

  • Client’s surname
  • Client’s first name
  • Second name (if applicable)
  • Gender
  • Date of birth
  • Suburb
  • Postal code
  • NSAF details - Service Officers can select NSAF details to display the NSAF details screen

Existing care recipient section - displays if a matching/similar care recipient ID is selected.

  • Similar care recipients table - a matching care recipient can be selected by using the Select button in the Select one column:
    • Similarity
    • Care recipient ID
    • Given name
    • Family name
    • Middle name
    • Date of birth
    • Gender
    • Postcode
    • Suburb
    • Country of origin
    • Death date
    • Select one
  • The Enter care recipient ID* input field - Service Officers can key a known care recipient’s ID if applicable

Selected care recipient section - the matching care recipient record’s details are listed in a table:

  • Care recipient ID
  • Given name
  • Family name
  • Middle name
  • Date of birth
  • Gender
  • Postcode
  • Suburb
  • Country of origin
  • Death date

Before creating a new care recipient record, Service Officers must check if the held NSAF can be matched with an existing care recipient record.

Service Officers must:

  • check the details displayed in the NSAF care recipient details section; and
  • compare the details against the information in the Existing care recipient section; or
  • select a similar care recipient record, or
  • key a known care recipient’s ID in the Enter care recipient ID* input field

If a matching care recipient is:

  • identified, go to Step 6
  • not identified:
    • Select Create new care recipient
    • Go to Step 7

6

Matching care recipient + Read more ...

Note:

  • Once Service Officers select Add to care recipient to complete the Match NSAF process, they do not have an option to undo this process
  • Refer to Multiple ACMPS records remediation for the process to remediate any incorrect matches

In the Match National Screening and Assessment Form NSAF screen, Service Officers can either:

  • select a matching care recipient by clicking the Select button in the Select one column of the Similar care recipients table, or
  • key a known care recipient’s ID in the Enter care recipient ID* input field

This populates the Selected care recipient section.

The Selected care recipient section displays the following table:

  • Care recipient ID
  • Given name
  • Family name
  • Middle name
  • Date of birth
  • Gender
  • Postcode
  • Suburb
  • Country of origin
  • Death date

Does the Service Officer want to continue with the Match NSAF process?

  • Yes,
    • select Add to care recipient
    • the Held NSAFs screen displays, with message SUCCESS: NSAF added to existing care recipient 4XXXXXXXX
  • No, select either:
    • Create new care recipient and go to Step 7, or
    • Cancel and return to the Held NSAFs screen

Procedure ends here.

7

Create new care recipient + Read more ...

  • Once Service Officers have selected Create new care recipient to complete the Match NSAF process, they do not have an option to undo this process
  • Refer to Multiple ACMPS records remediation for the process to remediate any incorrect matches

In the Match National Screening and Assessment Form (NSAF) screen, if the Service Officer is satisfied that the NSAF cannot be matched to an existing care recipient, select Create new care recipient.

Does the Service Officer want to continue creating a new care recipient for the NSAF?

  • Yes,
    • select Create new care recipient
    • the Held NSAFs screen displays, with message SUCCESS: New care recipient 4XXXXXXXX created with this NSAF
    • procedure ends here
  • No, select either:
    • Add to care recipient and repeat Step 6, or
    • Cancel and return to the Held NSAFs screen
    • Procedure ends here

8

Using the Transfer National Screening and Assessment Form screen + Read more ...

Note:

  • All roles can view the Transfer National Screening and Assessment Form screen
  • Service Officers who have the following security roles can complete the Transfer NSAF process:
    • Business Administrator
    • IT Administrator
  • The Care recipient ID search field only accepts care recipient IDs as a 10 digit character ID, or the Invalid characters error message displays

The Transfer National Screening and Assessment Form screen displays the following:

  • Losing care recipient assessment details section - displays in a table format containing either a single or multiple NSAF records:
    • Unique event ID
    • Package ID
    • Effective date
    • Care recipient ID
    • Given name
    • Family name
    • State/Territory
    • Select
  • Gaining care recipient details section
    • Care recipient ID search field
    • a blank Gaining care recipient details table

To complete the Transfer National Screening and Assessment Form process:

  • Select the relevant NSAF record from the Select column in the Losing care recipient assessment details table
  • Key the care recipient ID in the Care recipient ID search field under the Gaining care recipient details section
  • The Gaining care recipient details table populates with the following details:
    • Care recipient ID
    • Given name
    • Middle name
    • Family name
    • Marital status
    • Date of birth
    • Gender

Does the Service Officer want to complete the Transfer National Screening and Assessment Form process?

  • Yes,
    • Select Submit
    • Go to Step 9
  • No,
    • Select Back to assessment summary to return to the Assessment summary screen
    • Procedure ends here

Person or organisation request access to NSAF information

Table 4: This table describes how Service Officers respond to enquiries from a person or an organisation about a care recipient’s NSAF details.

Step

Action

1

Organisation types + Read more ...

Note:

  • The NSAF is classified as 'Protected Information' under the Aged Care Act 1997
  • Do not refer these callers to the Department of Health or the state evaluation units.

Before the release of any information, the care recipient or nominee/POA (not the provider) must complete a Consent to release an electronic Aged Care Client Record form (AC010).

If the caller:

  • passes security checks and is registered to access the Aged Care Provider Portal (ACPP), the Service Officer can provide NSAF information. Procedure ends here
  • passes security checks and is not registered to access the ACPP, go to Step 2
  • fails the security checks, go to Step 3

2

Organisation must be registered for aged care the online claiming channel + Read more ...

Advise the caller that they must register to access the ACPP before Services Australia can provide NSAF information.

Direct the caller to the registration form:

  • Register, amend or remove users for Aged Care Provider Portal form (AC004)

Procedure ends here (until the caller has completed the form and is registered to access the ACPP).

Note: once the caller registers to access the ACPP, it will be treated as a new enquiry.

3

Unable to provide NSAF information + Read more ...

Advise the caller that Services Australia cannot provide NSAF information as:

  • their identity could not be confirmed
  • consent from the care recipient and/or the care recipient’s nominee has not been provided, or
  • they are not registered to access the ACPP

End the call – procedure ends here.
