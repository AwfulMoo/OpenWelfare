4

The client profile + Read more ...

In the Client Profile screen, the care recipient’s personal details are displayed:

  • Client number
  • Surname
  • First name
  • Second name
  • Date of birth
  • Client notes
  • Sex
  • Death indicator

The care recipient's history will display with the most recent history at the top of the list.

Each line represents an individual data transaction - this includes item, type, category, 'to' and 'from' dates and other related information.

Item and types are:

  • ADM = an admission to care
  • NH4 = an appraisal
  • NH5 = an ACCR (Aged Care Client Record)/NSAF and all ACAT approvals
  • B = Residential respite care entry
  • P = Residential permanent care entry
  • COM TC = Transition Care (Flexible Care) admission
  • COM RC = Short-term Restorative Care (Flexible Care) admission
  • RCA = Respite care days extension
  • TCE = Transition care extension

Note: for details about the care recipient’s National Screening Assessment Form (NSAF), go to Step 5.

Procedure ends here.

5

National Screening Assessment Form (NSAF) + Read more ...

To check for a current (valid) National Screening Assessment Form (NSAF), Service Officers must select the most recent NH5 item.

The type indicates the type and level of care the care recipient is approved for:

  • PR = Permanent Residential
  • PL = Permanent Low
  • PH = Permanent High
  • BL = Respite Low
  • BH = Respite High

Note: a care recipient can have concurrent approval for multiple types of care. The most recent NSAF entered supersedes any earlier approvals of the same care type.

Resolve excess care recipient social leave

Table 4: This table describes how Service Officers resolve the excess use of care recipient social leave.

Step

Action

1

Non claimable social leave + Read more ...

A care recipient is entitled for 52 days claimable social leave in a financial year. If a care recipient exceeds this amount within a financial year, these days are known as non-claimable social leave days.

Exceeding claimable social leave days will prevent the monthly claim from being finalised. The process to resolve non-claimable social leave days is different depending on the claim channel (online or manual).

Is this a manual claim via PaNDA (email, fax or post)?

  • Yes, go to Step 2
  • No, go to Step 8

2

Manual claims + Read more ...

A manual claim will not finalise in SPARC where the claim contains care recipients with non-claimable social leave days. An error message is displayed advising this issue.

Service Officers will need to identify the affected care recipients and resolve the issue prior to finalising the claim again.

To identify affected care recipients with the non-claimable social leave days, Service Officers will need to cross check all care recipient’s social leave balance in the current claim against the previous payment statement. The social leave balance is represented as the RC/Leave Days left column within the payment statement.

To check the previous payment statement, see Payment statement for aged care services.

Has the Service Officer identified the affected care recipients with social leave days?

  • Yes, go to Step 3
  • No, Service Officers must repeat this process prior to continuing

3

Calculating the number of claimable social leave days + Read more ...

Service Officers must manually calculate the number of claimable social leave days available for the care recipient.

The RC/Leave Days left column within the previous payment statement will advise the number of paid social leave days remaining for a care recipient.

Service Officers must use the days listed in the RC/Leave Days left column with their manual calculations.

For example, the June claim.

  • For the June claim, the care recipient was on social leave for the period 5 - 20 June
  • The RC/Leave Days left column in the May payment statement states 6 paid social leave days remaining
  • The new social leave start date will be 05/06/yyyy
  • The new end leave date will be 11/06/yyyy
  • The non-claimable social leave days will be 11/06/yyyy to 20/06/yyyy

This process must be repeated for all affected care recipients with social leave.

Has the Service Officer manually calculated the number of claimable social leave days and non-claimable social leave days for all affected care recipients?

  • Yes, go to Step 4
  • No, Service Officers must repeat this step prior to continuing

4

Coding the social leave + Read more ...

Service Officers must record the dates in SPARC

  • Log on to SPARC. The Main Menu screen will display
  • Select Claim Processing Menu - option 4
  • Press [Enter]
  • Select Occupancy Variation - option 2
  • Press [Enter]
  • Key in the service ID or the service name
  • Press [Enter]. The Occupancy Variation - RAC Service Occupancy screen will display
  • Select the care recipient by placing the cursor next to the Client ID in the 'Sel' column. Note: press [F8] or [F7] to access next or previous page
  • Press [F6] for Add/Update Client Leave. The Add/Update Client Leave screen will display
  • Complete the fields
    • Key '1' in the Leave Code field
    • Key the start leave date in the Start Leave field – this date as indicated on the monthly claim form
    • Key the end leave date in the End Leave field - this date was manually calculated in Step 2
  • Press [F6] to open a new leave line
  • Complete the fields for the new leave line
    • Key ‘2’ in the Leave Code field
    • Key the start date in the Start Leave field – the start date of the non-claimable social leave will be the same date as end date of claimable social leave
    • Key the end leave date in the End Leave field - the end date of non-claimable social leave will be the end date of social leave indicated on the monthly claim form
  • Press [F5] to commit. The information message NHP04191 Update Successful will display

Has the Service Officer completed this process for all affected care recipients?

  • Yes, go to Step 5
  • No, Service Officers must repeat this step prior to continuing

5

Add a care recipient note + Read more ...

Add a care recipient note to the claim notifying the approved aged care service that the relevant care recipient has exceeded claimable social leave of 'nn days' and the balance of the social leave has been processed as non-claimable.

To add a care recipient note, see Step 3 in Table 2.

Has the Service Officer completed this process for all affected care recipients?

  • Yes, go to Step 6
  • No, Service Officers must repeat this step prior to continuing

6

Add a payment statement note + Read more ...

Add a payment statement note to the claim. The details are as per the agreed care recipient note, e.g. the relevant care recipient has exceeded claimable social leave of 'nn days' and the balance of the social leave has been processed as non-claimable.

To add a payment statement note:

  • Log on to SPARC. The Main Menu screen will display
  • Select Claim Processing Menu - option 4
  • Press [Enter]. The Claim Processing Menu screen will display
  • Select Payment Statement Provider Notes - option 20
  • Press [Enter]. The Claim Processing Menu screen will display
  • Select Add Payment Statement Provider Notes - option 1
  • Press [Enter]
  • Key in the service ID or the service name
  • Press [Enter]. The Add Payment Statement Provider Notes screen will display
  • Key in the relevant details
  • Press [F5] to Commit

Has the Service Officer completed this process for all affected care recipients?

  • Yes, go to Step 7
  • No, Service Officers must repeat this step prior to continuing

7

Submit + Read more ...

Submit the monthly claim for finalisation.

To process the monthly claim, see Table 2.

Procedure ends here.

8

Online claims + Read more ...

Aged care services are unable to finalise their online claims via the Aged Care Provider Portal if the claim contains care recipients with non-claimable social leave days. An error message will display advising this issue.

Aged care services must identify the affected care recipients and resolve the issue through their current process. The aged care service can resubmit the claim for finalisation after they have resolved the social leave issue.

Deleting care recipient assessment in ACMPS

Table 11: This table describes how Service Officers delete a care recipient assessment details in ACMPS. All requests to delete assessment details must be directed to the Aged Care Programmes Branch for action.

Requests to delete assessment details are only actioned under rare occasions requested by the Department of Health and it is a delegation request. An example is when the care recipient’s assessment details were matched to an incorrect care recipient’s profile.

Step

Action

1

Requests to delete assessment details + Read more ...

Requests to delete assessment details are only actioned under rare occasions requested by the Department of Health and it is a delegation request.

  • Log on to ACMPS
  • On the ACMPS Home screen, select Claim & ACCR from the left hand side menu. The Claim & ACCR Search screen will display
  • Select ACCR Search. The ACCR Search screen will display

2

ACCR search + Read more ...

Key the relevant search fields with information to search the Social Application.

To search for the specific ID, in the:

  • the Social Application ID field - key the transaction number as per the spreadsheet
  • the Request Category field - select the relevant category type from the drop down option
    Note: must be set as ‘Aged Care Client Record (Eaccr/accr), ‘Aged Care Client Record (Eaccr/accr) Old’ or Aged care Client Record/NSAF – IC’
  • Select Search. The search results are displayed

Note:

  • The search fields displayed are not mandatory. The search operation can be performed using any single parameter or multiple parameters. Select the search parameters from the drop down box. If needed, search rows can be copied/removed by clicking the plus (+) or minus (-) icon
  • To clear the search values, select Clear
  • The clear operation clears the values in the search fields, but retains any new search criteria that might have been added

3

Record to be deleted + Read more ...

Note: if there was only 1 assessment associated with the care recipient, the assessment and the care recipient records are deleted. If multiple assessment records are available, only the selected assessment record is deleted.

  • Select the Social Application ID record to be deleted
  • Select Delete Assessment
  • The Social Application ID record is deleted and the name of the applicant is changed to Archivable Transaction
  • The status is changed as ACCR – Deleted
  • Select Save
