Assessing MBS item restrictions

Table 1

Step

Action

1

Identify the restrictive item/s + Read more ...

Complete the following to identify restrictive item/s:

  • access and check the claims processing screen
  • when processing a claim and a restriction exists between one or more item numbers, the restrictive item/s will display in the bottom section of the claims processing screen

Or

  • Access and check the patient history:
    • Key ‘NHSI,CARDNUMBER,PATIENT’ and press [Enter]
    • Press [PF9]
    • The patient PIN is displayed

Note: check history by PIN in all instances to ensure the complete claiming history is returned. Not just the history associated with one card.

2

Check return messages when processing a claim + Read more ...

When processing a claim and an error appears due to a restriction between one or more item numbers, a return message will display.

Service Officers are to refer to the return message by selecting the [F1] function key. This provides the correct procedures to be followed.

3

Check QITI + Read more ...

Check QITI to see what restrictions are in place (e.g. same day, time dependency, age restriction, association rule etc.).

4

Check account/claim + Read more ...

Check the account or claim for additional information that may have been provided to support the restrictive situation between one or more item numbers. Where relevant also check patient history for claim text on the restrictive items.

If the claim is accompanied by clinical details, see Assessing complex Medicare Benefits Schedule (MBS) claims with clinical details.

5

Check assessing rulings + Read more ...

Check the assessing rulings through QITI by selecting the [F10]-ASS RULE function key.

6

Check MBS + Read more ...

Refer to the item number/s descriptors and associated explanatory notes.

7

Unable to decide + Read more ...

If none of the above steps have resolved the issue contact a Programme Support Officer (PSO) for assistance.

Restriction lift requests

Table 2

Step

Action

1

Decide when a restriction lift is required + Read more ...

Check:

  • the item description for legislative requirements and decide if the requirements prevent payment of the items
  • account/claim for restrictive items
  • Medicare patient history for other restrictive items on history:
    • Key ‘NHSI,CARDNUMBER,PATIENT’ and press [Enter]
    • Press [PF9]. The patient PIN is displayed
    • Key ‘NHSI,PIN,DOS/DDMMYY’ and press [Enter]

Note: check history by PIN in all instances to ensure the complete claiming history is returned. Not just the history associated with one card.

  • QITI to see what restrictions are in place (e.g. same day, time dependency, age restriction, association rule etc.)

Read:

  • the assessing rulings and MBS to decide if the items are eligible for payment
  • if a restriction lift is required, go to Step 2

2

Completing a restriction lift email + Read more ...

The Restriction Lift Request template is located on the Resources page.

Provide as much information as possible to assist with deciding if the restriction is eligible for lifting. Ensure all the necessary fields are completed, if the fields are not completed correctly the request will be returned.

If payment of a Medicare benefit is indicated based on clinical details, provide an explanation as to why the restriction is to be lifted. Where approval for payment of a Medicare benefit has been given by a team leader (TL) or programme support officer (PSO) attach a copy of the determination to the restriction lift request.

Note: if the restrictive or associated item is in patient culled history, a history line to have the item returned to Mainframe history will be needed. To use history line, contact the Medicare Assessing Helpdesk.

For multiple restriction lifts for the same patient go to Step 3.

For restriction lifts for multiple patients go to Step 4.

For cleft lip and cleft palate over 28 years restriction lift requests go to Step 5.

For sending a restriction lift request go to Step 6.

Note: for complex time dependency restriction lifts, see the Resources page.

3

Multiple restriction lifts for the same patient + Read more ...

  • Complete one template for each item number which requires a restriction lift
  • Multiple templates for the one patient can be sent in the one email request
  • When you get a claim for the same item number but there are multiple date of services, indicate the item number times and the number of services in the subject line. E.g. 132 x 2. Note: each date of service must be specified in the date of service field
  • Go to Step 6

4

Restriction lifts for multiple patients + Read more ...

  • complete one template for each patient
  • each template for each patient must be sent in separate emails
  • go to Step 6

5

Cleft lip and cleft palate over 28yrs restriction lift requests + Read more ...

The restriction lift request must include:

  • The approval letter provided by the Department of Health
  • All relevant patient details attached for the age restriction including previously paid Cleft Lip and Cleft Palate items
  • Details about if the service is for 1st or 2nd course of treatment
  • The previous number of services under that course of treatment (e.g. 11th service of the 1st course of treatment). This applies to items such as 75039 and 75048
  • Go to Step 6

6

Sending the completed restriction lift request + Read more ...

Email restriction lift requests to Medicare Assessing Helpdesk. Ensure:

  • the subject heading is in the following format:
    • RST Lift – State - Item xxxxx (patient name)
  • requests sent as needed and not kept to be sent in bulk

7

Payment of claims + Read more ...

Where a restriction lift is approved, Medicare Assessing Helpdesk will advise by email what date the claim can be processed. This approval must be received before the claim is processed.

Claims must be paid on the date advised by Medicare Assessing Helpdesk, if the requesting officer is on leave or unavailable on the day, another Service Officer should ensure the claim is processed.

Claims approved previously but left unpaid on the specified processing day must be re-submitted and approved before the claim can be reprocessed.

Multiple attendance items claimed – same DOL and same DOS – Return message 6-RST ADJ, 6-RST REJ, 6 SUS DUP

Table 3

Step

Action

1

Check item numbers + Read more ...

Are items being claimed for CDM services (items 229, 230, 233, 721, 723, and 732)?

  • Yes, go to Step 2
  • No, go to Step 3

2

CDM items can’t be claimed with attendance items + Read more ...

CDM items can’t be claimed with the below items:

  • 3, 4, 23, 24, 36, 37, 44, 47, 52, 53, 54, 57, 58, 59, 60, 65, 179, 181, 185, 187, 189, 191, 203, 206, 585, 588, 591, 594, 599, 600, 733, 737, 741, 745, 761, 763, 766, 769, 5000, 5003, 5020, 5023, 5040, 5043, 5060, 5063, 5200, 5203, 5207, 5208, 5220, 5223, 5227 and 5228

If claimed, pay the CDM item and reject the other attendance item. Use:

  • the rejection code in the error message on Mainframe, or
  • choose the proper rejection code based on the claiming channel

See Indicators, codes, modifiers and control lines for claims processing in Medicare for more information.

3

Read return message + Read more ...

Read the return message in Mainframe.

Has acceptable text been provided with the claim?

See Resources page for information and examples for acceptable text.

  • Yes, go to Step 5
  • No, go to Step 4

4

Pay the item with the higher schedule fee + Read more ...

Pay the item with the highest schedule fee.

Reject the other item(s) with the rejection code in the return message (it can be different depending on the claiming channel).

The process ends here.

5

Pay the claim + Read more ...

Pay all items in the claim using the code in the return message.

The process ends here.

3

Check item numbers + Read more ...

  • Yes, go to Step 4
  • No, go to Step 5

CDM items can’t be claimed with attendance items + Read more ...

CDM items can’t be claimed with the below items:

  • 3, 4, 23, 24, 36, 37, 44, 47, 52, 53, 54, 57, 58, 59, 60, 65, 179, 181, 185, 187, 189, 191, 203, 206, 585, 588, 591, 594, 599, 600, 733, 737, 741, 745, 761, 763, 766, 769, 5000, 5003, 5020, 5023, 5040, 5043, 5060, 5063, 5200, 5203, 5207, 5208, 5220, 5223, 5227 and 5228

  • CDM item has already been paid, reject the other attendance item with the rejection code in the return message. The process ends here

  • No, go to Step 6

6

Reject relevant items + Read more ...

Follow the instructions in the error message to reject the item being claimed. The process ends here.

7

Pay the claim + Read more ...
