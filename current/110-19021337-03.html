Discuss Income Stream data with customer

Table 2

Step

Action

1

Discuss Income Stream data + Read more ...

A match has occurred because the information on the customer's Centrelink record differs from information received from an income stream provider.

Prior to discussing the Income Stream data with the customer, check the customer record to ensure the funds/ asset has not already been assessed and/or debts raised:

  • Managed Investments Summary (MIS) screen
  • Superannuation Summary (SUPS) screen
  • Savings Summary (SVS) screen
  • Financial Investment Summary (IVIS) screen
  • Pensions Income and Asset Summary (PIAS) screen
  • Pensions Assessable Asset Summary (PAAS) screen
  • Assets Summary (NASS) screen

If identified, consider the use of that information and take the required actions prior to coding on SUPS. The information found and actions taken must be clearly documented in the Notes.

Discuss the Income Stream data with the customer and give them a chance to provide a verbal update.

The scope start date will either be:

  • the start date of the income stream, or
  • if the start date is greater than 7 years, the scope start date will be 1 July of the financial year, 7 years before the current financial year. (For example, currently in 18/19 financial year, scope start date will be 1 July 2011)
  • if the customer started receiving an income support payment after the income stream start date, the scope start date will be the date in which the customer’s payment started

Discuss the amounts listed in any of the income fields on the Income Stream Data.

Use questioning skills to get as much information as possible from the customer over the phone.

If the customer indicates the information is incorrect, review the customer record to determine an anomaly or mismatch.

Inform the customer:

'Would you mind holding the line while I check your record.'

Preliminary assessment

If there are at least 2 commonalities between the customer and the Income Stream data (e.g. name and date of birth, or name and address), it can be decided that the Income Stream data is not a mismatch and should be continued.

Alternative spelling or misspelling of a customer's name does not mean a mismatch.

Can an anomaly or mismatch be identified from the customer record?

  • Yes, go to Step 2
  • No, go to Step 3

3

Assessing verbal updates + Read more ...

Get the following information from the customer for each income stream product match.

The customer must provide answers to all the questions relevant to each matched income stream. If the information does not align with the Income Stream data and the reason is not clear, the information is deemed insufficient.

Encourage the customer to supply the documentation needed:

  • If they have the information but cannot provide it over the phone, documentation must be requested
  • If they cannot get the information at all, a request can be made to the third party

Account Based or Market Linked Income Stream

  • Commencement Date
  • Current gross payment amount ($)
  • Income Frequency
  • Original purchase price
  • Balance as at 1 July each year
  • Gross annual income at 1 July each year (grandfathered products only)
  • Current account balance
  • Commutation Date (if needed)
  • Latest commutation amount: (if needed)

Defined Benefit Income Stream

  • Commencement Date
  • Current gross payment amount ($)
  • Income frequency
  • Gross payment amount for each historic Consumer Price Index (CPI)
  • Undeducted purchase price (if needed)
  • Super child amount (if needed)
  • Amount of any arrears paid ($) (if needed)
  • Date arrears were paid (if needed)
  • Period covered by arrears (if needed)
  • Tax Free component amount (TFC) (if needed)
  • TFC calculation method (if needed)

Lifetime, Life Expectancy or Term income Stream

  • Commencement Date
  • Current gross payment amount ($)
  • Income frequency
  • Original purchase price
  • Balance as at 1 July each year
  • Current account balance
  • Commutation date (if needed)
  • Latest commutation amount (if needed)
  • Residual capital value (if needed)
  • TFC amount (if needed)
  • Indexation method (if needed)
  • Next indexation date (if needed)
  • Indexation % (if required)

Is the verbal update acceptable and aligns with the Income Stream data?

If the discrepancy can be fully explained for all matches, and no third party verification is needed, thoroughly document the explanation in Notes and finalise the intervention. Fully explained means there must be no more action needed as the information has already been assessed or does not need to be assessed. This must be for all matches.

  • Yes, go to Step 4
  • No, or the customer was unable to provide the information over the phone, go to Step 6

Customer providing documented evidence + Read more ...

If the customer cannot give a verbal update, but can give documentary evidence:

  • Tell the customer:

To avoid delays, the customer should phone back to notify 5-6 days after the documents have been lodged. When the customer phones back, confirm receipt and progress the AoIS intervention with the customer over the phone.

Request is not granted + Read more ...

  • Yes, go to Step 5
  • No, go to Step 6

5

Response from third party information request + Read more ...

Once the third party information is returned.

In Process Direct:

  • Under the action column, select Edit for the relevant Product name
  • Contact or attempt contact made: select Yes
  • Contact made with: select Third Party
  • Call type: Select Inbound/Outbound
  • Call Status: Successful
  • Number of Contact: displayed as appropriate
  • Anomaly/Mismatch: Select Deny
  • Extension Needed: Select No
  • Valid information received: Select Yes
  • Information Source: select Third party
  • Notes: make sure any additional notes relevant to the intervention are recorded
  • Select Save
  • Select Submit

If all information has been received, see Assessing and finalising AoIS interventions.

6

Information not returned by third party + Read more ...

Make one phone attempt to contact the third party to find out the correct postal address and to identify any reasons for delay in responding.

  • No, go to Step 8

7

Information not returned by third party + Read more ...

If the third party says they received the request but they need more time, tell them they need to return the information within 14 days. Extend the due date in the workflow.

In Process Direct:

  • Under the action column, select Edit for the relevant Product name
  • Contact or attempt contact made: select Yes
  • Contact made with: select Third Party
  • Call type: Select Outbound
  • Call Status: Select as appropriate Successful
  • Number of Contact: select as appropriate
  • Anomaly/Mismatch: Select Deny
  • Extension Needed: Select Yes
  • Valid information received: Select No
  • Information Source: select third party
  • Notes: make sure any additional notes relevant to the intervention are recorded
  • Select Save
  • Select Submit

If the third party says they did not receive the initial request for information or they need to be sent a new request, issue another request to the third party/parties. Take the following action:

In Process Direct:

  • Under the action column, select Edit for the relevant Product name
  • Contact or attempt contact made: select Yes
  • Contact made with: select Third Party
  • Call type: Select Inbound/Outbound
  • Call Status: select as appropriate Unsuccessful/Successful
  • Number of Contact: displayed as appropriate
  • Valid information received: select No
  • Action: Final Request for Documents
  • From: Third Party
  • Request: displayed as appropriate
  • Notes: make sure any additional notes relevant to the intervention are recorded
  • Select Save
  • Select Submit

8

No response after first request + Read more ...

If a request for information has been issued and not returned and one attempt to contact the third party has been made, all avenues have been exhausted.

Only after exhausting all avenues is a CADO Service Desk enquiry to be sent to the Compliance Helpdesk. Before issuing the request, seek and record Quality Development Officer (QDO) endorsement under the Notes. The Resources page contains a link to the documenting requirements for AoIS interventions.

Forward the following details to the Compliance Helpdesk via the CADO Service Desk online form:

  • CSAQuery: Compliance Helpdesk
  • CSAEnquiry: Compliance Helpdesk
  • Customer Reference Number (CRN)
  • Main Payment Type: UII - AoIS
  • Reference check: no response to QSS40

The Compliance Helpdesk will ask for help from Means Test to get the information needed. The information needed from the provider and the reasons for the referral must be fully documented in the referral to the CADO Service Desk.
